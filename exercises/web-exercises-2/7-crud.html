Section 7-1

CRUD機能実装

主要エンティティの登録・表示・更新・削除機能を Spring Boot + MyBatis で実装します。

CRUDとは

add_circle

Create（登録）

新しいデータをデータベースに追加

visibility

Read（表示）

データベースからデータを取得して表示

edit

Update（更新）

既存データを修正

delete

Delete（削除）

データをデータベースから削除

実装する層

層 役割 クラス名の例
Controller リクエスト受付、レスポンス返却 TaskController
Service ビジネスロジック TaskService
Mapper データベースアクセス TaskMapper
Entity/Form データ構造の定義 Task, TaskForm

Thymeleafでの画面実装

  • 一覧画面 - th:each でループ表示
  • 詳細画面 - th:text で値表示
  • 入力フォーム - th:field でバインド
  • リンク - th:href でURL生成
Section 7-2

バリデーション実装

サーバーサイドの入力検証を実装します。不正なデータの登録を防ぎましょう。

Bean Validationアノテーション

アノテーション 用途
@NotNull null不可
@NotBlank 空文字・空白不可（文字列用）
@Size(min, max) 文字数の範囲指定
@Min / @Max 数値の範囲指定
@Email メールアドレス形式
@Pattern 正規表現でのチェック

エラー処理

  • BindingResult でバリデーション結果を受け取る
  • th:errors でエラーメッセージを表示
  • カスタムエラーメッセージの設定

例外処理

error

グローバル例外ハンドラー

@ControllerAdvice でアプリ全体の例外を処理

web

エラーページ

404、500などのエラーページを用意

Section 7-3

認証・認可実装

Spring Securityによるログイン機能、ロール管理、アクセス制御を実装します。

認証（Authentication）

実装する機能

  • ログイン画面
  • ユーザー名・パスワードによる認証
  • ログアウト機能
  • パスワードのハッシュ化（BCrypt）

認可（Authorization）

ロール管理

ユーザーに役割（ADMIN、USER等）を付与

アクセス制御

ロールに応じたページへのアクセス制限

セキュリティ対策

対策 内容
CSRF対策 フォームにCSRFトークンを自動付与
XSS対策 Thymeleafで自動エスケープ
SQLインジェクション対策 MyBatisのプリペアドステートメント
