BEM命名規則 + セレクタ設計

verified
CSSポイント

BEM（Block-Element-Modifier）はCSS設計手法の1つで、.block__element--modifier の命名規則でセレクタの詳細度を低く保ちながら、わかりやすい構造を実現します。

assignment課題

以下のHTMLに対して、BEM命名規則に従ったCSSを設計してください。

<div class="card">
  <img class="card__image" src="..." alt="">
  <div class="card__body">
    <h3 class="card__title">タイトル</h3>
    <p class="card__text">説明文</p>
    <button class="card__button card__button--primary">詳細を見る</button>
    <button class="card__button card__button--outline">お気に入り</button>
  </div>
</div>

<div class="card card--featured">
  <!-- 同じ構造だが、featuredモディファイアで特別なスタイル -->
</div>
description仕様
  • Block: .card — 基本カードスタイル（白背景、角丸、影）
  • Elements: __image, __body, __title, __text, __button
  • Modifiers: --primary（青背景ボタン）、--outline（枠線ボタン）、--featured（ゴールドボーダーカード）
  • セレクタの詳細度を全て 0-1-0（クラス1つ）に保つ

  • ネストされたセレクタを使わないため、コンポーネントの再配置が容易で、スタイルの予測可能性が高まります。
  • 実務では Tailwind CSS が主流ですが、BEM の概念を理解しておくとCSS設計力の基盤になります。

:has() と :is() を使った高度なセレクタ

verified
CSSポイント

:has() は「子要素の状態で親をスタイリング」できる強力なセレクタです。:is() と :where() はセレクタリストを簡潔に書けます。

assignment課題

以下の要件を :has(), :is(), :where() を使って実装してください。

description仕様
  • 画像を含むカード（.card:has(img)）にだけ padding: 0 を適用
  • :is(h1, h2, h3) でまとめてフォントファミリーと色を指定
  • フォーカス中の input を含むフォームグループ（.form-group:has(input:focus)）にハイライトを追加
  • :where() でリセットスタイルを詳細度 0 で定義

codeベースコード

<!-- HTML -->
<div class="card">
  <img src="https://via.placeholder.com/400x200" alt="">
  <div class="card__body">
    <h3>画像付きカード</h3>
    <p>このカードは画像を含みます</p>
  </div>
</div>

<div class="card">
  <div class="card__body">
    <h3>テキストのみカード</h3>
    <p>このカードに画像はありません</p>
  </div>
</div>

<div class="form-group">
  <label>名前</label>
  <input type="text" placeholder="入力してください">
</div>

<style>
/* TODO: :has() — 画像を含むカードだけ padding を 0 にする */
/* .card:has(???) { ... } */

/* TODO: :is() — h1, h2, h3 をまとめてスタイリング */
/* :is(???) { font-family: ...; color: ...; } */

/* TODO: :has() — フォーカス中の input を持つグループをハイライト */
/* .form-group:has(???) { ... } */

/* TODO: :where() — 詳細度0のリセットスタイル */
/* :where(???) { list-style: none; padding: 0; margin: 0; } */
</style>
lightbulbヒント

:has() は親セレクタとして機能し、子の状態で親のスタイルを変更できます。:is() は最も高い引数の詳細度を取り、:where() は常に詳細度 0 です。

check_circle解答・解説 
/* :has() — 画像を含むカードだけ padding を変更 */
.card:has(img) {
  padding: 0;
}

.card:has(img) .card__body {
  padding: 20px;
}

/* :is() — 見出しをまとめてスタイリング */
:is(h1, h2, h3) {
  font-family: 'Noto Sans JP', sans-serif;
  color: #1d1d1f;
  line-height: 1.3;
}

/* :has() — フォーカス中の input を持つグループ */
.form-group:has(input:focus) {
  background: #f0f7ff;
  border-radius: 8px;
  padding: 12px;
}

/* :where() — 詳細度0のリセット */
:where(ul, ol) {
  list-style: none;
  padding: 0;
  margin: 0;
}
解説
  • :has() はCSSの「親セレクタ」と呼ばれ、子孫の状態に基づいて親要素をスタイリングできます。
  • :is(h1, h2, h3) は h1, h2, h3 と同じですが、長いセレクタをまとめるときに特に有用です。
  • :where() は :is() と同じ機能ですが、詳細度が常に 0 です。リセットCSSやデフォルトスタイルに適しています。
  • :has() は比較的新しいセレクタで、2023年以降の主要ブラウザでサポートされています。

ツールチップ + ポップオーバーの実装

verified
CSSポイント

ツールチップは position: absolute + ::after 擬似要素（三角形）で作成します。visibility と opacity のトランジションで滑らかに表示します。

assignment課題

以下の仕様を組み合わせてツールチップコンポーネントを実装してください。

description仕様
  • ホバーで上部にツールチップを表示（position: absolute; bottom: 100%;）
  • 三角形の矢印: ::after + border テクニック
  • 表示アニメーション: opacity: 0→1 + transform: translateY(4px)→0
  • 背景: #1e293b、文字: 白、角丸: 6px
  • 上・下・左・右の4方向バリエーションを用意

codeベースコード

  <button>ホバーしてね</button>
  <span class="tooltip">ツールチップのテキスト</span>
</div>

<div class="tooltip-wrapper">
  <button>下に表示</button>
  <span class="tooltip tooltip--bottom">下方向ツールチップ</span>
</div>

<style>
.tooltip-wrapper {
  position: relative;
  display: inline-block;
}

  /* TODO: 非表示: opacity: 0; visibility: hidden */
  /* TODO: transition で opacity, transform, visibility をアニメーション */
}

  /* TODO: 三角形の矢印を作成 */
  /* content: ""; position: absolute; top: 100%; left: 50% */
  /* border: 6px solid transparent; border-top-color: #1e293b */
}

  /* TODO: 表示状態にする（opacity: 1, visibility: visible, translateY(0)） */
}

/* TODO: .tooltip--bottom — 下方向バリエーション */
</style>
lightbulbヒント

三角形は border: 6px solid transparent; border-top-color: #1e293b; で作ります。visibility: hidden + opacity: 0 を組み合わせると、フェードアウト後にクリック不可になります。

check_circle解答・解説 
.tooltip-wrapper {
  position: relative;
  display: inline-block;
}

.tooltip {
  position: absolute;
  bottom: calc(100% + 8px);
  left: 50%;
  transform: translateX(-50%) translateY(4px);
  background: #1e293b;
  color: white;
  font-size: 0.8rem;
  padding: 6px 12px;
  border-radius: 6px;
  white-space: nowrap;
  opacity: 0;
  visibility: hidden;
  transition: opacity 0.2s, transform 0.2s, visibility 0.2s;
  pointer-events: none;
}

.tooltip::after {
  content: "";
  position: absolute;
  top: 100%;
  left: 50%;
  transform: translateX(-50%);
  border: 6px solid transparent;
  border-top-color: #1e293b;
}

.tooltip-wrapper:hover .tooltip {
  opacity: 1;
  visibility: visible;
  transform: translateX(-50%) translateY(0);
}

/* 下方向バリエーション */
.tooltip--bottom {
  bottom: auto;
  top: calc(100% + 8px);
}

.tooltip--bottom::after {
  top: auto;
  bottom: 100%;
  border-top-color: transparent;
  border-bottom-color: #1e293b;
}
解説
  • visibility: hidden + opacity: 0 は display: none と異なり、トランジション可能です。
  • pointer-events: none で非表示時にクリックやホバーの妨げにならないようにします。
  • 三角形は4辺の border を transparent にし、1辺だけ色をつけることで作成します。CSS の定番テクニックです。
  • calc(100% + 8px) で親要素の高さ + 余白分を計算し、ツールチップの位置を決めます。

verified
CSSポイント

タイムラインは ::before で縦線を引き、各アイテムに丸いドットを配置するパターンです。position と padding-left の組み合わせで実装します。

assignment課題

以下の仕様を組み合わせてタイムラインUIを実装してください。

description仕様
  • 縦の線: ::before 擬似要素、幅 2px、色 #e5e7eb
  • 各アイテムのドット: 12px の丸、背景 #007AFF、白い枠線
  • アイテムの左に padding-left: 36px で線とドットのスペースを確保

verified
CSSポイント

チャットUIは flex-direction: column + align-self で自分と相手のメッセージを左右に振り分けます。flex-grow でメッセージエリアを伸縮させます。

assignment課題

description仕様
