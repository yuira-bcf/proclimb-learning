T1 HTML基本構造・文書型

穴埋め — HTML5基本テンプレート

verified
学習ポイント

<!DOCTYPE html> はHTML5文書であることを宣言します。<html lang="ja"> で文書の言語を指定し、<meta charset="UTF-8"> で文字エンコーディングを設定します。

assignment課題

以下のHTMLコードの /* ① */ 〜 /* ③ */ を埋めて、正しいHTML5テンプレートを完成させてください。

/* ① */
<html lang="ja">
<head>
  /* ② */
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>はじめてのHTML</title>
</head>
<body>
  /* ③ */
</body>
</html>
visibility期待される結果
ブラウザに「こんにちは、HTML！」と表示される正しいHTML5文書
lightbulbヒント

① HTML5の文書型宣言は <!DOCTYPE html> です。② 文字エンコーディングは <meta charset="UTF-8"> で指定します。③ body内に表示したい内容を <h1> タグなどで書きます。

check_circle解答・解説 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>はじめてのHTML</title>
</head>
<body>
  <h1>こんにちは、HTML！</h1>
</body>
</html>
解説
  • <!DOCTYPE html> はHTML5文書であることをブラウザに伝える宣言です。省略すると互換モード（Quirks Mode）で表示される場合があります。
  • <meta charset="UTF-8"> は <head> 内の最初に記述するのがベストプラクティスです。
  • lang="ja" はスクリーンリーダーの言語判定やSEOに影響します。
D-02
T1 HTML基本構造・文書型

写経+改造 — headセクションの充実

verified
学習ポイント

<head> 内には文書のメタ情報（タイトル、説明文、外部CSSリンクなど）を配置します。<title> はブラウザのタブに表示されます。

assignment課題

Step 1（写経）: 以下のコードを写経して、ブラウザで表示を確認してください。

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>自己紹介ページ</title>
</head>
<body>
  <h1>自己紹介</h1>
  <p>私はWeb開発を学んでいます。</p>
</body>
</html>

Step 2（改造）: <head> 内に以下を追加してください。

  • <meta name="description"> で説明文を追加
  • <meta name="author"> で著者名を追加
  • <link rel="icon" href="favicon.ico"> でファビコンを指定
lightbulbヒント

<meta name="description" content="説明文"> の形式で記述します。content 属性に値を設定するのを忘れないでください。

check_circle解答・解説 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Web開発を学ぶ私の自己紹介ページです">
  <meta name="author" content="山田太郎">
  <link rel="icon" href="favicon.ico">
  <title>自己紹介ページ</title>
</head>
<body>
  <h1>自己紹介</h1>
  <p>私はWeb開発を学んでいます。</p>
</body>
</html>
解説
  • meta description は検索エンジンの結果ページ（SERP）に表示される説明文に使われます。
  • meta author は文書の著者を示すメタ情報です。
  • <link rel="icon"> はブラウザタブに表示されるファビコンを指定します。

T2 テキスト・リスト・リンク

穴埋め — 見出し・段落・強調

verified
学習ポイント

見出しは <h1>〜<h6> で階層構造を表します。<strong> は重要性を、<em> は強調を意味的に示します。

assignment課題

以下のHTMLコードの /* ① */ 〜 /* ④ */ を埋めてください。

</* ① */>Web開発入門</h1>
</* ② */>HTMLとは何か</h2>
<p>HTMLは</* ③ */>Webページの構造</strong>を定義する言語です。
正式名称は</* ④ */>HyperText Markup Language</em>です。</p>
visibility期待される結果
「Web開発入門」が大見出し、「HTMLとは何か」が中見出しで表示され、
「Webページの構造」が太字、「HyperText Markup Language」が斜体で表示される
lightbulbヒント

開始タグと終了タグは同じ要素名を使います。① は h1、② は h2 です。③ 太字は strong、④ 斜体は em です。

check_circle解答・解説 
<h1>Web開発入門</h1>
<h2>HTMLとは何か</h2>
<p>HTMLは<strong>Webページの構造</strong>を定義する言語です。
正式名称は<em>HyperText Markup Language</em>です。</p>
解説
  • 見出しは <h1> が最上位で、数字が大きくなるほどレベルが下がります。1ページに <h1> は原則1つです。
  • <strong> は「重要な内容」、<em> は「強調」を意味します。見た目（太字・斜体）ではなくセマンティクスで使い分けます。
  • <b> や <i> は見た目のみの指定で意味を持ちません。アクセシビリティの観点からは <strong> / <em> が推奨です。
D-04
T2 テキスト・リスト・リンク

穴埋め — リストとリンク

verified
学習ポイント

<ul> は順序なしリスト、<ol> は順序ありリストです。<a href="URL"> でハイパーリンクを作成します。

assignment課題

以下の /* ① */ 〜 /* ④ */ を埋めて、リストとリンクを完成させてください。

<h2>学習予定</h2>
</* ① */>
  <li>HTMLの基礎</li>
  <li>CSSの基礎</li>
  <li>JavaScriptの基礎</li>
</ol>

<h2>参考リンク</h2>
</* ② */>
  <li><a /* ③ */="https://developer.mozilla.org">MDN Web Docs</a></li>
  <li><a href="https://www.w3.org" /* ④ */="_blank">W3C</a></li>
</ul>
lightbulbヒント

① 順序ありリストは ol です。② 順序なしリストは ul です。③ リンク先URLを指定する属性は href です。④ 新しいタブで開く属性は target です。

check_circle解答・解説 
<h2>学習予定</h2>
<ol>
  <li>HTMLの基礎</li>
  <li>CSSの基礎</li>
  <li>JavaScriptの基礎</li>
</ol>

<h2>参考リンク</h2>
<ul>
  <li><a href="https://developer.mozilla.org">MDN Web Docs</a></li>
  <li><a href="https://www.w3.org" target="_blank">W3C</a></li>
</ul>
解説
  • <ol>（ordered list）は番号付きリスト、<ul>（unordered list）は箇条書きリストです。
  • href 属性はリンク先のURLを指定します。target="_blank" は新しいタブで開きます。
  • 外部リンクには rel="noopener noreferrer" を追加するのがセキュリティ上のベストプラクティスです。

穴埋め — 基本テーブルの構造

verified
学習ポイント

<table> の中に <thead>（ヘッダー行）と <tbody>（データ行）を分けて記述します。ヘッダーセルは <th>、データセルは <td> です。

assignment課題

以下の /* ① */ 〜 /* ④ */ を埋めて、商品一覧テーブルを完成させてください。

</* ① */>
  </* ② */>
    <tr>
      </* ③ */>商品名</th>
      <th>価格</th>
      <th>在庫</th>
    </tr>
  </thead>
  </* ④ */>
    <tr>
      <td>ノートPC</td>
      <td>98,000円</td>
      <td>5</td>
    </tr>
    <tr>
      <td>マウス</td>
      <td>2,500円</td>
      <td>20</td>
    </tr>
  </tbody>
</table>
lightbulbヒント

① テーブル全体を囲む要素は table です。② ヘッダー部分は thead、③ ヘッダーセルは th、④ データ部分は tbody です。

check_circle解答・解説 
<table>
  <thead>
    <tr>
      <th>商品名</th>
      <th>価格</th>
      <th>在庫</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>ノートPC</td>
      <td>98,000円</td>
      <td>5</td>
    </tr>
    <tr>
      <td>マウス</td>
      <td>2,500円</td>
      <td>20</td>
    </tr>
  </tbody>
</table>
解説
  • <thead> / <tbody> / <tfoot> でテーブルの構造を明示すると、スクリーンリーダーが正しく解釈できます。
  • <th> は自動的に太字・中央揃えになり、セマンティクスとして「見出しセル」であることを示します。
  • <tr>（table row）が行、<td>（table data）がセルを表します。
D-06
T3 テーブル

写経+改造 — セル結合テーブル

verified
学習ポイント

colspan 属性で横方向のセル結合、rowspan 属性で縦方向のセル結合ができます。結合数に応じて後続のセルを省略する必要があります。

assignment課題

Step 1（写経）: 以下の時間割テーブルを写経して表示を確認してください。

<table border="1">
  <thead>
    <tr>
      <th>時間</th>
      <th>月曜</th>
      <th>火曜</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1限</td>
      <td>HTML</td>
      <td>CSS</td>
    </tr>
    <tr>
      <td>2限</td>
      <td>HTML</td>
      <td>JavaScript</td>
    </tr>
  </tbody>
</table>

Step 2（改造）: 月曜の1限と2限の「HTML」を rowspan="2" で1つのセルに結合してください。

lightbulbヒント

1限の月曜セルに rowspan="2" を追加し、2限の行から月曜のセル（<td>HTML</td>）を削除します。

check_circle解答・解説 
<table border="1">
  <thead>
    <tr>
      <th>時間</th>
      <th>月曜</th>
      <th>火曜</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1限</td>
      <td rowspan="2">HTML</td>
      <td>CSS</td>
    </tr>
    <tr>
      <td>2限</td>
      <td>JavaScript</td>
    </tr>
  </tbody>
</table>
解説
  • rowspan="2" はそのセルが2行分を占有することを示します。結合したセルの分だけ次の行のセルを削除します。
  • colspan は横方向の結合で、同じ行内の後続セルを減らします。
  • 結合を使いすぎるとテーブルの構造が複雑になり、アクセシビリティが低下します。

T4 フォーム・入力要素

穴埋め — 基本フォーム要素

verified
学習ポイント

<form> はフォーム全体を囲みます。<label> と <input> を for/id で紐付けることで、アクセシビリティが向上します。

assignment課題

以下の /* ① */ 〜 /* ④ */ を埋めて、ログインフォームを完成させてください。

</* ① */ action="/login" method="post">
  <div>
    </* ② */ for="username">ユーザー名</label>
    <input type="text" id="username" name="username" /* ③ */>
  </div>
  <div>
    <label for="password">パスワード</label>
    <input type="/* ④ */" id="password" name="password" required>
  </div>
  <button type="submit">ログイン</button>
</form>
lightbulbヒント

① フォーム要素は form です。② ラベルは label です。③ 入力必須にする属性は required です。④ パスワード入力は password タイプです。

check_circle解答・解説 
<form action="/login" method="post">
  <div>
    <label for="username">ユーザー名</label>
    <input type="text" id="username" name="username" required>
  </div>
  <div>
    <label for="password">パスワード</label>
    <input type="password" id="password" name="password" required>
  </div>
  <button type="submit">ログイン</button>
</form>
解説
  • action 属性はフォーム送信先のURL、method はHTTPメソッド（GET/POST）を指定します。
  • <label for="id"> と <input id="id"> の for/id を一致させることで、ラベルクリックで入力欄にフォーカスが移ります。
  • type="password" は入力文字をマスク（●●●）して表示します。
  • required 属性はHTML5のバリデーションで、空欄での送信を防ぎます。
D-08
T4 フォーム・入力要素

写経+改造 — 選択系フォーム要素

verified
学習ポイント

<select> はドロップダウンメニュー、<input type="radio"> はラジオボタン、<input type="checkbox"> はチェックボックスです。

assignment課題

Step 1（写経）: 以下のアンケートフォームを写経してください。

<form>
  <fieldset>
    <legend>受講アンケート</legend>
    <div>
      <label for="course">受講コース</label>
      <select id="course" name="course">
        <option value="">選択してください</option>
        <option value="html">HTML/CSS</option>
        <option value="js">JavaScript</option>
      </select>
    </div>
    <div>
      <p>満足度</p>
      <label><input type="radio" name="rating" value="good"> 良い</label>
      <label><input type="radio" name="rating" value="normal"> 普通</label>
    </div>
  </fieldset>
</form>

Step 2（改造）: 以下を追加してください。

  • <select> に「Java」の選択肢を追加
  • 満足度に「悪い」の選択肢を追加
  • チェックボックスで「メール通知を受け取る」を追加
lightbulbヒント

ドロップダウンには <option value="java">Java</option> を追加します。チェックボックスは <input type="checkbox"> です。

check_circle解答・解説 
<form>
  <fieldset>
    <legend>受講アンケート</legend>
    <div>
      <label for="course">受講コース</label>
      <select id="course" name="course">
        <option value="">選択してください</option>
        <option value="html">HTML/CSS</option>
        <option value="js">JavaScript</option>
        <option value="java">Java</option>
      </select>
    </div>
    <div>
      <p>満足度</p>
      <label><input type="radio" name="rating" value="good"> 良い</label>
      <label><input type="radio" name="rating" value="normal"> 普通</label>
      <label><input type="radio" name="rating" value="bad"> 悪い</label>
    </div>
    <div>
      <label>
        <input type="checkbox" name="notify" value="email">
        メール通知を受け取る
      </label>
    </div>
  </fieldset>
</form>
解説
  • <fieldset> はフォーム要素をグループ化し、<legend> がそのグループのタイトルになります。
  • ラジオボタンは同じ name 属性を持つグループの中から1つだけ選択できます。
  • チェックボックスは複数選択が可能です。name が同じチェックボックスは配列として送信されます。

T5 メディア・埋め込み

穴埋め — 画像の埋め込み

verified
学習ポイント

<img> は空要素（閉じタグなし）です。src で画像パス、alt で代替テキストを必ず指定します。alt はアクセシビリティとSEOに不可欠です。

assignment課題

以下の /* ① */ 〜 /* ③ */ を埋めて、画像を表示させてください。

<figure>
  </* ① */ /* ② */="images/landscape.jpg" /* ③ */="美しい山の風景写真"
    width="600" height="400">
  <figcaption>2024年夏に撮影した山の写真</figcaption>
</figure>
lightbulbヒント

① 画像要素は img です。② 画像パスの属性は src です。③ 代替テキストの属性は alt です。

check_circle解答・解説 
<figure>
  <img src="images/landscape.jpg" alt="美しい山の風景写真"
    width="600" height="400">
  <figcaption>2024年夏に撮影した山の写真</figcaption>
</figure>
解説
  • alt 属性は画像が表示できない場合の代替テキストです。スクリーンリーダーはこのテキストを読み上げます。
  • width / height を指定するとCLS（Cumulative Layout Shift）を防ぎ、パフォーマンスが向上します。
  • <figure> と <figcaption> のセットで「図とキャプション」をセマンティックにマークアップできます。
D-10
T5 メディア・埋め込み

写経+改造 — 動画とオーディオ

verified
学習ポイント

<video> と <audio> はHTML5のメディア要素です。controls 属性で再生コントロールを表示します。

assignment課題

Step 1（写経）: 以下のビデオプレイヤーを写経してください。

<video controls width="640">
  <source src="video/sample.mp4" type="video/mp4">
  お使いのブラウザは動画再生に対応していません。
</video>

Step 2（改造）: 以下の属性・要素を追加してください。

  • poster="images/thumbnail.jpg" でサムネイル画像を指定
  • WebM形式のフォールバック <source> を追加
  • 別途 <audio> 要素でBGMプレイヤーを追加
lightbulbヒント

poster 属性は <video> の属性として追加します。<source> は複数記述でき、ブラウザは上から順に再生可能な形式を選びます。

check_circle解答・解説 
<video controls width="640" poster="images/thumbnail.jpg">
  <source src="video/sample.mp4" type="video/mp4">
  <source src="video/sample.webm" type="video/webm">
  お使いのブラウザは動画再生に対応していません。
</video>

<audio controls>
  <source src="audio/bgm.mp3" type="audio/mpeg">
  <source src="audio/bgm.ogg" type="audio/ogg">
  お使いのブラウザはオーディオ再生に対応していません。
</audio>
解説
  • poster 属性は動画再生前に表示されるサムネイル画像を指定します。
  • <source> を複数記述することで、ブラウザの対応形式に応じたフォールバックが可能です。
  • <video> / <audio> の閉じタグの前のテキストは、要素に非対応のブラウザで表示されるフォールバックテキストです。

T6 セマンティクス・アクセシビリティ

穴埋め — セマンティック要素の選択

verified
学習ポイント

HTML5のセマンティック要素（<header>, <nav>, <main>, <article>, <footer>）は文書の構造を明示し、アクセシビリティとSEOに貢献します。

assignment課題

以下の /* ① */ 〜 /* ⑤ */ を埋めて、セマンティックなページ構造を完成させてください。

</* ① */>
  <h1>サイトタイトル</h1>
  </* ② */>
    <ul>
      <li><a href="/">ホーム</a></li>
      <li><a href="/about">概要</a></li>
    </ul>
  </nav>
</header>

</* ③ */>
  </* ④ */>
    <h2>記事タイトル</h2>
    <p>記事の本文...</p>
  </article>
</main>

</* ⑤ */>
  <p>&copy; 2026 サイト名</p>
</footer>
lightbulbヒント

① サイトヘッダーは header、② ナビゲーションは nav、③ メインコンテンツは main、④ 独立した記事は article、⑤ フッターは footer です。

check_circle解答・解説 
<header>
  <h1>サイトタイトル</h1>
  <nav>
    <ul>
      <li><a href="/">ホーム</a></li>
      <li><a href="/about">概要</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>記事タイトル</h2>
    <p>記事の本文...</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 サイト名</p>
</footer>
解説
  • <header> はページやセクションの導入部、<footer> は末尾の情報を囲みます。
  • <nav> は主要なナビゲーションリンクのグループを示します。
  • <main> はページのメインコンテンツで、1ページに1つだけ使用します。
  • <article> は独立して配信・再利用可能なコンテンツ（ブログ記事、ニュースなど）を表します。

写経+改造 — WAI-ARIAの基礎

verified
学習ポイント

aria-label はスクリーンリーダー向けのラベルを提供します。role 属性はHTML要素のセマンティクスを補足・上書きできます。

assignment課題

Step 1（写経）: 以下のアクセシブルなナビゲーションを写経してください。

<nav aria-label="メインナビゲーション">
  <ul>
    <li><a href="/" aria-current="page">ホーム</a></li>
    <li><a href="/products">商品一覧</a></li>
    <li><a href="/contact">お問い合わせ</a></li>
  </ul>
</nav>

Step 2（改造）: 以下を追加してください。

  • パンくずリスト用の <nav aria-label="パンくず"> を追加
  • パンくず内の各リンクの間に aria-hidden="true" の区切り文字「>」を配置
lightbulbヒント

ページに複数の <nav> がある場合、aria-label で区別します。aria-hidden="true" はスクリーンリーダーに読み上げさせない要素に使います。

check_circle解答・解説 
<nav aria-label="メインナビゲーション">
  <ul>
    <li><a href="/" aria-current="page">ホーム</a></li>
    <li><a href="/products">商品一覧</a></li>
    <li><a href="/contact">お問い合わせ</a></li>
  </ul>
</nav>

<nav aria-label="パンくず">
  <ol>
    <li><a href="/">ホーム</a> <span aria-hidden="true">&gt;</span></li>
    <li><a href="/products">商品一覧</a> <span aria-hidden="true">&gt;</span></li>
    <li>商品詳細</li>
  </ol>
</nav>
解説
  • aria-label は要素に「目に見えないラベル」を付与します。複数の <nav> を区別するのに必須です。
  • aria-current="page" は「現在表示中のページ」を示し、スクリーンリーダーに伝わります。
  • aria-hidden="true" は装飾的な要素をスクリーンリーダーから隠します。

T7 メタデータ・SEO・OGP

穴埋め — 基本メタタグ

verified
学習ポイント

SEOに重要なメタタグは <title>（検索結果のタイトル）と <meta name="description">（検索結果の説明文）です。viewport メタタグはレスポンシブデザインに必須です。

assignment課題

以下の /* ① */ 〜 /* ④ */ を埋めて、SEO対策されたheadセクションを完成させてください。

<head>
  <meta charset="UTF-8">
  <meta name="/* ① */" content="width=device-width, initial-scale=1.0">
  </* ② */>東京カフェガイド｜おすすめカフェ情報</title>
  <meta name="/* ③ */" content="東京都内のおすすめカフェを厳選して紹介。Wi-Fi完備、電源ありの作業向けカフェ情報。">
  <link rel="/* ④ */" href="https://example.com/cafe-guide">
</head>
lightbulbヒント

① レスポンシブ用メタタグは viewport です。② タイトルタグは title です。③ 説明文は description です。④ 正規URLを示すリンクは canonical です。

check_circle解答・解説 
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>東京カフェガイド｜おすすめカフェ情報</title>
  <meta name="description" content="東京都内のおすすめカフェを厳選して紹介。Wi-Fi完備、電源ありの作業向けカフェ情報。">
  <link rel="canonical" href="https://example.com/cafe-guide">
</head>
解説
  • viewport メタタグはモバイルデバイスでの表示を制御します。これがないとスマホで小さく表示されます。
  • <title> はSEOで最も重要な要素の1つです。30〜60文字程度が推奨されます。
  • meta description は120〜160文字程度が推奨されます。検索結果のスニペットに使われます。
  • link rel="canonical" は重複ページがある場合に正規URLを示し、SEO評価の分散を防ぎます。

T7 メタデータ・SEO・OGP

写経+改造 — OGPタグの設定

verified
学習ポイント

OGP（Open Graph Protocol）タグはSNSでページをシェアした時の表示内容を制御します。og:title, og:description, og:image が基本3点セットです。

assignment課題

Step 1（写経）: 以下のOGPタグを写経してください。

<head>
  <meta charset="UTF-8">
  <title>サンプルブログ</title>
  <meta property="og:title" content="サンプルブログ">
  <meta property="og:description" content="Web開発について発信するブログです">
  <meta property="og:image" content="https://example.com/ogp.jpg">
  <meta property="og:url" content="https://example.com">
  <meta property="og:type" content="website">
</head>

Step 2（改造）: Twitter Card用のメタタグを追加してください。

  • twitter:card を summary_large_image に設定
  • twitter:site にアカウント名 @example を設定
lightbulbヒント

Twitter Card は <meta name="twitter:card" content="..."> の形式です。OGP は property 属性、Twitter Card は name 属性を使う点に注意してください。

check_circle解答・解説 
<head>
  <meta charset="UTF-8">
  <title>サンプルブログ</title>
  <meta property="og:title" content="サンプルブログ">
  <meta property="og:description" content="Web開発について発信するブログです">
  <meta property="og:image" content="https://example.com/ogp.jpg">
  <meta property="og:url" content="https://example.com">
  <meta property="og:type" content="website">
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:site" content="@example">
</head>
解説
  • OGP タグは property 属性を使い、Twitter Card は name 属性を使います。
  • twitter:card の値は summary（小さい画像）か summary_large_image（大きい画像）が一般的です。
  • OGP画像の推奨サイズは 1200x630px です。

T8 総合マークアップ

穴埋め — プロフィールカード

verified
学習ポイント

複数のHTML要素を組み合わせてコンポーネントを構築します。<div> はグルーピング用の汎用要素ですが、適切なセマンティック要素がある場合はそちらを優先します。

assignment課題

以下の /* ① */ 〜 /* ⑤ */ を埋めて、プロフィールカードを完成させてください。

<div class="profile-card">
  </* ① */ src="avatar.jpg" /* ② */="山田太郎のプロフィール写真" width="120" height="120">
  </* ③ */>山田太郎</h2>
  <p>フロントエンドエンジニア</p>
  </* ④ */>
    <li></* ⑤ */ href="https://twitter.com/example" target="_blank" rel="noopener noreferrer">Twitter</a></li>
    <li><a href="https://github.com/example" target="_blank" rel="noopener noreferrer">GitHub</a></li>
  </ul>
</div>
lightbulbヒント

① 画像要素は img、② 代替テキストは alt、③ 名前は見出し h2、④ リストは ul、⑤ リンクは a です。

check_circle解答・解説 
<div class="profile-card">
  <img src="avatar.jpg" alt="山田太郎のプロフィール写真" width="120" height="120">
  <h2>山田太郎</h2>
  <p>フロントエンドエンジニア</p>
  <ul>
    <li><a href="https://twitter.com/example" target="_blank" rel="noopener noreferrer">Twitter</a></li>
    <li><a href="https://github.com/example" target="_blank" rel="noopener noreferrer">GitHub</a></li>
  </ul>
</div>
解説
  • プロフィール画像の alt には「誰の」「何の写真か」を記述します。装飾画像なら alt=""（空文字）にします。
  • target="_blank" で新規タブを開く場合、rel="noopener noreferrer" をセキュリティのために追加します。
  • このカードは <div> で囲んでいますが、独立したコンテンツとして扱う場合は <article> も適切です。

T8 総合マークアップ

写経+改造 — ニュース記事ページ

verified
学習ポイント

実際のWebページは複数のセマンティック要素、テキスト要素、リスト、リンクなどを組み合わせて構築されます。<time> 要素で日時をマークアップするとSEOに有利です。

assignment課題

Step 1（写経）: 以下のニュース記事ページを写経してください。

<article>
  <header>
    <h1>HTML5の新機能が発表</h1>
    <p>投稿日: <time datetime="2026-03-01">2026年3月1日</time></p>
    <p>カテゴリ: <a href="/category/web">Web技術</a></p>
  </header>
  <p>HTML5の最新アップデートでは、以下の新機能が追加されました。</p>
  <ul>
    <li>新しいセマンティック要素</li>
    <li>フォームバリデーションの強化</li>
  </ul>
  <footer>
    <p>著者: <a href="/author/tanaka">田中一郎</a></p>
  </footer>
</article>

Step 2（改造）: 以下を追加してください。

  • 記事内に画像（<figure> + <figcaption>）を追加
  • 「関連記事」セクション（<aside>）を追加し、リンク2件を配置
  • 記事フッターに「更新日」を <time> 要素で追加
lightbulbヒント

<aside> は主要コンテンツに関連するがメインではない補足情報に使います。<time datetime="YYYY-MM-DD"> の datetime 属性は機械可読な日付形式です。

check_circle解答・解説 
<article>
  <header>
    <h1>HTML5の新機能が発表</h1>
    <p>投稿日: <time datetime="2026-03-01">2026年3月1日</time></p>
    <p>カテゴリ: <a href="/category/web">Web技術</a></p>
  </header>

  <figure>
    <img src="images/html5-update.jpg" alt="HTML5の新機能を示す図解" width="800" height="450">
    <figcaption>HTML5の主要な新機能の一覧</figcaption>
  </figure>

  <p>HTML5の最新アップデートでは、以下の新機能が追加されました。</p>
  <ul>
    <li>新しいセマンティック要素</li>
    <li>フォームバリデーションの強化</li>
  </ul>

  <aside>
    <h2>関連記事</h2>
    <ul>
      <li><a href="/articles/css-grid">CSS Gridレイアウト入門</a></li>
      <li><a href="/articles/js-es2026">JavaScript ES2026の新機能</a></li>
    </ul>
  </aside>

  <footer>
    <p>著者: <a href="/author/tanaka">田中一郎</a></p>
    <p>更新日: <time datetime="2026-03-05">2026年3月5日</time></p>
  </footer>
</article>
解説
  • <figure> + <figcaption> は画像と説明文をセマンティックに関連付けます。
  • <aside> は記事の補足情報（関連記事、サイドバーなど）に使います。
  • <time datetime="..."> の datetime 属性はISO 8601形式で記述します。検索エンジンはこの情報を利用します。
  • <article> 内の <header> / <footer> は記事固有のヘッダー/フッターで、ページ全体のものとは別です。
