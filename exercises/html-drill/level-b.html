マルチページサイトのテンプレート設計

verified
学習ポイント

マルチページサイトでは共通のヘッダー・フッター構造と一貫した <head> メタ情報が重要です。相対パスを正しく設定する必要があります。

assignment課題

以下の仕様で3ページ構成のサイトの共通テンプレートを設計してください。

  • 共通ヘッダー：ロゴ、ナビ（ホーム/サービス/お問い合わせ）、現在ページに aria-current="page"
  • 共通フッター：会社情報、コピーライト
  • 各ページ固有の <title>、meta description、OGPタグ
  • 共通CSS（style.css）と共通JS（script.js）の読み込み
  • ディレクトリ構成：index.html、services/index.html、contact/index.html
lightbulbヒント

サブディレクトリのページから共通CSSを読み込む場合、パスは ../style.css になります。ナビの現在ページリンクに aria-current="page" を付けてください。

check_circle解答・解説

index.html（トップページ）:

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ホーム | ABC Corporation</title>
  <meta name="description" content="ABC Corporationの公式サイトです。">
  <meta property="og:title" content="ABC Corporation">
  <meta property="og:url" content="https://example.com/">
  <link rel="stylesheet" href="style.css">
  <script src="script.js" defer></script>
</head>
<body>
  <header>
    <a href="/"><img src="logo.png" alt="ABC Corporation"></a>
    <nav>
      <ul>
        <li><a href="/" aria-current="page">ホーム</a></li>
        <li><a href="/services/">サービス</a></li>
        <li><a href="/contact/">お問い合わせ</a></li>
      </ul>
    </nav>
  </header>
  <main>
    <h1>ABC Corporationへようこそ</h1>
  </main>
  <footer>
    <p>ABC Corporation | 東京都千代田区</p>
    <p>&copy; 2026 ABC Corporation</p>
  </footer>
</body>
</html>

services/index.html（サブページ）:

<!-- head内のパスが ../style.css に変わる -->
<link rel="stylesheet" href="../style.css">
<script src="../script.js" defer></script>
<!-- ナビのaria-currentがサービスに移動 -->
<li><a href="/">ホーム</a></li>
<li><a href="/services/" aria-current="page">サービス</a></li>
解説
  • 相対パスはファイルの位置に応じて変わります。../ は1つ上のディレクトリです。
  • aria-current="page" はスクリーンリーダーに「現在のページ」を伝えます。CSSでスタイリングにも使えます。
  • OGPの og:url は各ページごとに正しいURLを設定します。

パフォーマンス最適化されたhead設計

verified
学習ポイント

<head> 内のリソース読み込み順序はパフォーマンスに大きく影響します。preconnect, preload, defer, async を適切に使い分けることが重要です。

assignment課題

以下のリソースを最適な順序・方法で読み込む <head> を設計してください。

  • Google Fonts（Noto Sans JP）— preconnect + link
  • 重要なCSS（critical.css）— 通常読み込み
  • 追加CSS（style.css）— preload で非同期読み込み
  • アナリティクス（analytics.js）— async で読み込み
  • アプリJS（app.js）— defer で読み込み
  • ヒーロー画像（hero.webp）— preload で事前読み込み
lightbulbヒント

async はダウンロード完了後すぐに実行、defer はDOM解析完了後に実行です。CSSの preload は <link rel="preload" href="..." as="style" onload="this.rel='stylesheet'"> で実現します。

check_circle解答・解説 
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>パフォーマンス最適化ページ</title>

  <!-- 1. preconnect（外部ドメインへの事前接続） -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

  <!-- 2. preload（重要リソースの事前読み込み） -->
  <link rel="preload" href="hero.webp" as="image">
  <link rel="preload" href="style.css" as="style"
    onload="this.onload=null;this.rel='stylesheet'">
  <noscript><link rel="stylesheet" href="style.css"></noscript>

  <!-- 3. 重要CSS（レンダリングブロック） -->
  <link rel="stylesheet" href="critical.css">

  <!-- 4. 外部フォント -->
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap" rel="stylesheet">

  <!-- 5. JavaScript -->
  <script src="analytics.js" async></script>
  <script src="app.js" defer></script>
</head>
解説
  • preconnect は DNS解決・TCP接続・TLSネゴシエーションを事前に行い、後続のリソース読み込みを高速化します。
  • preload はリソースを優先的にダウンロードします。CSSの場合は onload で適用をトリガーします。
  • async はスクリプトの実行順序を保証しません。独立したスクリプト（アナリティクス等）に適しています。
  • defer はDOM解析後に実行順序を保証します。DOM操作が必要なスクリプトに適しています。

技術ドキュメントページ

verified
学習ポイント

技術ドキュメントでは <code>, <pre>, <kbd>, <samp>, <var> など専用のテキスト要素を適切に使い分けます。

assignment課題

以下の仕様で技術ドキュメントページを作成してください。

  • ページ内目次（3セクション分のアンカーリンク）
  • 「インストール」セクション：コマンド例を <pre><code> で表示、キー操作を <kbd> で表示
  • 「APIリファレンス」セクション：関数名を <code>、変数を <var>、出力例を <samp> で表示
  • 「FAQ」セクション：<details> + <summary> でアコーディオンUI
  • 注意書きを <blockquote> + <cite> で表示
lightbulbヒント

<kbd> はキーボード入力（例: <kbd>Ctrl</kbd>+<kbd>C</kbd>）、<samp> はプログラムの出力結果、<var> は変数を表します。

check_circle解答・解説

  <nav aria-label="目次">
    <ol>
      <li><a href="#install">インストール</a></li>
      <li><a href="#api">APIリファレンス</a></li>
      <li><a href="#faq">FAQ</a></li>

  <section id="install">

  </section>

  <section id="api">

    <p>実行結果:</p>
    <p><samp>MyLib v2.0 initialized successfully.</samp></p>

    <blockquote>
      <p>初期化は必ずDOMContentLoadedイベント後に実行してください。</p>

  </section>

      <p>Chrome, Firefox, Safari, Edgeの最新2バージョンに対応しています。</p>
    </details>
  </section>
</main>
解説
  • <code> はインラインのコード断片、<pre><code> はコードブロックに使います。
  • <kbd> はキーボード入力を意味し、<samp> はプログラム出力を意味します。
  • <var> は数学的変数やプログラムの変数を表します。デフォルトでイタリック表示されます。
  • <blockquote> は長い引用、<cite> は出典を示します。

学習ポイント

複雑なテーブルでは colspan / rowspan の組み合わせに加え、colgroup と col でスタイリング対象の列グループを定義できます。

assignment課題

以下の仕様で料金比較テーブルを作成してください。

アクセシブルなデータテーブル

verified
学習ポイント

複雑なテーブルのアクセシビリティには headers 属性と id の組み合わせが効果的です。aria-describedby でテーブルの説明文を関連付けることもできます。

assignment課題

以下の仕様で売上データテーブルを作成してください。

  • 2段階ヘッダー：上段に四半期名（colspan="3"）、下段に月名
  • 行ヘッダー：商品カテゴリ名
  • 各ヘッダーに id を付与し、データセルに headers 属性で関連付け
  • aria-describedby でテーブルの補足説明を関連付け
  • 合計行と合計列を含める
lightbulbヒント

headers 属性にはスペース区切りで複数の id を指定できます。例: headers="q1 jan electronics" は「Q1」「1月」「家電」を関連付けます。

マルチステップフォーム

verified
学習ポイント

複雑なフォームは <fieldset> で論理グループに分け、進行状況を <progress> で可視化します。<datalist> で入力候補を提供できます。

assignment課題

以下の仕様でイベント申込フォーム（マルチステップ風）を作成してください。

  • プログレスバー（<progress>）で進行状況を表示
  • Step1「個人情報」：氏名、メール、電話番号
  • Step2「イベント選択」：参加イベント（<datalist> 付き入力）、参加人数（number、1〜10）、希望日（date、最小日付設定）
  • Step3「オプション」：食事プラン（ラジオボタン3択）、アレルギー（textarea）、写真掲載許可（checkbox）
  • <output> 要素で合計金額欄を設置
lightbulbヒント

<datalist id="events"> を <input list="events"> で関連付けます。<progress value="1" max="3"> で進行度を表現します。<output> は計算結果の表示用要素です。

構造化データ付きレシピページ

verified
学習ポイント

JSON-LD形式の構造化データ（Schema.org）を埋め込むと、検索結果にリッチスニペット（レシピカード、評価、調理時間など）が表示されます。

assignment課題

以下の仕様で構造化データ付きのレシピページを作成してください。

  • JSON-LD形式（<script type="application/ld+json">）で Recipe スキーマを記述
  • レシピ名、説明、画像URL、調理時間（prepTime, cookTime）、カロリー
  • 材料リスト（recipeIngredient）と手順（recipeInstructions）
  • 評価（aggregateRating）: 平均4.5、レビュー数28
  • HTML本文も構造化データと対応する内容で作成
visibility 表示結果
レシピページが表示される。Google検索ではリッチスニペットとして、レシピカード（画像、調理時間、カロリー、評価星）が表示される。ページ本文には材料一覧と手順が分かりやすく表示。
lightbulbヒント

JSON-LDは <head> または <body> 内に配置可能です。@context に "https://schema.org"、@type に "Recipe" を指定します。時間は ISO 8601 形式（例: PT30M）です。
