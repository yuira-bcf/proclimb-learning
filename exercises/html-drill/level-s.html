コーポレートサイト完全構築

verified
学習ポイント

本番品質のコーポレートサイトは、SEO、アクセシビリティ、パフォーマンス、セキュリティの全観点を統合して構築します。構造化データやCSP（Content Security Policy）も含みます。

assignment課題

以下の仕様で5ページ構成のコーポレートサイトのHTMLを設計してください（トップページのみ完全実装、他ページは構造のみ）。

  • 共通: DOCTYPE, charset, viewport, OGP, canonical, JSON-LD (Organization), favicon一式
  • トップページ: ヒーロー、サービス3件、実績数値、お知らせ一覧、CTA、フッター
  • サービスページ: サービス詳細3件、料金比較テーブル、FAQ（<details>）
  • 会社概要ページ: 代表挨拶、企業情報テーブル、アクセスマップ（<iframe>）
  • お知らせ一覧: 記事カード一覧（<article>）、ページネーション
  • お問い合わせ: フォーム（バリデーション付き）、プライバシーポリシー同意
  • 全ページ: スキップリンク、パンくず、構造化データ、aria-* 属性
lightbulbヒント

Organization の JSON-LD は @type: "Organization" で会社名、URL、ロゴ、連絡先を記述します。ページネーションは <nav aria-label="ページネーション"> で囲みます。

国際化対応サイト設計

verified
学習ポイント

国際化対応サイトでは hreflang で言語別URLを宣言し、dir 属性でRTL（右から左）言語に対応します。lang 属性を要素レベルで切り替えます。

assignment課題

以下の仕様で日英バイリンガルサイトの日本語版トップページを設計してください。

  • <link rel="alternate" hreflang="..."> で日本語版・英語版・デフォルト版のURLを宣言
  • 言語切替リンクに hreflang 属性と lang 属性を設定
  • 英語のテキストブロックに lang="en" を設定
  • OGPの og:locale と og:locale:alternate を設定
  • <html> の dir="ltr" を明示
lightbulbヒント

hreflang="x-default" はどの言語にもマッチしない場合のデフォルトURLです。og:locale は "ja_JP" のようにロケールを指定します。

check_circle解答・解説 
<!DOCTYPE html>
<html lang="ja" dir="ltr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>テックソリューション | ITで未来を創る</title>

  <link rel="alternate" hreflang="ja" href="https://example.com/ja/">
  <link rel="alternate" hreflang="en" href="https://example.com/en/">
  <link rel="alternate" hreflang="x-default" href="https://example.com/">

  <meta property="og:locale" content="ja_JP">
  <meta property="og:locale:alternate" content="en_US">
</head>
<body>
  <header>
    <nav>
      <ul>
        <li><a href="/ja/" aria-current="page">日本語</a></li>
        <li><a href="/en/" hreflang="en" lang="en">English</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <h1>ITで未来を創る</h1>
    <p>最新のテクノロジーでビジネスの課題を解決します。</p>

    <section lang="en">
      <h2>Our Mission</h2>
      <p>We create the future with cutting-edge technology.</p>
    </section>
  </main>
</body>
</html>
解説
  • hreflang タグは検索エンジンに言語バリエーションを伝え、各地域で適切なページを表示させます。
  • x-default はどの言語にもマッチしない場合のフォールバックURLです。
  • dir="ltr" はテキストの方向（左から右）を明示します。アラビア語等のRTL言語は dir="rtl" にします。
  • 言語切替リンクの lang 属性はリンクテキスト自体の言語を、hreflang はリンク先の言語を示します。

技術ブログ記事の完全マークアップ

verified
学習ポイント

技術ブログは見出し階層、コードブロック、画像、引用、リンク、リスト等の多様な要素を統合的に使用します。Article の JSON-LD 構造化データも重要です。

assignment課題

「HTML5セマンティクス完全ガイド」という技術ブログ記事を完全にマークアップしてください。

  • 記事メタ情報: 著者、公開日、更新日、カテゴリタグ、読了時間
  • 目次（ページ内アンカー、3セクション以上）
  • 各セクション: 見出し、段落、コードブロック（<pre><code>）、図表
  • 引用（<blockquote cite="...">）、略語（<abbr>）、キーボード操作（<kbd>）
  • 関連記事リスト、著者プロフィールカード、コメントセクション（<section aria-label="コメント">）
  • JSON-LD で Article スキーマを記述
lightbulbヒント

Article スキーマは @type: "Article"、headline、datePublished、author、image が必須項目です。著者プロフィールは <aside> か <footer> 内に配置します。

ヘルプセンター・ナレッジベース

verified
学習ポイント

ナレッジベースは検索、カテゴリ分類、ネストしたリスト、アコーディオン、パンくずリストなどを複合的に組み合わせた大規模コンテンツです。

assignment課題

以下の仕様でヘルプセンターのHTMLを完全構築してください。

  • 検索フォーム（role="search"）+ 検索結果件数（aria-live）
  • カテゴリ一覧（3カテゴリ、各5記事のネストリスト）
  • FAQ セクション（<details> で10問以上、JSON-LD FAQPage付き）
  • パンくずリスト（BreadcrumbList構造化データ付き）
  • 「役に立ちましたか？」のフィードバック UI（role="group"）
  • 関連記事のサジェスト（<aside>）
lightbulbヒント

role="search" は検索フォームのランドマークです。フィードバック UI は <fieldset> + role="group" + aria-label でグループ化します。

管理画面のダッシュボードテーブル

verified
学習ポイント

管理画面のテーブルは、ソート、フィルター、一括操作、ページネーション、レスポンシブ対応を全て含む複合的なUIコンポーネントです。

assignment課題

以下の仕様で商品管理ダッシュボードの完全なHTMLを構築してください。

  • フィルター: 検索入力、カテゴリ選択、ステータス選択
  • テーブル: チェックボックス列、商品名（ソート可能）、カテゴリ、価格（ソート可能）、在庫、ステータスバッジ、操作ボタン
  • 一括操作: 全選択チェックボックス、「削除」「非公開」の一括操作ドロップダウン
  • ページネーション: 件数表示 + ページリンク（前へ/1/2/3/次へ）
  • 全要素に適切な aria-* 属性
lightbulbヒント

全選択チェックボックスは aria-label="全件選択"、各行のチェックボックスは aria-label="商品名を選択" のように設定します。ステータスバッジは <span> でマークアップし、色だけでなくテキストでも状態を伝えます。

動画学習プラットフォームのメディアプレーヤーページ

verified
学習ポイント

動画プレーヤーでは複数ソース形式の提供、字幕・チャプターの <track> 要素、レスポンシブなポスター画像の <picture>、そしてプログレッシブエンハンスメントによるフォールバック戦略が重要です。

assignment課題

以下の仕様で動画学習プラットフォームのレッスン再生ページを構築してください。

  • <video> に MP4 / WebM の複数 <source> を設定
  • 字幕トラック（日本語・英語）とチャプタートラックを <track> で提供
  • ポスター画像を <picture> + <source>（WebP / JPEG、レスポンシブ media 属性）で実装
  • 動画の下に <figure> / <figcaption> でレッスン説明を配置
  • 動画非対応ブラウザ向けに <object> → <embed> → ダウンロードリンクの3段フォールバック
  • レッスン一覧サイドバーに <audio> 要素でポッドキャスト版を提供
lightbulbヒント

<track kind="subtitles"> は字幕、kind="chapters" はチャプターマーカーです。default 属性で初期表示する字幕を指定できます。<picture> の <source> は上から順に評価され、最初に一致したものが使用されます。

SPA（シングルページアプリケーション）では、タブパネル、モーダルダイアログ、アラートダイアログ、アコーディオン、ライブリージョンなど、動的UIコンポーネントのアクセシビリティを WAI-ARIA で正しく実装する必要があります。キーボードナビゲーションとフォーカス管理が特に重要です。

assignment課題

  • タブパネル（role="tablist"/"tab"/"tabpanel"）: 「概要」「タスク」「メンバー」の3タブ、aria-selected / aria-controls を設定

verified
学習ポイント

多言語ECサイトでは hreflang による言語切替指示、canonical による重複排除、JSON-LD 構造化データ（Product / BreadcrumbList / Organization）、OGP / Twitter Card の完全設定、パフォーマンス最適化メタ（preload / preconnect / dns-prefetch）を統合的に実装します。

assignment課題

ニュース記事ページでは JSON-LD の NewsArticle / FAQPage で検索結果のリッチスニペットを最大化し、RSS 自動検出、robots メタ、パフォーマンスメタを組み合わせてクローラビリティとユーザー体験を両立します。

assignment課題

以下の仕様でニュースメディアサイトの記事ページを構築してください。

  • JSON-LD NewsArticle: headline / datePublished / dateModified / author / publisher / image / mainEntityOfPage
  • JSON-LD FAQPage: 記事末尾のよくある質問（3問）を構造化
  • RSS フィード自動検出: <link rel="alternate" type="application/rss+xml">
  • robots メタ: index, follow, max-snippet:-1, max-image-preview:large
  • <link rel="sitemap"> でサイトマップ参照
  • 記事本文を <article> + <time> + <address> でセマンティックにマークアップ

SaaS管理画面ダッシュボード

verified
学習ポイント

SaaS管理画面はナビゲーション、データテーブル、フォーム、ダイアログ、グラフ、通知、パンくず、タブなどあらゆるHTML要素を統合します。セマンティクスとアクセシビリティを完全に維持しながら複雑なUIを構築する総合力が問われます。

assignment課題

以下の仕様でSaaSアナリティクスツールの管理画面HTMLを構築してください。

  • サイドバーナビ（<nav aria-label="サイドバー">）: アコーディオン式メニュー（aria-expanded）、現在地表示（aria-current）
  • パンくずリスト（<nav aria-label="パンくず">）+ ページタイトル
  • KPIカード4枚: <data> 要素で数値表示、<meter> で達成率
  • データテーブル: ソート可能（aria-sort）、ページネーション、行選択チェックボックス、一括操作ドロップダウン
  • フィルターフォーム: 日付範囲（date）、ステータスセレクト、検索テキスト、<output> で絞り込み結果件数
  • 通知バッジ（aria-label で件数を読み上げ）とトースト通知（aria-live）
  • 設定変更ダイアログ（role="dialog"）
lightbulbヒント

テーブルのソート状態は aria-sort="ascending" / "descending" / "none" で示します。行選択チェックボックスには aria-label="行 N を選択" を設定し、全選択チェックボックスには aria-label="全行を選択" を使います。<meter> には min / max / low / high / optimum を設定できます。

医療予約システムはマルチステップフォーム、カレンダーテーブル、ドクター一覧カード、アクセシビリティ、セマンティクス、メタデータのすべてを統合する最高難度の総合課題です。実際の業務システムに近い複雑さでHTMLの全知識を活用します。

assignment課題

以下の仕様で医療クリニックの予約システムページを構築してください。

  • JSON-LD MedicalOrganization + Physician 構造化データ
  • 診療科選択 → 医師選択 → カレンダーで日時選択 → 患者情報入力 の4ステッププログレスフォーム
  • 医師一覧: <article> カード（写真 <picture>、専門分野、資格、予約可能状況 <meter>）
  • カレンダーテーブル: <table> + <th abbr>、予約可能スロットを <button> で表示、aria-label で日時を読み上げ
  • 患者情報フォーム: 氏名、生年月日（date）、保険証番号（pattern）、症状テキストエリア、同意チェックボックス
  • 予約確認サマリー（<dl>）、キャンセルポリシー（<details>）、ライブリージョンでの予約完了通知
