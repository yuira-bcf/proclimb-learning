T1 HTML基本構造・文書型

多言語対応ページの基本構造

verified
学習ポイント

lang 属性は文書全体だけでなく、個別の要素にも設定できます。部分的に異なる言語を含むページでは要素レベルの lang が重要です。

assignment課題

以下の仕様に従い、HTMLファイルをゼロから作成してください。

  • HTML5の文書型宣言、日本語ページ（lang="ja"）
  • タイトル：「多言語サンプル」
  • メイン見出し「ようこそ」の下に、英語の段落（lang="en"）で "Welcome to our website." を配置
  • さらに中国語の段落（lang="zh"）で "欢迎来到我们的网站。" を配置
lightbulbヒント

個別要素の言語指定は <p lang="en"> のように書きます。<html lang="ja"> はページ全体のデフォルト言語です。

check_circle解答・解説 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>多言語サンプル</title>
</head>
<body>
  <h1>ようこそ</h1>
  <p lang="en">Welcome to our website.</p>
  <p lang="zh">欢迎来到我们的网站。</p>
</body>
</html>
解説
  • 要素レベルの lang はスクリーンリーダーがその部分を正しい発音で読み上げるために必要です。
  • CSS の :lang() 疑似クラスでスタイルを切り替えることも可能です。
C-02
T1 HTML基本構造・文書型

外部リソース読み込みページ

verified
学習ポイント

<link> で外部CSSを、<script> で外部JavaScriptを読み込みます。読み込み順序はパフォーマンスに影響します。

assignment課題

以下の仕様に従い、HTMLファイルをゼロから作成してください。

  • 外部CSS 2ファイル（reset.css, style.css）を読み込む
  • Google Fonts から Noto Sans JP を読み込む
  • 外部JavaScript（app.js）を defer 属性付きで読み込む
  • body内に見出し「外部リソースの読み込み」を配置
lightbulbヒント

CSSは <link rel="stylesheet" href="..."> で読み込みます。defer 属性はHTMLの解析後にスクリプトを実行します。

check_circle解答・解説 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>外部リソースの読み込み</title>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap" rel="stylesheet">
  <link rel="stylesheet" href="reset.css">
  <link rel="stylesheet" href="style.css">
  <script src="app.js" defer></script>
</head>
<body>
  <h1>外部リソースの読み込み</h1>
</body>
</html>
解説
  • rel="preconnect" は外部ドメインへの事前接続で、フォント読み込みを高速化します。
  • defer 属性はDOMの解析を妨げずにスクリプトを読み込み、解析後に実行します。
  • CSSの読み込み順序は重要で、reset.css を先に読み込むことで後のスタイルで上書きできます。

T2 テキスト・リスト・リンク

レシピページのマークアップ

verified
学習ポイント

コンテンツの意味に応じて <ol>（手順）と <ul>（材料）を使い分けます。<dl>（定義リスト）も活用できます。

assignment課題

以下の仕様でカレーのレシピページを作成してください。

  • 見出し「簡単カレーレシピ」
  • 材料リスト（順序なし）：玉ねぎ 2個、にんじん 1本、じゃがいも 3個、カレールー 1箱
  • 手順リスト（順序あり）：野菜を切る → 鍋で炒める → 水を加えて煮る → ルーを入れて煮込む
  • 調理情報を定義リスト（<dl>）で：調理時間 45分、難易度 初級、人数 4人分
lightbulbヒント

定義リストは <dl> の中に <dt>（用語）と <dd>（定義）のペアで記述します。

check_circle解答・解説 
<h1>簡単カレーレシピ</h1>

<h2>材料</h2>
<ul>
  <li>玉ねぎ 2個</li>
  <li>にんじん 1本</li>
  <li>じゃがいも 3個</li>
  <li>カレールー 1箱</li>
</ul>

<h2>手順</h2>
<ol>
  <li>野菜を一口大に切る</li>
  <li>鍋で野菜を炒める</li>
  <li>水を加えて柔らかくなるまで煮る</li>
  <li>カレールーを入れてとろみがつくまで煮込む</li>
</ol>

<h2>調理情報</h2>
<dl>
  <dt>調理時間</dt>
  <dd>45分</dd>
  <dt>難易度</dt>
  <dd>初級</dd>
  <dt>人数</dt>
  <dd>4人分</dd>
</dl>
解説
  • 材料は順序に意味がないので <ul>、手順は順番が重要なので <ol> が適切です。
  • <dl>（description list）は「用語と定義」のペアに使います。FAQ、メタデータ、用語集などに適しています。
  • 1つの <dt> に複数の <dd> を対応させることも可能です。
C-04
T2 テキスト・リスト・リンク

ナビゲーションメニュー

verified
学習ポイント

ナビゲーションはリストとリンクの組み合わせで構築します。ページ内リンク（アンカーリンク）は href="#id" で指定します。

assignment課題

以下の仕様でナビゲーション付きページを作成してください。

  • グローバルナビ：ホーム（/）、サービス（/services）、会社概要（/about）、お問い合わせ（/contact）
  • ページ内目次：3つのセクション（「サービス紹介」「料金プラン」「よくある質問」）へのアンカーリンク
  • 各セクションに id 属性を設定し、目次からジャンプできるようにする
  • ページ末尾に「トップに戻る」リンクを配置
lightbulbヒント

ページ内リンクは <a href="#section-id"> で、対象セクションに id="section-id" を付けます。ページトップへは href="#" または href="#top" で移動できます。

check_circle解答・解説 
<header>
  <nav aria-label="グローバルナビ">
    <ul>
      <li><a href="/">ホーム</a></li>
      <li><a href="/services">サービス</a></li>
      <li><a href="/about">会社概要</a></li>
      <li><a href="/contact">お問い合わせ</a></li>
    </ul>
  </nav>
</header>

<main>
  <h1>サービスページ</h1>

  <nav aria-label="ページ内目次">
    <ol>
      <li><a href="#services">サービス紹介</a></li>
      <li><a href="#pricing">料金プラン</a></li>
      <li><a href="#faq">よくある質問</a></li>
    </ol>
  </nav>

  <section id="services">
    <h2>サービス紹介</h2>
    <p>当社のサービス内容をご紹介します。</p>
  </section>

  <section id="pricing">
    <h2>料金プラン</h2>
    <p>各プランの料金をご案内します。</p>
  </section>

  <section id="faq">
    <h2>よくある質問</h2>
    <p>お客様からよくいただくご質問です。</p>
  </section>
</main>

<footer>
  <a href="#">トップに戻る</a>
</footer>
解説
  • href="#services" は id="services" を持つ要素までスクロールします。
  • グローバルナビとページ内目次はそれぞれ別の <nav> で囲み、aria-label で区別します。
  • <section> はテーマ別のコンテンツブロックを表し、通常見出しを持ちます。

C-05
T3 テーブル

成績表テーブル

verified
学習ポイント

<caption> でテーブルにタイトルを付け、<tfoot> で合計行を示します。scope 属性でヘッダーの対象範囲を明示するとアクセシビリティが向上します。

assignment課題

以下の仕様で成績表テーブルを作成してください。

  • キャプション：「2026年度 前期成績表」
  • ヘッダー行：科目、中間、期末、平均（各 <th scope="col">）
  • データ行3つ：HTML（80, 90, 85）、CSS（75, 85, 80）、JavaScript（70, 80, 75）
  • 行ヘッダー：各科目名を <th scope="row"> で記述
  • フッター行：合計を <tfoot> で表示（225, 255, 240）
lightbulbヒント

<caption> は <table> の直下（最初の子要素）に配置します。scope="col" は「この見出しは列のヘッダー」、scope="row" は「行のヘッダー」を意味します。

check_circle解答・解説 
<table>
  <caption>2026年度 前期成績表</caption>
  <thead>
    <tr>
      <th scope="col">科目</th>
      <th scope="col">中間</th>
      <th scope="col">期末</th>
      <th scope="col">平均</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">HTML</th>
      <td>80</td>
      <td>90</td>
      <td>85</td>
    </tr>
    <tr>
      <th scope="row">CSS</th>
      <td>75</td>
      <td>85</td>
      <td>80</td>
    </tr>
    <tr>
      <th scope="row">JavaScript</th>
      <td>70</td>
      <td>80</td>
      <td>75</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">合計</th>
      <td>225</td>
      <td>255</td>
      <td>240</td>
    </tr>
  </tfoot>
</table>
解説
  • <caption> はスクリーンリーダーがテーブルの目的を伝えるために使います。
  • scope="col" / scope="row" によりスクリーンリーダーがデータセルとヘッダーの関係を正しく読み上げます。
  • <tfoot> はHTMLソース上で <tbody> の前後どちらに書いても正しく表示されますが、<tbody> の後に書くのが現在の慣習です。
C-06
T3 テーブル

時間割テーブル（セル結合）

verified
学習ポイント

colspan と rowspan を組み合わせることで複雑なテーブルレイアウトを実現できます。結合するセル数に応じて後続セルを省略するのを忘れずに。

assignment課題

以下の仕様で時間割テーブルを作成してください。

  • 列：時限、月曜、火曜、水曜
  • 1限：月曜=HTML、火曜=CSS、水曜=JavaScript
  • 2限：月曜=HTML（1限と rowspan 結合）、火曜=CSS（1限と結合）、水曜=演習
  • 3限：月曜〜水曜を colspan="3" で「昼休み」と結合
  • 4限：月曜=JavaScript、火曜=演習、水曜=演習
lightbulbヒント

rowspan で結合したセルは次の行から対応するセルを削除します。colspan="3" は3列分を1つのセルに結合します。

check_circle解答・解説 
<table border="1">
  <thead>
    <tr>
      <th>時限</th>
      <th>月曜</th>
      <th>火曜</th>
      <th>水曜</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1限</td>
      <td rowspan="2">HTML</td>
      <td rowspan="2">CSS</td>
      <td>JavaScript</td>
    </tr>
    <tr>
      <td>2限</td>
      <td>演習</td>
    </tr>
    <tr>
      <td>3限</td>
      <td colspan="3">昼休み</td>
    </tr>
    <tr>
      <td>4限</td>
      <td>JavaScript</td>
      <td>演習</td>
      <td>演習</td>
    </tr>
  </tbody>
</table>
解説
  • 1限の月曜・火曜に rowspan="2" を指定したため、2限の行では月曜・火曜のセルを省略しています。
  • 3限の colspan="3" は月曜〜水曜の3列を1セルに統合します。
  • 複雑な結合テーブルはアクセシビリティが低下するため、可能であればシンプルな構造を推奨します。

C-07
T4 フォーム・入力要素

お問い合わせフォーム

verified
学習ポイント

フォームには様々な入力タイプ（text, email, tel, textarea）があります。HTML5バリデーション属性（required, pattern, minlength）でクライアント側の検証が可能です。

assignment課題

以下の仕様でお問い合わせフォームを作成してください。

  • フォーム送信先：/contact、メソッド：POST
  • 氏名（text、必須、最大50文字）
  • メールアドレス（email、必須）
  • 電話番号（tel、任意、プレースホルダー「090-1234-5678」）
  • お問い合わせ種別（select）：一般的な質問、技術的な質問、その他
  • お問い合わせ内容（textarea、必須、最低20文字、5行表示）
  • 送信ボタン
lightbulbヒント

maxlength で最大文字数、minlength で最小文字数を指定します。placeholder は入力欄のヒントテキストです。<textarea rows="5"> で行数を指定できます。

check_circle解答・解説 
<form action="/contact" method="post">
  <div>
    <label for="name">氏名</label>
    <input type="text" id="name" name="name" required maxlength="50">
  </div>
  <div>
    <label for="email">メールアドレス</label>
    <input type="email" id="email" name="email" required>
  </div>
  <div>
    <label for="tel">電話番号</label>
    <input type="tel" id="tel" name="tel" placeholder="090-1234-5678">
  </div>
  <div>
    <label for="category">お問い合わせ種別</label>
    <select id="category" name="category">
      <option value="">選択してください</option>
      <option value="general">一般的な質問</option>
      <option value="technical">技術的な質問</option>
      <option value="other">その他</option>
    </select>
  </div>
  <div>
    <label for="message">お問い合わせ内容</label>
    <textarea id="message" name="message" required minlength="20" rows="5"></textarea>
  </div>
  <button type="submit">送信する</button>
</form>
解説
  • type="email" はブラウザがメール形式の検証を自動で行います。モバイルでは @ 付きキーボードが表示されます。
  • type="tel" はモバイルで電話番号用キーボードが表示されます。形式の検証はされません。
  • 空の value="" の <option> をデフォルトにすると、未選択状態を表現できます。
  • <textarea> は <input> と異なり閉じタグが必要です。
C-08
T4 フォーム・入力要素

ユーザー登録フォーム

verified
学習ポイント

<fieldset> でフォーム要素をグループ化し、<legend> でグループ名を付けます。pattern 属性で正規表現によるバリデーションが可能です。

assignment課題

以下の仕様でユーザー登録フォームを作成してください。

  • 「基本情報」フィールドセット：ユーザー名（必須、英数字3〜20文字、pattern使用）、メール（必須）、パスワード（必須、8文字以上）
  • 「プロフィール」フィールドセット：生年月日（date）、性別（ラジオボタン：男性/女性/回答しない）
  • 利用規約への同意チェックボックス（必須）
  • 送信ボタンとリセットボタン
lightbulbヒント

英数字のパターンは pattern="[a-zA-Z0-9]{3,20}" です。type="date" はブラウザがカレンダーUIを表示します。<button type="reset"> でフォームをリセットできます。

check_circle解答・解説 
<form action="/register" method="post">
  <fieldset>
    <legend>基本情報</legend>
    <div>
      <label for="username">ユーザー名</label>
      <input type="text" id="username" name="username" required
        pattern="[a-zA-Z0-9]{3,20}" title="英数字3〜20文字で入力してください">
    </div>
    <div>
      <label for="email">メールアドレス</label>
      <input type="email" id="email" name="email" required>
    </div>
    <div>
      <label for="password">パスワード</label>
      <input type="password" id="password" name="password" required minlength="8">
    </div>
  </fieldset>

  <fieldset>
    <legend>プロフィール</legend>
    <div>
      <label for="birthday">生年月日</label>
      <input type="date" id="birthday" name="birthday">
    </div>
    <div>
      <p>性別</p>
      <label><input type="radio" name="gender" value="male"> 男性</label>
      <label><input type="radio" name="gender" value="female"> 女性</label>
      <label><input type="radio" name="gender" value="other"> 回答しない</label>
    </div>
  </fieldset>

  <div>
    <label>
      <input type="checkbox" name="agree" required>
      利用規約に同意する
    </label>
  </div>

  <button type="submit">登録する</button>
  <button type="reset">リセット</button>
</form>
解説
  • pattern 属性は正規表現でバリデーションルールを定義します。title 属性でエラーメッセージを指定できます。
  • type="date" はブラウザネイティブのカレンダーピッカーを表示します。
  • type="reset" はフォーム内の全入力値を初期状態に戻します。
  • <fieldset> + <legend> はスクリーンリーダーがフォームの構造を理解するのに役立ちます。

C-09
T5 メディア・埋め込み

レスポンシブ画像ギャラリー

verified
学習ポイント

<picture> 要素と <source> で画面サイズに応じた画像を出し分けできます。srcset と sizes 属性でレスポンシブな画像配信が可能です。

assignment課題

以下の仕様で画像ギャラリーを作成してください。

  • 3枚の画像を <figure> + <figcaption> でマークアップ
  • 1枚目は <picture> 要素で、768px以上はPC用画像、未満はモバイル用画像を表示
  • 2枚目は srcset で 1x / 2x の解像度別画像を指定
  • 3枚目は通常の <img> に loading="lazy" を指定
  • 全画像に適切な alt と width / height を設定
lightbulbヒント

<picture> の中に <source media="(min-width: 768px)" srcset="pc.jpg"> と <img>（フォールバック）を入れます。loading="lazy" は遅延読み込みです。

check_circle解答・解説 
<div class="gallery">
  <figure>
    <picture>
      <source media="(min-width: 768px)" srcset="images/photo1-pc.jpg">
      <img src="images/photo1-mobile.jpg" alt="海辺の夕日" width="600" height="400">
    </picture>
    <figcaption>海辺の夕日</figcaption>
  </figure>

  <figure>
    <img src="images/photo2.jpg"
      srcset="images/photo2.jpg 1x, images/photo2@2x.jpg 2x"
      alt="桜並木の風景" width="600" height="400">
    <figcaption>桜並木の風景</figcaption>
  </figure>

  <figure>
    <img src="images/photo3.jpg" alt="雪山のパノラマ"
      width="600" height="400" loading="lazy">
    <figcaption>雪山のパノラマ</figcaption>
  </figure>
</div>
解説
  • <picture> はアートディレクション（画面サイズで異なる構図の画像を表示）に使います。
  • srcset の 1x / 2x は画面の解像度（Retina対応）に応じた画像の出し分けです。
  • loading="lazy" はビューポート外の画像を遅延読み込みし、初期表示を高速化します。
C-10
T5 メディア・埋め込み

メディアプレイヤーページ

verified
学習ポイント

<iframe> で外部コンテンツ（YouTubeなど）を埋め込めます。<video> の <track> 要素で字幕を追加できます。

assignment課題

以下の仕様でメディアプレイヤーページを作成してください。

  • YouTube動画の埋め込み（<iframe>）：幅640、高さ360、title 属性付き
  • ローカル動画プレイヤー（<video>）：MP4形式、字幕トラック（<track>）付き
  • オーディオプレイヤー（<audio>）：MP3形式、preload="metadata"
  • 全要素を <section> で分け、見出しを付ける
lightbulbヒント

<track kind="subtitles" src="subs.vtt" srclang="ja" label="日本語"> で字幕を追加します。preload="metadata" はメタデータのみ事前読み込みします。

check_circle解答・解説 
<section>
  <h2>YouTube動画</h2>
  <iframe width="640" height="360"
    src="https://www.youtube.com/embed/dQw4w9WgXcQ"
    title="サンプル動画" frameborder="0"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowfullscreen></iframe>
</section>

<section>
  <h2>ローカル動画</h2>
  <video controls width="640">
    <source src="video/tutorial.mp4" type="video/mp4">
    <track kind="subtitles" src="subs/tutorial-ja.vtt" srclang="ja" label="日本語" default>
    お使いのブラウザは動画再生に対応していません。
  </video>
</section>

<section>
  <h2>オーディオ</h2>
  <audio controls preload="metadata">
    <source src="audio/podcast.mp3" type="audio/mpeg">
    お使いのブラウザはオーディオ再生に対応していません。
  </audio>
</section>
解説
  • <iframe> の title 属性はアクセシビリティに必須です。埋め込み内容を説明します。
  • <track> はWebVTT形式の字幕ファイルを指定します。default 属性で自動表示されます。
  • preload="metadata" は音声の長さなどメタデータのみ事前読み込みし、帯域幅を節約します。

verified
学習ポイント

ブログ記事はセマンティック要素を適切に組み合わせて構造化します。<article> は独立したコンテンツ、<section> はテーマ別のグループ、<aside> は補足情報です。

assignment課題

lightbulbヒント

verified
学習ポイント

assignment課題

以下の仕様で商品カード3枚を作成してください。

SEO最適化されたランディングページ

verified
学習ポイント

assignment課題

構造化データ（JSON-LD）の記述

verified
学習ポイント

JSON-LD（<script type="application/ld+json">）は検索エンジンにページの構造化データを伝えます。リッチスニペット（星評価、FAQ表示など）の表示条件になります。

assignment課題

以下の仕様でFAQページを作成し、JSON-LD構造化データを含めてください。

  • HTMLのFAQセクション：3つの質問と回答（<details> + <summary>）
  • JSON-LD で FAQPage スキーマを記述
  • パンくずリスト（BreadcrumbList）の構造化データも追加
lightbulbヒント

JSON-LD は <script type="application/ld+json"> タグ内に JSON 形式で記述します。@context に "https://schema.org" を指定します。

check_circle解答・解説 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>よくある質問 | サンプルサイト</title>
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "FAQPage",
    "mainEntity": [
      {
        "@type": "Question",
        "name": "入会金はかかりますか？",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "入会金は無料です。月額のみでご利用いただけます。"
        }
      },
      {
        "@type": "Question",
        "name": "途中解約は可能ですか？",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "いつでも解約可能です。違約金はかかりません。"
        }
      },
      {
        "@type": "Question",
        "name": "オンライン受講は可能ですか？",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "はい、全コースオンラインで受講可能です。"
        }
      }
    ]
  }
  </script>
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "BreadcrumbList",
    "itemListElement": [
      { "@type": "ListItem", "position": 1, "name": "ホーム", "item": "https://example.com/" },
      { "@type": "ListItem", "position": 2, "name": "よくある質問", "item": "https://example.com/faq" }
    ]
  }
  </script>
</head>
<body>
  <main>
    <h1>よくある質問</h1>
    <details>
      <summary>入会金はかかりますか？</summary>
      <p>入会金は無料です。月額のみでご利用いただけます。</p>
    </details>
    <details>
      <summary>途中解約は可能ですか？</summary>
      <p>いつでも解約可能です。違約金はかかりません。</p>
    </details>
    <details>
      <summary>オンライン受講は可能ですか？</summary>
      <p>はい、全コースオンラインで受講可能です。</p>
    </details>
  </main>
</body>
</html>
解説
  • JSON-LD はGoogleが推奨する構造化データ形式です。<head> 内に配置するのが一般的です。
  • FAQPage スキーマはGoogleの検索結果にFAQ表示（リッチスニペット）を出す条件になります。
  • BreadcrumbList はパンくずリストの構造化データで、検索結果でパンくず表示が出ます。
  • <details> + <summary> はJavaScript不要のアコーディオンUIを実現します。
