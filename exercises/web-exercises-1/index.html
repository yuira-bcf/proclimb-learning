WEB総合制作演習 I

これまでの全授業の集大成として、就職活動で使えるプロジェクトアーカイブを制作します。前半では企画・設計からフロントエンド実装までを行い、GitHub Pagesで公開します。

Phase 1: 企画・マーケティング

Day 1-3 / サービスの企画とマーケット分析

1
企画提案書の作成

プロジェクト概要、背景と課題、提案するソリューション、期待される効果をまとめた企画書を作成します。

コンセプト ターゲット スケジュール
2
マーケットリサーチ

3C分析、SWOT分析、STP分析、4P/4C分析、競合サイト比較を行い、差別化ポイントを明確にします。

3C分析 SWOT STP 競合分析

Phase 2: 設計

Day 4-5 / ペルソナ設計とサイト構造設計

3
ペルソナシート作成

ターゲットユーザーの基本情報、ライフスタイル、課題、カスタマージャーニーマップを具体化します。

ペルソナ ライフスタイル ジャーニーマップ
4
サイトマップ作成

サイト全体の構造図、ページ一覧、ユーザーフロー、URL設計を作成します。

IA設計 ページ構成 ユーザーフロー

Phase 3: デザイン

Day 6-7 / ビジュアルデザインとガイドライン策定

5
ワイヤーフレーム作成

全ページのワイヤーフレーム（PC版・スマートフォン版）をFigmaで作成します。

Figma レイアウト レスポンシブ
6
デザインガイドライン作成

カラーパレット、タイポグラフィ、UIコンポーネント、レイアウトルールを策定します。

カラー タイポ コンポーネント

Phase 4: フロントエンド実装

Day 8-12 / HTML/CSS/JavaScriptによる実装

7
HTML/CSSコーディング

セマンティックHTML、レスポンシブデザイン、アクセシビリティを考慮したマークアップを行います。

HTML5 CSS3 Flexbox Grid
8
JavaScript実装

インタラクティブなUI機能（アニメーション、動的機能）を3つ以上実装します。

DOM操作 イベント アニメーション
9
外部サービス連携

Googleマップ、SNS埋め込みなど外部サービスとの連携を実装します。

API連携 埋め込み SNS

Phase 5: プロジェクトアーカイブ構築・公開

Day 13-16 / プロジェクトアーカイブサイトの構築とGitHub公開

10
プロジェクトアーカイブサイト構築

制作物と制作ドキュメントを紹介するプロジェクトアーカイブサイトのベースを構築します。

index works about
11
ドキュメント配置

制作ドキュメント（企画書、リサーチ、ペルソナ等）をHTMLページとしてサイト内に配置します。

docs/ HTMLドキュメント
12
GitHub Pages公開

GitHubリポジトリにプッシュし、GitHub Pagesで公開。README.mdを充実させます。

Git GitHub README
