型判定ユーティリティ関数を実装

verified
学習ポイント

typeof 演算子は基本型を判定しますが、null が 'object' を返す、配列も 'object' になる等の落とし穴があります。

assignment課題

以下の仕様に従い、getType 関数を実装してください。

function getType(value) → string
- null → 'null'
- 配列 → 'array'
- それ以外 → typeof の結果をそのまま返す
terminal期待出力
getType(42)        → "number"
getType('hello')   → "string"
getType(null)      → "null"
getType([1, 2])    → "array"
getType({a: 1})    → "object"
getType(undefined) → "undefined"
lightbulbヒント

value === null で null 判定、Array.isArray(value) で配列判定ができます。判定順序が重要です。

check_circle解答・解説 
function getType(value) {
  if (value === null) return 'null';
  if (Array.isArray(value)) return 'array';
  return typeof value;
}

// テスト
console.log(getType(42));        // "number"
console.log(getType('hello'));   // "string"
console.log(getType(null));      // "null"
console.log(getType([1, 2]));    // "array"
console.log(getType({a: 1}));    // "object"
console.log(getType(undefined)); // "undefined"
解説
  • typeof null === 'object' はJavaScript の歴史的なバグです。null 判定は === で先に行う必要があります。
  • Array.isArray() は配列かどうかを確実に判定できます。typeof [] は 'object' を返すため使えません。
  • 早期 return パターンで判定の優先順位を明確にしています。

範囲チェック関数を実装

verified
学習ポイント

引数のバリデーションは関数設計の基本です。数値判定には Number.isFinite() が NaN や Infinity を除外できて安全です。

assignment課題

以下の仕様に従い、clamp 関数を実装してください。

function clamp(value, min, max) → number
- value が min 未満 → min を返す
- value が max 超過 → max を返す
- それ以外 → value をそのまま返す
- 数値でない引数が渡された場合は NaN を返す
terminal期待出力
clamp(5, 0, 10)    → 5
clamp(-3, 0, 10)   → 0
clamp(15, 0, 10)   → 10
clamp('abc', 0, 10) → NaN
lightbulbヒント

Number.isFinite() で3つの引数を検証してから、Math.min() と Math.max() を使う方法もあります。

check_circle解答・解説 
function clamp(value, min, max) {
  if (!Number.isFinite(value) || !Number.isFinite(min) || !Number.isFinite(max)) {
    return NaN;
  }
  return Math.min(Math.max(value, min), max);
}

console.log(clamp(5, 0, 10));      // 5
console.log(clamp(-3, 0, 10));     // 0
console.log(clamp(15, 0, 10));     // 10
console.log(clamp('abc', 0, 10));  // NaN
解説
  • Math.max(value, min) で下限を保証し、Math.min(result, max) で上限を保証するイディオムは頻出です。
  • Number.isFinite() は文字列を暗黙変換しないため、isFinite()（グローバル関数）より安全です。
  • CSSの clamp() 関数と同じ概念で、レスポンシブデザインでも活用されるパターンです。

メモ化関数を実装

verified
学習ポイント

メモ化（memoization）はクロージャを活用したキャッシュパターンです。同じ引数での再計算を防ぎ、パフォーマンスを改善します。

assignment課題

以下の仕様に従い、memoize 関数を実装してください。

function memoize(fn) → function
- 引数1つの関数 fn を受け取る
- 同じ引数で呼ばれた場合、キャッシュから結果を返す
- 初回呼び出し時のみ fn を実行し、結果をキャッシュする 
// テストコード
const expensiveCalc = (n) => {
  console.log(`計算中: ${n}`);
  return n * n;
};

const memoized = memoize(expensiveCalc);
console.log(memoized(5));  // "計算中: 5" → 25
console.log(memoized(5));  // （ログなし）→ 25（キャッシュから）
console.log(memoized(3));  // "計算中: 3" → 9
lightbulbヒント

クロージャで Map オブジェクトを保持し、引数をキーにして結果をキャッシュします。cache.has(arg) で既存チェックできます。

check_circle解答・解説 
function memoize(fn) {
  const cache = new Map();
  return (arg) => {
    if (cache.has(arg)) {
      return cache.get(arg);
    }
    const result = fn(arg);
    cache.set(arg, result);
    return result;
  };
}
解説
  • クロージャにより cache は memoize 関数の外からはアクセスできませんが、返された関数からは参照可能です。
  • Map はオブジェクトと異なり、あらゆる型をキーにできます。
  • 実務ではライブラリ（lodash の _.memoize など）が提供されますが、仕組みの理解が重要です。

デバウンス関数を実装

verified
学習ポイント

デバウンスは連続して呼ばれた場合、最後の呼び出しから一定時間後に1回だけ実行するパターンです。検索入力の API 呼び出し抑制などで使われます。

assignment課題

以下の仕様に従い、debounce 関数を実装してください。

function debounce(fn, delay) → function
- fn: 実行したい関数
- delay: 待機時間（ミリ秒）
- 返された関数が連続呼び出しされた場合、最後の呼び出しから delay ミリ秒後に fn を1回だけ実行 
// 使用例
const log = debounce((msg) => console.log(msg), 300);
log('a'); // キャンセルされる
log('b'); // キャンセルされる
log('c'); // 300ms 後に "c" が出力される
lightbulbヒント

クロージャで timerId を保持します。呼び出しのたびに clearTimeout(timerId) で前回のタイマーをキャンセルし、新しい setTimeout を設定します。

check_circle解答・解説 
function debounce(fn, delay) {
  let timerId = null;
  return (...args) => {
    clearTimeout(timerId);
    timerId = setTimeout(() => fn(...args), delay);
  };
}
解説
  • clearTimeout に null を渡しても安全です（何も起きません）。
  • ...args（レスト構文）で任意の引数を受け取り、fn(...args)（スプレッド構文）で展開して渡します。
  • 検索窓の入力に対する API リクエスト、ウィンドウリサイズ時の再計算など、実務で頻出のパターンです。

配列のグループ化関数を実装

verified
学習ポイント

reduce を使ってオブジェクトを組み立てるパターンは、集約・グループ化処理の定番です。

assignment課題

以下の仕様に従い、groupBy 関数を実装してください。

function groupBy(array, keyFn) → object
- array: オブジェクトの配列
- keyFn: 各要素からグループキーを返す関数
- 戻り値: キーごとに要素をまとめたオブジェクト 
const people = [
  { name: '田中', dept: '開発' },
  { name: '鈴木', dept: '営業' },
  { name: '佐藤', dept: '開発' },
  { name: '高橋', dept: '営業' },
  { name: '山田', dept: '人事' }
];

const result = groupBy(people, p => p.dept);
// { 開発: [{name:'田中',...}, {name:'佐藤',...}],
//   営業: [{name:'鈴木',...}, {name:'高橋',...}],
//   人事: [{name:'山田',...}] }
lightbulbヒント

reduce の初期値にオブジェクト {} を渡し、キーが存在しなければ配列を作成してから push します。

check_circle解答・解説 
function groupBy(array, keyFn) {
  return array.reduce((groups, item) => {
    const key = keyFn(item);
    if (!groups[key]) {
      groups[key] = [];
    }
    groups[key].push(item);
    return groups;
  }, {});
}
解説
  • reduce のアキュムレータとしてオブジェクトを使い、各要素をキーごとの配列に振り分けています。
  • groups[key] ??= []（Nullish 代入）を使うとより簡潔に書けます。
  • ES2024 では Object.groupBy() が標準メソッドとして追加されていますが、仕組みの理解が重要です。

ディープコピー関数を実装

verified
学習ポイント

オブジェクトの代入はシャローコピー（参照の共有）です。ネストしたオブジェクトまで完全にコピーするには再帰的な処理が必要です。

assignment課題

以下の仕様に従い、deepClone 関数を実装してください（Date, RegExp, Map, Set は対象外）。

function deepClone(obj) → object | array
- プリミティブ値 → そのまま返す
- 配列 → 各要素を再帰的にコピーした新しい配列
- オブジェクト → 各プロパティを再帰的にコピーした新しいオブジェクト 
const original = { a: 1, b: { c: 2, d: [3, 4] } };
const cloned = deepClone(original);
cloned.b.c = 99;
cloned.b.d.push(5);
console.log(original.b.c); // 2（元は変わらない）
console.log(original.b.d); // [3, 4]（元は変わらない）
lightbulbヒント

配列なら obj.map(item => deepClone(item))、オブジェクトなら Object.entries() でキー・値を取り出して再帰コピーします。プリミティブ判定は typeof obj !== 'object' || obj === null です。

check_circle解答・解説 
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }
  if (Array.isArray(obj)) {
    return obj.map(item => deepClone(item));
  }
  const cloned = {};
  for (const [key, value] of Object.entries(obj)) {
    cloned[key] = deepClone(value);
  }
  return cloned;
}
解説
  • 再帰の終了条件はプリミティブ値（null を含む）に到達した場合です。
  • 実務では structuredClone()（モダンブラウザ対応）が推奨されます。JSON.parse(JSON.stringify()) は関数や undefined が消えるため注意。
  • この問題はオブジェクトの参照と値の違い、再帰処理の理解を確認する良い練習です。

動的テーブル生成関数を実装

verified
学習ポイント

createElement と appendChild を組み合わせて、配列データからHTML要素を動的に生成するパターンはSPA開発の基礎です。

assignment課題

以下の仕様に従い、createTable 関数を実装してください。

function createTable(headers, rows) → HTMLTableElement
- headers: 文字列の配列（ヘッダー行）
- rows: 文字列配列の配列（データ行）
- thead > tr > th でヘッダー、tbody > tr > td でデータを生成
- 生成した table 要素を返す 
const headers = ['名前', '年齢', '職業'];
const rows = [
  ['田中', '25', 'エンジニア'],
  ['鈴木', '30', 'デザイナー'],
  ['佐藤', '28', 'マネージャー']
];

const table = createTable(headers, rows);
document.body.appendChild(table);
lightbulbヒント

document.createElement('table') でテーブルを作り、thead・tbody をそれぞれ追加します。forEach でループして tr・th・td を生成します。

check_circle解答・解説 
function createTable(headers, rows) {
  const table = document.createElement('table');

  // thead
  const thead = document.createElement('thead');
  const headerRow = document.createElement('tr');
  headers.forEach(text => {
    const th = document.createElement('th');
    th.textContent = text;
    headerRow.appendChild(th);
  });
  thead.appendChild(headerRow);
  table.appendChild(thead);

  // tbody
  const tbody = document.createElement('tbody');
  rows.forEach(rowData => {
    const tr = document.createElement('tr');
    rowData.forEach(text => {
      const td = document.createElement('td');
      td.textContent = text;
      tr.appendChild(td);
    });
    tbody.appendChild(tr);
  });
  table.appendChild(tbody);

  return table;
}
解説
  • textContent を使うことでXSS（クロスサイトスクリプティング）を防止しています。
  • 大量のDOM操作がある場合は DocumentFragment を使うとパフォーマンスが向上します。
  • 実務ではReactやVueなどのフレームワークがこの処理を抽象化しますが、基礎の理解は重要です。

Todoリスト — 追加・削除機能を実装

verified
学習ポイント

入力フォームからのデータ取得、DOM要素の追加・削除、イベントリスナーの動的な割り当てを組み合わせた実践的な課題です。

assignment課題

以下のHTML構造に対して、Todoの追加と削除ができるJavaScriptを実装してください。

<!-- HTML -->
<input id="todoInput" type="text" placeholder="Todoを入力">
<button id="addBtn">追加</button>
<ul id="todoList"></ul>
  • 「追加」ボタンクリックで、入力値を <li> として #todoList に追加
  • 各 <li> に「削除」ボタンを付け、クリックでその項目を削除
  • 空文字の場合は追加しない
  • 追加後、入力欄をクリアしてフォーカスを戻す
lightbulbヒント

削除ボタンのクリックで li.remove() を呼びます。input.focus() でフォーカスを戻せます。

check_circle解答・解説 
const input = document.getElementById('todoInput');
const addBtn = document.getElementById('addBtn');
const list = document.getElementById('todoList');

addBtn.addEventListener('click', () => {
  const text = input.value.trim();
  if (text === '') return;

  const li = document.createElement('li');
  li.textContent = text;

  const deleteBtn = document.createElement('button');
  deleteBtn.textContent = '削除';
  deleteBtn.addEventListener('click', () => li.remove());

  li.appendChild(deleteBtn);
  list.appendChild(li);

  input.value = '';
  input.focus();
});
解説
  • li.remove() は要素自身をDOMから削除するメソッドです。親要素の removeChild より簡潔です。
  • trim() でスペースのみの入力を防止しています。
  • 削除ボタンのイベントリスナー内で li を参照しているのはクロージャの活用例です。

イベント委譲（Event Delegation）は、子要素のイベントを親要素で一括処理するパターンです。動的に追加される要素にも対応でき、メモリ効率も良くなります。

assignment課題

以下のHTML構造に対して、イベント委譲を使ったクリック処理を実装してください。

<ul id="menu">

e.target.closest('li') でクリックされた li を特定します。dataset.action でカスタムデータ属性を読み取れます。

const menu = document.getElementById('menu');
const status = document.getElementById('status');

menu.addEventListener('click', (e) => {

フォームバリデーションを実装

verified
学習ポイント

submit イベントで e.preventDefault() を使いフォーム送信を制御します。バリデーションはユーザー体験の要です。

assignment課題

以下のフォームに対して、送信前のバリデーション処理を実装してください。

<form id="signupForm">
  <input name="username" placeholder="ユーザー名（3文字以上）">
  <input name="email" type="email" placeholder="メールアドレス">
  <input name="password" type="password" placeholder="パスワード（8文字以上）">
  <button type="submit">登録</button>
  <div id="errors"></div>
</form>
  • ユーザー名: 3文字以上
  • メール: @ を含む
  • パスワード: 8文字以上
  • エラーがあれば送信を中止し、#errors に一覧表示
lightbulbヒント

form.addEventListener('submit', (e) => { e.preventDefault(); ... }) で送信を制御します。form.elements.username.value でフォーム要素の値を取得できます。

check_circle解答・解説 
const form = document.getElementById('signupForm');
const errorsDiv = document.getElementById('errors');

form.addEventListener('submit', (e) => {
  e.preventDefault();
  const errors = [];

  const username = form.elements.username.value.trim();
  const email = form.elements.email.value.trim();
  const password = form.elements.password.value;

  if (username.length < 3) {
    errors.push('ユーザー名は3文字以上で入力してください');
  }
  if (!email.includes('@')) {
    errors.push('メールアドレスの形式が正しくありません');
  }
  if (password.length < 8) {
    errors.push('パスワードは8文字以上で入力してください');
  }

  errorsDiv.textContent = '';
  if (errors.length > 0) {
    errors.forEach(msg => {
      const p = document.createElement('p');
      p.textContent = msg;
      p.style.color = 'red';
      errorsDiv.appendChild(p);
    });
  } else {
    errorsDiv.textContent = '登録成功！';
    errorsDiv.style.color = 'green';
  }
});
解説
  • e.preventDefault() でフォームのデフォルト送信（ページ遷移）を防ぎます。
  • form.elements は name 属性でフォーム要素にアクセスできる HTMLFormControlsCollection です。
  • エラーメッセージを配列に集めてから一括表示するパターンは、複数フィールドのバリデーションで定番です。

T6 非同期処理・Promise

fetch APIでデータ取得関数を実装

verified
学習ポイント

fetch はHTTPリクエストを送るAPIで、Promiseを返します。async/await と組み合わせると読みやすいコードになります。

assignment課題

以下の仕様に従い、fetchUserData 関数を実装してください。

async function fetchUserData(userId) → object
- URL: https://jsonplaceholder.typicode.com/users/{userId}
- レスポンスが ok でない場合は Error を throw
- 成功時は JSON パースした結果を返す 
// 使用例
try {
  const user = await fetchUserData(1);
  console.log(user.name);  // "Leanne Graham"
  console.log(user.email); // "Sincere@april.biz"
} catch (error) {
  console.error('取得失敗:', error.message);
}
lightbulbヒント

fetch(url) は Response オブジェクトを返します。response.ok が false の場合にエラーを投げ、response.json() でJSONをパースします。

check_circle解答・解説 
async function fetchUserData(userId) {
  const response = await fetch(
    `https://jsonplaceholder.typicode.com/users/${userId}`
  );
  if (!response.ok) {
    throw new Error(`HTTP Error: ${response.status}`);
  }
  return await response.json();
}
解説
  • fetch はネットワークエラーの場合のみ reject します。HTTPエラー（404, 500等）は reject されないため、response.ok のチェックが必要です。
  • response.json() も Promise を返すため、await が必要です。
  • テンプレートリテラルでURLを組み立てています。実務では URL パラメータのエスケープにも注意が必要です。

Promise.all で並列リクエストを実装

verified
学習ポイント

Promise.all は複数のPromiseを並列実行し、全て完了するのを待ちます。1つでも失敗すると全体が reject されます。

assignment課題

以下の仕様に従い、複数ユーザーのデータを並列取得する fetchMultipleUsers 関数を実装してください。

async function fetchMultipleUsers(userIds) → array
- userIds: 数値の配列（例: [1, 2, 3]）
- 全ユーザーのデータを並列取得して配列で返す
- 1つでも失敗した場合はエラーを throw 
// 使用例
const users = await fetchMultipleUsers([1, 2, 3]);
users.forEach(u => console.log(u.name));
// "Leanne Graham"
// "Ervin Howell"
// "Clementine Bauch"
lightbulbヒント

userIds.map(id => fetchUserData(id)) で Promise の配列を作り、Promise.all() に渡します。

check_circle解答・解説 
async function fetchMultipleUsers(userIds) {

リトライ付き非同期関数を実装

verified
学習ポイント

ネットワークリクエストは一時的に失敗することがあります。自動リトライを実装することで、信頼性を向上させます。

assignment課題

以下の仕様に従い、retry 関数を実装してください。

async function retry(fn, maxRetries = 3) → any
- fn: Promise を返す関数
- maxRetries: 最大リトライ回数
- 成功したら結果を返す
- 全リトライ失敗で最後のエラーを throw 
// 使用例
let attempt = 0;
const unstable = async () => {
  attempt++;
  if (attempt < 3) throw new Error(`失敗 ${attempt}回目`);
  return '成功！';
};

const result = await retry(unstable, 5);
console.log(result); // "成功！"（3回目で成功）
lightbulbヒント

for ループで maxRetries 回まで try/catch で試行します。最後の試行でも失敗した場合に throw します。

check_circle解答・解説 
async function retry(fn, maxRetries = 3) {
  let lastError;
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error;
      console.log(`リトライ ${i + 1}/${maxRetries}: ${error.message}`);
    }
  }
  throw lastError;
}
解説
  • 成功時は即座に return でループを抜けます。失敗時はエラーを保存してループを続行します。
  • 実務ではリトライ間隔（指数バックオフ）を入れることが多いです: await new Promise(r => setTimeout(r, 2 ** i * 1000))
  • 全リトライ失敗時は最後のエラーを throw するのが一般的なパターンです。

try/catch の代わりに「成功/失敗を値として返す」パターンです。エラーの見落としを防ぎ、明示的なエラーハンドリングを促します。

assignment課題

以下の仕様に従い、trySafe 関数を実装してください。

async function trySafe(fn) → { ok: true, data } | { ok: false, error }
- fn: Promise を返す関数
- 成功時: { ok: true, data: 結果 }
- 失敗時: { ok: false, error: Errorオブジェクト } 
// 使用例
const result = await trySafe(() => fetchUserData(1));
if (result.ok) {
  console.log(result.data.name); // "Leanne Graham"
} else {
  console.log('エラー:', result.error.message);
}
lightbulbヒント

try ブロックで fn() を await し、成功時と失敗時でそれぞれ異なる形のオブジェクトを返します。

check_circle解答・解説 
async function trySafe(fn) {
  try {
    const data = await fn();
    return { ok: true, data };
  } catch (error) {
    return { ok: false, error };
  }
}

  • このパターンはRustの Result 型やGoのエラー戻り値に着想を得ています。

イベントエミッタークラスを実装

verified
学習ポイント

イベント駆動パターン（Pub/Sub）はJavaScript でよく使われる設計パターンです。クラスとMapを組み合わせて実装します。

assignment課題

以下の仕様に従い、EventEmitter クラスを実装してください。

class EventEmitter {
  on(event, handler) — イベントにハンドラを登録
  off(event, handler) — ハンドラを解除
  emit(event, ...args) — イベントを発火（登録済みハンドラを全て呼ぶ）
} 
const emitter = new EventEmitter();
const handler = (msg) => console.log(`受信: ${msg}`);

emitter.on('message', handler);
emitter.emit('message', 'こんにちは'); // "受信: こんにちは"
emitter.off('message', handler);
emitter.emit('message', 'さようなら'); // （何も出力されない）
lightbulbヒント

Map でイベント名をキー、ハンドラの配列を値として管理します。off は filter で該当ハンドラを除外します。

check_circle解答・解説 
class EventEmitter {
  constructor() {
    this.listeners = new Map();
  }

  on(event, handler) {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, []);
    }
    this.listeners.get(event).push(handler);
  }

  off(event, handler) {
    if (!this.listeners.has(event)) return;
    const handlers = this.listeners.get(event).filter(h => h !== handler);
    this.listeners.set(event, handlers);
  }

  emit(event, ...args) {
    if (!this.listeners.has(event)) return;
    this.listeners.get(event).forEach(handler => handler(...args));
  }
}
解説
  • Node.js の EventEmitter やブラウザの addEventListener と同じ概念を簡易実装しています。
  • off で filter を使うことで、参照が一致するハンドラだけを除外しています。
  • このパターンはコンポーネント間通信やプラグインシステムの基盤として使われます。

バリデーター builder パターンを実装

メソッドチェーンは return this で自身を返すことで実現します。直感的なAPIを設計する上で重要なパターンです。

以下の仕様に従い、メソッドチェーン可能な Validator クラスを実装してください。

  constructor(value) — 検証対象の値を受け取る

  minLength(n) — 最小文字数を検証

lightbulbヒント

各メソッドでバリデーションルールの関数を配列に追加し、return this でチェーンを継続します。validate() で全ルールを実行して結果を返します。

check_circle解答・解説 
class Validator {
  constructor(value) {
    this.value = value;
    this.errors = [];
  }

  isString() {
    if (typeof this.value !== 'string') {
      this.errors.push('文字列ではありません');
    }
    return this;
  }

  minLength(n) {
    if (typeof this.value === 'string' && this.value.length < n) {
      this.errors.push(`${n}文字以上で入力してください`);
    }
    return this;
  }

  maxLength(n) {
    if (typeof this.value === 'string' && this.value.length > n) {
      this.errors.push(`${n}文字以下で入力してください`);
    }
    return this;
  }

  matches(regex) {
    if (typeof this.value === 'string' && !regex.test(this.value)) {
      this.errors.push(`パターン ${regex} にマッチしません`);
    }
    return this;
  }

  validate() {
    return {
      valid: this.errors.length === 0,
      errors: this.errors
    };
  }
}
解説
  • 各検証メソッドが return this を返すことで、メソッドチェーンが可能になります。
  • jQuery、Zod（バリデーションライブラリ）、Express のミドルウェアなど、多くのライブラリがこのパターンを採用しています。
  • 各メソッドでは型チェックを先に行い、型が合わない場合はスキップすることで、連鎖的なエラーを防いでいます。
