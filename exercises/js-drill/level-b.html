設定オブジェクトのマージ・検証ユーティリティ

verified
学習ポイント

スプレッド構文でのマージ、型判定、デフォルト値の適用を組み合わせた実践的なユーティリティ設計です。

assignment課題

以下の仕様に従い、mergeConfig 関数を実装してください。

function mergeConfig(defaults, overrides, schema) → { config, errors }
- defaults: デフォルト設定オブジェクト
- overrides: ユーザー設定オブジェクト
- schema: 各キーの期待される型（例: { port: 'number', host: 'string' }）
- overrides の値で defaults を上書きするが、schema に合わない型はエラーとしてスキップ 
const defaults = { port: 3000, host: 'localhost', debug: false };
const overrides = { port: '8080', debug: true, host: '0.0.0.0' };
const schema = { port: 'number', host: 'string', debug: 'boolean' };

const { config, errors } = mergeConfig(defaults, overrides, schema);
console.log(config);
// { port: 3000, host: '0.0.0.0', debug: true }
// port は '8080'(string) なので型不一致でスキップされる
console.log(errors);
// ['port: expected number but got string']
lightbulbヒント

Object.entries(overrides) でループし、typeof value === schema[key] で型チェックします。合格した値だけをマージします。

check_circle解答・解説 
function mergeConfig(defaults, overrides, schema) {
  const config = { ...defaults };
  const errors = [];

  for (const [key, value] of Object.entries(overrides)) {
    if (!(key in schema)) continue;
    if (typeof value !== schema[key]) {
      errors.push(`${key}: expected ${schema[key]} but got ${typeof value}`);
      continue;
    }
    config[key] = value;
  }

  return { config, errors };
}
解説
  • スプレッド構文で defaults のシャローコピーを作り、型チェック合格した値だけを上書きします。
  • in 演算子で schema に定義されたキーかどうかを確認し、未知のキーはスキップします。
  • このパターンはライブラリやフレームワークの設定処理で実際に使われています。

文字列置換と正規表現、オブジェクトプロパティアクセスを組み合わせて、簡易テンプレートエンジンを作ります。

assignment課題

以下の仕様に従い、render 関数を実装してください。

function render(template, data) → string
- template: "Hello, {{name}}!" のようなプレースホルダを含む文字列
- data: { name: 'World' } のようなキー・値のオブジェクト

const tmpl = '{{greeting}}, {{name}}! あなたは{{age}}歳です。';
const data = { greeting: 'こんにちは', name: '田中', age: 25 };
console.log(render(tmpl, data));
// "こんにちは, 田中! あなたは25歳です。"

console.log(render('{{missing}}は未定義', {}));
// "は未定義"
lightbulbヒント

template.replace(/\{\{(\w+)\}\}/g, (match, key) => ...) で正規表現のキャプチャグループを使って置換します。

check_circle解答・解説 
function render(template, data) {
  return template.replace(/\{\{(\w+)\}\}/g, (match, key) => {
    return key in data ? String(data[key]) : '';
  });
}
解説
  • /\{\{(\w+)\}\}/g は {{key}} パターンをグローバルに検索します。(\w+) がキー名をキャプチャします。
  • replace のコールバックの第2引数にキャプチャグループの内容が渡されます。
  • String(data[key]) で数値なども文字列化しています。HandlebarsやMustacheの基本原理です。

パイプライン関数を実装

verified
学習ポイント

関数合成（Function Composition）は関数型プログラミングの基本です。reduce と高階関数を組み合わせて実装します。

assignment課題

以下の仕様に従い、pipe 関数を実装してください。

function pipe(...fns) → function
- 複数の関数を受け取り、左から右に順番に適用する新しい関数を返す
- 前の関数の戻り値が次の関数の引数になる 
const double = x => x * 2;
const addTen = x => x + 10;
const toString = x => `結果: ${x}`;

const transform = pipe(double, addTen, toString);
console.log(transform(5));  // "結果: 20"  (5*2=10, 10+10=20, "結果: 20")
console.log(transform(3));  // "結果: 16"  (3*2=6, 6+10=16, "結果: 16")
lightbulbヒント

fns.reduce((acc, fn) => fn(acc), initialValue) で関数を順番に適用できます。pipe は関数を返す関数（高階関数）です。

check_circle解答・解説 
function pipe(...fns) {
  return (value) => fns.reduce((acc, fn) => fn(acc), value);
}
解説
  • reduce のアキュムレータに前回の関数の戻り値が入り、次の関数に渡されます。
  • pipe は左から右に適用しますが、逆順（右から左）に適用する compose もあります: fns.reduceRight(...)
  • Ramda、lodash/fp など関数型ライブラリで広く使われるパターンです。

ミドルウェアパターンを実装

verified
学習ポイント

Express.js 等で使われるミドルウェアパターンは、クロージャと再帰（または反復）を組み合わせた高度なパターンです。

assignment課題

以下の仕様に従い、ミドルウェアを順番に実行する createApp 関数を実装してください。

function createApp() → { use(middleware), run(context) }
- use(fn): ミドルウェア関数 fn(context, next) を登録
- run(context): 登録済みミドルウェアを順番に実行
- 各ミドルウェアは next() を呼ぶことで次のミドルウェアに処理を渡す 
const app = createApp();

app.use((ctx, next) => {
  ctx.log = [];
  ctx.log.push('開始');
  next();
  ctx.log.push('終了');
});

app.use((ctx, next) => {
  ctx.log.push('処理中');
  ctx.result = ctx.input * 2;
  next();
});

const ctx = { input: 21 };
app.run(ctx);
console.log(ctx.log);    // ['開始', '処理中', '終了']
console.log(ctx.result); // 42
lightbulbヒント

ミドルウェアの配列とインデックスをクロージャで管理します。next 関数は次のインデックスのミドルウェアを呼び出す関数です。

check_circle解答・解説 
function createApp() {
  const middlewares = [];

  return {
    use(fn) {
      middlewares.push(fn);
    },
    run(context) {
      let index = 0;
      function next() {
        if (index < middlewares.length) {
          const fn = middlewares[index++];
          fn(context, next);
        }
      }
      next();
    }
  };
}
解説
  • next() が呼ばれるたびにインデックスを進め、次のミドルウェアを実行します。
  • 最初のミドルウェアで next() の前後にコードを書くと、リクエスト/レスポンスの前後処理ができます（オニオンモデル）。
  • Express.js、Koa.js、Next.js のミドルウェアの基本原理です。

データ変換パイプライン — 売上集計

verified
学習ポイント

filter → map → reduce のメソッドチェーンで、複雑なデータ変換をステップごとに記述できます。

assignment課題

売上データから、指定カテゴリの合計金額と平均単価を算出する summarizeSales 関数を実装してください。

const sales = [
  { product: 'りんご', category: 'fruit', price: 150, qty: 10 },
  { product: 'にんじん', category: 'vegetable', price: 100, qty: 20 },
  { product: 'バナナ', category: 'fruit', price: 120, qty: 15 },
  { product: 'キャベツ', category: 'vegetable', price: 200, qty: 5 },
  { product: 'みかん', category: 'fruit', price: 80, qty: 30 }
];

console.log(summarizeSales(sales, 'fruit'));
// { total: 5700, avgPrice: 116.67, items: 3 }
function summarizeSales(sales, category) → { total, avgPrice, items }
- total: そのカテゴリの売上合計（price × qty の合計）
- avgPrice: そのカテゴリの平均単価（小数第2位まで）
- items: 該当アイテム数
lightbulbヒント

filter でカテゴリを絞り、reduce で合計を算出します。平均は 合計 / 件数 で求め、Math.round(x * 100) / 100 で小数2位に丸めます。

check_circle解答・解説 
function summarizeSales(sales, category) {
  const filtered = sales.filter(s => s.category === category);
  const total = filtered.reduce((sum, s) => sum + s.price * s.qty, 0);
  const avgPrice = Math.round(
    (filtered.reduce((sum, s) => sum + s.price, 0) / filtered.length) * 100
  ) / 100;

  return { total, avgPrice, items: filtered.length };
}
解説
  • filter → reduce のチェーンで段階的にデータを変換しています。
  • Math.round(x * 100) / 100 は小数第2位まで丸める定番テクニックです。
  • ダッシュボードやレポート機能の実装で頻出するパターンです。

Proxy でリアクティブオブジェクトを実装

verified
学習ポイント

Proxy はオブジェクトへのアクセスを監視・制御できるメタプログラミング機能です。Vue 3 のリアクティブシステムの基盤技術です。

assignment課題

以下の仕様に従い、reactive 関数を実装してください。

function reactive(obj, onChange) → Proxy
- obj: 監視対象のオブジェクト
- onChange: プロパティ変更時に呼ばれるコールバック (key, newValue, oldValue) => void
- プロパティの読み書きは通常通り動作する 
const state = reactive({ count: 0, name: 'Alice' }, (key, newVal, oldVal) => {
  console.log(`${key}: ${oldVal} → ${newVal}`);
});

state.count = 1;  // "count: 0 → 1"
state.name = 'Bob'; // "name: Alice → Bob"
console.log(state.count); // 1
lightbulbヒント

new Proxy(obj, { set(target, key, value) { ... } }) でプロパティ設定をインターセプトします。set ハンドラは true を返す必要があります。

check_circle解答・解説 
function reactive(obj, onChange) {
  return new Proxy(obj, {
    set(target, key, value) {
      const oldValue = target[key];
      target[key] = value;
      onChange(key, value, oldValue);
      return true;
    }
  });
}
解説
  • Proxy の set トラップでプロパティ代入を検知し、コールバックを呼んでいます。
  • set ハンドラが true を返さないと strict mode で TypeError が発生します。
  • Vue 3 の reactive() はこの原理を発展させ、依存関係の追跡と自動再レンダリングを実現しています。

lightbulbヒント

クロージャでソート状態（現在のキー、方向）を管理します。ヘッダーにイベント委譲を使い、data.sort() 後に tbody を再描画します。

check_circle解答・解説 
function createSortableTable(containerId, data) {
  const container = document.getElementById(containerId);
  const headers = Object.keys(data[0]);
  let sortKey = null;
  let ascending = true;

  function renderTable() {
    container.textContent = '';
    const table = document.createElement('table');

    // thead
    const thead = document.createElement('thead');
    const headerRow = document.createElement('tr');
    headers.forEach(key => {
      const th = document.createElement('th');
      th.textContent = key + (sortKey === key ? (ascending ? ' ▲' : ' ▼') : '');
      th.style.cursor = 'pointer';
      th.dataset.key = key;
      headerRow.appendChild(th);
    });
    thead.appendChild(headerRow);
    table.appendChild(thead);

    // tbody
    const tbody = document.createElement('tbody');
    data.forEach(row => {
      const tr = document.createElement('tr');
      headers.forEach(key => {
        const td = document.createElement('td');
        td.textContent = row[key];
        tr.appendChild(td);
      });
      tbody.appendChild(tr);
    });
    table.appendChild(tbody);

    // イベント委譲
    thead.addEventListener('click', (e) => {
      const th = e.target.closest('th');
      if (!th) return;
      const key = th.dataset.key;
      ascending = sortKey === key ? !ascending : true;
      sortKey = key;
      data.sort((a, b) => {
        if (typeof a[key] === 'number') {
          return ascending ? a[key] - b[key] : b[key] - a[key];
        }
        return ascending
          ? String(a[key]).localeCompare(String(b[key]))
          : String(b[key]).localeCompare(String(a[key]));
      });
      renderTable();
    });

    container.appendChild(table);
  }

  renderTable();
}
解説
  • ソート状態をクロージャで管理し、ヘッダークリック時に data.sort() で並べ替えてから再描画しています。
  • localeCompare は日本語を含む多言語文字列のソートに適しています。
  • React / Vue ではこの手動DOM操作が不要になりますが、仕組みの理解は重要です。

lightbulbヒント

オーバーレイ div を position: fixed で画面全体に配置し、中央にモーダル本体を置きます。keydown イベントで Escape を検知します。

check_circle解答・解説 
function createModal({ title, content, onConfirm, onCancel }) {
  const overlay = document.createElement('div');
  Object.assign(overlay.style, {
    position: 'fixed', inset: '0', background: 'rgba(0,0,0,0.5)',
    display: 'none', alignItems: 'center', justifyContent: 'center', zIndex: '1000'
  });

  const modal = document.createElement('div');
  Object.assign(modal.style, {
    background: 'white', borderRadius: '12px', padding: '24px',
    maxWidth: '400px', width: '90%'
  });

  const h2 = document.createElement('h2');
  h2.textContent = title;
  const p = document.createElement('p');
  p.textContent = content;

  const btnGroup = document.createElement('div');
  btnGroup.style.display = 'flex';
  btnGroup.style.gap = '8px';
  btnGroup.style.justifyContent = 'flex-end';
  btnGroup.style.marginTop = '16px';

  const cancelBtn = document.createElement('button');
  cancelBtn.textContent = 'キャンセル';
  const confirmBtn = document.createElement('button');
  confirmBtn.textContent = '確認';

  btnGroup.append(cancelBtn, confirmBtn);
  modal.append(h2, p, btnGroup);
  overlay.appendChild(modal);
  document.body.appendChild(overlay);

  function close() {
    overlay.style.display = 'none';
    document.removeEventListener('keydown', handleKey);
  }

  function handleKey(e) {
    if (e.key === 'Escape') {
      close();
      onCancel?.();
    }
  }

  overlay.addEventListener('click', (e) => {
    if (e.target === overlay) { close(); onCancel?.(); }
  });
  cancelBtn.addEventListener('click', () => { close(); onCancel?.(); });
  confirmBtn.addEventListener('click', () => { close(); onConfirm?.(); });

  return {
    open() {
      overlay.style.display = 'flex';
      document.addEventListener('keydown', handleKey);
    },
    close
  };
}
解説
  • Object.assign(el.style, {...}) で複数のスタイルを一括設定しています。
  • onCancel?.() はオプショナルチェーンで、関数が未定義でもエラーになりません。
  • open 時にイベントリスナーを追加し、close 時に削除することでメモリリークを防いでいます。

コンテナにイベント委譲で click を登録し、e.target.dataset.action でボタン種別を判定します。カスタムイベントは new CustomEvent('counter:change', { detail: { value } }) で作成します。

check_circle解答・解説 
function createCounter(containerId, options = {}) {
  const { min = -Infinity, max = Infinity, step = 1 } = options;
  const container = document.getElementById(containerId);
  let value = min === -Infinity ? 0 : min;

  function render() {
    container.innerHTML = `
      <button data-action="decrement">-</button>
      <span class="counter-value">${value}</span>
      <button data-action="increment">+</button>
    `;
  }

  // イベント委譲
  container.addEventListener('click', (e) => {
    const action = e.target.dataset.action;
    if (!action) return;

    const oldValue = value;
    if (action === 'increment') {
      value = Math.min(value + step, max);
    } else if (action === 'decrement') {
      value = Math.max(value - step, min);
    }

    if (value !== oldValue) {
      render();
      container.dispatchEvent(new CustomEvent('counter:change', {
        detail: { value, oldValue },
        bubbles: true
      }));
    }
  });

  render();

  return {
    getValue() { return value; },
    reset() { value = min === -Infinity ? 0 : min; render(); }
  };
}
解説
  • イベント委譲により、ボタンが再描画されてもイベントリスナーの再登録が不要です。
  • CustomEvent の detail プロパティで任意のデータを渡せます。bubbles: true で親要素でもキャッチ可能です。
  • Math.min / Math.max で範囲制限を簡潔に実装しています。

function createShortcutManager() {
  const shortcuts = new Map();

  function parseCombo(combo) {
    const parts = combo.toLowerCase().split('+');
    return {
      ctrl: parts.includes('ctrl'),
      shift: parts.includes('shift'),
      alt: parts.includes('alt'),
      key: parts.filter(p => !['ctrl', 'shift', 'alt'].includes(p))[0]
    };
  }

  function handleKeydown(e) {
    for (const [combo, handler] of shortcuts) {
      const parsed = parseCombo(combo);
      if (
        e.ctrlKey === parsed.ctrl &&
        e.shiftKey === parsed.shift &&
        e.altKey === parsed.alt &&
        e.key.toLowerCase() === parsed.key
      ) {
        e.preventDefault();
        handler();
        return;
      }
    }
  }

  document.addEventListener('keydown', handleKeydown);

  return {
    register(combo, handler) {
      shortcuts.set(combo, handler);
    },
    unregister(combo) {
      shortcuts.delete(combo);
    },
    destroy() {
      shortcuts.clear();
      document.removeEventListener('keydown', handleKeydown);
    }
  };
}
解説
  • Map でショートカットとハンドラを管理し、keydown イベントで全登録ショートカットを照合しています。
  • e.preventDefault() でブラウザデフォルト動作（Ctrl+S の保存ダイアログ等）を抑止しています。
  • destroy() でイベントリスナーを確実に削除し、メモリリークを防止しています。
  • VS Code や Figma のショートカットシステムと同じ基本設計です。

function createLoader(fetchFn) {
  let state = { status: 'idle', data: null, error: null };
  const listeners = [];

  function setState(newState) {
    state = { ...state, ...newState };
    listeners.forEach(fn => fn(state));
  }

  return {
    async load() {
      setState({ status: 'loading', error: null });
      try {
        const data = await fetchFn();
        setState({ status: 'success', data });
        return data;
      } catch (error) {
        setState({ status: 'error', error });
        throw error;
      }
    },
    getState() {
      return { ...state };
    },
    onStateChange(fn) {
      listeners.push(fn);
      return () => {
        const i = listeners.indexOf(fn);
        if (i > -1) listeners.splice(i, 1);
      };
    }
  };
}
解説
  • async/await で非同期処理を同期的に記述し、try/catch でエラーハンドリングしています。
  • 状態遷移は idle → loading → success / error の有限ステートマシンパターンです。
  • React の useQuery（TanStack Query）やSWR の基本設計原理です。
  • onStateChange がアンサブスクライブ関数を返すのは、メモリリーク防止の定番パターンです。

Promise.allSettled(urls.map(url => fetch(url)...)) で全リクエストの結果を取得し、status === 'fulfilled' と 'rejected' で分類します。

check_circle解答・解説 
async function fetchAll(urls) {
  const promises = urls.map(async (url) => {
    const res = await fetch(url);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return { url, data: await res.json() };
  });

  const settled = await Promise.allSettled(promises);
  const results = [];
  const errors = [];

  settled.forEach((result, i) => {
    if (result.status === 'fulfilled') {
      results.push(result.value.data);
    } else {
      errors.push({ url: urls[i], error: result.reason.message });
    }
  });

  return {
    results,
    errors,
    summary: {
      total: urls.length,
      succeeded: results.length,
      failed: errors.length
    }
  };
}
解説
  • Promise.allSettled は全Promiseの完了を待ち、成功・失敗に関わらず結果を返します（Promise.all は1つでも失敗すると即座に reject）。
  • 各結果の status で成功/失敗を分類し、構造化されたレスポンスを返しています。
  • ダッシュボードで複数APIからデータを取得する場面や、バッチ処理で頻出するパターンです。

function trySafe(fn, fallback = undefined) {
  try {
    const value = fn();
    return { ok: true, value, error: null };
  } catch (error) {
    return { ok: false, value: fallback, error };
  }
}

async function trySafeAsync(fn, fallback = undefined) {
  try {
    const value = await fn();
    return { ok: true, value, error: null };
  } catch (error) {
    return { ok: false, value: fallback, error };
  }
}
解説

  • fallback でエラー時のデフォルト値を提供することで、後続処理が安全に継続できます。
  • Go言語の value, err := fn() や Rust の Result<T, E> と同じ発想です。
  • neverthrow ライブラリが TypeScript でこのパターンを型安全に提供しています。

async function withRetry(fn, options = {}) {
  const { maxRetries = 3, baseDelay = 1000, onRetry } = options;

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;

      const delay = baseDelay * Math.pow(2, attempt);
      onRetry?.(attempt + 1, error, delay);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
解説
  • 指数バックオフ（baseDelay * 2^attempt）により、サーバー負荷を抑えつつリトライします。
  • 最大試行回数を超えた場合は最後のエラーをそのまま throw し、呼び出し元で処理できます。
  • onRetry コールバックでリトライのログ出力やUI更新が可能です。
  • AWS SDK、axios-retry、fetch のラッパーなど実務で広く使われるパターンです。

class EventEmitter {
  constructor() {
    this._listeners = {};
  }
  on(event, fn) {
    (this._listeners[event] ||= []).push(fn);
    return this;
  }
  emit(event, ...args) {
    (this._listeners[event] || []).forEach(fn => fn(...args));
  }
}

class CountdownTimer extends EventEmitter {
  constructor(seconds) {
    super();
    this._initial = seconds;
    this._remaining = seconds;
    this._intervalId = null;
  }

  start() {
    if (this._intervalId) return;
    this._intervalId = setInterval(() => {
      this._remaining--;
      this.emit('tick', this._remaining);
      if (this._remaining <= 0) {
        this.pause();
        this.emit('complete');
      }
    }, 1000);
  }

  pause() {
    clearInterval(this._intervalId);
    this._intervalId = null;
    this.emit('pause');
  }

  reset() {
    this.pause();
    this._remaining = this._initial;
    this.emit('reset');
  }
}
解説
  • EventEmitter を基底クラスとして継承し、on / emit でイベント駆動の通信を実現しています。
  • ||=（論理OR代入）でリスナー配列を遅延初期化しています。
  • clearInterval でタイマーを確実に停止し、リソースリークを防いでいます。
  • Node.js の EventEmitter、ブラウザの EventTarget と同じ設計パターンです。

class Logger {
  static LEVELS = { debug: 0, info: 1, warn: 2, error: 3 };

  constructor({ prefix = '', level = 'debug' } = {}) {
    this._prefix = prefix;
    this._level = Logger.LEVELS[level] ?? 0;
    this._plugins = [];
  }

  use(plugin) {
    this._plugins.push(plugin);
    return this;
  }

  _log(level, message) {
    if (Logger.LEVELS[level] < this._level) return;

    let entry = { level, message, prefix: this._prefix };
    entry = this._plugins.reduce(
      (e, plugin) => plugin.transform ? plugin.transform(e) : e,
      entry
    );

    const tag = this._prefix ? `[${this._prefix}] ` : '';
    console.log(`${tag}${level.toUpperCase()}: ${entry.message}`);
  }

  debug(msg) { this._log('debug', msg); }
  info(msg)  { this._log('info', msg); }
  warn(msg)  { this._log('warn', msg); }
  error(msg) { this._log('error', msg); }
}
解説
  • static LEVELS でクラスレベルの定数を定義し、ログレベルの数値比較を行っています。
  • プラグインを reduce で順番に適用するパターンは、webpack プラグインや Express ミドルウェアと同じです。
  • use() が this を返すことで、メソッドチェーンが可能です。
  • Winston、Pino などのログライブラリの基本アーキテクチャです。
