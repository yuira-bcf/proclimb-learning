マルチモジュール構成のDI設計

verified
実務ポイント

大規模アプリケーションではapi/domain/infrastructureのマルチモジュール構成にします。各モジュール間の依存方向を制御し、ドメイン層が外部技術に依存しないようにします。

assignment課題

以下の仕様に従い、クリーンアーキテクチャに基づくマルチモジュール構成を設計してください。

  • domain モジュール: エンティティ、リポジトリインタフェース、ドメインサービス

  • domain → infrastructure への依存を禁止（DIP: 依存性逆転原則）

verified
実務ポイント

ECサイトの注文APIは、在庫確認→注文作成→決済→通知の一連のフローを含む総合的な実装です。トランザクション管理、例外処理、DTO変換を統合的に扱います。

assignment課題

以下の仕様に従い、ECサイトの注文APIを総合実装してください。

check_circle解答・解説 
// OrderService.java (要約)
@Service
public class OrderService {
    private final OrderMapper orderMapper;
    private final OrderItemMapper orderItemMapper;
    private final ProductMapper productMapper;
    private final ApplicationEventPublisher publisher;

    @Transactional
    public OrderResponse placeOrder(OrderRequest req) {
        // 1. 在庫確認&減算
        for (var item : req.getItems()) {
            Product p = productMapper.findById(item.getProductId());
            if (p == null) throw new ResourceNotFoundException(
                "Product not found: " + item.getProductId());
            if (p.getStock() < item.getQuantity())
                throw new InsufficientStockException(
                    p.getName(), item.getQuantity(), p.getStock());
            productMapper.decreaseStock(
                item.getProductId(), item.getQuantity());
        }
        // 2. 注文作成
        Order order = new Order();
        order.setCustomerName(req.getCustomerName());
        order.setStatus("PENDING");
        order.setOrderDate(LocalDateTime.now());
        orderMapper.insert(order);
        // 3. 明細作成
        for (var item : req.getItems()) {
            OrderItem oi = new OrderItem();
            oi.setOrderId(order.getId());
            oi.setProductId(item.getProductId());
            oi.setQuantity(item.getQuantity());
            orderItemMapper.insert(oi);
        }
        publisher.publishEvent(new OrderCreatedEvent(order));
        return OrderResponse.from(order);
    }

    @Transactional
    public void cancelOrder(Long id) {
        Order order = orderMapper.findById(id);
        if (order == null) throw new ResourceNotFoundException(
            "Order not found");
        if (!"PENDING".equals(order.getStatus()))
            throw new BusinessRuleException(
                "PENDING以外の注文はキャンセルできません");
        // 在庫復元
        List<OrderItem> items = orderItemMapper.findByOrderId(id);
        for (var item : items) {
            productMapper.increaseStock(
                item.getProductId(), item.getQuantity());
        }
        orderMapper.updateStatus(id, "CANCELLED");
    }
}
解説
  • 注文処理は「在庫確認→減算→注文作成→明細作成」の順で、どこかで失敗すると全てロールバックされます。
  • キャンセル処理では在庫の復元を忘れがちです。「逆操作」を意識するのが実務のポイントです。
  • イベント発行により、注文完了メール送信等の副次処理をServiceから分離できます。

        return ResponseEntity.ok()
            .contentType(MediaType.parseMediaType(dl.getContentType()))
            .header(HttpHeaders.CONTENT_DISPOSITION,

// FileService.java
@Service
public class FileService {
    private static final Set<String> ALLOWED = Set.of(
        "image/jpeg", "image/png", "application/pdf");
    private static final long MAX_SIZE = 5 * 1024 * 1024;

    @Transactional
    public FileResponse upload(MultipartFile file) {
        if (!ALLOWED.contains(file.getContentType()))
            throw new BadRequestException("許可されていないファイル形式です");
        if (file.getSize() > MAX_SIZE)
            throw new BadRequestException("ファイルサイズは5MB以下にしてください");

        String storedName = UUID.randomUUID() + "_"
            + file.getOriginalFilename();
        Path path = Path.of("uploads", storedName);
        try { file.transferTo(path); }
        catch (IOException e) {
            throw new RuntimeException("ファイル保存に失敗しました", e);
        }
        FileMeta meta = new FileMeta();
        meta.setOriginalName(file.getOriginalFilename());
        meta.setStoredName(storedName);
        meta.setContentType(file.getContentType());
        meta.setSize(file.getSize());
        fileMapper.insert(meta);
        return FileResponse.from(meta);
    }
}
解説
  • UUIDをファイル名のプレフィックスにすることで、同名ファイルの衝突を防ぎます。
  • ファイル種類の検証は拡張子ではなくContent-Type（MIME type）で行うのが安全です。
  • ダウンロード時の Content-Disposition ヘッダーで、ブラウザに保存ダイアログを表示させます。

// ExpenseService.java
@Service
public class ExpenseService {
    private static final Map<String, Set<String>> TRANSITIONS = Map.of(
        "DRAFT", Set.of("SUBMITTED"),
        "SUBMITTED", Set.of("APPROVED", "REJECTED")
    );

    @Transactional
    public ExpenseRequest transition(Long id, String newStatus,
            String operatorName) {
        ExpenseRequest req = expenseMapper.findById(id);
        if (req == null) throw new ResourceNotFoundException(
            "Expense not found");

        Set<String> allowed = TRANSITIONS.getOrDefault(
            req.getStatus(), Set.of());
        if (!allowed.contains(newStatus)) {
            throw new BusinessRuleException(
                req.getStatus() + " → " + newStatus + " は不正な遷移です");
        }

        expenseMapper.updateStatus(id, newStatus);
        ExpenseHistory history = new ExpenseHistory();
        history.setExpenseId(id);
        history.setFromStatus(req.getStatus());
        history.setToStatus(newStatus);
        history.setOperator(operatorName);
        history.setOperatedAt(LocalDateTime.now());
        historyMapper.insert(history);

        return expenseMapper.findById(id);
    }
}

@RestControllerAdvice
public class GlobalExceptionHandler {
    private static final Logger log =
        LoggerFactory.getLogger(GlobalExceptionHandler.class);

    @Value("${app.error.show-detail:false}")
    private boolean showDetail;

    @ExceptionHandler(AppException.class)
    public ResponseEntity<ProblemDetail> handleApp(
            AppException ex) {
        log.warn("[{}] {}: {}", MDC.get("traceId"),
            ex.getErrorCode(), ex.getMessage());
        ProblemDetail pd = ProblemDetail.forStatusAndDetail(
            ex.getHttpStatus(), ex.getMessage());
        pd.setType(URI.create(ex.getTypeUri()));
        pd.setProperty("errorCode", ex.getErrorCode());
        pd.setProperty("traceId", MDC.get("traceId"));
        ex.getProperties().forEach(pd::setProperty);
        return ResponseEntity.status(ex.getHttpStatus()).body(pd);
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ProblemDetail> handleValidation(
            MethodArgumentNotValidException ex) {
        log.warn("[{}] Validation failed: {} errors",
            MDC.get("traceId"),
            ex.getBindingResult().getErrorCount());
        ProblemDetail pd = ProblemDetail.forStatusAndDetail(
            HttpStatus.BAD_REQUEST, "入力値に誤りがあります");
        pd.setType(URI.create("/problems/validation-error"));
        pd.setProperty("traceId", MDC.get("traceId"));
        pd.setProperty("errorCode", "VALIDATION_ERROR");
        pd.setProperty("errors",
            ex.getBindingResult().getFieldErrors().stream()
                .map(e -> Map.of("field", e.getField(),
                    "message", e.getDefaultMessage()))
                .toList());
        return ResponseEntity.badRequest().body(pd);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ProblemDetail> handleUnexpected(
            Exception ex) {
        log.error("[{}] Unexpected error",
            MDC.get("traceId"), ex);
        String detail = showDetail
            ? ex.getMessage()

        pd.setType(URI.create("/problems/internal-error"));
        pd.setProperty("traceId", MDC.get("traceId"));
        pd.setProperty("errorCode", "INTERNAL_ERROR");
        return ResponseEntity.status(500).body(pd);

// CircuitBreaker.java
@Component
public class CircuitBreaker {
    private enum State { CLOSED, OPEN, HALF_OPEN }

    private final AtomicReference<State> state =
        new AtomicReference<>(State.CLOSED);
    private final AtomicInteger failureCount =
        new AtomicInteger(0);
    private volatile long lastFailureTime = 0;

    private static final int FAILURE_THRESHOLD = 3;
    private static final long TIMEOUT_MS = 30000;
    private static final Logger log =
        LoggerFactory.getLogger(CircuitBreaker.class);

    public <T> T execute(Supplier<T> action,
            Supplier<T> fallback) {
        if (state.get() == State.OPEN) {
            if (System.currentTimeMillis() - lastFailureTime
                    > TIMEOUT_MS) {
                state.set(State.HALF_OPEN);
                log.info("Circuit: OPEN → HALF_OPEN");
            } else {
                log.warn("Circuit OPEN: returning fallback");
                return fallback.get();
            }
        }
        try {
            T result = action.get();
            if (state.get() == State.HALF_OPEN) {
                state.set(State.CLOSED);
                failureCount.set(0);
                log.info("Circuit: HALF_OPEN → CLOSED");
            }
            failureCount.set(0);
            return result;
        } catch (Exception ex) {
            int count = failureCount.incrementAndGet();
            lastFailureTime = System.currentTimeMillis();
            if (count >= FAILURE_THRESHOLD) {
                state.set(State.OPEN);
                log.error("Circuit: → OPEN ({}回連続失敗)",
                    count);
            }
            if (state.get() == State.HALF_OPEN) {
                state.set(State.OPEN);
                log.warn("Circuit: HALF_OPEN → OPEN (再失敗)");
            }
            return fallback.get();
        }
    }
}

// RateLimitFilter.java
@Component
public class RateLimitFilter extends OncePerRequestFilter {
    private final ConcurrentHashMap<String, AtomicInteger>
        requestCounts = new ConcurrentHashMap<>();
    private static final int MAX_REQUESTS = 60;

    @Override
    protected void doFilterInternal(HttpServletRequest req,
            HttpServletResponse res, FilterChain chain)
            throws ServletException, IOException {
        String ip = req.getRemoteAddr();
        AtomicInteger count = requestCounts
            .computeIfAbsent(ip, k -> new AtomicInteger(0));
        if (count.incrementAndGet() > MAX_REQUESTS) {
            res.setStatus(429);
            res.setContentType("application/problem+json");
            res.getWriter().write("{\"type\":\"/problems/rate-limit\","
                + "\"status\":429,"
                + "\"detail\":\"リクエスト数の上限を超えました\"}");
            return;
        }
        res.setHeader("X-RateLimit-Remaining",
            String.valueOf(MAX_REQUESTS - count.get()));
        chain.doFilter(req, res);
    }

    @Scheduled(fixedRate = 60000)
    public void resetCounts() { requestCounts.clear(); }
}

// TodoService.java
@Service
public class TodoService {
    private final TodoMapper todoMapper;

    private static final Map<String, Set<String>> TRANSITIONS = Map.of(
        "OPEN", Set.of("IN_PROGRESS"),
        "IN_PROGRESS", Set.of("OPEN", "DONE"),
        "DONE", Set.of("IN_PROGRESS")
    );

    @Transactional
    public TodoResponse create(TodoForm form, Long userId) {
        Todo todo = new Todo();
        todo.setTitle(form.getTitle());
        todo.setDescription(form.getDescription());
        todo.setStatus("OPEN");
        todo.setPriority(form.getPriority());
        todo.setDueDate(form.getDueDate());
        todo.setUserId(userId);
        todo.setCreatedAt(LocalDateTime.now());
        todoMapper.insert(todo);
        return TodoResponse.from(todo);
    }

    public TodoResponse findById(Long id, Long userId) {
        Todo todo = todoMapper.findById(id);
        if (todo == null) throw new EntityNotFoundException("Todo", id);
        if (!todo.getUserId().equals(userId))
            throw new ForbiddenException("自分のTodoのみ閲覧できます");
        return TodoResponse.from(todo);
    }

    @Transactional
    public TodoResponse updateStatus(Long id, String newStatus,
            Long userId) {
        Todo todo = todoMapper.findById(id);
        if (todo == null) throw new EntityNotFoundException("Todo", id);
        if (!todo.getUserId().equals(userId))
            throw new ForbiddenException("自分のTodoのみ更新できます");
        Set<String> allowed = TRANSITIONS.getOrDefault(
            todo.getStatus(), Set.of());
        if (!allowed.contains(newStatus))
            throw new BusinessRuleViolationException(
                "STATUS_TRANSITION",
                todo.getStatus() + " → " + newStatus + " は不可");
        todoMapper.updateStatus(id, newStatus);
        return TodoResponse.from(todoMapper.findById(id));
    }

    public PageResponse<TodoResponse> search(Long userId,
            String status, Integer priority,
            int page, int size) {
        int offset = page * size;
        List<TodoResponse> content = todoMapper
            .search(userId, status, priority, size, offset)
            .stream().map(TodoResponse::from).toList();
        long total = todoMapper.countSearch(
            userId, status, priority);
        return new PageResponse<>(content, page, size, total);
    }
}

@Service
public class ArticleService {
    private final ArticleMapper articleMapper;
    private final TagMapper tagMapper;
    private final ArticleTagMapper articleTagMapper;

    @Transactional
    public ArticleResponse create(ArticleForm form, Long authorId) {
        Article article = new Article();
        article.setTitle(form.getTitle());
        article.setContent(form.getContent());
        article.setAuthorId(authorId);
        article.setStatus("DRAFT");
        article.setCreatedAt(LocalDateTime.now());
        articleMapper.insert(article);

        // タグの関連付け
        if (form.getTagIds() != null) {
            for (Long tagId : form.getTagIds()) {
                if (tagMapper.findById(tagId) == null)
                    throw new EntityNotFoundException("Tag", tagId);
                articleTagMapper.insert(article.getId(), tagId);
            }
        }
        return toResponse(articleMapper.findByIdWithTags(
            article.getId()));
    }

    @PreAuthorize("hasRole('ADMIN') or "
        + "@articleService.isAuthor(#id, authentication.name)")
    @Transactional
    public ArticleResponse publish(Long id) {
        Article article = articleMapper.findById(id);
        if (article == null)
            throw new EntityNotFoundException("Article", id);
        if (!"DRAFT".equals(article.getStatus()))
            throw new BusinessRuleViolationException(
                "PUBLISH_RULE",
                "DRAFT状態の記事のみ公開できます");
        articleMapper.updateStatus(id, "PUBLISHED");
        article.setUpdatedAt(LocalDateTime.now());
        return toResponse(articleMapper.findByIdWithTags(id));
    }

    public PageResponse<ArticleResponse> searchPublished(
            String tag, String keyword, String sort,
            int page, int size) {
        int offset = page * size;
        List<Article> articles = articleMapper
            .searchPublished(tag, keyword, sort, size, offset);
        long total = articleMapper.countSearchPublished(
            tag, keyword);
        return new PageResponse<>(
            articles.stream().map(this::toResponse).toList(),
            page, size, total);
    }

    public List<ArticleResponse> getPopular(int limit) {
        return articleMapper.findPopularByCommentCount(limit)
            .stream().map(this::toResponse).toList();
    }
}

@Service
public class CommentService {
    private final CommentMapper commentMapper;
    private final ArticleMapper articleMapper;

    @Transactional
    public CommentResponse create(Long articleId,
            CommentForm form, Long userId) {
        Article article = articleMapper.findById(articleId);
        if (article == null)
            throw new EntityNotFoundException("Article", articleId);
        if (!"PUBLISHED".equals(article.getStatus()))
            throw new BusinessRuleViolationException(
                "COMMENT_RULE",
                "公開済みの記事のみコメントできます");
        Comment comment = new Comment();
        comment.setArticleId(articleId);
        comment.setUserId(userId);
        comment.setContent(form.getContent());
        comment.setCreatedAt(LocalDateTime.now());
        commentMapper.insert(comment);
        return CommentResponse.from(comment);
    }

    @Transactional
    public void delete(Long commentId, Long userId, boolean isAdmin) {
        Comment comment = commentMapper.findById(commentId);
        if (comment == null)
            throw new EntityNotFoundException("Comment", commentId);
        if (!comment.getUserId().equals(userId) && !isAdmin)
            throw new ForbiddenException(
                "コメント作成者またはADMINのみ削除できます");
        commentMapper.deleteById(commentId);
    }
}

解説
  • 複数ドメイン（Article, Comment, Tag）を横断するAPIでは、ドメイン間のルール（「公開済み記事のみコメント可」等）をServiceで明示的にチェックする設計が重要です。
  • 多対多関係（Article-Tag）は中間テーブルで管理し、MyBatisの resultMap + collection タグでJOIN結果をネストしたオブジェクトにマッピングします。
  • ロール設計（ADMIN/AUTHOR/READER）により、操作権限を段階的に制御します。記事作成はAUTHOR以上、管理操作はADMINのみ許可するのが一般的です。
  • この課題を完成させることで「Spring Bootで実務レベルの複数ドメインAPIを設計・実装・テストできる」スキルが身につきます。
