S-01
T1 変数・データ型・演算子 模擬試験
var型推論と複合代入演算子の暗黙キャスト
emoji_events
Silver 頻出ポイント

複合代入演算子（+=, -= 等）は暗黙のキャストを含む。var の型推論は初期化式から決まり、再代入では型が変わらない。

quiz 問題

次のコードをコンパイル・実行した結果として正しいものはどれか。

public class CompoundAssign {
    public static void main(String[] args) {
        short s = 10;
        s += 3.5;           // 行A
        System.out.println("s = " + s);

        short t = 10;
        t = t + 3.5;        // 行B
        System.out.println("t = " + t);
    }
}
  • A. s = 13 と t = 13 が出力される
  • B. s = 13.5 と t = 13.5 が出力される
  • C. s = 13 が出力された後、行Bでコンパイルエラーになる
  • D. 行Aと行Bの両方でコンパイルエラーになる
lightbulb ヒント

s += 3.5; は s = (short)(s + 3.5); と等価です。複合代入演算子には暗黙のキャストが含まれます。一方、t = t + 3.5; では t + 3.5 の結果が double に拡張され、short に暗黙変換できません。

check_circle 解答を見る

正解: C

解説:

s += 3.5; — 複合代入演算子は内部的に s = (short)(s + 3.5); と同じ処理を行います。s + 3.5 は double の 13.5 になりますが、(short) にキャストされて 13 が代入されます。コンパイルエラーにはなりません。

t = t + 3.5; — こちらは通常の加算です。t + 3.5 の結果は double 型の 13.5 になります。double から short への暗黙の縮小変換はできないため、コンパイルエラーになります。

JLS 15.26.2 より、複合代入 E1 op= E2 は E1 = (T)(E1 op E2) と等価です（T は E1 の型）。これはSilverで頻出の引っかけポイントです。

code 課題

以下の仕様を満たす単位変換ライブラリを作成してください。

  1. enum Unit に CM, M, KM, INCH, FEET を定義し、各定数に「メートル換算係数」を持たせる
  2. Converter クラスに double value と Unit unit を持たせる
  3. Converter of(double value, Unit unit) — ファクトリメソッド
  4. Converter to(Unit target) — 変換メソッド（メソッドチェーン対応: return this;）
  5. double getValue() — 現在の値を返す

使用例:

double result = Converter.of(100, Unit.CM)
                         .to(Unit.M)
                         .getValue();
// result = 1.0

double result2 = Converter.of(1, Unit.KM)
                          .to(Unit.FEET)
                          .getValue();
// result2 = 3280.8398...
lightbulb ヒント

変換の考え方: まず現在の単位からメートルに変換し、次にメートルから目標単位に変換します。enum のコンストラクタで toMeter 係数を受け取ると整理しやすくなります。

check_circle 解答を見る

模範解答:

public class UnitConverter {

    enum Unit {
        CM(0.01), M(1.0), KM(1000.0),
        INCH(0.0254), FEET(0.3048);

        private final double toMeter;

        Unit(double toMeter) {
            this.toMeter = toMeter;
        }

        double getToMeter() {
            return toMeter;
        }
    }

    static class Converter {
        private double value;
        private Unit unit;

        private Converter(double value, Unit unit) {
            this.value = value;
            this.unit = unit;
        }

        static Converter of(double value, Unit unit) {
            return new Converter(value, unit);
        }

        Converter to(Unit target) {
            // 現在値をメートルに変換し、目標単位に変換
            double inMeters = this.value * this.unit.getToMeter();
            this.value = inMeters / target.getToMeter();
            this.unit = target;
            return this;
        }

        double getValue() {
            return value;
        }
    }

    public static void main(String[] args) {
        double r1 = Converter.of(100, Unit.CM)
                             .to(Unit.M)
                             .getValue();
        System.out.println("100 CM = " + r1 + " M");

        double r2 = Converter.of(1, Unit.KM)
                             .to(Unit.FEET)
                             .getValue();
        System.out.println("1 KM = " + r2 + " FEET");

        double r3 = Converter.of(5, Unit.FEET)
                             .to(Unit.INCH)
                             .to(Unit.CM)
                             .getValue();
        System.out.println("5 FEET = " + r3 + " CM");
    }
}

ポイント:

・enum のコンストラクタは暗黙的に private であり、public は指定できない

・メソッドチェーンは return this; で自分自身を返すことで実現

・ファクトリメソッド of() は static で定義し、コンストラクタを private にすることで生成方法を制御

String#concat() は新しい String を返し、元のオブジェクトは不変。== はインスタンスの同一性、equals() は値の等価性を比較する。

quiz 問題

次のコードの出力として正しいものはどれか。

public class StringCompare {
    public static void main(String[] args) {
        String s1 = "Hello";
        String s2 = s1.concat(" World");
        String s3 = s1 + " World";
        String s4 = "Hello World";

        System.out.println(s1);               // 行1
        System.out.println(s2 == s4);          // 行2
        System.out.println(s3 == s4);          // 行3
        System.out.println(s2.equals(s3));     // 行4

        StringBuilder sb = new StringBuilder("Test");
        sb.append("!");
        String s5 = sb.toString();
        String s6 = "Test!";
        System.out.println(s5 == s6);          // 行5
    }
}
  • A. Hello / false / false / true / false
  • B. Hello World / false / false / true / false
  • C. Hello / true / false / true / true
  • D. Hello / false / true / true / false
lightbulb ヒント

concat() は元の String を変更しません（Stringは不変）。s1 は "Hello" のまま。== で true になるのは同じ String Pool のインスタンスを指す場合のみです。concat() や StringBuilder.toString() はヒープに新しいオブジェクトを生成します。

check_circle 解答を見る

正解: A

解説:

行1: s1.concat(" World") は新しい String を返すだけで、s1 自体は変更されません。よって s1 は "Hello" のまま。

行2: s2 は concat() で生成されたヒープ上の新オブジェクト。s4 は文字列リテラルで String Pool のインスタンス。参照が異なるため false。

行3: s3 は s1 + " World" で生成。変数を含む + 連結はコンパイル時定数にならないため、ヒープ上に新オブジェクトが作られます。false。

行4: s2 と s3 は共に "Hello World" という値を持つので equals() は true。

行5: StringBuilder.toString() はヒープ上に新しい String を生成するため、String Pool の s6 とは別インスタンス。false。

emoji_events
Silver 頻出ポイント

StringBuilder の append(), indexOf(), replace(), delete() メソッドの使い分け。String の charAt(), substring() による文字列解析。

code 課題

以下のMarkdown記法をHTMLタグに変換するプログラムを StringBuilder を使って作成してください。

  • **太字** → <b>太字</b>
  • *斜体* → <i>斜体</i>
  • `コード` → <code>コード</code>

注意: ** と * の区別が必要です（** を先に処理）。

使用例:

String input = "これは**重要**な*ポイント*です。`System.out`を使います。";
String result = MarkdownParser.parse(input);
// result = "これは<b>重要</b>な<i>ポイント</i>です。<code>System.out</code>を使います。"
lightbulb ヒント

変換の順序がポイントです。**（2文字）を先に処理してから *（1文字）を処理しないと、**太字** が <i><i>太字</i></i> と誤変換されます。StringBuilder の indexOf() で開始位置を見つけ、replace() で置換するループを書きましょう。

check_circle 解答を見る

模範解答:

public class MarkdownParser {

    static String parse(String input) {
        StringBuilder sb = new StringBuilder(input);

        // 1. **太字** → <b>...</b>（2文字デリミタを先に処理）
        replaceMarkup(sb, "**", "<b>", "</b>");

        // 2. *斜体* → <i>...</i>
        replaceMarkup(sb, "*", "<i>", "</i>");

        // 3. `コード` → <code>...</code>
        replaceMarkup(sb, "`", "<code>", "</code>");

        return sb.toString();
    }

    private static void replaceMarkup(StringBuilder sb,
            String delim, String openTag, String closeTag) {
        int start;
        while ((start = sb.indexOf(delim)) != -1) {
            int end = sb.indexOf(delim, start + delim.length());
            if (end == -1) break; // 閉じデリミタがなければ終了

            // 閉じデリミタを先に置換（位置がずれないように）
            sb.replace(end, end + delim.length(), closeTag);
            // 開きデリミタを置換
            sb.replace(start, start + delim.length(), openTag);
        }
    }

    public static void main(String[] args) {
        String input = "これは**重要**な*ポイント*です。`System.out`を使います。";
        System.out.println(parse(input));
        // これは<b>重要</b>な<i>ポイント</i>です。<code>System.out</code>を使います。

        String input2 = "**A**と**B**は*別物*";
        System.out.println(parse(input2));
        // <b>A</b>と<b>B</b>は<i>別物</i>
    }
}

ポイント:

・StringBuilder.indexOf(String, int) で開始位置を指定して検索できる

・replace(int start, int end, String str) は指定範囲を置換する（String.replace() とは異なる）

・閉じデリミタを先に置換することで、開きタグ挿入による位置ズレの影響を受けない

T3 配列 模擬試験
Arrays.sort / copyOf / asList の挙動トラップ
emoji_events
Silver 頻出ポイント

Arrays.asList() で得たリストは固定サイズ。Arrays.copyOf() は新しい配列を返す（シャローコピー）。Arrays.sort() は元の配列を変更する。

quiz 問題

次のコードをコンパイル・実行した結果として正しいものはどれか。

import java.util.*;

public class ArrayTraps {
    public static void main(String[] args) {
        int[] original = {5, 3, 1, 4, 2};
        int[] copied = Arrays.copyOf(original, 3);
        Arrays.sort(original);

        System.out.println(Arrays.toString(copied));   // 行1
        System.out.println(Arrays.toString(original));  // 行2

        String[] names = {"C", "A", "B"};
        List<String> list = Arrays.asList(names);
        Collections.sort(list);
        System.out.println(names[0]);                  // 行3

        list.add("D");                                 // 行4
    }
}
  • A. [5, 3, 1] / [1, 2, 3, 4, 5] / A / 行4で UnsupportedOperationException
  • B. [1, 2, 3] / [1, 2, 3, 4, 5] / A / 行4で UnsupportedOperationException
  • C. [5, 3, 1] / [1, 2, 3, 4, 5] / C / 行4は正常に動作
  • D. [5, 3, 1] / [1, 2, 3, 4, 5] / A / 行4で IndexOutOfBoundsException
lightbulb ヒント

Arrays.copyOf() はコピー時点の値を新しい配列にコピーします。コピー後に元配列をソートしても、コピーには影響しません。Arrays.asList() は配列のビューを返すため、リストの変更は元配列に反映されますが、サイズ変更（add/remove）はできません。

check_circle 解答を見る

正解: A

解説:

行1: copyOf(original, 3) はコピー時点の先頭3要素 [5, 3, 1] を新配列にコピー。その後の sort(original) は copied に影響しない。

行2: sort() は元配列を破壊的にソートするため [1, 2, 3, 4, 5]。

行3: Arrays.asList(names) は names 配列のビューを返します。Collections.sort(list) はリストをソートしますが、このリストは元配列のビューなので、names 配列自体も ["A", "B", "C"] にソートされます。よって names[0] は "A"。

行4: Arrays.asList() が返すリストは固定サイズです。add() を呼ぶと UnsupportedOperationException がスローされます。

code 課題

int[9][9] の数独盤面を受け取り、以下の3つのルールをすべて満たすかを検証するメソッドを作成してください。

  1. 各行に 1〜9 が重複なく含まれる
  2. 各列に 1〜9 が重複なく含まれる
  3. 各 3×3 ブロックに 1〜9 が重複なく含まれる

メソッドシグネチャ: static boolean isValidSudoku(int[][] board)

0 は空欄を表し、検証対象外とします（1〜9のみ重複チェック）。

lightbulb ヒント

重複チェックには boolean[10]（インデックス1〜9を使用）が便利です。3×3ブロックの開始位置は (row / 3) * 3 と (col / 3) * 3 で計算できます。行・列・ブロックの3種類のチェックを別メソッドに分けると整理しやすくなります。

check_circle 解答を見る

模範解答:

public class SudokuValidator {

    static boolean isValidSudoku(int[][] board) {
        // 行チェック
        for (int r = 0; r < 9; r++) {
            boolean[] seen = new boolean[10];
            for (int c = 0; c < 9; c++) {
                int val = board[r][c];
                if (val == 0) continue;
                if (seen[val]) return false;
                seen[val] = true;
            }
        }
        // 列チェック
        for (int c = 0; c < 9; c++) {
            boolean[] seen = new boolean[10];
            for (int r = 0; r < 9; r++) {
                int val = board[r][c];
                if (val == 0) continue;
                if (seen[val]) return false;
                seen[val] = true;
            }
        }
        // 3x3ブロックチェック
        for (int br = 0; br < 3; br++) {
            for (int bc = 0; bc < 3; bc++) {
                boolean[] seen = new boolean[10];
                for (int r = br * 3; r < br * 3 + 3; r++) {
                    for (int c = bc * 3; c < bc * 3 + 3; c++) {
                        int val = board[r][c];
                        if (val == 0) continue;
                        if (seen[val]) return false;
                        seen[val] = true;
                    }
                }
            }
        }
        return true;
    }

    public static void main(String[] args) {
        int[][] valid = {
            {5,3,4, 6,7,8, 9,1,2},
            {6,7,2, 1,9,5, 3,4,8},
            {1,9,8, 3,4,2, 5,6,7},
            {8,5,9, 7,6,1, 4,2,3},
            {4,2,6, 8,5,3, 7,9,1},
            {7,1,3, 9,2,4, 8,5,6},
            {9,6,1, 5,3,7, 2,8,4},
            {2,8,7, 4,1,9, 6,3,5},
            {3,4,5, 2,8,6, 1,7,9}
        };
        System.out.println("Valid: " + isValidSudoku(valid));   // true

        int[][] invalid = {
            {5,3,4, 6,7,8, 9,1,2},
            {6,7,2, 1,9,5, 3,4,8},
            {1,9,8, 3,4,2, 5,6,7},
            {8,5,9, 7,6,1, 4,2,3},
            {4,2,6, 8,5,3, 7,9,1},
            {7,1,3, 9,2,4, 8,5,6},
            {9,6,1, 5,3,7, 2,8,4},
            {2,8,7, 4,1,9, 6,3,5},
            {3,4,5, 2,8,6, 1,7,5}  // 最後の行に5が重複
        };
        System.out.println("Invalid: " + isValidSudoku(invalid)); // false
    }
}

T4 フロー制御 模擬試験
switch式（arrow構文・yield）とラベル付きbreak
emoji_events
Silver 頻出ポイント

Java 14+ の switch 式では -> 構文が使え、yield でブロックから値を返す。ラベル付き break は外側のループを脱出する。

quiz 問題

次のコードの出力として正しいものはどれか。

public class SwitchAndLabel {
    public static void main(String[] args) {
        // Part 1: switch式
        String day = "WEDNESDAY";
        int numLetters = switch (day) {
            case "MONDAY", "FRIDAY", "SUNDAY" -> 6;
            case "TUESDAY" -> 7;
            case "WEDNESDAY" -> {
                System.out.print("Mid-week: ");
                yield 9;
            }
            case "THURSDAY", "SATURDAY" -> 8;
            default -> throw new IllegalArgumentException();
        };
        System.out.println(numLetters);

        // Part 2: ラベル付きbreak
        int count = 0;
        outer:
        for (int i = 0; i < 3; i++) {
            for (int j = 0; j < 3; j++) {
                if (i == 1 && j == 1) break outer;
                count++;
            }
        }
        System.out.println("count=" + count);
    }
}
  • A. Mid-week: 9 / count=4
  • B. Mid-week: 9 / count=5
  • C. 9 / count=4
  • D. Mid-week: 9 / count=3
lightbulb ヒント

switch式の -> 構文でブロック {} を使う場合、値を返すには yield を使います。System.out.print() の後に yield 9; があるので、"Mid-week: " が先に出力されます。ラベル付き break outer; は外側ループ自体を終了します。i=0 で j=0,1,2（3回）、i=1 で j=0（1回）、j=1 で break outer → 合計4回。

check_circle 解答を見る

正解: A

解説:

Part 1: day は "WEDNESDAY" なので、case "WEDNESDAY" のブロックに入ります。System.out.print("Mid-week: ") で改行なしに出力した後、yield 9; で switch 式の値として 9 を返します。その後 System.out.println(numLetters) で 9 が出力されるため、合わせて Mid-week: 9 と表示されます。

Part 2: ループの実行を追跡します。

  • i=0, j=0: count=1
  • i=0, j=1: count=2
  • i=0, j=2: count=3
  • i=1, j=0: count=4
  • i=1, j=1: break outer; で外側ループ終了

よって count=4 が出力されます。ラベル付き break は指定されたラベルのループ全体を脱出します。ラベルなしの break だと内側ループだけを脱出し、外側は継続します。

以下のコマンドを処理する対話式インタプリタを作成してください。

  • add 3 5 → 結果: 8 と表示し、履歴に追加
  • sub 10 3 → 結果: 7 と表示し、履歴に追加
  • mul 4 6 → 結果: 24 と表示し、履歴に追加

Scanner.nextLine() で1行読み取り、String.split(" ") でトークン分割します。コマンド名を switch で分岐し、quit の場合はフラグ変数を使うか、while ループの条件で制御します。履歴は ArrayList<String> で管理するのが簡単です。

模範解答:

import java.util.ArrayList;
import java.util.List;
import java.util.Scanner;

public class CommandInterpreter {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        List<String> history = new ArrayList<>();
        boolean running = true;

        System.out.println("=== 計算インタプリタ ===");
        System.out.println("コマンド: add/sub/mul/div NUM NUM, list, clear, quit");

        while (running) {
            System.out.print("> ");
            String line = sc.nextLine().trim();
            if (line.isEmpty()) continue;

            String[] tokens = line.split(" ");
            String cmd = tokens[0].toLowerCase();

            switch (cmd) {
                case "add": case "sub": case "mul": case "div":
                    if (tokens.length < 3) {
                        System.out.println("エラー: 数値を2つ指定してください");
                        continue;
                    }
                    try {
                        double a = Double.parseDouble(tokens[1]);
                        double b = Double.parseDouble(tokens[2]);
                        double result;
                        switch (cmd) {
                            case "add": result = a + b; break;
                            case "sub": result = a - b; break;
                            case "mul": result = a * b; break;
                            case "div":
                                if (b == 0) {
                                    System.out.println("エラー: 0で割れません");
                                    continue;
                                }
                                result = a / b;
                                break;
                            default: result = 0;
                        }
                        String entry = cmd + " " + a + " " + b
                                       + " = " + result;
                        history.add(entry);
                        System.out.println("結果: " + result);
                    } catch (NumberFormatException e) {
                        System.out.println("エラー: 数値の形式が不正です");
                    }
                    break;

                case "list":
                    if (history.isEmpty()) {
                        System.out.println("(履歴なし)");
                    } else {
                        for (int i = 0; i < history.size(); i++) {
                            System.out.println((i + 1) + ". "
                                               + history.get(i));
                        }
                    }
                    break;

                case "clear":
                    history.clear();
                    System.out.println("履歴をクリアしました");
                    break;

                case "quit":
                    running = false;
                    System.out.println("終了します");
                    break;

                default:
                    System.out.println("不明なコマンドです");
                    continue;
            }
        }
        sc.close();
    }
}

ポイント:

・while (running) + フラグ変数でメインループを制御。quit で running = false

・continue はエラー時にループの先頭に戻すために使用（履歴への追加をスキップ）

・switch 内の switch（ネスト）で演算を分岐。内側の break は内側の switch のみ脱出

・Scanner.close() でリソースを解放（Silver では try-with-resources も問われる）

lightbulb ヒント

staticイニシャライザはクラスが最初にロードされた時点で実行されます。main() メソッドを持つクラスのロード前に、参照されるクラスのロードが行われます。new Child() の呼出しで Parent と Child の両方がロードされ、static{} が親→子の順で実行されます。その後インスタンス生成時にインスタンス{} → コンストラクタの順で実行されます。

正解: B

1. クラスロード（staticイニシャライザ）: new Child() が呼ばれる前に、Parent と Child のクラスがロードされます。staticイニシャライザは親→子の順で実行されるため、A（Parent static）→ E（Child static）が出力されます。これは main() の "Start" より先に実行されます。

2. main() の実行: Start が出力されます。

3. new Child() の実行:

  • Child コンストラクタが super("hello") を呼ぶ

  • Parent(String) は this() で Parent() を呼ぶ → C
  • Parent(String) の残り → D
  • Child のインスタンスイニシャライザが実行 → F
  • Child コンストラクタの残り → G

class Product {

    private int price;

    Product(String name, int price) {
        this.name = name;
        this.price = price;
    }

    String getName()  { return name; }
    int getPrice()    { return price; }

    @Override
    public String toString() {
        return name + "(" + price + "円)";
    }
}

class CartItem {
    private Product product;
    private int quantity;

    CartItem(Product product, int quantity) {
        this.product = product;
        this.quantity = quantity;
    }

    Product getProduct() { return product; }
    int getQuantity()    { return quantity; }
    void addQuantity(int n) { this.quantity += n; }

    int getSubtotal() {
        return product.getPrice() * quantity;
    }

    @Override
    public String toString() {
        return product.getName() + " x" + quantity
               + " = " + getSubtotal() + "円";
    }
}

class Cart {
    private ArrayList<CartItem> items = new ArrayList<>();
    private static int orderCount = 0;

    void addItem(Product p, int qty) {
        // 同一商品があれば数量を加算
        for (CartItem item : items) {
            if (item.getProduct().getName().equals(p.getName())) {
                item.addQuantity(qty);
                return;
            }
        }
        // なければ新規追加
        items.add(new CartItem(p, qty));
    }

    void removeItem(String productName) {
        Iterator<CartItem> it = items.iterator();
        while (it.hasNext()) {
            if (it.next().getProduct().getName()
                    .equals(productName)) {
                it.remove();
                return;
            }
        }
        System.out.println(productName + " はカートにありません");
    }

    int getTotal() {
        int total = 0;
        for (CartItem item : items) {
            total += item.getSubtotal();
        }
        return total;
    }

    void printReceipt() {
        orderCount++;
        System.out.println("===== レシート #" + orderCount
                           + " =====");
        for (CartItem item : items) {
            System.out.println("  " + item);
        }
        System.out.println("  ─────────────────");
        System.out.println("  合計: " + getTotal() + "円");
        System.out.println("====================");
    }

    static int getOrderCount() {
        return orderCount;
    }
}

        Product apple  = new Product("りんご", 150);
        Product bread  = new Product("食パン", 250);
        Product milk   = new Product("牛乳", 200);

        Cart cart = new Cart();
        cart.addItem(apple, 3);

        System.out.println("合計: " + cart.getTotal() + "円");

interface Flyable {
    default String move() { return "fly"; }
}

interface Swimmable {
    default String move() { return "swim"; }
}

abstract class Animal {
    abstract String sound();
}

class Duck extends Animal implements Flyable, Swimmable {
    @Override
    String sound() { return "quack"; }

    @Override
    public String move() {
        return Flyable.super.move() + "&" + Swimmable.super.move();
    }
}

public class PolyTest {
    public static void main(String[] args) {
        Animal a = new Duck();
        System.out.println(a.sound());
        System.out.println(a instanceof Flyable);

        Flyable f = (Flyable) a;
        System.out.println(f.move());

        Swimmable s = (Swimmable) a;
        System.out.println(s.move());

        // System.out.println(((String) a)); // 行X
    }
}
  • A. quack / true / fly / swim
  • B. quack / true / fly&swim / fly&swim

lightbulb ヒント

Duck は move() をオーバーライドしており、Flyable.super.move() と Swimmable.super.move() を両方呼んでいます。これによりdefaultメソッドの衝突は解決済みです。f.move() も s.move() も実際のオブジェクト（Duck）の move() が呼ばれます（ポリモーフィズム）。

check_circle 解答を見る

正解: B

解説:

a.sound(): a の実体は Duck なので "quack" が返されます。

a instanceof Flyable: Duck は Flyable を実装しているため true。変数の型（Animal）ではなく、実際のオブジェクトの型で判定されます。

f.move(): 変数の型は Flyable ですが、実体は Duck です。Duck の move() が呼ばれ、"fly&swim" が返されます。Flyable の default メソッドが呼ばれるわけではありません。

s.move(): 同様に Duck の move() が呼ばれ、"fly&swim" が返されます。参照型が Swimmable でも結果は同じです。

行X について: (String) a は Animal から String へのキャストで、継承関係がないためコンパイルエラーになります。一方、インタフェースへのキャストはコンパイル時にはチェックされず、実行時に ClassCastException になる場合があります。

選択肢Aの誤り: f.move() が "fly" を返すと考えるのは、ポリモーフィズムを理解していない誤りです。オーバーライドされたメソッドは、参照型に関係なく実体のメソッドが呼ばれます。

模範解答:

import java.util.ArrayList;
import java.util.List;

interface Plugin {
    String getName();
    void execute(String input);
}

abstract class AbstractPlugin implements Plugin {
    // テンプレートメソッド: ログ付きで実行
    void run(String input) {
        System.out.println("[" + getName() + "] 開始");
        execute(input);
        System.out.println("[" + getName() + "] 完了");
    }
}

class TextPlugin extends AbstractPlugin {
    @Override
    public String getName() { return "TextPlugin"; }

    @Override
    public void execute(String input) {
        System.out.println("  文字数: " + input.length());
    }
}

class ImagePlugin extends AbstractPlugin {
    @Override
    public String getName() { return "ImagePlugin"; }

    @Override
    public void execute(String input) {
        System.out.println("  出力ファイル: " + input + ".png");
    }
}

class AudioPlugin extends AbstractPlugin {
    @Override
    public String getName() { return "AudioPlugin"; }

    @Override
    public void execute(String input) {
        System.out.println("  ♪ " + input);
    }
}

class PluginManager {
    private List<Plugin> plugins = new ArrayList<>();

    void registerPlugin(Plugin p) {
        plugins.add(p);
        System.out.println("登録: " + p.getName());
    }

    void executeAll(String input) {
        System.out.println("=== 全プラグイン実行 ===");
        for (Plugin p : plugins) {
            if (p instanceof AbstractPlugin) {
                // テンプレートメソッド経由で実行（ログ付き）
                ((AbstractPlugin) p).run(input);
            } else {
                // AbstractPluginを継承していない場合
                p.execute(input);
            }
        }
        System.out.println("=== 実行完了 ===");
    }
}

public class PluginSystemApp {
    public static void main(String[] args) {
        PluginManager manager = new PluginManager();

        manager.registerPlugin(new TextPlugin());
        manager.registerPlugin(new ImagePlugin());
        manager.registerPlugin(new AudioPlugin());

        manager.executeAll("HelloWorld");

        System.out.println();
        manager.executeAll("テスト入力");
    }
}

ポイント:

・interface → abstract class → 具象クラスの3層構造は実務でも頻出のパターン

・AbstractPlugin.run() がテンプレートメソッドとして機能し、共通のログ処理を一元管理

・PluginManager は List<Plugin> で管理するため、新しいプラグインを追加しても既存コードの変更が不要（開放閉鎖の原則）

・instanceof でキャスト前に型チェックすることで ClassCastException を防止

模範解答:

import java.util.ArrayList;
import java.util.List;

// カスタム例外
class ValidationException extends Exception {
    private final String fieldName;

    ValidationException(String fieldName, String message) {
        super(message);
        this.fieldName = fieldName;
    }

    String getFieldName() { return fieldName; }

    @Override
    public String toString() {
        return "[" + fieldName + "] " + getMessage();
    }
}

// バリデーション結果の集約クラス
class ValidationResult {
    private List<String> errors = new ArrayList<>();

    void addError(String message) {
        errors.add(message);
    }

    boolean hasErrors() {
        return !errors.isEmpty();
    }

    List<String> getErrors() {
        return errors;
    }

    void throwIfInvalid() throws ValidationException {
        if (hasErrors()) {
            String combined = String.join("; ", errors);
            throw new ValidationException("multiple",
                                          combined);
        }
    }
}

// バリデータインタフェース
interface Validator<T> {
    void validate(T value, ValidationResult result);
}

// 文字列バリデータ
class StringValidator implements Validator<String> {
    private final String fieldName;
    private final int maxLength;
    private final boolean required;

    StringValidator(String fieldName, int maxLength,
                    boolean required) {
        this.fieldName = fieldName;
        this.maxLength = maxLength;
        this.required = required;
    }

    @Override
    public void validate(String value,
                         ValidationResult result) {
        if (value == null || value.isEmpty()) {
            if (required) {
                result.addError(fieldName
                                + " は必須です");
            }
            return; // null/空なら以降のチェック不要
        }
        if (value.length() > maxLength) {
            result.addError(fieldName + " は"
                + maxLength + "文字以内にしてください"
                + "（現在" + value.length() + "文字）");
        }
    }
}

// 数値バリデータ
class NumberValidator implements Validator<Integer> {
    private final String fieldName;
    private final int min;
    private final int max;

    NumberValidator(String fieldName, int min, int max) {
        this.fieldName = fieldName;
        this.min = min;
        this.max = max;
    }

    @Override
    public void validate(Integer value,
                         ValidationResult result) {
        if (value == null) {
            result.addError(fieldName + " は必須です");
            return;
        }
        if (value < min || value > max) {
            result.addError(fieldName + " は" + min
                + "〜" + max + "の範囲で入力してください"
                + "（入力値: " + value + "）");
        }
    }
}

public class ValidationApp {
    public static void main(String[] args) {
        // バリデータを用意
        StringValidator nameValidator =
            new StringValidator("名前", 10, true);
        StringValidator emailValidator =
            new StringValidator("メール", 50, true);
        NumberValidator ageValidator =
            new NumberValidator("年齢", 0, 150);

        // --- ケース1: 正常な入力 ---
        System.out.println("=== ケース1: 正常入力 ===");
        ValidationResult r1 = new ValidationResult();
        nameValidator.validate("山田太郎", r1);
        emailValidator.validate("taro@example.com", r1);
        ageValidator.validate(25, r1);
        printResult(r1);

        // --- ケース2: エラーあり ---
        System.out.println("=== ケース2: エラーあり ===");
        ValidationResult r2 = new ValidationResult();
        nameValidator.validate("", r2);
        emailValidator.validate(
            "very-long-email-address-that-exceeds@limit.com",
            r2);
        ageValidator.validate(200, r2);
        printResult(r2);

        // --- ケース3: 例外スロー ---
        System.out.println("=== ケース3: 例外スロー ===");
        ValidationResult r3 = new ValidationResult();
        nameValidator.validate(null, r3);
        ageValidator.validate(-5, r3);
        try {
            r3.throwIfInvalid();
        } catch (ValidationException e) {
            System.out.println("例外捕捉: " + e);
        }
    }

    static void printResult(ValidationResult result) {
        if (result.hasErrors()) {
            System.out.println("エラー件数: "
                + result.getErrors().size());
            for (String err : result.getErrors()) {
                System.out.println("  ✗ " + err);
            }
        } else {
            System.out.println("バリデーションOK");
        }
        System.out.println();
    }
}

ポイント:

・ValidationException は Exception を継承した検査例外（checked exception）。呼び出し元で try-catch か throws が必須

・Validator<T> のジェネリクス型パラメータにより、String用・Integer用など型安全なバリデータを実装可能

・エラーを即座にスローせず ValidationResult に集約することで、全フィールドを一括チェックできる

・throwIfInvalid() で最終的に例外をスローするかどうかを制御できる柔軟な設計

quiz 問題

次のコードをコンパイル・実行した結果として正しいものはどれか。

import java.util.*;
import java.util.function.*;

public class FunctionalTest {
    public static void main(String[] args) {
        // Part 1: 関数型インタフェース
        Predicate<String> isLong = s -> s.length() > 3;
        Function<String, Integer> toLen = String::length;
        Consumer<String> printer = System.out::println;

        String word = "Hello";
        System.out.print(isLong.test(word) + " ");     // 行1
        System.out.print(toLen.apply(word) + " ");      // 行2
        // printer.accept(word);

        // Part 2: unmodifiableList
        List<String> original = new ArrayList<>();
        original.add("A");
        original.add("B");

        List<String> unmod =
            Collections.unmodifiableList(original);
        System.out.print(unmod.size() + " ");           // 行3

        original.add("C");
        System.out.print(unmod.size() + " ");           // 行4

        try {
            unmod.add("D");                             // 行5
        } catch (UnsupportedOperationException e) {
            System.out.print("blocked");                // 行6
        }
    }
}
  • A. true 5 2 2 blocked
  • B. true 5 2 3 blocked
  • C. false 5 2 3 blocked
  • D. true 5 2 2（行5で例外がスローされず正常終了）
lightbulb ヒント

"Hello".length() は 5。length() > 3 は true。String::length はインスタンスメソッド参照で、Function<String, Integer> に適合します。Collections.unmodifiableList() は元リストのビューを返すため、元リストへの変更は反映されます。ただし、ビュー自体への変更は UnsupportedOperationException になります。

check_circle 解答を見る

正解: B

解説:

行1: isLong.test("Hello") — "Hello".length() は 5。5 > 3 は true。

行2: toLen.apply("Hello") — String::length はインスタンスメソッド参照です。"Hello".length() が呼ばれ 5 が返されます。

行3: unmod.size() — この時点で original は ["A", "B"] の2要素。unmod はそのビューなのでサイズは 2。

行4: original.add("C") の後の unmod.size() — unmodifiableList() は元リストのビューを返します。元リストに "C" が追加されたため、ビューにも反映されてサイズは 3 になります。これが最大の引っかけポイントです。

行5-6: unmod.add("D") はビューへの変更操作なので UnsupportedOperationException がスローされ、"blocked" が出力されます。

メソッド参照の4種類:

  • String::length — インスタンスメソッド参照（任意のオブジェクト）
  • System.out::println — インスタンスメソッド参照（特定のオブジェクト）
  • Math::abs — staticメソッド参照
  • ArrayList::new — コンストラクタ参照

  3. EventScheduler クラス:

TreeMap はキーが自然順序（LocalDate は日付順）でソートされます。addEvent() では computeIfAbsent() を使うと、キーが存在しない場合に新しい ArrayList を自動生成できます。時間の重複判定は「イベントAの開始 < イベントBの終了 かつ イベントBの開始 < イベントAの終了」です。

import java.time.LocalDate;
import java.time.LocalDateTime;
import java.util.*;
import java.util.function.Predicate;
import java.util.stream.Collectors;

enum Category {
    MEETING, STUDY, EXERCISE, SOCIAL
}

class Event {
    private String name;
    private LocalDateTime start;
    private LocalDateTime end;
    private Category category;

    Event(String name, LocalDateTime start,
          LocalDateTime end, Category category) {
        this.name = name;
        this.start = start;
        this.end = end;
        this.category = category;
    }

    String getName()          { return name; }
    LocalDateTime getStart()  { return start; }
    LocalDateTime getEnd()    { return end; }
    Category getCategory()    { return category; }

    // 時間重複判定
    boolean overlapsWith(Event other) {
        return this.start.isBefore(other.end)
            && other.start.isBefore(this.end);
    }

    @Override
    public String toString() {
        return String.format("[%s] %s (%s〜%s)",
            category, name,
            start.toLocalTime(), end.toLocalTime());
    }
}

class EventScheduler {
    private TreeMap<LocalDate, List<Event>> schedule
        = new TreeMap<>();

    void addEvent(Event e) {
        LocalDate date = e.getStart().toLocalDate();
        schedule.computeIfAbsent(date,
            k -> new ArrayList<>()).add(e);
    }

    List<Event> getEventsOn(LocalDate date) {
        List<Event> events =
            schedule.getOrDefault(date, List.of());
        // 開始時刻でソートして返す
        List<Event> sorted = new ArrayList<>(events);
        sorted.sort(Comparator.comparing(
            Event::getStart));
        return sorted;
    }

    List<String> findConflicts() {
        List<String> conflicts = new ArrayList<>();
        for (List<Event> dayEvents
                : schedule.values()) {
            for (int i = 0; i < dayEvents.size(); i++) {
                for (int j = i + 1;
                     j < dayEvents.size(); j++) {
                    Event a = dayEvents.get(i);
                    Event b = dayEvents.get(j);
                    if (a.overlapsWith(b)) {
                        conflicts.add(a.getName()
                            + " ↔ " + b.getName());
                    }
                }
            }
        }
        return conflicts;
    }

    List<Event> filterByCategory(Category category) {
        // Predicateを使ったフィルタリング
        Predicate<Event> matcher =
            e -> e.getCategory() == category;

        List<Event> result = new ArrayList<>();
        for (List<Event> dayEvents
                : schedule.values()) {
            for (Event e : dayEvents) {
                if (matcher.test(e)) {
                    result.add(e);
                }
            }
        }
        result.sort(Comparator.comparing(
            Event::getStart));
        return result;
    }

    void printSchedule() {
        for (Map.Entry<LocalDate, List<Event>> entry
                : schedule.entrySet()) {
            System.out.println("--- " + entry.getKey()
                               + " ---");
            List<Event> sorted =
                getEventsOn(entry.getKey());
            for (Event e : sorted) {
                System.out.println("  " + e);
            }
        }
    }
}

public class SchedulerApp {
    public static void main(String[] args) {
        EventScheduler scheduler = new EventScheduler();
        LocalDate today = LocalDate.of(2026, 3, 7);

        // イベント追加
        scheduler.addEvent(new Event("朝会",
            today.atTime(9, 0), today.atTime(9, 30),
            Category.MEETING));
        scheduler.addEvent(new Event("Java勉強",
            today.atTime(10, 0), today.atTime(12, 0),
            Category.STUDY));
        scheduler.addEvent(new Event("ランチMTG",
            today.atTime(11, 30), today.atTime(13, 0),
            Category.MEETING));
        scheduler.addEvent(new Event("ジョギング",
            today.atTime(18, 0), today.atTime(19, 0),
            Category.EXERCISE));

        LocalDate tomorrow = today.plusDays(1);
        scheduler.addEvent(new Event("読書会",
            tomorrow.atTime(14, 0),
            tomorrow.atTime(16, 0),
            Category.STUDY));

        // 全スケジュール表示
        System.out.println("=== スケジュール一覧 ===");
        scheduler.printSchedule();

        // 特定日のイベント
        System.out.println("\n=== " + today
                           + " のイベント ===");
        for (Event e : scheduler.getEventsOn(today)) {
            System.out.println("  " + e);
        }

        // 重複検出
        System.out.println("\n=== 時間の重複 ===");
        List<String> conflicts =
            scheduler.findConflicts();
        if (conflicts.isEmpty()) {
            System.out.println("  重複なし");
        } else {
            for (String c : conflicts) {
                System.out.println("  ⚠ " + c);
            }
        }

        // カテゴリフィルタ
        System.out.println("\n=== MEETING のみ ===");
        for (Event e : scheduler.filterByCategory(
                Category.MEETING)) {
            System.out.println("  " + e);
        }
    }
}

ポイント:

・TreeMap はキーの自然順序（LocalDate は日付昇順）で自動ソートされるため、日付順の一覧表示が容易

・computeIfAbsent() でキーが存在しない場合のみ新しい ArrayList を生成する。ラムダ k -> new ArrayList<>() が簡潔

・Comparator.comparing(Event::getStart) はメソッド参照を使った Comparator 生成パターン

・Predicate<Event> を使ったフィルタリングは、条件を変数として扱えるため柔軟性が高い

・時間の重複判定 A.start < B.end && B.start < A.end は区間の重なり判定の定番アルゴリズム
