穴埋め — プリミティブ型の変換

verified
JavaSilverポイント

暗黙の型変換（拡大変換: int→double）は自動で行われますが、縮小変換（double→int）には明示的なキャストが必要です。char は内部的に Unicode の数値で管理されており、int との相互変換が可能です。

assignment課題

以下のコードの /* ① */ 〜 /* ③ */ を埋めて、プリミティブ型の変換プログラムを完成させてください。

public class TypeConversion {
    public static void main(String[] args) {
        // 1. int → double（暗黙の型変換）
        int intVal = 42;
        double doubleVal = /* ① */;
        System.out.println("int→double: " + intVal + " → " + doubleVal);

        // 2. double → int（明示的キャスト）
        double pi = 3.14159;
        int truncated = /* ② */;
        System.out.println("double→int: " + pi + " → " + truncated);

        // 3. int → char（文字コード変換）
        int code = 65;
        char letter = /* ③ */;
        System.out.println("int→char: " + code + " → " + letter);
    }
}
terminal期待出力
int→double: 42 → 42.0
double→int: 3.14159 → 3
int→char: 65 → A
lightbulbヒント

① int を double に代入するだけで暗黙変換されます。② (int) キャストを使います。小数部は切り捨てられます。③ (char) キャストで数値を文字に変換できます。65 は ASCII で 'A' です。

check_circle解答・解説 
public class TypeConversion {
    public static void main(String[] args) {
        // 1. int → double（暗黙の型変換）
        int intVal = 42;
        double doubleVal = intVal;  // ① 暗黙の拡大変換
        System.out.println("int→double: " + intVal + " → " + doubleVal);

        // 2. double → int（明示的キャスト）
        double pi = 3.14159;
        int truncated = (int) pi;  // ② 明示的な縮小変換
        System.out.println("double→int: " + pi + " → " + truncated);

        // 3. int → char（文字コード変換）
        int code = 65;
        char letter = (char) code;  // ③ int→char キャスト
        System.out.println("int→char: " + code + " → " + letter);
    }
}
解説
  • int → double は拡大変換（widening）のため、キャストなしで自動変換されます。情報の損失がないためコンパイラが許可します。
  • double → int は縮小変換（narrowing）のため、(int) キャストが必須です。小数部は四捨五入ではなく切り捨てられます。
  • char は内部的に 0〜65535 の符号なし整数値です。(char) 65 は Unicode コードポイント 65 の文字 'A' になります。
  • JavaSilverでは「どの型変換にキャストが必要か」を問う問題が頻出です。byte → short → int → long → float → double の順が拡大変換です。

写経+改造 — BMI計算プログラム

verified
JavaSilverポイント

int 同士の除算では小数部が切り捨てられます（例: 5 / 2 は 2）。double との混在演算では int が自動的に double に拡大変換されます。

assignment課題

Step 1（写経）: 以下のコードをそのまま写経して実行し、出力を確認してください。

public class BmiCalc {
    public static void main(String[] args) {
        double weightKg = 60.0;
        double heightM = 1.65;

        double bmi = weightKg / (heightM * heightM);
        System.out.println("BMI = " + bmi);

        long rounded = Math.round(bmi * 10);
        double result = rounded / 10.0;
        System.out.println("BMI（小数第1位）= " + result);
    }
}

Step 2（改造）: 身長を cm（センチメートル） で受け取り、内部で m に変換するように改造してください。変数名は heightCm とし、値は 165（int型）にしてください。

terminal期待出力（改造後）
身長: 165cm → 1.65m
BMI = 22.03856749311295
BMI（小数第1位）= 22.0
lightbulbヒント

int heightCm = 165; のとき、heightCm / 100 は int 同士の除算で 1 になります。heightCm / 100.0 と書くことで double の結果が得られます。

check_circle解答・解説 
public class BmiCalc {
    public static void main(String[] args) {
        double weightKg = 60.0;
        int heightCm = 165;

        // cm → m 変換（100.0 で割ることで double になる）
        double heightM = heightCm / 100.0;
        System.out.println("身長: " + heightCm + "cm → " + heightM + "m");

        double bmi = weightKg / (heightM * heightM);
        System.out.println("BMI = " + bmi);

        long rounded = Math.round(bmi * 10);
        double result = rounded / 10.0;
        System.out.println("BMI（小数第1位）= " + result);
    }
}
解説
  • 165 / 100 は int 同士の除算で結果は 1 です。165 / 100.0 と書くと int が double に拡大変換され、結果は 1.65 になります。
  • Math.round() は double を受け取り long を返します。/ 10.0 で再び double に変換するテクニックは頻出パターンです。
  • int 除算の「罠」はJavaSilverで頻出です。5 / 2 が 2 になることを理解しておきましょう。
  • 変数の型を int から double に変えるだけで計算結果が変わる点は、型の重要性を示す好例です。

String は不変（immutable）です。substring(), indexOf(), replace(), toUpperCase() などのメソッドは元の文字列を変更せず、新しい String を返します。

assignment課題

以下のコードの /* ① */ 〜 /* ④ */ を埋めて、String メソッドの動作確認プログラムを完成させてください。

public class StringMethods {
    public static void main(String[] args) {
        String text = "Hello, Java World!";

        // 1. 部分文字列の取得（7文字目から11文字目まで）
        String sub = text./* ① */(7, 11);
        System.out.println("substring: " + sub);

        // 2. 文字列の検索（"Java" の開始位置）
        int pos = text./* ② */("Java");
        System.out.println("indexOf: " + pos);

        // 3. 文字列の置換（"Java" を "Spring" に）
        String replaced = text./* ③ */("Java", "Spring");
        System.out.println("replace: " + replaced);

        // 4. 大文字に変換
        String upper = text./* ④ */();
        System.out.println("toUpperCase: " + upper);
    }
}
terminal期待出力
substring: Java
indexOf: 7
replace: Hello, Spring World!
toUpperCase: HELLO, JAVA WORLD!
lightbulbヒント

① substring(begin, end) は begin を含み end を含みません（半開区間）。② indexOf() は見つからない場合 -1 を返します。③ replace(old, new) で置換。④ toUpperCase() で全て大文字に。

check_circle解答・解説 
public class StringMethods {
    public static void main(String[] args) {
        String text = "Hello, Java World!";

        // 1. 部分文字列の取得（7文字目から11文字目まで）
        String sub = text.substring(7, 11);       // ① substring
        System.out.println("substring: " + sub);

        // 2. 文字列の検索（"Java" の開始位置）
        int pos = text.indexOf("Java");            // ② indexOf
        System.out.println("indexOf: " + pos);

        // 3. 文字列の置換（"Java" を "Spring" に）
        String replaced = text.replace("Java", "Spring"); // ③ replace
        System.out.println("replace: " + replaced);

        // 4. 大文字に変換
        String upper = text.toUpperCase();          // ④ toUpperCase
        System.out.println("toUpperCase: " + upper);
    }
}
解説
  • substring(7, 11) はインデックス 7 〜 10 の4文字を返します。end は含まない（半開区間）点がJavaSilver頻出です。
  • indexOf() は最初に見つかった位置を返します。見つからなければ -1 です。lastIndexOf() は末尾から検索します。
  • replace() は該当する全ての箇所を置換します。元の String は変わらず、新しい String が返されます。
  • これらのメソッドはすべて元の文字列を変更しません（Stringの不変性）。戻り値を変数に受け取る必要があります。

verified
JavaSilverポイント

contains(), startsWith(), endsWith() は boolean を返すメソッドです。条件式の中で直接使えるため、if 文との組み合わせがJavaSilverで出題されます。

assignment課題

Step 1（写経）: 以下のコードをそのまま写経して実行してください。

public class EmailValidator {
    public static boolean isValid(String email) {
        if (email == null || email.isEmpty()) {
            return false;
        }
        if (!email.contains("@")) {
            return false;
        }
        if (!email.endsWith(".com")) {
            return false;
        }
        return true;
    }

    public static void main(String[] args) {
        System.out.println("test@example.com → " + isValid("test@example.com"));
        System.out.println("invalid-email → " + isValid("invalid-email"));
        System.out.println("user@site.org → " + isValid("user@site.org"));
    }
}

Step 2（改造）: .com だけでなく .jp でも有効と判定されるように改造してください。さらに、@ の前に1文字以上あることもチェックしてください。

terminal期待出力（改造後）
test@example.com → true
invalid-email → false
user@site.org → false
info@company.jp → true
@noname.com → false
lightbulbヒント

.jp 対応は email.endsWith(".com") || email.endsWith(".jp") で実現できます。@ の前に1文字以上あるかは email.indexOf("@") > 0（0より大きい = 先頭ではない）で判定できます。

check_circle解答・解説 
public class EmailValidator {
    public static boolean isValid(String email) {
        if (email == null || email.isEmpty()) {
            return false;
        }
        // @ が含まれ、かつ先頭でないこと
        int atIndex = email.indexOf("@");
        if (atIndex < 1) {
            return false;
        }
        // .com または .jp で終わること
        if (!email.endsWith(".com") && !email.endsWith(".jp")) {
            return false;
        }
        return true;
    }

    public static void main(String[] args) {
        System.out.println("test@example.com → " + isValid("test@example.com"));
        System.out.println("invalid-email → " + isValid("invalid-email"));
        System.out.println("user@site.org → " + isValid("user@site.org"));
        System.out.println("info@company.jp → " + isValid("info@company.jp"));
        System.out.println("@noname.com → " + isValid("@noname.com"));
    }
}
解説
  • indexOf("@") が -1 なら @ がない、0 なら先頭にある（名前部分がない）ことを意味します。< 1 で両方をチェックできます。
  • endsWith(".com") || endsWith(".jp") の否定は !endsWith(".com") && !endsWith(".jp") です（ド・モルガンの法則）。
  • null チェックを最初に行うのは NullPointerException を防ぐ基本パターンです。
  • String の isEmpty() は長さ 0 かを判定します。Java 11 以降は isBlank()（空白のみも true）も使えます。

配列の length はフィールド（メソッドではない）です。new int[5] で生成した配列の各要素は 0 で初期化されます。インデックスは 0 から始まります。

assignment課題

以下のコードの /* ① */ 〜 /* ③ */ を埋めて、配列の合計値と平均値を求めるプログラムを完成させてください。

public class ArraySum {
    public static void main(String[] args) {
        int[] scores = {85, 92, 78, 95, 88};

        int sum = 0;
        for (int i = 0; i < /* ① */; i++) {
            sum = sum + /* ② */;
        }
        System.out.println("合計: " + sum);

        double avg = (double) sum / /* ③ */;
        System.out.println("平均: " + avg);
    }
}
terminal期待出力
合計: 438
平均: 87.6
lightbulbヒント

① 配列の長さは scores.length です（括弧なし）。② 配列の i 番目の要素は scores[i] です。③ 平均を求めるには合計を要素数で割ります。(double) キャストで小数結果を得ます。

check_circle解答・解説 
public class ArraySum {
    public static void main(String[] args) {
        int[] scores = {85, 92, 78, 95, 88};

        int sum = 0;
        for (int i = 0; i < scores.length; i++) {   // ① scores.length
            sum = sum + scores[i];                     // ② scores[i]
        }
        System.out.println("合計: " + sum);

        double avg = (double) sum / scores.length;     // ③ scores.length
        System.out.println("平均: " + avg);
    }
}
解説
  • length はフィールドであり、() を付けません。String の length() はメソッドなので () が必要です。この違いはJavaSilver頻出です。
  • 配列のインデックスは 0 から length - 1 まで。i < scores.length で正しく全要素を走査できます。
  • (double) sum / scores.length では、まず sum が double に変換され、その後 scores.length も自動的に double に拡大変換されます。
  • sum / scores.length のように int 同士で割ると 87（切り捨て）になってしまいます。

写経+改造 — 最大値・最小値を求める

verified
JavaSilverポイント

配列の走査パターン（for文、拡張for文）と、ArrayIndexOutOfBoundsException が発生する条件はJavaSilverの重要テーマです。

assignment課題

Step 1（写経）: 以下の最大値を求めるコードを写経して実行してください。

public class FindMax {
    public static void main(String[] args) {
        int[] data = {34, 67, 23, 89, 12, 56, 78};

        int max = data[0];
        for (int i = 1; i < data.length; i++) {
            if (data[i] > max) {
                max = data[i];
            }
        }
        System.out.println("最大値: " + max);
    }
}

Step 2（改造）: 最小値も同時に求めて出力するように改造してください。変数名は min としてください。

terminal期待出力（改造後）
最大値: 89
最小値: 12
lightbulbヒント

int min = data[0]; を追加し、ループ内で if (data[i] < min) をチェックすれば、1回のループで最大値と最小値を同時に求められます。

check_circle解答・解説 
public class FindMax {
    public static void main(String[] args) {
        int[] data = {34, 67, 23, 89, 12, 56, 78};

        int max = data[0];
        int min = data[0];
        for (int i = 1; i < data.length; i++) {
            if (data[i] > max) {
                max = data[i];
            }
            if (data[i] < min) {
                min = data[i];
            }
        }
        System.out.println("最大値: " + max);
        System.out.println("最小値: " + min);
    }
}
解説
  • 初期値を data[0] にすることで、配列内の値と比較する基準を設定しています。0 や Integer.MAX_VALUE を使う方法もありますが、data[0] が最もシンプルです。
  • ループを i = 1 から始めるのは、data[0] は既に初期値として使っているためです。i = 0 から始めても動作しますが、1回分の無駄な比較が発生します。
  • 1つのループで複数の集計を行うパターンは効率的です。ループを2回書くより高速です。
  • 空配列（data.length == 0）の場合、data[0] で ArrayIndexOutOfBoundsException が発生します。実務ではガード条件を入れましょう。

穴埋め — FizzBuzzプログラム

verified
JavaSilverポイント

if-else if-else の評価順序はJavaSilver頻出です。条件は上から順に評価され、最初に true になったブロックのみが実行されます。条件の順序を間違えると意図しない結果になります。

assignment課題

以下のコードの /* ① */ 〜 /* ③ */ を埋めて、FizzBuzz プログラムを完成させてください。1〜20の数字について、3の倍数なら "Fizz"、5の倍数なら "Buzz"、15の倍数なら "FizzBuzz"、それ以外は数字を出力します。

public class FizzBuzz {

                System.out.println("FizzBuzz");
            } else if (/* ② */) {
                System.out.println("Fizz");
            } else if (/* ③ */) {
                System.out.println("Buzz");
            } else {
                System.out.println(i);
            }
        }
    }
}

lightbulbヒント

「15の倍数」の条件を最初に書くのがポイントです。i % 15 == 0（または i % 3 == 0 && i % 5 == 0）を最初に判定しないと、3の倍数や5の倍数の条件に先にマッチしてしまいます。

check_circle解答・解説 
public class FizzBuzz {
    public static void main(String[] args) {
        for (int i = 1; i <= 20; i++) {
            if (i % 15 == 0) {            // ① 15の倍数（最も厳しい条件を先に）
                System.out.println("FizzBuzz");
            } else if (i % 3 == 0) {      // ② 3の倍数
                System.out.println("Fizz");
            } else if (i % 5 == 0) {      // ③ 5の倍数
                System.out.println("Buzz");
            } else {
                System.out.println(i);
            }
        }
    }
}
解説
  • %（剰余演算子）は「割り算の余り」を返します。i % 3 == 0 は「i を 3 で割った余りが 0」= 3の倍数です。
  • 15の倍数（= 3の倍数かつ5の倍数）の条件を最初に判定する必要があります。i % 3 == 0 を先に書くと、15は「Fizz」になってしまいます。
  • i % 15 == 0 の代わりに i % 3 == 0 && i % 5 == 0 と書いても同じ結果です。
  • JavaSilverでは if-else if チェーンの評価順序を変えたときの出力を問う問題が出ます。条件の順序は「より限定的な条件を先に」が鉄則です。

Step 2（改造）: 上三角だけ（j >= i の場合だけ）出力するように改造してください。それ以外の箇所はスペース4文字分を出力してください。

public class MultiTable {
    public static void main(String[] args) {
        for (int i = 1; i <= 9; i++) {
            for (int j = 1; j <= 9; j++) {
                if (j >= i) {
                    System.out.printf("%4d", i * j);

        }

解説

  • println() は改行あり、printf() / print() は改行なしです。行末の println() で改行しています。

public class Counter {

        if (count > 0) {

public class FileReader {
    public static String readFile(String filename) {
        if (filename == null) {
            throw new IllegalArgumentException("ファイル名がnullです");
        }
        if (filename.isEmpty()) {
            throw new IllegalArgumentException("ファイル名が空です");

        return "ファイル [" + filename + "] の内容";

import java.util.HashMap;

public class WordCount {
    public static void main(String[] args) {
        String[] words = {"java", "python", "java", "go", "python", "java"};

        HashMap<String, Integer> countMap = new HashMap<>();
        for (String word : words) {
            int current = countMap.getOrDefault(word, 0);
            countMap.put(word, current + 1);
        }

        // キーをカウントの降順でソート
        ArrayList<String> keys = new ArrayList<>(countMap.keySet());
        keys.sort((a, b) -> countMap.get(b) - countMap.get(a));

        System.out.println("単語カウント（多い順）:");
        for (String key : keys) {
            System.out.println("  " + key + ": " + countMap.get(key));
        }
    }
}
解説
  • getOrDefault(key, defaultValue) はキーが存在しない場合にデフォルト値を返します。get() だと null が返り、NullPointerException の原因になります。
  • keySet() は Map の全キーを Set として返します。これを ArrayList のコンストラクタに渡してリストに変換しています。
  • keys.sort((a, b) -> countMap.get(b) - countMap.get(a)) はラムダ式による比較です。b - a で降順、a - b で昇順になります。
  • HashMap は要素の順序を保証しません。順序が必要な場合は LinkedHashMap（挿入順）や TreeMap（キーの自然順序）を使います。
  • ラムダ式 (a, b) -> 式 は関数型インタフェース（ここでは Comparator<String>）の簡潔な記法です。
