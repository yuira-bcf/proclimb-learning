D-01
T1 変数・データ型・演算子

プリミティブ型の初期値と型変換

verified
JavaSilverポイント

プリミティブ型の暗黙の型変換（ワイドニング）と、フィールドのデフォルト初期値は頻出テーマです。ローカル変数には初期値が自動設定されない点にも注意しましょう。

assignment課題

次のコードを読み、出力結果を予測してください。

public class TypeDemo {
    static int sNum;          // フィールド
    static double sDouble;    // フィールド
    static boolean sFlag;     // フィールド
    static char sChar;         // フィールド

    public static void main(String[] args) {
        System.out.println("A: " + sNum);
        System.out.println("B: " + sDouble);
        System.out.println("C: " + sFlag);
        System.out.println("D: [" + sChar + "]");

        int x = 10;
        double y = x;             // 暗黙の型変換
        System.out.println("E: " + y);

        double d = 9.7;
        int z = (int) d;          // 明示的キャスト
        System.out.println("F: " + z);

        char c = 'A';
        int n = c;                // charからintへの暗黙変換
        System.out.println("G: " + n);
    }
}

A〜G の各行の出力を答えてください。

lightbulbヒント

フィールド（static変数・インスタンス変数）にはデフォルト初期値が設定されます: int=0, double=0.0, boolean=false, char='\u0000'（null文字）。

int → double は暗黙変換（ワイドニング）、double → int はキャスト必須（ナローイング、小数部切り捨て）。

char は内部的に0〜65535の整数値を持ちます。'A' の Unicode値は 65 です。

check_circle解答・解説 
A: 0
B: 0.0
C: false
D: [ ]
E: 10.0
F: 9
G: 65

解説:

A〜D: staticフィールドにはデフォルト初期値が自動設定されます。intは0、doubleは0.0、booleanはfalse、charは'\u0000'（null文字、表示上は空白）です。

E: int(10) → double への暗黙変換（ワイドニング）により 10.0 になります。情報の欠落がないため、キャスト不要です。

F: double(9.7) → int への明示的キャスト（ナローイング）では、小数部が切り捨てられ 9 になります。四捨五入ではありません。

G: char('A') → int への暗黙変換で、Unicodeコードポイント 65 が得られます。

check_circle解答・解説

  • 【?3】= %

D-03
T2 文字列操作

StringメソッドとString Pool

verified
JavaSilverポイント

Stringリテラルはプール（String Pool）で共有されるため == でtrueになりますが、newで生成した場合は別オブジェクトになります。== と equals() の違いは最頻出テーマです。

assignment課題

次のコードを読み、出力結果を予測してください。

public class StringDrill {
    public static void main(String[] args) {
        String s1 = "Hello";
        String s2 = "Hello";
        String s3 = new String("Hello");

        System.out.println("A: " + (s1 == s2));
        System.out.println("B: " + (s1 == s3));
        System.out.println("C: " + s1.equals(s3));

        String s4 = "Hello World";
        System.out.println("D: " + s4.substring(6));
        System.out.println("E: " + s4.substring(0, 5));
        System.out.println("F: " + s4.indexOf("World"));
        System.out.println("G: " + s4.indexOf("Java"));
        System.out.println("H: " + s4.length());
    }
}

A〜H の各行の出力を答えてください。

lightbulbヒント

== は参照（アドレス）の比較、equals() は文字列の内容の比較です。

substring(beginIndex) はbeginIndex以降の文字列を返します。substring(begin, end) はbeginからend-1までです。

indexOf() は見つかった位置（0始まり）を返し、見つからなければ -1 を返します。

check_circle解答・解説 
A: true
B: false
C: true
D: World
E: Hello
F: 6
G: -1
H: 11

解説:

A (true): s1とs2は同じリテラル "Hello" なので、String Poolにより同一オブジェクトを参照します。== はtrueです。

B (false): s3は new String() で生成したため、ヒープ上に新しいオブジェクトが作られます。s1とは別の参照なので == はfalseです。

C (true): equals() は文字列の中身を比較するため、trueです。

D (World): substring(6) はインデックス6以降を返します。"Hello World" の6番目は 'W' です。

E (Hello): substring(0, 5) はインデックス0〜4の5文字を返します。

F (6): "World" はインデックス6から始まります。

G (-1): "Java" は含まれないので -1 を返します。

H (11): "Hello World" は空白を含めて11文字です。

D-07
T4 フロー制御

switchのfall-throughとbreak

verified
JavaSilverポイント

switch文でbreakを書き忘れるとfall-through（次のcaseにそのまま突入）が起きます。この挙動を問う問題は非常に頻出です。default節の位置にも注意しましょう。

assignment課題

次のコードを読み、出力結果を予測してください。

public class SwitchDrill {
    public static void main(String[] args) {
        int day = 3;
        switch (day) {
            case 1:
                System.out.println("月曜");
                break;
            case 2:
                System.out.println("火曜");
                break;
            case 3:
                System.out.println("水曜");
            case 4:
                System.out.println("木曜");
            case 5:
                System.out.println("金曜");
                break;
            default:
                System.out.println("週末");
        }

        System.out.println("---");

        String grade = "B";
        if (grade.equals("A")) {
            System.out.println("優秀");
        } else if (grade.equals("B")) {
            System.out.println("良好");
        } else if (grade.equals("C")) {
            System.out.println("普通");
        } else {
            System.out.println("不明");
        }
    }
}

すべての出力行を順番に答えてください。

lightbulbヒント

case 3 にはbreakがありません。一致したcaseの処理を実行した後、breakに出会うまで後続のcaseの処理も実行し続けます（fall-through）。

if-else if は上から順に条件を評価し、最初に一致した分岐のみ実行します。

check_circle解答・解説 
水曜
木曜
金曜
---
良好

解説:

switch部分: day=3 なので case 3 に一致し「水曜」を出力します。しかし case 3 にbreakがないため、case 4（「木曜」）、case 5（「金曜」）にfall-throughします。case 5 にbreakがあるので、そこで脱出します。defaultは実行されません。

if-else部分: grade は "B" なので、2番目の条件 grade.equals("B") に一致し「良好」のみ出力されます。

D-08
T4 フロー制御

ループとbreak/continue

verified
JavaSilverポイント

for / while / do-while の実行順序の違い、およびbreak（ループ脱出）とcontinue（次の繰り返しへスキップ）の動作は試験頻出です。do-whileは「最低1回実行」される点に注意です。

assignment課題

次のコードの空欄を埋めて、コメントに示された出力結果になるようにしてください。

public class LoopDrill {
    public static void main(String[] args) {
        // 1〜5を出力（3はスキップ）
        for (int i = 1; i <= 5; i++) {
            if (i == 3) 【?1】;
            System.out.print(i + " ");
        }
        System.out.println();
        // 出力: 1 2 4 5

        // 1から順に出力、3で停止
        for (int i = 1; i <= 5; i++) {
            if (i == 3) 【?2】;
            System.out.print(i + " ");
        }
        System.out.println();
        // 出力: 1 2

        // do-while: 条件がfalseでも1回は実行
        int x = 10;
        【?3】 {
            System.out.print(x + " ");
            x++;
        } 【?4】 (x < 5);
        System.out.println();
        // 出力: 10
    }
}

【?1】〜【?4】に入るキーワードを答えてください。

lightbulbヒント

continue はそのループの残りの処理をスキップして次の繰り返しに進みます。break はループ全体を終了します。

do-while文の構文は do { ... } while (条件); です。条件判定がループ本体の後にあるため、最低1回は実行されます。

check_circle解答・解説

解答:

  • 【?1】= continue
  • 【?2】= break
  • 【?3】= do
  • 【?4】= while

解説:

?1 (continue): i==3 のとき continue で System.out.print() をスキップし、次のi=4に進みます。結果: 1 2 4 5

?2 (break): i==3 のとき break でforループ全体を脱出します。i=3以降は出力されません。結果: 1 2

?3, ?4 (do, while): do { ... } while (x < 5); の形です。x=10 で条件 x < 5 はfalseですが、do-whileは最低1回実行されるので 10 が出力されます。

D-09

フィールド・メソッド・thisキーワード

verified
JavaSilverポイント

thisキーワードはインスタンス自身への参照です。コンストラクタやメソッドの引数名がフィールド名と同じ場合、thisで明示的にフィールドを指定する必要があります。

assignment課題

次のコードを読み、出力結果を予測してください。

public class Person {
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void greet() {
        System.out.println("I am " + name + ", age " + age);
    }

    void birthday() {
        age++;
        System.out.println("Happy Birthday! Now " + age);
    }

    public static void main(String[] args) {
        Person p1 = new Person("Alice", 25);
        Person p2 = new Person("Bob", 30);

        p1.greet();
        p2.greet();
        p1.birthday();
        p1.greet();
    }
}

すべての出力行を順番に答えてください。

lightbulbヒント

各Personオブジェクトは独立したフィールド（name, age）を持ちます。p1.birthday()はp1のageだけを変更し、p2には影響しません。

check_circle解答・解説 
I am Alice, age 25
I am Bob, age 30
Happy Birthday! Now 26
I am Alice, age 26

解説:

1行目: p1("Alice", 25) の greet() で I am Alice, age 25。

2行目: p2("Bob", 30) の greet() で I am Bob, age 30。

継承チェーンとsuperの呼び出し順序

verified
JavaSilverポイント

コンストラクタの呼び出し順序は「親クラス → 子クラス」です。明示的にsuper()を書かなくても、コンパイラが自動的に親クラスの引数なしコンストラクタを挿入します。オーバーライドされたメソッドがどちらのクラスの実装を呼ぶかも重要です。

assignment課題

次のコードを読み、出力結果を予測してください。

    String name;

    Animal() {
        System.out.println("Animal constructor");
        name = "Unknown";

        this.name = name;
    }

    void speak() {
        System.out.println(name + " speaks");
    }
}

        super(name);
        System.out.println("Dog constructor");

public class InheritDrill {

try-catch-finallyの実行順序

verified
JavaSilverポイント

finallyブロックは例外の有無に関わらず必ず実行されます。try内で例外が発生した場合、tryの残りは実行されずcatchに移行する点、複数catchの順序（子クラス→親クラス）も重要です。

assignment課題

次のコードを読み、出力結果を予測してください。

public class ExceptionDrill {
    public static void main(String[] args) {
        System.out.println("Start");
        try {
            System.out.println("Try-1");
            int result = 10 / 0;
            System.out.println("Try-2");
        } catch (ArithmeticException e) {
            System.out.println("Catch: " + e.getMessage());
        } finally {
            System.out.println("Finally");
        }
        System.out.println("End");

        System.out.println("===");

        try {
            System.out.println("A");
            String s = null;
            System.out.println(s.length());
        } catch (NullPointerException e) {
            System.out.println("B");
        } catch (Exception e) {
            System.out.println("C");
        } finally {
            System.out.println("D");
        }
    }
}

すべての出力行を順番に答えてください。

lightbulbヒント

10 / 0 は ArithmeticException をスローします。例外発生後、tryの残り（"Try-2"）はスキップされます。

null.length() は NullPointerException をスローします。catchは上から順にマッチングされ、最初に一致したブロックのみ実行されます。

check_circle解答・解説 
Start
Try-1
Catch: / by zero
Finally
End
===
A
B
D

解説:

前半:

  • 「Start」を出力後、tryブロックに入り「Try-1」を出力。
  • 10 / 0 で ArithmeticException が発生。「Try-2」はスキップ。
  • catchブロックで「Catch: / by zero」を出力。
  • finallyブロックで「Finally」を出力（例外の有無に関わらず実行）。
  • 例外が処理されたので、通常通り「End」を出力。

後半:

  • 「A」を出力後、null.length() で NullPointerException が発生。
  • 最初のcatch (NullPointerException) に一致し「B」を出力。catch (Exception) はスキップ。

ラムダ式は関数型インタフェース（抽象メソッドが1つのインタフェース）の実装を簡潔に書く構文です。LocalDateはイミュータブル（不変）で、plusDays()などは新しいオブジェクトを返す点が頻出です。

lightbulbヒント

ラムダ式の基本構文: (引数) -> 式 または (引数) -> { 文; }

1つの引数の場合は括弧を省略可: name ->

LocalDateの plusDays(n) は新しいLocalDateオブジェクトを返します。元のオブジェクトは変更されません（イミュータブル）。

check_circle解答・解説

解答:

  • 【?1】= name ->
  • 【?2】= ->
