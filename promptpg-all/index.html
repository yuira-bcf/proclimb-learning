Section 0-1

概要

この章では、AIを活用したプロンプトプログラミングの手法を学びます。

題材として一覧、登録、確認、完了の4画面を持つToDoアプリを取り上げ、各画面ごとにプロンプトを作成していきます。

この章で学ぶこと

📝

プロンプトの書き方

AIに的確な指示を出すためのプロンプト作成技術

🔄

対話的な開発

AIからの質問に答えながら仕様を詰めていく手法

📋

仕様書・設計書の生成

プロンプトから仕様書・設計書を自動生成する方法

学習の流れ

  1. 開発規約の作成：使用技術やディレクトリ構成を決める
  2. 画面遷移の設計：画面の流れとURL設計を行う
  3. 各画面のプロンプト作成：一覧・登録・確認・完了画面のプロンプトを作成
  4. 仕様書・設計書の出力：全体構成を確認し、ドキュメントを生成
  5. プロンプトのダウンロード：作成したプロンプトを保存
warning 重要：この章の目的

この章ではプロンプトの作成方法を学びます。
実際のコーディングは行いません。作成したプロンプトをAIに送信してコードを生成してもらう形式です。

lightbulb 学習のポイント

プロンプトプログラミングでは「何を作りたいか」を明確に伝えることが重要です。
技術的な詳細はAIが補完してくれるため、要件を整理する力を身につけましょう。

Section 0-4

プロンプトの生成

各工程ごとにプロンプトを作成するために必要な項目と、具体的なプロンプト例を紹介します。

warning この章の目的

この章ではプロンプトの作成方法を学びます。
各工程ではプロンプトの生成を行い、実際の実装は行いません。
作成したプロンプトをAIに送信して、コードを生成してもらいましょう。

1. 開発規約の作成

整理すべき項目

項目 内容 サンプル
言語 使用するプログラミング言語 Java
フレームワーク 使用するフレームワーク Spring Boot
データベース 使用するデータベース H2メモリベース
その他 追加で必要な情報 CRUD開発に必要な依存関係を教えて
lightbulb 初学者向けポイント

ディレクトリ構成や命名規則など、詳細な設計はAIに任せてOKです。
最低限「何を使って」「何を作りたいか」を伝えれば、AIが適切な構成を提案してくれます。

プロンプト例 
あなたはJavaプログラマーです。

ToDoアプリを作成したいです。

【使用技術】
- 言語：Java
- フレームワーク：Spring Boot
- データベース：H2メモリベース
- 新規で最小構成のSpring Bootプロジェクトを作成する

【提案要求】
- CRUD開発に必要な依存関係
- 推奨されるディレクトリ構成
- クラス名やメソッド名の命名規則

プロンプト生成において不明箇所があれば教えてください。
なければプロンプトを生成しテキストファイルでダウンロードできる形式で出力してください。

AIからの質問例

🤖 AIからこのような質問が来ることがあります

  • 画面構成：想定する画面は何ですか？（例：一覧・新規作成・編集・詳細・削除確認 など）
  • プロンプト形式：todo_*_prompt.txt はプレーンテキストでOK？それともMarkdown形式にしますか？
  • プロジェクト構成：Maven/Gradleのどちらで最小Spring Bootプロジェクトを作成しますか？
  • アプリ種別：MVC（画面あり）/REST API（画面なし）のどちらを想定しますか？
  • ToDoの項目：最低限のフィールドは何にしますか？（例：id, title, description, dueDate, status など）
lightbulb 質問への回答のコツ

AIからの質問には、できるだけ具体的に回答しましょう。
回答内容がプロンプトに反映され、より適切なコードが生成されます。

2. 画面遷移

整理すべき項目

項目 内容 サンプル

画面の流れ どの順番で画面を移動するか 一覧→登録→確認→完了
戻る操作 前の画面に戻れるか 登録→一覧、確認→登録
lightbulb 初学者向けポイント

URLやファイル名はAIが適切に決めてくれます。
「どんな画面があって、どう移動するか」を伝えれば十分です。

プロンプト例 
あなたはJavaプログラマーです。

3. 確認画面

【画面の流れ】
一覧 → 登録 → 確認 → 完了 → 一覧に戻る

【戻る操作】
- 登録画面から一覧に戻れる
- 確認画面から登録画面に戻れる

Column 効果的なプロンプトのコツ

プロンプトを作成する際のポイントを紹介します。

  1. 具体的に書く：曖昧な表現を避け、具体的な仕様を明記する
  2. 箇条書きを活用：項目ごとに整理して見やすくする
  3. 出力形式を指定：必要なファイル名やクラス名を明示する
  4. 技術スタックを明記：使用する言語やフレームワークを最初に伝える
  5. 段階的に依頼：一度に全部ではなく、画面ごとに分けて依頼する

7. 仕様書・設計書の出力

全ての画面のプロンプトが完成したら、全体構成を確認するための仕様書・設計書を出力してもらいましょう。

出力の流れ

  1. 全画面のプロンプト完成：一覧・登録・確認・完了画面のプロンプトを作成
  2. 仕様書・設計書の依頼：下記のプロンプトを送る
  3. 出力内容の確認：全体構成に問題がないか確認
  4. 必要に応じて修正：不足や修正点があれば伝える
lightbulb なぜ仕様書・設計書が必要？

各画面を個別に作成した後、全体の整合性を確認することが重要です。
仕様書・設計書があれば、開発の全体像を把握でき、実装時の手戻りを防げます。

仕様書・設計書出力プロンプト例 
これまで作成した全てのプロンプトをもとに、ToDoアプリの仕様書・設計書を出力してください。

【出力してほしい内容】
1. システム概要
   - アプリケーション名
   - 目的・概要
   - 使用技術

2. 画面一覧
   - 画面名、URL、概要の一覧表

3. 画面遷移図
   - テキストベースの画面遷移図

4. データベース設計
   - テーブル定義（カラム名、型、制約）

5. 機能一覧
   - 各画面の機能詳細

ダウンロードできる形式で出力してください。

9. 実装用プロンプトの生成

全てのプロンプトファイルが揃ったら、実装用のプロンプトを生成してAIに実際のコードを作成してもらいます。

手順

  1. プロンプトファイルの準備：ダウンロードした全てのプロンプトファイル（01〜07）を用意
  2. ファイルのアップロード：AIチャットに全てのプロンプトファイルをアップロード
  3. 実装指示の送信：下記の実装用プロンプトを送信
  4. コードの確認：生成されたコードを確認し、必要に応じて修正を依頼

# 実装指示
このリポジトリに実ファイルを作成して実装してください。

## 技術スタック
- Java 17以上
- Spring Boot 3.x
- Spring Web MVC
- Spring Data JPA
- H2 Database（インメモリ）
- Thymeleaf テンプレートエンジン

## 実装する機能
1. ToDo一覧表示（検索・ソート機能付き）
2. ToDo新規登録
3. ToDo編集・更新
4. ToDo削除
5. 画面遷移（一覧→登録→確認→完了→一覧）

## 作成するファイル構成
```
src/main/java/com/example/todo/
├── TodoApplication.java
├── controller/TodoController.java
├── entity/Todo.java
├── repository/TodoRepository.java
└── service/TodoService.java

src/main/resources/
├── application.properties
└── templates/
    ├── index.html（一覧画面）
    ├── create.html（登録画面）
    ├── confirm.html（確認画面）
    └── complete.html（完了画面）
```
