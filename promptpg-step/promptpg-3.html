Section 3-1

フォームオブジェクトの作成

フォームのデータを格納するFormクラスを作成します。Lombokを使用してコードを簡潔にします。

smart_toy AIプロンプト

Spring BootでToDoアプリ用のフォームクラスを作成してください。

要件:
- クラス名: TodoForm
- フィールド:
  - title（タイトル）: String
  - description（説明）: String
  - dueDate（期限日）: LocalDate
  - priority（優先度）: Integer
- Lombokを使用（@Data, @NoArgsConstructor, @AllArgsConstructor）
- 各フィールドに適切なバリデーションアノテーションを付与

バリデーションメッセージも日本語で設定してください。
Section 3-2

フォームの送信と受け取り

HTMLフォームからPOSTされたデータを、@ModelAttributeでControllerで受け取ります。

smart_toy AIプロンプト

Spring BootでフォームデータをPOSTで受け取る処理を実装してください。

要件:
- 入力画面（GET /todos/new）でフォームオブジェクトを初期化
- フォーム送信（POST /todos/confirm）でデータを受け取り
- @ModelAttributeを使用
- Thymeleafのth:objectとth:fieldを使用したフォーム

入力画面のHTML（form.html）とControllerの両方を示してください。
Thymeleaf属性 説明
th:object フォームにバインドするオブジェクトを指定
th:field フォームフィールドとオブジェクトのプロパティをバインド
th:action フォームの送信先URLを指定
Section 3-3

確認画面への遷移

入力されたデータを確認画面に表示し、ユーザーに確認させてから登録処理を行います。

smart_toy AIプロンプト

Spring Bootで入力→確認→完了の画面遷移を実装してください。

画面フロー:
1. 入力画面（/todos/new）
   - フォームに入力
   - 「確認」ボタンで次へ

2. 確認画面（/todos/confirm）
   - 入力内容を表示（読み取り専用）
   - 「戻る」ボタンで入力画面へ（入力値を保持）
   - 「登録」ボタンで完了画面へ

3. 完了画面（/todos/complete）
   - 登録完了メッセージを表示

RedirectAttributesを使用したデータの持ち回り方法を示してください。
check_circle 確認ポイント

「戻る」ボタンを押した際に、入力した値が保持されていることを確認してください。
