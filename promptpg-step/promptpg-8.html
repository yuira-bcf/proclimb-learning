Section 8-1

概要

このステップでは、既存のToDoを編集する機能を実装します。

一覧画面から編集ボタンをクリックすると、現在のデータが入力された状態で編集画面が表示されます。

Section 8-2

仕様

編集の流れ

  1. 一覧画面の編集ボタンをクリック
  2. 編集画面が表示される（既存データがフォームに入力済み）
  3. 内容を修正して確認ボタンをクリック
  4. 確認画面で内容を確認
  5. 更新ボタンで保存

URL設計

URLメソッド説明
/edit/{id}GET編集画面表示（データ読み込み）
/confirmPOST確認画面（新規・編集共通）
/completePOST保存処理（新規・更新を判定）

Section 8-4

プロンプトの生成

プロンプト例 
あなたはJavaプログラマーです。

Step 7で作成したアプリケーションに、編集機能を追加してください。

【要件】
1. 一覧画面に編集ボタンを追加する
2. 編集ボタンをクリックすると、既存データが入力された編集画面を表示する
3. 確認画面・完了画面は新規登録と共通で使用する
4. 新規と編集の判定はIDの有無で行う（idがnullなら新規、値があれば更新）

【TodoFormの変更】
- idフィールドを追加（Long型）

【Serviceクラスの追加メソッド】
- findById(Long id): IDでデータを取得
- update(TodoForm form): データを更新（updatedAtを更新）

【Controllerの追加・変更】
- GET /edit/{id}: 編集画面表示（データを読み込んでFormにセット）
- POST /complete: IDの有無で新規登録か更新かを判定

【テンプレートの変更】
- index.html: 編集ボタン追加
- edit.html: hiddenフィールドでIDを保持
- confirm.html: hiddenフィールドでIDを保持

各ファイルのコードを実装してください。

新規・編集の判定

TodoService.java（抜粋）
public void saveOrUpdate(TodoForm form) {
    if (form.getId() == null) {
        // 新規登録
        save(form);
    } else {
        // 更新
        update(form);
    }
}
