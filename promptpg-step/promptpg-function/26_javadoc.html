Section 26-1

工程概要

Javadocコメントを記述し、HTMLドキュメントを自動生成します。

ドキュメンテーション

主な技術

Build / Documentation

この機能で行うこと

build

Mavenプラグイン設定

maven-javadoc-pluginでビルド時にドキュメントを生成

class

クラスレベルのコメント

概要、@author、@version、@since、@seeを記述

functions

メソッドレベルのコメント

概要、@param、@return、@throwsを記述

folder

パッケージドキュメント

package-info.javaでパッケージの概要を記述

Section 26-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
pom.xml maven-javadoc-pluginの設定
package-info.java パッケージレベルのドキュメント
各Javaファイル Javadocコメントの追加
target/site/apidocs/ 生成されるHTMLドキュメント

主要なJavadocタグ

  • @param - メソッドのパラメータを説明
  • @return - 戻り値を説明
  • @throws - 発生する例外を説明
  • @see - 関連するクラスやメソッドへのリンク
  • {@code} - コードスニペットを表示
  • {@link} - インラインリンクを作成
Section 26-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリにJavadocを追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- 各クラスにはコメントがほとんどない
- APIドキュメントが存在しない

【実装要件】
1. pom.xmlにmaven-javadoc-pluginを追加
   - UTF-8エンコーディング
   - 日本語ロケール
   - ビルド時にJavadoc jarを生成

2. 各クラスにJavadocコメントを追加
   - クラスレベル: 概要、@author、@version、@since、@see
   - メソッドレベル: 概要、@param、@return、@throws
   - フィールドレベル: 各フィールドの説明

3. package-info.javaの作成
   - 各パッケージの概要を説明
   - 主要クラスへのリンク

4. Javadocタグの使用例
   - {@code コード}
   - {@link クラス名}
   - {@inheritDoc}
   - 
、
    、
  • などのHTML

5. 以下のクラスにJavadocを追加
   - TodoService（全メソッド）
   - TodoController（全エンドポイント）
   - Todo（エンティティ）
   - TodoForm（フォームクラス）

【技術スタック】
- Java 17+
- Maven
- maven-javadoc-plugin 3.6.0

必要なコードの変更点を提示してください。
また、Javadoc生成コマンド（mvn javadoc:javadoc）の実行方法も説明してください。
lightbulb ポイント

Swagger/OpenAPIを併用すると、REST APIのドキュメントも自動生成できます。フロントエンド開発者との連携に便利です。
