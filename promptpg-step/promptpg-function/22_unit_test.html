Section 22-1

工程概要

Service層・Controller層のテストを実装し、アプリケーションの品質を保証します。

Section 22-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
TodoServiceTest.java Service層の単体テスト
TodoControllerTest.java Controller層の単体テスト（MockMvc使用）
TodoMapperTest.java Mapper層のテスト（@MybatisTest使用）
application-test.properties テスト用設定ファイル
TestDataFactory.java テストデータ生成ユーティリティ

テストの主要アノテーション

  • @ExtendWith(MockitoExtension.class) - Mockitoを有効化
  • @Mock - モックオブジェクトを作成
  • @InjectMocks - モックを注入したテスト対象を作成
  • @WebMvcTest - Controller層のテスト用
  • @MockBean - Spring Contextにモックを登録
Section 22-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリに単体テストを追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- TodoService、TodoController、TodoMapperが実装済み
- テストコードは未作成

【実装要件】
1. Service層のテスト（TodoServiceTest.java）
   - @ExtendWith(MockitoExtension.class)でMockitoを使用
   - TodoMapperを@Mockでモック化
   - findAll(), findById(), create(), update(), delete()のテスト
   - 正常系・異常系（例外発生時）の両方をテスト
   - AssertJを使用したアサーション

2. Controller層のテスト（TodoControllerTest.java）
   - @WebMvcTestでMockMvcを使用
   - TodoServiceを@MockBeanでモック化
   - GET/POST/PUT/DELETEのエンドポイントテスト
   - バリデーションエラー時の動作テスト
   - リダイレクト先の検証

3. Mapper層のテスト（TodoMapperTest.java）
   - @MybatisTestでMyBatisのテスト
   - H2インメモリDBを使用
   - CRUD操作のテスト

4. テスト設定
   - application-test.properties（テスト用DB設定）
   - @DisplayNameで日本語テスト名を付与

【技術スタック】
- JUnit 5
- Mockito
- MockMvc
- AssertJ
- H2 Database（テスト用）

必要なコードの変更点を提示してください。
