テストの主要アノテーション

  • @ExtendWith(MockitoExtension.class) - Mockitoを有効化
  • @Mock - モックオブジェクトを作成
  • @InjectMocks - モックを注入したテスト対象を作成
  • @WebMvcTest - Controller層のテスト用
  • @MockBean - Spring Contextにモックを登録

サンプルソースコード

TodoServiceTest.java 
@ExtendWith(MockitoExtension.class)
class TodoServiceTest {

    @Mock
    private TodoMapper todoMapper;

    @InjectMocks
    private TodoService todoService;

    @Test
    @DisplayName("全件取得：正常系")
    void findAll_ReturnsAllTodos() {
        // Arrange（準備）
        List<Todo> expected = List.of(
            new Todo(1, "タスク1"),
            new Todo(2, "タスク2")
        );
        when(todoMapper.findAll()).thenReturn(expected);

        // Act（実行）
        List<Todo> actual = todoService.findAll();

        // Assert（検証）
        assertThat(actual).hasSize(2);
        assertThat(actual.get(0).getTitle()).isEqualTo("タスク1");
        verify(todoMapper, times(1)).findAll();
    }

    @Test
    @DisplayName("ID検索：存在しないID")
    void findById_NotFound_ThrowsException() {
        // Arrange
        when(todoMapper.findById(999)).thenReturn(null);

        // Act & Assert
        assertThatThrownBy(() -> todoService.findById(999))
            .isInstanceOf(ResourceNotFoundException.class)
            .hasMessage("ToDo not found: 999");
    }
}

lightbulb コード解説

  • @Mock - TodoMapperをモック化（実際のDBアクセスは行わない）
  • @InjectMocks - モックを注入してTodoServiceを構築
  • when().thenReturn() - モックの振る舞いを定義
  • assertThat() - AssertJの流暢なアサーション
  • verify() - モックメソッドの呼び出し回数を検証
TodoControllerTest.java 
@WebMvcTest(TodoController.class)
class TodoControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private TodoService todoService;

    @Test
    @DisplayName("一覧画面：200 OKを返す")
    void list_ReturnsOk() throws Exception {
        when(todoService.findAll()).thenReturn(List.of());

        mockMvc.perform(get("/todos"))
            .andExpect(status().isOk())
            .andExpect(view().name("todos/list"))
            .andExpect(model().attributeExists("todos"));
    }

    @Test
    @DisplayName("作成：バリデーションエラー")
    void create_ValidationError_ReturnsForm() throws Exception {
        mockMvc.perform(post("/todos")
                .param("title", "")  // 空のタイトル
                .with(csrf()))
            .andExpect(status().isOk())
            .andExpect(view().name("todos/form"))
            .andExpect(model().hasErrors());
    }

    @Test
    @DisplayName("作成成功：リダイレクト")
    void create_Success_Redirects() throws Exception {
        mockMvc.perform(post("/todos")
                .param("title", "新しいタスク")
                .with(csrf()))
            .andExpect(status().is3xxRedirection())
            .andExpect(redirectedUrl("/todos"));
    }
}

lightbulb コード解説

  • @WebMvcTest - Controller層のみをテスト（Serviceはモック）
  • MockMvc - HTTPリクエストをシミュレート
  • @MockBean - Spring Contextにモックを登録
  • with(csrf()) - CSRFトークンをリクエストに追加
  • andExpect() - レスポンスの検証（ステータス、ビュー名、モデル属性など）

【AAA パターン（テストの基本構造）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Arrange（準備）  │ テストデータの準備、モックの設定
  Act（実行）      │ テスト対象メソッドの実行
  Assert（検証）   │ 結果の検証、期待値との比較

【モックの種類】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  @Mock     │ Mockitoが生成するモック（Spring不要）
  @MockBean │ Spring Contextに登録されるモック（DIで使用）
  @Spy      │ 実際のオブジェクトをラップ（一部メソッドのみモック化）

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- TodoService、TodoController、TodoMapperが実装済み

【実装要件】
1. Service層のテスト（TodoServiceTest.java）
   - @ExtendWith(MockitoExtension.class)でMockitoを使用
   - TodoMapperを@Mockでモック化
   - findAll(), findById(), create(), update(), delete()のテスト
   - 正常系・異常系（例外発生時）の両方をテスト
   - AssertJを使用したアサーション

2. Controller層のテスト（TodoControllerTest.java）
   - @WebMvcTestでMockMvcを使用
   - TodoServiceを@MockBeanでモック化
   - GET/POST/PUT/DELETEのエンドポイントテスト
   - バリデーションエラー時の動作テスト
   - リダイレクト先の検証

3. Mapper層のテスト（TodoMapperTest.java）
   - @MybatisTestでMyBatisのテスト
   - H2インメモリDBを使用
   - CRUD操作のテスト

4. テスト設定
   - application-test.properties（テスト用DB設定）
   - @DisplayNameで日本語テスト名を付与
