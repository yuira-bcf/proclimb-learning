Section 14-1

工程概要

ログインユーザーごとにToDoを管理し、自分のデータのみ閲覧・編集できるようにします。

学習テーマ

ユーザーごとのToDo管理

Controller / Repository / View

user_id外部キーの追加

filter_list

一覧表示の絞り込み

自分のToDoのみ表示

アクセス制御

Section 14-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容

TodoMapper.xml user_idによる絞り込みクエリ

Thymeleaf Security拡張

  • sec:authentication="name" - ログインユーザー名を表示
  • sec:authorize="isAuthenticated()" - 認証済みの場合のみ表示
  • sec:authorize="hasRole('ADMIN')" - ADMIN権限の場合のみ表示
Section 14-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリにユーザー別ToDo管理機能を追加してください。

【要件】
- ToDoにuser_id（作成者）を追加し、ユーザーと紐付け
- ログインユーザーは自分のToDoのみ表示・編集可能
- 新規作成時に自動的にログインユーザーのIDを設定
- 他ユーザーのToDoにアクセスした場合は403エラー

【実装箇所】

■ Todo.java
- user_idフィールドを追加
- getter/setterを追加

■ schema.sql
- ALTER TABLE todo ADD user_id INT;
- ALTER TABLE todo ADD FOREIGN KEY (user_id) REFERENCES users(id);

■ TodoController.java
- @AuthenticationPrincipal UserDetails userDetailsでログインユーザー取得
- findAll()をfindByUserId(userId)に変更
- 新規作成時にtodo.setUserId(userId)を設定

■ TodoMapper.xml
- findByUserId(Integer userId)クエリを追加
- INSERT文にuser_idを追加

■ list.html（Thymeleaf + Security）
- sec:authentication="name"でユーザー名表示
- 件数表示「あなたのToDoは○件です」

【技術スタック】
- Spring Boot 3.x
- Spring Security (@AuthenticationPrincipal)
- MyBatis
- Thymeleaf Security拡張

必要なコードを提示してください。
lightbulb ポイント

セキュリティ強化として「@PreAuthorize("@todoSecurityService.isOwner(#id, principal)")でメソッドレベルのセキュリティを実装してください」を追加するとより堅牢になります。
