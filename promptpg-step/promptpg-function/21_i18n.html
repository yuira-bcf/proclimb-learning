Section 21-1

工程概要

アプリケーションを日本語・英語などの複数言語に対応させ、ユーザーが言語を切り替えられるようにします。

Section 21-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
WebConfig.java LocaleResolver、LocaleChangeInterceptorの設定
messages_ja.properties 日本語メッセージファイル
messages_en.properties 英語メッセージファイル
application.properties メッセージソース設定の追加
各種HTMLテンプレート #{key}形式のメッセージ参照に修正

国際化の主要コンポーネント

  • MessageSource - メッセージをロケールに応じて取得
  • LocaleResolver - 現在のロケールを決定・変更
  • LocaleChangeInterceptor - URLパラメータで言語切り替え
  • #{key} - Thymeleafでのメッセージ参照形式

サンプルソースコード

WebConfig.java 
@Configuration
public class WebConfig implements WebMvcConfigurer {

    // ロケールをセッションで管理
    @Bean
    public LocaleResolver localeResolver() {
        SessionLocaleResolver resolver = new SessionLocaleResolver();
        resolver.setDefaultLocale(Locale.JAPANESE);  // デフォルト: 日本語
        return resolver;
    }

    // ?lang=ja または ?lang=en で言語切り替え
    @Bean
    public LocaleChangeInterceptor localeChangeInterceptor() {
        LocaleChangeInterceptor interceptor = new LocaleChangeInterceptor();
        interceptor.setParamName("lang");  // パラメータ名
        return interceptor;
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(localeChangeInterceptor());
    }
}

lightbulb コード解説

  • SessionLocaleResolver - ユーザーのロケール設定をセッションに保存
  • setDefaultLocale() - 初回アクセス時のデフォルト言語を設定
  • LocaleChangeInterceptor - URLパラメータでロケールを変更するインターセプター
  • setParamName("lang") - ?lang=ja のようなパラメータで言語切り替え

list.html（Thymeleafテンプレート） 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <!-- #{key} でメッセージを参照 -->
    <title th:text="#{app.title}">ToDo App</title>
</head>
<body>
    <!-- 言語切り替えリンク -->
    <nav>
        <a th:href="@{''(lang='ja')}">日本語</a>
        <a th:href="@{''(lang='en')}">English</a>
    </nav>

    <h1 th:text="#{todo.list.title}">ToDo List</h1>

    <form th:action="@{/todos}" method="post">
        <label th:text="#{todo.form.title}">Title</label>
        <input type="text" name="title" />
        <button th:text="#{todo.button.save}">Save</button>
    </form>
</body>
</html>

lightbulb コード解説

  • #{key} - Thymeleafでメッセージプロパティを参照する構文
  • @{''(lang='ja')} - 現在のURLに?lang=jaパラメータを追加
  • タグ内のテキスト - プロパティが見つからない場合のフォールバック表示

Section 21-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

auto_awesome AIプロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリに国際化（i18n）機能を追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- 現在は日本語のみ対応
- 画面のラベルやメッセージがハードコードされている

【実装要件】
1. 多言語対応の設定
   - SessionLocaleResolverでセッションベースのロケール管理
   - LocaleChangeInterceptorでURLパラメータ(?lang=ja/?lang=en)による言語切り替え
   - デフォルト言語は日本語

2. メッセージファイルの作成
   - messages_ja.properties（日本語）
   - messages_en.properties（英語）
   - ページタイトル、ラベル、ボタン、成功/エラーメッセージを定義

3. Thymeleafテンプレートの修正
   - ハードコードされた文字列を#{key}形式に置き換え
   - 言語切り替えリンクをヘッダーに追加

4. バリデーションメッセージの国際化
   - ValidationMessages_ja.properties
   - ValidationMessages_en.properties

5. コントローラーでのメッセージ取得
   - MessageSourceをインジェクション
   - LocaleContextHolder.getLocale()で現在のロケールを取得

【技術スタック】
- Spring Boot 3.x
- MessageSource / LocaleResolver
- Thymeleaf
- Jakarta Validation
