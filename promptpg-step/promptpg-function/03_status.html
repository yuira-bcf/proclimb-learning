ステータス管理初級

未着手・進行中・完了の3つの状態を管理

Section 03-1

工程概要

Java Enumを使用して未着手・進行中・完了の3つの状態を管理し、タスクの進捗を可視化します。

学習テーマ

Enum活用

主な技術

Java Enum / Thymeleaf

関連レイヤー

Entity / View

この機能で行うこと

code

Enum作成

TodoStatus列挙型の定義（NOT_STARTED, IN_PROGRESS, COMPLETED）

data_object

Entity修正

completedフィールドからstatusフィールドへ変更

badge

ステータス表示

色分けしたバッジで状態を視覚的に表示

filter_alt

フィルタ機能

ステータス別の絞り込み表示を実装

Section 03-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
TodoStatus.java ステータス列挙型（新規作成）
Todo.java statusフィールドに変更
schema.sql statusカラム追加
list.html ステータスバッジ表示

ステータス定義

  • NOT_STARTED ("未着手") - グレーバッジ
  • IN_PROGRESS ("進行中") - 青バッジ
  • COMPLETED ("完了") - 緑バッジ
Section 03-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

ToDoアプリにステータス管理機能を実装してください。

【現状】
- completed (Boolean) で完了/未完了を管理

【変更後】
- status (TodoStatus) で3段階の状態を管理

【ステータス定義】
1. NOT_STARTED (未着手) - デフォルト値
2. IN_PROGRESS (進行中)
3. COMPLETED (完了)

【実装内容】
1. TodoStatus.java（Enum）
   - 各ステータスに表示名（日本語）を持たせる
   - getDisplayName()メソッドを実装

2. Todo.java
   - completedフィールドをstatusフィールドに変更
   - 型をTodoStatusに変更

3. schema.sql
   - statusカラム（VARCHAR）に変更

4. TodoMapper.xml
   - ステータスの永続化設定

5. list.html
   - ステータスをバッジで表示（色分け）
   - 未着手: グレー、進行中: 青、完了: 緑
   - ステータスで絞り込むドロップダウン追加

6. form.html
   - ステータス選択のラジオボタンまたはドロップダウン

【要件】
- MyBatisでEnumを扱うためのTypeHandler設定も含める
- 一覧画面でステータスのフィルタリングができること

必要なコードの変更点を提示してください。
lightbulb ポイント

MyBatisでEnumを扱う場合、EnumTypeHandlerを使用するか、文字列として保存する方法があります。
