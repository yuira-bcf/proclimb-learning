Section 01-1

工程概要

基本のToDoアプリに説明文、重要度、作成日時などのフィールドを追加し、より実用的なアプリケーションに拡張します。

学習テーマ

エンティティ設計

主な技術

JPA / MyBatis

関連レイヤー

Entity / Repository / View

この機能で行うこと

data_object

Entityクラスの拡張

新しいフィールド（description, priority, createdAt, updatedAt）を追加

storage

DBスキーマ変更

schema.sqlに新規カラムを追加

edit_note

フォームの拡張

入力フィールドを追加し、新しい項目を登録できるようにする

list

一覧表示の拡張

追加フィールドを一覧画面に表示

Section 01-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
Todo.java description, priority, createdAt, updatedAt フィールド追加
schema.sql 新規カラムの定義
TodoMapper.xml INSERT/UPDATE文の修正
form.html 入力フィールド追加
list.html 一覧表示の拡張

追加するフィールド

  • description (String) - タスクの詳細説明
  • priority (Integer) - 優先度（1:高, 2:中, 3:低）
  • createdAt (LocalDateTime) - 作成日時
  • updatedAt (LocalDateTime) - 更新日時
Section 01-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Phase 1で作成したToDoアプリのEntityを拡張してください。

【現在のTodo Entity】
- id (Long)
- title (String)
- completed (Boolean)

【追加するフィールド】
1. description (String) - タスクの詳細説明（任意入力、最大500文字）
2. priority (Integer) - 優先度（1:高, 2:中, 3:低、デフォルト2）
3. createdAt (LocalDateTime) - 作成日時（自動設定）
4. updatedAt (LocalDateTime) - 更新日時（自動設定）

【変更が必要なファイル】
1. Todo.java - フィールド追加、getter/setter追加
2. schema.sql - 新規カラム追加
3. TodoMapper.xml - INSERT/UPDATE文の修正
4. form.html - 入力フィールド追加
5. list.html - 一覧表示に優先度と説明を追加

【要件】
- createdAtは新規作成時に自動設定
- updatedAtは更新時に自動設定
- priorityはドロップダウンで選択
- descriptionはtextareaで入力

必要なコードの変更点を提示してください。
lightbulb ポイント

LocalDateTimeを使用する場合、@DateTimeFormatアノテーションでフォーマットを指定します。フォーム入力とのマッピングに注意しましょう。
