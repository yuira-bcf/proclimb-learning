Section 02-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIプロンプト

あなたはJavaプログラマーです。

ToDoアプリにBean Validationを使用したバリデーション機能を実装してください。

【検証ルール】
1. title
   - 必須入力（@NotBlank）
   - 最大100文字（@Size(max=100)）
   - エラーメッセージ: 「タイトルは必須です」「タイトルは100文字以内で入力してください」

2. description
   - 任意入力
   - 最大500文字（@Size(max=500)）
   - エラーメッセージ: 「説明は500文字以内で入力してください」

3. priority
   - 1〜3の範囲（@Min(1), @Max(3)）
   - エラーメッセージ: 「優先度は1〜3で指定してください」

【変更が必要なファイル】
1. Todo.java - バリデーションアノテーション追加
2. TodoController.java - @Valid, BindingResultの使用
3. form.html - th:errorsでエラー表示、エラー時のスタイル
4. messages.properties - カスタムエラーメッセージ（日本語）

【要件】
- エラー時はフォームを再表示し、入力値を保持
- エラーがある入力欄は赤枠で表示（is-invalidクラス）
- エラーメッセージは入力欄の下に赤文字で表示

必要なコードの変更点を提示してください。

auto_awesome 生成されるソースコード

Todo.java（バリデーション追加） 
@NotBlank(message = "タイトルは必須です")
@Size(max = 100, message = "タイトルは100文字以内で入力してください")
private String title;

@Size(max = 500, message = "説明は500文字以内で入力してください")
private String description;

@Min(value = 1, message = "優先度は1〜3で指定してください")
@Max(value = 3, message = "優先度は1〜3で指定してください")
private Integer priority;
TodoController.java（バリデーション実行） 
@PostMapping("/todos")
public String create(
        @Valid @ModelAttribute Todo todo,
        BindingResult bindingResult,  // @Validの直後に配置！
        Model model) {

    if (bindingResult.hasErrors()) {
        // エラーあり → フォーム再表示
        return "todo/form";
    }

    todoService.save(todo);
    return "redirect:/todos";
}
lightbulb コード解説
  • @NotBlank - null、空文字、空白のみを不可（文字列専用）
  • @Size(max=100) - 文字列の長さを制限
  • @Min / @Max - 数値の範囲を指定
  • @Valid - Controllerでバリデーション実行を宣言
  • BindingResult - エラー情報を受け取るオブジェクト
  • hasErrors() - エラーがあるかどうかを判定

lightbulb ポイント

BindingResultは必ず@Validの直後に配置します。間に他の引数があるとエラーになります。
