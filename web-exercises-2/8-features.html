Section 8-1

検索機能実装

キーワード検索や条件検索を実装し、ユーザーが必要なデータを素早く見つけられるようにします。

キーワード検索

実装のポイント

  • 検索フォームの作成（GET リクエスト）
  • SQL の LIKE 句を使用
  • 複数カラムを対象に検索
  • 検索結果のハイライト表示（任意）
SQL例
SELECT * FROM tasks
WHERE title LIKE CONCAT('%', #{keyword}, '%')
   OR description LIKE CONCAT('%', #{keyword}, '%')

条件検索（フィルタリング）

filter_list

ステータスで絞り込み

未完了/完了などの状態でフィルタ

category

カテゴリで絞り込み

プルダウンで選択

date_range

日付範囲で絞り込み

開始日〜終了日を指定

Section 8-2

ソート・ページネーション

一覧画面の並び替えとページ分割を実装します。

ソート機能

ソート項目の例 用途
作成日時（新しい順/古い順） 最新のデータを優先表示
更新日時 最近更新されたデータを優先表示
タイトル（あいうえお順） 名前でアルファベット順に並び替え
優先度（高い順/低い順） 重要なデータを優先表示
実装のポイント

URLパラメータ（?sort=created_at&order=desc）でソート条件を受け取り、動的にSQLを生成します。

ページネーション

実装する機能

  • 1ページあたりの表示件数設定（10件、20件、50件など）
  • ページ番号リンク
  • 「前へ」「次へ」ボタン
  • 総件数・現在のページ番号の表示
SQL例（MySQL）
SELECT * FROM tasks
ORDER BY created_at DESC
LIMIT #{limit} OFFSET #{offset}
