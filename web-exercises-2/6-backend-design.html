Section 6-1

仕様書（機能仕様書）作成

Webアプリケーションの機能仕様を詳細に記述します。実装の指針となる重要なドキュメントです。

システム概要

  • システムの目的 - 何を実現するシステムか
  • システム構成図 - フロント ↔ バックエンド ↔ DB
  • 使用技術一覧 - Spring Boot、MyBatis、MySQL等

機能一覧

項目 内容
機能名 機能の名称
概要 機能の説明
優先度 Must / Should / Could
対応画面 関連する画面

画面一覧・画面遷移図

list

画面一覧

画面名、URL、HTTPメソッド（GET/POST）を一覧化

account_tree

画面遷移図

画面間の遷移を図で表現

各機能の詳細仕様

  • 画面レイアウト - ワイヤーフレームへのリンク
  • 入力項目一覧 - 項目名・型・必須/任意・バリデーションルール
  • 処理フロー - 入力→処理→出力の流れ
  • 出力内容・画面遷移先
  • エラー時の動作 - エラーメッセージ、遷移先

非機能要件

セキュリティ要件
  • 認証方式（Spring Security）
  • CSRF対策
  • XSS対策
  • SQLインジェクション対策
対応環境
  • 対応ブラウザ（Chrome、Firefox、Safari、Edge）
  • 対応デバイス（PC、タブレット、スマートフォン）
Section 6-2

データベース設計書作成

データベースの構造設計を行います。テーブル間のリレーションを明確にしましょう。

ER図

テーブル間のリレーション（1対多・多対多）を図で表現します。

ER図に含める情報

  • テーブル名
  • 主キー（PK）
  • 外部キー（FK）
  • リレーションの種類（1:N、N:N）

テーブル定義

テーブルごとに詳細を記載します。

項目 内容
カラム名 フィールドの名前（英語、スネークケース）
データ型 INT, VARCHAR, TEXT, DATE, TIMESTAMP 等
NULL許可 NOT NULL か NULL 許可か
デフォルト値 初期値がある場合
制約 PK / FK / UNIQUE / NOT NULL
説明 フィールドの用途

インデックス設計

検索性能のために設定するインデックスを記載します。

インデックスの候補
  • 検索条件によく使うカラム
  • ソートに使用するカラム
  • 外部キーカラム

初期データ

マスタデータの初期値やテストデータを用意します。
