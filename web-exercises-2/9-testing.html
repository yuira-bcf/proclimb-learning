Section 9-1

テスト計画

テストを体系的に行うための計画を立てます。何をどこまでテストするかを明確にしましょう。

テスト対象の範囲

  • 全機能（CRUD、検索、ソート、追加機能）
  • 全画面
  • 認証・認可
  • エラー処理

テスト種別

check_circle

機能テスト

各機能が仕様通りに動作するかを確認

web

画面テスト

画面表示、レイアウト、レスポンシブ対応を確認

security

セキュリティテスト

不正アクセス、権限チェック、入力値の検証

テスト環境

項目 内容
ブラウザ Chrome、Firefox、Safari、Edge
OS Windows、macOS
デバイス PC、タブレット、スマートフォン
Section 9-2

テスト実施

テストケースに基づいてテストを実行し、バグを発見・修正します。

テストケースの種類

正常系テスト

想定通りの入力で正常に動作するかを確認

  • 正しいデータで登録できる
  • 一覧が正しく表示される
  • 更新が反映される
異常系テスト

不正な入力でエラー処理されるかを確認

  • 必須項目が空の場合
  • 文字数オーバーの場合
  • 不正な形式の入力
境界値テスト

制限値の境界で正しく動作するかを確認

  • 最小値・最大値
  • 0件・1件・大量データ
  • 文字数の上限

バグ修正

バグ発見時の対応

  1. 再現手順を記録
  2. 原因を特定
  3. 修正を実施
  4. 再テストで確認
  5. 報告書に記録
Section 9-3

テスト報告書作成

テスト結果をドキュメントとしてまとめます。

テストケース一覧

項目 内容
テストID 一意の識別番号（TC-001等）
テスト対象機能 テストする機能名
テスト内容 テストの詳細な手順
期待結果 正常な場合の動作
実施結果 OK / NG
備考 NGの場合の詳細等

テスト結果サマリー

  • 総テストケース数
  • OK数
  • NG数
  • 成功率（OK / 総数 × 100%）

バグ一覧

項目 内容
バグID 一意の識別番号
内容 バグの説明
重大度 高 / 中 / 低
対応状況 修正済み / 未修正 / 対応不要

ブラウザ・デバイス対応状況

確認ポイント
  • 各ブラウザでの動作確認結果
  • レスポンシブ表示の確認結果
  • タッチ操作の確認（スマートフォン）
