第１章　CSS基本設定


1. CSSとは

CSS（シー・エス・エス）とは、Cascading Style Sheets（カスケーディング・スタイル・シート）の略称です。
Webページの見た目（デザイン）を整えるための言語であり、
HTMLで作られた文書のレイアウトや色、文字の大きさ、背景などを指定できます。

HTMLはページの「構造」や「内容（テキストや画像）」を定義する言語。
CSSはそのHTML要素に装飾や配置などの"スタイル"を付ける役割を持っています。
JavaScriptは、HTML/CSSを操作して、リアルタイムにWebページに動きを付ける言語です。


2.CSSを使わないと、どうなるのか

HTMLのみでWebページを作ると、見た目が無機質で無骨な印象のページにしかなりません。
ユーザーにとっては使いにくいページ、となってしまいます。

<img src="1-1-2.CSS を使わないと、どうなるのか.png">

Webページの目的は、集客や認知、成約、購入などです。
それらを見て、感じて、購入ボタンを押したりするのは、最後は必ず人です。

人にとって使いやすいページにする、良い体験が出来るページにする。
それを左右するのが、CSSとなります。



3.CSS適用方法

CSSは、HTMLに読み込まれることで、その装飾をブラウザに反映させます。
全部で3つの適用方法があるので、順番に説明していきます。

3-1. CSS専用ファイルを読み込んで反映（外部スタイル）

index.htmlというHTMLファイルがあるように、CSS「style.css」というファイルを作ることが一般的です。
このstyle.cssをHTMLで読み込ませて、スタイルを反映させる方法が、実務で最も多い手法となります。

フォルダ構成のイメージ

Chapter1_3-1
 ├── index.html
 └── CSS
　　　└── style.css

▶index.html

<!doctype html>
<html lang="ja">
<head>
		<meta charset="utf-8">
		<meta name="description" content="カフェメニュー紹介ページ">
		  <title>Café Latte｜メニュー</title>

  		<!-- CSSファイルの読み込み -->
  		<link rel="stylesheet" href="css/style.css">
</head>

<body>
  <h1>ようこそ Café Latte へ</h1>
  <p>季節限定のメニューをご紹介します。</p>
</body>
</html>


CSSファイルの読込は、下記のコードの部分で行われています。

<link rel="stylesheet" href="css/style.css">


HTMLでは、外部のファイルや、ネット上のリソースを適用したりするのに、
この<link>タグを<head>タグ内に記述して利用・反映します。
CSSも外部ファイルとして存在する場合、上記方法で読み取ってもらいます。

POINT！
「rel="stylesheet"」 は「これから読み込むファイルはCSSですよ」というおまじない。
href の中には CSSファイルまでの道順（ファイルパス） を正確に書きます。
index.html と同じ階層なら href="style.css"、
フォルダ内なら href="css/style.css" のように書き分けましょう

実際のCSSファイルの中身（style.css）

▶style.css


/* style.css */
body {
  background-color: #fdf6ec;
  font-family: "Hiragino Sans", "Helvetica", sans-serif;
}

h1 {
  color: #6b3e26;
  text-align: center;
}

p {
  color: #444;
  line-height: 1.8;
text-align: center;
}

CSSの構文・書き方は後述します。
上記では、<body>の背景をベージュに。<h1>はブラウンで中央寄せ、<p>はグレーで中央寄せ
などのスタイルを指定しています。

4.CSS適用の優先順位

外部スタイル、内部スタイル、インライン。
この3つの適用方法で、たまに同じ要素にそれぞれ違うスタイルが当たる場合があります。
そのような状態のことを「競合する」と言います。

競合しても正しく表示されるように、CSSではどの適用を優先するかという順位が決まっています。

① CSSの適用による優先順位
　　インライン ＞ 内部スタイル＞ 外部スタイル

より詳細な指定が優先されるよう設計されています。


もうひとつ、同じ適用スタイルの中でも、タグ名やクラス、idなどの指定方法にも優先順位があります。
それが「詳細度」と呼ばれるものです。

② 詳細度による優先順位
　　#id名 ＞ .class名 ＞ タグ名

　　　指定方法による詳細度の持ち点例
　　　　#id		 …… 100点
　　　　.class		 …… 10点
　　　　タグ名		 …… 1点
　　　競合した場合は、上記の詳細度の点数の合計が高いものが優先される
　　　　※ 合計点数が同じ場合は、最後に書いたものが優先される

5. CSSの基本の書き方を身につけよう

5-1.CSSの基本文法

CSSは次の3つの要素で構成されています

セレクタ {
  プロパティ: 値;
}


用語				意味					例
セレクタ	（selector）	どの部分にデザインを適用するか		h1, .menu, #header
プロパティ	（property）	何を変えるか（色・大きさなど）		color, font-size
値		（value）		どう変えるか（具体的な内容）		red, 20px

　プロパティ（意味）		　	値（意味）

　font-size（文字の大きさ）		28px（28というピクセル単位の大きさ）
　text-align（1行の中の文字の位置）	center（中央寄せ）

クラスとid
HTMLの属性として、classやidを付与することができます。
これを利用することで、同じタグでもそれぞれ違うスタイルを適用することができます。
classは重複可能で、同じクラス名を別のタグに付与できます。
idはそのページの中では、同じ名前を使うことは出来ません。
idは必ず一意の値（ユニーク）でなければいけません。



7.プロパティと値

プロパティは、指定されたセレクタの要素の「何を変えるのか」、値は「どう変えるのか」を指定します。

例：

p {
  color: #333;        /* 文字の色をグレーに */
  font-size: 18px;    /* 文字サイズを18pxに */
  line-height: 1.6;   /* 行間を広めに */
}



7-1. 1行ごとに

「プロパティ: 値;」の形式で書くのがルールです！
「:（コロン）」と「;（セミコロン）」は必須です。忘れないようにしましょう。


7-2.CSSを書くときのルール

ルール				内容
① 半角英数字で書く		全角や日本語は使えません。必ず半角で！
② 小文字で統一する		CSSは大文字小文字を区別しないが、読みやすさのために小文字が基本。
③ 文の区切りで			改行やスペースをある程度入れることで見やすくなります。
　 改行・スペースは自由		英文と一緒で、単語の途中には入れられません。

 ・1行で書く例

h1 { color: #6b3e26; font-size: 28px; }


1行でまとめてもOKですが、少し読みにくいです。
読みやすくするために改行して書くのが一般的です。

h1 {
  color: #6b3e26;
  font-size: 28px;
}


・複数のプロパティを指定するとき
1つの要素に複数のプロパティを適用するときは、単純に書き足していきます。

h1 {
  color: #6b3e26;
  font-size: 28px;
  text-align: center;
}

第２章　文字・文章の装飾


1. 文字の大きさを変える「font-size プロパティ」

文字の大きさを指定するプロパティが font-size です。
大きさは 数値＋単位（px、rem、%）で指定します。
 キーワード（small、largeなど） でも指定出来ますが、使用することは稀です。


1-1指定方法

指定方法	説明
数値指定	px（ピクセル）やrem（ルート基準）、%（割合）などの単位を使う
キーワード指定	small、medium、large などの7段階キーワードで指定できる


1-2 フォルダ構成

Chapter2_1-2
 └── index.html
 └── style.css


▶style.css

@charset "UTF-8";

html {
	font-size: 100%; /* ページ全体の基準サイズ */
}

h1 {
	font-size: 2rem; /* 基準の2倍サイズ */
}

p {
	font-size: 18px; /* 絶対値指定（18ピクセル） */
}


POINT
<html>タグにもスタイルを指定できます。ここに指定したものは、そのページの基準値となります。
html { font-size:100%; }とすると、原則ブラウザ標準の16pxという単位が基準になります。
「rem」という単位は、このhtmlの値を基準にして、大きさを決定します。

 1-3. 適切な文字サイズの考え方

本文	：	14〜18px が一般的（読みやすさ重視）
見出し	：	本文の約1.5〜2.5倍

デザインの統一感：サイズの種類は2〜5段階以内にまとめる


POINT！
文字サイズがバラバラだとページが雑多に見えます。
使用サイズをあらかじめ決めておくと、統一感が出て見やすくなります



1-4. 文字の太さを変える「font-weight プロパティ」

文字の太さを指定するプロパティです。

値			説明
normal			標準の太さ
bold			太字
lighter / bolder		相対的に細く／太く
数値（100〜900）	より細かく太さを指定（例：font-weight: 700;）

 1-6. 見出しとジャンプ率を考えよう

ジャンプ率とは「見出し」と「本文」の文字サイズの比率のこと。
ジャンプ率が高いとメリハリがあり、低いと落ち着いた印象になります。

ジャンプ率		雰囲気
高い（差が大）		活発・ポップ
低い（差が小）		落ち着いた・上品

POINT！
Webサイトのテーマに合わせて文字の"強弱"をコントロールすると、
読みやすくデザイン性の高いページに仕上がります

2 文字を読みやすくするコツ

2-1.行の高さを整える line-height

● line-height とは
1行ぶんの高さ（行間を含む高さ）を決めるプロパティ。
基本は単位なしの倍率を利用した指定をする。

● 指定方法と意味
指定			例			特徴
単位なし（推奨）		1.6, 1.8			文字サイズ*倍率が1行の高さになる
px / rem / %		28px, 1.6rem, 160%	絶対／相対を固定値で指定、後々の調整が面倒くさくなりやすい
実務では 1.5〜1.8 のレンジが"読みやすい黄金ゾーン"。
日本語の本文は字面が詰まりやすいので 1.6〜1.8 から試すのがおすすめ。

2-2 見出しと本文のバランス

見出し（h1〜h3）はサイズを大きくするだけでなく、line-heightをやや小さめ（例：1.2〜1.4）にすると締まって見える。
本文（p）は大きめ（例：1.6〜1.8）で余裕を持たせ、段落のブロック感を出す。

3-1. text-align とは？

段落や見出しなどインライン要素の整列位置（左右・中央・両端）を決めるプロパティ。
日本語の段落は 既定で左揃え（left） です。

指定値		こうなる	使いどころ

第３章　幅と高さ

1-1. 要素の大きさや領域

要素には、自身が持つ領域があります。
その領域の中に、外余白(margin)、枠線(border)、内余白(padding)、要素が含まれています。
要素自体の幅と高さは「枠線」「内余白」「要素」で、大きさが決まっています。

<img src="3-1-1. 要素の大きさや領域.png">

そして、幅は「width(ウィドスまたはウィズス)」
高さは「height(ハイト)」と表現します。

これらをプロパティとして指定し、適切な単位を含んだ値を指定して、
大きさを調整します。


1-2.Webで使う主な単位（ざっくり早見表）

単位		%
読み方		パーセント
どんな基準？	親要素のサイズ比
種別		相対
例		width: 60%
特徴		親の大きさが変われば、自分も比率に併せて伸縮

単位		rem
読み方		レム
種別		相対
どんな基準？	ルート（html）のフォントサイズ
例		font-size: 1.25rem
特徴		どこでも同じ基準＝扱いやすい◎

単位		vw
読み方		ビューポート幅の%
種別		相対
どんな基準？	ブラウザの画面幅基準
例		width: 50vw
特徴		ブラウザの画面幅に応じて伸縮（横）

単位		vh
読み方		ビューポート高の%
種別		相対
どんな基準？	ブラウザの画面高さ基準
例		height: 50vh
特徴		ブラウザの画面高さに応じて伸縮（縦）

単位		px
読み方		ピクセル
種別		絶対
どんな基準？	物理ピクセル（論理）
例		border-radius: 8px
特徴		親や画面幅に左右されず、常に"固定"の大きさを維持する

まずは px（固定） と %（親に追従） を押さえればOK！
レスポンシブでは vw/vh も便利です。


1-3 実務の指針（困ったらこれ）

テキストサイズ   　…　　基本は rem（例：本文 1rem = 16px）
コンテナ幅	　… 	PCは max-width: 960px など + width: 90% で余白確保
カードの余白 	　… 	rem で文字1文字単位で余白を入れるとバランスが良い
全幅ヒーロー	　…	height: 60vh や min-height: 60vh が扱いやすい

豆知識
max-widthやmin-heightなどは、最大・最低の幅の指定。
%などで指定していると追従するが、最大・最低の大きさを超えたり割ったりしないよう、
その値で止めることが出来る。レスポンシブなどの設定に良く使う。

第４章 余白

余白を上手に活用することで、Webページ全体のバランスや見やすさが格段に上がります。
ここでは外余白のmarginと、内余白のpaddingを使っていきます。


1-1. 余白ってなに？

おさらいとして、要素の領域図を再度確認しましょう。

<img src="4-1-1.余白.png">

1-2.ボックスの構造

要素の構造は、外から順に……
　margin → border → padding → content

名前		説明						
margin		要素の枠線より外側のすきま。隣の要素までの距離になります		
border		内余白を囲んでいる枠線。ここが境界線				
padding	要素と枠線の間の内側のすきま	
content		要素。文字や画像など。			


 POINT：
要素の大きさ（幅と高さ）には「要素」「内余白」「枠線」が含まれます。
背景色（background-color）は「padding」までは塗られますが、「margin」部分には塗られません。


1-3. 外側の余白：margin プロパティ

margin（マージン）は、要素の外のスキマを作ります。
隣の要素と距離を取ったり、中央に配置したりするのに使います。

基本の書き方
margin: 10px;    			 /* すべての方向に10pxの余白 */
margin: 10px 20px;   		/* 上下10px、左右20px */
margin: 10px 20px 30px; 		/* 上10px、左右20px、下30px */
margin: 10px 20px 30px 40px;	/* 上→右→下→左（時計回り） */


個別に指定したいとき
margin-top: 20px;		 /* 上の余白 */
margin-right: 10px;  		/* 右の余白 */
margin-bottom: 30px; 		/* 下の余白 */
margin-left: 50px;    		/* 左の余白 */

<img src="4-1-3. 外側の余白：margin プロパティ.png">



ちょっと便利な使い方
中央寄せ：横幅を決めて margin: 0 auto; と書くと、要素が中央に配置されます。

第５章　線

1-1. 線を引くってどういうこと？

ボーダーは、要素のまわりに描かれる線のことです。
たとえばカードのフチを囲ったり、見出しを目立たせたり、区切り線を引いたりできます。

ボックスの構造（おさらい）
<img src="5-1-1. 線を引くってどういうこと？.png">

2-1. 線の太さを変えよう（border-width）

線の太さを変えるには border-width プロパティを使います。
たとえば「細い線」や「太い線」を指定できます。

border-width: thin;  		/* 細い線 */

第6章 レイアウト

ここまでは要素そのものに対するスタイルの当て方について学んできました。
その要素をどう配置するのか、が最後の決めるべき重要な点となります。

1. Flexbox

「Flexbox」とは Flexible Box Layout（フレキシブル・ボックス・レイアウト）の略で、
要素を箱（ボックス）に入れ、その中で配置を指定することで、柔軟に画面上に並べられるCSSの仕組みです。

これを使うことで、複数の要素を横並びにしたり、縦並びにしたり。
画面幅が変わったらそれに併せて折り返したりなど……
レスポンシブ設計には欠かせない概念となっています。

昔はこの横並びひとつ取っても、プロパティなどの設定が面倒くさかったのが、
いまでは基本は「display:flex」と書くだけで良くなりました。

<img src="6-1. Flexbox.png">

1-1. Flexboxの基本構造

Flexboxでは、親要素と子要素があって初めて機能します。

名前		意味
Flexコンテナ	親箱（全体を包む要素）
Flexアイテム	子箱（中に並ぶ要素）


HTMLで「親の中に子を入れる」形を作り、
CSSで「親に display: flex; をつける」と、子が"横並び"になります。

第7章　 グリッドレイアウト

FlexBoxの他に、もうひとつ横並びを実現できるプロパティがあります。

 1-1. CSSグリッドとは？

CSSグリッド（Grid Layout）は、ページの中に**見えないマス目（グリッド）**を作り、
要素をタイル状にきれいに並べるための仕組みです。

Flexbox		 →	 1列や1行の並びに強い
Grid 		→ 	2次元（タイル状）に並べるのが得意

つまり、「縦×横のレイアウトをまとめてコントロールできる」超便利機能です！

名称			役割
Gridコンテナ（親）	全体を包む箱（マス目の外枠）
Gridアイテム（子）	中に並ぶ各ブロック（マスの中身）

結果：ウィンドウを広げるとボックスが広がり、狭めると自動で縮む！
レスポンシブデザインの第一歩です

POINT
frの前に指定している数字は対比の倍率指定となります。
1fr 2fr 3fr などと指定すると、3列で1:2:3という幅でそれぞれの子要素に違う幅で指定できます。

1-8. まとめ


プロパティ		意味				例
display: grid;		グリッドを使う宣言		-
grid-template-columns	列の幅・数を指定		200px 200px 200px
grid-template-rows	行の高さを指定			200px 200px
gap			すきまの幅を指定		gap: 10px;

grid-template-columns: repeat(3, 1fr); は、1fr を 3回繰り返す。

同じ値のみを指定するならこの記法が楽。
見た目がスッキリ、わかりやすい！





第9章　デフォルトCSSとリセットCSS

同じHTMLを書いたのに、ChromeとFirefoxなど、別々のブラウザで見たら"見た目がちょっと違う"…？
なんてことがよくあります。

それは「ブラウザごとに最初からCSSが入っている」からです。
そんな"ばらつき"をなくすために使うのが 「リセットCSS」 です


1. CSSをリセット

ブラウザには最初から「標準スタイル（デフォルトCSS）」が設定されています。

たとえば：
・タイトルタグ（<h1>）の大きさ
・段落（<p>）の余白
・フォームの見た目
これらはブラウザごとに少しずつ違います。
そのままだと「Chromeで見たときとEdgeで見たときでズレてる!?」なんてことも。

そこで登場するのが リセットCSS 
→ デフォルトのCSSを打ち消して、すべてのブラウザで見た目をそろえます。


2-1. リセットCSSの利用方法

リセットCSSは、Web上で無料公開されているものを使うのが一般的です。
ここでは有名な ress.css（レッス）を使います。




 公開元：
https://github.com/filipelinhares/ress

そうすると、h1やpタグなどに最初から効いているCSSのほとんどが、
程よい感じに削除されます。


2-2. HTMLに読み込む方法

HTMLの<head>内に、以下のように書くだけ！

2-2.フォルダ構成

Chapter8_2-2
 └── index.html
 └── style.css

▶index.html

<!DOCTYPE html>
<html lang="ja">
<head>
	<meta charset="UTF-8" />
	<title>CSSリセットサンプル</title>

	<!-- リセットCSS（外部読み込み） -->
	<link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">

	 <!-- 自分のCSS -->
	<link rel="stylesheet" href="style.css" />
</head>
<body>
	<h1>お問い合わせ</h1>
	<form>
		<label>お名前</label><br>
		<input type="text"><br><br>

		<label>ご意見・ご質問</label><br>
		<textarea></textarea><br><br>

		<button>送信</button>
	</form>
</body>
</html>



POINT：読み込み順が超重要！
ress.cssを 先に読み込む → その後に自作のCSSを読み込む。
逆にしてしまうと、自分のCSSがリセットCSSによって初期化されてしまいます。


間違った例（やってはいけない順番）

<!-- 間違い -->
<link rel="stylesheet" href="style.css" />
<link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">

「リセットは最初、デザインはあとから」が鉄則です！


2-3. 自作CSS（style.css）の例

3. 「ress.css」を使うとこうなる！

比較項目		リセット前					リセット後
入力欄			長さがバラバラ					全ブラウザで統一
テキストの余白		ブラウザごとに違う				すべて同じ
見出しの余白		ブラウザによって大きすぎることも		スッキリ均一に


4. よくある疑問Q&A

Q1. リセットCSSって必ず必要？
	→ 小規模なページでは不要なこともありますが、複数ブラウザ対応なら必須級です。

Q2. どのリセットCSSを使えばいいの？
	→ 有名どころは以下の3つ：

	ress.css（デフォルトの差を最小化）
	ormalize.css（標準に合わせて整える）
	sanitize.css（セキュリティ考慮の新しいタイプ）

Q3. オフライン環境で使いたい場合は？
	→ あらかじめネットからファイルをダウンロードして css/ress.min.css として保存し、
　　　　　 <link>タグでそのファイルパスを読み込ませます。


5. まとめ

ポイント		内容
リセットCSSとは	ブラウザごとの初期スタイルをリセットするCSS
有名なファイル		ress.css や normalize.css
 読み込み順		①リセットCSS → ②自作CSS
 効果			すべてのブラウザで見た目が統一される！
