各ページの内容

home

index.html（トップページ）

第一印象で興味を引くデザイン。自己紹介の要約、制作物のハイライト、各ページへの導線。

person

about.html（自己紹介）

経歴、学んだこと、目指すキャリア、強みなどを紹介。

work

works.html（制作物一覧）

制作したWebサイト・アプリの一覧。スクリーンショット、使用技術、GitHubリンク。

timeline

process.html（制作プロセス）

企画→設計→実装の流れを紹介。各ドキュメントへのリンク集。

code

skills.html（スキルシート）

使用できる技術、ツール、スキルレベルの一覧。

mail

contact.html（連絡先）

連絡先情報、SNSリンク、問い合わせフォーム。

Section 5-3

GitHub Pages公開

GitHubリポジトリにプッシュし、GitHub Pagesで公開します。誰でもアクセスできる状態にしましょう。

公開手順

folder

1. リポジトリ作成

GitHubで portfolio-site という名前でリポジトリを作成

cloud_upload

2. コードをプッシュ

git add → git commit → git push

settings

3. GitHub Pages設定

Settings → Pages → Source を main branch に設定

check_circle

4. 公開確認

https://[username].github.io/portfolio-site/ にアクセス

Section 5-4

README.mdの作成

各リポジトリの README.md は採用担当者が最初に見る場所です。充実した内容を記載しましょう。

READMEに記載する内容

  • プロジェクト名と概要 - 1〜2文で何を作ったか
  • スクリーンショット - 完成画面のキャプチャ画像（必須）
  • 使用技術一覧 - 言語・フレームワーク・ツール
  • 機能一覧 - 実装した機能
  • 工夫したポイント - 技術的チャレンジ
  • ポートフォリオサイトへのリンク
コミットメッセージについて

コミットメッセージは丁寧に書きましょう。「修正」ではなく「ログイン時のバリデーションエラーメッセージを日本語化」のように具体的に記載します。
