Section 4-1

HTML/CSSコーディング

ワイヤーフレームとデザインガイドラインに基づいて、セマンティックHTMLとレスポンシブCSSで実装します。

セマンティックHTML

意味のある適切なタグを使用して、アクセシビリティとSEOを向上させます。

タグ 用途
<header> ページやセクションのヘッダー
<nav> ナビゲーション
<main> ページのメインコンテンツ
<section> 関連するコンテンツのまとまり
<article> 独立したコンテンツ
<footer> ページやセクションのフッター

レスポンシブデザイン

PC・タブレット・スマートフォンに対応したレイアウトを実装します。

desktop_mac

デスクトップ（1024px以上）

12カラムグリッド、広い余白、横並びレイアウト

tablet

タブレット（768px〜1023px）

8カラムグリッド、適度な余白

smartphone

スマートフォン（767px以下）

4カラムグリッド、縦並びレイアウト、ハンバーガーメニュー

CSS - メディアクエリの例
/* モバイルファースト */
.container {
  padding: 16px;
}

/* タブレット */
@media (min-width: 768px) {
  .container {
    padding: 24px;
  }
}

/* デスクトップ */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 32px;
  }
}

Flexbox と Grid

Flexbox

一方向（横または縦）のレイアウトに最適

  • ナビゲーション
  • カードの横並び
  • 中央揃え
Grid

二次元（縦横）のレイアウトに最適

  • ページ全体の構造
  • ギャラリー
  • 複雑なレイアウト
Section 4-2

JavaScript実装

インタラクティブなUI機能を3つ以上実装します。ユーザー体験を向上させる機能を追加しましょう。

実装する機能の例

menu

ハンバーガーメニュー

モバイル時のナビゲーション開閉

photo_library

画像スライダー/カルーセル

画像の自動切り替え、手動切り替え

expand_more

アコーディオン

FAQ等のコンテンツ展開/折りたたみ

animation

スクロールアニメーション

スクロール時の要素フェードイン

open_in_new

モーダルウィンドウ

画像拡大、詳細情報の表示

check_circle

フォームバリデーション

入力値のリアルタイムチェック

DOM操作の基本

document.querySelector()、addEventListener()、classList.toggle() などを活用して、ユーザーの操作に応じた動的な変更を実装します。
