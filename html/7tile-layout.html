Section 7-1

タイル型レイアウトとは

タイル型レイアウトは、カードや画像をグリッド状に並べるレイアウトです。プロジェクトアーカイブサイトや画像ギャラリーでよく使われます。

1.1 タイル型の特徴

Grid

視覚的な一覧性

多くのコンテンツを一度に見渡せます。

Flex

レスポンシブに最適

画面幅に応じて自動的に列数が変化します。

UX

直感的な操作

クリックで詳細を見る、フィルタリングなどの機能と相性が良いです。

CSS Gridでのギャラリー

2.1 基本のグリッドギャラリー

chat プロンプト 
## 機能
タイル型のギャラリーセクションを作成

## 実装要件

CSS Grid（グリッドレイアウト）

display: grid; grid-template-columns: repeat(auto-fill, minmax(最小幅, 最大幅));

display: gridでグリッドコンテナを作成します。grid-template-columnsで列の定義を行い、repeat()関数で繰り返しパターンを指定します。minmax(280px, 1fr)は「最小280px、最大は均等分割」という意味です。

カードUIのスタイリング

position: relative; border-radius: 8px; overflow: hidden;

position: relativeは子要素の絶対配置の基準点になります。border-radiusで角を丸くし、overflow: hiddenではみ出した部分（拡大時の画像など）を隠します。

ホバーエフェクト

transition: transform 0.3s; .gallery-item:hover img { transform: scale(1.05); }

transitionでアニメーションの対象プロパティと時間を指定します。:hover疑似クラスでマウスオーバー時のスタイルを定義し、transform: scale()で要素を拡大縮小します。

2.2 auto-fillとauto-fitの違い

chat プロンプト 
## 機能
auto-fillとauto-fitの違いを説明するサンプル

## 実装要件
- auto-fill: アイテムが少なくても空の列スペースを確保
- auto-fit: アイテムが少ない場合は要素が伸びて埋める
- 両方のパターンを比較できるようにする

## 使用するCSS
- grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))
- grid-template-columns: repeat(auto-fit, minmax(200px, 1fr))
auto-fill vs auto-fit の違い
CSS 
/* auto-fill: 空の列も作成される */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

/* auto-fit: 空の列は潰れて、既存の要素が伸びる */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

menu_book 構文説明

auto-fill

grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

コンテナの幅に収まる限り、できるだけ多くの列を作成します。アイテム数が少なくても、空の列スペースが確保されるため、アイテムは最小幅を保ちます。

auto-fit

grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

auto-fillと同様に列を作成しますが、空の列は0幅に折りたたまれます。そのため、アイテム数が少ない場合は既存のアイテムが伸びてコンテナ全体を埋めます。

使い分け

アイテム数が少ない場合に要素を広げたいならauto-fit、最小幅を保ちたいならauto-fillを使います。

ホバーエフェクト

3.1 オーバーレイ表示

chat プロンプト 
## 機能
ホバー時にオーバーレイを表示するエフェクト

## 実装要件
- 画像の上に半透明の黒オーバーレイを配置
- 通常時は透明、ホバー時にフェードイン
- オーバーレイ内にタイトルと説明を中央配置
- スムーズなトランジション

## 使用するCSS
- position: absolute / inset: 0
- background: rgba(0, 0, 0, 0.7)
- opacity: 0 → 1
- transition: opacity 0.3s
- display: flex / justify-content: center / align-items: center

    flex-direction: column;
    justify-content: center;
    align-items: center;
    color: white;
    opacity: 0;
    transition: opacity 0.3s;
}

.gallery-item:hover .gallery-overlay {
    opacity: 1;
}

.gallery-overlay h3 {
    font-size: 1.25rem;
    margin-bottom: 8px;
}

.gallery-overlay p {
    font-size: 0.875rem;
    opacity: 0.8;

insetプロパティ

inset: 0;

inset: 0はtop: 0; right: 0; bottom: 0; left: 0;の短縮形です。絶対配置された要素を親要素全体に広げる際によく使われます。

opacity によるフェードイン

opacity: 0; transition: opacity 0.3s; /* :hover時 */ opacity: 1;

初期状態でopacity: 0（完全に透明）にしておき、ホバー時にopacity: 1にすることで、フェードインエフェクトを実現します。transitionでスムーズに変化させます。

menu_book 構文説明

transform: translateY()

transform: translateY(100%); /* :hover時 */ transform: translateY(0);

translateY(100%)は要素を自身の高さ分だけ下方向に移動させます。初期状態で画面外に隠し、ホバー時にtranslateY(0)で元の位置に戻すことでスライドインエフェクトを実現します。

transition の ease

transition: transform 0.3s ease;

easeはイージング関数の一つで、開始と終了を緩やかにするアニメーションです。他にease-in（開始が緩やか）、ease-out（終了が緩やか）、linear（一定速度）があります。

filter プロパティ

filter: grayscale(100%); filter: blur(2px); filter: brightness(1.2);

filterプロパティで画像にさまざまな視覚効果を適用できます。grayscale()はグレースケール、blur()はぼかし、brightness()は明るさを調整します。複数のフィルターを組み合わせることも可能です。

Section 7-4

object-fitで画像調整

4.1 object-fitの使い方

chat プロンプト 
## 機能
画像を指定サイズに収めるobject-fitの実装

## 実装要件
- 異なるサイズの画像を統一した枠に収める
- 縦横比を保ちながらトリミング
- トリミング位置を調整可能に

## 使用するCSS
- object-fit: cover（枠を覆う）
- object-fit: contain（枠内に収める）
- object-fit: fill（引き伸ばし）
- object-position: top center（位置調整）
object-fit で画像調整
CSS 
/* 枠内に収まるようにトリミング（よく使う） */
img {
    width: 100%;
    height: 200px;
    object-fit: cover;
}

/* 枠内に全体が見えるように縮小 */
img {
    object-fit: contain;
}

/* object-position: トリミング位置の調整 */
img {
    object-fit: cover;
    object-position: top center;  /* 上部を優先 */
}

Exercise

7章 練習問題

Q1

画像を枠内でトリミングして表示するためのCSSプロパティはどれですか？

  1. object-fit: contain;
  2. object-fit: cover;
  3. overflow: hidden;
  4. background-size: cover;
Answer

7章 解説・解答

Q1の解答: B

object-fit: cover;は、縦横比を保ちながら指定した枠を完全に覆うように画像をトリミングします。containは全体が見えるように縮小、overflow: hiddenははみ出した部分を隠すだけです。
