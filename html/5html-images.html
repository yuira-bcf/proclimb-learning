Section 5-1

画像を表示する <img> タグ

画像を入れるときは <img>タグ を使います。
<img>タグは、終了タグを書く必要がありません。

Section 5-2

基本の書き方

HTML 
<img src="sakura.jpg" alt="美しい桜">
  • src ⇒「source（ソース）」画像ファイルの場所（パス）を指定
  • alt ⇒「alternative text（代替テキスト）」画像が表示されない場合のテキストを指定

画像を入れるときは、必ずsrc と alt の2つの属性がセットになります。

属性名 意味 例
src 画像のファイルパス src="images/photo.jpg"
alt 画像の説明（代替テキスト） alt="海辺の風景"

POINT

もし画像が見つからなかったとき、ブラウザーは alt の内容を代わりに表示します。
また、視覚に障がいがある人が使う読み上げソフト（スクリーンリーダー）も、
alt の文章を読み上げてくれます

例：海の写真を表示する

HTML 
<img src="sea.jpg" alt="真夏の海の写真">

表示結果：

  • 真夏の海の写真（画像が表示されている場合）
  • 「真夏の海の写真」（画像が見つからない場合はテキストで代わりに表示）
Section 5-3

画像の場所を教える「ファイルパス」を覚えよう！

Webページに画像を表示するとき、HTMLファイルは
「どこに画像ファイルがあるか」を道案内（パス）で指定します。

相対パスとは？

相対パスとは、今いる場所（HTMLファイル）から見た画像の場所を指定する書き方です。
たとえば、HTMLファイル（index.html）と画像（sea.jpg）が
同じフォルダにある場合は、下記のような書き方になります。

フォルダ構造 
academia
  └ src
    ├ index.html　　← いまいる場所
    └ sea.jpg　　　 ← ★アクセスするファイル
HTML 
<img src="sea.jpg" alt="夏の海">

同じフォルダにあるなら「画像ファイル名」だけで呼び出せます。

フォルダの中にある画像を指定する

もし画像を別のフォルダ（例：imagesフォルダ）に入れている場合は、
フォルダ名を指定した後に、ファイル名を指定します。

フォルダ構造 
academia
  └ src
    ├ index.html　　← いまいる場所
    └ images　　　 ← 同じ階層のフォルダ
        └ sea.jpg　 ← ★アクセスするファイル
HTML 
<img src="images/sea.jpg" alt="夏の海">

index.htmlと同じ階層にある 「imagesフォルダの中にあるsea.jpgを使うよ！」という意味になります。

1つ上のフォルダにある画像を指定する

もし画像ファイルが1つ上の階層（親フォルダ）にある場合は、
「1つ上に戻る」という意味の 「../」 を使います

フォルダ構造 
academia
  ├ images　　　　 ← 親フォルダと同階層のフォルダ
  │  └ sea.jpg　　  ← ★アクセスしたいファイル
  └ src
      └ index.html　　← いまいる場所
HTML 
<img src="../images/sea.jpg" alt="夏の海">

../ は「1つ上のフォルダに戻る」合図です。階層が深いときは ../../ のように"../"を増やせばOK！

指定したい場所 記号 例
同じフォルダ内 なし（直接ファイル名でOK） index.html
1つ上のフォルダ ../ ../images/photo.jpg

※フォルダの区切りは「/」で表現します。

まとめ

  • 相対パスとは、今いるHTMLファイルから見た「画像までの道」
  • 同じ階層なら：sea.jpg
  • 下のフォルダなら：images/sea.jpg
  • 上のフォルダなら：../sea.jpg
Section 5-4

絶対パスとは

「絶対パス」は、ファイルの"完全な住所"を指定する書き方です。
インターネット上で公開されている画像やデータを使うときに使います。

たとえば、相対パスが「今の場所からの道案内」なのに対して、
絶対パスは「郵便番号付きの住所」を書くイメージです。

書き方の例

HTML 
<img src="http://example.com/images/sea.jpg" alt="夏の海">

「http://」や「https://」から始まり、ドメイン（Webサイトの名前）＋ファイルまでのパスをすべて書きます。
このように、どのページからでも同じ場所のファイルを呼び出せるのが絶対パスの特徴です

POINT

他のWebサイトや別フォルダにあるページへリンクするときも、
絶対パスで書くと「どこからでも正確にアクセス」できます。

種類 特徴 例
相対パス 現在のファイルからの道順を書く images/sea.jpg
絶対パス Web上の住所を全部書く https://example.com/images/sea.jpg
Section 5-5

メールアドレスにリンクをつけよう

Webページから直接メールを送れるリンクを作るには、
<a>タグの中で mailto:（メールトゥと読みます）を使います

基本の書き方

HTML 
<a href="mailto:info@example.com">お問い合わせ</a>

mailto: のあとに送り先のメールアドレスを書くだけ！
リンクをクリックすると、パソコンやスマホに設定されているメールアプリが自動で起動します

記述部分 意味
mailto: 「メールを送るためのリンクですよ」という指示
info@example.com 実際の送り先メールアドレス

つまり、mailto: + メールアドレス ＝ メールリンク完成

まとめ

第5章のまとめ

  • <img>タグで画像を表示（終了タグ不要）
  • src属性で画像の場所、alt属性で代替テキストを指定
  • 相対パスは今いるファイルからの道順
  • 絶対パスはWebの完全な住所
  • mailto:でメールリンクが作れる
