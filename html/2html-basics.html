Section 2-1

HTMLの基本構造

HTMLドキュメントには決まった構造があります。この構造を理解することで、正しいHTMLを書けるようになります。

1.1 タグと要素

HTMLは「タグ」を使ってコンテンツをマークアップします。タグには開始タグと終了タグがあり、これらで囲まれた部分を「要素」と呼びます。

## 使用するHTML
- <p>タグ（段落）を例として使用
タグの構造 HTML 
<開始タグ>コンテンツ</終了タグ>

例：
<p>これは段落です</p>
↑開始タグ   ↑コンテンツ    ↑終了タグ
空要素（自己終了タグ）

一部のタグは終了タグを持ちません。これを「空要素」と呼びます。
例：<img>、<br>、<hr>、<input>、<meta>、<link>

Section 2-2

見出しと段落

見出しと段落は、Webページのコンテンツを構成する最も基本的な要素です。適切に使い分けることで、読みやすいページを作成できます。

見出しは<h1>から<h6>まであり、数字が小さいほど重要度が高くなります。

menu_book 構文説明

見出しタグ（h1-h6）

<h1>最重要見出し</h1> <h2>大見出し</h2> <h3>中見出し</h3> ... <h6>最小見出し</h6>

見出しを表すタグです。h1が最も重要で、h6に向かって重要度が下がります。h1はページに1つだけ使用し、階層構造を崩さないようにします（h1→h3のように飛ばすのはNG）。

段落タグ（p）

<p>段落のテキスト</p>

段落（パラグラフ）を表します。<p>タグで囲まれたテキストは、前後に自動的に余白が入ります。文章のまとまりを示すために使用します。

見出しの使い方の注意点
  • <h1>はページに1つだけ使用する
  • 見出しレベルを飛ばさない（h1→h3はNG、h1→h2→h3が正しい）
  • 見た目の大きさ調整のために見出しタグを選ばない（それはCSSの役割）

画像タグ（img src="" alt=""）

<img src="画像パス" alt="代替テキスト"> <img src="画像パス" alt="説明" width="幅" height="高さ">

画像を表示します。srcで画像ファイルのパスを、altで画像が表示できない場合やスクリーンリーダー用の代替テキストを指定します。どちらも必須属性です。終了タグは不要な空要素です。

属性 必須 説明
src 必須 画像ファイルのパス
alt 必須 代替テキスト（画像が表示できない場合やスクリーンリーダー用）
width 推奨 画像の幅（ピクセル）
height 推奨 画像の高さ（ピクセル）
loading 任意 lazy: 遅延読み込み、eager: 即時読み込み

テーブルの作成

テーブル（表）は、データを行と列で整理して表示するのに使います。価格表やスケジュール表などに適しています。

chat プロンプト 
## 機能
基本的なテーブル（表）を作成

## 実装要件
- thead（ヘッダー行）とtbody（本体行）を分離
- th（見出しセル）とtd（データセル）を使い分け
- 3列×3行程度のデータを表示

## 使用するHTML
- <table>（テーブル全体）
- <thead>, <tbody>（行のグループ化）
- <tr>（行）
- <th>（見出しセル）
- <td>（データセル）

<table>
    <thead>
        <tr>
            <th>商品名</th>
            <th>価格</th>
            <th>在庫</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>りんご</td>
            <td>100円</td>
            <td>50個</td>
        </tr>
        <tr>
            <td>バナナ</td>
            <td>80円</td>
            <td>30個</td>
        </tr>
    </tbody>
</table>
menu_book 構文説明

テーブルタグ（table, tr, th, td）

<table> <thead> <!-- ヘッダー部分 --> <tr><th>見出し</th></tr> </thead> <tbody> <!-- 本体部分 --> <tr><td>データ</td></tr> </tbody> </table>

<table>でテーブル全体を囲み、<tr>で行、<th>で見出しセル、<td>でデータセルを作成します。<thead>と<tbody>でヘッダーと本体を分離すると、構造が明確になります。

タグ 役割
<table> テーブル全体を囲む
<thead> ヘッダー行のグループ
<tbody> 本体行のグループ
<tfoot> フッター行のグループ
<tr> 行（Table Row）
<th> 見出しセル（Table Header）
<td> データセル（Table Data）

基本的なフォームを作成

## 実装要件
- formタグでフォーム全体を囲む
- action属性で送信先を指定
- labelとinputを関連付け
- 必須入力項目を設定
- 送信ボタンを配置

## 使用するHTML
- <form>（フォーム）
- <label>（ラベル）
- <input>（入力欄）
- <button>（ボタン）
基本的なフォーム HTML 
<form action="/submit" method="POST">
    <!-- テキスト入力 -->
    <label for="name">お名前</label>
    <input type="text" id="name" name="name" required>

    <!-- メール入力 -->
    <label for="email">メールアドレス</label>
    <input type="email" id="email" name="email" required>

    <!-- 送信ボタン -->
    <button type="submit">送信</button>
</form>
menu_book 構文説明

フォーム要素（form, input, button, select, textarea）

<form action="送信先URL" method="POST"> <label for="id">ラベル</label> <input type="種類" id="id" name="名前"> <textarea name="名前"></textarea> <select name="名前"> <option value="値">選択肢</option> </select> <button type="submit">送信</button> </form>

<form>でフォーム全体を囲み、actionで送信先、methodで送信方法を指定します。<input>はtype属性で入力の種類（text, email, passwordなど）を変えられます。<label>のforと<input>のidを一致させると関連付けられます。

7.4 バリデーション属性

chat プロンプト 
## 機能
フォームのバリデーション（入力制限）を設定

## 実装要件
- 必須入力の設定
- 文字数の最小・最大値
- 数値の範囲制限
- 正規表現パターンによるチェック
- プレースホルダーで入力例を表示

## 使用するHTML属性
- required（必須）
- minlength, maxlength（文字数）
- min, max（数値範囲）
- pattern（正規表現）
- placeholder（入力例）
入力制限 HTML 
<!-- 必須入力 -->
<input type="text" name="name" required>

<!-- 文字数制限 -->
<input type="text" name="username" minlength="3" maxlength="20">

<!-- 数値範囲 -->
<input type="number" name="age" min="0" max="120">

<!-- パターン（正規表現） -->
<input type="text" name="postal" pattern="[0-9]{3}-[0-9]{4}"
       placeholder="123-4567">

<!-- プレースホルダー（入力例） -->
<input type="text" name="name" placeholder="山田太郎">
Section 2-8

セマンティックHTML

セマンティックHTMLとは、コンテンツの意味（セマンティクス）を適切に表現するタグを使用することです。検索エンジンやスクリーンリーダーがページ構造を理解しやすくなります。

8.1 主要なセマンティックタグ

chat プロンプト 
## 機能
セマンティックHTMLでページ構造を作成

## 実装要件
- header（ヘッダー）とnav（ナビ）
- main（メインコンテンツ）
- article（記事）とsection（セクション）
- aside（サイドバー）
- footer（フッター）

## 使用するHTML
- <header>, <nav>
- <main>（1ページに1つ）
- <article>, <section>
- <aside>, <footer>
ページ構造の例 HTML 
<body>
    <header>
        <nav>
            <!-- ナビゲーションメニュー -->
        </nav>
    </header>

    <main>
        <article>
            <h1>記事タイトル</h1>
            <section>
                <h2>セクション1</h2>
                <p>内容...</p>
            </section>
        </article>

        <aside>
            <!-- サイドバー -->
        </aside>
    </main>

    <footer>
        <!-- フッター -->
    </footer>
</body>
menu_book 構文説明

セマンティックタグ（header, nav, main, article, section, footer）

<header>ヘッダー領域</header> <nav>ナビゲーション</nav> <main>主要コンテンツ（1ページに1つ）</main> <article>独立した記事</article> <section>テーマごとのセクション</section> <aside>補足情報（サイドバー）</aside> <footer>フッター領域</footer>

セマンティック（意味的）なタグを使うと、コンテンツの役割が明確になります。検索エンジンやスクリーンリーダーがページ構造を理解しやすくなり、SEOやアクセシビリティが向上します。<main>は1ページに1つだけ使用します。

タグ 意味・用途
<header> ページやセクションのヘッダー
<nav> ナビゲーションリンクのセット
<main> ページの主要コンテンツ（1ページに1つ）
<article> 独立した記事やコンテンツ
<section> テーマ的なグループ
<aside> 本文から独立した補足情報（サイドバーなど）
<footer> ページやセクションのフッター

8.2 divとspanの使いどころ

divとspan

<div>と<span>は意味を持たない汎用コンテナです。セマンティックタグで適切なものがない場合にのみ使用しましょう。

  • <div>: ブロック要素のグループ化
  • <span>: インライン要素のグループ化
Exercise

2章 練習問題

Q1

画像を挿入するタグとして正しいものはどれですか？

  1. <image src="photo.jpg">
  2. <img src="photo.jpg" alt="写真">
  3. <picture src="photo.jpg"></picture>
  4. <img href="photo.jpg">
Q2

外部リンクを新しいタブで開くために必要な属性はどれですか？

  1. target="_new"
  2. target="_blank"
  3. open="new"
  4. link="external"
Q3

ページの主要コンテンツを囲むセマンティックタグはどれですか？

  1. <content>
  2. <body>
  3. <main>
  4. <article>
Answer

2章 解説・解答

Q1の解答: B

画像は<img>タグで挿入します。src属性で画像パスを、alt属性で代替テキストを指定します。alt属性はアクセシビリティのために必須です。

Q2の解答: B

target="_blank"を指定すると、リンクが新しいタブ（またはウィンドウ）で開きます。セキュリティのため、rel="noopener"も併記することが推奨されます。

Q3の解答: C

<main>タグはページの主要コンテンツを囲むセマンティックタグです。1ページに1つだけ使用できます。<article>は独立した記事を表し、<body>はページ全体のコンテンツを囲みます。
