Section 8-1

Google Maps埋め込み

Google Mapsを埋め込むことで、店舗やオフィスの場所をわかりやすく表示できます。

1.1 埋め込みコードの取得

1

Google Mapsで場所を検索

maps.google.com で表示したい場所を検索します。

2

共有をクリック

「共有」ボタンをクリックし、「地図を埋め込む」タブを選択します。

3

HTMLをコピー

表示されるiframeコードをコピーしてHTMLに貼り付けます。

1.2 レスポンシブ対応

chat プロンプト 
## 機能
Google Mapsをレスポンシブに埋め込む

## 実装要件
- 画面幅に応じてマップサイズが変化
- 16:9のアスペクト比を維持
- iframeを親要素いっぱいに広げる
- 遅延読み込みでパフォーマンス向上

## 使用するAPI/CSS
- Google Maps Embed API
- aspect-ratio: 16 / 9 または padding-bottom: 56.25%
- position: absolute / relative
- loading="lazy"
レスポンシブGoogle Maps HTML 
<div class="map-container">
    <iframe
        src="https://www.google.com/maps/embed?pb=..."
        style="border:0;"
        allowfullscreen=""
        loading="lazy"
        referrerpolicy="no-referrer-when-downgrade">
    </iframe>
</div>
レスポンシブマップ CSS
CSS 
.map-container {
    position: relative;
    width: 100%;
    padding-bottom: 56.25%;  /* 16:9のアスペクト比 */
    height: 0;
    overflow: hidden;
}

.map-container iframe {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
}

/* または aspect-ratio を使用 */
.map-container-modern {
    aspect-ratio: 16 / 9;
}

.map-container-modern iframe {
    width: 100%;
    height: 100%;
}

16:9
aspect-ratio方式（モダン）

menu_book 構文説明

iframe埋め込み（Google Maps）

<iframe src="https://www.google.com/maps/embed?pb=..." loading="lazy"></iframe>

iframe要素は外部コンテンツを埋め込むためのHTMLタグです。src属性に埋め込みURLを指定します。loading="lazy"で遅延読み込みを有効にし、ページの読み込み速度を改善できます。

padding-bottomによるアスペクト比維持

padding-bottom: 56.25%; /* 9/16 = 0.5625 */

従来の手法では、height: 0とpadding-bottomを組み合わせてアスペクト比を維持します。16:9の場合は56.25%（9÷16×100）になります。モダンな方法ではaspect-ratio: 16 / 9が使えます。

Section 8-2

YouTube動画埋め込み

2.1 基本的な埋め込み

chat プロンプト 
## 機能
YouTube動画をレスポンシブに埋め込む

## 実装要件
- 16:9のアスペクト比を維持
- 最大幅を設定して中央配置
- フルスクリーン再生を許可
- 角丸でモダンな見た目

## 使用するAPI/CSS
- YouTube IFrame Player API
- aspect-ratio: 16 / 9
- max-width: 800px
- border-radius: 8px
- allowfullscreen属性
YouTube埋め込み HTML 
<div class="video-container">
    <iframe
        src="https://www.youtube.com/embed/VIDEO_ID"
        title="YouTube video player"
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
        allowfullscreen>
    </iframe>
</div>
menu_book 構文説明

iframe埋め込み（YouTube）

<iframe src="https://www.youtube.com/embed/VIDEO_ID" allowfullscreen></iframe>

YouTube動画を埋め込むには、動画URLを埋め込み用URLに変換します。watch?v=VIDEO_IDをembed/VIDEO_IDに置き換えます。allowfullscreen属性でフルスクリーン再生を許可します。

allow属性

allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"

allow属性はiframe内で許可する機能を指定します。autoplayは自動再生、picture-in-pictureはピクチャインピクチャ機能の許可です。

2.2 パラメータオプション

chat プロンプト 
## 機能
YouTube埋め込みに各種オプションを設定

## 実装要件
- 自動再生（ミュート必須）
- ループ再生
- 開始位置の指定
- 関連動画の非表示
- コントロールバーの表示/非表示

## 使用するパラメータ
- autoplay=1&mute=1
- loop=1&playlist=VIDEO_ID
- start=30（秒指定）
- rel=0
- controls=0
パラメータ付きURL HTML 
<!-- 自動再生（ミュート必須） -->
src="https://www.youtube.com/embed/VIDEO_ID?autoplay=1&mute=1"

<!-- ループ再生 -->
src="https://www.youtube.com/embed/VIDEO_ID?loop=1&playlist=VIDEO_ID"

<!-- 開始位置指定（秒） -->
src="https://www.youtube.com/embed/VIDEO_ID?start=30"

<!-- 関連動画非表示 -->
src="https://www.youtube.com/embed/VIDEO_ID?rel=0"

<!-- コントロール非表示 -->
src="https://www.youtube.com/embed/VIDEO_ID?controls=0"
パラメータ 値 説明
autoplay 0/1 自動再生（mute=1が必要）
mute 0/1 ミュート
loop 0/1 ループ再生
start 秒数 開始位置
rel 0/1 関連動画の表示

2.3 レスポンシブ対応

レスポンシブ動画コンテナ CSS

    aspect-ratio: 16 / 9;
    max-width: 800px;
    margin: 0 auto;
}

.video-container iframe {
    width: 100%;
    height: 100%;
    border: none;
    border-radius: 8px;
}

Section 8-4

メール・電話リンク

4.1 メールリンク（mailto）

chat プロンプト 
## 機能
クリックでメールアプリを起動するリンク

## 実装要件
- メールアドレスを指定
- 件名を事前設定
- 本文のテンプレートを設定
- CC、BCCの設定も可能

## 使用する記法
- mailto:メールアドレス
- ?subject=件名
- &body=本文（%0D%0Aで改行）
- &cc=CCアドレス
- &bcc=BCCアドレス
メールリンク HTML 
<!-- 基本 -->
<a href="mailto:info@example.com">メールを送る</a>

<!-- 件名付き -->
<a href="mailto:info@example.com?subject=お問い合わせ">お問い合わせ</a>

<!-- 件名＋本文 -->
<a href="mailto:info@example.com?subject=お問い合わせ&body=お名前：%0D%0Aご用件：">
    お問い合わせフォーム
</a>

<!-- CC、BCC付き -->
<a href="mailto:info@example.com?cc=sub@example.com&bcc=admin@example.com">
    メールを送る
</a>
menu_book 構文説明

SNSシェアボタンのリンク構造

<a href="mailto:アドレス?subject=件名&body=本文">...</a> <a href="https://twitter.com/intent/tweet?text=テキスト&url=URL">...</a>

メールリンク（mailto:）やSNSシェアボタンは、特殊なURLスキームやシェア用APIのURLを使います。クエリパラメータ（?以降）で件名、本文、シェアするテキストなどを事前設定できます。

URLエンコード

改行: %0D%0A スペース: %20 日本語: encodeURIComponent()で変換

URLに含められない特殊文字はパーセントエンコーディングで変換します。改行は%0D%0A、スペースは%20です。日本語はJavaScriptのencodeURIComponent()で変換できます。

URLエンコード

改行は%0D%0A、スペースは%20でエンコードします。

4.3 PC時のクリック防止

chat プロンプト 
## 機能
PC表示時に電話リンクのクリックを無効化

## 実装要件
- モバイルではタップで発信可能
- PC（768px以上）ではクリック無効
- テキストとして表示のみ
- リンクの下線も削除

## 使用するCSS
- @media (min-width: 768px)
- pointer-events: none
- text-decoration: none
- color: inherit
PC時の電話リンク無効化 CSS
CSS 
/* PC時は電話リンクを無効化 */
@media (min-width: 768px) {
    a[href^="tel:"] {
        pointer-events: none;
        text-decoration: none;
        color: inherit;
    }
}

Section 8-5

OGP設定

OGP（Open Graph Protocol）を設定すると、SNSでシェアされた時にタイトル、説明、画像が表示されます。

5.1 基本的なOGP設定

chat プロンプト 
## 機能
SNSシェア時に表示されるOGP（Open Graph Protocol）を設定

## 実装要件
- タイトル、説明文、URL、画像を設定
- Twitter Card用のメタタグも追加
- 画像は1200x630pxを推奨
- 画像URLは絶対パスで指定

## 使用するメタタグ
- og:title, og:description, og:type
- og:url, og:image, og:site_name
- twitter:card, twitter:site
- twitter:title, twitter:description, twitter:image
OGPメタタグ HTML 
<head>
    <!-- 基本のOGP -->
    <meta property="og:title" content="ページタイトル">
    <meta property="og:description" content="ページの説明文（100文字程度）">
    <meta property="og:type" content="website">
    <meta property="og:url" content="https://example.com/page/">
    <meta property="og:image" content="https://example.com/images/ogp.jpg">
    <meta property="og:site_name" content="サイト名">
    <meta property="og:locale" content="ja_JP">

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:site" content="@username">
    <meta name="twitter:title" content="ページタイトル">
    <meta name="twitter:description" content="説明文">
    <meta name="twitter:image" content="https://example.com/images/ogp.jpg">
</head>
menu_book 構文説明

OGPメタタグ（og:title, og:description, og:image）

<meta property="og:title" content="タイトル"> <meta property="og:description" content="説明文"> <meta property="og:image" content="画像URL">

OGP（Open Graph Protocol）はSNSでシェアされた際に表示される情報を定義するメタタグです。og:titleはタイトル、og:descriptionは説明文、og:imageはサムネイル画像を指定します。画像URLは絶対パスで記述する必要があります。

Twitter Card

<meta name="twitter:card" content="summary_large_image">

X（Twitter）専用のメタタグです。summary_large_imageは大きな画像付きのカード形式、summaryは小さな画像付きの形式で表示されます。OGPタグがあれば一部は自動的に使われますが、明示的に設定することを推奨します。

プロパティ 説明
og:title シェア時に表示されるタイトル
og:description 説明文
og:type website、article など
og:url 正規URL
og:image OGP画像（絶対URL）
OGP画像の推奨サイズ

1200 x 630px が推奨です。このサイズならFacebook、Twitter、LINEなど多くのSNSで綺麗に表示されます。

Section 8-6

ファビコン設定

6.1 ファビコンの設定

chat プロンプト 
## 機能
ブラウザタブやブックマークに表示されるファビコンを設定

## 実装要件
- 基本のファビコン（.ico）を設定
- SVGファビコンでモダン対応
- Apple Touch Icon（180x180px）
- 各種サイズ対応（16x16, 32x32）
- Android用webmanifest

## 使用するlink要素
- rel="icon" href="/favicon.ico"
- rel="icon" type="image/svg+xml"
- rel="apple-touch-icon" sizes="180x180"
- rel="manifest" href="/site.webmanifest"
ファビコン設定 HTML 
<head>
    <!-- 基本的なファビコン -->
    <link rel="icon" href="/favicon.ico">

    <!-- SVGファビコン（モダンブラウザ） -->
    <link rel="icon" type="image/svg+xml" href="/favicon.svg">

    <!-- Apple Touch Icon -->
    <link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">

    <!-- 各種サイズ対応 -->
    <link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
    <link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">

    <!-- Android用 -->
    <link rel="manifest" href="/site.webmanifest">
</head>

6.2 推奨サイズ一覧

用途 サイズ 形式
ブラウザタブ 16x16, 32x32 PNG / ICO
Apple Touch Icon 180x180 PNG
Android 192x192, 512x512 PNG
モダンブラウザ 任意 SVG
ファビコン生成ツール

realfavicongenerator.net などのツールを使うと、1つの画像から各種サイズを自動生成できます。

Q1

SNSでシェアされた時に表示される画像を設定するOGPタグはどれですか？

  1. <meta property="og:picture">
  2. <meta property="og:image">
  3. <meta name="share-image">
  4. <link rel="og-image">
Answer

8章 解説・解答

Q1の解答: B

<meta property="og:image">でOGP画像を設定します。画像URLは絶対パス（https://から始まるURL）で指定する必要があります。
