Section 5-1

エンゲージメントレポートの概要

GA4の「エンゲージメント」レポートでは、ユーザーがサイト内で「何をしたか」を把握できます。どのページが見られているか、どんなイベントが発生しているか、成果（キーイベント）につながっているかを分析します。

エンゲージメントレポートへのアクセス

アクセス手順

  1. 左メニューの「レポート」をクリック
  2. 「ライフサイクル」カテゴリを展開
  3. 「エンゲージメント」を選択

エンゲージメントレポートの構成

レポート名 概要
概要 エンゲージメント全体を俯瞰できるサマリー画面
イベント 発生したイベントの一覧と発生回数
コンバージョン（キーイベント） キーイベントとしてマークしたイベントの状況
ページとスクリーン 閲覧されたページの一覧とパフォーマンス
ランディングページ ユーザーが最初に訪問したページの分析

エンゲージメントとは

GA4における「エンゲージメント」の定義

GA4では、以下のいずれかに該当するセッションを「エンゲージメントのあったセッション」と定義しています。

  • 10秒以上継続したセッション
  • 2つ以上のページビュー（スクリーンビュー）があったセッション
  • キーイベントが発生したセッション

この定義により、単にページを開いただけの「質の低い訪問」と、実際に関心を持って閲覧した「質の高い訪問」を区別できます。

Section 5-2

イベントの理解

GA4はすべてのユーザー行動を「イベント」として記録します。ページの表示も、ボタンのクリックも、フォームの送信も、すべてがイベントです。

イベントの種類

GA4のイベントは、大きく4つの種類に分類されます。

🔧

自動収集イベント

GA4のタグを設置するだけで自動的に収集されるイベント

例：page_view、first_visit、session_start

📊

拡張計測機能イベント

管理画面でオンにすると収集されるイベント

例：scroll、click（外部リンク）、file_download、video_start

⭐

推奨イベント

Googleが定義した名前で実装することが推奨されるイベント

例：login、sign_up、purchase、add_to_cart

✨

カスタムイベント

独自に定義して実装するイベント

例：form_submit、contact_click、pdf_download

主要な自動収集イベント

イベント名 説明
page_view ページが表示されたとき
first_visit ユーザーが初めてサイトを訪問したとき
session_start セッションが開始されたとき
user_engagement ユーザーがページを一定時間以上閲覧したとき

拡張計測機能イベント

拡張計測機能を有効にすると、以下のイベントが自動収集されます。

イベント名 説明 条件
scroll ページを90%までスクロール ページの90%到達時に1回発火
click 外部リンクのクリック 離脱クリックとして計測
view_search_results サイト内検索の実行 URLにクエリパラメータが含まれる場合
video_start YouTube動画の再生開始 埋め込みYouTube動画
video_progress 動画の再生進捗 10%、25%、50%、75%到達時
video_complete 動画の再生完了 動画を最後まで再生
file_download ファイルのダウンロード PDF、文書、圧縮ファイル等のリンククリック
form_start フォーム入力の開始 フォーム要素への最初の入力
form_submit フォームの送信 フォームの送信ボタンクリック
拡張計測機能の有効化

拡張計測機能は、GA4の管理画面「データストリーム」から設定します。

  1. 管理 → プロパティ設定 → データの収集と修正 → データストリーム
  2. 該当するストリームを選択
  3. 「拡張計測機能」をオン

イベントレポートの見方

「イベント」レポートでは、発生したイベントの一覧と、それぞれの発生回数を確認できます。

表示される主な指標

  • イベント数：そのイベントが発生した回数
  • 総ユーザー数：そのイベントを発生させたユーザー数
  • ユーザーあたりのイベント数：1ユーザーあたりの平均発生回数
Section 5-3

キーイベント（コンバージョン）

キーイベントは、ビジネスにとって重要なイベントを特別にマークしたものです。以前は「コンバージョン」と呼ばれていました。

キーイベントとは

キーイベントの定義

キーイベントとは、ビジネス目標の達成に直結する重要な行動を指します。

  • ECサイト：購入完了（purchase）
  • BtoBサイト：資料請求、問い合わせ送信
  • メディアサイト：会員登録、メルマガ登録
  • コーポレートサイト：採用応募、お問い合わせ

キーイベントの設定方法

イベントをキーイベントとしてマークする方法は2つあります。

方法1：管理画面から設定

  1. 管理 → プロパティ設定 → データの表示 → キーイベント
  2. 「新しいキーイベント」をクリック
  3. キーイベントとしてマークするイベント名を入力
  4. 保存

方法2：イベントレポートから設定

  1. レポート → エンゲージメント → イベント
  2. キーイベントにしたいイベントの行を見つける
  3. 「キーイベントとしてマークを付ける」トグルをオン

キーイベントの設計ポイント

設計時の注意点
  • 数を絞る：すべての行動をキーイベントにすると、重要性が薄まる
  • ビジネス目標と紐づける：「成果」と言えるものだけを設定
  • マイクロCVとマクロCV：最終成果（購入）と中間成果（カート追加）を区別
種類 説明 例
マクロCV 最終的なビジネス成果 購入、契約、申込完了
マイクロCV 成果に至る途中の重要な行動 カート追加、資料ダウンロード、会員登録

キーイベントレポートの活用

キーイベントレポートでは、以下の情報を確認できます。

  • キーイベント数：発生した回数
  • 総ユーザー数：キーイベントを達成したユーザー数
  • キーイベント率：セッションまたはユーザーに対する達成率

POINT

集客レポートやページレポートと組み合わせることで、「どのチャネルからのユーザーが成果につながりやすいか」「どのページを経由すると成果が出やすいか」といった分析が可能になります。

Section 5-4

ページとスクリーン

「ページとスクリーン」レポートでは、ユーザーが閲覧したページごとのパフォーマンスを確認できます。どのページが人気があり、どのページがエンゲージメントを生んでいるかを把握します。

ディメンションの選択

ページとスクリーンレポートでは、以下のディメンションでページを分類できます。

ディメンション 説明 例
ページパスとスクリーンクラス URLのパス部分 /products/item001
ページタイトルとスクリーン名 HTMLのtitleタグの内容 商品詳細ページ | サイト名
ページパス + クエリ文字列 クエリパラメータを含むパス /search?q=keyword
コンテンツグループ 設定したグループ分け 商品ページ、記事ページ

主要な指標

指標 説明
表示回数 ページが表示された回数（ページビュー）
ユーザー そのページを閲覧したユーザー数
ユーザーあたりのビュー 1ユーザーあたりの平均表示回数
平均エンゲージメント時間 そのページでの平均滞在時間
イベント数 そのページで発生したイベントの総数
キーイベント そのページで発生したキーイベント数

ページ分析のポイント

分析の観点
  • 人気ページの把握：表示回数が多いページはサイトの主力コンテンツ
  • エンゲージメントの質：滞在時間が長いページはコンテンツが読まれている
  • 成果への貢献：キーイベント数が多いページは成果に寄与している
  • 改善候補の発見：PVが多いがエンゲージメント時間が短いページは改善候補

分析例

「表示回数は上位だが、平均エンゲージメント時間が極端に短いページ」は、ユーザーの期待と内容のミスマッチがある可能性があります。ページの内容や導線を見直す候補となります。

Section 5-5

ランディングページ分析

ランディングページとは、ユーザーがサイトに訪問した際に最初に表示されたページのことです。このページの質がその後の行動を大きく左右します。

ランディングページレポートの特徴

ランディングページの重要性
  • ユーザーの第一印象を決めるページ
  • SEOや広告の着地点として評価される
  • 離脱するか回遊するかの分岐点

ランディングページの主要指標

指標 説明 評価ポイント
セッション そのページから開始したセッション数 流入の多いページを把握
ユーザー そのページから流入したユーザー数 新規獲得の入口として評価
新しいユーザー 初回訪問でそのページに着地したユーザー数 SEOや広告の獲得ページとして評価
平均エンゲージメント時間 着地後の平均滞在時間 コンテンツの質を評価
キーイベント そのページ経由で発生したキーイベント数 成果への貢献度を評価

ランディングページ改善の観点

🚪

集客ページの最適化

SEOや広告で獲得したユーザーを逃さないためのCTA配置や導線設計

⚡

ファーストビューの改善

ページ読み込み速度、ファーストビューでの訴求内容の最適化

🔗

内部リンクの設計

回遊を促すための関連コンテンツへの導線

POINT

ランディングページレポートと集客レポートを組み合わせることで、「どのチャネルからどのページに着地しているか」「チャネルとページの相性」を分析できます。

Section 5-6

エンゲージメント指標の活用

GA4のエンゲージメント指標を正しく理解し、活用することで、サイトの改善ポイントを的確に把握できます。

主要なエンゲージメント指標

指標 定義 活用方法
エンゲージメント率 エンゲージメントのあったセッションの割合 訪問の質を評価。低い場合はコンテンツやUIの改善を検討
平均エンゲージメント時間 アクティブにページを閲覧していた平均時間 コンテンツがしっかり読まれているかを評価
セッションあたりのイベント数 1セッションで発生したイベントの平均数 ユーザーのアクティブ度を評価
セッションあたりのページビュー 1セッションで閲覧されたページの平均数 サイト内回遊の度合いを評価

UAの「直帰率」との違い

直帰率から「エンゲージメント率」へ

UAでは「直帰率」（1ページだけ見て離脱した割合）が重要な指標でしたが、GA4では「エンゲージメント率」に置き換わりました。

  • UA直帰率：1ページだけで離脱 = 悪いとは限らない
  • GA4エンゲージメント率：滞在時間や行動も考慮した「質」の指標

1ページしか見なくても、10秒以上滞在していれば「エンゲージメントあり」と判定されます。

セグメント別分析の重要性

エンゲージメント指標は、セグメント別に比較することで価値が高まります。

比較分析の例
  • デバイス別：モバイルとPCでエンゲージメント率に差があるか
  • チャネル別：どの流入元からのユーザーがよく読んでいるか
  • 新規/リピーター別：リピーターの方がエンゲージメントが高いか
  • ページカテゴリ別：どのコンテンツ種別がエンゲージメントが高いか

まとめ

エンゲージメントレポートは、ユーザーがサイト内で「何をしたか」を把握する基盤です。

  • イベントで行動を把握し、キーイベントで成果を測る
  • ページとスクリーンでコンテンツの人気と質を評価
  • ランディングページで流入の入口を最適化
  • エンゲージメント指標で訪問の質を総合的に評価
