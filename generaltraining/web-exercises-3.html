12. CSVエクスポート

14. ユーザー別ToDo管理
15. 権限管理（管理者・一般ユーザー）

【出力形式】
以下の項目を含めて仕様書を作成してください：
1. システム概要
2. 機能一覧（機能ID、機能名、概要、画面/API）
3. 画面一覧（画面ID、画面名、URL、概要）
4. API一覧（エンドポイント、メソッド、概要、リクエスト/レスポンス）
5. 非機能要件（セキュリティ、パフォーマンス、可用性）
6. 制約事項・前提条件
lightbulb 「画面遷移図も含めてください」「Markdown形式で出力してください」など、出力形式を指定すると、より使いやすい仕様書が生成されます。
08 DB設計書

DB設計書生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、DB設計書を生成できます。

edit_note AIへの入力プロンプト
以下のToDoアプリケーションのDB設計書を作成してください。

【アプリケーション概要】
- Spring Boot + MyBatis + MySQL で構築したToDoアプリケーション
- ユーザー認証、ToDo管理、カテゴリ管理、監査ログ機能を実装

【必要なテーブル】
1. users（ユーザー情報）
   - ログイン認証用
   - 権限管理（ADMIN/USER）

2. todos（ToDo情報）
   - 基本項目：タイトル、説明、ステータス、優先度、期限
   - ユーザーとの関連付け
   - カテゴリとの関連付け
   - ファイル添付対応

3. categories（カテゴリ情報）
   - ToDoの分類用
   - ユーザーごとのカテゴリ管理

4. audit_logs（監査ログ）
   - 操作履歴の記録
   - ユーザー、操作種別、対象データ、日時

5. attachments（添付ファイル）
   - ToDoに紐づくファイル情報

【出力形式】
以下の項目を含めてDB設計書を作成してください：
1. ER図（テキストベースでの関連図）
2. テーブル定義書（各テーブルごとに）
   - テーブル名、論理名、概要
   - カラム一覧（カラム名、データ型、NULL許可、デフォルト値、説明）
   - 主キー、外部キー、インデックス
3. リレーション説明
4. 初期データ（マスタデータ）
5. 命名規則
lightbulb 「CREATE TABLE文も出力してください」「MySQLの構文で」など、使用するDBを指定すると、すぐに使えるSQL文も生成されます。
09 テスト報告書

テスト報告書生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、テスト報告書を生成できます。

edit_note AIへの入力プロンプト
以下のToDoアプリケーションのテスト報告書を作成してください。

【アプリケーション概要】
- Spring Boot + MyBatis + Thymeleaf で構築したToDoアプリケーション
- 基本CRUD + 26の追加機能を実装

【テスト対象機能】
■ 認証・認可
- ログイン/ログアウト
- 権限に応じたアクセス制御

■ ToDo基本操作
- 一覧表示、新規登録、編集、削除
- バリデーション（必須入力、文字数制限）

■ 検索・フィルタ
- キーワード検索
- ステータス絞り込み
- ソート（日付、優先度）

■ 追加機能
- ページネーション
- 一括削除
- CSVエクスポート
- ファイルアップロード
- メール送信

■ API
- REST APIエンドポイント
- エラーレスポンス

【出力形式】
以下の項目を含めてテスト報告書を作成してください：
1. テスト概要（目的、期間、環境）
2. テスト項目一覧
   - テストID、機能名、テスト観点、手順、期待結果、実施結果、備考
3. テスト結果サマリ
   - 総テスト数、成功数、失敗数、未実施数
   - 機能別の結果集計
4. 発見した不具合一覧（あれば）
   - 不具合ID、概要、重要度、対応状況
5. 特記事項・課題
6. 結論と推奨事項
lightbulb 「境界値テスト」「異常系テスト」など、テスト種別を指定すると、より網羅的なテスト項目が生成されます。
