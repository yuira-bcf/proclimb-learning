プロンプトガイド

📅 作成日：2026-XX-XX 👤 作成者：（氏名を入力） 📌 Ver 1.0
🎯 このガイドの目的 本演習の成果物はすべてAI（ChatGPT/Claude等）を活用して制作します。
このガイドでは、各ドキュメント・各工程で使用するプロンプトのベストプラクティスを紹介します。

📌 プロンプト作成の基本原則

1. 具体的に指示する
曖昧な指示ではなく、出力形式・文量・含めるべき項目を具体的に指定する。
2. コンテキストを与える
サービス名・ターゲット・これまでに作成した情報を毎回伝える。
3. 段階的に進める
1つのプロンプトで全部を出そうとせず、ステップを分けて会話形式で詰めていく。
4. 出力を検証・修正する
AIの出力をそのまま使わず、自分で確認・修正して最終版にする。
5. フィードバックを重ねる
「もっと具体的に」「表形式で」「〇〇を追加して」と追加指示で品質を上げる。
6. 自分の言葉に変換する
最終的には自分が理解し説明できる内容にする。面接で聞かれても答えられるように。

企画段階

🤖 テーマ決め
Webサービスのテーマを考えています。以下の条件に合うWebサービスのアイデアを5つ提案してください。 条件： ・Webサイト（フロントエンド）＋管理システム（Spring Boot）の両方が必要 ・CRUD操作とユーザー認証が含まれる ・就職活動のポートフォリオとして見栄えがする ・実用性がある（実際に使えそうなサービス） 各アイデアについて以下を記載してください。 ・サービス名の案 ・サービス概要（2〜3文） ・Webサイト部分の内容 ・管理システム部分の内容 ・技術的な面白さ

フロントエンド実装段階

🤖 HTML/CSS コード生成
以下のデザインガイドラインとワイヤーフレームに基づいて、トップページのHTMLとCSSを作成してください。 【デザインガイドライン】 （デザインガイドラインの内容を貼り付け） 【ワイヤーフレームの構成】 - ヘッダー：ロゴ（左）、ナビゲーション（右） - ヒーローセクション：キャッチコピー、サブテキスト、CTAボタン - サービス紹介：3カラムのカード - ... 【技術要件】 ・セマンティックHTMLを使用 ・レスポンシブ対応（モバイルファースト） ・CSS変数でカラーパレットを定義 ・Flexbox / Grid を使用したレイアウト

バックエンド実装段階

🤖 Spring Boot CRUD 生成
Spring Bootで以下のCRUD機能を実装してください。 【テーブル定義】 （DB設計書のテーブル定義を貼り付け） 【実装してほしいもの】 1. Entity クラス 2. Mapper インターフェース（MyBatis） 3. Mapper XML 4. Service クラス 5. Controller クラス 6. Thymeleaf テンプレート（一覧画面・登録画面・編集画面） 【要件】 ・バリデーション（Bean Validation）を含める ・エラーメッセージは日本語 ・ページネーション対応 ・フラッシュメッセージで操作結果を表示

ドキュメント作成段階

🤖 ドキュメントHTML化
以下の内容をHTMLページとして整形してください。 【ページの種類】（企画提案書 / テスト報告書 etc.） 【内容】 （これまでAIで生成した文章を貼り付け） 【HTML要件】 ・ポートフォリオサイトのCSSに合わせたマークアップ ・目次（ページ内リンク）を含める ・表はtableタグで構造化 ・コードブロックはpreタグで表示 ・レスポンシブ対応

⚠️ 注意事項

AIの出力をそのまま使わないでください AIが生成した内容は必ず以下を確認してください。
・情報の正確性（特に技術的な内容）
・自分のプロジェクトとの整合性
・文章の自然さ・読みやすさ
・コードの動作確認
