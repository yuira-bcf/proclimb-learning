プロジェクトアーカイブ作成ガイド

企画から実装、公開まで。就職活動に使えるプロジェクトアーカイブを作成するための完全ガイドです。

プロジェクトアーカイブとは

プロジェクトアーカイブとは、あなたのスキルや実績を示す作品集です。就職活動において、履歴書や職務経歴書だけでは伝えきれない「実際に何ができるのか」を具体的に示すことができます。本演習では、企画・設計・デザイン・実装のすべての工程を経験し、その成果物をプロジェクトアーカイブとしてまとめます。

visibility
スキルの可視化
コードや成果物を通じて、自分の技術力を具体的に示すことができます。
psychology
思考プロセスの提示
企画書や設計書により、問題解決の思考プロセスをアピールできます。
trending_up
成長の証明
学習の軌跡を見せることで、成長意欲と継続力を伝えられます。
handshake
面接での会話材料
具体的な成果物があることで、面接での質問に自信を持って答えられます。

作成ステップ

1
企画フェーズ
テーマ決定・企画立案
作成するWebサービスのテーマを決め、企画書をまとめます。ターゲットユーザーやサービスのコンセプトを明確にし、何を作るのかを具体化します。
企画提案書 マーケットリサーチ ペルソナ設定

checklist やること

  • テーマ選定：自分が興味を持てる業種・サービスを選ぶ（例：カフェ、学習塾、ECサイトなど）
  • 3C分析：Customer（顧客）・Competitor（競合）・Company（自社）の観点で市場を分析
  • SWOT分析：強み・弱み・機会・脅威を整理
  • STP分析：セグメンテーション・ターゲティング・ポジショニングを決定
  • ペルソナ作成：具体的な1人のユーザー像を設定（名前・年齢・職業・悩みなど）
ポイント：AIを活用して市場分析やペルソナ作成を効率化できます。ただし、AIの出力をそのまま使わず、自分のサービスに合わせて調整することが重要です。
description 企画提案書テンプレート
2
設計フェーズ
サイト構造・画面設計
サイト全体の構造（サイトマップ）を設計し、各画面のワイヤーフレームを作成します。ユーザーの導線や情報の配置を検討します。
サイトマップ ワイヤーフレーム 画面遷移図

checklist やること

  • サイトマップ作成：全ページの階層構造を図式化（トップ、サービス紹介、会社概要、お問い合わせなど）
  • ワイヤーフレーム作成：各ページのレイアウトを線画で設計（Figmaを使用）
  • ナビゲーション設計：ユーザーが迷わない導線を考える
  • コンテンツ整理：各ページに掲載する情報を洗い出す
  • レスポンシブ検討：PC/タブレット/スマホでの表示を考慮
ポイント：ワイヤーフレームは「骨格」です。色やデザインは後回しにして、情報の配置と優先順位に集中しましょう。ユーザーが最初に見るべき情報は何かを意識してください。
account_tree サイトマップテンプレート
3
デザインフェーズ
デザインガイドライン作成
カラーパレット、タイポグラフィ、UIコンポーネントなどのデザインルールを策定します。Figmaを使ってモックアップを作成します。
デザインガイドライン Figmaモックアップ コンポーネント設計

checklist やること

  • カラーパレット決定：メインカラー、サブカラー、アクセントカラー、テキストカラーを設定
  • タイポグラフィ設定：フォントファミリー、サイズ、ウェイトのルールを決める
  • コンポーネント設計：ボタン、カード、フォームなどの共通UIパーツを定義
  • 余白・グリッドルール：統一感のあるレイアウトのための基準を設定
  • モックアップ作成：Figmaで主要ページのデザインカンプを作成
ポイント：デザインに迷ったら、競合サイトや参考サイトを3つ以上分析しましょう。良いと思った要素を自分のデザインに取り入れることで、プロのような仕上がりになります。
palette デザインガイドラインテンプレート
4
フロント実装フェーズ
Webサイト実装
HTML/CSS/JavaScriptを使ってWebサイトを構築します。レスポンシブ対応やアニメーションなど、ユーザー体験を向上させる実装を行います。
HTML/CSS JavaScript レスポンシブ対応

checklist やること

  • HTML構造化：セマンティックなHTMLでページの骨格を作成（header, main, section, footerなど）
  • CSS設計：デザインガイドラインに基づいてスタイルを適用（CSS変数の活用推奨）
  • レスポンシブ対応：メディアクエリを使って各デバイスサイズに対応
  • JavaScript実装：ハンバーガーメニュー、スムーススクロール、フォームバリデーションなど
  • アクセシビリティ：alt属性、aria属性、キーボード操作への対応
ポイント：AIにコードを生成してもらう際は、デザインガイドラインとワイヤーフレームの情報を詳しく伝えましょう。「このボタンはメインカラー#FF3B30で、角丸は8px」のように具体的に指示すると精度が上がります。
5
バックエンド設計フェーズ
機能設計・DB設計
管理システムの機能要件を定義し、データベース設計を行います。テーブル定義やER図を作成し、実装の準備をします。
仕様書 DB設計書 ER図

checklist やること

  • 機能一覧作成：管理画面で必要な機能をリストアップ（CRUD、検索、認証など）
  • 画面仕様書：各画面の入力項目、ボタン、遷移先を定義
  • テーブル設計：必要なテーブル、カラム、データ型を決定
  • ER図作成：テーブル間のリレーションを図式化
  • API設計：エンドポイント、リクエスト/レスポンス形式を定義（REST API形式）
ポイント：DB設計は後から変更するとコストが高いです。「将来こんな機能が必要になるかも」と思ったら、設計段階で考慮しておきましょう。正規化のルールも意識してください。
storage DB設計書テンプレート
6
バックエンド実装フェーズ
Spring Boot アプリケーション開発
Java / Spring Boot を使って管理システムを開発します。CRUD機能、認証機能、検索機能などを実装します。
Spring Boot Thymeleaf MyBatis

checklist やること

  • プロジェクト作成：Spring Initializrで依存関係を設定（Web, Thymeleaf, MyBatis, MySQL）
  • Entity/DTO作成：テーブルに対応するJavaクラスを作成
  • Mapper作成：MyBatisでSQL文を記述
  • Service作成：ビジネスロジックを実装
  • Controller作成：URLマッピングと画面遷移を実装
  • View作成：Thymeleafテンプレートで画面を作成
  • 認証機能：Spring Securityでログイン/ログアウトを実装
ポイント：まずは最小限の機能（例：一覧表示と登録）を動かしてから、編集・削除・検索と機能を追加していきましょう。一度に全部作ろうとすると、デバッグが困難になります。
7
テストフェーズ
テスト・デバッグ
作成したシステムのテストを行い、バグを修正します。テストケースを作成し、品質を担保します。
テスト報告書 バグ修正 品質確認

checklist やること

  • テストケース作成：各機能の正常系・異常系のテストパターンを洗い出す
  • 機能テスト：すべてのボタン、リンク、フォームが正しく動作するか確認
  • バリデーションテスト：入力値の検証が正しく動作するか確認
  • ブラウザテスト：Chrome, Firefox, Safari, Edgeで表示崩れがないか確認
  • レスポンシブテスト：PC, タブレット, スマホで正しく表示されるか確認
  • バグ修正：発見したバグを修正し、再テスト
ポイント：テストは「壊そうとする」気持ちで行いましょう。「普通の使い方」だけでなく、「空文字で送信」「長すぎる文字列」「特殊文字」など、想定外の操作も試してください。
bug_report テスト報告書テンプレート
8
公開フェーズ
プロジェクトアーカイブ公開・振り返り
GitHub Pagesでプロジェクトアーカイブを公開し、プロジェクト全体を振り返ります。学んだことや改善点を技術記事としてまとめます。
GitHub Pages 振り返り 技術記事

checklist やること

  • GitHub Pages設定：リポジトリの設定からGitHub Pagesを有効化
  • README作成：プロジェクトの概要、使用技術、セットアップ手順を記載
  • 振り返り作成：プロジェクトで学んだこと、苦労したこと、改善点をまとめる
  • 技術記事執筆：開発中に得た知見を記事にまとめる（Qiita, Zennなど）
  • 最終確認：公開されたサイトをすべてのリンクをクリックして確認
  • 面接準備：「なぜこの技術を選んだか」「苦労した点」などの質問に答えられるよう準備
ポイント：振り返りは「良かった点」「改善点」「次に活かすこと」の3つの視点でまとめましょう。失敗談も正直に書くことで、成長意欲をアピールできます。面接では「この経験から何を学んだか」を語れることが重要です。
rate_review 振り返りテンプレート

成功のためのヒント

1
AIを活用しよう
ChatGPTやClaudeを活用して、企画書やコードの作成を効率化しましょう。ただし、AIの出力はそのまま使わず、必ず自分で確認・修正してください。
2
ドキュメントを丁寧に
コードだけでなく、企画書や設計書も重要です。「なぜそう設計したのか」を説明できるドキュメントを残しましょう。
3
段階的に進めよう
完璧を目指さず、まずは動くものを作りましょう。機能を段階的に追加していくアプローチが効果的です。
4
GitHubで管理しよう
こまめにコミットし、変更履歴を残しましょう。コミットメッセージは分かりやすく書くことで、開発プロセスをアピールできます。
5
面接を意識しよう
「なぜこの技術を選んだのか」「どんな困難があったか」など、面接で聞かれそうな質問を想定して準備しましょう。
6
フィードバックを活かそう
講師や同期からのフィードバックを積極的に取り入れ、継続的に改善していきましょう。
さっそく始めましょう
プロジェクトアーカイブテンプレートを使って、あなただけのプロジェクトアーカイブを作成しましょう。テンプレートには必要な構成がすべて揃っています。
テンプレートを開く download テンプレートをダウンロード

folder_open VSCodeでプロジェクト設定

1
ZIPファイルを解凍
ダウンロードしたportfolio-templates.zipを任意の場所に解凍します。
2
VSCodeでフォルダを開く
VSCodeを起動し、メニューから「ファイル」→「フォルダーを開く」を選択。解凍したportfolio-templatesフォルダを選択します。
3
Live Server拡張機能をインストール
VSCode左側の拡張機能アイコン（四角が4つ）をクリックし、「Live Server」を検索してインストールします。
4
Live Serverで確認
index.htmlを右クリックし「Open with Live Server」を選択。ブラウザでプレビューしながら編集できます。
