# 制作 ガイド

## 就職活動プロジェクトアーカイブ制作（全32日間）

---

## 本演習について

本演習はこれまでの全授業（HTML/CSS、JavaScript、UI/UX、Figma、マーケティング、GA4、セキュリティ、システム開発、Java、Spring Framework、プロンプトプログラミング、ChatGPT活用）の集大成として、**就職活動で使えるプロジェクトアーカイブを個人で制作する**演習です。

授業（講義）はありません。全日程が**個人制作時間**です。
これまでの授業で学んだすべてが活用できます。必要なチュートリアルを自分でピックアップして参照してください。

---

### 制作ドキュメント一覧（全10種）

以下のドキュメントをすべてHTMLページとして制作し、プロジェクトアーカイブサイト内に配置します。

---

#### 01. 企画提案書

クライアントへの提案を想定した企画書。

**記載内容:**
- プロジェクト概要（サービス名・コンセプト・ターゲット）
- 背景と課題（なぜこのサービスが必要か）
- 提案するソリューション
- 期待される効果
- スケジュール概要
- 使用技術一覧

**活用するチュートリアル:** マーケティング基礎、Webマーケティング基礎、ChatGPTアイデア発想

---

#### 02. マーケットリサーチ

市場調査・競合分析の結果をまとめた資料。

**記載内容:**
- 3C分析（Customer / Competitor / Company）
- SWOT分析
- STP分析（Segmentation / Targeting / Positioning）
- 4P / 4C 分析
- 競合サイト比較（3サイト以上のスクリーンショット付き分析）
- 差別化ポイント

**活用するチュートリアル:** 戦略分析（3C・SWOT）、戦術（STP分析）、戦術（4P・4C）

---

#### 03. ペルソナシート

ターゲットユーザーを具体化した人物像。

**記載内容:**
- 基本情報（名前・年齢・性別・職業・居住地・年収）
- ライフスタイル（平日の過ごし方・休日の過ごし方）
- 価値観・性格
- 情報収集の方法（SNS・検索・口コミなど）
- 抱えている課題・悩み
- サービスに期待すること
- カスタマージャーニーマップ（認知→興味→比較→行動→共有）

**活用するチュートリアル:** ペルソナ設計、カスタマージャーニー

---

#### 04. サイトマップ

サイト全体の構造を示す資料。

**記載内容:**
- ページ階層図（トップ → 下層ページの関係）
- 各ページの目的と主要コンテンツ
- ユーザーフロー（ペルソナの行動動線）
- URL設計

**活用するチュートリアル:** 情報アーキテクチャ、Web制作フロー

---

#### 05. ワイヤーフレーム

各ページのレイアウト設計図。

**記載内容:**
- 全ページのワイヤーフレーム（PC版）
- 主要ページのスマートフォン版
- 各要素の配置意図の説明
- Figmaの画面キャプチャまたはHTMLで再現

**活用するチュートリアル:** Figma全般、レイアウト・グリッド、UIコンポーネント

---

#### 06. デザインガイドライン

デザインの統一基準を定めた資料。

**記載内容:**
- デザインコンセプト（キーワード3つ程度）
- カラーパレット（メイン・サブ・アクセント・背景・テキスト色とカラーコード）
- タイポグラフィ（フォント名・サイズ・行間・用途別一覧）
- ボタン・フォーム・カードなどのUIコンポーネントスタイル
- アイコン・画像の方針
- 余白・角丸などの数値ルール

**活用するチュートリアル:** デザイン基礎4原則、タイポグラフィ、色の基礎、配色設計、Web色設計

---

#### 07. 仕様書（機能仕様書）

Webアプリケーションの機能仕様。

**記載内容:**
- 機能一覧表（機能名・概要・優先度）
- 画面一覧（画面名・URL・概要）
- 画面遷移図
- 各機能の詳細仕様
  - 画面レイアウト
  - 入力項目と制約（バリデーションルール）
  - 処理フロー
  - 出力内容
- 非機能要件（セキュリティ・パフォーマンス）

**活用するチュートリアル:** 要件定義、Webアプリ・バックエンド、開発ライフサイクル

---

#### 08. データベース設計書

データベースの構造設計。

**記載内容:**
- ER図（テーブル間のリレーション図）
- テーブル定義書（テーブルごとにカラム名・型・制約・説明）
- インデックス設計
- 初期データ・テストデータ

**活用するチュートリアル:** MyBatis、データベース操作、アプリケーション準備

---

#### 09. テスト報告書

テスト実施結果の報告。

**記載内容:**
- テスト計画（テスト対象・範囲・方法）
- テストケース一覧（正常系・異常系・境界値）
- テスト実施結果（OK / NG / 件数集計）
- 発見されたバグと対応状況
- ブラウザ・デバイス対応状況

**活用するチュートリアル:** 開発ライフサイクル、セキュリティ対策

---

#### 10. 振り返り・技術記事

制作全体の振り返りと技術的な学びの記録。

**記載内容:**
- プロジェクト全体の振り返り
  - うまくいったこと
  - 苦労したこと・解決方法
  - 次回への改善点
- 技術的なトピック記事（1つ以上）
  - 例：「Spring SecurityでログインEを実装した手順」
  - 例：「レスポンシブデザインで学んだこと」
  - 例：「プロンプトプログラミングの活用で開発効率がどう変わったか」
- 使用技術の棚卸しと自己評価

**活用するチュートリアル:** ChatGPTまとめ・ベストプラクティス

---

### 制作物（Webサイト＋Webアプリケーション）

ドキュメントで設計した内容を実際に制作します。

**Webサイト（フロントエンド）:**
- レスポンシブ対応のWebサイト（5ページ以上）
- HTML / CSS / JavaScript による実装
- インタラクティブなUI（アニメーション、動的機能）

**Webアプリケーション（サーバーサイド）:**
- Spring Boot によるWebアプリケーション
- CRUD機能 + 認証 + 追加機能
- プロンプトプログラミングを活用した開発

## 制作テーマの選び方

テーマは自由ですが、以下の条件を満たすものを選んでください。

**必須条件:**
- Webサイト（フロントエンド）とWebアプリ（サーバーサイド）の両方が必要なテーマ
- CRUD操作が含まれるテーマ
- ユーザー認証が必要なテーマ
- 自分が興味を持てるテーマ

**テーマ例:**

| テーマ | Webサイト部分 | Webアプリ部分 |
|--------|-------------|-------------|
| カフェ/レストラン | 店舗紹介サイト | 予約管理システム |
| フリーランス向けサービス | サービス紹介LP | 案件管理・タスク管理システム |
| 学習塾 | 塾の紹介サイト | 生徒管理・成績管理システム |
| イベント運営 | イベント告知サイト | 参加者管理・チケット管理システム |
| 社内ツール | サービス紹介ページ | 社内掲示板・ナレッジ共有システム |
| ECサイト | 商品紹介・ブランドサイト | 在庫管理・注文管理システム |

## 制作ドキュメントの要件

各ドキュメントHTMLページに含めるべき要素の詳細です。

---

### 全ドキュメント共通

---

### 01. 企画提案書 の詳細要件

```
■ プロジェクト概要
  - サービス名
  - コンセプト（一言で表現）
  - サービスの説明（3〜5文）
  - ターゲットユーザー（概要レベル）

■ 背景と課題
  - 市場の現状と課題
  - 既存サービスの問題点
  - 解決すべき課題の明確化

■ 提案するソリューション
  - サービスの特長（3つ程度）
  - 競合との差別化ポイント
  - 想定する効果

■ 制作範囲
  - Webサイト（フロントエンド）の範囲
  - Webアプリ（サーバーサイド）の範囲
  - 使用技術一覧

■ スケジュール
  - マイルストーンベースの概要スケジュール
```

---

### 02. マーケットリサーチ の詳細要件

```
■ 市場分析
  - 対象市場の規模・トレンド
  - ユーザーのニーズ・行動傾向

■ 3C分析
  - Customer（顧客）：誰が使うか、何を求めているか
  - Competitor（競合）：既存サービスの強み・弱み
  - Company（自社）：自分のサービスの強み

■ SWOT分析
  - Strengths（強み）
  - Weaknesses（弱み）
  - Opportunities（機会）
  - Threats（脅威）

■ STP分析
  - Segmentation（市場細分化）
  - Targeting（ターゲット選定）
  - Positioning（ポジショニング）

■ 競合サイト分析
  - 3サイト以上をスクリーンショット付きで分析
  - 各サイトの良い点・改善点
  - 自分のサービスへの示唆
```

---

### 03. ペルソナシート の詳細要件

```
■ 基本プロフィール
  - 名前（架空）、年齢、性別、職業、居住地、年収、家族構成

■ ライフスタイル
  - 平日のタイムスケジュール
  - 休日の過ごし方
  - 趣味・関心事

■ デジタル行動
  - よく使うデバイス
  - よく使うSNS・Webサービス
  - 情報収集の方法

■ 課題とニーズ
  - 現在抱えている課題・不満
  - 理想の状態
  - サービスに期待すること

■ カスタマージャーニーマップ
  - 認知 → 興味 → 比較検討 → 行動 → 共有・継続
  - 各段階でのタッチポイント
  - 各段階での感情・行動・課題
```

---

### 04. サイトマップ の詳細要件

```
■ サイト構造図
  - トップページからの階層構造（ツリー図）
  - Webサイト部分とWebアプリ部分の関係

■ ページ一覧
  - ページ名、URL、ページの目的、主要コンテンツ

■ ユーザーフロー
  - ペルソナがサイトに訪問してから目的を達成するまでの動線
  - 主要なユーザーシナリオ（2〜3パターン）
```

---

### 05. ワイヤーフレーム の詳細要件

```
■ 全ページのワイヤーフレーム
  - PC版（全ページ）
  - スマートフォン版（主要ページ）

■ 各ワイヤーフレームに付記する情報
  - 要素の配置意図（なぜその位置に配置したか）
  - インタラクション（クリック時・ホバー時の動作）
  - レスポンシブ時の変化

■ 表現方法
  - Figmaで作成し、スクリーンショットをHTMLに埋め込む
  - または HTMLでワイヤーフレームを直接表現する
```

---

### 06. デザインガイドライン の詳細要件

```
■ デザインコンセプト
  - キーワード3つ（例：「信頼感」「シンプル」「温かみ」）
  - コンセプトの説明

■ カラーパレット
  - メインカラー、サブカラー、アクセントカラー
  - 背景色、テキスト色、リンク色
  - 各色のカラーコード（HEX）と使用場面
  - 実際の色見本をCSSで表示

■ タイポグラフィ
  - 見出し用フォント、本文用フォント
  - フォントサイズ一覧（h1〜h6、本文、キャプション等）
  - 行間（line-height）、字間（letter-spacing）

■ UIコンポーネント
  - ボタン（プライマリ・セカンダリ・デンジャー）
  - フォーム要素（input・select・textarea）
  - カード、テーブル、リスト
  - 各コンポーネントの実物サンプルをHTMLで表示

■ レイアウトルール
  - グリッドシステム（カラム数・ガター幅）
  - 余白のルール（セクション間・要素間）
  - 角丸・シャドウの数値
```

```
  - エラー時の動作

  - 対応ブラウザ・デバイス
```
