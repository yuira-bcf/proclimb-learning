Section 0

プログラミング3大原則

ここまで何気なく進めてきましたが、プログラミングには絶対に覚えておかなければいけない3大原則があります。

プログラミングの3大原則

  1. 順次実行 ─ ソースコードは必ず上から順に読み込まれて実行される
  2. 条件分岐 ─ 条件が成立するかしないかで、処理Aか処理Bかを判断し、判定された処理のみを実行する
  3. 繰り返し ─ 条件が成立する間、同じ処理を繰り返し続ける

Webサイトのみならず、世の中のすべてのアプリやシステムは、必ずこの原則に基づいて実行されています。

ここからは、2つめの原則「条件分岐」について学んでいきます。

Section 1

条件分岐とは？

人間の判断を思い出してみましょう。

  • 「もし」、 雨が降っている「なら」 、傘を持っていく
  • 「もし」、 テストが80 点以上「なら」、 合格にする
  • 「もし」、 財布に500円以上ある「なら」、 コーヒーを買う

この「もし〜なら〜する」という考え方をプログラムの世界で実現するのが条件分岐。プログラミングではこれを実現するのに「if文」と言われる構文を使用します。

Section 2

if文の基本構文

JavaScript 
if (条件) {
  実行する処理;
}

読み方：「もし（条件）が成立すれば、{ }の中を実行する」

デベロッパーツールのコンソールに書いてみよう↓↓

例1：点数が80点以上なら「合格」と表示

JavaScript 
let score = 85;

if (score >= 80) {
  console.log("合格！");
}

実行結果：合格！

POINT

上記構文は、人が読みやすいようにあえて改行と字下げ（インデント）を加えています。プログラミングの性質上、if(score>=80){console.log("合格！");}でも正しく実行できます。しかし、処理は1つだけとは限りません。複雑なものになると数個、数十個。改行を入れていったら数十行、数百行になることもあります。それが1行に書かれていたら人間が読み取るのは大変なので、改行を入れて書くのがマナーです。

Section 3

条件の書き方（比較演算子）

if文の後の()の中には、条件式を書きます。条件式は、比較演算子を区切りに、左と右にあるものを比較させる式です。条件が成立すれば「true（正や真と呼ぶ）」、不成立なら「false（誤や偽と呼ぶ）」となり、trueのときのみ{}の中の処理を実行します。falseのときは何も起こりません。

記号 意味 例 説明
== 等しい a == b aとbが同じならtrue
!= 等しくない a != b aとbが違えばtrue
> より大きい a > b aがbより大きい
< より小さい a < b aがbより小さい
>= 以上 a >= b aがb以上
<= 以下 a <= b aがb以下

例2：お金が500円以上なら買える

JavaScript 
let money = 400;
if (money >= 500) {
  console.log("コーヒーが買えます");
}

実行結果：条件不成立（false）のため、何も起こらない

Section 4

if〜else（そうでなければ）

条件分岐にはいくつか種類があります。現実でも、こうだったらA、じゃなかったらB、という具合に、条件によって選ぶものを変えたりしますよね。

それと同じことが、if文で処理することができます。それが、if ~ else文です。

JavaScript 
if (条件) {
  真のときの処理;
} else {
  偽のときの処理;
}

例3：テストが80点以上なら合格、そうでなければ不合格

JavaScript 
let score = 65;

if (score >= 80) {
  console.log("合格です");
} else {
  console.log("不合格です");
}

実行結果：不合格です

Section 5

if〜else if〜else（条件が3つ以上のとき）

条件によって、さらに細かく処理を分けることもできます。条件の数だけ、間に else if文を入れます。

JavaScript 
if (score >= 80) {
  console.log("優秀です");　// scoreが80以上のときの処理
} else if (score >= 60) {
  console.log("合格です");　// scoreが80未満だけど、60以上のときの処理
} else {
  console.log("再テストです");　// 全部の条件がfalseのときの処理
}

例4：成績が90点以上ならAランク、90点未満～70点以上ならBランク、それ以外はCランク

JavaScript 
let score = 90;

if (score >= 90) {
  console.log("Aランク");
} else if (score >= 70) {
  console.log("Bランク");
} else {
  console.log("Cランク");
}

結果：Aランク

Section 6

条件の組み合わせ（論理演算子）

論理演算子というのを使うと、2つの条件を一度に判定することができます。
if(条件A 論理演算子 条件B) {……} という形で記述します。

記号 意味 例 結果
&& かつ（AND） (a >= 0 && a <= 100) 両方trueならtrue
|| または（OR） (a >= 0 || a <= 100) どちらかがtrueならtrue
! ではない（NOT） !(a > 0) 条件がfalseならtrue

例5：成績が80点以上、かつ欠席日数が0なら表彰対象

JavaScript 
let score = 85;
let absences = 0;

if (score >= 80 && absences == 0) {
  console.log("表彰対象です");
}

結果：表彰対象です

例6：成績が80点以上、または欠席日数が0なら表彰対象

JavaScript 
let score = 60;
let absences = 0;

if (score >= 80 || absences == 0) {
  console.log("表彰対象です");
}

実行結果：表彰対象です

Section 7

条件文を使ったミニアプリ

ファイル：Chapter_3-7 \ index.html

「年齢によって料金を変える」

HTML 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>条件分岐の練習</title>
</head>
<body>
  <script>
    let age = 15;
    let price;

    if (age < 13) {
      price = 500;
    } else if (age < 20) {
      price = 800;
    } else {
      price = 1000;
    }

    console.log("あなたの料金は " + price + " 円です。");
  </script>
</body>
</html>

結果：あなたの料金は 800 円です。

Section 8

比較でよくあるエラー

ミス 原因 修正
= と == を混同 = は代入（右の値を左に入れる） 比較は == を使う
=== を使わない 型が違うとfalseになることも 厳密に比較したいなら ===
文字比較で大文字小文字違い "A" ≠ "a" 同じ大文字・小文字にそろえる

== と === の違い

JavaScript 
let a = 5; // 数値型（number）
let b = "5"; // 文字列型（string）

console.log(a == b);  // true（5という値だけ比較）
console.log(a === b); // false（型も比較）

初心者のうちは「==」を使うのがおすすめです

Section 9

条件の中に変数を使う

条件内では演算子を使わなくても、変数のみでtrue/false を判定することもできます。

JavaScript 
let isMember = true;
if (isMember) {
  console.log("会員限定ページにアクセスできます");
} else {
  console.log("ログインが必要です");
}

結果：会員限定ページにアクセスできます

※ isMember に false が入っていると、「ログインが必要です」となります。

Section 10

じゃんけんアプリ（実践ミニゲーム）

ファイル：Chapter_3-10 \ index.html

HTML 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>じゃんけん</title>
</head>
<body>
  <script>
    let user = "グー";
    let computer = "チョキ";

    if (user === computer) {
      console.log("あいこです！");
    } else if (
      (user === "グー" && computer === "チョキ") ||
      (user === "チョキ" && computer === "パー") ||
      (user === "パー" && computer === "グー")
    ) {
      console.log("あなたの勝ちです");
    } else {
      console.log("あなたの負けです");
    }
  </script>
</body>
</html>

条件が3つ組み合わさると、こうして「論理演算子（&& / ||）」が活躍します。

まとめ

第3章のまとめ

  • プログラミング3大原則：順次実行・条件分岐・繰り返し
