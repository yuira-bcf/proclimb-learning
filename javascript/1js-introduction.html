Section 0

そもそも、なぜWebページは見れるのか

普段、私たちは何気なくインターネットを使ってさまざまなWebページを当たり前のように閲覧しています。どういう理屈で、何がどう関連して目に映る形になっているか、考えたことはありますか？

身近すぎてほとんど気にしたことがないと思いますが、Webページを基本的にHTMLと呼ばれる構造言語、CSSと呼ばれる装飾言語、JavaScriptと呼ばれるプログラミング言語。

大きくこれら3つの言語がテキスト（文章のこと）で書かれたファイルを読み込むことで表示されています。そしてそれらのファイルはどこに存在しているのかというと「サーバ」と呼ばれるところにあります。

そのサーバに対して「このページを表示して！」というリクエストを、私たちはスマホなどで検索結果などのリンクをクリックすることで送信しています。

ルータなどがそのリクエストのURLを見て、該当のサーバまでリクエストを届け、受け取ったサーバが「あーはい、じゃあこのページね」という形で、該当のHTMLファイルの情報を私たちのデバイスに返します（レスポンス）

そうして、初めて私たちは手元のデバイスでWebページを閲覧出来ている。そんな仕組になっています。

Section 1

JavaScriptの概要と基本構造

HTML・CSS・JavaScriptの関係をまず知ろう！

HTML/CSS/JavaScriptは、以下のような役割で分かれています。

分担 言語 たとえ 役割
① 骨組み HTML 建物の柱や屋根、壁など ページの構造・内容を決める
② デザイン CSS 塗装や壁紙、カーテンなど 色・文字の大きさ・レイアウト
③ 動き JavaScript エレベータや自動ドアなど クリック・変化・アニメーションなど

HTMLが構造を作り、CSSが装飾やレイアウトを決める。JavaScriptが、アクションによって一定の決まった処理を実行します。

Section 2

JavaScriptは何ができるの？

JavaScriptは「ページに反応をつける」言語。たとえば、次のようなことができます

種類 できること 具体的な例
表示 ページに文字や画像を追加・変更する 「こんにちは○○さん！」と表示
動き 画像をスライドさせる／ボタンで開閉 スライドショーやメニュー表示
入力チェック 入力内容を調べる フォームが空欄ならエラーを出す
自動更新 時間に合わせて変化 デジタル時計、リアルタイム表示
通信 サーバーとデータをやり取り 天気・株価・チャットなど

JavaScriptは"動く仕掛け人"

HTMLやCSSがどんなにキレイでも、JavaScriptがなければページはただの静止画です。

あなたが普段使っている：

  • YouTubeの再生ボタン
  • Amazonのカート
  • Twitterの「いいね」

これらも全部 JavaScriptの力 です！

Section 3

JavaScriptとJavaの違い（よくある勘違い）

「JavaScript」って名前に"Java"が入っているので、「似てるのかな？」と思う人が多いですが、実は……JavaとJavaScriptはまったくの別物！

比較 JavaScript Java
主な用途 Webブラウザ上で動く サーバー・スマホアプリなど
実行方法 ブラウザが直接実行 JVMという専用環境で動く
覚える量 少なめ（簡単） 多め（本格派）
歴史 1995年 Netscape社が開発 1991年 Sun社が開発
Section 4

JavaScriptを書くには何が必要？

答え：テキストエディタとブラウザだけ！

特別なアプリやツールは不要です。

  1. メモ帳（またはVSCodeなど）に決まった文章（コード）を書く
  2. ファイルを .html で保存
  3. ブラウザで開くだけ！
Section 5

はじめてのJavaScript（超入門コード）

次のフォルダと、ファイルをVScodeで開いてみよう

フォルダ：JavaScript\Chapter_1-5　　ファイル名：index.html

▶index.html

HTML 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>JavaScriptのはじめての例</title>
</head>
<body>
  <h1 id="title">こんにちは！</h1>
  <button onclick="changeText()">文字を変える</button>

  <script>
    // 「changeText」という関数（動きのかたまり）を作る
    function changeText() {
      // id="title" の要素を探して中身を変更
      document.getElementById("title").textContent = "JavaScriptで変えたよ！";
    }
  </script>
</body>
</html>

▶ 実行方法

  1. 開発基本演習_ソースフォルダの中の「Chapter_1-5」を開く
  2. その中の「index.html」というファイルを右クリック
  3. メニューの「プログラムから開く」から「Google chrome」をクリック
  4. ブラウザに表示された「文字を変える」ボタンをクリック
結果

画面に表示されていた「こんにちは！」が「JavaScriptで変えたよ！」に変わります！

このリアルタイムでWebページを変更するのが、JavaScriptならではの機能となります！

Section 6

どうして動くの？（しくみを図解）

処理の流れ

[HTML のボタン]
<button onclick="changeText()"> 文字を変える </button>

↓　ユーザーがクリック！

[ユーザーアクション、イベント（click）が発生]
onclick="changeText()"

↓

[JavaScript がイベントを感知！決められた処理を実行]
function changeText() { document.getElementById("title").textContent = "JavaScript で変えたよ！ "; }

↓

[変える文字を探し出して、タイトル文字を書き換える]
<h1 id="title"> こんにちは！</h1>
↓
<h1 id="title"> JavaScript で変えたよ！</h1>

つまり、JavaScriptは：
ユーザーのアクション、すなわち「イベント（出来事）」に反応して動くプログラムなんです。

Section 7

簡単にJavaScriptのコードを確認する方法、デベロッパーツール

実はVSCodeなどを開かなくても、JavaScriptの簡単なコードを実行して確認する方法があります。それが「デベロッパーツール」というブラウザツールと「コンソール」機能です。

デベロッパーツールとは

各ブラウザに標準搭載されている「開発者ツール」と呼ばれるツール。ここでWebページが開かれたときのサーバ状態やデータ、コードの確認が行えたりします。

7-1. デベロッパーツールの起動の仕方とコンソールの表示方法

  1. Googlechromeを起動
  2. 右上の縦に並んだ3点マークをクリック
  3. その他のツール ⇒ デベロッパーツールをクリック
    ※ Ctrl + Shift + i でも開きます
  4. コンソールと書かれたタブをクリック

これだけでOK、真っ白な画面にカーソルが当たっている状態になります。あとはここにJavaScriptのコードを書いて、Enterで実行するだけ。

※ 最初は英語になっています。language for Japaneseというボタンをクリックすると日本語になります。

7-2. コンソールとは？

作成したプログラミングコードが正しく動くかを、テスト（デバックと呼ぶ）したりするための、簡易的なテキストエディタです。コードの実行も出来、かつそれに対して結果を返し、ダメなときはエラー文を表示したりしてくれます。

デベロッパーツールのコンソール機能は、JavaScriptに限定した形で「結果」や「エラー」を表示してくれます。

7-3. コードを実行してみる

↓の文章をコピペしてEntarキーを押してみよう

JavaScript 
console.log("JavaScriptの練習中です！");

コンソール表示結果：JavaScriptの練習中です！

この console.log() というのは、()の中に書いたものを、コンソールに表示させる、という処理がプログラムされたメソッドと呼ばれる処理セットです。

Section 8

よくあるエラーとtips

エラー 原因 対策
Uncaught ReferenceError 変数や関数が未定義 スペルを確認
Unexpected token カッコやクォーテーションの抜け カッコなどの数をチェック
Cannot read properties of null 要素のidが間違い HTMLと一致しているか確認

tips

  • 改行できない ⇒ Shiftを押しながらEnter で改行される
  • 間違えて入力した ⇒ バックスペースで消すか、Ctrl + z でひとつ前の状態に戻る
  • 同じコードを書くのが面倒 ⇒ 一番最初に方向キーの↑を1回押すと、1つ前のコードを復元
Section 9

JavaScriptでできる面白いこと（実例）

次の①～③をコンソールにコピペして、実際に動きを確認してみましょう。

① ページの色を変える

JavaScript 
document.body.style.backgroundColor = "lightblue";

ページ全体が水色になります！

② 時間で文字を変える

JavaScript 
let hour = new Date().getHours();
if (hour < 12) {
  document.write("おはよう");
} else if (hour < 18) {
  document.write("こんにちは");
} else {
  document.write("こんばんは");
}

時間帯によって挨拶が変わります！

③ アラート（ポップアップ）を出す

JavaScript 
alert("こんにちは！JavaScriptの世界へようこそ");

実行するとメッセージボックスが出ます

こんなふうに、複雑じゃないプログラムならJavaScriptは簡単に、誰にでも実行できます。とりあえず学習を初めてみるプログラミング言語としても最適です。

Section 10

学習のすすめ方

JavaScriptに限らず、プログラミング言語ははっきり言って難しいです。初めて学習する人は「学習」そのもののスタンスを変えることが、継続的、かつ伸びていくポイントになります。

プログラミングを学ぶ際のおすすめスタンス

  1. すべてを覚えようとしない ─ 全部を覚えているエンジニアはいない
  2. 理解出来なくても次に進む ─ ある程度まとめたら次へ。理解は後から付いてくる
  3. 何も見ずに書けるようにならなくていい ─ プロも案外わからないもの。調べて書くのが当たり前
  4. 悩んでいる暇があるなら間違えて結果を得る ─ 実務でも何十回もエラーを出してようやく出来上がる
  5. 思い込みを辞める、素直な気持ちで ─ エラーは嘘を付かない。合っているはず、はやめよう
  6. 読んで覚えるより書いて覚える ─ とにかくいろんなパターンを書く、すると見えてくる

プログラミングは完璧を目指して学習するものではありません。それよりも、いかに試し、間違い、調べ、修正し、また試すか。このサイクルをどれだけ早く回すかが、実務でもカギになります。

まずは、わからなくても調べてそれとなく納得できればそれでOK。コードの単語ひとつひとつより、何が起きて、何をしているのかを説明できるかの方がプログラミングではとても大切です。
