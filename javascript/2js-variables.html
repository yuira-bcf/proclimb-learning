Section 1

「変数」ってなに？

みなさん、こんなときに"メモ"を取ることはありませんか？

  • 買物しなければいけないものを覚えておきたいとき
  • 今日の仕事のタスクに優先順位をつけたいとき
  • 金額を計算したいとき

プログラムの世界にも、それをするためのものがあります。それが――変数（へんすう）です！

イメージでいうと…変数 は、ラベル（名前）の付いた箱、と良く言われます。

Section 2

変数の作り方（宣言）

JavaScriptでは、次のように書きます

JavaScript 
let name = "たろう";
部分 意味
let 「これから箱を作るよ！」という合図（宣言）
name 箱の名前（基本は好きに決めて良い）
= 右側の文字や数字を、左の箱に入れる（代入）
"たろう" 入れる中身（値）

実際の例

JavaScript 
let age = 18;
let message = "こんにちは！";
let passed = true;

変数を使ってみよう！

ファイル：Chapter_2-3 \ index.html

▶ 確認方法

  1. ファイルをGooglechromeで開く
  2. デベロッパーツールを起動（Ctrl + Shift +i）
  3. コンソールタブをクリック
結果

こんにちは、たろうさん（18歳）！

変数は、letなどで新規作成をし、使いたいところでは変数名のみを書きます。変数などを使うことを「呼び出す」と、プログラミングでは頻繁に表現します。変数は呼び出されると、そのときに中に入っている値を展開します。

POINT

プログラミングでは「＝」と書くと、右側の値を左側の変数に「代入」する、という意味になります。学校で学んだ＝とは使い方が違いますが、これがプログラミングの大前提です。「同じ」を意味するイコールは、また別の書き方があります（後述します）

Section 4

変数を使って計算してみよう

変数には、計算結果を入れることもできます。

ファイル：Chapter_2-3 \ index.html の <script>タグ内に追記していこう↓↓

JavaScript 
let price = 120;
let count = 3;
let total = price * count;

console.log("合計は " + total + " 円です。");

出力結果：合計は 360 円です。

上記は計算結果を変数totalに入れていますが、詳細な処理の流れは以下のようになっています。

  1. 変数priceを作成し、「120」という数字の値を入れる
  2. 変数countを作成し、「3」という数字の値を入れる
  3. 変数totalを作成し、priceとcountの値は掛け算する
  4. 各変数の値が展開され、120 x 3 という計算が行われる
  5. 120 x 3 は「360」という数字の値になるため、let total = 360; という形になる
  6. 変数totalに「360」という数字の値が代入される
  7. console.logでtotalの部分が展開され、360が表示される

こうして並べるとたくさんあって難しそうですが、小さなひとつひとつの処理を、確実に、順番通りに行っているだけだったりします。

プログラミングで用いられる計算の記号：算術演算子

演算子 意味 使い方 結果
+ 足し算 1 + 1 2
- 引き算 2 - 1 1
* 掛け算 2 * 3 6
/ 割り算 10 / 2 5
% あまり 10 % 3 1
Section 5

文字列（string）を組み合わせる

ここまで既に何度か書いてますが……"〇〇"や'〇〇'で囲まれた値のことを「文字列(string)」と呼びます。「"」はダブルクオーテーション、「'」はシングルクオーテーションと言います。

変数と文字列をつなげるには「 +」 を使って繋げます。これを「文字列連結」と呼びます。

ファイル：Chapter_2-3 \ index.html の <script>タグ内に追記↓↓

JavaScript 
let firstName = "田中";
let lastName = "花子";
let fullName = firstName + "  " + lastName;

console.log(fullName); // 田中 花子

※ クオーテーションは必ず同じものをセットで使いましょう（開始と終わりで違うとエラーになります）

Section 6

データの種類（型）

JavaScriptの変数にはいろんなデータの型（タイプ）があります。

種類 例 説明
文字列（string） "こんにちは" 文字を表す。必ず「"」または「'」で囲む
数値（number） 10, 3.14 数を表す。""はいらない
真偽値（boolean） true, false 「はい or いいえ」「正 or 誤」「真 or 偽」
未定義（undefined） （値を入れてない） まだ中身がない状態
null null 「空っぽである」と定義したいとき

プログラミングの値は、それぞれデータの型があります。JavaScriptではあまり重要視されませんが、Javaや昨今のプログラミングの考え方としてデータ型は重要視されてきています。

※ 他のプログラミング言語ではデータ型の呼び方が違ったりします。

例：

JavaScript 
let name = "さくら"; // 文字（string）
let age = 20; // 数字（number）
let isPassed = true; // 真偽（boolean）
let data; // 未定義(undefined)
let empty = null; // 空っぽ（null）
Section 7

変数の中身を変える（再代入）

ファイル：Chapter_2-3 \ index.html の <script>タグ内に追記↓↓

JavaScript 
let score = 80;
console.log(score); // 80

score = 90; // 中身を変える
console.log(score); // 90

一度作った変数には、後から新たな値を入れて上書き更新することが出来ます。

Section 8

「変えたくない値」は const を使う

一度作った変数が上書きされてほしくないときは、const という宣言で変数を作ります。これで作られた変数は、後から値を代入できず、エラーになります。

JavaScript 
const TAX = 0.1;
TAX = 0.08; // エラーになる！
POINT

定数として作成した変数名は大文字にする、という慣習があります。小文字でもプログラム上は問題ありませんが、人がコードを見たときに区別が出来るような工夫です。そういった、対人間に対する配慮をしながら、プログラミングを組むことも良くあります。

Section 9

変数名のルール

OK例

例 説明
userName 単語をつなぐときは、次の単語の頭文字を大文字にする（キャメルケースと呼ぶ）
price1 数字はOK（ただし先頭はNG）
_total 記号は_ と $ はOK

NG例

例 理由
1user 先頭が数字
let JavaScriptの命令などに使われる単語と同じ（予約語と呼ぶ）
ユーザー 日本語でも動くが推奨されない
POINT

変数名は小文字と大文字を厳密に区別します。「username」と「userName」は、それぞれ別々の変数として認識されます。誤字・脱字には注意しましょう。

Section 10

演習問題（チャレンジ）

ファイル：Chapter_2-10 \ index.html の <script>タグ内に書いて、ブラウザで確認しよう

問1

自分の名前と、年齢を変数に入れて、コンソールに表示しよう。

例）清水 崇 / 40

問2

足し算、引き算、掛け算、割り算 のすべてを使った計算で、答えが100になる計算結果をひとつの変数に入れて、コンソールに100と表示しよう。

問3

年齢と名前の2つの変数を用意して、文字連結を使って簡単な自己紹介文をコンソールに表示してみよう。

例）[]が変数の部分：名前は[清水 崇]です。[40]歳です。

問4

変数の中身を2回、再代入して、1回ごとにコンソールに表示してみよう。

問5

const に値を入れて、中身を変更するとエラーが出るのを確認しよう。

解答例を見る

問1

let name = "あなたの名前";
let age = 20; // あなたの年齢に変えてね
console.log(name);
console.log(age);

問2

let math = (50 - 30) + (20 / 2) * 8;
console.log(math);

問3

let userName = "たろう";
let userAge = 18;
console.log("私の名前は" + userName + "です。" + userAge + " 歳です！");

問4

let score = 80;
console.log(score);
score = 90;
console.log(score);
score = 100;
console.log(score);

問5

const TAX = 0.1;
console.log(TAX);
TAX = 0.2; // エラーになるよ
まとめ

第2章のまとめ

  • 変数はラベル（名前）の付いた箱、値を保存して使い回せる
  • letで変数を宣言、=で値を代入する
  • 算術演算子（+, -, *, /, %）で計算ができる
  • 文字列連結は + で繋げる
  • データ型には string, number, boolean, undefined, null がある
  • 再代入で変数の中身を変えられる
  • constは再代入できない定数を作る
  • 変数名はキャメルケースで命名し、先頭に数字は使えない
