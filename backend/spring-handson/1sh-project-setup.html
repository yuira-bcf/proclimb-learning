Section 1-1

Spring Initializrでプロジェクト生成

Spring Bootプロジェクトは、Spring Initializrというウェブツールを使って簡単に生成できます。ブラウザで設定を選ぶだけで、必要なファイル一式がダウンロードされます。

1-1-1 Spring Initializrにアクセスする

ブラウザで以下のURLを開いてください。

open_in_new アクセス先

https://start.spring.io

Spring公式が提供するプロジェクト生成ツールです。

表示されたら、以下の設定を1つずつ確認しながら選択してください。

settings Initializrの設定値

項目 設定値 補足
ProjectMavenビルドツール
LanguageJavaデフォルト
Spring Boot3.4.x最新安定版を選択
Groupcom.exampleパッケージのグループ名
Artifacttodoプロジェクト名
PackagingJar組み込みTomcat付き
Java21LTS版

1-1-2 依存関係を追加する

画面右側の「ADD DEPENDENCIES」ボタンをクリックし、以下の4つの依存関係を検索して追加してください。

extension 追加する依存関係（4つ）

Spring Web
Webアプリを作るための基盤。HTTPリクエストを受け取るControllerを書ける。
Thymeleaf
HTMLテンプレートエンジン。Javaのデータを動的にHTMLに埋め込める。
Spring Data JPA
データベース操作をシンプルに書ける。SQLを直接書かずにデータを取得・保存。
H2 Database
軽量なインメモリDB。インストール不要で開発時にすぐ使える。

Validation は第7章で追加します。まずはこの4つだけで始めましょう。

すべて追加したら、画面下部の 「GENERATE」 ボタンをクリックしてください。todo.zip がダウンロードされます。

1-1-3 プロジェクトをIDEで開く

ダウンロードした todo.zip を展開し、VS Codeでフォルダを開いてください。

tips_and_updates VS Codeの準備

File > Open Folder から展開した todo フォルダを選択します。拡張機能「Extension Pack for Java」がインストールされていれば、Mavenの依存解決が自動で始まります。右下のプログレスバーが完了するまで待ちましょう（初回は2〜3分かかることがあります）。

1-2-3 TodoApplication.javaを確認する

src/main/java/com/example/todo/TodoApplication.java を開いてみましょう。これがアプリケーションのエントリポイント（起動クラス）です。

TodoApplication.java 
package com.example.todo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class TodoApplication {
    public static void main(String[] args) {
        SpringApplication.run(TodoApplication.class, args);
    }
}
info @SpringBootApplicationとは

このアノテーション1つで、コンポーネントスキャン・自動設定・設定クラスの3つの機能が有効になります。Spring Bootの「お約束」として覚えておきましょう。このファイルは編集する必要はありません。

Section 1-3

application.propertiesの初期設定

1-3-1 設定ファイルを編集する

src/main/resources/application.properties を開いてください。最初は空ですが、ここにアプリケーションの設定を書いていきます。

以下の内容を手で入力してください。

src/main/resources/application.properties 
# サーバー設定
server.port=8080

# H2データベース設定
spring.datasource.url=jdbc:h2:mem:tododb
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# H2 Console（ブラウザでDBを確認するツール）
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console

# JPA設定
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

tune 各設定の意味

spring.datasource.url=jdbc:h2:mem:tododb
メモリ上にtododbという名前のDBを作る。アプリ停止で消える。
spring.h2.console.enabled=true
ブラウザでDBの中身を確認できるH2 Consoleを有効化する。
spring.jpa.hibernate.ddl-auto=update
Entityクラスに合わせてテーブルを自動作成・更新する。
spring.jpa.show-sql=true
実行されるSQLをコンソールに表示する（デバッグに便利）。

設定が完了したら、アプリケーションを起動しましょう。

VS Codeの場合は、TodoApplication.java を開いて main メソッド上の Run リンクをクリックしてください。ターミナルの場合は次のコマンドを実行します。

ターミナル

Section 1-4

Hello Worldを表示する

1-4-1 Controllerクラスを作成する

いよいよ最初のコードを書きます。ブラウザからのリクエストを受け取る Controller クラスを作成しましょう。

src/main/java/com/example/todo/ の中に controller パッケージ（フォルダ）を新規作成し、その中に HelloController.java を作成してください。

以下のコードを手で入力してください。

src/main/java/com/example/todo/controller/HelloController.java 
package com.example.todo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class HelloController {

    @GetMapping("/")
    public String hello(Model model) {
        model.addAttribute("message", "Hello, Spring Boot!");
        return "hello";
    }
}

lightbulb コードの解説
  • @Controller ── このクラスがWebリクエストを処理するControllerであることをSpringに伝えます
  • @GetMapping("/") ── ブラウザで http://localhost:8080/ にアクセスしたときに、このメソッドが呼ばれます
  • model.addAttribute("message", ...) ── HTMLに渡すデータをセットします
  • return "hello" ── templates/hello.html を表示するという意味です

1-4-2 HTMLテンプレートを作成する

次に、Controllerが表示するHTMLファイルを作ります。src/main/resources/templates/ フォルダに hello.html を新規作成してください。

以下のHTMLを手で入力してください。

src/main/resources/templates/hello.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Hello Spring Boot</title>
</head>
<body>
    <h1 th:text="${message}">デフォルトメッセージ</h1>
    <p>Spring Bootアプリケーションが正常に動作しています！</p>
</body>
</html>
info th:text とは

th:text="${message}" は Thymeleaf の構文です。Controllerで model.addAttribute("message", "Hello, Spring Boot!") とセットした値が、この部分に表示されます。タグの中に書いた「デフォルトメッセージ」は、Thymeleafが無効な場合のフォールバックです。

  1. アプリが起動中なら一度停止して、再度起動してください
  2. ブラウザで http://localhost:8080/ にアクセスしてください

Exercise

1章 練習問題

Q1

Spring Initializrでプロジェクトを生成する際、Webアプリケーションを作るために最低限必要な依存関係はどれですか。

  1. Spring Data JPA
  2. Spring Web
  3. H2 Database
  4. Spring Security
Q2

以下のControllerコードで、ブラウザからどのURLにアクセスしたときに hello() メソッドが呼ばれますか。

Java
@Controller
public class HelloController {
    @GetMapping("/greeting")
    public String hello() { return "hello"; }
}
  1. http://localhost:8080/hello
  2. http://localhost:8080/greeting
  3. http://localhost:8080/HelloController
  4. http://localhost:8080/
Q3

Spring Bootプロジェクトで、Thymeleaf HTMLテンプレートファイルを配置するディレクトリはどこですか。

  1. src/main/java/templates/
  2. src/main/resources/static/
  3. src/main/resources/templates/
  4. src/main/webapp/
Answer

1章 解説・解答

Q1の解答: B

Spring Web には組み込みTomcatとSpring MVCが含まれており、これだけでHTTPリクエストを受け取るWebアプリケーションを作れます。JPA・H2はデータベースを使う場合に必要で、SecurityはWebアプリの基本には含まれません。

Q2の解答: B

@GetMapping("/greeting") は、GETリクエストで /greeting というパスにアクセスしたときにメソッドを実行する指定です。URLは http://localhost:8080/greeting になります。メソッド名やクラス名はURLに関係ありません。

Q3の解答: C

Thymeleafのテンプレートは src/main/resources/templates/ に配置します。Spring Bootが自動的にこのディレクトリを参照します。static/ はCSS・JS・画像などの静的ファイル用です。Controllerの return "hello" は templates/hello.html を意味します。
