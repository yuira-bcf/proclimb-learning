Section 3-1

データベースとEntityの関係

前章までのTodoデータはControllerにハードコードされていました。アプリを再起動するとデータは消えません（最初から同じデータが表示されるだけ）が、追加・変更はできません。データを永続化するために、データベースを使いましょう。

3-1-1 テーブルとEntityの対応

JPAでは、データベースのテーブルとJavaのクラスを1対1で対応させます。このJavaクラスを Entity（エンティティ） と呼びます。

ポイント: テーブルのカラム名は due_date（スネークケース）、Javaのフィールド名は dueDate（キャメルケース）。JPAが自動で変換してくれます。

3-1-2 H2データベースとは

H2は インメモリデータベース です。通常のデータベース（MySQL、PostgreSQL）のように別途インストールする必要がなく、アプリケーションに組み込まれて動作します。

info 本番環境では

H2は開発・学習用です。本番環境ではMySQL、PostgreSQL等の永続データベースを使います。Spring Bootは設定ファイルを変えるだけでDBを切り替えられるため、学習中はH2を使い、後から本番用DBに移行できます。

Section 3-2

Todo Entityクラスを作成する

3-2-1 entityパッケージを作成する

src/main/java/com/example/todo/ の中に entity パッケージ（フォルダ）を新規作成してください。この中にEntityクラスを配置します。

3-2-2 Todo.javaを作成する

entity パッケージに Todo.java を新規作成し、以下のコードを手で入力してください。1つずつアノテーションの意味を解説するので、打ちながら理解しましょう。

src/main/java/com/example/todo/entity/Todo.java 
package com.example.todo.entity;

import jakarta.persistence.Column;
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import jakarta.persistence.Table;
import java.time.LocalDate;
import java.time.LocalDateTime;

@Entity
@Table(name = "todos")
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 255)
    private String title;

    @Column(length = 500)
    private String description;

    @Column(name = "due_date")
    private LocalDate dueDate;

    @Column(name = "created_at")
    private LocalDateTime createdAt;

    // デフォルトコンストラクタ（JPAに必須）
    public Todo() {
    }

    // Getter / Setter
    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public String getDescription() {
        return description;
    }

    public void setDescription(String description) {
        this.description = description;
    }

    public LocalDate getDueDate() {
        return dueDate;
    }

    public void setDueDate(LocalDate dueDate) {
        this.dueDate = dueDate;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }

    public void setCreatedAt(LocalDateTime createdAt) {
        this.createdAt = createdAt;
    }
}

label JPAアノテーションの意味

@Entity
「このクラスはDBテーブルに対応するEntityだよ」とJPAに伝える。これがないとテーブルが作られない。
@Table(name = "todos")
テーブル名を指定。省略するとクラス名がそのままテーブル名になる（Todo → todo）。
@Id
主キー（Primary Key）であることを示す。テーブルの各行を一意に識別するカラム。
@GeneratedValue(strategy = GenerationType.IDENTITY)
IDを自動採番する。INSERTするたびにDBが自動で1, 2, 3...と番号を振る。
@Column(nullable = false, length = 255)
カラムの制約を指定。nullable=false でNOT NULL制約、length で最大文字数を設定。
tips_and_updates デフォルトコンストラクタが必須な理由

JPAはDBからデータを取り出すとき、まず引数なしのコンストラクタでインスタンスを作り、その後setterでフィールドに値を設定します。そのため、引数なしのコンストラクタ（デフォルトコンストラクタ）が必須です。

Section 3-3

H2 Consoleでテーブルを確認する

3-3-1 アプリケーションを起動する

Entityクラスを作成したら、アプリケーションを起動してください。起動すると、JPAが自動的にEntityに対応するテーブルをH2データベースに作成します。

コンソールに以下のようなSQL文が表示されるはずです（spring.jpa.show-sql=true を設定したため）。

コンソール出力 
Hibernate: create table todos (
    id bigint generated by default as identity,
    title varchar(255) not null,
    description varchar(500),
    due_date date,
    created_at timestamp(6),
    primary key (id)
)
lightbulb 自動テーブル作成の仕組み

spring.jpa.hibernate.ddl-auto=update の設定により、Entityクラスの構造に合わせてテーブルが自動生成されます。フィールドを追加すれば、カラムも自動で追加されます。

3-3-2 H2 Consoleにアクセスする

ブラウザで http://localhost:8080/h2-console にアクセスしてください。ログイン画面が表示されます。

以下の値を入力してください。

「Connect」をクリックすると、左側に TODOS テーブルが表示されます。テーブル名をクリックして「Run」を押すと、テーブル構造を確認できます。

warning JDBC URLに注意

H2 ConsoleのデフォルトJDBC URLは jdbc:h2:~/test です。これを jdbc:h2:mem:tododb に変更し忘れると、アプリとは別のDBに接続してしまい、TODOSテーブルが見つかりません。

Section 3-4

初期データを投入する

3-4-1 data.sqlを作成する

Spring Bootは起動時に src/main/resources/data.sql というファイルがあれば、自動でSQLを実行してくれます。ここに初期データのINSERT文を書きましょう。

src/main/resources/ に data.sql を新規作成し、以下のSQLを手で入力してください。

src/main/resources/data.sql 
INSERT INTO todos (title, description, due_date, created_at)
VALUES ('Spring Bootの環境構築', 'Initializrでプロジェクトを作成する', '2026-03-10', '2026-03-01 09:00:00');

INSERT INTO todos (title, description, due_date, created_at)
VALUES ('Thymeleafの基本を学ぶ', 'テンプレートエンジンの使い方を習得', '2026-03-15', '2026-03-01 10:00:00');

INSERT INTO todos (title, description, due_date, created_at)
VALUES ('Entityクラスを作成する', 'JPA EntityでTodoテーブルを定義', '2026-03-20', '2026-03-01 11:00:00');

3-4-2 application.propertiesに設定を追加する

data.sql を確実に実行するために、application.properties に以下の1行を追加してください。

application.properties に追加 
# data.sql を Hibernate のテーブル作成後に実行する
spring.jpa.defer-datasource-initialization=true
info なぜこの設定が必要か

Spring Boot 2.5以降、data.sqlはHibernateのテーブル作成前に実行されるようになりました。テーブルがまだ存在しない状態でINSERTを実行するとエラーになるため、defer-datasource-initialization=true でテーブル作成後に実行するよう指定します。

3-4-3 実行して確認しましょう

アプリケーションを再起動して、H2 Consoleで SELECT * FROM TODOS; を実行してください。

Exercise

3章 練習問題

Q1

JPAのEntityクラスで、主キーフィールドに付与するアノテーションの正しい組み合わせはどれですか。

  1. @Column と @GeneratedValue
  2. @Id と @GeneratedValue
  3. @Entity と @Id
  4. @Table と @Column
Q2

以下のEntityクラスで、データベースに作成されるカラム名はどれですか（Spring Bootのデフォルト設定の場合）。

Java
@Column
private LocalDate dueDate;
  1. dueDate
  2. due_date
  3. DUEDATE
  4. DUE_DATE
Q3

H2 Consoleにブラウザからアクセスするために、application.propertiesに必要な設定はどれですか。

  1. spring.h2.console.enabled=true
  2. spring.h2.browser.enabled=true
  3. spring.datasource.console=true
  4. h2.console.auto-start=true
Answer

3章 解説・解答

Q1の解答: B

主キーには @Id（主キーであることを示す）と @GeneratedValue（自動採番を指定する）の2つを組み合わせて使います。@Entity はクラスレベル、@Table や @Column はテーブル・カラムの詳細設定で、主キーの定義とは直接関係ありません。

Q2の解答: B

Spring Boot（Hibernate）のデフォルトの命名戦略では、Javaのキャメルケース（dueDate）はスネークケース（due_date）に自動変換されます。大文字の位置にアンダースコアが挿入され、すべて小文字に変換されます。

Q3の解答: A

H2 Consoleはデフォルトで無効です。spring.h2.console.enabled=true を設定することで、ブラウザから /h2-console にアクセスできるようになります。パスは spring.h2.console.path で変更可能です。
