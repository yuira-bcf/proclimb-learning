Section 3-1

データベースとJavaクラスの関係

前章までのTodoデータはControllerにハードコードされていました。アプリを再起動するとデータは消えません（最初から同じデータが表示されるだけ）が、追加・変更はできません。データを永続化するために、データベースを使いましょう。

3-1-1 テーブルとJavaクラスの対応

MyBatisでは、データベースのテーブルとJavaのクラスを対応させてデータをやり取りします。テーブルの各行がJavaオブジェクト1つに対応します。

ポイント: テーブルのカラム名は due_date（スネークケース）、Javaのフィールド名は dueDate（キャメルケース）。MyBatisの設定 map-underscore-to-camel-case=true で自動変換されます。

3-1-2 H2データベースとは

H2は インメモリデータベース です。通常のデータベース（MySQL、PostgreSQL）のように別途インストールする必要がなく、アプリケーションに組み込まれて動作します。

info 本番環境では

H2は開発・学習用です。本番環境ではMySQL、PostgreSQL等の永続データベースを使います。Spring Bootは設定ファイルを変えるだけでDBを切り替えられるため、学習中はH2を使い、後から本番用DBに移行できます。

Section 3-2

Todo クラスとschema.sqlを作成する

3-2-1 entityパッケージを作成する

src/main/java/com/example/todo/ の中に entity パッケージ（フォルダ）を新規作成してください。この中にデータを表すJavaクラスを配置します。

3-2-2 Todo.javaを作成する

entity パッケージに Todo.java を新規作成し、以下のコードを手で入力してください。MyBatisではアノテーション不要の普通のJavaクラス（POJO）を使います。

src/main/java/com/example/todo/entity/Todo.java 
package com.example.todo.entity;

import java.time.LocalDate;
import java.time.LocalDateTime;

public class Todo {

    private Long id;
    private String title;
    private String description;
    private LocalDate dueDate;
    private boolean completed;
    private LocalDateTime createdAt;

    // getter & setter
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }

    public String getDescription() { return description; }
    public void setDescription(String description) { this.description = description; }

    public LocalDate getDueDate() { return dueDate; }
    public void setDueDate(LocalDate dueDate) { this.dueDate = dueDate; }

    public boolean isCompleted() { return completed; }
    public void setCompleted(boolean completed) { this.completed = completed; }

    public LocalDateTime getCreatedAt() { return createdAt; }
    public void setCreatedAt(LocalDateTime createdAt) { this.createdAt = createdAt; }
}

swap_horiz MyBatisのカラム名マッピング（camelCase ↔ snake_case）

設定: mybatis.configuration.map-underscore-to-camel-case=true
この設定1つで、SQLの結果カラム名（snake_case）をJavaフィールド名（camelCase）に自動変換します。
SQLカラム名
due_date
created_at
→
Javaフィールド名

createdAt
tips_and_updates MyBatisではアノテーションが不要

MyBatisではJavaクラスに特別なアノテーション（@Entityや@Column等）は不要です。普通のJavaクラスにgetter/setterを用意するだけでOKです。MyBatisがSQLの結果をフィールドに自動マッピングしてくれます。

3-2-3 schema.sqlでテーブルを作成する

MyBatisではテーブルの自動生成機能がないため、自分でCREATE TABLE文を書く必要があります。src/main/resources/ に schema.sql を新規作成し、以下のSQLを手で入力してください。

src/main/resources/schema.sql 
CREATE TABLE todo (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    description VARCHAR(500),
    due_date DATE,
    completed BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

compare MyBatisのテーブル管理方式

schema.sql
テーブル定義をSQLで自分で書く。カラム名・型・制約を明示的に指定するため、テーブル構造が明確。
data.sql
初期データをINSERT文で投入する。schema.sqlでテーブルが作られた後に実行される。
spring.sql.init.mode=always
application.propertiesに設定済み。起動時にschema.sql → data.sqlの順で実行される。
lightbulb SQLを自分で書くメリット

テーブル構造をSQLで明示的に定義するため、型やデフォルト値、制約が一目でわかります。また、本番環境でもそのままSQL文を使ってテーブルを作れるため、開発と本番の差が小さくなります。

Section 3-3

H2 Consoleでテーブルを確認する

3-3-1 アプリケーションを起動する

schema.sqlとTodoクラスを作成したら、アプリケーションを起動してください。起動すると、Spring Bootが schema.sql を読み取り、H2データベースにテーブルを作成します。

コンソールに正常に起動したログが表示されるはずです。

コンソール出力（抜粋） 
Started TodoApplication in 2.3 seconds (process running for 2.7)
lightbulb テーブル作成の仕組み

spring.sql.init.mode=always の設定により、起動時に schema.sql が実行されテーブルが作成されます。schema.sqlを編集すればテーブル構造を変更できます。

3-3-2 H2 Consoleにアクセスする

ブラウザで http://localhost:8080/h2-console にアクセスしてください。ログイン画面が表示されます。

以下の値を入力してください。

「Connect」をクリックすると、左側に TODO テーブルが表示されます。テーブル名をクリックして「Run」を押すと、テーブル構造を確認できます。

warning JDBC URLに注意

H2 ConsoleのデフォルトJDBC URLは jdbc:h2:~/test です。これを jdbc:h2:mem:tododb に変更し忘れると、アプリとは別のDBに接続してしまい、TODOテーブルが見つかりません。

Section 3-4

初期データを投入する

3-4-1 data.sqlを作成する

Spring Bootは起動時に src/main/resources/data.sql というファイルがあれば、schema.sqlの後に自動でSQLを実行してくれます。ここに初期データのINSERT文を書きましょう。

src/main/resources/ に data.sql を新規作成し、以下のSQLを手で入力してください。

src/main/resources/data.sql 
INSERT INTO todo (title, description, due_date, completed, created_at)
VALUES ('Spring Bootの環境構築', 'Initializrでプロジェクトを作成する', '2026-03-10', false, '2026-03-01 09:00:00');

INSERT INTO todo (title, description, due_date, completed, created_at)
VALUES ('Thymeleafの基本を学ぶ', 'テンプレートエンジンの使い方を習得', '2026-03-15', false, '2026-03-01 10:00:00');

INSERT INTO todo (title, description, due_date, completed, created_at)
VALUES ('Todoクラスを作成する', 'MyBatisで使うJavaクラスとテーブルを定義', '2026-03-20', false, '2026-03-01 11:00:00');

第1章の application.properties で spring.sql.init.mode=always を設定済みです。これにより起動時に以下の順序で実行されます。

info schema.sql が先に実行される理由

Spring Bootはファイル名の規約で実行順序を決めています。schema.sql（テーブル定義）が先に実行され、その後 data.sql（データ投入）が実行されます。テーブルが存在しない状態でINSERTするとエラーになるため、この順序が重要です。

Exercise

3章 練習問題

Q1

MyBatisでデータベースのテーブルを作成するには、どのファイルにCREATE TABLE文を書きますか。

  1. Todo.java にアノテーションを付ける
  2. src/main/resources/schema.sql にSQL文を書く
  3. application.properties にテーブル定義を書く
  4. MyBatisが自動でテーブルを作成する
Q2

MyBatisで、SQLのカラム名 due_date をJavaフィールド dueDate に自動マッピングするために必要な設定はどれですか。

  1. mybatis.configuration.map-underscore-to-camel-case=true
  2. spring.jpa.hibernate.ddl-auto=update
  3. mybatis.auto-mapping=enabled
  4. 設定不要。MyBatisがデフォルトで変換する
Q3

H2 Consoleにブラウザからアクセスするために、application.propertiesに必要な設定はどれですか。

  1. spring.h2.console.enabled=true
  2. spring.h2.browser.enabled=true
  3. spring.datasource.console=true
  4. h2.console.auto-start=true
Answer

3章 解説・解答

Q1の解答: B

MyBatisにはテーブル自動生成機能がないため、src/main/resources/schema.sql にCREATE TABLE文を自分で書きます。Spring Bootが起動時にこのファイルを読み込んで実行します。Javaクラスにはアノテーションは不要で、普通のPOJO（getter/setterを持つクラス）として書きます。

Q2の解答: A

mybatis.configuration.map-underscore-to-camel-case=true を設定すると、SQLのスネークケース（due_date）がJavaのキャメルケース（dueDate）に自動変換されます。この設定がないと、カラム名とフィールド名が一致しないため、値が null になってしまいます。

Q3の解答: A

H2 Consoleはデフォルトで無効です。spring.h2.console.enabled=true を設定することで、ブラウザから /h2-console にアクセスできるようになります。パスは spring.h2.console.path で変更可能です。
