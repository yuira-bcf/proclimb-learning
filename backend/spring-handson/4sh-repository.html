Section 4-1

4-1-2 JpaRepositoryの魔法

Spring Data JPAの JpaRepository インターフェースを継承するだけで、よく使うDB操作メソッドが 自動で使える ようになります。SQLを1行も書く必要がありません。

auto_awesome JpaRepositoryが提供するメソッド（抜粋）

メソッド 実行されるSQL 用途
findAll() SELECT * FROM todos 全件取得
findById(id) SELECT * FROM todos WHERE id = ? 1件取得
save(entity) INSERT / UPDATE 保存・更新
deleteById(id) DELETE FROM todos WHERE id = ? 削除
count() SELECT COUNT(*) FROM todos 件数取得
lightbulb なぜSQLを書かなくていいのか

Spring Data JPAが、インターフェースの型情報（Todo と Long）からEntity構造を読み取り、実行時にSQL文を自動生成して実行してくれるからです。この仕組みによって開発者はビジネスロジックに集中できます。

Section 4-2

TodoRepositoryインターフェースを作成する

4-2-1 repositoryパッケージを作成する

src/main/java/com/example/todo/ の中に repository パッケージ（フォルダ）を新規作成してください。

4-2-2 TodoRepository.javaを作成する

repository パッケージに TodoRepository.java を新規作成し、以下のコードを手で入力してください。

src/main/java/com/example/todo/repository/TodoRepository.java 
package com.example.todo.repository;

import com.example.todo.entity.Todo;
import org.springframework.data.jpa.repository.JpaRepository;

public interface TodoRepository extends JpaRepository<Todo, Long> {

}

これだけです。中身が空のインターフェースですが、JpaRepository<Todo, Long> を継承しているだけで、前述の findAll(), findById(), save(), deleteById() 等のメソッドがすべて使えます。

description 型パラメータの意味

JpaRepository<Todo, Long>
Todo
操作対象のEntityクラス
Long
主キー（id）の型
tips_and_updates classではなくinterfaceで書く理由

Repositoryは interface（インターフェース）として書きます。Spring Data JPAが実行時に自動でこのインターフェースの実装クラスを生成してくれるため、開発者はメソッドの「仕様」だけ定義すればOKです。

Section 4-3

Controllerからリポジトリを使う

4-3-1 DI（依存性注入）を理解する

ControllerでRepositoryを使うには、DI（Dependency Injection: 依存性注入） という仕組みを使います。難しく聞こえますが、やることはシンプルです。

settings_input_component DI（依存性注入）のイメージ

close DIなし（自分で作る）
// 自分でnewする
TodoRepository repo
= new TodoRepository();
Springでは動かない
check DI（Springが渡す）
// コンストラクタで受け取る
public TodoController(
TodoRepository repo) {
this.repo = repo;
}
Springが自動でインスタンスを渡す

DI = 「使いたい部品をコンストラクタの引数に書くだけ」。Springが起動時に自動でインスタンスを作って渡してくれます。

4-3-2 TodoControllerを書き換える

TodoController.java をDBからデータを取得するように書き換えましょう。ファイル全体を以下の内容に置き換えてください。

src/main/java/com/example/todo/controller/TodoController.java 
package com.example.todo.controller;

import com.example.todo.entity.Todo;
import com.example.todo.repository.TodoRepository;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import java.util.List;

@Controller
public class TodoController {

    private final TodoRepository todoRepository;

    // コンストラクタインジェクション
    public TodoController(TodoRepository todoRepository) {
        this.todoRepository = todoRepository;
    }

    @GetMapping("/todos")
    public String list(Model model) {
        // DBから全件取得
        List<Todo> todos = todoRepository.findAll();
        model.addAttribute("todos", todos);
        return "todo/list";
    }

    @GetMapping("/todos/{id}")
    public String detail(@PathVariable("id") Long id, Model model) {
        // DBから1件取得
        Todo todo = todoRepository.findById(id).orElse(null);
        model.addAttribute("todo", todo);
        return "todo/detail";
    }
}

compare 前章との変更点

Before: ハードコードされた仮データ
List<String> todos = List.of("...", "...", "...");
↓
After: データベースから取得
List<Todo> todos = todoRepository.findAll();
info findById().orElse(null) とは

findById() は Optional<Todo> を返します。Optionalは「データがあるかもしれないし、ないかもしれない」ことを表す型です。.orElse(null) で「データがなければnullを返す」という意味になります。Optionalの詳細はJava基礎で学習しています。

Exercise

4章 練習問題

Q1

JpaRepository<Todo, Long> の2つの型パラメータが表すものの正しい組み合わせはどれですか。

  1. Todo = テーブル名, Long = カラム数
  2. Todo = Entityクラス, Long = 主キーの型
  3. Todo = Repository名, Long = レコード数
  4. Todo = パッケージ名, Long = バージョン番号
Q2

Spring BootでControllerにRepositoryを渡すDI（依存性注入）の正しい方法はどれですか。

  1. TodoRepository repo = new TodoRepository();
  2. コンストラクタの引数として TodoRepository を受け取る
  3. TodoRepository.getInstance() で取得する
  4. ApplicationContext.getBean(TodoRepository.class) で取得する
Q3

JpaRepositoryのメソッドで、findById(1L) の戻り値の型はどれですか。

  1. Todo
  2. List<Todo>
  3. Optional<Todo>
  4. null
Answer

4章 解説・解答

Q1の解答: B

JpaRepository<Todo, Long> の第1型パラメータは操作対象のEntityクラス（Todo）、第2型パラメータは主キーフィールド（id）のJava型（Long）です。これによりSpring Data JPAは、どのテーブルに対してどのような型のIDでクエリを実行すべきかを判断します。

Q2の解答: B

Spring BootではDI（依存性注入）として、コンストラクタインジェクションが推奨されています。コンストラクタの引数に TodoRepository を宣言するだけで、SpringがTodoRepositoryのインスタンスを自動生成して渡してくれます。new で自分で作ってはいけません（Spring管理外になり、DB接続等が機能しません）。選択肢Dも技術的には可能ですが、推奨されません。

Q3の解答: C

findById() は Optional<Todo> を返します。Optionalは「値が存在するかもしれないし、存在しないかもしれない」ことを明示的に表す型です。指定したIDのデータが存在しない場合、Optionalは空（empty）になります。.orElse() や .orElseThrow() で値を取り出すか、例外をスローできます。
