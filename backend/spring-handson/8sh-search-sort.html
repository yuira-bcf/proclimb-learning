Section 8-2

@Selectで検索メソッドを作る

MyBatisでは、@Select アノテーションにネイティブSQL（実際のデータベースSQL）を直接記述して検索メソッドを定義します。SQLを自分で書くため、動作が明確で理解しやすいのが特徴です。

8-2-1 LIKE検索のSQLを理解する

SQLの LIKE は文字列の部分一致検索に使います。% はワイルドカードで、任意の文字列にマッチします。

LIKE パターン 意味 マッチ例
'%Spring%' Springを含む（部分一致） Spring Boot入門、学ぶSpring
'Spring%' Springで始まる（前方一致） Spring Boot入門
'%Spring' Springで終わる（後方一致） 学ぶSpring

auto_fix_high MyBatisでのLIKE検索

@Select(" SELECT * FROM todo WHERE title LIKE CONCAT('%', #{keyword}, '%') ")
アノテーション ネイティブSQL 条件 CONCAT で % とパラメータを結合

8-2-2 TodoMapperに検索メソッドを追加する

TodoMapper.java を開き、以下の検索メソッドを追加してください。

TodoMapper.java にメソッド追加 
// タイトルにキーワードを含むTodoを検索
@Select("SELECT * FROM todo WHERE title LIKE CONCAT('%', #{keyword}, '%')")
List<Todo> searchByTitle(String keyword);

MyBatisではSQLを自分で直接記述します。CONCAT('%', #{keyword}, '%') で、パラメータの前後に % を付けて部分一致検索を実現します。

lightbulb なぜ CONCAT を使うのか

#{keyword} はMyBatisの安全なパラメータバインドです。'%#{keyword}%' と書くと文字列リテラルとして扱われてしまうため、CONCAT 関数で文字列を結合します。これによりSQLインジェクションを防ぎつつ、部分一致検索が実現できます。

code MyBatis @Select の特徴

ネイティブSQL

データベースのSQL文をそのまま書くため、動作が明確で学習しやすい。SQLの知識がそのまま活かせます。

#{property}

MyBatisのパラメータバインド。PreparedStatement を使用するため安全にパラメータを埋め込めます。

Section 8-3

Controllerに検索ロジックを追加する

Mapperに検索メソッドを追加したので、次はControllerでURLパラメータを受け取り、検索結果を返すようにします。

8-3-1 @RequestParamでキーワードを受け取る

TodoController.java の list メソッドを以下のように変更してください。importに RequestParam と List を追加するのも忘れずに。

TodoController.java のimportに追加 
import org.springframework.web.bind.annotation.RequestParam;
import java.util.List;
TodoController.java の list メソッドを変更 
// 一覧表示（検索対応）
@GetMapping("/todos")
public String list(@RequestParam(name = "keyword", required = false) String keyword,
                   Model model) {
    List<Todo> todos;
    if (keyword != null && !keyword.isEmpty()) {
        todos = todoMapper.searchByTitle(keyword);
    } else {
        todos = todoMapper.findAll();
    }
    model.addAttribute("todos", todos);
    model.addAttribute("keyword", keyword);
    return "todos/list";
}

Section 8-4

ソート用のMapperメソッドを追加する

検索に加えて、期限日の昇順（近い順）・降順（遠い順）やタイトル順でソートできるようにしましょう。MyBatisではSQLの ORDER BY 句を使ってソートを実現します。

8-4-1 Mapperにソート対応メソッドを追加する

MyBatisではSQLに ORDER BY 句を直接書いてソートします。TodoMapper.java にソート対応のメソッドを追加しましょう。

TodoMapper.java にメソッド追加 
// 期限日の昇順で全件取得
@Select("SELECT * FROM todo ORDER BY due_date ASC")
List<Todo> findAllOrderByDueDate();

// 期限日の降順で全件取得
@Select("SELECT * FROM todo ORDER BY due_date DESC")
List<Todo> findAllOrderByDueDateDesc();

// タイトル順で全件取得
@Select("SELECT * FROM todo ORDER BY title ASC")
List<Todo> findAllOrderByTitle();

// キーワード検索 + 期限日の昇順
@Select("SELECT * FROM todo WHERE title LIKE CONCAT('%', #{keyword}, '%') ORDER BY due_date ASC")
List<Todo> searchByTitleOrderByDueDate(String keyword);

// キーワード検索 + 期限日の降順
@Select("SELECT * FROM todo WHERE title LIKE CONCAT('%', #{keyword}, '%') ORDER BY due_date DESC")
List<Todo> searchByTitleOrderByDueDateDesc(String keyword);

// キーワード検索 + タイトル順
@Select("SELECT * FROM todo WHERE title LIKE CONCAT('%', #{keyword}, '%') ORDER BY title ASC")
List<Todo> searchByTitleOrderByTitle(String keyword);
lightbulb MyBatisのソートの考え方

MyBatisではSQLを自分で書くため、ソート順ごとに別のメソッドを定義します。メソッド数は増えますが、各メソッドのSQLが明確で動作が予測しやすいのがメリットです。より高度な方法として、XMLマッパーで動的SQLを使う方法もありますが、この教材ではシンプルなアノテーション方式で進めます。

8-4-2 Controllerにソートロジックを追加する

TodoController.java の list メソッドをさらに修正して、ソートパラメータに応じて呼び出すMapperメソッドを切り替えます。

TodoController.java の list メソッドを変更（最終版） 
// 一覧表示（検索 + ソート対応）
@GetMapping("/todos")
public String list(@RequestParam(name = "keyword", required = false) String keyword,
                   @RequestParam(name = "sort", required = false,
                                 defaultValue = "dueDateAsc") String sortParam,
                   Model model) {
    List<Todo> todos;

    if (keyword != null && !keyword.isEmpty()) {
        // 検索 + ソート
        switch (sortParam) {
            case "dueDateDesc":
                todos = todoMapper.searchByTitleOrderByDueDateDesc(keyword);
                break;
            case "titleAsc":
                todos = todoMapper.searchByTitleOrderByTitle(keyword);
                break;
            default:
                todos = todoMapper.searchByTitleOrderByDueDate(keyword);
        }
    } else {
        // 全件 + ソート
        switch (sortParam) {
            case "dueDateDesc":
                todos = todoMapper.findAllOrderByDueDateDesc();
                break;
            case "titleAsc":
                todos = todoMapper.findAllOrderByTitle();
                break;
            default:
                todos = todoMapper.findAllOrderByDueDate();
        }
    }

    model.addAttribute("todos", todos);
    model.addAttribute("keyword", keyword);
    model.addAttribute("currentSort", sortParam);
    return "todos/list";
}

sort ソートパラメータと呼び出しメソッドの対応

sort=dueDateAsc → ORDER BY due_date ASC
sort=dueDateDesc → ORDER BY due_date DESC
sort=titleAsc → ORDER BY title ASC

8章 練習問題

Q1

MyBatisの @Select で「titleにキーワードを含むTodoを検索する」SQLとして正しいものはどれですか。

  1. SELECT * FROM todo WHERE title = #{keyword}
  2. SELECT * FROM todo WHERE title LIKE CONCAT('%', #{keyword}, '%')
  3. SELECT * FROM todo WHERE title CONTAINS #{keyword}
  4. SELECT * FROM todo WHERE title LIKE '%#{keyword}%'
Q2

MyBatisで期限日の昇順（近い順）にTodoを取得するSQLとして正しいものはどれですか。

  1. SELECT * FROM todo SORT BY due_date ASC
  2. SELECT * FROM todo ORDER BY due_date ASC
  3. SELECT * FROM todo WHERE ORDER due_date ASC
  4. SELECT * FROM todo GROUP BY due_date ASC
Q3

@RequestParam(name = "keyword", required = false) の説明として正しいものはどれですか。

  1. URLパラメータ「keyword」が必ず必要で、ない場合は400エラーになる
  2. URLパラメータ「keyword」がなくてもよく、ない場合はnullが設定される
  3. リクエストボディからJSON形式の「keyword」を取得する
  4. URLパスの一部として「keyword」を取得する
Answer

8章 解説・解答

Q1の解答: B

MyBatisではネイティブSQLを直接記述します。部分一致検索（LIKE '%keyword%'）には CONCAT 関数を使って CONCAT('%', #{keyword}, '%') と書きます。選択肢Aは完全一致検索です。選択肢Cの CONTAINS はSQLの標準的な構文ではありません。選択肢Dは #{keyword} が文字列リテラルの中に入ってしまい、パラメータバインドが正しく動作しません。

Q2の解答: B

SQLでソート順を指定するには ORDER BY 句を使います。ASC は昇順（小さい順）、DESC は降順（大きい順）です。SORT BY はSQLの標準構文ではなく、GROUP BY はグループ化（集約）に使う別の構文です。WHERE 句の中に ORDER は書けません。

Q3の解答: B

@RequestParam はURLパラメータ（クエリストリング）を受け取るアノテーションです。required = false を指定すると、パラメータが存在しない場合にnullが設定されます（required = true がデフォルトで、パラメータがないと400エラー）。URLパスの一部を受け取るのは @PathVariable、リクエストボディのJSONを受け取るのは @RequestBody です。
