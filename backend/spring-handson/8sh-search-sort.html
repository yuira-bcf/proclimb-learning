Section 8-2

カスタムクエリメソッドを作る

Spring Data JPAには、メソッド名のルールに従ってメソッドを定義するだけで、SQLを書かずにクエリを実行できる機能があります。

8-2-1 Spring Dataのメソッド名規則

Spring Data JPAはメソッド名を解析して、自動的にSQLクエリを生成します。

メソッド名 生成されるSQL
findByTitle(String title) WHERE title = ?
findByTitleContaining(String keyword) WHERE title LIKE '%keyword%'
findByDueDateBefore(LocalDate date) WHERE due_date < ?
findByTitleAndDescription(...) WHERE title = ? AND description = ?
findAllByOrderByDueDateAsc() ORDER BY due_date ASC
メソッド名の構造
findBy Title Containing
操作 フィールド名 条件キーワード

8-2-2 TodoRepositoryに検索メソッドを追加する

TodoRepository.java を開き、以下のメソッドを追加してください。

TodoRepository.java にメソッド追加 
package com.example.todo.repository;

import com.example.todo.entity.Todo;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.List;

public interface TodoRepository extends JpaRepository<Todo, Long> {

    // タイトルにキーワードを含むTodoを検索
    List<Todo> findByTitleContaining(String keyword);
}

たったこれだけです。メソッドの中身（SQL）はSpring Data JPAが自動生成します。インターフェースにメソッド定義を追加するだけで検索機能が使えるのが、Spring Data JPAの強力な特徴です。

lightbulb なぜ実装を書かなくてよいのか

Spring Data JPAは、アプリケーション起動時にRepositoryインターフェースのメソッド名を解析し、対応するSQL（実際にはJPQL）を自動生成します。findByTitleContainingは「Titleフィールドに対してContaining（部分一致）で検索する」と解釈されます。

Section 8-3

@Queryでカスタム検索を書く

メソッド名からの自動生成は便利ですが、複雑な検索条件ではメソッド名が長くなりすぎます。そのような場合は@Queryアノテーションで直接JPQLを書くことができます。

8-3-1 @Queryアノテーションの書き方

TodoRepository.java に以下のメソッドを追加してみましょう。これは先ほどのfindByTitleContainingと同じ処理をJPQLで書いたものです。

TodoRepository.java のimportに追加 
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
TodoRepository.java にメソッド追加 
// @Queryを使ったカスタム検索
@Query("SELECT t FROM Todo t WHERE t.title LIKE %:keyword%")
List<Todo> searchByTitle(@Param("keyword") String keyword);
JPQLの構文
SELECT t FROM Todo t WHERE t.title LIKE %:keyword%
SQLとの違い

JPQLはテーブル名ではなくEntityクラス名を、カラム名ではなくフィールド名を使います。

:keyword

@Param("keyword")で指定した引数の値がここに代入されます。SQLインジェクションも防げます。

lightbulb どちらを使うべき？

単純な条件ならメソッド名規則（findByTitleContaining）が簡潔でおすすめです。複数テーブルのJOINや複雑な条件が必要な場合は@Queryを使いましょう。今回はどちらの方法も知っておくために両方を実装しましたが、次のセクションではfindByTitleContainingを使います。

Section 8-4

検索フォームを一覧画面に追加する

Repositoryに検索メソッドを追加したので、次はControllerとHTMLテンプレートから使えるようにします。

8-4-1 Controllerの一覧メソッドを修正する

TodoController.java の list メソッドを以下のように変更してください。importにRequestParamを追加するのも忘れずに。

TodoController.java のimportに追加 
import org.springframework.web.bind.annotation.RequestParam;
TodoController.java の list メソッドを変更 
// 一覧表示（検索対応）
@GetMapping("/todo")
public String list(@RequestParam(name = "keyword", required = false) String keyword,
                   Model model) {
    List<Todo> todos;
    if (keyword != null && !keyword.isEmpty()) {
        todos = todoRepository.findByTitleContaining(keyword);
    } else {
        todos = todoRepository.findAll();
    }
    model.addAttribute("todos", todos);
    model.addAttribute("keyword", keyword);
    return "todo/list";
}

importにjava.util.Listも必要です。忘れずに追加してください。

Section 8-5

ソート機能を追加する

検索に加えて、期限日の昇順（近い順）・降順（遠い順）でソートできるようにしましょう。

8-5-1 Sort.byを使ったソート

Spring Data JPAのSortクラスを使うと、動的にソート順を変更できます。TodoController.java の list メソッドをさらに修正しましょう。

TodoController.java のimportに追加 
import org.springframework.data.domain.Sort;
TodoController.java の list メソッドを変更（最終版） 
// 一覧表示（検索 + ソート対応）
@GetMapping("/todo")
public String list(@RequestParam(name = "keyword", required = false) String keyword,
                   @RequestParam(name = "sort", required = false,
                                 defaultValue = "dueDateAsc") String sortParam,
                   Model model) {
    Sort sort;
    switch (sortParam) {
        case "dueDateDesc":
            sort = Sort.by(Sort.Direction.DESC, "dueDate");
            break;
        case "titleAsc":
            sort = Sort.by(Sort.Direction.ASC, "title");
            break;
        default:
            sort = Sort.by(Sort.Direction.ASC, "dueDate");
    }

    List<Todo> todos;
    if (keyword != null && !keyword.isEmpty()) {
        todos = todoRepository.findByTitleContaining(keyword, sort);
    } else {
        todos = todoRepository.findAll(sort);
    }
    model.addAttribute("todos", todos);
    model.addAttribute("keyword", keyword);
    model.addAttribute("currentSort", sortParam);
    return "todo/list";
}

findByTitleContaining に Sort 引数を追加したので、Repositoryのメソッドも合わせて更新します。

TodoRepository.java のメソッドを変更 
import org.springframework.data.domain.Sort;

public interface TodoRepository extends JpaRepository<Todo, Long> {

    // タイトルにキーワードを含むTodoを検索（ソート対応）
    List<Todo> findByTitleContaining(String keyword, Sort sort);

    // @Queryを使ったカスタム検索（参考）
    @Query("SELECT t FROM Todo t WHERE t.title LIKE %:keyword%")
    List<Todo> searchByTitle(@Param("keyword") String keyword);
}
Sort の仕組み
Sort.by(Direction.ASC, "dueDate") → ORDER BY due_date ASC
Sort.by(Direction.DESC, "dueDate") → ORDER BY due_date DESC
Sort.by(Direction.ASC, "title") → ORDER BY title ASC

8-5-2 ソートリンクをHTMLに追加する

list.html の検索フォームの下に、ソート切り替えリンクを追加してください。

list.html に追加（検索フォームの下） 
<div style="margin:8px 0; font-size:small;">
  並び替え:
  <a th:href="@{/todo(keyword=${keyword}, sort='dueDateAsc')}">期限日 ↑</a> |
  <a th:href="@{/todo(keyword=${keyword}, sort='dueDateDesc')}">期限日 ↓</a> |
  <a th:href="@{/todo(keyword=${keyword}, sort='titleAsc')}">タイトル順</a>

Thymeleafの @{/todo(keyword=${keyword}, sort='dueDateAsc')} は、URLパラメータ付きのリンクを生成します。検索キーワードを保持したままソートが切り替わります。

8章 練習問題

Q1

Spring Data JPAで「titleフィールドにキーワードを含むデータを検索する」メソッド名として正しいものはどれですか。

  1. findByTitleLike(String keyword)
  2. findByTitleContaining(String keyword)
  3. searchByTitle(String keyword)
  4. findByTitleEquals(String keyword)
Q2

以下の@Queryアノテーションの記述で、空欄に入るJPQLとして正しいものはどれですか。
@Query("______")
List<Todo> searchByTitle(@Param("keyword") String keyword);

  1. SELECT * FROM todo WHERE title LIKE '%' + :keyword + '%'
  2. SELECT t FROM Todo t WHERE t.title LIKE %:keyword%
  3. SELECT Todo FROM Todo WHERE title CONTAINS :keyword
  4. FIND Todo WHERE Todo.title HAS :keyword
Q3

@RequestParam(name = "keyword", required = false) の説明として正しいものはどれですか。

  1. URLパラメータ「keyword」が必ず必要で、ない場合は400エラーになる
  2. URLパラメータ「keyword」がなくてもよく、ない場合はnullが設定される
  3. リクエストボディからJSON形式の「keyword」を取得する
  4. URLパスの一部として「keyword」を取得する
Answer

8章 解説・解答

Q1の解答: B

Spring Data JPAのメソッド名規則で、部分一致検索（LIKE '%keyword%'）を行うにはContainingキーワードを使います。findByTitleContainingは「titleに指定した文字列を含むレコードを検索する」という意味になります。Likeは自分で%を付ける必要があり、Equalsは完全一致です。searchByTitleはSpring Data JPAの命名規則に合致しないため、自動でクエリが生成されません。

Q2の解答: B

JPQLはSQLに似ていますが、テーブル名の代わりにEntityクラス名（Todo）を、カラム名の代わりにフィールド名（t.title）を使います。パラメータは:keywordの形式で、@Param("keyword")で引数と紐づけます。LIKE検索の%は文字列の前後に直接置くことができます。選択肢Aは通常のSQL構文であり、JPQLでは使えません。

Q3の解答: B

@RequestParamはURLパラメータ（クエリストリング）を受け取るアノテーションです。required = falseを指定すると、パラメータが存在しない場合にnullが設定されます（required = trueがデフォルトで、パラメータがないと400エラー）。URLパスの一部を受け取るのは@PathVariable、リクエストボディのJSONを受け取るのは@RequestBodyです。
