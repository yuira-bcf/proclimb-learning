6-1-2 編集フォーム表示のControllerメソッド

まず、編集画面を表示するメソッドを TodoController.java に追加します。URLに含まれる {id} でどのTodoを編集するか判別します。

新しいimport文を追加してください。

TodoController.java（import追加） 
import org.springframework.web.bind.annotation.PathVariable;

そして、以下のメソッドを追加してください。

TodoController.java（メソッドを追加） 
// 編集フォーム表示
@GetMapping("/todos/{id}/edit")
public String showEditForm(@PathVariable Long id, Model model) {
    Todo todo = todoMapper.findById(id);
    if (todo == null) {
        return "redirect:/todos";
    }
    model.addAttribute("todo", todo);
    return "todos/edit";
}
lightbulb @PathVariableとは

URLの {id} 部分の値をメソッドの引数に自動で渡してくれます。例えば /todos/3/edit にアクセスすると、id に 3 がセットされます。

Section 6-2

findById()とnullチェック

6-2-1 findById()の戻り値とnullチェック

MyBatisの findById() は、データが見つからない場合に null を返します。そのため、取得後に nullチェック を行って、データが存在しない場合の処理を記述する必要があります。

まず TodoMapper.java に findById メソッドを追加しましょう。

TodoMapper.java にメソッドを追加 
@Select("SELECT * FROM todo WHERE id = #{id}")
Todo findById(Long id);

inventory_2 findById() の戻り値

データがある場合
Todoオブジェクトが返る
Todo{id=3, title="買い物"}
データがない場合
null が返る（存在しないID）
null

6-2-2 nullチェックのパターン

MyBatisではデータが存在しない場合に null が返るため、if文でnullチェックを行います。

code nullチェックのパターン

if (todo == null) { return "redirect:/todos"; }
データが見つからなければ一覧画面にリダイレクト。今回はこれを使用。
if (todo == null) { throw new IllegalArgumentException(...); }
データが見つからなければ例外を投げてエラーにする。
if (todo != null) { /* 処理 */ }
データがある場合だけ処理を実行する。
nullチェックの使い方（比較） 
// 方法1: nullならリダイレクト（今回のアプローチ）
Todo todo = todoMapper.findById(id);
if (todo == null) {
    return "redirect:/todos";
}

// 方法2: nullなら例外を投げる
Todo todo = todoMapper.findById(id);
if (todo == null) {
    throw new IllegalArgumentException("不正なID: " + id);
}

// 方法3: nullチェックしてから処理
Todo todo = todoMapper.findById(id);
if (todo != null) {
    model.addAttribute("todo", todo);
}

difference create.html との違い

<input type="hidden" th:field="*{id}">
IDを隠しフィールドで送信。update()でWHERE条件に使うために必要。
<input type="hidden" th:field="*{createdAt}">
作成日時を保持。これがないとupdate()時にnullで上書きされてしまう。
th:action="@{/todos/{id}/edit(id=${todo.id})}"
送信先URLにIDを含める。Thymeleafの変数式でURL内のパラメータを置換。

6-3-2 更新処理のControllerメソッドを追加する

TodoController.java に更新処理のメソッドを追加してください。

TodoController.java（メソッドを追加） 
// 更新処理
@PostMapping("/todos/{id}/edit")
public String updateTodo(@PathVariable Long id,
                         @ModelAttribute Todo todo) {
    todo.setId(id);
    todoMapper.update(todo);
    return "redirect:/todos";
}

TodoMapper.java に update メソッドを追加してください。

TodoMapper.java にメソッドを追加 
@Update("UPDATE todo SET title = #{title}, description = #{description}, due_date = #{dueDate}, completed = #{completed} WHERE id = #{id}")
void update(Todo todo);

info insert()とupdate()の使い分け

MyBatisでは新規登録と更新で別のメソッド（insert() / update()）を使います。update() の SQL には WHERE id = #{id} を指定して、対象レコードを特定します。フォームの隠しフィールドで送られた id がこの条件に使われます。

Section 6-4

削除処理を実装する

6-4-1 deleteById()メソッド

削除には TodoMapper の deleteById() メソッドを使います。IDを指定するだけで対象のレコードが削除されます。

まず TodoMapper.java に deleteById メソッドを追加してください。

TodoMapper.java にメソッドを追加 
@Delete("DELETE FROM todo WHERE id = #{id}")
void deleteById(Long id);

次に TodoController.java に以下のメソッドを追加してください。

TodoController.java（メソッドを追加） 
// 削除処理
@PostMapping("/todos/{id}/delete")
public String deleteTodo(@PathVariable Long id) {
    todoMapper.deleteById(id);
    return "redirect:/todos";
}
warning なぜPOSTで削除するのか

GETリクエスト（リンク）で削除すると、ブラウザのプリフェッチやクローラーが意図せず削除を実行してしまう危険があります。データを変更する操作は必ずPOSTで行いましょう。HTMLのフォームは GET と POST のみ対応しているため、DELETE メソッドは使いません。

6-4-2 削除ボタンのHTMLパターン

削除はリンクではなく、フォーム + ボタンで実装します。次のセクションで一覧画面に組み込みます。

削除ボタンのHTMLパターン 
<form th:action="@{/todos/{id}/delete(id=${todo.id})}"
      method="post"
      onsubmit="return confirm('本当に削除しますか？')">
    <button type="submit">削除</button>
</form>
lightbulb confirm() について

onsubmit="return confirm('...')" はJavaScriptの確認ダイアログを表示します。「OK」を押すと送信、「キャンセル」で送信を中止します。誤操作による削除を防ぐための簡易的な対策です。

6-5-3 Controllerの全体像とまとめ

この章までで作成した TodoController.java の全体像です。

src/main/java/com/example/todo/controller/TodoController.java（全体） 
package com.example.todo.controller;

import com.example.todo.entity.Todo;
import com.example.todo.mapper.TodoMapper;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.PostMapping;
import java.time.LocalDateTime;

@Controller
public class TodoController {

    private final TodoMapper todoMapper;

    public TodoController(TodoMapper todoMapper) {
        this.todoMapper = todoMapper;
    }

    // 一覧表示
    @GetMapping("/todos")
    public String list(Model model) {
        model.addAttribute("todos", todoMapper.findAll());
        return "todos/list";
    }

    // 新規作成フォーム表示
    @GetMapping("/todos/new")
    public String showCreateForm(Model model) {
        model.addAttribute("todo", new Todo());
        return "todos/create";
    }

    // 新規作成処理
    @PostMapping("/todos")
    public String createTodo(@ModelAttribute Todo todo) {
        todo.setCreatedAt(LocalDateTime.now());
        todoMapper.insert(todo);
        return "redirect:/todos";
    }

    // 編集フォーム表示
    @GetMapping("/todos/{id}/edit")
    public String showEditForm(@PathVariable Long id, Model model) {
        Todo todo = todoMapper.findById(id);
        if (todo == null) {
            return "redirect:/todos";
        }
        model.addAttribute("todo", todo);
        return "todos/edit";
    }

    // 更新処理
    @PostMapping("/todos/{id}/edit")
    public String updateTodo(@PathVariable Long id,
                             @ModelAttribute Todo todo) {
        todo.setId(id);
        todoMapper.update(todo);
        return "redirect:/todos";
    }

    // 削除処理
    @PostMapping("/todos/{id}/delete")
    public String deleteTodo(@PathVariable Long id) {
        todoMapper.deleteById(id);
        return "redirect:/todos";
    }
}

6章 練習問題

Q1

URLの /todos/{id}/edit で {id} の値をControllerメソッドの引数で受け取るために使うアノテーションはどれですか。

  1. @RequestParam
  2. @PathVariable
  3. @ModelAttribute
  4. @RequestBody
Q2

todoMapper.findById(99) を呼び出したとき、ID=99のデータが存在しない場合の戻り値として正しいのはどれですか。

  1. null
  2. Optional.empty()
  3. 例外が発生する
  4. 空のTodoオブジェクト
Q3

削除処理をGETリクエスト（リンク）ではなくPOSTリクエスト（フォーム）で実装する理由として正しいのはどれですか。

  1. GETリクエストでは削除が技術的に不可能だから
  2. POSTの方が処理速度が速いから
  3. GETだとブラウザのプリフェッチやクローラーが意図せず削除を実行する可能性があるから
  4. GETだとデータベースに接続できないから
Answer

6章 解説・解答

Q1の解答: B

@PathVariable はURL内の {id} などのプレースホルダーの値を引数にマッピングします。@RequestParam はクエリパラメータ（?key=value）用、@ModelAttribute はフォームデータのバインド用、@RequestBody はJSONなどのリクエストボディ用です。

Q2の解答: A

MyBatisの @Select で単一オブジェクトを返すメソッドは、該当データが存在しない場合に null を返します。そのためControllerでは if (todo == null) のようにnullチェックを行い、データが存在しない場合の処理（リダイレクトや例外）を記述する必要があります。

Q3の解答: C

GETリクエストは「安全なリクエスト」（副作用がない）と見なされます。ブラウザのプリフェッチ機能やWebクローラーがGETリクエストを自動実行することがあり、削除リンクが意図せず呼ばれる危険があります。POSTリクエストはユーザーの明示的なアクション（ボタンクリック）が必要なため、安全です。
