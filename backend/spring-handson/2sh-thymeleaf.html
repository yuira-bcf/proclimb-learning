Section 2-1

Controllerの役割を理解する

前章ではHello Worldを表示しましたが、今回はControllerの仕組みをもう少し深く理解し、さまざまなデータをHTMLに渡す方法を学びます。

2-1-1 MVCパターンとControllerの位置づけ

Spring Bootは MVC（Model-View-Controller） パターンを採用しています。Controllerはブラウザからのリクエストを受け取り、処理結果をViewに渡す「交通整理役」です。

hub MVCパターンの全体像

2-1-2 @Controllerと@GetMappingの仕組み

Controllerには2つの重要なアノテーション（注釈）があります。

label アノテーションの役割

@Controller
「このクラスはWebリクエストを処理する役割だよ」とSpringに伝える。起動時に自動でインスタンスが作られる。
@GetMapping("/パス")
「GETリクエストでこのパスにアクセスされたらこのメソッドを呼んで」という指定。URLとメソッドを紐づける。
lightbulb GETリクエストとは

ブラウザのアドレスバーにURLを入力してアクセスすると「GET」リクエストが送信されます。データの「取得」に使います。フォームの送信は「POST」リクエストで、第5章で学びます。

Section 2-2

Modelにデータを渡す

2-2-1 文字列を渡す

前章の HelloController を改良して、Todoアプリ用の TodoController を作りましょう。

src/main/java/com/example/todo/controller/ に TodoController.java を新規作成し、以下のコードを手で入力してください。

src/main/java/com/example/todo/controller/TodoController.java 
package com.example.todo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class TodoController {

    @GetMapping("/todos")
    public String list(Model model) {
        model.addAttribute("title", "Todoリスト");
        return "todo/list";
    }
}
info コードの解説
  • model.addAttribute("title", "Todoリスト") ── "title"という名前で文字列をHTMLに渡す
  • return "todo/list" ── templates/todo/list.html を表示する（サブフォルダも指定可能）

2-2-2 リストを渡す

次に、複数のTodoを文字列リストとして渡してみましょう。TodoController.java の list メソッドに以下の行を追加してください。

TodoController.java（list メソッドに追加） 
@GetMapping("/todos")
public String list(Model model) {
    model.addAttribute("title", "Todoリスト");

    // 仮データとしてリストを渡す
    List<String> todos = List.of(
        "Spring Bootの環境構築",
        "Thymeleafの基本を学ぶ",
        "Entityクラスを作成する"
    );
    model.addAttribute("todos", todos);

    return "todo/list";
}

import文も追加が必要です。ファイルの先頭に 以下の1行を追加してください。

import文を追加 
import java.util.List;

);
→

Section 2-3

Thymeleafで動的表示する

2-3-1 一覧ページのHTMLを作成する

src/main/resources/templates/ に todo フォルダを新規作成し、その中に list.html を作成してください。

以下のHTMLを手で入力してください。

src/main/resources/templates/todo/list.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title th:text="${title}">Todoリスト</title>
</head>
<body>
    <h1 th:text="${title}">タイトル</h1>

    <!-- リストが空の場合のメッセージ -->
    <p th:if="${#lists.isEmpty(todos)}">
        Todoはまだありません。
    </p>

    <!-- Todoリストをループ表示 -->
    <ul>
        <li th:each="todo : ${todos}" th:text="${todo}">
            サンプルTodo
        </li>
    </ul>

    <p>合計: <span th:text="${#lists.size(todos)}">0</span> 件</p>
</body>
</html>

code Thymeleafの主要構文

構文 役割 例
th:text テキストを置き換える th:text="${title}"
th:each リストをループ表示 th:each="item : ${list}"
th:if 条件に合うときだけ表示 th:if="${condition}"
th:href リンクURLを動的に生成 th:href="@{/path}"

2-3-2 実行して確認しましょう

アプリケーションを再起動して、ブラウザで http://localhost:8080/todos にアクセスしてください。

check_circle 期待される表示

localhost:8080/todos

  • Spring Bootの環境構築

合計: 3 件

3つのTodoが箇条書きで表示されていれば成功です。

Section 2-4

複数ページを作る

2-4-1 詳細ページ用のControllerメソッドを追加する

一覧ページに加えて、各Todoの詳細を表示するページを作りましょう。TodoController.java に以下のメソッドを追加してください。

TodoController.java に追加するメソッド 
@GetMapping("/todos/{id}")
public String detail(@PathVariable("id") int id, Model model) {
    // 仮データ（後でDBから取得するように変更します）
    List<String> todos = List.of(
        "Spring Bootの環境構築",
        "Thymeleafの基本を学ぶ",
        "Entityクラスを作成する"
    );

    if (id >= 0 && id < todos.size()) {
        model.addAttribute("todo", todos.get(id));
        model.addAttribute("id", id);
    }

    return "todo/detail";
}

import文も追加が必要です。

import文を追加 
import org.springframework.web.bind.annotation.PathVariable;

link URLパスとパス変数

GET /todos/0 → detail() の id = 0 → "Spring Bootの環境構築"
GET /todos/1 → detail() の id = 1 → "Thymeleafの基本を学ぶ"
GET /todos/2 → detail() の id = 2 → "Entityクラスを作成する"
{id} の部分がURLの値と自動でマッチします。@PathVariable でメソッド引数として受け取ります。

2-4-2 詳細ページのHTMLを作成する

src/main/resources/templates/todo/ に detail.html を新規作成し、以下のHTMLを手で入力してください。

src/main/resources/templates/todo/detail.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Todo詳細</title>
</head>
<body>
    <h1>Todo詳細</h1>
    <p>ID: <span th:text="${id}">0</span></p>
    <p>内容: <span th:text="${todo}">サンプル</span></p>

    <a th:href="@{/todos}">← 一覧に戻る</a>
</body>
</html>

2-4-3 一覧ページにリンクを追加する

list.html の <li> タグを修正して、各Todoをクリックすると詳細ページに遷移するようにしましょう。list.html の <ul> 部分を以下に書き換えてください。

list.html の <ul> 部分を修正 
<ul>
    <li th:each="todo, iterStat : ${todos}">
        <a th:href="@{/todos/{id}(id=${iterStat.index})}"
           th:text="${todo}">サンプルTodo</a>
    </li>
</ul>
lightbulb iterStatとは

th:each="todo, iterStat : ${todos}" の iterStat はループ状態を表すオブジェクトです。iterStat.index で現在のインデックス（0始まり）を取得できます。@{/todos/{id}(id=${iterStat.index})} はThymeleafのURL構文で、パス変数に値を埋め込みます。

2-4-4 実行して確認しましょう

アプリケーションを再起動して、ブラウザで確認しましょう。

  1. http://localhost:8080/todos にアクセス
  2. 各Todoのリンクをクリックして、詳細ページに遷移することを確認
  3. 詳細ページの「一覧に戻る」リンクで戻れることを確認

2章 練習問題

Q1

以下のThymeleafのコードで、items リストに ["りんご", "バナナ", "みかん"] が渡された場合、生成されるHTMLのli要素は何個ですか。

HTML
<ul>
    <li th:each="item : ${items}" th:text="${item}"></li>
</ul>
  1. 1個
  2. 2個
  3. 3個
  4. 4個
Q2

Controllerのメソッドで return "todo/list" と記述した場合、Thymeleafが参照するHTMLファイルのパスはどれですか。

  1. src/main/java/todo/list.html
  2. src/main/resources/templates/todo/list.html
  3. src/main/resources/static/todo/list.html
  4. templates/todo/list.html
Q3

以下のControllerメソッドにおいて、model.addAttribute("name", "太郎") で渡したデータをHTMLで表示する正しいThymeleaf構文はどれですか。

  1. <p th:text="name"></p>
  2. <p th:text="${name}"></p>
  3. <p th:value="${name}"></p>
  4. <p text="${name}"></p>
Answer

2章 解説・解答

Q1の解答: C

th:each はリストの要素数だけHTMLタグを繰り返し生成します。["りんご", "バナナ", "みかん"] は3つの要素があるため、<li> が3個生成されます。テンプレート上では1行で書いても、実際のHTMLでは要素数分に展開されます。

Q2の解答: B

Thymeleafは src/main/resources/templates/ をルートとして、戻り値の文字列に .html を付けたパスを参照します。return "todo/list" は src/main/resources/templates/todo/list.html を表示します。static/ はCSS・JS・画像用のフォルダです。

Q3の解答: B

Thymeleafで変数を参照するには ${変数名} の形式で書きます。th:text="${name}" が正解です。${} を付けないとリテラル文字列として扱われます。th:value はフォーム要素のvalue属性用で、テキスト表示には th:text を使います。
