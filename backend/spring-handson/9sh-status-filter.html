Section 9-1

ステータス管理の設計

これまでのTodoアプリには「完了したかどうか」を管理する仕組みがありませんでした。この章では、Todoにステータス（未着手・進行中・完了）を持たせ、一覧画面からワンクリックで切り替えたり、ステータスでフィルタリングしたりできるようにします。

9-1-1 なぜステータスが必要か

現実のタスク管理では、タスクには複数の状態があります。単純な「完了/未完了」ではなく、「まだ手をつけていない」「作業中」「終わった」という3つの状態を管理することで、進捗を正確に把握できるようになります。

ステータスは順番通りに進むだけでなく、戻すこともできる設計にします

9-1-2 Enumで状態を表現するメリット

ステータスをStringで管理すると、タイプミスによるバグが起きやすくなります。JavaのEnumを使えば、取りうる値をコンパイル時に限定できるため安全です。

lightbulb Enumとは

Enum（列挙型）は、決められた定数の集合を表すJavaの型です。例えば「曜日」「季節」「ステータス」のように、取りうる値が限定されている場合に使います。switch文やif文で安全に分岐できるのが大きなメリットです。

String vs Enum の比較

String（危険）
todo.setStatus("complated");
// タイプミスに気づけない！
Enum（安全）
todo.setStatus(TodoStatus.COMPLETED);
// コンパイルエラーで検出！
Section 9-2

Enumを作成しEntityに追加する

まずはTodoStatus Enumを新しいファイルとして作成し、次にTodo Entityにstatusフィールドを追加します。

9-2-1 TodoStatus Enumを作成する

com.example.todoパッケージに新しいファイルTodoStatus.javaを作成してください。以下のコードを手で入力しましょう。

src/main/java/com/example/todo/TodoStatus.java 
package com.example.todo;

public enum TodoStatus {
    NOT_STARTED("未着手"),
    IN_PROGRESS("進行中"),
    COMPLETED("完了");

    private final String displayName;

    TodoStatus(String displayName) {
        this.displayName = displayName;
    }

    public String getDisplayName() {
        return displayName;
    }
}
info コードの解説
  • NOT_STARTED, IN_PROGRESS, COMPLETED -- 3つの定数を定義しています。データベースにはこの英語名で保存されます。
  • displayName -- 画面に表示する日本語名を保持するフィールドです。
  • getDisplayName() -- Thymeleafテンプレートから th:text="${todo.status.displayName}" のように呼び出して日本語表示できます。

9-2-2 Todo Entityにstatusフィールドを追加する

既存のTodo.javaを開いて、以下のフィールドを追加してください。dueDateフィールドの下あたりに書くのがよいでしょう。

Todo.java に追加するフィールド 
    private TodoStatus status = TodoStatus.NOT_STARTED;

続けて、getterとsetterも追加しましょう。

Todo.java に追加するgetter/setter 
    public TodoStatus getStatus() {
        return status;
    }

    public void setStatus(TodoStatus status) {
        this.status = status;
    }

storage MyBatisのEnum自動マッピング

Javaコード側
todo.setStatus(TodoStatus.COMPLETED);

Enumの定数名をそのまま使用。型安全。

データベース側
status = 'COMPLETED'

MyBatisが自動で文字列名に変換して保存。

MyBatisはデフォルトでEnumのname()（定数名）を使ってVARCHAR列と相互変換します。JPAのように@Enumeratedアノテーションは不要です。

lightbulb なぜアノテーションが不要なのか

MyBatisにはEnumTypeHandlerという仕組みが組み込まれており、JavaのEnum型とデータベースのVARCHAR型を自動的に変換してくれます。Enumの定数名（NOT_STARTED、COMPLETEDなど）がそのまま文字列としてDBに保存され、読み出し時には自動的にEnum定数に戻されます。Todo.javaはただのPOJO（普通のJavaクラス）なので、特別なアノテーションは一切必要ありません。

Section 9-3

ステータス切り替え機能を実装する

一覧画面でボタンを押すだけでステータスを次の状態に進められる「トグル機能」を実装します。NOT_STARTED → IN_PROGRESS → COMPLETED → NOT_STARTED ... と巡回する仕組みです。

9-3-1 TodoStatusにnext()メソッドを追加する

TodoStatus.javaを開いて、次の状態を返すメソッドを追加しましょう。getDisplayName()の下に以下のメソッドを入力してください。

TodoStatus.java に追加するメソッド 
    public TodoStatus next() {
        TodoStatus[] values = TodoStatus.values();
        return values[(this.ordinal() + 1) % values.length];
    }
info ordinal() と values() とは
  • ordinal() -- Enumの定義順を0から返します。NOT_STARTED=0, IN_PROGRESS=1, COMPLETED=2
  • values() -- 全てのEnum定数を配列で返します
  • % values.length -- 剰余演算で配列の末尾から先頭に巡回します（2+1=3, 3%3=0 → NOT_STARTEDに戻る）

9-3-2 Controllerにtoggleエンドポイントを追加する

TodoController.javaに以下のメソッドを追加してください。

TodoController.java に追加 
    @PostMapping("/{id}/toggle-status")
    public String toggleStatus(@PathVariable Long id) {
        Todo todo = todoMapper.findById(id);
        if (todo == null) {
            throw new IllegalArgumentException("不正なID: " + id);
        }
        todo.setStatus(todo.getStatus().next());
        todoMapper.update(todo);
        return "redirect:/todos";
    }

このメソッドは、URLパスに含まれるIDでTodoを検索し、現在のステータスのnext()を呼んで次の状態に進め、更新してから一覧にリダイレクトします。MyBatisのfindById()はnullを返す可能性があるため、nullチェックを行っています。

9-3-3 一覧テンプレートにステータス切り替えボタンを追加する

list.htmlのTodo一覧テーブルの各行に、ステータス表示と切り替えボタンを追加しましょう。既存のテーブルのtd要素の中に以下を追加してください。

list.html - ステータス列の追加 
<!-- テーブルヘッダーに追加 -->
<th>ステータス</th>

<!-- テーブルボディのth:eachの中に追加 -->
<td>
    <form th:action="@{'/todos/' + ${todo.id} + '/toggle-status'}" method="post" style="display:inline;">
        <button type="submit" th:text="${todo.status.displayName}"
                th:classappend="${todo.status.name() == 'COMPLETED'} ? 'btn-success' :
                               (${todo.status.name() == 'IN_PROGRESS'} ? 'btn-primary' : 'btn-secondary')">
        </button>
    </form>
</td>

Section 9-4

フィルタ機能を実装する

Todoが増えてくると、全件表示では見づらくなります。ステータスごとに絞り込めるフィルタ機能を追加しましょう。

9-4-1 Mapperにフィルタ用メソッドを追加する

TodoMapper.javaに以下のメソッドを追加してください。

TodoMapper.java に追加 
    @Select("SELECT * FROM todo WHERE status = #{status}")
    List<Todo> findByStatus(TodoStatus status);

MyBatisでは@Selectアノテーションに直接SQLを書きます。#{status}は引数の値がバインドされるプレースホルダです。TodoStatus型の引数を渡すと、MyBatisが自動的にEnumの定数名（文字列）に変換してSQLに渡します。

auto_fix_high MyBatisのSQLマッピングの仕組み

@Select("SELECT * FROM todo WHERE status = #{status}")
↓ SQLアノテーション                                ↓ パラメータバインド
↓
SELECT * FROM todo WHERE status = 'NOT_STARTED'

9-4-2 Controllerのlist()メソッドを修正する

既存のlist()メソッドを以下のように変更してください。@RequestParamでステータスのフィルタ値を受け取り、値がある場合はフィルタリング、ない場合は全件表示する仕組みです。

TodoController.java - list()を修正 
    @GetMapping
    public String list(@RequestParam(required = false) TodoStatus status,
                       Model model) {
        List<Todo> todos;
        if (status != null) {
            todos = todoMapper.findByStatus(status);
        } else {
            todos = todoMapper.findAll();
        }
        model.addAttribute("todos", todos);
        model.addAttribute("currentStatus", status);
        model.addAttribute("statuses", TodoStatus.values());
        return "list";
    }
info ポイント解説
  • @RequestParam(required = false) -- パラメータが省略可能であることを示します。省略時はnullになります。
  • TodoStatus.values() -- テンプレートでフィルタボタンを動的に生成するために全ステータスを渡しています。
  • currentStatus -- 現在選択中のフィルタを画面でハイライトするために渡します。

9-4-3 テンプレートにフィルタボタンを追加する

list.htmlのテーブルの上に、以下のフィルタボタン群を追加してください。

list.html - フィルタボタン 
<div style="margin-bottom: 16px;">
    <a th:href="@{/todos}" th:classappend="${currentStatus == null} ? 'active'">すべて</a>
    <span th:each="s : ${statuses}">
        <a th:href="@{/todos(status=${s})}"
           th:text="${s.displayName}"
           th:classappend="${currentStatus != null and currentStatus.name() == s.name()} ? 'active'">
        </a>
    </span>
</div>

th:each="s : ${statuses}" で全ステータスをループし、それぞれのフィルタリンクを動的に生成しています。currentStatusと一致するボタンにはactiveクラスが付きます。

実行して確認しましょう。ブラウザで http://localhost:8080/todos を開き、フィルタボタンをクリックすると、URLが /todos?status=COMPLETED のように変わり、該当するTodoだけが表示されるはずです。

9章 練習問題

Q1

MyBatisでJavaのEnum型をデータベースに保存する際の動作について正しいものはどれですか。

  1. Enumは保存できないため、事前にString型に変換する必要がある
  2. MyBatisのEnumTypeHandlerにより、Enumの定数名（name()）が自動的に文字列として保存される
  3. Enumの定義順（ordinal: 0, 1, 2...）が自動的に数値として保存される
  4. @Enumerated アノテーションを付けないとエラーになる
Q2

MyBatisのTodoMapperに@Select("SELECT * FROM todo WHERE status = #{status}")と定義したメソッドにTodoStatus.NOT_STARTEDを渡したとき、実行されるSQLとして正しいものはどれですか。

  1. SELECT * FROM todo WHERE status LIKE 'NOT_STARTED'
  2. SELECT * FROM todo WHERE status = 'NOT_STARTED'
  3. SELECT * FROM todo WHERE status = 0
  4. SELECT status FROM todo WHERE status = 'NOT_STARTED'
Q3

@RequestParam(required = false) TodoStatus status とコントローラのメソッド引数に書いた場合、URLに?statusパラメータが含まれないとき、status変数の値はどうなりますか。

  1. 空文字列（""）が代入される
  2. TodoStatus.NOT_STARTED がデフォルトで代入される
  3. nullが代入される
  4. 例外（MissingServletRequestParameterException）がスローされる
Answer

9章 解説・解答

Q1の解答: B

MyBatisにはデフォルトでEnumTypeHandlerが組み込まれており、JavaのEnum型を自動的にその定数名（name()の戻り値）で文字列としてデータベースに保存します。JPAの@Enumeratedのような特別なアノテーションは不要です。データベース側のカラムはVARCHAR型にしておけば、"NOT_STARTED"、"COMPLETED"のように人間が読める形で保存されます。

Q2の解答: B

MyBatisの#{status}はプリペアドステートメントのパラメータとしてバインドされます。TodoStatus.NOT_STARTEDを渡すと、EnumTypeHandlerが自動的に文字列"NOT_STARTED"に変換し、SELECT * FROM todo WHERE status = 'NOT_STARTED'が実行されます。@Selectに書いたSQLがそのまま使われるため、Spring Data JPAのようなメソッド名からの自動生成ではなく、開発者がSQLを明示的に記述します。

Q3の解答: C

required = falseを指定すると、パラメータが省略された場合にnullが代入されます。デフォルトではrequired = trueであり、パラメータが省略されると例外がスローされます。nullの可能性がある場合は、Controller側でif (status != null)のようにnullチェックを行う必要があります。
