Section 10-2

Bootstrapを導入する

Bootstrap 5をCDN経由で読み込むだけで、すぐに使い始められます。npmやダウンロードは不要です。

10-2-1 CDNリンクの追加

まだlayout.htmlは作成していないので、まずはlist.htmlの<head>タグ内に以下の2行を追加してください。CSSは<head>内に、JavaScriptは</body>の直前に配置します。

list.html の <head> 内に追加 
<!-- Bootstrap CSS -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
      rel="stylesheet">
list.html の </body> 直前に追加 
<!-- Bootstrap JS -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
lightbulb CDNとは

CDN（Content Delivery Network）は、ライブラリのファイルを世界中のサーバーから配信するサービスです。自分のプロジェクトにファイルをダウンロードしなくても、URLを指定するだけでBootstrapを利用できます。開発中はこの方法が最も手軽です。

Section 10-3

共通レイアウトを作る

すべてのページに共通するヘッダー・フッター・CDNリンクを1つのファイルにまとめ、各ページから再利用できるようにします。Thymeleafのフラグメント機能を使います。

10-3-1 layout.htmlを作成する

src/main/resources/templates/にlayout.htmlを新規作成し、以下のコードを手で入力してください。

src/main/resources/templates/layout.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org" lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title th:text="${pageTitle} ? ${pageTitle} + ' - Todo App' : 'Todo App'">Todo App</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
          rel="stylesheet">
    <style>
        .navbar-brand { font-weight: 700; }
        .footer { background: #f8f9fa; padding: 20px 0; margin-top: 40px; text-align: center;
                  color: #6c757d; font-size: 0.85rem; }
    </style>
</head>
<body>
    <!-- ナビバー -->
    <nav class="navbar navbar-expand-lg navbar-dark bg-dark">
        <div class="container">
            <a class="navbar-brand" th:href="@{/todos}">Todo App</a>
            <ul class="navbar-nav ms-auto">
                <li class="nav-item">
                    <a class="nav-link" th:href="@{/todos}">一覧</a>
                </li>
                <li class="nav-item">
                    <a class="nav-link" th:href="@{/todos/create}">新規登録</a>
                </li>
            </ul>
        </div>
    </nav>

    <!-- メインコンテンツ -->
    <main class="container mt-4">
        <div th:replace="~{${viewName} :: content}">ここにページ固有のコンテンツが入る</div>
    </main>

    <!-- フッター -->
    <div class="footer">
        <div class="container">
            &copy; 2026 Todo App - Spring Boot ハンズオン
        </div>
    </div>

    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

dashboard レイアウト構造

Navbar（layout.html が提供）
コンテンツ領域
th:replace="~{${viewName} :: content}"
list.html / create.html / edit.html の
th:fragment="content" がここに挿入される
Footer（layout.html が提供）

10-3-2 Controllerでレイアウトを使う設定を追加する

レイアウトを使うには、Controllerからlayout.htmlにviewNameを渡す必要があります。各メソッドのreturn文を変更しましょう。

TodoController.java - 変更箇所 
    // 変更前:
    // return "list";

    // 変更後:
    model.addAttribute("viewName", "list");
    model.addAttribute("pageTitle", "Todo一覧");
    return "layout";

同様に、createメソッドとeditメソッドも変更してください。

TodoController.java - 新規登録画面 
    // 新規登録画面
    model.addAttribute("viewName", "create");
    model.addAttribute("pageTitle", "新規登録");
    return "layout";

    // 編集画面
    model.addAttribute("viewName", "edit");
    model.addAttribute("pageTitle", "Todo編集");
    return "layout";
info 仕組みの解説

Controllerがreturn "layout"とすると、Thymeleafはまずlayout.htmlを読み込みます。その中のth:replace="~{${viewName} :: content}"がviewNameに設定されたテンプレート（例: list.html）のth:fragment="content"部分を読み込んで差し替えます。

10-3-3 各テンプレートをフラグメント化する

各テンプレートのコンテンツ部分をth:fragment="content"で囲みます。list.htmlを開いて、以下のように書き換えてください。

list.html - フラグメント化 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org" lang="ja">
<body>
<div th:fragment="content">
    <h2 class="mb-4">Todo一覧</h2>

    <!-- ここに既存のフィルタボタン、テーブル等を配置 -->

</div>
</body>
</html>

<head>やBootstrapのCDNリンクはlayout.htmlが管理するので、各テンプレートからは削除してかまいません。create.html、edit.htmlも同じ要領でth:fragment="content"で囲んでください。

src/main/resources/templates/
├── layout.html ← NEW! 共通レイアウト
├── list.html ← th:fragment="content" で囲む
├── create.html ← th:fragment="content" で囲む
└── edit.html ← th:fragment="content" で囲む

一覧画面をデザインする

一覧画面（list.html）にBootstrapのクラスを適用して、見た目を整えていきましょう。

10-4-1 フィルタボタンにBootstrapクラスを適用する

前章で作ったフィルタボタンにBootstrapのボタングループスタイルを適用します。list.htmlのth:fragment="content"の中を次のように書き換えてください。

list.html - フィルタ部分 
<div class="d-flex justify-content-between align-items-center mb-4">
    <h2>Todo一覧</h2>
    <a th:href="@{/todos/create}" class="btn btn-primary">
        + 新規登録
    </a>
</div>

<!-- フィルタボタン -->
<div class="btn-group mb-3" role="group">
    <a th:href="@{/todos}"
       th:classappend="${currentStatus == null} ? 'btn-dark' : 'btn-outline-dark'"
       class="btn">すべて</a>
    <span th:each="s : ${statuses}">
        <a th:href="@{/todos(status=${s})}"
           th:text="${s.displayName}"
           th:classappend="${currentStatus != null and currentStatus.name() == s.name()}
                           ? 'btn-dark' : 'btn-outline-dark'"
           class="btn">
        </a>
    </span>
</div>

10-4-2 テーブルにBootstrapクラスを適用する

テーブルタグにtable table-hoverクラスを追加し、ステータスにはバッジを使いましょう。

list.html - テーブル部分 
<table class="table table-hover">
    <thead class="table-light">
        <tr>
            <th>タイトル</th>
            <th>期限日</th>
            <th>ステータス</th>
            <th>操作</th>
        </tr>
    </thead>
    <tbody>
        <tr th:each="todo : ${todos}">
            <td>
                <strong th:text="${todo.title}"></strong>
                <br>
                <small class="text-muted" th:text="${todo.description}"></small>
            </td>
            <td th:text="${#temporals.format(todo.dueDate, 'yyyy/MM/dd')}"></td>
            <td>
                <form th:action="@{'/todos/' + ${todo.id} + '/toggle-status'}" method="post"
                      style="display:inline;">
                    <button type="submit" class="badge border-0"
                            th:classappend="${todo.status.name() == 'COMPLETED'} ? 'bg-success' :
                                           (${todo.status.name() == 'IN_PROGRESS'} ? 'bg-primary' : 'bg-secondary')"
                            th:text="${todo.status.displayName}">
                    </button>
                </form>
            </td>
            <td>
                <a th:href="@{'/todos/' + ${todo.id} + '/edit'}"
                   class="btn btn-outline-primary btn-sm">編集</a>
                <form th:action="@{'/todos/' + ${todo.id} + '/delete'}" method="post"
                      style="display:inline;">
                    <button type="submit" class="btn btn-outline-danger btn-sm"
                            onclick="return confirm('本当に削除しますか？')">削除</button>
                </form>
            </td>
        </tr>
    </tbody>
</table>

Section 10-5

フォーム画面をデザインする

新規登録画面（create.html）と編集画面（edit.html）にもBootstrapのフォームスタイルを適用しましょう。

10-5-1 新規登録画面にBootstrapを適用する

create.htmlを開いて、th:fragment="content"の中を以下のように書き換えてください。

create.html - Bootstrapフォーム 
<div th:fragment="content">
    <h2 class="mb-4">新規登録</h2>

    <div class="row justify-content-center">
        <div class="col-md-8">
            <div class="card">
                <div class="card-body">
                    <form th:action="@{/todos}" th:object="${todo}" method="post">
                        <div class="mb-3">
                            <label for="title" class="form-label">タイトル</label>
                            <input type="text" id="title" th:field="*{title}"
                                   class="form-control" placeholder="例: Spring Bootの勉強">
                            <div class="text-danger" th:if="${#fields.hasErrors('title')}"
                                 th:errors="*{title}"></div>
                        </div>
                        <div class="mb-3">
                            <label for="description" class="form-label">説明</label>
                            <textarea id="description" th:field="*{description}"
                                      class="form-control" rows="3"></textarea>
                        </div>
                        <div class="mb-3">
                            <label for="dueDate" class="form-label">期限日</label>
                            <input type="date" id="dueDate" th:field="*{dueDate}"
                                   class="form-control">
                            <div class="text-danger" th:if="${#fields.hasErrors('dueDate')}"
                                 th:errors="*{dueDate}"></div>
                        </div>
                        <div class="d-flex gap-2">
                            <button type="submit" class="btn btn-primary">登録する</button>
                            <a th:href="@{/todos}" class="btn btn-outline-secondary">キャンセル</a>
                        </div>
                    </form>
                </div>
            </div>
        </div>
    </div>
</div>
info 適用したクラスの解説
  • card / card-body -- フォームを白いカードで囲み、視覚的に区切ります
  • form-control -- テキスト入力にBootstrapのスタイルを適用します
  • form-label -- ラベルのフォントサイズや余白を統一します
  • mb-3 -- margin-bottom を1rem分確保し、フォーム要素間に余白を作ります
  • text-danger -- バリデーションエラーメッセージを赤字にします
  • row justify-content-center + col-md-8 -- フォームを中央配置で適度な幅に制限します

edit.htmlも同様の構造で書き換えてください。フォームのth:actionと送信ボタンのテキストが異なるだけです。

10-5-2 バリデーションエラーのスタイルを強化する

Bootstrapでは、入力フィールドにエラーがある場合にis-invalidクラスを付けると、赤い枠線が表示されます。Thymeleafのth:classappendを使って動的に付与しましょう。

バリデーションエラーのスタイル強化 
<input type="text" id="title" th:field="*{title}"
       class="form-control"
       th:classappend="${#fields.hasErrors('title')} ? 'is-invalid'"
       placeholder="例: Spring Bootの勉強">
<div class="invalid-feedback" th:if="${#fields.hasErrors('title')}"
     th:errors="*{title}"></div>

text-dangerの代わりにinvalid-feedbackクラスを使うと、is-invalidが付いた入力フィールドの直下にエラーメッセージが表示されます。実行して、タイトルを空にしてフォーム送信し、赤い枠線とエラーメッセージが表示されることを確認してください。

Exercise

10章 練習問題

Q1

Thymeleafでlayout.htmlのth:replace="~{${viewName} :: content}"を使うとき、各ページのテンプレートで必要な記述はどれですか。

  1. th:include="content" をルート要素に付ける
  2. th:fragment="content" をコンテンツを囲むdivに付ける
  3. th:block="content" をbodyタグに付ける
  4. 特に何も書かなくてもbody全体が自動的に挿入される
Q2

Bootstrapでフォームの入力フィールドにバリデーションエラーの赤枠を付けるために使用するクラスはどれですか。

  1. form-error
  2. has-error
  3. is-invalid
  4. text-danger
Q3

Bootstrapのグリッドシステムで、PC画面（md以上）で横幅の8/12を占め、中央配置するクラスの組み合わせとして正しいものはどれですか。

  1. class="container-8 center"
  2. class="row"の中にclass="col-md-8 mx-auto"
  3. class="grid-8 text-center"
  4. class="col-8 align-center"
Answer

10章 解説・解答

Q1の解答: B

th:replaceは指定されたフラグメントで要素を置き換えます。各ページのテンプレートではth:fragment="content"を付けた要素を用意し、layout.htmlのth:replace="~{${viewName} :: content}"がそのフラグメントを読み込んで差し替えます。これにより、ヘッダー・フッター・CDNリンクなどを共通化でき、変更時も1箇所の修正で済みます。

Q2の解答: C

Bootstrap 5ではis-invalidクラスを入力フィールドに付けると赤い枠線が表示されます。合わせてinvalid-feedbackクラスを持つ要素を直後に配置すると、エラーメッセージが赤字で表示されます。text-dangerは汎用的な赤字テキストクラスであり、フォームフィールドの枠線色は変わりません。

Q3の解答: B

Bootstrapのグリッドシステムは12カラム制で、rowの中にcol-md-8を配置すると横幅の8/12を占めます。mx-autoは左右のmarginをautoにするユーティリティクラスで、中央配置になります。またはrow justify-content-centerを親要素に付ける方法もあります。
