Section 5-1

登録フォームのHTMLを作成する

前章までで、データベースからTodo一覧を取得して表示できるようになりました。この章では、ブラウザのフォームからデータを入力してデータベースに保存する機能を実装します。

5-1-1 この章で作る機能の全体像

まず、これから実装する「新規登録」機能の流れを確認しましょう。

sync_alt 新規登録の処理フロー

GET /todos/new
フォーム表示
→
POST /todos
データ受信
→
repository.save()
DB保存
→
redirect:/todos
一覧へ戻る

この流れを1つずつ実装していきましょう。

5-1-2 フォーム表示用のControllerメソッドを追加する

まず、フォーム画面を表示するためのメソッドをControllerに追加します。TodoController.java を開いて、以下のメソッドを追加してください。

TodoController.java（メソッドを追加） 
@GetMapping("/todos/new")
public String showCreateForm(Model model) {
    model.addAttribute("todo", new Todo());
    return "todos/create";
}
lightbulb なぜ空のTodoを渡すのか

new Todo() で空のTodoオブジェクトを作りModelに入れています。これはThymeleafの th:object でフォームとオブジェクトを紐づけるために必要です。フォーム入力値が自動的にこのオブジェクトのフィールドにセットされます。

5-1-3 登録フォームのHTMLを作成する

src/main/resources/templates/todos/ フォルダに create.html を新規作成してください。

以下のHTMLを手で入力してください。

src/main/resources/templates/todos/create.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Todo新規作成</title>
</head>
<body>
    <h1>Todoを新規作成</h1>

    <form th:action="@{/todos}" th:object="${todo}" method="post">
        <div>
            <label for="title">タイトル</label>
            <input type="text" id="title" th:field="*{title}">
        </div>
        <div>
            <label for="description">説明</label>
            <textarea id="description" th:field="*{description}"
                      rows="3"></textarea>
        </div>
        <div>
            <label for="dueDate">期限日</label>
            <input type="date" id="dueDate" th:field="*{dueDate}">
        </div>
        <button type="submit">登録</button>
        <a th:href="@{/todos}">キャンセル</a>
    </form>
</body>
</html>

description Thymeleafのフォーム属性

th:action="@{/todos}"
フォーム送信先のURLを指定。@{...} はThymeleafのURL式。
th:object="${todo}"
フォーム全体をTodoオブジェクトに紐づける。Controllerで渡したオブジェクトを参照。
th:field="*{title}"
*{...} は th:object のフィールドを参照。name属性・value属性が自動設定される。
method="post"
フォーム送信時にHTTP POSTリクエストを送る。データの作成・更新にはPOSTを使う。
Section 5-2

@PostMappingでフォームデータを受信する

5-2-1 GETとPOSTの違いを理解する

Webアプリケーションでは、HTTPメソッドによってリクエストの意味が異なります。

compare_arrows GETリクエストとPOSTリクエスト

GET リクエスト
  • データの取得に使う
  • URLにパラメータが見える
  • ブラウザの「戻る」で再実行される
  • ブックマーク可能
例: 一覧表示、詳細表示、フォーム画面表示
POST リクエスト
  • データの送信・作成に使う
  • データはリクエストボディに入る
  • ブラウザの「戻る」で再送信されない
  • ブックマーク不可
例: 新規登録、更新、削除

5-2-2 POSTリクエストを受け取るメソッドを作成する

TodoController.java に以下のメソッドを追加してください。@PostMapping アノテーションを使い、フォームから送信されたデータを受け取ります。

TodoController.java（メソッドを追加） 
@PostMapping("/todos")
public String createTodo(@ModelAttribute Todo todo) {
    todo.setCreatedAt(LocalDateTime.now());
    todoRepository.save(todo);
    return "redirect:/todos";
}

import文も追加してください。

TodoController.java（import追加） 
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.PostMapping;
import java.time.LocalDateTime;
lightbulb @ModelAttribute の仕組み

@ModelAttribute は、フォームから送信されたデータを自動的にJavaオブジェクトに変換（バインド）してくれます。フォームの name 属性（th:fieldで自動設定）とオブジェクトのフィールド名が一致していれば、値が自動的にセットされます。

link フォームデータのバインド

// フォームの送信データ
title=買い物に行く
description=牛乳とパン
dueDate=2026-03-10
→
// Todoオブジェクト
todo.title = "買い物に行く"
todo.description = "牛乳とパン"
todo.dueDate = 2026-03-10
Spring が name 属性とフィールド名を照合して自動変換する
Section 5-3

repository.save()でデータを保存する

5-3-1 save()メソッドの動作を理解する

JpaRepository の save() メソッドは、渡されたエンティティをデータベースに保存します。重要なのは、新規登録と更新の両方に使えるということです。

save save() の動作

id が null の場合
INSERT（新規登録）が実行される
INSERT INTO todo (title, ...) VALUES (?, ...)
id が存在する場合
UPDATE（更新）が実行される
UPDATE todo SET title=?, ... WHERE id=?

フォームから新規作成する場合、id はセットされていないため null です。そのため save() はINSERTを実行します。

5-3-2 Controllerの全体像を確認する

ここまでの変更を加えた TodoController.java の全体像を確認しましょう。

src/main/java/com/example/todo/controller/TodoController.java 
package com.example.todo.controller;

import com.example.todo.entity.Todo;
import com.example.todo.repository.TodoRepository;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.PostMapping;
import java.time.LocalDateTime;

@Controller
public class TodoController {

    private final TodoRepository todoRepository;

    public TodoController(TodoRepository todoRepository) {
        this.todoRepository = todoRepository;
    }

    // 一覧表示（第4章で作成済み）
    @GetMapping("/todos")
    public String list(Model model) {
        model.addAttribute("todos", todoRepository.findAll());
        return "todos/list";
    }

    // 新規作成フォーム表示
    @GetMapping("/todos/new")
    public String showCreateForm(Model model) {
        model.addAttribute("todo", new Todo());
        return "todos/create";
    }

    // 新規作成処理
    @PostMapping("/todos")
    public String createTodo(@ModelAttribute Todo todo) {
        todo.setCreatedAt(LocalDateTime.now());
        todoRepository.save(todo);
        return "redirect:/todos";
    }
}
info URLの使い分け

GET /todos と POST /todos は同じURLですが、HTTPメソッドが異なるため別のメソッドで処理されます。@GetMapping はGETリクエスト、@PostMapping はPOSTリクエストにだけ反応します。

PRGパターンとリダイレクト

5-4-1 PRGパターンとは

先ほどのコードで return "redirect:/todos" と書きました。これは PRG（Post-Redirect-Get）パターン と呼ばれる重要なWebアプリの設計パターンです。

refresh PRG（Post-Redirect-Get）パターン

PRGパターンを使わない場合の問題
POST /todos
データ登録
→
HTMLを直接返す
→
F5で再読込
二重登録される！
PRGパターンを使う場合
POST /todos
データ登録
→
302 Redirect
リダイレクト
→
GET /todos
一覧表示
→
F5で再読込
GETなので安全！
tips_and_updates 覚えておこう

Spring Bootでリダイレクトする場合は、戻り値の文字列を "redirect:/パス" にします。テンプレート名ではなくURLとして解釈され、ブラウザに302レスポンスが返ります。

Exercise

5章 練習問題

Q1

Thymeleafのフォームで th:object="${todo}" と書いた場合、各入力フィールドでオブジェクトのプロパティを参照する正しい構文はどれですか。

  1. th:field="${todo.title}"
  2. th:field="*{title}"
  3. th:value="${title}"
  4. th:field="#{title}"
Q2

PRGパターンを使わずにPOST処理後にHTMLを直接返すと、どのような問題が起きる可能性がありますか。

  1. ページの表示速度が遅くなる
  2. ブラウザのF5（再読込）でデータが二重登録される
  3. データベースに接続できなくなる
  4. CSSが適用されなくなる
Q3

JpaRepository の save() メソッドについて正しい説明はどれですか。

  1. 常にINSERT（新規登録）を実行する
  2. 常にUPDATE（更新）を実行する
  3. idがnullならINSERT、idが存在すればUPDATEを実行する
  4. DELETE（削除）も実行できる
Answer

5章 解説・解答

Q1の解答: B

th:object で指定したオブジェクトのプロパティは *{...}（アスタリスク式）で参照します。*{title} は ${todo.title} と同じ意味ですが、th:object と組み合わせることでコードが簡潔になります。#{...} はメッセージ式（国際化）、@{...} はURL式です。

Q2の解答: B

POST処理後にHTMLを直接返すと、ブラウザのURLがPOST先のままになります。この状態でF5（再読込）を押すと、同じPOSTリクエストが再送信され、同じデータが重複して登録されます。PRGパターンではPOST後にリダイレクト（302）するため、ブラウザのURLがGET先に変わり、F5を押してもGETリクエストが実行されるだけで安全です。

Q3の解答: C

save() メソッドは、渡されたエンティティのIDを見て自動的に動作を切り替えます。IDが null または未設定の場合はINSERT（新規登録）、既に存在するIDが設定されている場合はUPDATE（更新）を実行します。削除は deleteById() など別のメソッドを使います。
