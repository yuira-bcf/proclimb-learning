16-1

アノテーションとは？

アノテーション（Annotation）とは、Javaのクラスやメソッドに「目印」を付ける仕組みです。@ から始まる記述で、Springはこの目印を読み取ってクラスの役割を自動で判断します。

label アノテーション = クラスに貼る「ラベル」

public class
TodoService
ただのJavaクラス
Springは認識しない
→
@Service
public class
TodoService
Springが管理するBean
Springが自動で認識する

ポイント: アノテーションを付けることで、Springが「このクラスはサービス層の処理を担当するクラスだ」と認識し、自動でインスタンスを生成・管理してくれます。これにより new TodoService() を書く必要がなくなります。

アノテーションの基本構文

アノテーションはクラスやメソッドの直前の行に記述します。

アノテーションの記述位置 
@Controller                     // ← クラスに付けるアノテーション
public class TodoController {

    @GetMapping("/todo")        // ← メソッドに付けるアノテーション
    public String list(Model model) {
        return "todo/list";
    }
}
info アノテーションの種類
  • クラスに付けるもの — @Controller, @Service, @Repository, @Component
  • メソッドに付けるもの — @GetMapping, @PostMapping, @Transactional
  • 引数に付けるもの — @PathVariable, @RequestParam
  • フィールドに付けるもの — @Autowired, @NotBlank

16-3

コンポーネントスキャンの仕組み

コンポーネントスキャンとは、Springがアプリケーション起動時にパッケージ内のクラスを自動で探索し、アノテーションが付いたクラスをBeanとして登録する仕組みです。

search コンポーネントスキャンの動作 ── 起動時に何が起きるか

1
アプリケーション起動
@SpringBootApplication が付いたメインクラスを起動すると、Springはそのクラスの同じパッケージ以下を自動スキャンします。
2
クラスを探索
パッケージ内の全クラスをチェックし、アノテーションが付いたクラスを発見します。
発見される
@Controller TodoController
@Service TodoService
@Repository TodoRepository
無視される
Todo (Entityクラス)
TodoForm (フォームクラス)
Utils (アノテーションなし)
3
Beanとして登録
発見されたクラスのインスタンスを自動で生成し、Spring の DIコンテナ（Bean の保管庫）に登録します。以降はSpringが管理し、必要な場所に自動で注入してくれます。

重要: @SpringBootApplication は内部に @ComponentScan を含んでいます。そのため、メインクラスと同じパッケージまたはそのサブパッケージに置いたクラスが自動でスキャン対象になります。別パッケージに置くと認識されないので注意してください。

folder スキャン対象のパッケージ構成

com.example.webapp/
WebappApplication.java ← @SpringBootApplication（起点）
controller/
TodoController.java @Controller
service/
TodoService.java @Service
repository/
TodoRepository.java @Repository
entity/
Todo.java （アノテーションなし＝Beanではない）

まとめ: Springのアノテーションとコンポーネントスキャンにより、クラスに目印を付けるだけでSpringが自動でインスタンス生成・管理・注入を行ってくれます。開発者は new を書く必要がなく、各層の役割に集中してコードを書くことができます。
