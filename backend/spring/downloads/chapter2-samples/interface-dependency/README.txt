=============================================
  インターフェース依存のサンプルコード
=============================================

【概要】
インターフェース依存のメリットを示すサンプルです。
インターフェースを型として使用することで、
実装クラスの変更が1箇所で済みます。

【ファイル説明】
- I.java : インターフェース（methodXを宣言）
- B.java : インターフェースIを実装したクラス
- C.java : インターフェースIを実装した別のクラス
- A.java : 使う側のクラス（インターフェースを型として使用）

【実行方法】
1. すべてのファイルをコンパイル
   javac *.java

2. 実行
   java A

【ポイント】
A.java で「I i = new B();」を「I i = new C();」に変更するだけで
実装を切り替えられます。

クラス依存と比較すると：
- クラス依存：2箇所の変更が必要（型名 + 変数名）
- インターフェース依存：1箇所の変更で済む（newの部分のみ）

これにより「依存性が低い」コードになり、
保守性・拡張性が向上します。
