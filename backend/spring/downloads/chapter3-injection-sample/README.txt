================================================================================
  Beanの参照を渡す方法 サンプルプロジェクト
================================================================================

【概要】
SpringでBeanの参照を渡す5つの方法を比較できるサンプルプログラムです。

【5つの方法】
1. フィールドに渡す（フィールドインジェクション）
2. セッターメソッド経由で渡す（セッターインジェクション）
3. コンストラクタ経由で渡す（コンストラクタインジェクション）
4. コンストラクタ経由で渡す（@Autowired省略版）
5. コンストラクタ経由で渡す（Lombok使用版）

================================================================================
【VSCodeでの実行方法】
================================================================================

1. VSCodeでこのフォルダを開く
   - ファイル → フォルダーを開く → このフォルダを選択

2. Extension Pack for Java がインストールされていることを確認

3. InjectionSampleApplication.java を開く

4. main メソッドの上に表示される「Run」をクリック

5. ターミナルに5つの方法の実行結果が表示されます

================================================================================
【ファイル構成】
================================================================================

src/main/java/com/example/demo/
├── InjectionSampleApplication.java   ... メインクラス
├── example/
│   ├── Example.java                  ... インターフェース
│   └── impl/
│       ├── FieldInjectionExample.java          ... 方法1
│       ├── SetterInjectionExample.java         ... 方法2
│       ├── ConstructorInjectionExample.java    ... 方法3
│       ├── ConstructorInjectionOmitExample.java    ... 方法4
│       └── ConstructorInjectionLombokExample.java  ... 方法5
└── service/
    ├── SomeService.java              ... サービスインターフェース
    └── impl/
        └── SomeServiceImpl.java      ... サービス実装

================================================================================
【推奨される方法】
================================================================================

✅ 推奨：コンストラクタ経由（方法3, 4, 5）
   - finalフィールドで不変性を保証
   - テストしやすい
   - 依存関係が明確

❌ 非推奨：フィールドに渡す（方法1）
   - テストが困難
   - 依存関係が隠れる
