Section 7-1

基本データ型の型変換

異なる型のデータを一緒に計算する場合、型変換が必要になります。Javaでは自動的に行われる変換と、明示的に指定が必要な変換があります。型変換を正しく理解することで、データの精度損失やオーバーフローを防げます。

public class ArithmeticCastDemo {
    public static void main(String[] args) {
        // 整数同士の除算は整数結果
        int a = 5, b = 2;
        System.out.println(a / b);  // 2（小数部切り捨て）

        // 一方をdoubleにすると小数結果
        System.out.println((double) a / b);  // 2.5
        System.out.println(a / (double) b);  // 2.5

        // byte/short/charの演算結果はintになる
        byte x = 10, y = 20;
        // byte z = x + y;  // コンパイルエラー
        int z = x + y;      // OK
        byte w = (byte)(x + y);  // キャストが必要
    }
}
Section 7-2

基本データ型から参照型への型変換

コレクション（List、Mapなど）は参照型しか格納できません。そのため、intやdoubleなどの基本データ型をオブジェクトとして扱うためのラッパークラスが用意されています。オートボクシング機能により、変換は自動的に行われますが、nullの扱いには注意が必要です。

public class BoxingDemo {
    public static void main(String[] args) {
        // オートボクシング（基本型 → ラッパー）
        Integer num = 100;  // int → Integer

        // オートアンボクシング（ラッパー → 基本型）
        int n = num;  // Integer → int

        // 明示的な変換

        int i2 = i1.intValue();            // アンボクシング
    }
}
warning nullに注意

ラッパークラスはnullを持てます。nullをアンボクシングするとNullPointerExceptionが発生します。

Exercise

7章 練習問題

Q1

次のコードでコンパイルエラーになるものはどれですか？

  1. int a = 10; long b = a;
  2. double d = 3.14; int n = d;
  3. byte b = 10; int i = b;
  4. float f = 1.0f; double d = f;
Q2

次のコードの実行結果として正しいものはどれですか？

double d = 9.7;
int n = (int) d;
System.out.println(n);
  1. 10
  2. 9
  3. 9.7
  4. コンパイルエラー
Q3

次のコードの実行結果として正しいものはどれですか？

Integer num = null;
int n = num;
System.out.println(n);
  1. 0
  2. null
  3. NullPointerException
  4. コンパイルエラー
Answer

7章 解説・解答

Q1の解答: B

doubleからintへの変換は縮小変換（データ損失の可能性あり）のため、明示的なキャストが必要です。int n = (int) d;と書く必要があります。A、C、Dはすべて拡大変換なので自動的に変換されます。

Q2の解答: B

doubleからintへのキャストでは小数部が切り捨てられます（四捨五入ではありません）。9.7の小数部が切り捨てられ、結果は9になります。

Q3の解答: C

Integerがnullの状態でintにアンボクシングしようとすると、NullPointerExceptionが発生します。ラッパークラスはnullを持てますが、基本データ型にはnullが存在しないため、変換時にエラーになります。
