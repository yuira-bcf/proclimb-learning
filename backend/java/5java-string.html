Section 5-1

Stringクラスの基本

文字列操作はほぼすべてのプログラムで必要になります。ユーザー入力の処理、ファイル名の生成、ログ出力、API通信など、文字列を扱わないプログラムはほとんどありません。Stringクラスのメソッドを使いこなすことで、効率的なコードが書けるようになります。

5-1-1 Stringの生成方法

Javaで文字列を生成するには主に2つの方法があります。

add_circle 文字列の2つの生成方法

recommend リテラル（推奨）
String s1 = "Hello";
String s2 = "Hello";
文字列プールに格納され、同じ内容なら同じオブジェクトを共有する。メモリ効率が良い。
warning new演算子
String s3 = new String("Hello");
String s4 = new String("Hello");
毎回ヒープに新しいオブジェクトを生成。プールは使わない。通常は使わない。

参照の比較: s1 == s2 は true（同じプール内オブジェクト）。s1 == s3 は false（newで別オブジェクトが生成される）。文字列の内容比較には必ず equals() を使いましょう。

5-1-2 イミュータブル性

Stringクラスの最も重要な特徴はイミュータブル（不変）であることです。一度生成された文字列オブジェクトの内容を後から変更することはできません。

lock イミュータブル（不変）とは

String s = "Hello";
s.toUpperCase(); // 元は変わらない（戻り値を使っていない）
s = s.toUpperCase(); // 新しいオブジェクトを変数に代入
"Hello"
← 元のオブジェクト（変更されない）
"HELLO"
← 新しく生成されたオブジェクト

試験頻出: Stringメソッドの戻り値を変数に代入しないと、元の文字列は変わりません。s.toUpperCase(); だけでは s は "Hello" のままです。

public class StringImmutableDemo {

        String s = "Hello";
        s.toUpperCase();  // 元の文字列は変更されない

}

5-1-3 文字列プール

Javaは文字列リテラルをメモリ効率よく管理するために、文字列プール（String Pool）という仕組みを持っています。同じ内容の文字列リテラルは1つのオブジェクトとして共有されます。

hub 文字列プール（String Pool）の仕組み

String Pool（ヒープ内の特殊領域）
"Hello"
"World"
String a = "Hello";
String b = "Hello";
→ a == b は true（同じオブジェクト）
ヒープ（プール外）
"Hello"
"Hello"
String c = new String("Hello");
String d = new String("Hello");
→ c == d は false（別オブジェクト）

注意: new String("Hello") はプールを使わず新しいオブジェクトを生成するため、== で比較すると false になります。文字列の比較には必ず equals() を使いましょう。

lightbulb intern()メソッド

new String("Hello").intern() を使うと、プール内に同じ内容の文字列があればそのオブジェクトを返します。ただし実務で使う場面は少なく、試験でも出題頻度は低いです。

length() - 文字列の長さを取得

文字列.length()

文字列に含まれる文字数をintで返します。空文字列""の場合は0を返します。

charAt() - 指定位置の文字を取得

文字列.charAt(インデックス)

0から始まるインデックスで指定した位置の文字をcharで返します。範囲外のインデックスを指定するとStringIndexOutOfBoundsExceptionが発生します。

文字列.substring(開始インデックス) // または 文字列.substring(開始, 終了)

equals() - 文字列の等価比較

文字列1.equals(文字列2)

文字列の内容が等しいかをbooleanで返します。==は参照の比較なので、文字列の比較にはequals()を使用します。

public class StringBuilderDemo {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder();

        // 文字列の追加
        sb.append("Hello");
        sb.append(" ");
        sb.append("World");
        System.out.println("append後: " + sb);

        // メソッドチェーン
        sb.append("!").append(" ").append("Java");
        System.out.println("チェーン後: " + sb);

        // 挿入
        sb.insert(0, "[");  // 先頭に挿入
        System.out.println("insert後: " + sb);

        // 削除
        sb.delete(0, 1);    // 範囲削除
        System.out.println("delete後: " + sb);

        // 置換
        sb.replace(0, 5, "Hi");
        System.out.println("replace後: " + sb);

        // 反転
        sb.reverse();
        System.out.println("reverse後: " + sb);

        // Stringに変換
        String result = sb.toString();
        System.out.println("結果: " + result);
    }
}

StringBuilderの生成

StringBuilder sb = new StringBuilder();

変更可能な文字列を扱うクラスです。頻繁な文字列操作ではStringより効率的です。

Section 5-4

文字列のフォーマット

数値やオブジェクトを含む文字列を組み立てるとき、+ 演算子による連結だけでなく、書式指定子（フォーマット指定子）を使った方法があります。可読性が高く、桁数の制御も容易です。

5-4-1 String.format() と printf()

String.format() は書式指定子に値を埋め込んだ新しい文字列を返します。System.out.printf() は同じ書式で直接コンソールに出力します。

table_chart 主要なフォーマット指定子

指定子 対応する型 例 出力
%s 文字列（String） format("%s", "Java") Java
%d 整数（int, long） format("%d", 42) 42
%f 浮動小数点（float, double） format("%f", 3.14) 3.140000
%.2f 小数点以下2桁 format("%.2f", 3.14159) 3.14
%b 真偽値（boolean） format("%b", true) true
%n 改行（OS依存） format("Hello%nWorld") Hello
World
chapter02/StringFormatDemo.java Java 
public class StringFormatDemo {
    public static void main(String[] args) {
        // String.format() - 書式付き文字列を生成
        String name = "Alice";
        int age = 25;
        double score = 89.5;

        String msg = String.format("名前: %s, 年齢: %d歳", name, age);
        System.out.println(msg);  // 名前: Alice, 年齢: 25歳

        // printf() - 直接コンソールに出力（改行なし）
        System.out.printf("スコア: %.1f点%n", score);  // スコア: 89.5点

        // 桁数の指定
        System.out.printf("商品コード: %05d%n", 42);    // 商品コード: 00042
        System.out.printf("金額: %,d円%n", 1234567);    // 金額: 1,234,567円
    }
}
menu_book 構文説明

String.format() - 書式付き文字列の生成

String result = String.format("書式文字列", 引数1, 引数2, ...);

書式指定子に対応する引数を埋め込んだ新しい文字列を返します。printf()と同じ書式を使えますが、出力ではなく文字列を返す点が異なります。

System.out.printf() - 書式付き出力

System.out.printf("書式文字列", 引数1, 引数2, ...);

書式指定子に対応する引数を埋め込んでコンソールに出力します。println()と違い自動改行されないため、必要に応じて%nを使います。

5-4-2 formatted() メソッド

Java 15以降では、文字列のインスタンスメソッドとして formatted() が使えます。String.format() と同じ結果を、より簡潔に記述できます。

compare String.format() vs formatted() の比較

従来: String.format()
String msg = String.format(
  "名前: %s, 年齢: %d"
  , name, age);
recommend Java 15+: formatted()
String msg =
  "名前: %s, 年齢: %d"
  .formatted(name, age);

ポイント: formatted() はインスタンスメソッドなので、テキストブロックとの組み合わせで特に読みやすくなります。動作は String.format() と同じです。

chapter02/FormattedDemo.java Java 
public class FormattedDemo {
    public static void main(String[] args) {
        String name = "Bob";
        int age = 30;
        double salary = 350000.0;

        // formatted() を使ったフォーマット
        String profile = "名前: %s, 年齢: %d歳".formatted(name, age);
        System.out.println(profile);

        // テキストブロックとの組み合わせ
        String card = """
            ========================
            社員カード
            ------------------------
            名前: %s
            年齢: %d歳
            給与: %,.0f円
            ========================
            """.formatted(name, age, salary);
        System.out.println(card);
    }
}

5章 練習問題

Q1

次のコードの実行結果として正しいものはどれですか？

String s = "Hello World";
System.out.println(s.substring(6).toUpperCase());
  1. HELLO
  2. WORLD
  3. Hello World
  4. world
Q2

以下のコードの出力は何ですか？

String s = "Hello";
s.toUpperCase();
System.out.println(s);
  1. HELLO
  2. Hello
  3. hello
  4. コンパイルエラー
Q3

次のコードの実行結果として正しいものはどれですか？

StringBuilder sb = new StringBuilder("Hello");
sb.append(" World").reverse();
System.out.println(sb);
  1. Hello World
  2. dlroW olleH
  3. olleH dlroW
  4. コンパイルエラー
Answer

5章 解説・解答

Q1の解答: B

substring(6)はインデックス6から末尾までの部分文字列"World"を返します。さらにtoUpperCase()で大文字に変換されるため、出力は"WORLD"になります。Stringはイミュータブルなので、各メソッドは新しい文字列を返します。

Q2の解答: B

Stringはイミュータブル（不変）なので、toUpperCase()は新しい文字列を返しますが、元の変数sは変更されません。結果を反映するにはs = s.toUpperCase();とする必要があります。

Q3の解答: B

StringBuilderはミュータブル（可変）なので、append(" World")で内容が"Hello World"になり、続けてreverse()で文字列が反転されます。メソッドチェーンが可能なのは、各メソッドがStringBuilder自身を返すためです。結果は"dlroW olleH"になります。
