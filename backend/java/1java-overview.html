Section 1-1

Javaの概要

Javaは1995年にサン・マイクロシステムズ（現Oracle）によって開発されたプログラミング言語です。「Write Once, Run Anywhere（一度書けばどこでも動く）」をスローガンに、プラットフォームに依存しない実行環境を提供しています。

1.1 Java言語の特徴

Javaには以下のような特徴があります：

💻

オブジェクト指向

クラスとオブジェクトを基本単位としてプログラムを構成します。継承、カプセル化、ポリモーフィズムといった概念を活用できます。

🔒

型安全性

静的型付け言語であり、コンパイル時に型チェックが行われます。これにより実行時エラーを未然に防ぐことができます。

📦

ガベージコレクション

メモリ管理を自動的に行うため、プログラマがメモリの解放を意識する必要がありません。

🌐

プラットフォーム非依存

Java仮想マシン（JVM）上で動作するため、OSに依存せずプログラムを実行できます。

1.2 実行環境の特徴

Javaプログラムは、以下の流れで実行されます：

1

ソースコードの作成

.java拡張子のファイルにJavaのソースコードを記述します。

2

コンパイル

javacコマンドでソースコードをバイトコード（.classファイル）に変換します。

3

実行

javaコマンドでJVMがバイトコードを解釈・実行します。

lightbulb バイトコードとは

バイトコードは、JVMが理解できる中間コードです。ネイティブコード（機械語）とは異なり、どのプラットフォームのJVMでも実行可能な形式になっています。

Section 1-2

Java環境とJDK

Javaプログラムを開発・実行するためには、適切な環境を整える必要があります。このセクションでは、開発に必要なツールと、それぞれの役割を理解します。環境構築はプログラミング学習の第一歩であり、ここをしっかり理解することで、後のトラブルシューティングにも役立ちます。

2.1 Java環境とは

Javaの開発・実行には、以下の環境が必要です：

環境 正式名称 用途
JDK Java Development Kit 開発に必要なツール一式（コンパイラ、デバッガなど）
JRE Java Runtime Environment 実行に必要な環境（JVMとライブラリ）
JVM Java Virtual Machine バイトコードを解釈・実行する仮想マシン
warning 注意

Java 11以降、JREは単独で配布されなくなりました。JDKにJREが含まれているため、開発者はJDKをインストールすれば開発と実行の両方が可能です。

2.2 環境セットアップ

Javaの開発を始めるには、JDKをインストールし、環境変数を設定する必要があります。以下の手順に沿ってセットアップしましょう。

layers JDK・JRE・JVMの関係 ── 入れ子構造

JDK（Java Development Kit）
開発ツール一式：javac（コンパイラ）、jdb（デバッガ）、jar など
JRE（Java Runtime Environment）
実行環境：標準ライブラリ（java.lang, java.util 等）
JVM（Java Virtual Machine）
バイトコードを解釈・実行する仮想マシン

ポイント: JDKをインストールすれば、JREとJVMも含まれるため開発と実行の両方が可能です。つまり、JDKだけインストールすればOKです。

build JDKインストール手順

1
JDKをダウンロード

以下のいずれかのサイトからJDK 21（LTS）をダウンロードします。

Oracle JDK
Oracle公式の商用版。学習・個人利用は無料
Adoptium (Eclipse Temurin) 推奨
オープンソースの無料JDK。商用利用もOK
2
インストーラーを実行

ダウンロードしたインストーラー（.msi / .pkg / .deb）を実行し、画面の指示に従ってインストールします。

3
環境変数を設定（Windows）

Windowsでは手動で環境変数を設定する必要があります。

① JAVA_HOME を設定
JAVA_HOME = C:\Program Files\Java\jdk-21
② PATH に追加
PATH = ...;%JAVA_HOME%\bin

info macOS / Linuxでは、Adoptiumインストーラーが自動で設定する場合が多いです。手動設定の場合は ~/.zshrc や ~/.bashrc に export JAVA_HOME=$(/usr/libexec/java_home) を追加します。

4
インストールを確認

コマンドプロンプト（またはターミナル）を新しく開いて、以下のコマンドを実行します。

バージョン確認コマンド Terminal

$ javac -version

route 環境変数の役割 ── なぜ設定が必要なのか

JAVA_HOME ── JDKの場所を教える
JAVA_HOME = C:\Program Files\Java\jdk-21
→
他のツール（Maven、IDE等）がJDKを見つけるために使う
PATH ── コマンドを使えるようにする
PATH += %JAVA_HOME%\bin
→
どこからでも java javac コマンドが実行可能になる

ポイント: PATHを設定しないと、javaコマンドを実行しても「'java' は認識されていません」というエラーになります。PATHはOSに「このフォルダの中にあるプログラムも探して」と伝える設定です。

warning よくあるトラブルと対処法
'java' は認識されていません
PATH環境変数にJDKのbinフォルダが追加されていない。設定後、コマンドプロンプトを再起動して確認。
javacは使えるがjava -versionで古いバージョンが表示される
古いJDK/JREが残っている可能性あり。where java（Windows）やwhich java（Mac/Linux）でパスを確認し、不要なものを削除。
JAVA_HOMEに\binを含めてしまった
JAVA_HOMEはjdk-21フォルダまでを指定。\binはPATHの方で追加します。
