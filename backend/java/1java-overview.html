Section 1-1

Javaの概要

Javaは1995年にサン・マイクロシステムズ（現Oracle）によって開発されたプログラミング言語です。「Write Once, Run Anywhere（一度書けばどこでも動く）」をスローガンに、プラットフォームに依存しない実行環境を提供しています。

1-1-1 Java言語の特徴

Javaには以下のような特徴があります：

💻

オブジェクト指向

クラスとオブジェクトを基本単位としてプログラムを構成します。継承、カプセル化、ポリモーフィズムといった概念を活用できます。

🔒

型安全性

静的型付け言語であり、コンパイル時に型チェックが行われます。これにより実行時エラーを未然に防ぐことができます。

📦

ガベージコレクション

メモリ管理を自動的に行うため、プログラマがメモリの解放を意識する必要がありません。

🌐

プラットフォーム非依存

Java仮想マシン（JVM）上で動作するため、OSに依存せずプログラムを実行できます。

1-1-2 Javaの歴史とバージョン

Javaは1995年の登場以来、継続的に進化を続けています。特にLTS（Long Term Support）バージョンは長期サポートが提供され、企業での採用が多いバージョンです。

ポイント: 学習・開発には最新のLTS版（Java 21）を使いましょう。LTS版は約2年ごとにリリースされ、長期間のセキュリティアップデートが提供されます。

1-1-3 他言語との比較

Javaの特徴をより明確に理解するために、他の主要プログラミング言語と比較してみましょう。

ポイント: Javaは「静的型付け + 自動メモリ管理 + 完全OOP」という組み合わせが特徴です。コンパイル時にエラーを検出でき、かつメモリ管理の複雑さからも解放されるため、大規模な業務システムの開発に適しています。

ポイント: C++のように直接機械語にコンパイルするのではなく、バイトコードという中間形式に変換するのがJavaの大きな特徴です。バイトコードはどのOSのJVMでも実行できるため、「Write Once, Run Anywhere」が実現されます。

lightbulb バイトコードとは

バイトコードは、JVMが理解できる中間コードです。ネイティブコード（機械語）とは異なり、どのプラットフォームのJVMでも実行可能な形式になっています。

Section 1-2

Java環境とJDK

Javaプログラムを開発・実行するためには、適切な環境を整える必要があります。このセクションでは、開発に必要なツールと、それぞれの役割を理解します。環境構築はプログラミング学習の第一歩であり、ここをしっかり理解することで、後のトラブルシューティングにも役立ちます。

1-2-1 Java環境とは

Javaの開発・実行には、以下の環境が必要です：

環境 正式名称 用途
JDK Java Development Kit 開発に必要なツール一式（コンパイラ、デバッガなど）
JRE Java Runtime Environment 実行に必要な環境（JVMとライブラリ）
JVM Java Virtual Machine バイトコードを解釈・実行する仮想マシン

layers JDK・JRE・JVMの関係 ── 入れ子構造

JDK（Java Development Kit）
開発ツール一式：javac（コンパイラ）、jdb（デバッガ）、jar など
JRE（Java Runtime Environment）
実行環境：標準ライブラリ（java.lang, java.util 等）
JVM（Java Virtual Machine）
バイトコードを解釈・実行する仮想マシン

ポイント: JDKをインストールすれば、JREとJVMも含まれるため開発と実行の両方が可能です。つまり、JDKだけインストールすればOKです。

warning 注意

Java 11以降、JREは単独で配布されなくなりました。JDKにJREが含まれているため、開発者はJDKをインストールすれば開発と実行の両方が可能です。

Section 1-3

Hello Worldプログラム

環境構築が完了したら、最初のJavaプログラムを書いてみましょう。プログラミング学習の伝統として、最初に作るプログラムは画面に「Hello World!」と表示する「Hello World」プログラムです。

1-3-1 はじめてのJavaプログラム

以下がJavaの最小限のプログラムです。テキストエディタ（メモ帳やVS Code）で入力し、Hello.javaというファイル名で保存しましょう。

Hello.java 
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World!");
    }
}

このプログラムをコンパイル・実行するには、コマンドプロンプト（ターミナル）で以下のコマンドを実行します。

コンパイルと実行 
$ javac Hello.java    ← コンパイル（Hello.class が生成される）
$ java Hello          ← 実行（.class は付けない）
Hello World!
lightbulb ファイル名のルール

Javaではファイル名とクラス名を一致させる必要があります。public class Hello と宣言した場合、ファイル名は必ず Hello.java にしなければなりません（大文字・小文字も一致させます）。

1-3-2 プログラムの解剖

たった4行のプログラムですが、Javaの重要な要素が詰まっています。1行ずつ見ていきましょう。

biotech Hello Worldプログラムの構造

public class Hello {
public: どこからでもアクセス可能 class: クラスの宣言 Hello: クラス名（=ファイル名）
public static void main(String[] args) {
public: アクセス修飾子 static: インスタンス不要 void: 戻り値なし main: エントリーポイント String[] args: コマンドライン引数
System.out.println("Hello World!");
System.out: 標準出力 println: 改行付きで出力 "Hello World!": 出力する文字列

ポイント: public static void main(String[] args) はJavaプログラムのエントリーポイント（実行開始地点）です。この形式は決まり文句として暗記しましょう。JVMはプログラム実行時にこのメソッドを最初に呼び出します。

要素 意味 補足
public アクセス修飾子 どこからでもアクセスできることを示す
class クラスの宣言 Javaのプログラムはすべてクラスの中に書く
static 静的メソッド オブジェクトを作らずに呼び出せる
void 戻り値の型 このメソッドは値を返さない
main メソッド名 JVMが最初に呼び出す特別なメソッド
String[] args 引数 コマンドラインから渡される文字列の配列
System.out.println 出力メソッド コンソールに文字列を表示し改行する
lightbulb println と print の違い

System.out.println() は出力後に改行します。改行なしで出力したい場合は System.out.print() を使います。

1-3-3 よくあるコンパイルエラー

はじめてプログラムを書くと、タイプミスなどでエラーが発生しがちです。よくあるエラーとその対処法を確認しておきましょう。

Section 1-4

プログラムの構造

Javaプログラムには一定の構造があります。このセクションでは、プログラムを構成する要素とその階層関係、そして名前の付け方のルール（命名規則）を学びます。

1-4-1 Javaプログラムの階層構造

Javaプログラムは、以下の階層構造で構成されます。上位の要素が下位の要素を包含する形になっています。

layers Javaプログラムの階層構造

① package（パッケージ） クラスをグループ化する名前空間
package com.example.app;
② import（インポート） 他のパッケージのクラスを利用
import java.util.Scanner;
③ class（クラス） プログラムの基本単位
public class Hello {
④ field（フィールド） データ（変数）
private String name;
⑤ method（メソッド） 処理（関数）
public void greet() {
⑥ statement（文） 個々の命令
System.out.println(name);

ポイント: Javaプログラムはpackage → import → class → field/method → statementという階層で構成されます。初学者は最初にこの「入れ子構造」を意識しましょう。

Greeting.java ── 構造の実例 
package com.example.app;         // ① パッケージ宣言

import java.time.LocalDate;      // ② インポート

public class Greeting {           // ③ クラス宣言

    private String name;          // ④ フィールド（データ）

    public void greet() {         // ⑤ メソッド（処理）
        LocalDate today = LocalDate.now();  // ⑥ 文
        System.out.println("Hello, " + name + "! Today is " + today);
    }
}

1-4-2 命名規則（ネーミングコンベンション）

Javaには、コードの可読性を高めるための命名規則があります。これはJavaの文法ではなく「慣習」ですが、ほぼすべてのJavaプロジェクトで従われている重要なルールです。

text_format Javaの命名規則

要素 規則 例 NG例
パッケージ すべて小文字 com.example.app Com.Example.App
クラス PascalCase（大文字始まり） StudentManager studentManager
メソッド camelCase（小文字始まり） calculateTotal() CalculateTotal()
変数 camelCase（小文字始まり） userName user_name
定数 UPPER_SNAKE_CASE MAX_RETRY_COUNT maxRetryCount

code 命名規則の適用例

// パッケージ名 → すべて小文字
package com.example.shop;
 
// クラス名 → PascalCase
public class ShoppingCart {
 
    // 定数 → UPPER_SNAKE_CASE
    static final int MAX_ITEM_COUNT = 100;
 
    // 変数名 → camelCase
    private int itemCount;
 
    // メソッド名 → camelCase
    public void addItem() { ... }
}
lightbulb 良い名前を付けるコツ

名前は「何をするか」「何を表すか」が一目で分かるようにしましょう。例えば、変数名 x よりも studentAge の方が、後からコードを読む人（自分を含む）にとって圧倒的にわかりやすくなります。

1章 練習問題

Q1

JDK、JRE、JVMの関係について正しい記述はどれですか。

  1. JDKにはJREが含まれるが、JVMは別途インストールが必要である
  2. JDKにはJREとJVMの両方が含まれており、JDKをインストールすれば開発と実行の両方が可能である
  3. JREにはJDKが含まれており、JREをインストールすればコンパイルも可能である
  4. JVM、JRE、JDKはそれぞれ独立しており、すべて個別にインストールする必要がある
Q2

次のJavaプログラムについて、コンパイル・実行できるために必要な条件として正しいものはどれですか。

public class MyApp {
    public static void main(String[] args) {
        System.out.println("Hello!");
    }
}
  1. ファイル名は myapp.java でなければならない
  2. ファイル名は MyApp.java でなければならない
  3. ファイル名は何でもよい
  4. ファイル名は Main.java でなければならない
Q3

Javaの命名規則として正しい組み合わせはどれですか。

  1. クラス名: studentManager、メソッド名: GetName、定数: maxValue
  2. クラス名: StudentManager、メソッド名: getName、定数: MAX_VALUE
  3. クラス名: Student_Manager、メソッド名: get_name、定数: MAX_VALUE
  4. クラス名: STUDENTMANAGER、メソッド名: getname、定数: max_value
Answer

1章 解説・解答

Q1の解答: B

JDKはJREを含み、JREはJVMを含む入れ子構造（JDK ⊃ JRE ⊃ JVM）になっています。JDKをインストールすれば、コンパイラ（javac）などの開発ツールに加え、JRE（実行環境）とJVM（仮想マシン）も含まれるため、開発と実行の両方が可能です。Java 11以降、JREは単独で配布されなくなりました。

Q2の解答: B

Javaでは、publicクラスのクラス名とファイル名を一致させる必要があります。このプログラムのクラス名は MyApp なので、ファイル名は MyApp.java でなければなりません。大文字・小文字も区別されるため、myapp.java ではコンパイルエラーになります。

Q3の解答: B

Javaの命名規則は以下の通りです。クラス名はPascalCase（各単語の先頭を大文字：StudentManager）、メソッド名・変数名はcamelCase（最初の単語は小文字、以降の単語の先頭を大文字：getName）、定数はUPPER_SNAKE_CASE（すべて大文字でアンダースコア区切り：MAX_VALUE）で記述します。
