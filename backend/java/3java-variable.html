Section 3-1

リテラル

リテラルとは、ソースコード中に直接記述される値のことです。数値、文字、文字列などがあります。

1.1 数値リテラル

種類 例 説明
整数（10進数） 123 通常の整数表記
整数（2進数） 0b1010 0bまたは0Bで始まる
整数（8進数） 0755 0で始まる
整数（16進数） 0xFF 0xまたは0Xで始まる
long型 123L 末尾にLまたはlを付ける
浮動小数点（double） 3.14 デフォルトはdouble型
浮動小数点（float） 3.14f 末尾にFまたはfを付ける
lightbulb アンダースコア区切り

Java 7以降、数値リテラルにアンダースコアを入れて読みやすくできます：1_000_000

public class LiteralDemo {
    public static void main(String[] args) {
        // 文字リテラル（シングルクォート）
        char c = 'A';

        // 文字列リテラル（ダブルクォート）
        String s = "Hello";

        // エスケープシーケンス
        String newline = "Line1\nLine2";  // 改行
        String tab = "Column1\tColumn2";  // タブ

        System.out.println(c);
        System.out.println(s);
        System.out.println(newline);
        System.out.println(tab);
        System.out.println(quote);
    }
}

文字リテラル

char 変数名 = '文字';

シングルクォートで囲んだ1文字を表します。char型は2バイトのUnicode文字を格納できます。

文字列リテラル

String 変数名 = "文字列";

ダブルクォートで囲んだ0文字以上の文字の並びを表します。空文字列""も有効です。

エスケープシーケンス

\n（改行）, \t（タブ）, \\（バックスラッシュ）, \"（ダブルクォート）

バックスラッシュで始まる特殊な文字表現です。直接入力できない文字や特殊な意味を持つ文字を表現します。

エスケープシーケンス 意味
\n改行
\tタブ
\\バックスラッシュ
\"ダブルクォート
\'シングルクォート

真偽値リテラル

boolean 変数名 = true; または boolean 変数名 = false;

boolean型はtrue（真）とfalse（偽）の2つの値のみを持ちます。条件分岐やループの制御に使用します。

nullリテラル

参照型 変数名 = null;

nullは参照型変数が「何も参照していない」状態を表します。基本データ型（int, booleanなど）には使用できません。

Section 3-2

変数や定数の宣言と初期化

変数はプログラムの中でデータを一時的に保存するための「箱」です。適切な変数名と型を選ぶことは、コードの可読性とバグ防止に直結します。チーム開発では命名規則を統一することで、コードレビューの効率が大幅に向上します。

2.1 変数と命名規則

変数名のルール：

  • 英字、数字、アンダースコア、ドル記号が使用可能
  • 数字で始めることはできない
  • 予約語は使用できない
  • 大文字と小文字は区別される

変数宣言

型名 変数名;

変数を使用する前に、型と名前を宣言します。変数名は英字・アンダースコア・ドル記号で始め、その後は数字も使用可能です。予約語（class、intなど）は使用できません。

        int number;

        int age = 25;

変数宣言と初期化（分離）

型名 変数名; 変数名 = 値;

宣言と値の代入を別々の行で行います。宣言後、使用前に必ず値を代入する必要があります。

変数宣言と初期化（同時）

型名 変数名 = 値;

宣言と同時に初期値を設定します。初期化忘れを防ぐため、この方法が推奨されます。

複数変数の同時宣言

型名 変数名1, 変数名2, 変数名3;

同じ型の変数をカンマで区切って1行で宣言できます。それぞれに初期値を設定することも可能です。

warning ローカル変数の初期化

ローカル変数は使用前に必ず初期化が必要です。初期化されていない変数を使用するとコンパイルエラーになります。

finalキーワードを付けると、一度代入した値を変更できなくなります。定数名は慣習的にUPPER_SNAKE_CASE（大文字とアンダースコア）で記述します。円周率やプログラム全体で使う設定値などに使用します。

    }
}

文字列リテラルによる初期化

String 変数名 = "文字列";

最も一般的な文字列の初期化方法です。同じ内容のリテラルは文字列プールで共有されるため、メモリ効率が良いです。

文字列の連結

文字列1 + 文字列2

+演算子で文字列を連結できます。数値と文字列を連結すると、数値は自動的に文字列に変換されます。

import java.util.ArrayList;

public class VarDemo {
    public static void main(String[] args) {
        // varを使った宣言
        var number = 10;           // int型と推論
        var name = "Alice";        // String型と推論
        var list = new ArrayList<String>();  // ArrayList<String>型と推論

        // varが使えない場合
        // var x;                  // 初期化が必要
        // var y = null;           // 型が推論できない
        // var z = {1, 2, 3};      // 配列初期化子は不可

        list.add("Hello");
        System.out.println("number: " + number);
        System.out.println("name: " + name);
        System.out.println("list: " + list);
    }
}
menu_book 構文説明

var（ローカル変数の型推論）

var 変数名 = 初期値;

varを使うと、コンパイラが右辺の値から型を推論します。必ず初期化が必要で、nullのみでは型を推論できません。ローカル変数にのみ使用可能で、フィールドや引数には使えません。ジェネリクスを使う場合に冗長な型宣言を省略できます。

lightbulb varの使用場面

varはローカル変数にのみ使用可能です。フィールド、メソッドの引数、戻り値の型には使用できません。
