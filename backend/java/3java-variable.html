Section 3-1

リテラル

リテラルとは、ソースコード中に直接記述される値のことです。数値、文字、文字列などがあります。

}

Section 3-2

変数や定数の宣言と初期化

変数はプログラムの中でデータを一時的に保存するための「箱」です。適切な変数名と型を選ぶことは、コードの可読性とバグ防止に直結します。チーム開発では命名規則を統一することで、コードレビューの効率が大幅に向上します。

3-2-1 変数と命名規則

変数名のルール：

  • 英字、数字、アンダースコア、ドル記号が使用可能
  • 数字で始めることはできない
  • 予約語は使用できない
  • 大文字と小文字は区別される

変数宣言と初期化（分離）

型名 変数名; 変数名 = 値;

宣言と値の代入を別々の行で行います。宣言後、使用前に必ず値を代入する必要があります。

変数宣言と初期化（同時）

型名 変数名 = 値;

宣言と同時に初期値を設定します。初期化忘れを防ぐため、この方法が推奨されます。

複数変数の同時宣言

warning ローカル変数の初期化

public class VarDemo {

        // varを使った宣言

        var name = "Alice";        // String型と推論

menu_book 構文説明

var（ローカル変数の型推論）

var 変数名 = 初期値;

varを使うと、コンパイラが右辺の値から型を推論します。必ず初期化が必要で、nullのみでは型を推論できません。ローカル変数にのみ使用可能で、フィールドや引数には使えません。ジェネリクスを使う場合に冗長な型宣言を省略できます。

checklist var の使用可否一覧

✓ var name = "Hello"; // String型と推論される
✓ var count = 10; // int型と推論される
✓ var list = new ArrayList<String>(); // 冗長な型宣言を省略
✗ var x; // 初期化なしはコンパイルエラー
✗ var n = null; // nullでは型を推論できない
✗ var arr = {1, 2, 3}; // 配列初期化子は不可
lightbulb varの使用場面

varはローカル変数にのみ使用可能です。フィールド、メソッドの引数、戻り値の型には使用できません。

Exercise

3章 練習問題

Q1

次のうち、コンパイルエラーにならないリテラルの記述はどれですか。

  1. int a = 0b1012;
  2. int b = 1_000_000;
  3. int c = _100;
  4. int d = 100_;
Q2

基本データ型（プリミティブ型）について正しい記述はどれですか。

  1. byte型の範囲は0～255である
  2. char型は1バイトのASCII文字を格納する
  3. float型のリテラルは末尾にfまたはFを付ける必要がある
  4. boolean型のデフォルト値は0である
Q3

var（ローカル変数の型推論）の使用について正しいものはどれですか。

  1. varはクラスのフィールド（メンバ変数）の宣言に使用できる
  2. var x = null; は型が推論できるためコンパイルできる
  3. varはメソッドの引数の型として使用できる
  4. var list = new ArrayList<String>(); はArrayList<String>型と推論される
Answer

3章 解説・解答

Q1の解答: B

1_000_000はJava 7以降で使用可能なアンダースコア区切りで、正しいリテラルです。Aの0b1012は2進数リテラル（0b開始）に2が含まれておりエラーです（2進数は0と1のみ）。Cの_100はアンダースコアで始まるため変数名として解釈されます。Dの100_はアンダースコアがリテラルの末尾にあるためコンパイルエラーです。アンダースコアは数字と数字の間にのみ配置できます。

Q2の解答: C

浮動小数点リテラルはデフォルトでdouble型となるため、float型の変数に代入するには末尾にfまたはFを付ける必要があります。Aのbyte型の範囲は-128～127です。Bのchar型は2バイトのUnicode文字を格納します。Dのboolean型のデフォルト値はfalseであり、数値の0ではありません（Javaではbooleanと数値の間に暗黙的な変換はありません）。

Q3の解答: D

varはローカル変数にのみ使用でき、右辺の値から型を推論します。var list = new ArrayList<String>();は右辺からArrayList<String>型と正しく推論されます。Aのフィールド（メンバ変数）やCのメソッド引数にはvarは使用できません。Bのvar x = null;はnullのみでは型を推論できないためコンパイルエラーになります。
