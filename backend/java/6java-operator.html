Section 6-1

演算子と優先順位

演算子はデータを加工・比較するための基本ツールです。算術演算だけでなく、代入や比較、論理演算など多様な演算子があります。演算子の優先順位を理解することで、括弧を適切に使い、意図通りの計算結果を得られるようになります。

演算子 説明 例
+加算5 + 3 → 8
-減算5 - 3 → 2
*乗算5 * 3 → 15
/除算5 / 3 → 1（整数除算）
%剰余5 % 3 → 2
++インクリメントi++, ++i
--デクリメントi--, --i

menu_book 構文説明

論理演算子

&&（AND）, ||（OR）, !（NOT）

boolean値を組み合わせて複合条件を作成します。&&は両方true、||はどちらかtrue、!は値を反転します。

lightbulb 短絡評価とは

&&は左辺がfalseなら右辺を評価しません。||は左辺がtrueなら右辺を評価しません。これにより効率的な評価とNullPointerExceptionの回避ができます。

Section 6-2

データの比較

プログラムでは「2つの値が等しいか」を判定する場面が頻繁にあります。しかし、Javaでは基本データ型と参照型で比較の仕組みが異なるため、正しく理解しないとバグの原因になります。特に文字列の比較は初心者がつまずきやすいポイントです。

public class ComparisonDemo {
    public static void main(String[] args) {
        // 基本データ型：値を比較
        int a = 10;
        int b = 10;
        System.out.println(a == b);  // true

        // 参照型：参照（アドレス）を比較
        String s1 = new String("Hello");
        String s2 = new String("Hello");
        System.out.println(s1 == s2);  // false（異なるオブジェクト）
        System.out.println(s1.equals(s2));  // true（内容が同じ）
    }
}

public class StringPoolDemo {
    public static void main(String[] args) {
        // 文字列リテラルは文字列プールで共有される
        String s1 = "Hello";
        String s2 = "Hello";
        System.out.println(s1 == s2);  // true（同じオブジェクト）

        // newで作成すると新しいオブジェクト
        String s3 = new String("Hello");
        System.out.println(s1 == s3);  // false

        // 文字列比較には必ずequals()を使用
        System.out.println(s1.equals(s3));  // true
    }
}
warning 重要

文字列の比較には常にequals()メソッドを使用してください。==は参照の比較であり、意図しない結果になることがあります。

chapter03/NullCheckDemo.java Java 
import java.util.Objects;

public class NullCheckDemo {
    public static void main(String[] args) {
        String str = null;

        // nullとの比較は==を使用
        if (str == null) {
            System.out.println("strはnullです");
        }

        // nullに対してメソッドを呼ぶとNullPointerException
        // str.length();  // 実行時エラー

        // 安全な比較方法
        if (str != null && str.length() > 0) {
            // 短絡評価により、strがnullの場合length()は呼ばれない
        }

        // Objects.equals()を使う方法（Java 7以降）
        Objects.equals(str, "Hello");  // strがnullでもOK
    }
}

6章 練習問題

Q1

次のコードの実行結果として正しいものはどれですか？

int a = 5;
int b = a++;
System.out.println(a + ", " + b);
  1. 5, 5
  2. 6, 5
  3. 5, 6
  4. 6, 6
Q2

次のコードの実行結果として正しいものはどれですか？

String s1 = "Hello";
String s2 = "Hello";
String s3 = new String("Hello");
System.out.println((s1 == s2) + ", " + (s1 == s3));
  1. true, true
  2. true, false
  3. false, false
  4. false, true
Q3

次のコードの実行結果として正しいものはどれですか？

String str = null;
if (str != null && str.length() > 0) {
    System.out.println("有効");
} else {
    System.out.println("無効");
}
  1. 有効
  2. 無効
  3. NullPointerException
  4. コンパイルエラー
Answer

6章 解説・解答

Q1の解答: B

後置インクリメントa++は「現在の値を返してから1を加算」します。そのためbには加算前の値5が代入され、その後aは6になります。前置++aの場合は先に加算してから値を返します。

Q2の解答: B

文字列リテラル"Hello"は文字列プールで共有されるため、s1 == s2はtrue（同じオブジェクト）です。一方、new String("Hello")は新しいオブジェクトを生成するため、s1 == s3はfalse（参照が異なる）です。

Q3の解答: B

&&は短絡評価を行います。左辺str != nullがfalseなので、右辺str.length()は評価されず、NullPointerExceptionは発生しません。条件全体がfalseとなり「無効」が出力されます。
