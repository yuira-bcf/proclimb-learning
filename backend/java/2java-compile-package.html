Section 2-1

コマンドラインでのJavaプログラムのコンパイルと実行

IDEを使わずにコマンドラインでJavaプログラムを動かす方法を学びます。普段はIDEがこの作業を自動化してくれますが、内部で何が起きているかを理解することで、ビルドエラーの原因特定やCI/CD環境での作業に対応できるようになります。

2-1-2 ソースファイルの作成

最も基本的なJavaプログラム「Hello World」を作成してみましょう：

## 生成先
- ディレクトリ: chapter01

## 実装要件
- publicクラスとしてHelloWorldクラスを作成
- mainメソッドをエントリーポイントとして定義
- コンソールに「Hello, World!」を出力
- mainメソッドを含め、System.out.printlnで各処理の実行結果を出力して動作確認できるようにする

## 使用するJava構文
- public class宣言
- public static void main(String[] args)
- System.out.println()
chapter01/HelloWorld.java Java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
menu_book 構文説明

クラス宣言

public class クラス名 { }

Javaプログラムの基本単位です。publicは他のクラスからアクセス可能であることを示します。クラス名は大文字で始めるのが慣習です。

mainメソッド

public static void main(String[] args) { }

プログラムのエントリーポイント（開始点）です。publicはどこからでもアクセス可能、staticはインスタンス化不要、voidは戻り値なし、String[] argsはコマンドライン引数を受け取る配列です。

標準出力

System.out.println(出力内容);

コンソールに文字列を出力し、改行します。print()は改行なし、println()は改行ありです。

lightbulb ファイル名の規則

publicクラスを含むファイルは、クラス名と同じファイル名にする必要があります。上記の場合、HelloWorld.javaという名前で保存します。

2-1-3 コンパイル

javacコマンドを使ってソースファイルをコンパイルします：

コンパイルコマンド Terminal 
$ javac HelloWorld.java

コンパイルが成功すると、HelloWorld.classというバイトコードファイルが生成されます。

2-1-4 クラスの実行

javaコマンドでプログラムを実行します：

実行コマンド Terminal 
$ java HelloWorld
Hello, World!
warning 注意

実行時には.class拡張子を付けません。クラス名のみを指定します。

ポイント: コンパイルエラーはプログラム実行前に検出されるため、修正が容易です。実行時エラーはプログラム実行中に発生するため、テストやデバッグで発見する必要があります。

2-1-5 ソースファイルとクラスファイルの関係

  • 1つのソースファイルには複数のクラスを定義できますが、publicクラスは1つだけ
  • ソースファイル内の各クラスに対して、それぞれのクラスファイルが生成される
  • 内部クラスがある場合は外部クラス$内部クラス.classという名前で生成される

2-1-6 ソースファイルモードでの実行

Java 11以降では、コンパイルせずに直接ソースファイルを実行できます：

ソースファイルモード（Java 11以降） Terminal 
$ java HelloWorld.java
Hello, World!

この機能は単一ファイルのプログラムをテストする際に便利です。

Section 2-2

パッケージ宣言とインポート

大規模なプロジェクトでは、数百から数千のクラスを扱うことがあります。パッケージはこれらのクラスを論理的に整理し、名前の衝突を防ぐ仕組みです。Spring BootやMavenプロジェクトでは、パッケージ構成がプロジェクト構造の基盤となるため、この概念をしっかり理解しておくことが重要です。

2-2-1 パッケージ

パッケージは、関連するクラスをグループ化するための仕組みです。名前空間を提供し、クラス名の衝突を防ぎます。

## 生成先
- ディレクトリ: chapter01/src/com/example/myapp
- ファイル名: MyClass.java

## 実装要件
- com.example.myappパッケージに所属するクラスを作成
- publicクラスとしてMyClassを定義
- パッケージ宣言はファイルの先頭に記述
- mainメソッドを含め、System.out.printlnでクラスの実行を確認できるようにする

## 使用するJava構文
- package文
- public class宣言
chapter01/src/com/example/myapp/MyClass.java Java 
package com.example.myapp;

public class MyClass {
    public static void main(String[] args) {
        System.out.println("MyClassが実行されました");
    }
}
menu_book 構文説明

パッケージ宣言

package パッケージ名;

クラスが所属するパッケージを宣言します。ファイルの先頭（コメント以外）に記述します。パッケージ名はドット区切りで階層を表し、通常はドメイン名を逆にした形式（例：com.example.myapp）を使用します。

2-2-2 パッケージ化

パッケージ化するには：

  • ソースファイルの先頭にpackage文を記述
  • パッケージ名に対応するディレクトリ構造にファイルを配置

folder_open パッケージとディレクトリ構造の対応 

src/
├── com/
│   └── example/
│       ├── model/
│       │   └── User.java         ← package com.example.model;
│       └── service/
│           └── UserService.java  ← package com.example.service;
└── org/
    └── utils/
        └── Helper.java       ← package org.utils;
Rule: パッケージ名のドット（. ）がディレクトリの区切りに対応します

2-2-3 パッケージ化されたクラスのコンパイルと実行

コンパイルと実行 Terminal 
# コンパイル（srcディレクトリで実行）
$ javac com/example/myapp/MyClass.java

# 実行（完全修飾クラス名を指定）
$ java com.example.myapp.MyClass

2-2-4 クラスパス

クラスパスは、JVMがクラスファイルを検索する場所を指定します。

クラスパスの指定 Terminal 
# -cpまたは-classpathオプションで指定
$ java -cp ./classes com.example.myapp.MyClass

# 複数のパスを指定（Windowsは;、Unix系は:で区切る）
$ java -cp ./classes:./lib/* com.example.myapp.MyClass

2-2-5 インポート

他のパッケージのクラスを使用するには、import文を使います。

chat プロンプト 
## 機能
Javaのインポート文の使い方を確認するサンプルプロジェクト

## 生成先
- ディレクトリ: chapter01
- ファイル名: ImportDemo.java

## 実装要件
- 特定のクラス（ArrayList）を個別にインポート
- ワイルドカードを使ってパッケージ全体をインポート
- staticインポートで定数やメソッドを直接使用可能にする
- mainメソッドを含め、System.out.printlnで各インポートの動作結果を確認できるようにする

## 使用するJava構文
- import文（単一クラス指定）
- import文（ワイルドカード *）
- import static文
chapter01/ImportDemo.java Java 
// 特定のクラスをインポート
import java.util.ArrayList;

// パッケージ内の全クラスをインポート（ワイルドカード）
import java.util.*;

// staticメンバのインポート
import static java.lang.Math.PI;
import static java.lang.Math.*;

public class ImportDemo {
    public static void main(String[] args) {
        // ArrayListの使用
        ArrayList<String> list = new ArrayList<>();
        list.add("Hello");
        System.out.println(list);

        // staticインポートで直接使用
        System.out.println("円周率: " + PI);
        System.out.println("平方根: " + sqrt(16));
    }
}
menu_book 構文説明

単一クラスのインポート

import パッケージ名.クラス名;

特定のクラスを1つだけインポートします。使用するクラスを明示できるため、コードの可読性が高まります。

ワイルドカードインポート

import パッケージ名.*;

パッケージ内の全クラスをインポートします。ただし、サブパッケージは含まれません。

staticインポート

import static パッケージ名.クラス名.メンバ名;

クラスのstaticメンバ（フィールドやメソッド）を直接使用できるようにします。Math.PIの代わりにPIと書けるようになります。

compare_arrows import文の種類と使い分け

推奨 import java.util.ArrayList; ← 特定クラスをインポート
注意 import java.util.*; ← パッケージ内の全クラス（サブパッケージは含まない）
static import static java.lang.Math.PI; ← staticメンバを直接使用可能
自動 java.lang.* ← String, System等は import不要
lightbulb java.langパッケージ

java.langパッケージのクラス（String、Systemなど）は自動的にインポートされるため、明示的なimport文は不要です。

2-2-6 標準APIのパッケージ

Javaの標準ライブラリには多くのパッケージが含まれています：

パッケージ 説明
java.lang 基本クラス（String、Math、Systemなど）
java.util ユーティリティクラス（List、Map、Dateなど）
java.io 入出力処理（File、InputStreamなど）

2章 練習問題

Q1

パッケージ宣言について正しい記述はどれですか。

  1. パッケージ宣言はソースファイル内のどこにでも記述できる
  2. 1つのソースファイルに複数のパッケージ宣言を記述できる
  3. パッケージ宣言はソースファイルの先頭（コメントを除く）に1つだけ記述する
  4. パッケージ宣言はimport文の後に記述する
Q2

以下のコードをコンパイル・実行するためのコマンドとして正しいものはどれですか。

package com.example;
public class Sample {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
  1. java Sample
  2. java Sample.class
  3. java com.example.Sample
  4. java com/example/Sample
Q3

import文について正しい記述はどれですか。

  1. import java.util.*; と記述すると、java.utilパッケージとそのサブパッケージのすべてのクラスがインポートされる
  2. java.langパッケージのクラスは明示的にインポートしなくても使用できる
  3. import staticを使うと、他のパッケージのクラスをインスタンス化せずに使用できる
  4. import文はパッケージ宣言の前に記述する
Answer

2章 解説・解答

Q1の解答: C

パッケージ宣言（package文）はソースファイルの先頭に1つだけ記述します（コメントは除く）。パッケージ宣言の後にimport文、その後にクラス定義という順序になります。1つのソースファイルに複数のパッケージ宣言を記述することはできません。

Q2の解答: C

パッケージ化されたクラスを実行するには、完全修飾クラス名（パッケージ名を含むクラス名）をドット区切りで指定します。java com.example.Sampleが正しい形式です。実行時に.class拡張子は付けず、パス区切り文字（/）ではなくドット（.）を使用します。

Q3の解答: B

java.langパッケージのクラス（String、System、Mathなど）は自動的にインポートされるため、明示的なimport文は不要です。ワイルドカードインポート（*）はサブパッケージを含みません。import staticはstaticメンバ（フィールドやメソッド）を直接参照するための機能であり、インスタンス化とは無関係です。import文はパッケージ宣言の後に記述します。
