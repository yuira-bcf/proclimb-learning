Section 1-1

VSCodeとは何か

Visual Studio Code（通称 VSCode）は、Microsoft が2015年にリリースしたオープンソースのソースコードエディタです。Windows、macOS、Linuxの主要なOSすべてで動作し、現在では世界中の開発者にもっとも利用されているエディタの一つとなっています。

VSCodeの概要

VSCodeは「コードエディタ」に分類されるソフトウェアです。コードエディタとは、プログラミングのソースコードを記述・編集するための専用テキストエディタです。一般的なテキストエディタ（メモ帳など）と異なり、シンタックスハイライト（色分け表示）、コード補完、デバッグ機能など、開発に特化した機能が備わっています。

VSCodeは Electron というフレームワークをベースに開発されており、Web技術（HTML、CSS、JavaScript）で構築されています。そのため、デスクトップアプリでありながら、Webブラウザのような柔軟なUI表現が可能です。

POINT

VSCodeは「IDE（統合開発環境）」ではなく「コードエディタ」として設計されていますが、豊富な拡張機能によりIDEに匹敵する機能を実現できます。軽量さと高機能を両立した、現代の開発に最適なツールです。

他のエディタ・IDEとの比較

開発ツールにはさまざまな選択肢があります。VSCodeの位置づけを理解するために、代表的なエディタやIDEと比較してみましょう。

Section 1-2

VSCodeの特徴と強み

VSCodeが多くの開発者に支持されている理由は、その優れた特徴にあります。ここでは、VSCodeの主要な強みを4つの観点から解説します。

1. 軽量で高速な動作

VSCodeはElectronベースでありながら、非常に高速に起動・動作します。大規模なIDEのように数十秒の起動時間がかかることはなく、数秒でエディタが立ち上がります。メモリ使用量も比較的少なく、スペックの低いPCでも快適に動作します。

2. 豊富な拡張機能（Extensions）

VSCodeの最大の強みの一つが、拡張機能のエコシステムです。Visual Studio Marketplace には数万の拡張機能が公開されており、あらゆるプログラミング言語、フレームワーク、ツールに対応できます。拡張機能をインストールすることで、VSCodeを自分の開発スタイルに合わせてカスタマイズできます。

  • 言語サポート：Python、Java、Go、Rust、C++ など
  • フォーマッター：Prettier、Black など
  • リンター：ESLint、Pylint など
  • AIアシスタント：GitHub Copilot、Codeium など

3. クロスプラットフォーム対応

Windows、macOS、Linux の3つの主要プラットフォームで動作します。どのOSを使っていても同じ操作感で開発できるため、チーム開発においてOSの違いが障壁になりません。さらに、設定の同期機能（Settings Sync）を使えば、複数のデバイス間で設定を統一できます。

4. Git統合とターミナル

VSCodeにはGitのソース管理機能が標準で組み込まれています。ファイルの変更状態の確認、コミット、プッシュ、プルなどの操作をエディタ内で行えます。また、統合ターミナルを内蔵しており、別のターミナルアプリを開かずにコマンド操作が可能です。

💫 VSCodeの4大特徴
⚡ 軽量・高速 数秒で起動、低メモリ
🔧 豊富な拡張機能 数万の拡張が利用可能
💻 クロスプラットフォーム Win / Mac / Linux
📁 Git統合 ソース管理を内蔵
Section 1-3

Windowsへのインストール

Windows環境にVSCodeをインストールする手順を解説します。インストールは非常に簡単で、数分で完了します。

ステップ1: インストーラーのダウンロード

  1. ブラウザで https://code.visualstudio.com/ にアクセスします
  2. トップページの「Download for Windows」ボタンをクリックします
  3. インストーラー（VSCodeUserSetup-x64-X.XX.X.exe）がダウンロードされます
POINT

「User Installer」と「System Installer」の2種類があります。通常は「User Installer」で問題ありません。System Installerは、PCの全ユーザーにインストールする場合に使用します。

ステップ2: インストーラーの実行

  1. ダウンロードした .exe ファイルをダブルクリックして実行します
  2. 使用許諾契約書を確認し「同意する」を選択して「次へ」をクリック
  3. インストール先フォルダを確認（通常はデフォルトのまま）して「次へ」
  4. 追加タスクの選択画面で以下のオプションを推奨チェック：
    • 「PATHへの追加」（推奨: 必ずチェック）
    • 「エクスプローラーのファイルコンテキストメニューに追加」
    • 「エクスプローラーのディレクトリコンテキストメニューに追加」
  5. 「インストール」ボタンをクリックしてインストールを開始
  6. 完了後、「Visual Studio Codeを実行する」にチェックを入れて「完了」
注意

「PATHへの追加」は必ずチェックしてください。これにより、コマンドプロンプトやPowerShellから code コマンドでVSCodeを起動できるようになります。

ステップ3: 起動の確認

インストールが完了したら、以下の方法でVSCodeが正常に起動できることを確認しましょう。

  • デスクトップアイコンからダブルクリックで起動
  • スタートメニューから「Visual Studio Code」を検索して起動
  • コマンドプロンプトで code と入力して起動
Section 1-4

macOSへのインストール

macOS環境にVSCodeをインストールする方法は主に2つあります。Homebrewを使う方法（推奨）と、公式サイトから.dmgファイルをダウンロードする方法です。

方法1: Homebrewを使ったインストール（推奨）

Homebrew はmacOSのパッケージマネージャーで、コマンド一つでソフトウェアをインストール・管理できます。VSCodeのインストールにも最も手軽な方法です。

Terminal
# Homebrewがインストール済みの場合
brew install --cask visual-studio-code

# インストール確認
code --version
POINT

Homebrewがまだインストールされていない場合は、まず https://brew.sh/ の手順に従ってHomebrewをインストールしてください。

方法2: .dmgファイルからのインストール

  1. https://code.visualstudio.com/ にアクセスし、「Download for Mac」をクリック
  2. ダウンロードされた .zip ファイルを展開します
  3. 展開された「Visual Studio Code.app」を アプリケーションフォルダにドラッグ＆ドロップ
  4. アプリケーションフォルダからVSCodeを起動

codeコマンドの有効化

macOSではターミナルから code コマンドを使うために、追加の設定が必要です。

  1. VSCodeを起動
  2. Cmd + Shift + P でコマンドパレットを開く
  3. 「Shell Command: Install 'code' command in PATH」と入力して実行
  4. 成功メッセージが表示されたら設定完了
Terminal - 確認
# codeコマンドが使えるか確認
code --version

# フォルダをVSCodeで開く
code ~/Desktop/my-project

Section 1-5

初期設定と日本語化

VSCodeをインストールしたら、最初に行うべき初期設定を解説します。日本語化と基本的な設定を行うことで、快適な開発環境が整います。

日本語化 - Japanese Language Pack

VSCodeのデフォルト言語は英語です。日本語で使用するには、公式の日本語拡張機能をインストールします。

  1. VSCodeを起動
  2. 左サイドバーの拡張機能アイコン（四角形が4つのアイコン）をクリック、または Ctrl + Shift + X（macOS: Cmd + Shift + X）
  3. 検索窓に「Japanese Language Pack」と入力
  4. 「Japanese Language Pack for Visual Studio Code」をインストール
  5. 右下に表示される「Restart」ボタンをクリックしてVSCodeを再起動
  6. 再起動後、メニューやUIが日本語になっていることを確認
POINT

日本語化後も、コマンドパレット（Ctrl+Shift+P）では英語名でもコマンドを検索できます。オンラインの情報は英語が多いため、英語のコマンド名にも慣れておくと便利です。

基本的な初期設定

VSCodeの設定は GUI（設定画面）または settings.json（JSON形式の設定ファイル）で行えます。

設定画面の開き方

  • GUI：Ctrl + ,（macOS: Cmd + ,）
  • settings.json：コマンドパレット → 「Preferences: Open User Settings (JSON)」

推奨する初期設定

settings.json
{
    // フォントサイズ
    "editor.fontSize": 14,

    // フォントファミリー（JetBrains Mono推奨）
    "editor.fontFamily": "'JetBrains Mono', 'Fira Code', Consolas, monospace",

    // タブサイズ
    "editor.tabSize": 2,

    // 保存時に自動フォーマット
    "editor.formatOnSave": true,

    // ミニマップを非表示（初心者は非表示推奨）
    "editor.minimap.enabled": false,

    // 自動保存を有効化
    "files.autoSave": "afterDelay",
    "files.autoSaveDelay": 1000,

    // 末尾の空白を自動削除
    "files.trimTrailingWhitespace": true,

    // ファイル末尾に空行を挿入
    "files.insertFinalNewline": true
}

設定の種類

VSCodeの設定にはスコープ（適用範囲）があります。

設定の種類 適用範囲 保存場所
ユーザー設定 すべてのプロジェクトに適用 OS固有のユーザーフォルダ
ワークスペース設定 特定のプロジェクトのみ .vscode/settings.json

ワークスペース設定はユーザー設定より優先されます。チーム開発では、.vscode/settings.json をGitリポジトリに含めることで、チーム全員の設定を統一できます。

Exercise

1章 練習問題

Q1

VSCodeの説明として正しいものはどれですか？

  1. Microsoftが開発した有料の統合開発環境（IDE）である
  2. Microsoftが開発したオープンソースのコードエディタである
  3. Windowsでしか動作しないテキストエディタである
  4. 拡張機能のインストールはできない軽量エディタである
Q2

Windowsへのインストール時に、特にチェックを推奨されるオプションはどれですか？

  1. デスクトップにショートカットを作成する
  2. PATHへの追加
  3. ファイルの関連付けをすべてVSCodeにする
  4. スタートメニューに登録しない
Q3

VSCodeの設定について正しいものはどれですか？

  1. 設定はGUIからのみ変更でき、JSONファイルでは変更できない
  2. ユーザー設定はワークスペース設定より優先される
  3. ワークスペース設定はユーザー設定より優先される
  4. 設定は一度行うと変更できない
Answer

1章 解説・解答

Q1の解答: B

VSCodeは Microsoft が開発したオープンソースのコードエディタです。IDEではなくコードエディタに分類されますが、拡張機能によりIDEに匹敵する機能を実現できます。無料で利用でき、Windows・macOS・Linuxのクロスプラットフォームで動作します。

Q2の解答: B

「PATHへの追加」を有効にすると、コマンドプロンプトやPowerShellから code コマンドでVSCodeを起動できるようになります。これにより、ターミナルから code . と入力してカレントフォルダをVSCodeで開くなど、効率的な操作が可能になります。

Q3の解答: C

VSCodeの設定にはユーザー設定とワークスペース設定があり、ワークスペース設定がユーザー設定より優先されます。設定はGUIとsettings.json（JSON形式）の両方から変更可能で、いつでも何度でも変更できます。ワークスペース設定は .vscode/settings.json に保存されます。
