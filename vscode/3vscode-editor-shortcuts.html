Section 3-1

IntelliSense（コード補完）

IntelliSense（インテリセンス）は、VSCodeに組み込まれた強力なコード補完機能です。変数名や関数名の自動補完、パラメータヒント、定義へのジャンプなど、コーディングを大幅に効率化する機能を提供します。

コード補完の基本

コードを入力している途中で、IntelliSenseが自動的に補完候補を表示します。候補が表示されたら、矢印キーで選択して Tab または Enter で確定できます。

  • 自動表示：文字を入力すると候補が自動的に表示される
  • 手動呼び出し：Ctrl + Space（macOS: Cmd + Space）で候補を手動表示
  • 候補の詳細：候補にカーソルを合わせると、型情報やドキュメントが表示される
JavaScript - IntelliSenseの例
// 変数の型をIntelliSenseが推論する
const message = "Hello, World!";
message.  // ← ここで "." を入力するとStringのメソッド候補が表示される
// .toUpperCase(), .toLowerCase(), .includes(), .split() など

// 関数のパラメータヒント
document.getElementById("app");
// ↑ 関数名の後に "(" を入力すると、引数の型が表示される

定義へのジャンプと参照の検索

IntelliSenseを使えば、関数や変数の定義場所にすばやくジャンプしたり、どこで使われているかを確認したりできます。

POINT

IntelliSenseの精度は言語によって異なります。TypeScript、JavaScript、Python、Java などは標準で高精度な補完が利用できます。その他の言語では、対応する拡張機能をインストールすることで精度が向上します。

Section 3-2

マルチカーソル編集

マルチカーソルは、複数の場所に同時にカーソルを配置し、一括編集できるVSCodeの強力な機能です。同じ変更を複数箇所に適用するとき、手作業で1つずつ修正する必要がなくなります。

マルチカーソルの作り方

操作 Windows macOS 説明
任意位置にカーソル追加 Alt + Click Option + Click クリックした位置に新しいカーソルを配置
同じ単語を追加選択 Ctrl + D Cmd + D 現在の選択と同じ単語を次々選択
同じ単語を全選択 Ctrl + Shift + L Cmd + Shift + L 同じ単語すべてにカーソルを配置
上下にカーソル追加 Ctrl + Alt + ↑/↓ Cmd + Option + ↑/↓ 現在行の上下にカーソルを追加

マルチカーソルの活用例

以下のHTMLコードで、すべての div を section に変更する例を見てみましょう。

変更前 - HTML
<div class="header">Header</div>
<div class="main">Main Content</div>
<div class="sidebar">Sidebar</div>
<div class="footer">Footer</div>

手順：最初の div を選択 → Ctrl + D を繰り返し押して全ての div を選択 → section と入力

変更後 - HTML
<section class="header">Header</section>
<section class="main">Main Content</section>
<section class="sidebar">Sidebar</section>
<section class="footer">Footer</section>
注意

Ctrl + D は選択を1つずつ追加します。意図しない箇所が選択された場合は Ctrl + U で最後の選択を取り消せます。また Esc ですべてのマルチカーソルを解除できます。

Section 3-3

検索と置換

VSCodeの検索・置換機能は非常に強力で、単一ファイル内の検索からプロジェクト全体にわたるファイル横断検索まで対応しています。正規表現検索も利用でき、複雑なパターンマッチングも可能です。

ファイル内の検索と置換

機能 Windows macOS
検索 Ctrl + F Cmd + F
置換 Ctrl + H Cmd + H
次の一致に移動 Enter / F3 Enter / F3
前の一致に移動 Shift + Enter / Shift + F3 Shift + Enter / Shift + F3

検索バーには以下のオプションがあります（アイコンで切り替え）。

  • Aa：大文字/小文字を区別する（Match Case）
  • ab：単語全体を一致させる（Match Whole Word）
  • .*：正規表現を使用する（Use Regular Expression）

ファイル横断検索

プロジェクト全体からテキストを検索するには、ファイル横断検索を使います。

  • Windows：Ctrl + Shift + F
  • macOS：Cmd + Shift + F

サイドバーの検索ビューが開き、プロジェクト内のすべてのファイルから一致するテキストが一覧表示されます。「含めるファイル」「除外するファイル」のフィルターも設定できます。

検索フィルターの例
含めるファイル: *.js, *.ts
除外するファイル: node_modules, dist, .git

正規表現検索の活用例

Section 3-4

スニペット

スニペットとは、よく使うコードパターンをテンプレートとして登録し、短いキーワードで呼び出せる機能です。定型的なコードを毎回手入力する手間を省き、開発速度を大幅に向上させます。

組み込みスニペットの利用

VSCodeには言語ごとに多数の組み込みスニペットが用意されています。例えば、HTMLファイルで ! と入力して Tab を押すと、HTMLの基本テンプレートが展開されます。

HTML - 「!」スニペットの展開結果
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>

その他のよく使うHTMLスニペットの例：

  • div → <div></div>
  • a → <a href=""></a>
  • link:css → <link rel="stylesheet" href="style.css">
  • script:src → <script src=""></script>

ユーザースニペットの作成

独自のスニペットを作成して登録することもできます。コマンドパレットで「Preferences: Configure User Snippets」と入力し、対象の言語を選択します。

javascript.json - ユーザースニペットの例
{
    "Console Log": {
        "prefix": "clog",
        "body": [
            "console.log('${1:message}', ${2:value});",
            "$0"
        ],
        "description": "console.logを挿入"
    },
    "Arrow Function": {
        "prefix": "af",
        "body": [
            "const ${1:name} = (${2:params}) => {",
            "    ${3:// TODO}",
            "    $0",
            "};"
        ],
        "description": "アロー関数を挿入"
    }
}
プロパティ 説明
prefix スニペットを呼び出すためのキーワード
body 展開されるコードの本体（配列の各要素が1行）
description IntelliSenseで表示される説明文
${1:placeholder} タブストップ（Tab で移動する入力箇所）
$0 最終カーソル位置
POINT

スニペットの ${1}、${2} はタブストップと呼ばれ、Tab キーを押すと順番にカーソルが移動します。${1:placeholder} のように初期値を設定でき、そのまま上書き入力するだけで効率的に編集できます。

Exercise

3章 練習問題

Q1

IntelliSenseで関数や変数の定義場所にジャンプするショートカットはどれですか？

  1. Ctrl + P
  2. Ctrl + G
  3. F12
  4. Ctrl + Shift + F
Q2

マルチカーソルで、現在選択している単語と同じ単語を「次のマッチ」として追加選択するショートカットはどれですか？（Windows）

  1. Ctrl + L
  2. Ctrl + D
  3. Ctrl + Shift + L
  4. Alt + Click
Q3

ユーザースニペットのJSON設定で、${1:placeholder} の意味として正しいものはどれですか？

  1. スニペット展開後の最終カーソル位置
  2. 1番目のタブストップで、初期値が「placeholder」
  3. スニペットの説明文（description）
  4. スニペットを呼び出すためのキーワード（prefix）
Answer

3章 解説・解答

Q1の解答: C

F12 は「定義へ移動（Go to Definition）」のショートカットです。関数名や変数名にカーソルを合わせて F12 を押すと、その定義場所にジャンプします。Alt + F12 で定義のインラインプレビュー、Shift + F12 で参照箇所の一覧表示も可能です。

Q2の解答: B

Ctrl + D は、現在選択中の単語と同じテキストを「次のマッチ」として追加選択します。繰り返し押すと、次々にマッチが選択されます。一方、Ctrl + Shift + L は同じ単語のすべてのマッチを一度に選択し、Alt + Click は任意の位置にカーソルを手動で追加する操作です。

Q3の解答: B

${1:placeholder} は1番目のタブストップで、初期値として「placeholder」が入力された状態になります。Tab キーを押すと ${2}、${3}... の順にカーソルが移動します。$0 が最終カーソル位置で、最後にカーソルが止まる場所を示します。
