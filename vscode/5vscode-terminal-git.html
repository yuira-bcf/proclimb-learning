Section 5-1

統合ターミナルの基本

VSCodeには統合ターミナル（Integrated Terminal）が内蔵されています。外部のターミナルアプリを別途開く必要がなく、エディタの中で直接コマンドを実行できるため、開発効率が大幅に向上します。

ターミナルの開き方

統合ターミナルを開くには、以下のいずれかの方法を使います。

  • ショートカットキー： Ctrl + `（Windows/Linux）、Cmd + `（macOS）
  • メニュー： 「ターミナル」 → 「新しいターミナル」
  • コマンドパレット： Ctrl+Shift+P → 「Terminal: Create New Terminal」

ターミナルはエディタ画面の下部に表示されます。パネル上部のハンドルをドラッグすると、表示領域の高さを調節できます。

複数ターミナルの管理

VSCodeでは複数のターミナルインスタンスを同時に開くことができます。たとえば、1つ目でサーバーを起動し、2つ目でGitコマンドを実行するといった使い方が可能です。

  • 新しいターミナル：ターミナルパネル右上の「+」ボタン、または Ctrl+Shift+`
  • ターミナル切替：ドロップダウンリストから選択、または Ctrl+PageUp/PageDown
  • ターミナル削除：ゴミ箱アイコンで不要なターミナルを閉じる

シェルの切り替え

使用するシェルはOS環境によって異なりますが、VSCodeの設定で変更が可能です。

VSCode統合ターミナル - シェルの選択肢
💻 PowerShell Windows標準
🐱 Git Bash Git for Windows
📄 Command Prompt Windows従来型
🍎 zsh / bash macOS / Linux
⚙ WSL Windows Subsystem for Linux
info POINT

Web開発ではGit Bashやzshがよく使われます。Windowsユーザーは「Git for Windows」をインストールすると、Git BashをVSCodeのデフォルトシェルとして設定できます。設定は Ctrl+Shift+P → 「Terminal: Select Default Profile」から変更します。

Section 5-2

ターミナルの便利機能

VSCodeの統合ターミナルには、開発を効率化するさまざまな便利機能が備わっています。ここでは代表的な機能を紹介します。

ターミナルの分割

1つのターミナルパネル内で、画面を左右に分割して複数のシェルを同時に表示できます。

  • 分割方法：ターミナルパネル右上の分割アイコン、または Ctrl+Shift+5
  • 用途例：左でサーバー起動、右でファイル操作やテスト実行

タスクランナーとの統合

VSCodeの「タスク」機能を使うと、よく使うコマンドをワンクリックで実行できます。tasks.jsonに定義したタスクは、Ctrl+Shift+B（ビルドタスク）や Ctrl+Shift+P → 「Tasks: Run Task」から呼び出せます。

JSON .vscode/tasks.json 
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "開発サーバー起動",
      "type": "shell",
      "command": "npm run dev",
      "group": "build",
      "presentation": {
        "reveal": "always",
        "panel": "new"
      }
    },
    {
      "label": "テスト実行",
      "type": "shell",
      "command": "npm test",
      "group": "test"
    }
  ]
}

ターミナルプロファイル設定

ターミナルの見た目や動作は、VSCodeの設定（settings.json）でカスタマイズできます。

JSON settings.json（抜粋） 
{
  // デフォルトのシェルプロファイル（Windows）
  "terminal.integrated.defaultProfile.windows": "Git Bash",

  // フォントサイズ
  "terminal.integrated.fontSize": 14,

  // カーソルスタイル
  "terminal.integrated.cursorStyle": "line",

  // スクロールバック行数
  "terminal.integrated.scrollback": 5000
}

Section 5-3

ソース管理ビュー（Git SCM）

VSCodeにはGitとの連携機能が標準で組み込まれています。サイドバーの「ソース管理」アイコン（分岐のアイコン）をクリックすると、SCM（Source Control Management）パネルが開きます。ここからGUIでGit操作を行えます。

ソース管理パネルの見方

ソース管理パネルには、現在の作業ディレクトリ内で変更されたファイルが一覧表示されます。

  • M（Modified）：変更されたファイル
  • U（Untracked）：新規作成でまだGit管理下にないファイル
  • D（Deleted）：削除されたファイル
  • A（Added）：ステージング済みの新規ファイル
Git ステージング・コミットの流れ
📝 作業ディレクトリ ファイルを編集
➔
📦 ステージング git add（+ボタン）
➔
✅ コミット git commit
➔
☁ リモート git push

変更の確認とステージング

ソース管理パネルでファイル名をクリックすると、差分ビュー（Diff View）が開き、変更内容を確認できます。

  • ステージングする：ファイル名右側の「+」ボタンをクリック
  • ステージングを解除する：ステージ済みファイル右側の「-」ボタンをクリック
  • 全ファイルをステージ：「変更」セクション右側の「+」をクリック
info POINT

ソース管理パネルを使えば、コマンドラインでの git add・git status と同じ操作を視覚的に行えます。特にどのファイルが変更されたかが一目でわかるのが大きなメリットです。

Section 5-4

Git基本操作

VSCodeのGUI（ソース管理ビュー）から、Gitの基本操作を行う方法を見ていきましょう。コマンドラインを使わなくても、日常的なGit操作のほとんどをVSCode上で完結できます。

コミット（Commit）

変更をステージングした後、ソース管理パネル上部のメッセージ入力欄にコミットメッセージを入力し、チェックマーク（Commit）ボタンをクリックします。

操作手順 VSCodeでのコミット 
1. ソース管理パネルを開く（Ctrl+Shift+G）
2. 変更ファイルの「+」ボタンでステージング
3. メッセージ欄に「機能Aを追加」などと入力
4. チェックマーク（✓）ボタンをクリック
   → コミット完了

プッシュ（Push）とプル（Pull）

コミット後、リモートリポジトリ（GitHubなど）に反映するにはPushを行います。逆に、他の人の変更を取り込むにはPullを行います。

  • Push：ソース管理パネルの「...」メニュー → 「Push」、またはステータスバーの同期アイコン
  • Pull：ソース管理パネルの「...」メニュー → 「Pull」
  • 同期（Sync）ボタンは、PullとPushを一度に行います

ブランチの切り替え・作成

ステータスバー左下にはブランチ名が表示されています。ここをクリックすると、ブランチの切り替えや新規作成ができます。

  • 既存ブランチに切替：ブランチ名をクリック → リストから選択
  • 新しいブランチ作成：ブランチ名をクリック → 「新しいブランチの作成」を選択 → ブランチ名を入力
VSCodeで行うGit基本操作の全体像
📂 Clone リポジトリの複製
📝 Stage + Commit 変更を記録
🚀 Push リモートに送信
📥 Pull リモートから取得
🔌 Branch ブランチの作成・切替
🔁 Merge ブランチの統合
warning 注意

Pushする前に必ずPullを行い、リモートの最新変更を取り込んでおきましょう。これを怠ると、コンフリクト（競合）が発生しやすくなります。

Section 5-5

差分表示とマージ

VSCodeには強力な差分表示（Diff）機能が内蔵されています。変更内容の確認やコンフリクトの解消を、視覚的にわかりやすく行えます。

差分ビュー（Diff View）

ソース管理パネルで変更ファイルをクリックすると、サイドバイサイドDiffが表示されます。左が変更前、右が変更後です。

  • 追加行：緑色の背景で表示
  • 削除行：赤色の背景で表示
  • 変更行：赤（削除）＋緑（追加）のペアで表示

インラインDiff

サイドバイサイド表示だけでなく、インライン表示に切り替えることもできます。Diffビューの右上にある切替ボタンで変更できます。インライン表示では、変更前後が1つのビュー内に表示されるため、画面の小さい環境でも使いやすくなっています。

マージコンフリクトの解消

複数の人が同じファイルの同じ箇所を編集した場合、マージコンフリクト（競合）が発生します。VSCodeではコンフリクト箇所がハイライト表示され、以下の選択肢がボタンで提示されます。

  • Accept Current Change：自分の変更を採用
  • Accept Incoming Change：相手の変更を採用
  • Accept Both Changes：両方の変更を残す
  • Compare Changes：変更内容を比較して確認
テキスト コンフリクト発生時のファイル表示例 
<<<<<<< HEAD
const greeting = "こんにちは";
=======
const greeting = "Hello";
>>>>>>> feature-branch

VSCodeでは上記のようなマーカーを自動検出し、クリック操作で簡単に解消できます。手動で編集することも可能です。

lightbulb ヒント

コンフリクトを減らすコツは、こまめにPull（最新取得）してからPush（送信）すること、そして作業ブランチを細かく分けることです。チーム開発では、プルリクエスト（Pull Request）を使ったレビューフローも有効です。

Exercise

練習問題

Q1

VSCodeの統合ターミナルを開くショートカットキー（Windows/Linux）を答えてください。また、新しいターミナルを追加で開くショートカットキーも答えてください。

Q2

Gitのソース管理パネルで、ファイルの横に表示されるアルファベット記号「M」「U」「D」はそれぞれ何を意味しますか？日本語で説明してください。

Q3

マージコンフリクトが発生した際、VSCodeが表示する4つの解消オプションをすべて挙げ、それぞれの意味を簡潔に説明してください。

Answer

解説・解答

Q1 の解答

統合ターミナルを開くショートカットキーは Ctrl + `（バッククォート）です。新しいターミナルを追加で開くショートカットキーは Ctrl+Shift+` です。macOSの場合は Ctrl を Cmd に読み替えます。

Q2 の解答

M（Modified）：既存のファイルが変更されたことを示します。
U（Untracked）：新しく作成されたファイルで、まだGitの管理下に登録されていない（追跡されていない）ことを示します。
D（Deleted）：ファイルが削除されたことを示します。

Q3 の解答

Accept Current Change：自分の変更（HEADの内容）を採用し、相手の変更を破棄します。
Accept Incoming Change：相手の変更（マージ元の内容）を採用し、自分の変更を破棄します。
Accept Both Changes：自分と相手の両方の変更を残します（順番に並べて両方を保持）。
Compare Changes：変更内容を比較表示して、どの部分が異なるかを確認できます。
