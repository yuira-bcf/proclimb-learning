Section 19-1

工程概要

@Transactionalでデータベース操作の整合性を保証し、エラー発生時のロールバックを実装します。

学習テーマ

データ整合性の保証

主な技術

@Transactional / rollbackFor / Propagation

関連レイヤー

Service / Repository

この機能で行うこと

@Transactionalでメソッドを囲む

visibility

読み取り専用トランザクション

readOnly = trueで最適化

ロールバック制御

rollbackFor / noRollbackFor

call_split

トランザクション伝播

Propagation.REQUIRES_NEWなど

Section 19-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
TodoService.java @Transactional適用、ロールバック制御
AuditService.java REQUIRES_NEWで独立トランザクション

@Transactionalの属性

  • readOnly - 読み取り専用（最適化のヒント）デフォルト: false
  • timeout - タイムアウト（秒）デフォルト: -1（制限なし）
  • rollbackFor - ロールバック対象の例外 デフォルト: RuntimeException
  • noRollbackFor - ロールバックしない例外 デフォルト: なし
  • propagation - トランザクションの伝播 デフォルト: REQUIRED
Section 19-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリにトランザクション管理機能を追加してください。

【要件】
- TodoServiceに@Transactionalを適用してデータベース操作の整合性を保証
- ToDoと履歴を同一トランザクションで保存し、どちらかで例外が発生したら両方ロールバック
- 読み取り専用メソッドには@Transactional(readOnly = true)を適用
- rollbackFor属性でチェック例外もロールバック対象に設定
- 監査ログ記録にはREQUIRES_NEWを使用し、本体がロールバックしても監査ログは残す

【実装箇所】

■ TodoService.java
- save()メソッドに@Transactionalを追加
- ToDoの保存と履歴の保存を同一トランザクション内で実行
- どちらかで例外が発生すると両方ロールバック

- findAll(), findById()には@Transactional(readOnly = true)を追加
- 読み取り専用でDB接続を最適化

- @Transactional(rollbackFor = Exception.class)でチェック例外もロールバック
- @Transactional(noRollbackFor = BusinessException.class)で特定例外はコミット

■ AuditService.java
- @Transactional(propagation = Propagation.REQUIRES_NEW)
- 監査ログは本体のトランザクションとは独立して処理
- 本体がロールバックしても監査ログは残す

【技術スタック】
- Spring Boot 3.x
- Spring Transaction
- MyBatis
