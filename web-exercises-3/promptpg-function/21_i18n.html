Section 21-1

工程概要

アプリケーションを日本語・英語などの複数言語に対応させ、ユーザーが言語を切り替えられるようにします。

Section 21-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
WebConfig.java LocaleResolver、LocaleChangeInterceptorの設定
messages_ja.properties 日本語メッセージファイル
messages_en.properties 英語メッセージファイル
application.properties メッセージソース設定の追加
各種HTMLテンプレート #{key}形式のメッセージ参照に修正

国際化の主要コンポーネント

  • MessageSource - メッセージをロケールに応じて取得
  • LocaleResolver - 現在のロケールを決定・変更
  • LocaleChangeInterceptor - URLパラメータで言語切り替え
  • #{key} - Thymeleafでのメッセージ参照形式
Section 21-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリに国際化（i18n）機能を追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- 現在は日本語のみ対応
- 画面のラベルやメッセージがハードコードされている

【実装要件】
1. 多言語対応の設定
   - SessionLocaleResolverでセッションベースのロケール管理
   - LocaleChangeInterceptorでURLパラメータ(?lang=ja/?lang=en)による言語切り替え
   - デフォルト言語は日本語

2. メッセージファイルの作成
   - messages_ja.properties（日本語）
   - messages_en.properties（英語）
   - ページタイトル、ラベル、ボタン、成功/エラーメッセージを定義

3. Thymeleafテンプレートの修正
   - ハードコードされた文字列を#{key}形式に置き換え
   - 言語切り替えリンクをヘッダーに追加

4. バリデーションメッセージの国際化
   - ValidationMessages_ja.properties
   - ValidationMessages_en.properties

5. コントローラーでのメッセージ取得
   - MessageSourceをインジェクション
   - LocaleContextHolder.getLocale()で現在のロケールを取得

【技術スタック】
- Spring Boot 3.x
- MessageSource / LocaleResolver
- Thymeleaf
- Jakarta Validation

必要なコードの変更点を提示してください。
