Section 23-1

工程概要

時間のかかる処理を非同期で実行し、ユーザー体験を向上させます。

Section 23-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
AsyncConfig.java 非同期処理の設定（スレッドプール設定）
AsyncService.java 非同期処理を行うサービス
NotificationService.java 非同期で通知を送信するサービス
Application.java @EnableAsync の追加
BatchJobService.java バッチ処理の非同期実行

スレッドプール設定のパラメータ

  • corePoolSize - 基本スレッド数（常に維持）
  • maxPoolSize - 最大スレッド数
  • queueCapacity - キューの容量
  • threadNamePrefix - スレッド名のプレフィックス
Section 23-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリに非同期処理機能を追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- 全ての処理が同期的に実行されている
- メール送信やレポート生成など時間のかかる処理がある

【実装要件】
1. 非同期処理の有効化
   - @EnableAsyncをメインクラスまたはConfigクラスに追加
   - AsyncConfigでThreadPoolTaskExecutorを設定
   - corePoolSize=5, maxPoolSize=10, queueCapacity=100

2. 非同期サービスの実装
   - @Asyncアノテーションを使用した非同期メソッド
   - 戻り値なしの非同期メソッド（void）
   - CompletableFutureを返す非同期メソッド

3. 使用例
   - ToDo作成時に非同期で通知を送信
   - レポート生成を非同期で実行
   - 複数の非同期処理の結果を待ち合わせ（CompletableFuture.allOf）

4. 例外処理
   - 非同期処理での例外をハンドリング
   - AsyncUncaughtExceptionHandlerの実装

5. スケジューリング（オプション）
   - @Scheduledで定期実行タスクを追加
   - Cron式でのスケジュール設定

【技術スタック】
- Spring Boot 3.x
- @Async / @EnableAsync
- CompletableFuture
- ThreadPoolTaskExecutor

必要なコードの変更点を提示してください。
lightbulb ポイント

@Asyncメソッドが同一クラス内で呼び出された場合、非同期にならないことに注意。別のBeanから呼び出す必要があります。
