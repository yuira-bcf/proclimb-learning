ソートオプション

  • created_at DESC - 作成日 新しい順（デフォルト）
  • created_at ASC - 作成日 古い順
  • priority ASC - 優先度 高い順
  • priority DESC - 優先度 低い順
  • title ASC - タイトル 昇順
  • deadline ASC - 期限 近い順
Section 05-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

ToDoアプリにソート機能を実装してください。

【ソートオプション】
1. 作成日（新しい順）- デフォルト
2. 作成日（古い順）
3. 優先度（高い順: 1→2→3）
4. 優先度（低い順: 3→2→1）
5. タイトル（昇順: A-Z）
6. 期限（近い順）

【実装内容】
1. TodoMapper.java
   - List findAllWithSort(@Param("sortBy") String sortBy, @Param("sortOrder") String sortOrder);

2. TodoMapper.xml
   - タグで動的ORDER BYを実装
   - SQLインジェクション対策として、許可されたカラム名のみを使用

3. TodoController.java
   - @RequestParam でソート条件を受け取り
   - デフォルト値を設定（sortBy="created_at", sortOrder="DESC"）

4. list.html
   - ソート選択のドロップダウンを追加
   - 現在のソート条件を保持
   - ソート変更時に自動的にフォーム送信

【セキュリティ注意点】
- ソートカラム名は${}ではなく、で安全に指定
- ユーザー入力を直接SQLに使用しない

【SQL例】

  
    ORDER BY priority ${sortOrder}
  
  
    ORDER BY title ${sortOrder}
  
  
    ORDER BY created_at DESC
  


必要なコードの変更点を提示してください。
lightbulb ポイント

ORDER BYのカラム名には${}を使用しますが、<choose>タグで許可されたカラムのみを指定することでSQLインジェクションを防止できます。
