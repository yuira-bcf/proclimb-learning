Section 02-1

工程概要

Bean Validationを使用して入力値の検証機能を実装し、不正なデータの登録を防止します。

学習テーマ

入力検証

主な技術

Bean Validation

関連レイヤー

Entity / Controller / View

この機能で行うこと

edit_attributes

アノテーション追加

@NotBlank, @Size等の検証ルールをEntityに追加

settings

Controller修正

@Valid, BindingResultの使用で検証実行

error_outline

エラー表示

th:errorsでメッセージ表示、入力欄の赤枠表示

translate

メッセージカスタマイズ

messages.propertiesで日本語エラーメッセージを設定

Section 02-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
Todo.java バリデーションアノテーション追加
TodoController.java @Valid, BindingResult追加
form.html エラーメッセージ表示
messages.properties カスタムエラーメッセージ（日本語）

使用するアノテーション

  • @NotBlank - 空文字・null不可
  • @Size(max=100) - 文字数制限
  • @Min / @Max - 数値の範囲指定
  • @Email - メール形式チェック
  • @Pattern - 正規表現パターン
Section 02-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

ToDoアプリにBean Validationを使用したバリデーション機能を実装してください。

【検証ルール】
1. title
   - 必須入力（@NotBlank）
   - 最大100文字（@Size(max=100)）
   - エラーメッセージ: 「タイトルは必須です」「タイトルは100文字以内で入力してください」

2. description
   - 任意入力
   - 最大500文字（@Size(max=500)）
   - エラーメッセージ: 「説明は500文字以内で入力してください」

3. priority
   - 1〜3の範囲（@Min(1), @Max(3)）
   - エラーメッセージ: 「優先度は1〜3で指定してください」

【変更が必要なファイル】
1. Todo.java - バリデーションアノテーション追加
2. TodoController.java - @Valid, BindingResultの使用
3. form.html - th:errorsでエラー表示、エラー時のスタイル
4. messages.properties - カスタムエラーメッセージ（日本語）

【要件】
- エラー時はフォームを再表示し、入力値を保持
- エラーがある入力欄は赤枠で表示（is-invalidクラス）
- エラーメッセージは入力欄の下に赤文字で表示

必要なコードの変更点を提示してください。
lightbulb ポイント

BindingResultは必ず@Validの直後に配置します。間に他の引数があるとエラーになります。
