Section 05-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIへの入力プロンプト

あなたはJavaプログラマーです。

Step 4で作成したTodoServiceを使用するController層を作成してください。

【役割】
- HTTPリクエストの受付
- ServiceからのデータをModelに設定
- 適切なViewを返す
- PRGパターンでフォーム二重送信防止

【技術スタック】
- Spring MVC（@Controller）
- Thymeleaf
- コンストラクタインジェクション

【TodoServiceのメソッド】
- findAll(): List
- findById(Long id): Todo
- create(Todo todo): Todo
- update(Todo todo): void
- delete(Long id): void
- toggleComplete(Long id): void

【作成するエンドポイント】
1. GET /todos - 一覧表示（list.html）
2. GET /todos/new - 新規作成フォーム（form.html）
3. POST /todos - 新規作成処理 → リダイレクト
4. GET /todos/{id}/edit - 編集フォーム（form.html再利用）
5. POST /todos/{id} - 更新処理 → リダイレクト
6. POST /todos/{id}/delete - 削除処理 → リダイレクト
7. POST /todos/{id}/toggle - 完了切替 → リダイレクト

【ファイル配置】
src/main/java/com/example/todo/controller/TodoController.java

【要件】
- @Controllerアノテーションを付与
- @RequiredArgsConstructorでServiceを注入
- @PathVariableでURLパラメータ取得
- redirect:で PRGパターン実装

コードを実装してください。
lightbulb ポイント

PRGパターン（Post-Redirect-Get）を使うと、ブラウザの更新ボタンで二重送信されることを防げます。
