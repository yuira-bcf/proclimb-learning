Section 6-1

探索レポートとは

GA4の「探索」機能は、標準レポートでは得られない柔軟な分析を可能にする高度な機能です。自分でディメンションと指標を組み合わせ、独自の視点でデータを深掘りできます。

探索レポートへのアクセス

アクセス手順

  1. GA4の左メニューから「探索」をクリック
  2. テンプレートギャラリーが表示される
  3. 使用したいテンプレートを選択、または「空白」から新規作成

標準レポートと探索レポートの違い

観点 標準レポート 探索レポート
カスタマイズ性 限定的（フィルタ、比較程度） 高い（自由にディメンション・指標を組み合わせ）
分析の深さ 概要・傾向把握向け 詳細分析・仮説検証向け
視覚化 固定されたグラフ・表 複数の視覚化形式から選択可能
セグメント 比較機能で対応 高度なセグメント作成が可能
保存・共有 標準で全員に表示 作成者が保存・共有設定

探索テンプレートの種類

📊

自由形式（Free Form）

テーブルやグラフを自由に作成。最も汎用的なテンプレート

🔽

ファネルデータ探索

ステップごとの離脱・進行を可視化。コンバージョンファネル分析に最適

🔀

経路データ探索

ユーザーの行動経路をツリー形式で可視化。ナビゲーション分析に有用

👥

セグメントの重複

複数セグメントの重なりをベン図で表示。ユーザー層の理解に活用

📈

コホートデータ探索

同一時期のユーザー群の行動変化を追跡。リテンション分析に最適

💰

ユーザーのライフタイム

ユーザーの生涯価値を分析。LTV視点の評価に使用

Section 6-2

自由形式（Free Form）

自由形式は、最も基本的かつ汎用的な探索テンプレートです。表やグラフを自由に作成し、標準レポートでは見られないクロス集計を行えます。

自由形式の画面構成

画面の3つのエリア
  • 変数パネル（左）：使用するセグメント、ディメンション、指標を設定
  • タブの設定（中央）：行・列・値に何を配置するか設定
  • 出力エリア（右）：設定に基づいた結果が表示

基本的な操作手順

自由形式レポートの作成手順

  1. ディメンションを追加：変数パネルの「ディメンション」横の＋をクリックし、使用したいディメンションを選択
  2. 指標を追加：同様に「指標」横の＋をクリックし、使用したい指標を選択
  3. 行に配置：追加したディメンションを「行」エリアにドラッグ
  4. 値に配置：追加した指標を「値」エリアにドラッグ
  5. 結果を確認：右側の出力エリアに表が表示される

視覚化オプション

自由形式では、データの表示形式を選択できます。

形式 用途
テーブル 詳細な数値の確認。最も基本的な形式
ドーナツグラフ 構成比の視覚化。シェアの把握に
折れ線グラフ 時系列の変化。トレンド把握に
散布図 2つの指標の関係性。相関分析に
棒グラフ カテゴリ別比較。ランキング表示に
地図 地域別データの視覚化

自由形式の活用例

分析例：チャネル×デバイス別キーイベント

目的：どのチャネルのどのデバイスが成果に貢献しているかを把握

設定：

  • 行：セッションのデフォルトチャネルグループ
  • 列：デバイスカテゴリ
  • 値：キーイベント、キーイベント率

得られるインサイト：「Organic Searchはデスクトップでキーイベント率が高い」「Paid Socialはモバイルからのキーイベントが多い」などの発見

Section 6-3

ファネルデータ探索

ファネルデータ探索は、ユーザーが特定のステップをどの程度進行・離脱しているかを可視化する機能です。購入プロセスや会員登録フローの分析に最適です。

ファネル分析とは

ファネルの概念

「ファネル（funnel）」は漏斗を意味し、ユーザーがステップを進むごとに人数が減っていく様子を表します。

  • ステップ1：商品一覧ページを見た → 1,000人
  • ステップ2：商品詳細ページを見た → 500人（50%進行）
  • ステップ3：カートに追加した → 100人（20%進行）
  • ステップ4：購入完了した → 30人（30%進行）

ファネルの設定方法

ファネル作成の手順

  1. 探索から「ファネルデータ探索」テンプレートを選択
  2. 「タブの設定」パネルで「ステップ」の鉛筆アイコンをクリック
  3. 各ステップの条件を設定（イベント、ページ、パラメータなど）
  4. 「適用」をクリックしてファネルを表示

ステップの条件設定

各ステップは、以下のような条件で定義できます。

条件タイプ 例
イベント page_view、add_to_cart、purchase
ページ/スクリーン 特定のページパスを閲覧
パラメータ イベントパラメータの値で絞り込み

ファネルの種類

📍

クローズドファネル

ステップ1から開始したユーザーのみを対象とする。厳密なプロセス分析向け

🌐

オープンファネル

どのステップからでも参加可能。より広い視点での分析向け

クローズド vs オープンの選択
  • クローズド：「TOPページ→商品一覧→商品詳細→購入」という正規ルートを分析したい場合
  • オープン：「商品詳細に直接来た人も含めて、購入までの流れを見たい」場合

ファネル分析の活用例

分析例：ECサイトの購入ファネル

  • ステップ1：商品詳細ページ表示（page_view、ページパス contains /products/）
  • ステップ2：カートに追加（add_to_cart）
  • ステップ3：決済開始（begin_checkout）
  • ステップ4：購入完了（purchase）

着眼点：どのステップで大きく離脱しているかを特定し、その箇所のUI/UX改善を検討

Section 6-4

経路データ探索

経路データ探索は、ユーザーがサイト内でどのような経路をたどっているかを、ツリー形式で可視化する機能です。ナビゲーションの分析やユーザー行動の理解に役立ちます。

経路データ探索の特徴

経路データ探索でわかること
  • 特定のページを見た後、ユーザーはどこに移動するか
  • 特定のイベントの前に、ユーザーはどんな行動をしていたか
  • 想定外の行動パターンはないか
  • 離脱が多い箇所はどこか

経路の方向

経路データ探索では、2つの方向で分析できます。

➡️

始点から（順方向）

特定のページやイベントから始まり、その後ユーザーがどこに行ったかを追跡

例：TOPページを見た後の行動を分析

⬅️

終点から（逆方向）

特定のページやイベントに至るまで、ユーザーがどこから来たかを追跡

例：購入完了に至るまでの経路を分析

経路データ探索の設定

基本的な設定手順

  1. 探索から「経路データ探索」テンプレートを選択
  2. 「始点」または「終点」を選択
  3. 起点となるイベントまたはページを指定
  4. 分析したいノード（分岐点）をクリックして深掘り

ノードタイプの選択

ノードタイプ 用途
イベント名 イベント単位で経路を追跡。行動の種類を見たい場合
ページタイトルとスクリーン名 ページ単位で経路を追跡。ページ遷移を見たい場合
ページパスとスクリーンクラス URLパス単位で追跡。技術的な分析向け
分析のコツ

経路データ探索は情報量が多くなりがちです。以下のコツを意識しましょう。

  • 分析の目的（何を知りたいか）を明確にしてから始める
  • セグメントを使って対象ユーザーを絞り込む
  • 気になるノードをクリックして深掘りする

活用例：キーイベント前の行動分析

分析例：購入ユーザーの行動経路

  • 終点を「purchase」イベントに設定
  • 購入に至るまでにユーザーがどのページを経由したかを可視化
  • よく経由されるページを「成功パターン」として強化
  • あまり経由されないページは導線の見直しを検討
Section 6-5

その他の探索テンプレート

自由形式、ファネル、経路以外にも、特定の分析目的に適した探索テンプレートがあります。

セグメントの重複

セグメントの重複とは

複数のセグメント（ユーザー群）がどの程度重なっているかをベン図で視覚化します。

活用例：

  • 「モバイルユーザー」と「購入者」の重なりを確認
  • 「新規ユーザー」と「高エンゲージメントユーザー」の関係を分析
  • 複数のキャンペーン参加者の重複を把握

コホートデータ探索

コホート分析とは

同じ時期に獲得したユーザー群（コホート）の、その後の行動変化を追跡する分析手法です。

主な用途：リテンション（再訪問率）分析

例：

  • 「1月に初回訪問したユーザー」が、2月、3月とどの程度再訪問しているか
  • キャンペーンで獲得したユーザーの定着率を測定

コホート分析の設定項目

  • コホートへの登録条件：どの条件でコホートにユーザーを含めるか（例：first_visit）
  • リピートの条件：再訪問の定義（例：any_event）
  • コホートの粒度：日別、週別、月別

ユーザーのライフタイム

ライフタイム分析とは

ユーザーがサイトとの関係を通じて生み出す累積的な価値を分析します。

主な用途：

  • LTV（顧客生涯価値）の把握
  • 獲得チャネル別の長期的な価値比較
  • 高価値ユーザーの特性分析

注意：eコマース設定が必要な指標もあります

ユーザーエクスプローラー

ユーザーエクスプローラーとは

個別のユーザー（匿名化されたID単位）の行動履歴を時系列で確認できる機能です。

活用例：

  • 購入者の具体的な行動パターンを理解
  • 問題が発生したユーザーの行動を追跡
  • 理想的なユーザージャーニーの発見
Section 6-6

探索レポートの実践演習

GA4のデモアカウントを使って、探索レポートの実践演習を行いましょう。以下の演習を通じて、探索機能の使い方を体験してください。

演習1：自由形式でクロス集計

課題

「チャネル別・デバイス別のセッション数とエンゲージメント率」を表示する自由形式レポートを作成してください。

手順

  1. 探索 → 空白から新規作成
  2. ディメンション追加：「セッションのデフォルトチャネルグループ」「デバイスカテゴリ」
  3. 指標追加：「セッション」「エンゲージメント率」
  4. 行に「セッションのデフォルトチャネルグループ」を配置
  5. 列に「デバイスカテゴリ」を配置
  6. 値に「セッション」「エンゲージメント率」を配置

確認ポイント

どのチャネル×デバイスの組み合わせが最もエンゲージメント率が高いか確認してください。

演習2：ファネル分析

課題

デモアカウント（Google Merchandise Store）の購入ファネルを作成し、離脱ポイントを特定してください。

手順

  1. 探索 → ファネルデータ探索
  2. ステップを編集して以下を設定：
    • ステップ1：view_item（商品閲覧）
    • ステップ2：add_to_cart（カート追加）
    • ステップ3：begin_checkout（決済開始）
    • ステップ4：purchase（購入完了）
  3. 各ステップ間の進行率・離脱率を確認

確認ポイント

どのステップで最も離脱が多いかを特定し、改善仮説を立ててみてください。

演習3：経路分析

課題

「購入完了」に至るまでのユーザー経路を、逆方向から分析してください。

手順

  1. 探索 → 経路データ探索
  2. 「終点」を選択
  3. 終点のイベントを「purchase」に設定
  4. ノードタイプを「ページタイトルとスクリーン名」に変更
  5. 購入前にどのページが多く経由されているか確認

確認ポイント

購入ユーザーがよく経由するページを「成功パターン」として把握してください。

探索レポートのまとめ

探索レポートは、標準レポートでは得られない深いインサイトを提供する強力な機能です。

  • 自由形式：柔軟なクロス集計と視覚化
  • ファネル：プロセスの離脱ポイントを特定
  • 経路：ユーザーの行動パターンを可視化
  • コホート：ユーザーの継続率を追跡
  • セグメントの重複：ユーザー層の関係性を把握
探索レポート利用時の注意点
  • データ保持期間：探索で使用できるデータは、設定された保持期間内のみ（デフォルト2ヶ月、最大14ヶ月）
  • サンプリング：大量データを扱う場合、サンプリングが発生することがある
  • しきい値：少人数のデータはプライバシー保護のため非表示になることがある
