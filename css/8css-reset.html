Introduction

デフォルトCSSとリセットCSS

同じHTMLを書いたのに、ChromeとFirefoxなど、別々のブラウザで見たら"見た目がちょっと違う"…？
なんてことがよくあります。

それは「ブラウザごとに最初からCSSが入っている」からです。
そんな"ばらつき"をなくすために使うのが 「リセットCSS」 です

Section 8-1

CSSをリセット

ブラウザには最初から「標準スタイル（デフォルトCSS）」が設定されています。

たとえば：

  • タイトルタグ（<h1>）の大きさ
  • 段落（<p>）の余白
  • フォームの見た目

これらはブラウザごとに少しずつ違います。
そのままだと「Chromeで見たときとEdgeで見たときでズレてる!?」なんてことも。

そこで登場するのが リセットCSS
→ デフォルトのCSSを打ち消して、すべてのブラウザで見た目をそろえます。

Section 8-2

リセットCSSの利用方法

リセットCSSは、Web上で無料公開されているものを使うのが一般的です。
ここでは有名な ress.css（レッス）を使います。

公開元：
https://github.com/filipelinhares/ress

そうすると、h1やpタグなどに最初から効いているCSSのほとんどが、
程よい感じに削除されます。

Section 8-3

HTMLに読み込む方法

HTMLの<head>内に、以下のように書くだけ！

フォルダ構成

フォルダ構成 
Chapter8_2-2
 └── index.html
 └── style.css
HTML 
<!DOCTYPE html>
<html lang="ja">
<head>
	<meta charset="UTF-8" />
	<title>CSSリセットサンプル</title>

	<!-- リセットCSS（外部読み込み） -->
	<link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">

	 <!-- 自分のCSS -->
	<link rel="stylesheet" href="style.css" />
</head>
<body>
	<h1>お問い合わせ</h1>
	<form>
		<label>お名前</label><br>
		<input type="text"><br><br>

		<label>ご意見・ご質問</label><br>
		<textarea></textarea><br><br>

		<button>送信</button>
	</form>
</body>
</html>

POINT：読み込み順が超重要！

ress.cssを 先に読み込む → その後に自作のCSSを読み込む。
逆にしてしまうと、自分のCSSがリセットCSSによって初期化されてしまいます。

間違った例（やってはいけない順番）

HTML（間違い） 
<!-- 間違い -->
<link rel="stylesheet" href="style.css" />
<link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">

「リセットは最初、デザインはあとから」が鉄則です！

自作CSS（style.css）の例

CSS 
body {
  font-family: "Hiragino Sans", sans-serif;
  line-height: 1.6;
  margin: 40px;
}

h1 {
  color: #0bd;
  border-bottom: 3px solid #0bd;
  padding-bottom: 8px;
}

input, textarea {
  border: 2px solid #ccc;
  border-radius: 4px;
  padding: 8px;
  width: 300px;
}

button {
  background: #0bd;
  color: #fff;
  border: none;
  padding: 10px 20px;
  border-radius: 4px;
  cursor: pointer;
}

「ress.css」を使うとこうなる！

比較項目 リセット前 リセット後
入力欄 長さがバラバラ 全ブラウザで統一
テキストの余白 ブラウザごとに違う すべて同じ
見出しの余白 ブラウザによって大きすぎることも スッキリ均一に
Section 8-4

よくある疑問Q&A

Q1. リセットCSSって必ず必要？

→ 小規模なページでは不要なこともありますが、複数ブラウザ対応なら必須級です。

Q2. どのリセットCSSを使えばいいの？

→ 有名どころは以下の3つ：

  • ress.css（デフォルトの差を最小化）
  • normalize.css（標準に合わせて整える）
  • sanitize.css（セキュリティ考慮の新しいタイプ）

Q3. オフライン環境で使いたい場合は？

→ あらかじめネットからファイルをダウンロードして css/ress.min.css として保存し、
<link>タグでそのファイルパスを読み込ませます。
