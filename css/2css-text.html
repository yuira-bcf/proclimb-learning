Section 2-1

文字の大きさを変える「font-size プロパティ」

文字の大きさを指定するプロパティが font-size です。
大きさは 数値＋単位（px、rem、%）で指定します。
キーワード（small、largeなど） でも指定出来ますが、使用することは稀です。

指定方法

指定方法 説明
数値指定 px（ピクセル）やrem（ルート基準）、%（割合）などの単位を使う
キーワード指定 small、medium、large などの7段階キーワードで指定できる

フォルダ構成

フォルダ構成 
Chapter2_1-2
 └── index.html
 └── style.css
CSS 
@charset "UTF-8";

html {
	font-size: 100%; /* ページ全体の基準サイズ */
}

h1 {
	font-size: 2rem; /* 基準の2倍サイズ */
}

p {
	font-size: 18px; /* 絶対値指定（18ピクセル） */
}

POINT

<html>タグにもスタイルを指定できます。ここに指定したものは、そのページの基準値となります。
html { font-size:100%; }とすると、原則ブラウザ標準の16pxという単位が基準になります。
「rem」という単位は、このhtmlの値を基準にして、大きさを決定します。

HTML 
<!doctype html>
<html lang="ja">
<head>
	<meta charset="UTF-8">
	<meta name="description" content="Café Latteのメニュー紹介ページ">
	<title>Café Latte｜メニュー紹介</title>

	<!-- CSSファイルを読み込む -->
	<link rel="stylesheet" href="style.css">
</head>

<body>
	<h1>Café Latteへようこそ！</h1>
	<p>本日のおすすめは「キャラメルラテ」と「抹茶モカ」です。</p>
	 <p>香ばしいエスプレッソとふわふわのフォームミルクで、
	心まであたたまるひとときをお楽しみください。</p>
</body>
</html>

適切な文字サイズの考え方

本文：14〜18px が一般的（読みやすさ重視）
見出し：本文の約1.5〜2.5倍

デザインの統一感：サイズの種類は2〜5段階以内にまとめる

POINT！

文字サイズがバラバラだとページが雑多に見えます。
使用サイズをあらかじめ決めておくと、統一感が出て見やすくなります

Section 2-2

文字の太さを変える「font-weight プロパティ」

文字の太さを指定するプロパティです。

値 説明
normal 標準の太さ
bold 太字
lighter / bolder 相対的に細く／太く
数値（100〜900） より細かく太さを指定（例：font-weight: 700;）

フォルダ構成

フォルダ構成 
Chapter2_1-5
 └── index.html
 └── style.css
CSS 
@charset "UTF-8";

h1 {
    font-weight: bold;
    /* 太めの指定 */
}

/* p {
    font-weight: 400;
} */

/* おまけの指定 */
body :nth-child(2) {
    /* bodyの中の、2番目の要素を指定 */
    font-weight: 700;
    /* 太めの指定 */
}

body :nth-child(3) {
    /* bodyの中の、3番目の要素を指定 */
    font-weight: 400;
    /* 標準の指定 */
}

body :nth-child(4) {
    /* bodyの中の、4番目の要素を指定 */
    font-weight: 100;
    /* 細めの指定 */
}
HTML 
<!doctype html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="description" content="Café Latteのメニュー紹介ページ">
  <title>Café Latte｜メニュー紹介</title>

  <!-- CSSファイルを読み込む -->
  <link rel="stylesheet" href="style.css">

<body>

  <p>ようこそCafé Latteへ！こだわりの豆と、やさしいミルクの香りが広がる癒しの一杯をどうぞ。</p>

  香ばしいキャラメルの甘みが、エスプレッソの苦味と絶妙にマッチします。</p>

  <p>※ 当店では季節ごとに限定ブレンドをご用意しています。お楽しみに！</p>

POINT！

boldは「より目立たせたい部分」に最適
数値指定はフォントによって効かない場合もあります。

表示イメージ

要素 指定 見た目
<h1> bold 強調された太字タイトル
<p> 400 読みやすい標準の本文

見出しとジャンプ率を考えよう

ジャンプ率とは「見出し」と「本文」の文字サイズの比率のこと。
ジャンプ率が高いとメリハリがあり、低いと落ち着いた印象になります。

ジャンプ率 雰囲気
高い（差が大） 活発・ポップ
低い（差が小） 落ち着いた・上品

POINT！

Webサイトのテーマに合わせて文字の"強弱"をコントロールすると、
読みやすくデザイン性の高いページに仕上がります

まとめ

項目 プロパティ名 説明
文字の大きさ font-size 単位は px・rem・% など
フォントの種類 font-family 明朝体・ゴシック体などの書体を指定
文字の太さ font-weight 太字・細字・数値指定
ジャンプ率 （比率の考え方） 見出しと本文のサイズ比を意識する
Section 2-3

文字を読みやすくするコツ

行の高さを整える line-height

line-height とは

1行ぶんの高さ（行間を含む高さ）を決めるプロパティ。
基本は単位なしの倍率を利用した指定をする。

指定方法と意味

指定 例 特徴
単位なし（推奨） 1.6, 1.8 文字サイズ*倍率が1行の高さになる
px / rem / % 28px, 1.6rem, 160% 絶対／相対を固定値で指定、後々の調整が面倒くさくなりやすい

実務では 1.5〜1.8 のレンジが"読みやすい黄金ゾーン"。
日本語の本文は字面が詰まりやすいので 1.6〜1.8 から試すのがおすすめ。

見出しと本文のバランス

見出し（h1〜h3）はサイズを大きくするだけでなく、line-heightをやや小さめ（例：1.2〜1.4）にすると締まって見える。
本文（p）は大きめ（例：1.6〜1.8）で余裕を持たせ、段落のブロック感を出す。

フォルダ構成

フォルダ構成 
Chapter2_2-3
└── index.html
└── style.css
HTML 
<!doctype html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="description" content="Café Latteの読みやすいタイポグラフィ実験ページ">
  <title>Café Latte｜読みやすい文字組み</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header>
    <h1>Café Latte</h1>
    <p class="lead"><strong>太字は"ここぞ"で。</strong>
		本文は行間をしっかり取り、やさしく読み進められる紙面を目指します。</p>
  </header>

  <main class="content">
    <section>
      <h2>本日のおすすめ</h2>
      <p>
        キャラメルラテはエスプレッソのほろ苦さと香ばしい甘さが調和した人気メニュー。
        <strong>季節限定ブレンド</strong>は軽やかな香り立ちで、午後のひと休みにぴったりです。
      </p>
    </section>

    <section>
      <h2>読みやすさのレシピ</h2>
      <p>
        見出しにはやや小さめの行間、本文には大きめの行間を設定します。
        文字は通常の太さを基準にし、強調したい語だけ<strong>ピンポイントで太字</strong>に。
        これだけでページのリズムが生まれ、ストレスなく読み進められます。
      </p>
    </section>
  </main>

  <footer>
    <p class="note">※ 本ページはデモです。実際のメニュー価格とは異なる場合があります。</p>
  </footer>
</body>
</html>
CSS 
@charset "UTF-8";

/* ベース */
html { font-size: 100%; }
body {
  font-family: "Hiragino Sans", "Yu Gothic", sans-serif;
  color: #333;
  margin: 40px 20px;
  background: #fffdf8;
}

/* 見出し：サイズは大きめ、行間はややタイト */
h1 {
  font-size: 2rem;
  line-height: 1.25;     /* ぎゅっと締める */
  color: #6b3e26;        /* カフェブラウン */
  margin: 0 0 12px;
}
h2 {
  font-size: 1.25rem;
  line-height: 1.35;
  color: #6b3e26;
  margin: 28px 0 8px;
}

/* 本文：行間を広めにして読みやすく */
p {
  font-size: 1rem;       /* ≒16px（ブラウザ既定） */
  line-height: 1.7;      /* おすすめゾーン */
  margin: 0 0 16px;
}

/* リード文は少し大きく */
.lead { font-size: 1.0625rem; line-height: 1.8; }

/* 注記など小さめの文は、それでも行間は確保 */
.note { font-size: .875rem; line-height: 1.6; color: #666; }

/* 強調：太字はピンポイント！ */
strong { font-weight: 700; }
Section 2-4

文章を揃えよう：text-align プロパティ

text-align とは？

段落や見出しなどインライン要素の整列位置（左右・中央・両端）を決めるプロパティ。
日本語の段落は 既定で左揃え（left） です。

指定値 こうなる 使いどころ
left 左揃え 基本の段落。読みやすさ重視
right 右揃え 署名・日付・価格表の金額末尾など
center 中央揃え 見出し・短いキャッチコピー
justify 両端揃え 段落の左右端をピタッと揃える（長文の紙面感を出したいとき）

※ 日本語の justify は字間の伸縮で調整されるため、ブラウザやフォントによっては不自然に感じることがあります。
印象を見て採用判断しましょう。

中央揃えは"短文だけ"が鉄則

中央揃えは行頭の位置が毎行ズレるため、長文では読みにくい。
2〜3行程度の短い文・見出しには効果的。
記事本文や説明文などでの利用は、理由がない限り極力避けましょう。

CSS 
@charset "UTF-8";

/* ベース */
html { font-size: 100%; }
body {
  font-family: "Hiragino Sans", "Yu Gothic", sans-serif;
  color: #333;
  margin: 40px 20px;
  background: #fffdf8;
}

/* 見出しとレイアウト */
h1 { color: #6b3e26; margin: 0 0 8px; line-height: 1.25; }
h2 { color: #6b3e26; margin: 24px 0 8px; line-height: 1.3; }
p  { font-size: 1rem; line-height: 1.7; margin: 0 0 14px; }

/* text-align デモ */
.left    { text-align: left; }     /* 本文の基本 */
.center  { text-align: center; }   /* 見出し・短文 */
.right   { text-align: right; }    /* 価格列など */
.justify { text-align: justify; text-justify: inter-ideograph; } /* 両端揃え（任意） */

/* 価格表：2列グリッドで桁揃え 詳細は後述します*/
.price {
  display: grid;
  grid-template-columns: 1fr auto;
  column-gap: 16px;
  align-items: baseline;
}

/* 装飾プラス */
.hero .catch { color: #6b3e26; opacity: .9; }
.note { color: #666; font-size: .875rem; }
