8-1

演習概要

8-1-1. この演習の目的

第7章のTodoアプリでは完成版の設計書を提供しましたが、本章では自分で設計書を書くところから始めます。ECサイトの「商品管理機能」を題材に、6種類の設計書テンプレートを埋めてプロンプトを作成し、コードを生成する応用演習です。

warning 注意: この演習は自力で取り組む演習です

第7章と異なり、完成版の設計書は提供しません。ヒントとチェックリストを参考にしながら、自分で設計書を作成してください。分からない場合は各章のテンプレートと記入例を参照してください。

8-1-2. 演習の題材: ECサイト商品管理

簡易的なECサイトの商品管理機能（管理者向け）を開発します。

shopping_cart ECサイト商品管理の概要

作るもの

  • 商品の一覧表示
  • 商品の登録（名前・価格・説明・カテゴリ）
  • 商品の編集
  • 商品の削除
  • カテゴリ別フィルタ

作らないもの

  • ユーザー認証
  • カート機能
  • 注文・決済
  • 画像アップロード
  • 在庫管理

要件定義（自分で作成）

8-2-1. 要件定義書を書こう

templates/requirements-template.md をコピーして、ECサイト商品管理の要件定義書を作成してください。

tips_and_updates ヒント: 機能要件

以下の機能をFR-001〜FR-006として定義してみましょう。商品追加 / 商品一覧 / 商品詳細 / 商品編集 / 商品削除 / カテゴリフィルタ

tips_and_updates ヒント: 技術スタック

8-3-2. 設計のヒント

tips_and_updates ヒント: エンドポイント設計

Todoアプリの/todosを/productsに置き換えて考えましょう。カテゴリフィルタはGET /products?category=食品のようにクエリパラメータで実現できます。

tips_and_updates ヒント: DB設計

productsテーブルには、id / name / price / description / category / on_sale / created_at / updated_at のカラムが必要です。priceはINTEGER（0以上）、categoryはVARCHAR、on_saleはBOOLEANで定義します。

tips_and_updates ヒント: バリデーション

商品名: @NotBlank + @Size(max=200) / 価格: @NotNull + @Min(0) + @Max(9999999) / カテゴリ: @NotBlank

tips_and_updates ヒント: カテゴリ選択

カテゴリはselectタグ（プルダウン）で選択させます。選択肢は固定値（食品/日用品/家電/書籍/その他）で、Enum定義するか、文字列リストとしてControllerから渡します。

プロンプト作成ヒント

8-4-1. プロンプトの構成

第7章と同じ3段階のプロンプトで作成します。

smart_toy 3段階プロンプト構成

プロンプト1: Entity + Repository

DB設計書のテーブル定義を貼り付けて、ProductエンティティとProductRepositoryを生成

プロンプト2: Service + Controller + Form

API設計書のエンドポイント一覧 + バリデーションルールを貼り付けて、ProductService / ProductController / ProductFormを生成

プロンプト3: Thymeleafテンプレート

画面設計書の入力項目定義 + レイアウト概要を貼り付けて、各HTMLテンプレートを生成

8-4-2. Todoアプリとの違い

ECサイトの商品管理はTodoアプリより以下の点が複雑です。プロンプトにこれらの要件を含めることを忘れないでください。

追加要素 Todoアプリ ECサイト商品管理
数値入力 なし 価格（Integer, @Min(0)）
プルダウン選択 なし カテゴリ（select要素）
フィルタ機能 なし カテゴリ別フィルタ
金額表示 なし 価格のフォーマット（カンマ区切り + 円）

8-6

発展課題

8-6-1. チャレンジ課題

基本の商品管理機能が完成したら、以下の発展課題に挑戦してみましょう。各課題について、まず設計書を書いてからプロンプトを作成してください。

Level 1 商品検索機能

商品名でキーワード検索できるようにする。一覧画面に検索フォームを追加し、LIKEクエリで部分一致検索を実装する。

Level 2 価格帯フィルタ

価格の範囲（〜1,000円 / 1,000〜5,000円 / 5,000円〜）でフィルタできるようにする。クエリパラメータで最小価格・最大価格を指定する。

Level 3 カテゴリマスタ分離

カテゴリを固定文字列ではなく、別テーブル（categories）で管理する。productsテーブルにcategory_id外部キーを追加し、1:Nのリレーションを設計する。

8-6-2. 講座のまとめ

celebration 講座完了おめでとうございます!

全8章を通じて、以下のスキルを身につけました。

  • 6種類の設計書（要件定義・機能仕様・API設計・DB設計・画面設計・テスト仕様）の書き方
  • Markdownテンプレートを使った効率的な設計書作成
  • 設計書をAIプロンプトに変換する手法
  • 設計書駆動でSpring Bootアプリケーションを開発する一連の流れ

これからの開発では、コードを書く前にまず設計書を書く習慣をつけてください。設計書があれば、AIの力を最大限に引き出すことができます。
