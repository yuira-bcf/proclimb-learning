5-1

DB設計書の構成

5-1-1. DB設計書とは

DB設計書は、データベースの構造（テーブル・カラム・リレーション）を定義するドキュメントです。どのデータをどのような型でどのテーブルに保存するかを正確に記述します。

5-1-2. DB設計書の主要セクション

storage DB設計書の構成要素

1 テーブル一覧 システムで使う全テーブルの概要
2 テーブル定義 カラム名・型・NULL可否・制約
3 ER図 テーブル間のリレーション
4 初期データ シードデータ・マスタデータ
5-2

テーブル定義書

5-2-1. テーブル定義の書き方

テーブル定義は、各カラムのカラム名・論理名・型・長さ・NULL可否・デフォルト値・主キーを表で定義します。

table_view todosテーブル定義

# カラム名 論理名 型 長さ NULL デフォルト PK 備考
1 id ID BIGINT - NO AUTO_INCREMENT YES 主キー
2 title タイトル VARCHAR 100 NO - - タスク名
3 description 説明 TEXT - YES NULL - 詳細説明
4 completed 完了フラグ BOOLEAN - NO false - 完了状態
5 created_at 作成日時 DATETIME - NO CURRENT_TIMESTAMP - 自動設定
6 updated_at 更新日時 DATETIME - NO CURRENT_TIMESTAMP - 自動更新

5-2-2. データ型の選び方

lightbulb よく使うデータ型
  • BIGINT: 主キー（IDカラム）に使用
  • VARCHAR(N): 文字数制限がある文字列（タイトル、名前など）
  • TEXT: 長い文字列（説明文、本文など）
  • BOOLEAN: true/falseの二値（フラグ系）
  • DATETIME: 日時情報（作成日時、更新日時）
  • INTEGER: 数値（数量、順序など）
  • DATE: 日付のみ（期限日など）
5-3

ER図の記述

5-3-1. ER図とは

ER図（Entity-Relationship Diagram）は、テーブル間の関係を視覚的に表したものです。Markdownではテキストベースで簡易的に記述します。

hub Todoアプリ ER図（単一テーブル）

todos
PK id : BIGINT
    title : VARCHAR(100)
    description : TEXT
    completed : BOOLEAN
    created_at : DATETIME
    updated_at : DATETIME

5-3-2. 複数テーブルのリレーション

テーブルが複数ある場合は、外部キー（FK）でリレーションを表現します。例えば、ユーザーとタスクの1対多（1:N）の関係は以下のようになります。

hub ER図: users と todos（1:N）

users
PK id
    username
    email
1 : N
------→
user_id FK
todos
PK id
FK user_id
    title
    completed
Markdownでの記述例 
## ER図

+-------------------+        +-------------------+
| users             |        | todos             |
+-------------------+        +-------------------+
| PK id             |<----+  | PK id             |
|    username        |     +--| FK user_id        |
|    email           |        |    title           |
|    created_at      |        |    completed       |
+-------------------+        |    created_at      |
        1                     +-------------------+
                                      N

### リレーション
| 親テーブル | 子テーブル | 関係 | 外部キー | 削除時 |
|-----------|-----------|------|---------|--------|
| users     | todos     | 1:N  | user_id | CASCADE |
5-4

初期データ定義

5-4-1. 初期データ（シードデータ）

開発時やテスト時に投入するサンプルデータを定義します。Spring Bootではdata.sqlファイルに記述します。

data.sql（初期データ） 
INSERT INTO todos (title, description, completed, created_at, updated_at)
VALUES
  ('買い物に行く', '牛乳とパンを買う', false, NOW(), NOW()),
  ('レポートを書く', '今週の進捗レポート', false, NOW(), NOW()),
  ('部屋の掃除', NULL, true, NOW(), NOW());
lightbulb 初期データのポイント
  • 正常なデータだけでなく、NULL値を含むデータも入れる（description: NULL）
  • completedがtrue/false両方のデータを入れる
  • 日本語の文字化けテスト用に日本語データを含める
5-5

テンプレートからプロンプトへ

5-5-1. DB設計書からJPA Entityへ

テーブル定義書は、JPA Entityクラスの生成プロンプトにそのまま使えます。

smart_toy DB設計書からEntityを生成するプロンプト

以下のDB設計書に基づいて、Spring Data JPAのEntityクラスを作成してください。

## テーブル定義: todos
| カラム名 | 型 | NULL | デフォルト | 備考 |
|---------|-----|------|-----------|------|
| id | BIGINT | NO | AUTO_INCREMENT | PK |
| title | VARCHAR(100) | NO | - | タスク名 |
| description | TEXT | YES | NULL | 詳細説明 |
| completed | BOOLEAN | NO | false | 完了フラグ |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 作成日時 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP | 更新日時 |

要件:
- @Entity, @Id, @GeneratedValue アノテーション使用
- created_at/updated_at は @CreationTimestamp, @UpdateTimestamp
- Lombokは使わず、getter/setterを明示的に記述
- パッケージ: com.example.todo.entity

5-5-2. 生成されるEntityのイメージ

Todo.java（生成イメージ） 
package com.example.todo.entity;

import jakarta.persistence.*;
import org.hibernate.annotations.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "todos")
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String title;

    @Column(columnDefinition = "TEXT")
    private String description;

    @Column(nullable = false)
    private Boolean completed = false;

    @CreationTimestamp
    private LocalDateTime createdAt;

    @UpdateTimestamp
    private LocalDateTime updatedAt;

    // getter/setter...
}
check_circle テーブル定義とEntityの対応

テーブル定義の各行がEntityのフィールドに、型がJavaの型に、制約がJPAアノテーションにそれぞれ変換されています。設計書があれば、この変換をAIが正確に行えます。

edit_note

練習問題

1

以下のDB型とJavaの型の対応を答えてください。(a) BIGINT (b) VARCHAR(100) (c) TEXT (d) BOOLEAN (e) DATETIME

2

「メモ帳アプリ」のmemosテーブル定義を作成してください。カラム名・型・NULL可否・デフォルト値を含めてください。

3

memosテーブルの初期データ（INSERT文）を3件分書いてください。NULLを含むデータも入れてください。

解答を表示
問1の解答

(a) BIGINT → Long (b) VARCHAR → String (c) TEXT → String (d) BOOLEAN → Boolean (e) DATETIME → LocalDateTime

問2の解答例

| id | BIGINT | NO | AUTO_INCREMENT | PK |

| title | VARCHAR(100) | NO | - | メモタイトル |

| body | TEXT | NO | - | メモ本文 |

| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 作成日時 |

| updated_at | DATETIME | NO | CURRENT_TIMESTAMP | 更新日時 |

問3の解答例

INSERT INTO memos (title, body, created_at, updated_at) VALUES ('会議メモ', '来週の予定を確認', NOW(), NOW()), ('買い物リスト', '牛乳、卵、パン', NOW(), NOW()), ('アイデアメモ', NULL, NOW(), NOW());

※3件目のbodyがNULLになっています（NULLを含むデータの例）。ただし、bodyをNOT NULLにした場合はNULLではなく空文字を使います。
