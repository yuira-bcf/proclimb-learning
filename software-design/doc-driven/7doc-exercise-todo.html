# 要件定義書: Todoアプリ

## 1. プロジェクト概要
### 1.1 背景・目的
個人のタスク管理を効率化するWebアプリケーション。
日々のやるべきことを登録・管理し、完了状況を把握する。

### 1.2 対象ユーザー
- 一般ユーザー: タスクを管理したい個人

### 1.3 スコープ
#### 対象範囲
- タスクのCRUD（作成・読取・更新・削除）
- タスクの完了/未完了切り替え

#### 対象外
- ユーザー認証（ログイン機能）
- 複数ユーザー対応
- カテゴリ・タグ機能

## 2. 機能要件
| ID | 機能名 | 説明 | 優先度 |
|----|--------|------|--------|
| FR-001 | タスク追加 | タイトルと説明を入力して新規作成 | 高 |
| FR-002 | タスク一覧 | 全タスクを一覧表示 | 高 |
| FR-003 | タスク詳細 | タスクの詳細を表示 | 高 |
| FR-004 | タスク編集 | タイトル・説明を編集 | 中 |
| FR-005 | タスク削除 | 確認後にタスクを削除 | 高 |
| FR-006 | 完了切替 | 完了/未完了を切り替え | 高 |

## 3. 非機能要件
- DB: H2（インメモリ）
- 対応ブラウザ: Chrome最新版
- 文字コード: UTF-8

## 4. 技術スタック
| レイヤー | 技術 |
|----------|------|
| バックエンド | Spring Boot 3.4 |
| テンプレート | Thymeleaf |
| DB | H2 Database |
| CSS | Bootstrap 5 |
| ビルド | Gradle |

## 5. 受け入れ基準
- [ ] 全CRUD操作が動作すること
- [ ] バリデーションが正しく動作すること
- [ ] 日本語が文字化けしないこと

設計書からプロンプト作成

7-5-1. プロンプト1: Entity + Repository

smart_toy プロンプト1: Entity + Repository生成

以下のDB設計書に基づいて、Spring Data JPAのEntityクラスとRepositoryインターフェースを作成してください。

## テーブル定義: todos
| カラム名 | 型 | NULL | デフォルト | 備考 |
|---------|-----|------|-----------|------|
| id | BIGINT | NO | AUTO_INCREMENT | PK |
| title | VARCHAR(100) | NO | - | タスク名 |
| description | TEXT | YES | NULL | 詳細説明 |
| completed | BOOLEAN | NO | false | 完了フラグ |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 自動設定 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP | 自動更新 |

## 要件
- パッケージ: com.example.todo
- @Entity, @Id, @GeneratedValue使用
- @CreationTimestamp, @UpdateTimestamp使用
- Lombokは使わない（getter/setter明示）
- JpaRepositoryを継承したTodoRepositoryも作成

7-5-2. プロンプト2: Service + Controller

smart_toy プロンプト2: Service + Controller生成

以下の設計書に基づいて、TodoServiceクラスとTodoControllerクラスを作成してください。

## エンドポイント
| メソッド | パス | 概要 | Controllerメソッド |
|---------|------|------|------------------|
| GET | /todos | 一覧 | list() |

| POST | /todos/{id}/delete | 削除 | delete() |
| POST | /todos/{id}/toggle | 完了切替 | toggleComplete() |

## バリデーション
- title: @NotBlank, @Size(max=100)
- description: @Size(max=500)
- エラー時はフォーム再表示（入力値保持）

## 要件
- TodoServiceはTodoRepositoryに依存
- コンストラクタインジェクション使用
- POST成功後はredirect:/todosでリダイレクト
- TodoFormクラスも別途作成
