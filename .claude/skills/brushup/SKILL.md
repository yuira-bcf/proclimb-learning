# /brushup - 章ファイル一括修正

指定した章ファイルのサブセクション番号、Material Icons、練習問題を一括で検証・修正します。

## 使い方
```
/brushup [ファイルパス or ディレクトリ]
```

例:
- `/brushup backend/java/1java-overview.html` — 単一ファイル
- `/brushup backend/java/` — ディレクトリ内の全章ファイル
- `/brushup software-design/` — カテゴリ全体

## 検証・修正項目

### 1. サブセクション番号
- `<h3>` のサブセクション番号が `N-S-X` 形式であることを確認
- 連番が飛んでいないか確認
- 旧形式（`1.1`, `10.2.3` 等）を新形式に変換

### 2. Material Icons
- `<link>` で Material Icons CSS が読み込まれていることを確認
- 不足している場合は追加

### 3. 練習問題
- 章末に練習問題が3問あることを確認
- `exercise-list` / `answer-list` パターンが正しいことを確認
- 練習問題の内容が章のトピックに合っていることを確認（不一致の場合は差し替え）

### 4. 共通CSS読み込み
- `../../shared/common.css` が読み込まれていることを確認

## 出力形式
修正前後の差分サマリーをテーブルで出力する:
```
| ファイル | サブセクション番号 | Material Icons | 練習問題 | CSS |
|---------|------------------|----------------|---------|-----|
| 1java-overview.html | OK | 追加 | 3問OK | OK |
```
