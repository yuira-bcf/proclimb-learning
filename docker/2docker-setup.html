Section 2-1

Docker Desktopとは

Docker Desktop は、Windows や macOS 上で Docker を簡単に使えるようにするデスクトップアプリケーションです。Docker Engine、Docker CLI、Docker Compose、Kubernetes などの主要ツールがすべて同梱されており、インストールするだけで開発環境が整います。

🛠 Docker Desktop のアーキテクチャ
🖥 Docker Desktop（GUIアプリケーション） ダッシュボード・設定画面・ログ閲覧・イメージ管理
⚙ Docker Engine コンテナ実行
💻 Docker CLI コマンド操作
🎼 Docker Compose 複数コンテナ管理
☵ Kubernetes オーケストレーション
💾 軽量Linux VM（HyperKit / WSL 2） Windows / macOS 上で Linux カーネルを提供する仮想化レイヤー

Docker Desktop の主な機能

  • GUI ダッシュボード：実行中のコンテナやイメージの一覧をグラフィカルに管理
  • リソース管理：CPU・メモリ・ディスクの割り当て量を設定画面から調整
  • Kubernetes 統合：ローカル Kubernetes クラスタをワンクリックで有効化
  • 拡張機能（Extensions）：セキュリティスキャン、ログ管理などのプラグイン
  • Docker Scout：イメージの脆弱性スキャンと推奨事項の表示

ライセンスについて

注意：有料化の対象

Docker Desktop は、従業員250人以上または年間収益1,000万ドル以上の企業で利用する場合、有料のサブスクリプションが必要です。個人利用・教育目的・小規模企業・オープンソースプロジェクトでは引き続き無料で利用できます。

Section 2-2

Windowsへのインストール

Windows に Docker Desktop をインストールするには、WSL 2（Windows Subsystem for Linux 2）が必要です。WSL 2 は Windows 上で軽量な Linux カーネルを動かす仕組みで、Docker のバックエンドとして使用されます。

システム要件

  • Windows 10 バージョン 2004 以降、または Windows 11
  • 64ビットプロセッサ（SLAT対応）
  • 4GB以上のRAM（8GB以上推奨）
  • BIOS で仮想化が有効化されていること
  • WSL 2 がインストール済みであること

インストール手順

Step 1：WSL 2 の有効化

PowerShell（管理者として実行） PowerShell 
# WSL をインストール（WSL 2 がデフォルト）
wsl --install

# インストール済みの場合はバージョン確認
wsl --version

# 再起動後、WSL 2 をデフォルトに設定
wsl --set-default-version 2

Step 2：Docker Desktop のダウンロード

  1. Docker公式サイトにアクセス
  2. 「Download for Windows」をクリック
  3. ダウンロードされた Docker Desktop Installer.exe を実行

Step 3：インストーラーの実行

  1. 「Use WSL 2 instead of Hyper-V」にチェックが入っていることを確認
  2. 「Add shortcut to desktop」を選択（任意）
  3. 「Ok」をクリックしてインストール開始
  4. 完了後、PCを再起動
POINT

Windows では WSL 2 バックエンドを使用することで、Hyper-V を使う方式よりもパフォーマンスが大幅に向上します。特にファイルI/Oの速度が改善されるため、WSL 2 の使用を強く推奨します。

Section 2-3

macOSへのインストール

macOS への Docker Desktop インストールは比較的シンプルです。Intel チップ搭載 Mac と Apple Silicon（M1/M2/M3/M4）搭載 Mac でインストーラーが異なる点に注意しましょう。

システム要件

  • macOS 12（Monterey）以降
  • 4GB以上のRAM（8GB以上推奨）
  • Apple Silicon（M1以降）または Intel プロセッサ

インストール手順

方法1：公式サイトからダウンロード

  1. Docker公式サイトにアクセス
  2. Mac のチップに合ったインストーラーをダウンロード
    • Apple Silicon（M1/M2/M3/M4）: 「Mac with Apple chip」
    • Intel: 「Mac with Intel chip」
  3. Docker.dmg をダブルクリック
  4. Docker アイコンを Applications フォルダにドラッグ
  5. アプリケーションから Docker を起動

方法2：Homebrew を使う（推奨）

Homebrew でのインストール Bash 
# Homebrew がインストール済みか確認
brew --version

# Docker Desktop をインストール
brew install --cask docker

# インストール完了後、アプリケーションから Docker を起動
POINT

Apple Silicon Mac では、ARM64 ネイティブの Docker イメージを使うとパフォーマンスが最適です。x86_64 向けイメージも --platform linux/amd64 オプションで動作しますが、エミュレーションによる速度低下があります。

Section 2-4

Linuxへのインストール

Linux では Docker Desktop を使う方法と、Docker Engine を直接インストールする方法があります。サーバー用途では Docker Engine のみをインストールするのが一般的です。ここでは Ubuntu を例に解説します。

Docker Engine のインストール（Ubuntu）

Step 1：古いバージョンの削除 Bash 
# 古い Docker パッケージを削除
sudo apt-get remove docker docker-engine docker.io containerd runc
Step 2：リポジトリの設定 Bash 
# 必要なパッケージをインストール
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

# Docker の公式 GPG キーを追加
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# リポジトリを追加
echo \
  "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
Step 3：Docker Engine のインストール Bash 
# パッケージ一覧を更新
sudo apt-get update

# Docker Engine をインストール
sudo apt-get install docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# sudo なしで docker コマンドを使えるようにする
sudo usermod -aG docker $USER

# ログアウトして再ログインすると設定が反映される
注意

usermod コマンドでユーザーを docker グループに追加した後は、一度ログアウトして再ログインする必要があります。これを忘れると docker コマンド実行時に「permission denied」エラーが発生します。

Section 2-5

インストール確認とHello World

Docker のインストールが完了したら、正しく動作するか確認しましょう。Docker には公式の動作確認用イメージ hello-world が用意されています。

バージョン確認

Docker のバージョン確認 Bash 
# Docker のバージョンを確認
$ docker --version
Docker version 27.4.1, build b9d17ea

# 詳細な情報を表示
$ docker version
Client:
 Version:           27.4.1
 API version:       1.47
 Go version:        go1.22.10
 ...

# Docker Compose のバージョン確認
$ docker compose version
Docker Compose version v2.32.1

Hello World の実行

追加の動作確認

Nginx コンテナで Web サーバーを起動 Bash 
# Nginx コンテナをバックグラウンドで起動
$ docker run -d -p 8080:80 --name my-nginx nginx

# ブラウザで http://localhost:8080 にアクセスして確認

# 実行中のコンテナを確認
$ docker ps
CONTAINER ID   IMAGE   COMMAND                  PORTS                  NAMES
a1b2c3d4e5f6   nginx   "/docker-entrypoint.…"   0.0.0.0:8080->80/tcp   my-nginx

# コンテナを停止・削除
$ docker stop my-nginx
$ docker rm my-nginx
POINT

-d はバックグラウンド実行、-p 8080:80 はポートマッピング（ホストの8080番 → コンテナの80番）、--name はコンテナに名前を付けるオプションです。これらは今後頻繁に使うオプションです。

2章 練習問題

Q1

Windows で Docker Desktop を使用する際に推奨されるバックエンドはどれですか？

  1. Hyper-V
  2. VirtualBox
  3. WSL 2（Windows Subsystem for Linux 2）
  4. VMware
Q2

docker run hello-world を実行したとき、ローカルにイメージがない場合どうなりますか？

  1. エラーが発生して終了する
  2. 空のコンテナが作成される
  3. Docker Hub から自動的にイメージがダウンロードされる
  4. 手動でイメージをビルドするよう求められる
Q3

docker run -d -p 8080:80 nginx コマンドの -p 8080:80 が意味することとして正しいものはどれですか？

  1. コンテナの8080番ポートをホストの80番ポートにマッピングする
  2. ホストの8080番ポートをコンテナの80番ポートにマッピングする
  3. コンテナ内で8080番と80番の両方のポートを開放する
  4. ホスト側で8080番から80番までの範囲のポートを使用する
Answer

2章 解説・解答

Q1の解答: C

WSL 2（Windows Subsystem for Linux 2）は、Windows 上で軽量な Linux カーネルを実行する仕組みです。Docker Desktop は WSL 2 をバックエンドとして使用することで、Hyper-V を使う方式よりもファイルI/Oの速度やメモリ効率が大幅に改善されます。

Q2の解答: C

Docker は docker run 実行時にローカルにイメージが見つからない場合、自動的に Docker Hub（デフォルトのレジストリ）からイメージをダウンロード（pull）します。これは出力の「Unable to find image locally」「Pulling from library/hello-world」というメッセージで確認できます。

Q3の解答: B

-p オプションのフォーマットは ホスト側ポート:コンテナ側ポート です。-p 8080:80 は「ホストの8080番ポートへのアクセスを、コンテナの80番ポートに転送する」という意味です。ブラウザで http://localhost:8080 にアクセスすると、コンテナ内の Nginx（80番ポート）に接続されます。
