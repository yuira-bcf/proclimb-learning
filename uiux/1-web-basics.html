Section 1-1

インターネットとは何か

インターネットとは、世界中のコンピュータ同士がデータをやり取りできるようにつながった巨大なネットワークです。 現在ではスマートフォン、タブレット、パソコン、ゲーム機、テレビなど、あらゆる機器がインターネットにつながっています。 しかし、その本質はとてもシンプルです。インターネットはあくまで 「データを送る」「データを受け取る」という仕組みにすぎません。

World Wide Web（WWW）の誕生は、インターネットの歴史において画期的な出来事でした。これにより、情報の共有とアクセスが格段に容易になり、私たちの日常生活やビジネスに大きな影響を与えることになりました。

1.ネットワーク（Network）とは？

まず「インターネット」という言葉を理解する前に、もっと基本の ネットワーク を理解する必要があります。 ネットワークとは、複数のコンピュータが互いに情報をやり取りできる状態を指します。 例えば、

  • 学校のパソコン室
  • 会社のオフィス
  • 家のWi-Fiに接続されたスマホやPC

といったインターネットに接続された電子機器はすべてネットワークを構成しています。

ネットワークの種類：LAN と WAN

一言にネットワークと言ってもいくつかの種類があります。それがLAN（Local Area Network）とWAN（Wide Area Network）の２つです。

種類説明
LAN（Local Area Network）限られた範囲のネットワーク（家・学校・会社など）
WAN（Wide Area Network）国や地域など "広い範囲" を結ぶネットワーク

LANとWANのネットワーク構成図

LAN同士がつながり、さらに世界中のネットワークが接続され続けた結果、巨大なWANが形成されました。 これが インターネット です。また、インターネットをひと言で表すなら、ネットワーク（LAN）同士が連結した"ネットワークの集合体"と言えます。世界中の企業、大学、個人が持つLANが、ケーブルや電波、海底ケーブルなどで繋がった結果、私たちは世界中の様々な情報をデータとして、やり取りができるようになっています。

クライアントとサーバ

インターネットを理解するうえで必ず知っておくべき概念として、データを要求する側の"クライアント"とデータを提供する側の"サーバ"があります。私たちが日頃当たり前に行っているWebサイトを表示するとき、実施にはスマホやパソコン（クライアント）からサーバに「○○のページをください」と要求（リクエスト）をサーバが「はいどうぞ」とデータを応答（レスポンス）するというシンプルなやり取りが行われています。 Webデザインやプログラミング、UI/UXを学ぶ上では、この「データの流れ」を掴むことが最初の一歩になります。

インターネットが成り立つ理由

インターネットは世界中のコンピュータ同士がつながった巨大なネットワークですが、ただ"つながっている"だけでは通信はできません。人間同士が会話するには 同じ言語 が必要なように、コンピュータ同士が通信するには 共通の約束事（プロトコル） が必要です。その代表が TCP/IP と呼ばれる仕組みです。 このTCP/IPの仕組みは初心者にとって難解であるため、ここでは「インターネット中のすべての通信は、TCP/IP という共通言語で行われている」ということだけ覚えてください。

2．Web とインターネットの違い

多くの人は「インターネット」と「Web」を混同して使っていますが、この2つは全く別物です。 インターネットとは、さまざまなコンピュータ同士を結ぶ 通信インフラ（道や高速道路） のようなものであり、Webとは、 Webサイトやメール、オンラインゲーム、動画配信サービス、SNSなどあらゆるサービスがインターネット上で動くサービスのひとつひとつであり、その中でもWebは、ブラウザを使って情報を閲覧する仕組みのことを指します。

Webを支える3つの基本技術

Webは次の3つが揃って初めて成立します。

  • URL（Webページの住所）
  • HTTP（データのやり取りのルール）
  • HTML（ページの内容を記述する言語）

これらを1つずつ理解していけば、Webの正体が見えてきます。

3．IPアドレス・ドメイン・DNS の役割

Webを見るとき、必ず「通信相手を探す」プロセスが必要になります。そのときに活躍するのが、IPアドレス・ドメイン・DNSという3つの仕組みです。

IPアドレス：インターネット上の住所

インターネットの世界でデータを送るためには、相手の住所（IPアドレス）を必ず指定する必要があります。

例：192.168.10.5

IP は TCP/IP の一部で、「データをどこへ届けるか」を決める"住所担当" の役割を持ちます。TCP/IP の "IP" がこの住所の技術であり、「IPアドレス」という名前になっています。人間には覚えづらいですが、コンピュータにとっては明確な住所になっており、PCやスマホなどの電子機器を始め会社のオフィスで使用されている複合機やデータセンターに設置されているサーバーなど、データを要求したり提供したりするコンピュータ毎にIPアドレスが付与されています。 また、会社のオフィスや学校など限られた範囲のネットワーク（LAN）で使用されるIPアドレスをローカルIP、国や地域など "広い範囲" を結ぶネットワーク（WAN）で使用されるIPアドレスをグローバルIPとも言いますが、一般的にIPアドレスというとLANで使用されるローカルIPを指していることが多いです。

ドメイン：人間が覚えやすい名前

"192.168.10.5"のようなIPアドレスを覚えるのは大変なので、人間が覚えやすい形で名前をつけたものが ドメイン です。 企業名や店舗名、サービス名をドメインにすることができます。 また、「.com」や「.jp」はトップドメイン（TLD）と言われ、インターネット上の住所に分類がされており、私達人間でいうところの"苗字"の様な役割を持っています。

例：
google.com
nhk.or.jp
example.jp

DNS：インターネットの電話帳

DNS（Domain Name System）は、ドメイン名をIPアドレスに変換するための仕組みです。例えばブラウザで「google.com」と入力したとき、私たちの目には見えないインターネットの中では次のようなやりとりが一瞬で行われます。

  1. ブラウザ → 「google.comのIPアドレスを教えて」
  2. DNS → 「142.250.196.206 だよ」
  3. ブラウザ → そのIPアドレスへアクセス
  4. GoogleのサーバがWebページを返す

この"名前から数字の順書"の変換が、DNSの役割です。

4.URL

URL（Uniform Resource Locator）は、Web 上のリソース（ページ・画像・ファイルなど）にアクセスするための住所 です。 ブラウザは URL を手がかりに、どこへアクセスすべきか、何を取得すべきかを判断します。 URLは、大きく "スキーム、ドメイン、パス、パラメータ"の4つのパーツ に分解して理解できます。

URLの構造

スキーム（scheme）

スキームは、「どの方法（プロトコル*）で通信するか？」を指定する部分 です。Web では主に HTTPS または HTTP （保護なしバージョン）の2 種類が使われます。 スキームの次には、オーソリティが続きます。これはスキームから :// の文字パターンで区切られます。 *プロトコルとは、コンピューターネットワーク上でデータを交換または転送するための一連の手順です。

URLのスキーム

ドメイン（Domain）

ドメインは、「どのサーバにアクセスするか？」を示す住所の名前 です。 ブラウザはこのドメインを DNS に問い合わせて、最終的に IP アドレス（数字の住所）に変換します。

URLのドメイン

パス（Path）

パスは、「そのサーバの中のどの場所（ページ）か」を表す部分 です。初期の Web では物理フォルダの位置を表していましたが、現在は多くの場合、サーバ側のルーティング（処理の道順）として使われています。 Webサイトは階層構造になっているので、パスを見るだけでそのページがサイト内のどの場所にあるのかおおまかな内容が予測できることも多いです。

URLのパス

クエリパラメータ（Query Parameter）

クエリパラメータは、「ページに渡す条件・絞り込み内容」 を指定する部分です。?item=12とcolor=blueは検索条件となり、 条件が複数ある場合は&でつなぎます。EC サイト、検索フォーム、ログインなど、動的ページでは必ずと言っていいほど利用されます。

5．Webブラウザはどうやってページを表示しているか

私たちがスマートフォンやパソコンで Web ページを開くとき、裏側では 複数の技術が連携して"ページを取ってくる" 動作が行われています。URL を入力してから画面が表示されるまでの流れを、①DNS → ②TCP/IP → ③HTTP → ④描画 の順に整理して理解します。

（1） URL を受け取る（ユーザーの入力）

ユーザーがブラウザに"https://example.com/news"のような URL を入力すると、ブラウザは「このサイトを表示する準備」を開始します。しかしこの時点では、ブラウザはまだ "どのコンピュータにアクセスすればいいか" を知りません。

(2) DNSで"サーバの住所（IPアドレス）"を調べる

URL の中には ドメイン名（例：example.com） が含まれていますが、インターネットでは 数字で表される IP アドレス が必要です。そこで、ブラウザは DNS に問い合わせます。

DNSの流れ

ステップ内容
１．ブラウザ「example.com の IP アドレスはどこ？」
２．DNS「203.0.113.45 です」
３．ブラウザ「ありがとう、そこにアクセスしよう！」

この "名前 → 住所" の変換を行っているのが DNS（Domain Name System）です。

(3) TCP/IP が"データを確実に送り届ける"

IP アドレスが分かったら、ブラウザはサーバへデータを送ります。しかしデータはそのまま送るのではなく、以下の2つのルールで処理されます。

◎TCP（Transmission Control Protocol）
役割：データを小さく分割し、順番どおり・確実に届ける
TCP が行うこと：

  • データを細かいパケット（小包）に分ける
  • 各パケットに番号を付ける
  • 届いたかどうか確認する
  • 壊れたら再送する
  • サーバ側で組み立て直す

→ 郵便で言えば、小包毎に分けて確実に届ける仕組み

◎IP（Internet Protocol）
役割：そのパケットを"目的地のサーバ"に運ぶ
IP が行うこと：

  • 宛先の IP アドレスを確認
  • ルータがパケットをリレーして運ぶ
  • サーバに到達させる

→ 郵便で言えば、住所を見て届ける仕組み

TCPは小包に分割して確実に届け、IPは住所（IPアドレス）へ届けるようなものであり、この TCP/IP が土台にあるためWeb ページが確実に表示されるのです。

(4) HTTP が"ページをください"とやり取りする

TCP/IP によってサーバに安全にデータが届くと、今度は HTTP（Hypertext Transfer Protocol） というルールに従って会話が始まります。

● ブラウザ → サーバ（HTTP リクエスト）
「/news のページを送ってください！」

ブラウザは必要な情報を明確に伝えます。

● サーバ → ブラウザ（HTTP レスポンス）
「はい、これが HTML と CSS と JavaScript のデータです」
さらに、次のような ステータスコード も返されます。

コード意味
200正常に取得成功
404ページが存在しない
500サーバ内部エラー

UX設計において 404 や 500 の画面デザインは重要です。

(5) ブラウザが受け取ったデータを"画面に描画する"

HTML / CSS / JavaScript を受け取ったブラウザは、次の順序でページを表示します。

  1. HTMLを解析して画面の構造を作る
  2. CSSを適用してデザインを整える
  3. JavaScriptを実行して動きをつける
  4. 必要な画像や追加データを読み込む

といった処理を高速で行い、最終的に画面として表示します。

６．まとめ

Webが表示されるとき次のことが全て一瞬で、裏側で起こっています。

  1. URL入力
  2. DNS：ドメイン→IPに変換
  3. TCP：データを小包に分割
  4. IP：サーバへ届ける
  5. HTTP：ページをください！（HTTP リクエスト）
  6. HTML/CSS/JS が返ってくる（HTTP レスポンス）
  7. ブラウザが描画して表示

UI/UX の観点で重要なポイント

UX は「画面のデザイン」だけでなく通信の仕組みに強く依存しているため、ゆくゆくは次のような観点も意識した設計や機能改善を行っていくことが必要になってきます。今は「このようなポイントがある」とだけ覚えてください。

  • API や画像が重いと「TCP/IP の段階」が遅くなる
  • DNS の応答が遅いと体感速度が悪化
  • HTTP レスポンスが遅いと"読み込み中"が長くなる
  • JavaScript が重いと「描画フェーズ」で固まる
  • キャッシュがあると非常に速くなる（UX向上）
Section 1-2

Webの歴史

World Wide Web（WWW）の誕生は、インターネットの歴史において画期的な出来事でした。これにより、情報の共有とアクセスが格段に容易になり、私たちの日常生活やビジネスに大きな影響を与えることになりました。

インターネットの基礎

インターネットは、もともと米国国防総省のARPA（後のDARPA）によって1960年代に開発されたARPANETに端を発します。このネットワークは、複数のコンピュータを遠隔地で接続し、情報を共有するために設計されました。当初の目的は、戦時における安全かつ信頼性の高い通信手段を提供することでしたが、その後の発展を通じて、学術機関や研究機関にも利用が広がりました。

インターネットの基本的な仕組みは、データをパケットという小さな単位に分割して送信し、受信側で再構成するというもので、この技術により、データの効率的かつ信頼性の高い転送が可能となりました。

1970年代から1980年代にかけて、TCP/IPというプロトコルが標準化され、これが現在のインターネットの基盤となっています。この標準化は、異なるネットワーク間の相互接続を可能にし、グローバルなインターネットの出現に大きく寄与しました。

ARPANET（アーパネット）

1969年、米国防総省のARPA（Advanced Research Projects Agency）は、初のパケット通信ネットワークであるARPANETアーパネットを立ち上げました。 ARPANETは、分散したネットワーク間でデータを送受信する技術の礎を築き、インターネットの直接的な前身となりました。

ARPANETの初期ネットワーク図　出典: 　wikipedia

TCP/IP プロトコル の開発

1970年代後半、ロバート・カーンとヴィントン・サーフによって開発されたTCP/IP（Transmission Control Protocol/Internet Protocol）プロトコルは、異なるネットワーク間の通信を可能にする技術（コンピュータがインターネット上でデータを安全かつ正確にやり取りするための仕組み）として重要な役割を果たしました。 TCP（Transmission Control Protocol）は、データを小さなパケットに分けて送り出し、相手側でそれを元通りに組み立てます。IP（Internet Protocol）は、そのパケットがどこへ向かうべきかを指定する住所のような役割を果たします。このプロトコルのおかげで、異なる種類のコンピュータやデバイス（パソコン、スマホ、サーバーなど）が同じネットワーク上でやり取りできるようになり、今日のインターネットの基盤として機能し続けています。

インターネットの技術的背景

インターネットは世界中のコンピュータやネットワークを接続する巨大なネットワークのことで、情報をやり取りするためのインフラです。1960年代にアメリカ国防総省のARPANETプロジェクトとして始まり、徐々に学術機関や政府機関に広がっていきました。1980年代には、インターネットプロトコル（IP）とトランスミッションコントロールプロトコル（TCP）が標準化され、現在のインターネットの基盤が築かれました。

WWWの誕生

WWWは、1989年にスイスのCERN（欧州原子核研究機構）で働いていたティム・バーナーズ＝リーによって発明されました。当時、研究者たちは大量の情報を効率的に共有する方法を探しており、バーナーズ＝リーの提案がその解決策となりました。

ハイパーテキスト

ハイパーテキストは、文書内の特定の部分を他の文書にリンクさせる技術です。これにより、情報間の相互参照が容易になり、複雑な情報体系を整理して理解しやすくすることができます。

Hypertext Editing System (HES) IBM 2250 ディスプレイ・コンソール　出典: wikipedia

WWWの起源

WWW（ワールド・ワイド・ウェブ）の起源は、1940年代から60年代にかけて提唱された概念と技術にそのルーツを持っています。

ヴァネヴァー・ブッシュの「メメックス」や、ダグラス・エンゲルバートの「オーグメンテーション」といった先駆的なアイデアは、人間の知識や情報を整理し、効率的にアクセスするための基盤を構築しました。特に、エンゲルバートのハイパーテキストやコンピュータマウスの発明は、後にWWWが誕生する際の重要な技術的要素となりました。

メメックス

1945年、アメリカの科学者ヴァネヴァー・ブッシュは、「As We May Think」という論文で、知識を整理し、簡単にアクセスできる「メメックス」という概念を提唱しました。これは、後のハイパーテキストやウェブのアイデアの先駆けとなりました。明に繋がりました。

メメックスの概念図

オーグメンテーション

1960年代、ダグラス・エンゲルバートは、コンピュータを用いて人間の知識を拡張するという「オーグメンテーション（Augmentation）」のアイデアを推進しました。彼のチームは、世界初のコンピュータマウスを発明し、テキストを相互にリンクするハイパーテキストの基本概念を実現しました。

Section 1-3

WWWの初期の発展と普及

WWWの導入は、1990年代初頭に急速に広がり、多くの企業や個人がウェブサイトを立ち上げるようになりました。これにより、インターネットは一部の専門家のものから、一般大衆が利用できるものへと変貌を遂げました。

初期ブラウザとその進化

初期のウェブブラウザは、インターネットの一般化に大きく貢献しました。1990年にティム・バーナーズ＝リーが開発した「WorldWideWeb」は、最初のウェブブラウザであり、同時にウェブエディターでもありました。このブラウザにより、ユーザーはウェブページを閲覧し、ハイパーリンクを通じて他のページへと移動することが可能になりました。

その後、1993年にリリースされたMosaicは、初めてのグラフィカルユーザーインターフェースを持つブラウザで、画像の表示が可能でした。Mosaicは非常に人気を博し、後にNetscape Navigatorとして進化し、1990年代のウェブの普及に大きく貢献しました。続いて、マイクロソフトがInternet Explorerをリリースし、ブラウザ市場で激しい競争が繰り広げられました。これらのブラウザは、次々と新機能を導入し、ウェブの視覚的体験を豊かにしました。1990年代末から2000年代初頭にかけては、標準化と互換性の向上が進み、ウェブの利便性が飛躍的に向上しました。

WorldWideWeb

ティム・バーナーズ＝リーが開発した最初のウェブブラウザは「WorldWideWeb」と名付けられました。このブラウザは、ハイパーテキストを表示するだけでなく、ウェブページの作成も可能なエディタとして機能しました。

世界で一番最初のブラウザの名は「WorldWideWeb」　出典: GIGAZINE

Mosaic

1993年、NCSA（National Center for Supercomputing Applications）で開発されたMosaicブラウザは、グラフィカルユーザーインターフェース（GUI）を備えた最初のブラウザであり、ウェブの普及に大きく貢献しました。 Mosaicはテキストだけでなく、画像も表示できる画期的な機能を持っていました。

NCSA Mosaicブラウザのスクリーンショット　出典: wikipedia

ブラウザ戦争

MosaicをベースにしたNetscape Navigatorが、1994年に登場し、瞬く間に市場を席巻しました。 このブラウザは、インターネットの普及とともに広く利用されるようになり、商業用ウェブの発展を支えました。 その後、MicrosoftはInternet Explorer（IE）をリリースし、Netscapeに対抗しました。 1990年代後半には、IEとNetscapeの間で「ブラウザ戦争」と呼ばれる激しい競争が繰り広げられました。

Internet Explorerのスクリーンショット　出典: Neowin

ブラウザ戦争の影響

この競争は、ウェブ標準の制定や技術革新の加速をもたらし、現在の多様なウェブエクスペリエンスの基盤を築きました。

初期のウェブサイト

初期のウェブサイトは、主にテキストで構成されており、デザインはシンプルで、リンクを通じて情報を探しやすくすることが最優先されていました。MosaicやNetscapeの登場により、ウェブサイトに画像が導入されるようになり、ウェブの視覚的な魅力が増しました。これにより、コンテンツの多様化が進み、ウェブはよりリッチなメディアとなりました。

時系列で振り返る「ウェブサイト、進化の歴史」　出典:INFO CUBIC

CSSの誕生

1996年に発表されたCSS（Cascading Style Sheets）は、ウェブページのデザインとレイアウトを分離するためのスタイルシート言語です。これにより、ウェブデザインはより柔軟で美しいものとなり、コンテンツとデザインの分離が実現しました。

https://washimo-web.jp/　出典:WaShimo

動的コンテンツの導入

JavaScriptやFlashなどの技術が導入されることで、ウェブページは静的なコンテンツから動的なコンテンツへと進化しました。 これにより、ユーザーインタラクションが増え、ウェブサイトの可能性が大きく広がりました。

現代のウェブ技術への影響と教訓

WWWの初期の発展は、現代のウェブ技術に多大な影響を与えました。当時の技術革新と試行錯誤は、今日のウェブがどのように機能し、どのように利用されているかに深く関わっています。

初期ブラウザとその進化

初期のインターネットとウェブの発展には、多くの試行錯誤が伴いました。 技術的な課題だけでなく、標準化やセキュリティの問題、ユーザーエクスペリエンスの最適化など、多岐にわたる課題に直面しました。例えば、初期のウェブサイトはブラウザによる表示の違いや、回線速度の遅さなど、技術的制約によってデザインが制限されていました。 また、セキュリティの重要性も早期に認識され、暗号化技術やセキュアな通信プロトコルの開発が進みました。 これらの初期の試行錯誤から得られた教訓は、今日のウェブ標準の確立や、ユーザビリティとアクセシビリティの向上に大きく貢献しています。初期の失敗と成功は、インターネットとウェブが今日のように進化するための貴重な経験となりました。

オープンスタンダードの重要性

オープンスタンダードの重要性 オープンスタンダードは、技術の発展と普及を促進し、互換性を維持するために重要です。WWWの発展においても、HTMLやCSS、HTTPなどのオープンスタンダードが採用され、ウェブ全体の一貫性と相互運用性が確保されました。

ユーザビリティとアクセシビリティ

初期のウェブデザインから学んだもう一つの重要な教訓は、ユーザビリティとアクセシビリティの重要性です。全てのユーザーがウェブを使いやすくするためのデザインと技術の実装が求められました。文字サイズや色のコントラスト、ナビゲーションの明確さなど、ユーザビリティに配慮したウェブデザインは、ユーザーの体験を向上させるための基本要素となりました。

出典https://basixs.com/times/columns/web-accessibility

セキュリティの課題とその対応

初期の脅威: 初期のウェブでは、セキュリティがまだ十分に確立されておらず、データの盗難やハッキングなどの脅威が存在していました。

現代の対策: SSL/TLSの導入やHTTPSの普及により、ウェブ通信のセキュリティが強化され、ユーザーのプライバシー保護が進みました。
