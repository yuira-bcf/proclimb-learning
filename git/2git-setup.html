Section 2-1

Gitのインストール（Windows/Mac/Linux）

Gitを使い始めるには、まずお使いのコンピュータにGitをインストールする必要があります。OS別にインストール方法を見ていきましょう。

💻 OS別インストール手順フロー
💻 OSを確認 Windows / Mac / Linux
→
📥 Gitをインストール OS別の方法で導入
→
✅ バージョン確認 git --version
→
⚙ 初期設定 git config で設定

1.1 Windowsの場合

Windowsでは「Git for Windows」をインストールします。

  1. https://git-scm.com/ にアクセス
  2. 「Download for Windows」ボタンをクリック
  3. ダウンロードしたインストーラー（.exe）を実行
  4. 基本的にデフォルト設定のまま「Next」を押して進める
  5. インストール完了後、「Git Bash」を開いてバージョンを確認
バージョン確認（Git Bash） Bash 
git --version
# 出力例: git version 2.43.0.windows.1
lightbulb Git Bashとは

Git for WindowsにはGit Bashというターミナルが付属しています。Linux/Macと同じコマンドが使えるため、Windowsユーザーにとって非常に便利です。コマンドプロンプトやPowerShellからもGitは使えますが、学習にはGit Bashがおすすめです。

1.2 Macの場合

Macでは複数の方法でGitをインストールできます。

方法1: Xcode Command Line Toolsで導入 Terminal 
# ターミナルで以下を実行
xcode-select --install
# ダイアログが表示されたら「インストール」をクリック
方法2: Homebrewで導入（推奨） Terminal 
# Homebrewがインストール済みの場合
brew install git

# バージョン確認
git --version
# 出力例: git version 2.43.0

1.3 Linuxの場合

Ubuntu/Debianの場合 Terminal 
sudo apt update
sudo apt install git

git --version
# 出力例: git version 2.43.0
CentOS/Fedoraの場合 Terminal 
# CentOS
sudo yum install git

# Fedora
sudo dnf install git
Section 2-2

初期設定（git config）

Gitをインストールしたら、最初にユーザー名とメールアドレスを設定します。この情報はコミットのたびに記録されるため、正しく設定することが重要です。

2.1 ユーザー名とメールアドレスの設定

必須の初期設定 Bash 
# ユーザー名の設定
git config --global user.name "あなたの名前"

# メールアドレスの設定
git config --global user.email "your-email@example.com"

# 設定の確認
git config --global user.name
git config --global user.email
warning GitHubを使う場合の注意

GitHubと連携する場合は、GitHubに登録したメールアドレスと同じものを設定してください。一致しない場合、GitHubのコントリビューション（草）グラフに反映されません。

2.2 git config の3つのスコープ

git config には3つのスコープ（適用範囲）があります。より狭い範囲の設定が優先されます。

⚙ git config の3つのスコープ
--system（システム全体） /etc/gitconfig - 全ユーザーに適用
▼ 上書き
--global（ユーザー全体） ~/.gitconfig - 現在のユーザーに適用
▼ 上書き
--local（リポジトリ単位） .git/config - 現在のリポジトリのみに適用

下のスコープほど優先度が高い（localが最優先）

スコープの使い分け Bash 
# グローバル設定（通常はこれを使う）
git config --global user.name "個人の名前"

# ローカル設定（特定のプロジェクトだけ別の設定にしたい場合）
git config --local user.email "work-email@company.com"

# システム設定（管理者向け、通常は使わない）
git config --system core.autocrlf true
lightbulb 仕事と個人で設定を分ける

仕事用と個人用で異なるメールアドレスを使いたい場合、グローバルに個人用のアドレスを設定し、仕事用リポジトリだけ --local で会社のアドレスを設定するのが便利です。

Section 2-3

ターミナル/コマンドプロンプトの基本

Gitはコマンドライン（ターミナル）から操作するツールです。GUIツールもありますが、コマンドラインの基本を覚えておくとGitの仕組みがよく理解できます。

3.1 よく使うコマンド

コマンド 説明 使用例
pwd 現在のディレクトリを表示 pwd
ls ファイル一覧を表示 ls -la
cd ディレクトリを移動 cd my-project
mkdir 新しいディレクトリを作成 mkdir my-project
touch 新しいファイルを作成 touch index.html
cat ファイルの中身を表示 cat index.html
clear 画面をクリア clear
基本的なターミナル操作例 Bash 
# 現在の場所を確認
pwd
# /Users/username

# デスクトップに移動
cd ~/Desktop

# プロジェクト用フォルダを作成
mkdir my-first-project

# 作成したフォルダに移動
cd my-first-project

# ファイル一覧を表示（まだ何もない）
ls -la

# 新しいファイルを作成
touch index.html

# 1つ上のディレクトリに戻る
cd ..
warning Windowsユーザーへの注意

Windowsの「コマンドプロンプト」ではLinux/Macと異なるコマンドが使われます（dir、cd など）。Git Bashを使えばLinuxと同じコマンドが使えるため、学習にはGit Bashの利用を推奨します。

Section 2-4

エディタの設定

Gitでコミットメッセージを編集する際やマージコンフリクトを解決する際に、テキストエディタが起動されます。デフォルトではVimが設定されていることが多いですが、使い慣れたエディタに変更できます。

4.1 エディタの変更

エディタの設定例 Bash 
# VS Codeに設定（推奨）
git config --global core.editor "code --wait"

# nanoに設定（シンプルなターミナルエディタ）
git config --global core.editor "nano"

# Vimに設定（デフォルト）
git config --global core.editor "vim"

# Sublime Textに設定
git config --global core.editor "subl -n -w"
lightbulb VS Code を推奨する理由

VS Codeは無料で使え、Git連携機能が充実しています。--wait オプションは、VS Codeでファイルを閉じるまでGitが待機するために必要です。初学者には最もおすすめのエディタです。

warning Vimが起動してしまったら

Vimに慣れていない場合、突然Vimが起動すると操作方法が分からず困ることがあります。Vimを終了するには :q と入力してEnterキーを押します。保存して終了するには :wq、保存せず終了するには :q! です。

4.2 その他の便利な設定

おすすめの追加設定 Bash 
# デフォルトブランチ名を main に設定
git config --global init.defaultBranch main

# 改行コードの自動変換（Windowsの場合）
git config --global core.autocrlf true

# 改行コードの自動変換（Mac/Linuxの場合）
git config --global core.autocrlf input

# カラー表示を有効化
git config --global color.ui auto

# プッシュ時のデフォルト動作を設定
git config --global push.default current
Section 2-5

設定の確認方法

設定が正しく行われたか確認する方法を覚えておきましょう。設定ミスはあとからトラブルの原因になることがあります。

5.1 設定一覧の表示

設定の確認コマンド Bash 
# 全ての設定を一覧表示
git config --list

# 出力例:
# user.name=田中太郎
# user.email=tanaka@example.com
# core.editor=code --wait
# init.defaultbranch=main
# color.ui=auto

# 特定の設定値だけ確認
git config user.name
# 出力: 田中太郎

git config user.email
# 出力: tanaka@example.com

# 設定のスコープ（どのファイルに書かれているか）も表示
git config --list --show-origin

5.2 設定ファイルの直接編集

設定ファイルは直接テキストエディタで編集することもできます。

設定ファイルを直接開く Bash 
# グローバル設定ファイルをエディタで開く
git config --global --edit

# 設定ファイルの場所
# Mac/Linux: ~/.gitconfig
# Windows: C:\Users\ユーザー名\.gitconfig
.gitconfig ファイルの内容例 INI 
[user]
    name = 田中太郎
    email = tanaka@example.com
[core]
    editor = code --wait
[init]
    defaultBranch = main
[color]
    ui = auto
lightbulb 設定を削除したい場合

間違った設定をした場合は、git config --global --unset 設定名 で削除できます。例えば git config --global --unset user.name でユーザー名の設定を削除できます。

Exercise

2章 練習問題

Q1

Gitの初期設定で、最初に必ず設定すべき2つの項目はどれですか？

  1. ユーザー名とパスワード
  2. ユーザー名とメールアドレス
  3. メールアドレスとエディタ
  4. リポジトリ名とブランチ名
Q2

git config の3つのスコープのうち、最も優先度が高いのはどれですか？

  1. --system
  2. --global
  3. --local
  4. 全て同じ優先度
Q3

Gitのインストール後にバージョンを確認するコマンドはどれですか？

  1. git -v
  2. git version
  3. git --version
  4. git check
Answer

2章 解説・解答

Q1の解答: B

Gitの初期設定で最初に設定すべきなのはユーザー名（user.name）とメールアドレス（user.email）です。この2つはコミットのたびに記録される情報で、誰がその変更を行ったかを識別するために使われます。

Q2の解答: C

--local（リポジトリ単位）が最も優先度が高いです。優先順位は local > global > system です。同じ設定項目が複数のスコープで定義されている場合、より狭い範囲の設定が優先されます。

Q3の解答: C

git --version でGitのバージョンを確認できます。なお git version（Bの選択肢）も同じ結果を返しますが、一般的に使われるのは --version オプションです。git -v もGit 2.34.0以降では使用可能ですが、最も標準的な書き方は git --version です。
