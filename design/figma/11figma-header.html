ヘッダーとグローバルナビゲーションの作成

ロゴ・メニュー・ボタンをオートレイアウトで整列

この章の内容

  • 5-1グローバルナビゲーションの概要
  • 5-2グローバルナビゲーションのパーツを作成する

ページの上部に表示されるロゴやメニュー、ボタンなど 主要ページへのリンクをまとめたセクションを「ヘッダー(Header)」と呼びます。 ヘッダーに設置されたページ内リンクや、他ページに遷移するリンクのことは「グローバルナビゲーション」と呼ばれ、 どんなページにも、必須のUIとなります。

Section 5-1

グローバルナビゲーションの概要

メニューの並びや間隔は、ユーザーがクリック or タップしやすいよう、配慮して作成する必要があります。 ヘッダー・グローバルナビゲーションの作成には 「オートレイアウト」機能 を使用していきます。

▼ 作成するヘッダー・グローバルナビゲーション

Section 5-2

グローバルナビゲーションのパーツを作成する

全体作業フロー
  1. ロゴテキストを作成し、スタイルを設定
  2. メニューテキストを作成し、スタイルを設定
  3. メニューテキストをオートレイアウト化
  4. Button インスタンスを設置と内容変更
  5. ①②④で作成したものを、オートレイアウト化して整列
  6. ⑤で作成したフレームをコンポーネント化、リネーム

① ロゴテキストを作成し、スタイルを設定（Componentsページにて作業）

  1. テキストを配置します：T
  2. [ Edix Works ] と入力します
  3. テキストスタイル → [ title/md ]、色スタイル →[ portfolio / black ]を設定します。
  4. テキストのレイヤー名称を変更します。[ Edix Works ] →[ LogoText ]

② メニューテキストを作成し、スタイルを設定

次に、ナビゲーション（リンク）となる3つのメニューテキストを作成していきます。

※ 今回はUIを作成するのみで、実際にはリンク機能は持たせません

  1. LogoTextの右側、適当な位置にテキストを配置します：T
  2. [ Service ] と入力します
  3. テキストスタイル → 「text/md」、色スタイル → [ portfolio / black ]を設定します。
  1. Service を選択し、[ Shift + Alt(Option) +ドラッグ]で右隣にテキストを複製します
  1. [ Ctrl(Cmd) + D ] で、もう一つ複製し、合計3つの[ Service ]を並べます
  2. 真ん中のテキスト →[ Works ]、一番右のテキスト →[ About ]に変更します

③ メニューテキストをオートレイアウト化

メニューテキストをすべて作成したら、オートレイアウト機能を使って、 フレーム化しつつ「整列」させていきます。

  1. [ Service ] [ Works ] [ About ] の3つのテキストを選択し、オートレイアウト化します：Shift + A
    ※ 複数選択は、Shiftを押しながら各要素をクリックしていきます。
    ※ または、ドラッグで各要素を囲むと、範囲内の要素をすべて選択した状態になります。
  1. メニューフレーム のオートレイアウト項目を、下記に設定します（デフォルトで設定済みのものもアリ）
    • フロー：水平
    • 配置：左揃え
    • 間隔：40 （フレーム内での、オブジェクト同士の距離設定）
    • 各パディング：0
  2. フレームの名称 → [ navMenu ]に変更します。
オートレイアウトの留意点
  • 前提として、オートレイアウトとは「フレーム」の1種である
  • ただのフレーム → 複数の要素を内包できる親要素（グループ化のようなイメージ）
  • オートレイアウト化したフレーム → ただのフレーム ＋ 縦や横で「整列」させる機能がついたフレーム
  • パディング（内側の余白）や、要素の並びの型を選択できる
  • 新たな要素を追加すると、すべて自動で整列させる（要素を重ねたりする表現はできない ← フレームでは可）
  • オートレイアウトする直前の要素の配置によって、デフォルトで設定される各項目の状態が異なる
    ※ 最初の配置により、Figmaが「よしなに判断」し、初期設定してしまうクセ

④ Button インスタンスの設置と内容変更

ではナビゲーションの最後の要素である、ボタンを配置していきます。

  1. 左パネルの[アセット]タブをクリック
  2. すべてのライブラリ → [このファイル内で作成]をクリック
  3. [ Button ] を [ navMenu ]の右隣にドラッグ＆ドロップし、インスタンスを作成します
    ※ 同じページにマスターコンポーネントがある場合は、Alt + ドラッグでマスターから複製も可能です
  1. 配置されたボタンのテキストを[ Contact ]に変更します
  2. [ファイル]タブに切替、レイヤーの[ ◇　Button ]の名称を[ navButton ]に変更します。

これでナビゲーションのパーツは完成です。 あとは最初に作成した「LogoText」と、このナビゲーションをひとつにまとめ、 「ヘッダー」と呼ばれるセクションとなるように仕上げていきます。

POINT
  • レイヤー名称の右にある「鍵アイコン」と「瞳アイコン」がONになっていないか、たまに確認しましょう
  • 鍵アイコン → 要素の選択の可否を決めるスイッチ（ONのとき：要素を一切選択出来なくなる）
  • 瞳アイコン → キャンバス上での表示可否を決めるスイッチ（ONのとき：キャンバス上で見えなくなる）
    ※ 「いくらクリックしても選択できない！」というときはまずここを確認しましょう 自分でも知らないうちに触れてしまい、鍵をONにしてることが意外とよくあります

⑤ ①③④で作成したものを、オートレイアウト化して整列

ここではオートレイアウトのフレームの中に、さらにオートレイアウトのフレームを作成していきます。 オートレイアウトをネスト（入れ子）にすることで、より柔軟な整列を実現します。

  1. [ LogoText ] [ navMenu ] [ navButton ] をすべて選択し、オートレイアウト化します：Shift + A
  2. レイヤーで新たに作成されたフレームの、オートレイアウト項目を下記に設定します。
    • フロー：水平
    • サイズ変更：W1440
    • 配置：中央
    • 間隔：自動
    ※ 「間隔」の値を空白にしてEnterを押すと自動に設定されます。
  1. 作成したフレームの名称を [ header ]に変更します
POINT

これがヘッダーセクションの親となるフレームとなります。このフレームは3つの要素を内包しており、 そのうちのひとつはオートレイアウト化したフレーム（navMenu）です。 つまり、この時点でオートレイアウトフレームが入れ子になっている状態となります。 ここからさらに、オートレイアウトフレームを増やして、より柔軟な整列状態を作っていきます。

  1. [ navMenu ] [ navButton ]の2つのみを 選択し、オートレイアウト化します：Shift + A
  2. レイヤーで、新たに作成されたフレームの名称を[ groNav ]に変更します
  3. groNavのオートレイアウト項目を、下記に設定します。
    • サイズ変更：コンテンツを内包（数字の右にある[∨]をクリックすると選択可能）
    • 間隔：48
    • 配置：左揃え
POINT
  • コンテンツを内包 → 子要素の幅 + 子要素同士の間隔 + パディング ＝ 要素の幅
  • コンテナに合わせて拡大 → 親要素の幅に併せて、自分自身の幅の比率を自動調整
  • 幅を固定 → 親要素、子要素の大きさに関わらず、常に一定の幅を保持

★ つまり自分の幅を子に合わせるか、親に合わせるか、という基準です（高さも同様に設定可能）

  1. header のオートレイアウトの設定に下記を追加します
    • 水平パディング：20
    • 垂直パディング：40
  1. headerをコンポーネント化します：Ctrl(Cmd) + Alt(Option) + K

これで、ヘッダーとグローバルナビゲーションが完成となります。 次からはいよいよデザインに入っていきます。
