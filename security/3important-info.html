Section 3-1

企業にとって重要な情報とは

企業にとって重要な情報とは、「その情報が企業外に漏れる（漏えいする）と、企業の事業運営上で重大な問題を引き起こす可能性のある情報」として定義されます。

この「重要な情報」の概念を正確に理解することは、効果的な情報セキュリティ対策を実施する上での出発点となります。

重要な情報の特徴

  • 漏えいすると事業運営に重大な影響を与える
  • 法的責任が発生する可能性がある
  • 企業の競争力や信頼に関わる
Section 3-2

情報漏えいが企業に与える具体的な影響

1. 顧客からの信頼失墜

信頼回復の困難さ

個人情報や機密情報が漏えいすると、顧客は企業の情報管理能力に疑問を抱き、今後の取引を見直したり、他社への乗り換えを検討したりします。一度失った信頼を回復するには長期間を要し、場合によっては完全に回復できないこともあります。

2. 法的責任の発生

個人情報保護法などの関連法規に違反した場合、行政処分や刑事罰の対象となる可能性があります。また、被害を受けた個人や企業から損害賠償請求を受けるリスクもあります。

3. 経済的損失

情報漏えいの対応には多大なコストがかかります。

直接的な費用だけでも数億円規模になることがあります。

4. 競合他社への優位性の喪失

技術情報や営業戦略などの企業秘密が漏えいすると、競合他社に自社の優位性を奪われ、市場での競争力を失う可能性があります。

5. 事業継続の困難

重要なシステムが攻撃を受けて停止した場合、業務が継続できなくなり、売上の減少や取引先との関係悪化を招く可能性があります。

Section 3-3

個人情報の具体例

個人情報保護法では、「特定の個人を識別することができる情報」を個人情報と定義しています。現代のデジタル社会では、個人情報の種類と形態が多様化しており、従来考えられていた以上に広範囲な情報が個人情報に該当する可能性があります。

基本的な個人情報（従来から認識されているもの）

情報の種類 説明
氏名、生年月日 最も基本的な個人識別情報
住所、電話番号 連絡先に関する情報
性別、血液型 身体的特徴に関する情報
職業、収入 社会的・経済的地位に関する情報
家族構成、学歴 個人の背景に関する情報

デジタル時代の個人情報（新たに重要性が認識されているもの）

情報の種類 説明
メールアドレス、SNSアカウント デジタルコミュニケーションに関する情報
生体情報 指紋、顔認証データ、虹彩情報、声紋など、本人確認に使用される身体的特徴
クレジットカード情報 カード番号、有効期限、セキュリティコード、暗証番号
金融機関情報 銀行口座番号、取引履歴、資産情報
オンラインサービス情報 ログインID、パスワード、利用履歴、購入履歴

複合的な個人情報

組み合わせによる特定

現代では、複数の情報が組み合わされてデータベースに保存されることが一般的です。そのため、一見すると重要でない情報でも、他の情報と組み合わされることで個人を特定できる可能性があり、慎重な取り扱いが必要になります。

単体では個人を特定できないが、組み合わせることで特定可能になる情報の例：

  • 年齢と居住地域の組み合わせ
  • 職種と勤務先エリアの組み合わせ
  • 趣味嗜好と行動履歴の組み合わせ
  • 位置情報と時間情報の組み合わせ など

AIによる個人特定

また、技術の進歩により、従来は個人を特定できないと考えられていた情報でも、AIや機械学習技術を使って分析することで個人を特定できるケースが増えています。このため、個人情報の範囲は今後さらに拡大していく可能性があり、常に最新の動向を把握しておくことが重要です。

Section 3-4

マイナンバー（特定個人情報）

マイナンバー（個人番号）は、個人情報の中でも特に厳格な管理が求められる「特定個人情報」に該当します。2015年に制度が開始されて以来、マイナンバーを取り扱う事業者には、通常の個人情報よりもはるかに厳しい管理義務が課せられています。

特定個人情報としてのマイナンバーの特徴

マイナンバーは、国民一人ひとりに付与される12桁の番号で、社会保障・税・災害対策の分野で活用されています。この番号は生涯変わることがなく、また他の個人情報と容易に照合できるため、個人のプライバシーに与える影響が極めて大きいとされています。

厳しい罰則

マイナンバーの不適切な取り扱いに対しては、個人情報保護法よりも重い罰則が設けられており、最大で4年以下の懲役または200万円以下の罰金が科せられる可能性があります。

厳格な取扱いルールの具体例

1. 取扱者の厳格な限定

マイナンバーを取り扱うことができる従業員を明確に指定し、その者以外は一切アクセスできないようにする必要があります。また、取扱者には定期的な教育を実施し、責任の重さを十分に理解させる必要があります。

2. 取扱エリアの物理的管理

マイナンバーを含む書類やデータを取り扱う場所は、関係者以外が立ち入れないよう物理的にアクセスを制限する必要があります。また、書類の保管場所は施錠可能なキャビネットや金庫を使用し、パソコンの画面は外部から見えない位置に設置するなどの配慮が必要です。

3. 取得から廃棄までの全工程管理

マイナンバーのライフサイクル全体（取得→利用→保存→廃棄）の各段階で適切な管理措置を講じる必要があります。特に廃棄時には、復元不可能な方法で確実に削除・破棄し、その記録を残す必要があります。

4. ガイドラインの遵守

個人情報保護委員会が定める「特定個人情報の適正な取扱いに関するガイドライン」に基づいて、組織的・人的・物理的・技術的安全管理措置を講じる必要があります。

企業での実際の取り扱い場面

マイナンバーを扱う業務

多くの企業では、給与計算、年末調整、社会保険手続き、源泉徴収票の作成などの業務でマイナンバーを取り扱います。これらの業務を担当する人事・経理部門の従業員は、マイナンバーの重要性と取扱いルールを十分に理解し、日常業務の中で確実に実践する必要があります。

コラム 情報漏えいの損害賠償額

過去の判例では、個人情報漏えいに対する損害賠償額として、1人あたり数千円から数万円程度が認められるケースが多くあります。一見すると少額に思えますが、漏えいした件数が多ければ、総額は膨大な金額になります。

例えば、10万件の個人情報が漏えいし、1人あたり1万円の賠償となった場合、総額は10億円にもなります。これに加えて、対応コスト、信頼回復のための費用、機会損失などを考えると、企業にとっての実質的な損害はさらに大きくなります。
