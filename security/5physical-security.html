Section 5-1

事務所の中の机の管理

ペーパーレスが謳われPC、電子データでの業務が増えていますが、IT企業以外でのオフィス環境で依然として紙での業務が行われていることも少なくはありません。

机上放置のリスク

基本的で重要な問題

机の上に放置した情報は、誰かに持ち去られる危険にさらされており、これは物理的セキュリティの最も基本的で重要な問題の一つです。

関係者以外の人が見たり触れたりできる状態にあると、意図しない情報漏洩や盗み見のリスクが生まれます。特に、来客者、清掃業者、他部署の従業員、修理業者などが事務所内を移動する際に、机上の資料に目を通す可能性があります。また、同僚であっても、担当プロジェクトが異なれば知る必要のない情報もあります。

IT企業でも見かける問題行動

危険な行為の例
  • 付箋にパスワードを書いてモニターに貼り付ける
  • システムの管理者権限情報などをメモに書き、机に置いたまま離席する

POINT!!

重要情報は「放置せず」、適切に管理および保護する必要があります。これは、情報セキュリティの基本中の基本であり、どんなに高度な技術的セキュリティ対策を講じても、物理的な管理が不適切であれば、そこから情報が漏洩してしまう可能性があります。

クリーンデスク・ポリシーという考え方があり、業務終了時には机上に何も残さない状態にすることが推奨されています。

Section 5-2

具体的なリスク事例

職場環境では、様々な関係者が事務所等に出入りします。リスクシナリオを理解することで、適切な予防策を講じることができます。開発現場に限らず一般的なオフィスでも同様です。

来客リスク

1. システム導入先からの来客

顧客企業の担当者がシステムの進捗確認や打ち合わせで訪問する際、開発チームの作業エリアを通ることがあります。この時、他の顧客のプロジェクト情報や、競合他社の情報が見える状態にあると、情報漏洩や競合情報の流出につながる可能性があります。

2. 外部ベンダーの技術者

インフラ整備、機器メンテナンス、ソフトウェアのサポート担当者などが定期的に訪問します。これらの技術者は専門知識を持っているため、画面に表示されている情報や机上の技術資料の重要性を理解し、悪意がある場合は情報を収集する可能性があります。

第三者立ち入りによる技術情報の流出リスク

🧹

清掃業者

夜間や早朝の清掃時間中、机上に技術仕様書やシステム構成図が置かれていると、それらの情報が写真撮影される可能性があります。特に、競合他社と契約している清掃業者の場合、意図的な情報収集が行われるリスクもあります。

🔧

修理・保守業者

パソコンの修理、ネットワーク機器の交換、空調設備の点検などで業者が入る際、作業エリア周辺の情報に触れる機会があります。技術者であれば、システム構成やセキュリティ設定に関する情報の価値を理解している場合があります。

👥

同僚・他部署による意図しない情報アクセス

同じ会社の従業員であっても、プロジェクトや部署が異なれば、知る必要のない情報があります。特に、顧客情報、他部署のプロジェクト詳細、給与・人事情報、競合分析資料などは、適切なアクセス制限が必要です。

対策のポイント

これらのリスクを防ぐためには、机の上はいつも整理しておき、重要な情報が放置された状態にならないよう注意する必要があります。また、来客時には事前に作業エリアの整理を行い、機密情報が見えない状態にすることが重要です。

Section 5-3

机上管理のポイント

働く上での机上管理として、ルールとして設定されていることの多いポイントを紹介します。

1. 帰宅時の基本ルール（クリーンデスク・ポリシー）

業務終了時には、机の上に何も残さない状態にします。これは「クリーンデスク・ポリシー」と呼ばれる国際的な標準的なセキュリティ対策です。紙の資料、USBメモリ、外付けハードディスク、メモ用紙、付箋など、すべてを適切な場所に保管します。

2. 画面の保護

パソコンの画面には、ソースコード、データベースの内容、システム設計図、顧客情報など業務上重要な情報が表示されていることがあります。

画面ロックのショートカット

離席時には必ず画面ロックをかけ、長時間離れる場合はアプリケーションを終了します。

  • Windows: Win + L
  • Mac: Command + Control + Q

3. 重要な情報が放置された状態の回避

仕様書などは、紙文書の書類は使用後すぐに適切な場所に保管します。印刷した資料は、不要になったらすぐにシュレッダーで処分します。

4. 情報の分類と管理

情報を「公開」「社内限り」「関係者限り」「極秘」などの機密度レベルに応じた分類を行い、それぞれに応じた取り扱いルールを設定します。例えば、顧客の本番データは「極秘」、社内のシステム設計書は「関係者限り」など。

5. アクセス権限の明確化

誰がどの情報にアクセスできるかを明確にし、必要最小限の権限のみを付与します。プロジェクトメンバー間でも、役割に応じて異なるアクセス権限を設定することがあります。

6. 保管場所の統一と施錠

重要な資料は指定された施錠可能なキャビネットや金庫に保管します。

7. デジタル情報の管理（ローカル保存の制限）

重要なファイルは個人のパソコンではなく、セキュリティが確保されたサーバーやクラウドストレージに保存します。バックアップやバージョン管理も含めて、組織的な管理を行います。

8. USBメモリ・外部媒体の管理

個人所有のUSBメモリや外付けハードディスクの業務利用は、原則として禁止されています。業務で使用する場合は、暗号化機能付きの承認された媒体のみを使用します。

Section 5-4

情報廃棄の重要性

ITエンジニアが扱う情報の廃棄は、一般的な書類処分よりもはるかに慎重さが求められます。技術情報、顧客データ、システム仕様書などには、企業の競争力の源泉となる情報や、悪用されると深刻な被害をもたらす可能性のある情報が含まれているためです。

廃棄による情報漏洩の典型事例

実際に起きた事例
  • 廃品回収に出したハードディスクから、顧客データが復元・流出
  • シュレッダーをかけずに廃棄した資料から、データベースの接続情報が漏洩
  • 社内ゴミ箱から、テスト用の本番データが第三者に発見される

POINT!!

重要な情報が読めないようにする対策は、必須の情報セキュリティ対策です。「廃棄」も情報のライフサイクル管理の重要な一部であり、作成・利用・保管と同じレベルの注意を払う必要があります。

個人の判断だけに任せるのではなく、組織として統一された廃棄ルールを設定し、定期的な監査や教育を通じて、適切な廃棄が実施されていることを確認することが重要です。

Section 5-5

情報の社外持ち出し管理

在宅勤務、客先での作業、出張先での開発など、社外で情報を扱う機会が頻繁にあります。そのため、情報の社外持ち出し管理は、他の職種以上に重要で複雑な課題となります。

ITエンジニア特有の持ち出し情報例

情報の種類 具体例
開発関連データ ソースコード、開発環境、テストデータ
顧客システム情報 データベースの構造、既存データのサンプル
インフラ・セキュリティ情報 サーバー設定情報、ネットワーク構成 など

承認プロセスの重要性

重要な情報を会社の外へ持ち出す場合は、必ず上司やプロジェクトマネージャーなどに確認を仰ぎます。これは単なる形式的な手続きではなく、客観的な視点からリスクを評価し、適切な対策を検討するために不可欠なプロセスです。

多くの開発現場では、何を、いつ、誰が、どこへ、何の目的で持ち出したかを詳細に記録しています。返却予定日も明記し、実際の返却時には確認者のサインを取得します。

リモート環境における対策

コロナ禍以降、在宅勤務で業務を行うIT企業、自宅で作業を行うエンジニアも増えたことにより、リモート環境における対策というのも増えました。

🏠

1. 自宅環境のセキュリティ確保

  • 家族による覗き見防止
  • 作業場所の確保
  • ネットワークセキュリティの確認
🔐

2. VPN接続の必須化

社外から社内システムにアクセスする場合は、多くがVPN接続を使用し通信内容の暗号化を行っています。VPN接続方法は、マニュアル化されており、定められた手順でアクセスし、IDやPWなど必要な認証行為を求められます。

💼

3. 紛失・盗難リスク

会社や現場から貸与されたPC等を通勤で持ち運ぶ機会というのが増えているため、紛失や盗難に遭わないよう対策を取る必要があります。例えば、PCの入った鞄は上棚に置かない、カフェや電車内でPCを立ち上げないです。

コラム ショルダーハッキングに注意

「ショルダーハッキング」とは、肩越しに画面やキーボード入力を覗き見て情報を盗み取る手法のことです。カフェや電車内、オフィスのオープンスペースなど、周囲に人がいる場所でパソコンを使用する際は特に注意が必要です。

対策としては、覗き見防止フィルター（プライバシーフィルター）の使用、周囲の状況確認、パスワード入力時の手元隠しなどがあります。
