Section 1-1

Springの概要について知ろう

この章では、現在Javaの主要フレームワークである「Spring Framework」について、Java基礎文法を理解しているビギナーの方を対象に、わかりやすく解説します。この章を読み終えると「Spring Frameworkの全体像」が把握できるようになります。

フレームワークとは？

フレームワークとは、ソフトウェアやアプリケーション開発を効率的に行うための「骨組み」のことです。

フレームワークを利用すると、開発に必要な基本機能があらかじめ用意されているため、すべてを一から作る必要がありません。その結果、開発にかかる時間とコストを大幅に削減できます。

一方で、フレームワーク特有のルールや使い方を習得する必要があるというデメリットもあります。

✓ メリット
🏗️

骨組みが既に用意されているため、開発時間とコストを削減できる

△ デメリット
📚

フレームワーク固有のルールや使用方法を学ぶ必要がある

SpringとSpring Frameworkの違い

「Spring」と「Spring Framework」は、しばしば同じ意味で使われますが、厳密には少し異なります。

Spring Frameworkは、Java開発における基盤となるフレームワークで、依存性の注入（DI）やアスペクト指向プログラミング（AOP）などのコア機能を提供します。

一方、Springは、「Spring Framework」を含む、多くの機能を提供する「フレームワークの集合体」を指します。

各フレームワークの概要

🚀

Spring Boot

Springアプリケーションを煩雑な設定なしで迅速に作成する機能を提供します。

💾

Spring Data

データベースへのアクセスを簡単にする機能を提供します。

🌐

Spring MVC

Webアプリケーションを簡単に作成する機能を提供します。

⚙️

Spring Batch

大量データの一括処理（バッチ処理）を行う機能を提供します。

🔐

Spring Security

認証・認可・セキュリティ関連の機能を提供します。

🔗

Spring Framework（DI）

依存性注入の機能を提供し、コンポーネント間の結合度を低く保ちます。

🎯

Spring Framework（AOP）

アスペクト指向プログラミングの機能を提供し、横断的関心事を分離します。

本書で扱う機能は、主に「Spring Boot」「Spring MVC」「DI」「AOP」「Spring Security」になります。現時点では「Spring」とは様々な機能（フレームワーク）を提供してくれる集合体だとイメージしてください。

Column サンプルファイルを活用しよう

本書の学習方法としておすすめするのは、サポートページから提供されている「サンプルコード」をダウンロードして、ファイルに貼り付ける方法です。

サンプルコードは動作確認済みです。まずはアプリケーションが動くことを確認し、その後、自分でコードを読み解いて学習することで、打ち間違いによるストレスから解放されます。

おすすめの学習ステップ：

  1. サンプルコードを使用してアプリケーションを動かす
  2. 各ソースコードに自分のメモをコメントで記述する
  3. 各ファイルの繋がりを意識するために、変数やクラス名などを自分なりに修正する
