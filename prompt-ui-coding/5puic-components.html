Section 5-1

ボタンコンポーネント

ボタンは最も頻繁に使うUIコンポーネントです。バリエーションを含めて一度に生成すると効率的です。

ボタンセット生成プロンプト

🔘 ボタンコンポーネント生成
ボタンコンポーネントのHTML/CSSを作成してください。 ## バリエーション 1. プライマリボタン（塗りつぶし、青） 2. セカンダリボタン（アウトライン、青枠） 3. ゴーストボタン（テキストのみ） 4. 危険ボタン（赤、削除用） ## サイズ - sm: 高さ32px、フォント14px、padding 0 12px - md: 高さ40px、フォント14px、padding 0 16px - lg: 高さ48px、フォント16px、padding 0 24px ## 共通仕様 - 角丸: 8px - フォント: 太字 - カーソル: pointer - 無効時: opacity 0.5、カーソル not-allowed - トランジション: 0.2s ease ## 状態 - デフォルト - ホバー - アクティブ（押下時） - フォーカス（アウトライン表示） - 無効（disabled） ## 命名規則 - BEM: .btn, .btn--primary, .btn--sm

生成結果のチェックポイント

確認すべき項目

  • すべてのバリエーションが揃っているか
  • ホバー・アクティブ状態が定義されているか
  • フォーカス時のアウトラインがあるか（アクセシビリティ）
  • disabled状態が適切か
  • クリックしやすいサイズか（最低44px推奨）
Section 5-2

カードコンポーネント

基本カード生成プロンプト

🃏 カードコンポーネント生成
汎用カードコンポーネントのHTML/CSSを作成してください。 ## 構成要素 - サムネイル画像（上部、アスペクト比16:9） - カテゴリバッジ（画像上に重ねて表示） - タイトル（h3、2行まで表示、超過は...） - 説明文（3行まで表示、超過は...） - メタ情報（日付、著者など） - アクションボタン ## デザイン仕様 - 背景: 白 - 角丸: 12px - 影: 軽い影（ホバーで強く） - 内側余白: 16px - 画像とコンテンツの間: 余白なし ## インタラクション - ホバー時: 少し浮き上がる + 影強化 - カード全体がクリック可能（a要素でラップ） - 画像は少しズーム（scale 1.05） ## レスポンシブ - 最小幅: 280px - 最大幅: 400px

カードバリエーション

タイプ 特徴 用途
画像カード 上部に大きな画像 ブログ、商品一覧
横型カード 左に画像、右にテキスト 検索結果、ニュース
テキストカード 画像なし FAQ、機能紹介
プロフィールカード アバター + 名前 + 説明 チーム紹介
価格カード プラン名 + 価格 + 機能リスト 料金表
Section 5-3

ナビゲーション

ヘッダーナビ生成プロンプト

📍 ヘッダーナビゲーション生成
レスポンシブ対応のヘッダーナビゲーションを作成してください。 ## PC表示（768px以上） - ロゴ: 左寄せ - ナビリンク: 中央または右寄せ、5項目 - CTAボタン: 右端 - 高さ: 64px - 背景: 白、下線付き ## モバイル表示（767px以下） - ロゴ: 左寄せ - ハンバーガーボタン: 右寄せ - メニュー: クリックでスライドダウン表示 - メニュー項目: 縦並び、全幅 ## デザイン - リンクホバー: 下線または色変化 - 現在地表示: アクティブ状態のスタイル - 影: スクロール時に追加（任意） ## アクセシビリティ - aria-label設定 - フォーカス表示 - キーボード操作対応

Section 5-4

フォーム要素

入力フィールドセット生成プロンプト

📝 フォーム要素生成
フォーム入力要素のコンポーネントセットを作成してください。 ## 含める要素 1. テキスト入力（text, email, password, tel） 2. テキストエリア 3. セレクトボックス 4. チェックボックス 5. ラジオボタン 6. 日付入力 ## 共通スタイル - ラベル: 入力欄の上、14px、グレー - 入力欄: 幅100%、高さ44px、padding 12px - ボーダー: 1px、グレー、角丸8px - フォーカス時: 青ボーダー、影 - エラー時: 赤ボーダー、エラーメッセージ表示 - プレースホルダー: 薄いグレー ## 状態 - デフォルト - フォーカス - 入力済み - エラー - 無効 ## アクセシビリティ - labelとinputの紐付け - エラーメッセージのaria-describedby

カスタムチェックボックス・ラジオ

☑️ カスタムチェックボックス
カスタムスタイルのチェックボックスを作成してください。 ## 仕様 - ネイティブのinputを非表示 - カスタムの四角形を表示（20x20px） - チェック時: 青背景 + 白いチェックマーク - ラベルは右側に配置 - クリック領域はラベルを含む全体 - フォーカス時のアウトライン表示 - アニメーション付き（チェック時）
注意点

カスタムフォーム要素は見た目は良くなりますが、アクセシビリティを損なわないよう注意が必要です。ネイティブ要素を完全に隠すのではなく、視覚的に隠しつつスクリーンリーダーには認識されるようにしましょう。

まとめ

第5章のまとめ

  • ボタンはバリエーション・サイズ・状態を一度に定義
  • カードは構成要素とインタラクションを明確に指定
  • ナビゲーションはレスポンシブとアクセシビリティを考慮
  • フォーム要素は状態（フォーカス、エラー等）を網羅
  • コンポーネントは再利用可能な形で生成する
