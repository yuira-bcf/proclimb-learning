Section 18-1

@Transactionalの基本

@Transactionalを使って、データベース操作の整合性を保証します。

🔄

@Transactional

トランザクション境界を宣言的に設定

↩️

Rollback

例外発生時に変更を取り消す

TodoService.java（基本的な使い方） 
import org.springframework.transaction.annotation.Transactional;
import org.springframework.stereotype.Service;

@Service
public class TodoService {

    private final TodoMapper todoMapper;
    private final HistoryMapper historyMapper;

    /**
     * クラスレベルで@Transactionalを付与
     * 全publicメソッドにトランザクションが適用される
     */
    @Transactional
    public void save(Todo todo) {
        // ToDoを保存
        todoMapper.insert(todo);

        // 履歴を保存（同一トランザクション内）
        History history = new History();
        history.setTodoId(todo.getId());
        history.setAction("CREATE");
        historyMapper.insert(history);

        // どちらかで例外が発生すると両方ロールバック
    }

    /**
     * 読み取り専用トランザクション
     */
    @Transactional(readOnly = true)
    public List<Todo> findAll() {
        return todoMapper.findAll();
    }
}

@Transactionalの属性

属性説明デフォルト
readOnly読み取り専用（最適化のヒント）false
timeoutタイムアウト（秒）-1（制限なし）
rollbackForロールバック対象の例外RuntimeException
propagationトランザクションの伝播REQUIRED
Section 18-2

ロールバックの制御

ロールバックの制御例 
@Service
public class TodoService {

    /**
     * チェック例外でもロールバックする
     */
    @Transactional(rollbackFor = Exception.class)
    public void saveWithRollback(Todo todo) throws Exception {
        todoMapper.insert(todo);

        if (todo.getTitle().contains("error")) {
            throw new Exception("強制エラー");  // ロールバックされる
        }
    }

    /**
     * 特定の例外ではロールバックしない
     */
    @Transactional(noRollbackFor = BusinessException.class)
    public void saveWithNoRollback(Todo todo) {
        todoMapper.insert(todo);

        if (someCondition) {
            // BusinessExceptionが発生してもコミットされる
            throw new BusinessException("警告メッセージ");
        }
    }

    /**
     * 一括処理（一部失敗しても続行）
     */
    @Transactional
    public BulkResult bulkInsert(List<Todo> todoList) {
        int success = 0;
        int failure = 0;

        for (Todo todo : todoList) {
            try {
                todoMapper.insert(todo);
                success++;
            } catch (Exception e) {
                failure++;
                // 個別の失敗は無視して続行
            }
        }

        return new BulkResult(success, failure);
    }
}
warningデフォルトのロールバック動作

RuntimeException（非チェック例外）は自動的にロールバックされます。
Exception（チェック例外）はデフォルトではロールバックされません。
チェック例外でもロールバックしたい場合はrollbackForを指定してください。

Section 18-3

トランザクションの伝播

トランザクション伝播の例 
import org.springframework.transaction.annotation.Propagation;

@Service
public class TodoService {

    private final TodoMapper todoMapper;
    private final AuditService auditService;

    /**
     * REQUIRED（デフォルト）: 既存トランザクションがあれば参加、なければ新規作成
     */
    @Transactional(propagation = Propagation.REQUIRED)
    public void save(Todo todo) {
        todoMapper.insert(todo);
        auditService.log("TODO_CREATED", todo.getId());  // 同一トランザクション
    }

    /**
     * REQUIRES_NEW: 常に新しいトランザクションを作成
     * 親トランザクションとは独立してコミット/ロールバック
     */
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void saveIndependent(Todo todo) {
        todoMapper.insert(todo);
        // 親でロールバックされてもこの処理はコミットされる
    }
}

@Service
public class AuditService {

    /**
     * REQUIRES_NEW: 監査ログは独立してコミット
     * 本体が失敗しても監査ログは残す
     */
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void log(String action, Integer targetId) {
        auditMapper.insert(new AuditLog(action, targetId));
    }
}

伝播属性（Propagation）

属性説明
REQUIRED既存があれば参加、なければ新規作成（デフォルト）
REQUIRES_NEW常に新規トランザクションを作成
SUPPORTS既存があれば参加、なければトランザクションなし
MANDATORY既存トランザクション必須（なければ例外）
NOT_SUPPORTEDトランザクションなしで実行
NEVERトランザクション禁止（あれば例外）
lightbulb@Transactionalの注意点
  • publicメソッドにのみ適用される
  • 同一クラス内のメソッド呼び出しでは適用されない（プロキシの制限）
  • インターフェース経由で呼び出す必要がある

📝 基本プロンプト 
Spring Boot 3.xのToDoアプリにトランザクション管理機能を追加してください。

【要件】
- TodoServiceに@Transactionalを適用してデータベース操作の整合性を保証
- ToDoと履歴を同一トランザクションで保存し、どちらかで例外が発生したら両方ロールバック
- 読み取り専用メソッドには@Transactional(readOnly = true)を適用
- rollbackFor属性でチェック例外もロールバック対象に設定
- 監査ログ記録にはREQUIRES_NEWを使用し、本体がロールバックしても監査ログは残す

【技術スタック】
- Spring Boot 3.x
- Spring Transaction
- MyBatis
🎯 ステップ別プロンプト 
# Step 1: 基本的なトランザクション適用
TodoServiceのcreate、update、deleteメソッドに@Transactionalを追加してください。

# Step 2: 読み取り専用トランザクション
findAll、findByIdなど読み取り専用メソッドに
@Transactional(readOnly = true)を適用してください。

# Step 3: ロールバック制御
チェック例外でもロールバックするようrollbackFor = Exception.classを設定し、
特定の業務例外ではロールバックしないようnoRollbackForを設定してください。

# Step 4: トランザクション伝播の設定
監査ログ記録サービスにPropagation.REQUIRES_NEWを設定し、
本体のトランザクションとは独立して処理されるようにしてください。
lightbulb プロンプトのコツ

「@Transactionalが同一クラス内のメソッド呼び出しで動作しない理由を解説して」のように、
プロキシの仕組みについて質問すると、トランザクションの理解が深まります。
