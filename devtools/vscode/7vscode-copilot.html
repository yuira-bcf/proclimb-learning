Section 7-1

GitHub Copilotとは

GitHub Copilotは、GitHub（Microsoft）とOpenAIが共同開発したAIコーディングアシスタントです。コードの文脈を理解し、次に書くべきコードをリアルタイムで提案してくれる「AIペアプログラマー」として、世界中の開発者に利用されています。

概要と仕組み

GitHub Copilotは、大規模言語モデル（LLM）をベースにしています。エディタで入力中のコードや、開いているファイルの内容、コメント、関数名、変数名などのコンテキスト（文脈）を分析し、続きのコードを予測・提案します。

GitHub Copilotの内部で使われるモデルは継続的にアップデートされており、GPT-4ベースのモデルやClaude等のモデルを選択可能になっています（2025年時点）。

Section 7-2

Copilotの導入と設定

GitHub CopilotをVSCodeで使うには、拡張機能のインストールとGitHubアカウントとの連携が必要です。手順を見ていきましょう。

Step 1: GitHubアカウントの準備

GitHub Copilotを使うには、GitHubアカウントが必要です。まだ持っていない場合は、github.com で無料アカウントを作成してください。アカウント作成後、Copilotの利用を有効化します。

  1. GitHubにログイン
  2. 右上のアイコン → Settings → 左メニューの「Copilot」
  3. 利用するプラン（Free / Pro）を選択して有効化

Step 2: VSCode拡張機能のインストール

VSCodeの拡張機能マーケットプレイスから、以下の拡張機能をインストールします。

  1. VSCodeのサイドバーで「拡張機能」アイコンをクリック（Ctrl+Shift+X）
  2. 検索欄に「GitHub Copilot」と入力
  3. 「GitHub Copilot」をインストール（チャット機能も含まれます）
info POINT

「GitHub Copilot」拡張機能をインストールすると、コード補完機能とCopilot Chat機能の両方が有効になります。以前はCopilot Chatは別の拡張機能でしたが、現在は統合されています。

Step 3: GitHubアカウントとの連携

拡張機能インストール後、初回使用時にGitHubアカウントとの認証を行います。

  1. VSCodeの右下にGitHubログインの通知が表示される
  2. 「Sign in to GitHub」をクリック
  3. ブラウザが開くので、GitHubにログインして認証を許可
  4. VSCodeに戻り、ステータスバーにCopilotアイコンが表示されれば完了

設定画面

Copilotの動作は、VSCodeの設定から調整できます。

JSON settings.json（Copilot関連） 
{
  // Copilotのインライン候補を有効化
  "github.copilot.enable": {
    "*": true,
    "plaintext": false,    // プレーンテキストでは無効
    "markdown": true,
    "yaml": true
  },

  // 候補の自動表示
  "editor.inlineSuggest.enabled": true
}
Section 7-3

コード補完機能

GitHub Copilotの最も基本的な機能が、インラインコード補完です。コードを書いている途中で、AIが次に書くべきコードを灰色のテキストで提案してくれます。

インライン候補の表示と受け入れ

Copilotの候補は自動的に表示されます。表示された候補に対して、以下の操作が可能です。

操作 キー 説明
候補を受け入れるTab表示されている候補全体を採用
単語単位で部分受入Ctrl+→候補を1単語ずつ受け入れる
候補を拒否Esc現在の候補を非表示にする
次の候補Alt+]別の候補を表示
前の候補Alt+[1つ前の候補に戻る

補完の活用例

たとえば、以下のようにコメントを書くだけで、Copilotが関数全体を提案してくれます。

JavaScript コメントからの関数生成例 
// 配列の中から最大値を返す関数
function findMax(arr) {
  // ← ここでCopilotが以下のコードを提案:
  //    return Math.max(...arr);
  // Tab を押すと候補が採用されます
}
JavaScript 関数名からの推測例 
// 関数名を書くだけで、Copilotが中身を提案
function validateEmail(email) {
  // Copilot提案:
  //   const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  //   return regex.test(email);
}

function formatDate(date) {
  // Copilot提案:
  //   const y = date.getFullYear();
  //   const m = String(date.getMonth() + 1).padStart(2, '0');
  //   const d = String(date.getDate()).padStart(2, '0');
  //   return `${y}-${m}-${d}`;
}
warning 注意

Copilotの提案は必ずしも正しいとは限りません。AIが生成したコードは必ず内容を確認し、テストを行ってから本番コードに採用してください。特にセキュリティに関わるコード（認証・暗号化など）は慎重にレビューしましょう。

Section 7-4

Copilot Chat

Copilot Chatは、VSCodeのサイドパネルやインラインで、AIと対話しながらコーディングできる機能です。コードの説明、修正、リファクタリング、テスト生成など、さまざまなタスクをチャット形式で依頼できます。

チャットパネルの開き方

  • サイドパネル：アクティビティバーのチャットアイコン、または Ctrl+Alt+I
  • インラインチャット：コード上で Ctrl+I（選択したコードについて直接質問）
  • クイックチャット：Ctrl+Shift+I（ウィンドウ上部にポップアップ）

コンテキスト指定

Copilot Chatでは、コンテキスト変数を使って、AIに参照させる情報を指定できます。

変数 説明 使い方例
@workspace ワークスペース全体のコードを参照 @workspace このプロジェクトの構成を教えて
#file 特定のファイルを参照 #file:app.js のエラーを修正して
#selection 選択中のコードを参照 この #selection を最適化して
@terminal ターミナルの出力を参照 @terminal このエラーの原因は？
Copilot Chatでできること
💬 コード説明 「このコードは何をしている？」
🔧 バグ修正 「このエラーを修正して」
🔄 リファクタリング 「このコードを改善して」
📋 テスト生成 「ユニットテストを書いて」
📄 ドキュメント生成 「JSDocコメントを追加して」
🎓 学習支援 「asyncの使い方を教えて」

スラッシュコマンド

Copilot Chatでは、スラッシュコマンドを使ってよく使う操作を素早く実行できます。

  • /explain - 選択したコードの説明
  • /fix - コードの問題を修正
  • /tests - テストコードの生成
  • /doc - ドキュメントコメントの生成
  • /optimize - コードの最適化提案
  • /new - 新しいファイルやプロジェクトの生成
lightbulb ヒント

Copilot Chatのエージェントモード（Copilot Pro以上）では、AIがファイルの作成・編集、ターミナルコマンドの実行まで自律的に行えます。たとえば「Expressでシンプルなサーバーを作って」と依頼すると、必要なファイルを一括で生成してくれます。

Section 7-5

効果的なプロンプトの書き方

GitHub Copilotの出力品質は、プロンプト（指示・文脈）の質に大きく左右されます。適切なプロンプトを書くことで、より正確で実用的なコードを生成できます。

コメント駆動開発

関数やクラスを書く前に、詳細なコメントを記述することで、Copilotに明確な指示を与えることができます。これは「コメント駆動開発」とも呼ばれます。

JavaScript 良いコメントの例 
// ユーザーの配列を受け取り、年齢が18歳以上のユーザーだけをフィルタし、
// 名前のアルファベット順にソートして、
// 名前と年齢だけを含む新しい配列を返す関数
function getAdultUsersSorted(users) {
  // Copilotがコメントの内容に基づいて正確なコードを提案
  return users
    .filter(user => user.age >= 18)
    .sort((a, b) => a.name.localeCompare(b.name))
    .map(user => ({ name: user.name, age: user.age }));
}

具体的な指示を書く

曖昧な指示よりも具体的な指示の方が、良い結果が得られます。

悪い例 良い例
// データを処理する // CSVデータをパースしてJSON配列に変換する
// APIを呼ぶ // fetchを使ってGET /api/usersを呼び、JSONレスポンスを返す非同期関数
// バリデーション // メールアドレスの形式チェック。@と.を含み、空文字でないことを確認

コンテキストを与える

Copilotは開いているファイルの内容や周囲のコードを参考にします。以下の工夫でより良い提案が得られます。

  • 関連ファイルを開いておく：型定義ファイルや関連モジュールを別タブで開く
  • インポート文を先に書く：使用するライブラリをインポートしておくと、そのAPIに基づいた提案が得られる
  • 型情報を付ける：TypeScriptの型アノテーションやJSDocを書くと精度が上がる
  • テストを先に書く：テストケースを先に書くことで、実装コードの提案精度が向上する（TDD）
効果的なCopilot活用フロー
📝 1. コメントで意図を明記 具体的な説明を書く
➔
🤖 2. Copilotが提案 候補コードが表示
➔
🔍 3. レビュー・修正 内容を確認して調整
➔
✅ 4. テスト実行 動作確認して採用
info POINT

Copilotを「自分の代わりにコードを書いてくれるツール」ではなく、「一緒にコードを考えてくれるパートナー」として活用しましょう。AIの提案をそのまま受け入れるのではなく、自分で内容を理解し、必要に応じて修正・改善する姿勢が大切です。

Exercise

練習問題

Q1

GitHub Copilotのインラインコード補完で、表示された候補を「全体を受け入れる」場合と「1単語ずつ部分的に受け入れる」場合のキー操作をそれぞれ答えてください。

Q2

Copilot Chatで使えるコンテキスト変数 @workspace と #file の違いを説明してください。

Q3

Copilotに効果的なプロンプトを与えるために、「コメント駆動開発」ではどのような工夫をしますか？具体例を挙げながら説明してください。

Answer

解説・解答

Q1 の解答

候補全体を受け入れるには Tab キーを押します。1単語ずつ部分的に受け入れるには Ctrl+→（macOSでは Cmd+→）を押します。部分受入は、提案の一部だけを採用したい場合に便利です。

Q2 の解答

@workspace は、ワークスペース（プロジェクト）全体のコードをコンテキストとしてCopilotに渡します。プロジェクトの構成や全体的な質問に適しています。
#file は、特定のファイルを指定してそのファイルの内容をコンテキストとして渡します。「#file:app.js のエラーを修正して」のように、特定のファイルに関する質問や操作に使います。
@workspace が広い範囲をカバーするのに対し、#file はピンポイントでファイルを指定できる点が違いです。

Q3 の解答

コメント駆動開発では、関数やクラスを実装する前に、処理内容を詳細に記述したコメントを書きます。これにより、Copilotがコメントの内容を読み取り、意図に沿ったコードを提案してくれます。

具体例：// 配列を受け取り、重複を除去して昇順にソートし、先頭5件だけを返す関数 のように、入力・処理内容・出力を具体的にコメントで記述します。「データを処理する」のような曖昧なコメントではなく、何を入力として受け取り、どのような処理を行い、何を返すかを明確にすることが重要です。
