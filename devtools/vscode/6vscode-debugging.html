Section 6-1

デバッグとは

デバッグ（Debug）とは、プログラム中の不具合（バグ）を見つけて修正する作業のことです。プログラミングにおいて最も時間がかかる作業の一つであり、効率的なデバッグ手法を身につけることは、開発者にとって非常に重要なスキルです。

デバッグの意義

プログラムは書いた通りに動きますが、それが必ずしも意図した通りとは限りません。バグの原因としては、ロジックの間違い、変数の初期化忘れ、型の不一致、非同期処理のタイミングなどさまざまです。デバッグは、「コードが実際にどう動いているか」を調べ、期待と現実のギャップを特定する作業です。

print文デバッグ vs ブレークポイントデバッグ

デバッグ手法には大きく2つのアプローチがあります。

デバッグ手法の比較
📄 print文デバッグ console.log() で値を出力
手軽だが情報が限定的
コードを都度追加・削除が必要
大規模コードでは非効率
🔴 ブレークポイントデバッグ 実行を一時停止して調査
変数の値をリアルタイム確認
ステップ実行で処理を追跡
コードを変更せずにデバッグ可能
info POINT

console.log() でのデバッグは手軽ですが、VSCodeのブレークポイントデバッグを使えば、プログラムの実行を任意の箇所で止めて、その時点での変数の値やコールスタック（関数の呼び出し履歴）をすべて確認できます。本格的な開発では、ブレークポイントデバッグを使いこなせるようになりましょう。

Section 6-2

ブレークポイントの使い方

ブレークポイント（Breakpoint）とは、プログラムの実行を一時停止させる「目印」のことです。デバッガーがブレークポイントの設定された行に到達すると、自動的に実行が中断され、変数の状態や処理の流れを確認できます。

ブレークポイントの設定方法

VSCodeでは、コードエディタの行番号の左側（ガター）をクリックするだけでブレークポイントを設定できます。設定すると赤い丸が表示されます。

  • 設定：行番号の左側をクリック（赤い丸が表示される）
  • 解除：赤い丸をもう一度クリック
  • 一時無効化：右クリック → 「ブレークポイントを無効にする」

条件付きブレークポイント

ブレークポイントに条件を設定すると、特定の条件を満たした場合のみ実行が停止します。ループ処理で特定の回数だけ止めたい場合などに便利です。

  • 行番号の左側を右クリック → 「条件付きブレークポイントの追加」
  • 例：i === 5 や user.name === "田中" のように条件式を入力
JavaScript 条件付きブレークポイントの活用例 
// ループの中で i === 50 のときだけ停止させたい場合
for (let i = 0; i < 100; i++) {
  // ← ここに条件付きブレークポイント: i === 50
  const result = processItem(items[i]);
  console.log(result);
}

ログポイント

ログポイント（Logpoint）は、ブレークポイントの一種で、実行を停止せずにデバッグコンソールにメッセージを出力します。コードに console.log() を書かなくても同様の効果が得られます。

  • 行番号の左側を右クリック → 「ログポイントの追加」
  • メッセージ例：変数iの値: {i}, resultの値: {result}
  • {} で囲んだ式は実行時の値に置換されます
lightbulb ヒント

ログポイントはダイヤモンド型（ひし形）のアイコンで表示されます。通常のブレークポイント（赤い丸）と区別できるので覚えておきましょう。本番環境に近い状態でのデバッグでは、実行を止めたくない場面が多いため、ログポイントが非常に役立ちます。

Section 6-3

launch.jsonの設定

デバッグの設定は .vscode/launch.json ファイルに記述します。このファイルでは、デバッグ対象のプログラム、使用するデバッガー、起動方法などを定義します。

launch.jsonの作成方法

初回のデバッグ設定は、以下の手順で行います。

  1. サイドバーの「実行とデバッグ」アイコン（再生マークに虫のアイコン）をクリック
  2. 「launch.jsonファイルを作成します」リンクをクリック
  3. 使用する環境（Node.js、Chrome、Pythonなど）を選択
  4. 自動生成された launch.json を確認・編集

主要プロパティ

JSON .vscode/launch.json 
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",          // デバッガーの種類
      "request": "launch",     // "launch"（起動）or "attach"（接続）
      "name": "Node.js起動",   // 構成名（表示名）
      "program": "${workspaceFolder}/app.js",  // 実行ファイル
      "cwd": "${workspaceFolder}",             // 作業ディレクトリ
      "console": "integratedTerminal",         // 出力先
      "env": {                 // 環境変数
        "NODE_ENV": "development"
      },
      "args": ["--port", "3000"]  // コマンドライン引数
    }
  ]
}
プロパティ 説明 例
typeデバッガーの種類"node", "chrome", "python"
request起動方法"launch"（新規起動）, "attach"（既存プロセスに接続）
name構成の表示名"サーバー起動"
program実行するファイルのパス"${workspaceFolder}/index.js"
env環境変数の設定{"PORT": "8080"}
argsプログラムへの引数["--verbose"]

言語別のテンプレート例

JSON Chrome（フロントエンド）デバッグ 
{
  "type": "chrome",
  "request": "launch",
  "name": "Chrome起動",
  "url": "http://localhost:3000",
  "webRoot": "${workspaceFolder}/src"
}

{
  "type": "debugpy",
  "request": "launch",
  "name": "Python起動",
  "program": "${workspaceFolder}/main.py",
  "console": "integratedTerminal"

Section 6-4

デバッグパネルの操作

デバッグを開始すると、VSCodeの画面上部にデバッグツールバーが表示されます。ここからプログラムの実行を制御し、変数の値やコールスタックを確認できます。

デバッグツールバーのボタン

デバッグ中に使う6つの主要ボタンを理解しましょう。

操作 ショートカット 説明
続行（Continue）F5次のブレークポイントまで実行を再開
ステップオーバーF10現在の行を実行し、次の行へ（関数の中には入らない）
ステップインF11現在の行を実行し、関数の中に入る
ステップアウトShift+F11現在の関数を抜けて呼び出し元に戻る
再起動Ctrl+Shift+F5デバッグセッションを再起動
停止Shift+F5デバッグセッションを終了
ステップ実行の動作イメージ
▶ F5 続行 次のBPまで実行
➔
➡ F10 ステップオーバー 次の行へ
➔
⬇ F11 ステップイン 関数の中へ
➔
⬆ Shift+F11 ステップアウト 関数から脱出

変数パネルとウォッチ

デバッグ中、左サイドバーの「実行とデバッグ」ビューには以下の情報が表示されます。

  • 変数（VARIABLES）：現在のスコープで定義されている変数と値の一覧
  • ウォッチ（WATCH）：監視したい変数や式を追加でき、値の変化をリアルタイムに追跡
  • コールスタック（CALL STACK）：現在の実行位置に至るまでの関数呼び出しの履歴
  • ブレークポイント（BREAKPOINTS）：設定されているすべてのブレークポイントの一覧と有効/無効の切り替え
デバッグビューの構成要素
📊 VARIABLES ローカル変数・グローバル変数の値を表示
👁 WATCH 指定した式の値をリアルタイム監視
📚 CALL STACK 関数呼び出しの履歴（どこから呼ばれたか）
🔴 BREAKPOINTS ブレークポイントの一覧管理
lightbulb ヒント

デバッグ中にコード上の変数にマウスカーソルを合わせると、ホバー表示で現在の値を確認できます。また、デバッグコンソール（Debug Console）では任意の式を入力して評価結果を確認することもできます。

Section 6-5

JavaScript/Node.jsのデバッグ実践

ここでは、実際のJavaScriptコードを使ってVSCodeのデバッグ機能を体験してみましょう。VSCodeにはNode.jsのデバッガーが標準で組み込まれているため、追加のインストールは不要です。

デバッグ対象のコード

以下のような簡単なNode.jsスクリプトを用意します。このコードには意図的にバグが含まれています。

JavaScript app.js 
// 商品の合計金額を計算する関数
function calculateTotal(items) {
  let total = 0;
  for (let i = 0; i <= items.length; i++) {  // バグ: <= ではなく < が正しい
    total += items[i].price * items[i].quantity;
  }
  return total;
}

// 税込み価格を計算する関数
function addTax(amount, taxRate) {
  return amount * (1 + taxRate);
}

// テストデータ
const cart = [
  { name: "ノートPC", price: 80000, quantity: 1 },
  { name: "マウス", price: 3000, quantity: 2 },
  { name: "キーボード", price: 8000, quantity: 1 }
];

// 実行
const subtotal = calculateTotal(cart);
const total = addTax(subtotal, 0.1);
console.log(`税込合計: ${total}円`);

デバッグの手順

  1. ブレークポイント設定：calculateTotal 関数の total += ... の行（5行目）にブレークポイントを設定
  2. デバッグ開始：F5 を押す。初回はデバッガーの選択画面が出るので「Node.js」を選択
  3. 変数確認：ブレークポイントで停止したら、VARIABLESパネルで i, total, items[i] の値を確認
  4. ステップ実行：F10（ステップオーバー）で1行ずつ進めながら i と total の変化を追跡
  5. バグ発見：i === 3 のとき items[3] が undefined になり、エラーが発生することを確認
  6. 修正：ループ条件を i <= items.length から i < items.length に修正

デバッグコンソールの活用

デバッグ中にデバッグコンソール（Debug Console）を使うと、任意の式を評価できます。停止中に以下のような式を入力して試してみましょう。

Debug Console デバッグコンソールでの式評価 
> items.length        // → 3
> items[i]            // → 現在のアイテム
> total               // → 現在の合計値
> items.map(x => x.name)  // → ["ノートPC", "マウス", "キーボード"]
warning 注意

Node.jsのデバッグを行うには、PCにNode.jsがインストールされている必要があります。インストールされていない場合は、Node.js公式サイトからLTS版をダウンロードしてインストールしてください。

Exercise

練習問題

Q1

VSCodeのデバッグ操作で、「ステップオーバー（F10）」と「ステップイン（F11）」の違いを説明してください。

Q2

launch.json の "request" プロパティに設定できる2つの値と、それぞれの違いを説明してください。

Q3

通常のブレークポイントとログポイントの違いを説明してください。また、ログポイントが特に有用な場面を1つ挙げてください。

Answer

解説・解答

Q1 の解答

ステップオーバー（F10）は、現在の行を実行して次の行に進みます。もし現在の行に関数呼び出しがあっても、その関数の中には入らず、関数全体を実行してから次の行に進みます。
ステップイン（F11）は、現在の行を実行して次の行に進みますが、もし関数呼び出しがある場合は、その関数の内部に入って1行目から実行を追跡します。関数内部の処理を詳しく調べたいときに使います。

Q2 の解答

"launch"：VSCodeがプログラムを新しく起動してデバッガーを接続します。通常のデバッグ開始時に使用します。
"attach"：すでに実行中のプロセスにデバッガーを後から接続します。たとえば、別のターミナルで起動したNode.jsサーバーにデバッガーを接続して調査する場合に使います。

Q3 の解答

通常のブレークポイントは、その行に到達するとプログラムの実行が一時停止します。停止した状態で変数やコールスタックを調査できます。
ログポイントは、プログラムを停止させずに、デバッグコンソールにメッセージを出力します。コードに console.log() を追加するのと同じ効果を、コードを変更せずに実現できます。
有用な場面の例：APIサーバーのデバッグで、リクエスト処理中に実行を止めるとタイムアウトが発生してしまう場合、ログポイントを使えば実行を止めずに変数の値を確認できます。
