Section 8-1

AI開発ツールの全体像

2024〜2025年にかけて、AIを活用した開発ツール（AIコーディングアシスタント）は急速に進化しました。前章で学んだGitHub Copilotに加え、ターミナルベースのAIエージェントが登場し、開発者のワークフローを大きく変えつつあります。ここでは主要なツールを比較しながら全体像を把握しましょう。

主要AI開発ツールの比較
🤖 GitHub Copilot VSCode内蔵型
コード補完 + チャット
エージェントモード対応
Microsoft / GitHub
💻 OpenAI Codex CLI ターミナルベース
codexコマンドで実行
自律的なコード生成・編集
OpenAI
⚡ Claude Code ターミナルベース
claudeコマンドで実行
MCP連携・高精度な推論
Anthropic
🔹 Cursor VSCode派生エディタ
AI機能がネイティブ統合
Composer・エージェント機能
Cursor, Inc.

ツールの分類

AI開発ツールは、動作環境によって大きく2つに分類できます。

分類 ツール例 特徴
エディタ統合型 GitHub Copilot, Cursor エディタ内でAI機能を使用。GUI操作が中心
CLI（ターミナル）型 Codex CLI, Claude Code ターミナルからAIに指示。ファイル操作やコマンド実行も自律的に行う

この章では、CLI型のAIエージェントであるOpenAI Codex CLIとClaude Codeを中心に学びます。これらはVSCodeの統合ターミナルからも利用でき、第5章で学んだターミナル操作の知識が活きてきます。

Section 8-2

OpenAI Codex CLIの導入

OpenAI Codex CLI（コマンド名：codex）は、OpenAIが提供するターミナルベースのAIコーディングエージェントです。自然言語で指示を出すと、コードの生成・編集・ファイル操作・シェルコマンドの実行まで自律的に行ってくれます。

概要

Codex CLIはオープンソースで開発されており、OpenAIのAPIを利用して動作します。主な特徴は以下の通りです。

  • ターミナルから codex コマンドで起動
  • 自然言語で開発タスクを依頼可能
  • サンドボックス環境でコードを安全に実行
  • ファイルの読み書き、コマンド実行を自律的に行う
  • Git連携で変更の差分確認・コミットも可能

インストール

Node.js（v22以上）がインストールされた環境で、npmを使ってインストールします。

Shell Codex CLIのインストール 
# グローバルインストール
npm install -g @openai/codex

# バージョン確認
codex --version

# OpenAI APIキーの設定
export OPENAI_API_KEY="sk-xxxxxxxxxxxx"
warning 注意

Codex CLIの利用にはOpenAIのAPIキーが必要です。APIキーはOpenAI Platformで取得できます。APIキーは秘密情報なので、コードに直接書き込まず、環境変数で管理してください。

基本コマンドと使い方

Shell Codex CLIの基本的な使い方 
# 対話モードで起動
codex

# プロンプトを直接指定して実行
codex "Expressでシンプルなサーバーを作成して"

# 自動承認モード（確認なしで実行）
codex --approval-mode full-auto "package.jsonを作成して"

# 静かモード（確認のみ表示）
codex -q "READMEを日本語で作成して"

Codex CLIは3つの承認モードを提供しています。

モード 説明 用途
suggest（デフォルト） ファイル変更・コマンド実行の前に確認 安全に操作したいとき
auto-edit ファイル編集は自動、コマンドは確認 ファイル編集を信頼する場合
full-auto すべて自動実行（サンドボックス内） 実験的な作業向け
info POINT

Codex CLIはサンドボックス（隔離環境）内でコマンドを実行するため、システムを破壊するリスクが低い設計になっています。ただし、full-auto モードではファイルの変更が即座に反映されるため、重要なファイルがあるディレクトリでは注意して使いましょう。

Section 8-3

Claude Codeの導入と活用

Claude Code（コマンド名：claude）は、Anthropicが提供するターミナルベースのAIコーディングエージェントです。Claudeモデルの高い推論能力を活かし、複雑なコーディングタスクやプロジェクト全体の理解を伴う作業に強みを持ちます。

インストール

Shell Claude Codeのインストール 
# npmでグローバルインストール
npm install -g @anthropic-ai/claude-code

# バージョン確認
claude --version

# 初回起動（認証を含む）
claude

初回起動時にAnthropicアカウントとの認証が行われます。ブラウザが開くので、ログインして認証を完了してください。

基本コマンド

Shell Claude Codeの基本操作 
# 対話モードで起動
claude

# プロンプトを指定して実行
claude "このプロジェクトの構成を説明して"

# ファイルを指定して質問
claude "app.jsのバグを修正して"

# パイプで入力を渡す
cat error.log | claude "このエラーの原因を教えて"

# 直接プロンプトを実行して終了
claude -p "package.jsonの依存関係を一覧にして"

Claude Codeの特徴的な機能

  • プロジェクト全体の理解：ワークスペース内の全ファイルを自動的に把握し、コードベース全体の文脈を理解
  • マルチファイル編集：複数のファイルにまたがる変更を一括で提案・実行
  • Git連携：変更内容の差分確認、コミットメッセージの自動生成
  • コマンド実行：テスト実行やビルドなど、必要なシェルコマンドを自動で実行
  • MCP連携：外部ツールやサービスとの連携が可能

MCP（Model Context Protocol）とは

MCP（Model Context Protocol）は、Anthropicが策定したAIモデルと外部ツール・データソースを接続するためのオープンプロトコルです。Claude Codeに外部サービスの機能を追加できます。

MCP（Model Context Protocol）の仕組み
⚡ Claude Code AIエージェント
↔
🔌 MCPサーバー ツール・データ提供
↔
🌐 外部サービス GitHub / DB / API

MCPを使うと、Claude Codeから以下のような外部サービスにアクセスできるようになります。

  • GitHub MCP：Issue作成・PR作成・コードレビューなど
  • ファイルシステムMCP：安全なファイル読み書きアクセス
  • データベースMCP：SQLクエリの実行やスキーマ確認
  • Web検索MCP：最新情報の検索・取得
JSON .claude/settings.json（MCP設定例） 
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/project"]
    }
  }
}
info POINT

MCPはAIと外部ツールの「共通語」のようなものです。MCPに対応したサーバーを追加するだけで、Claude Codeの能力を拡張できます。コミュニティ製のMCPサーバーも多数公開されており、自分でMCPサーバーを作成することも可能です。

Section 8-4

AI開発の実践ワークフロー

AIツールは開発のあらゆるフェーズで活用できます。ここでは、要件定義からリリースまでの開発フロー全体で、AIエージェントをどのように活用するかを見ていきましょう。

AI活用 開発ワークフロー
📄 1. 要件定義 AIに壁打ち・整理
➔
🛠 2. 設計 AIにDB設計・API設計を相談
➔
💻 3. 実装 AIにコード生成を依頼
➔
🔍 4. レビュー AIにコードレビューを依頼

1. 要件定義フェーズ

AIエージェントは要件の整理やアイデアの壁打ちに活用できます。

Shell 要件定義の壁打ち例 
# Claude Codeで要件整理
claude "Todoアプリを作りたい。必要な機能を洗い出して、
画面一覧とデータ構造を提案して"

# Codex CLIで技術選定
codex "React + TypeScript + Firebase でTodoアプリを作る場合の
プロジェクト構成を提案して"

2. 設計フェーズ

データベース設計やAPI設計をAIに相談できます。

Shell 設計の相談例 
# API設計の提案
claude "REST APIのエンドポイント設計をして。
リソース: users, todos, categories
認証: JWTトークン方式"

3. 実装フェーズ

最もAIの恩恵を受けるフェーズです。コードの自動生成、テスト作成、ドキュメント生成などを行えます。

Shell 実装の依頼例 
# プロジェクトの初期セットアップ
codex "Express + TypeScript のプロジェクトを初期化して、
基本的なフォルダ構成を作成して"

# 機能の実装
claude "src/routes/todo.ts にTodoのCRUD APIを実装して。
バリデーションとエラーハンドリングも含めて"

# テストの作成
claude "src/routes/todo.ts のユニットテストを
Jest で作成して"

4. レビュー・改善フェーズ

AIによるコードレビューで、バグやセキュリティの問題を事前に検出できます。

Shell レビューの依頼例 
# コードレビュー
claude "このプロジェクトのコードをレビューして。
セキュリティ・パフォーマンス・可読性の観点で問題点を指摘して"

# リファクタリング
claude "src/utils/ のヘルパー関数を
TypeScript のベストプラクティスに沿ってリファクタリングして"
Section 8-5

AIツール活用のベストプラクティス

AIツールは強力ですが、万能ではありません。効果的に活用するためには、いくつかの重要な原則と注意点を理解しておく必要があります。

セキュリティ上の注意点

  • APIキーや秘密情報の取り扱い：AIツールにAPIキー、パスワード、トークンなどの秘密情報を入力しないでください。環境変数や.envファイルで管理し、.gitignoreに必ず含めましょう
  • 機密コードの入力：社内の機密性の高いコードをAIツールに送信する際は、会社のセキュリティポリシーを確認してください。多くのAIツールはコードをモデル学習に使わない設定がありますが、確認が必要です
  • 生成コードの脆弱性チェック：AIが生成したコードにはセキュリティ上の脆弱性が含まれる可能性があります。特に認証、暗号化、入力バリデーションに関するコードは入念にチェックしましょう
warning 重要

AIが生成したコードをそのまま本番環境にデプロイしないでください。必ず人間がレビューし、テストを行い、セキュリティチェックを通してから使用しましょう。

コードレビューの重要性

AIが生成したコードは、以下の観点で必ずレビューしましょう。

  • 正確性：コードが要件を正しく満たしているか
  • セキュリティ：SQLインジェクションやXSSなどの脆弱性がないか
  • パフォーマンス：不必要なループや非効率な処理がないか
  • 可読性：他の開発者が理解しやすいコードか
  • テスト可能性：テストが書きやすい構造になっているか
  • ライセンス：AIが学習元のコードをそのまま出力していないか

AIの限界を理解する

AIツールには以下のような限界があることを常に意識しましょう。

限界 説明 対策
ハルシネーション 存在しないAPIやライブラリを使ったコードを生成する場合がある 公式ドキュメントで確認する
古い情報 学習データが最新でない場合、非推奨の書き方を提案する 最新のドキュメントを参照する
文脈の限界 大規模プロジェクトの全体像を完全に把握できない場合がある 必要な文脈を明示的に伝える
テスト不足 AIが生成したコードにバグやエッジケースの見落としがある 必ず自分でテストを実行する

効果的な活用の心得

  • AIは「ツール」であり「代替」ではない：AIが書いたコードを理解できるだけの知識は必要です。基礎学習を怠らないようにしましょう
  • 小さく試して確認する：大きなタスクを一度にAIに任せるのではなく、小さな単位に分けて結果を確認しながら進めましょう
  • バージョン管理を活用する：AIによる変更はGitで管理し、いつでも元に戻せるようにしておきましょう
  • 複数のツールを使い分ける：補完にはCopilot、大規模な変更にはClaude Code、というように場面に応じて使い分けましょう
school まとめ

AI開発ツールは、開発者の生産性を飛躍的に向上させる強力な味方です。しかし、最終的な責任はコードを書く（採用する）人間にあります。AIの提案を鵜呑みにせず、理解し、検証し、改善するというプロセスを大切にしましょう。AIと人間が協力することで、より良いソフトウェアを、より速く作ることができます。

Exercise

練習問題

Q1

AI開発ツールの「エディタ統合型」と「CLI（ターミナル）型」の違いを、それぞれの代表的なツール名を挙げながら説明してください。

Q2

MCP（Model Context Protocol）とは何ですか？Claude Codeとの関連で、具体的にどのようなことができるようになるか説明してください。

Q3

AIが生成したコードを使用する際に注意すべきセキュリティ上のポイントを3つ挙げ、それぞれの対策を説明してください。

Answer

解説・解答

Q1 の解答

エディタ統合型（例：GitHub Copilot, Cursor）は、コードエディタの中でAI機能が動作します。コード補完やチャット機能がエディタのUIに統合されており、GUI操作で利用できます。
CLI（ターミナル）型（例：OpenAI Codex CLI, Claude Code）は、ターミナルからコマンドでAIに指示を出します。ファイルの読み書きやシェルコマンドの実行まで自律的に行うエージェント機能を持ち、より大きなタスクを一括で依頼できるのが特徴です。

Q2 の解答

MCP（Model Context Protocol）は、Anthropicが策定したAIモデルと外部ツール・データソースを接続するためのオープンプロトコルです。Claude Codeと組み合わせることで、AIエージェントが外部サービスにアクセスできるようになります。

具体例：GitHub MCPサーバーを設定すると、Claude CodeからIssueの作成やPull Requestの作成・レビューが行えます。データベースMCPサーバーを設定すると、SQLクエリの実行やスキーマの確認ができます。これにより、Claude Codeがコーディングだけでなく、開発に関連するさまざまなサービスと連携できるようになります。

Q3 の解答

1. APIキー・秘密情報の漏洩リスク：AIツールにAPIキーやパスワードを直接入力しないようにします。秘密情報は環境変数や.envファイルで管理し、.gitignoreに含めます。

2. 生成コードのセキュリティ脆弱性：AIが生成したコードにSQLインジェクションやXSS（クロスサイトスクリプティング）などの脆弱性が含まれる可能性があります。特に認証・認可、データ入力のバリデーションに関するコードは、公式ドキュメントやセキュリティベストプラクティスに照らして入念にレビューします。

3. ハルシネーション（誤情報）のリスク：AIが存在しないAPIや非推奨のメソッドを使ったコードを生成する場合があります。生成されたコードで使われているライブラリやAPIが実在するか、最新のドキュメントで確認します。
