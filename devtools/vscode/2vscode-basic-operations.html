Section 2-1

画面構成を理解する

VSCodeの画面は、いくつかの明確な領域に分かれています。各領域の名前と役割を理解することで、操作の効率が格段に向上します。ここでは、VSCodeの主要なUI要素について解説します。

Section 2-2

ファイルとフォルダの操作

VSCodeでの開発は、まずフォルダを開くところから始まります。エクスプローラーを使ったファイル操作は、日常的に最も多く使う機能です。

フォルダを開く

VSCodeで作業を始めるには、プロジェクトのフォルダを開きます。以下のいずれかの方法で開くことができます。

  • メニューバー：「ファイル」→「フォルダを開く」
  • ショートカット：Ctrl + K → Ctrl + O（macOS: Cmd + K → Cmd + O）
  • ターミナル：プロジェクトのディレクトリで code . を実行
  • ドラッグ＆ドロップ：フォルダをVSCodeのウィンドウにドロップ
POINT

VSCodeでは個別のファイルではなく、プロジェクト全体のフォルダを開くのが基本です。フォルダを開くことで、エクスプローラーにファイルツリーが表示され、プロジェクト全体を俯瞰できます。

エクスプローラーでのファイル操作

サイドバーのエクスプローラーでは、以下の操作が可能です。

操作 方法
新規ファイル作成 エクスプローラー上部の「新しいファイル」アイコンをクリック
新規フォルダ作成 エクスプローラー上部の「新しいフォルダ」アイコンをクリック
名前変更 ファイルを右クリック →「名前の変更」またはF2キー
削除 ファイルを右クリック →「削除」またはDeleteキー
移動・コピー ドラッグ＆ドロップ（Ctrlを押しながらでコピー）

クイックオープン

プロジェクト内のファイルを素早く開くには、クイックオープン機能を使います。

  • ショートカット：Ctrl + P（macOS: Cmd + P）
  • ファイル名の一部を入力するだけで、候補が絞り込まれます
  • ファジー検索に対応しているため、「idx」と入力すれば「index.html」がマッチします
注意

クイックオープン Ctrl + P とコマンドパレット Ctrl + Shift + P は似ていますが、用途が異なります。クイックオープンはファイルを開くためのもの、コマンドパレットはコマンドを実行するためのものです。

Section 2-3

タブとエディタグループ

VSCodeでは複数のファイルをタブで管理し、さらにエディタを分割して同時に表示できます。これらの機能を使いこなすことで、複数ファイルを扱う作業効率が大幅に向上します。

タブの基本操作

ファイルを開くと、エディタエリアの上部にタブとして表示されます。

操作 ショートカット
次のタブに切替 Ctrl + Tab
前のタブに切替 Ctrl + Shift + Tab
タブを閉じる Ctrl + W（macOS: Cmd + W）
閉じたタブを復元 Ctrl + Shift + T（macOS: Cmd + Shift + T）
すべてのタブを閉じる Ctrl + K → Ctrl + W
POINT

エクスプローラーでファイルをシングルクリックすると「プレビューモード」で開かれ、タブ名がイタリック表示になります。別のファイルを開くと置き換わります。ダブルクリックすると「固定タブ」として残ります。

エディタの分割表示

複数のファイルを横に並べて同時に編集できます。HTMLとCSSを並べて表示する場合などに便利です。

  • 分割：Ctrl + \（macOS: Cmd + \）
  • タブのドラッグ：タブをドラッグしてエディタを分割位置にドロップ
  • グループ間の移動：Ctrl + 1、Ctrl + 2... で各グループにフォーカス

Section 2-4

コマンドパレット

コマンドパレットはVSCodeの最も重要な機能の一つです。ほぼすべてのVSCodeの機能にコマンドパレットからアクセスでき、メニューを探し回る必要がなくなります。

コマンドパレットの開き方

  • Windows/Linux：Ctrl + Shift + P または F1
  • macOS：Cmd + Shift + P または F1

コマンドパレットが開くと、入力欄に > マークが表示されます。ここにコマンド名（またはその一部）を入力すると、マッチするコマンドが候補として表示されます。

よく使うコマンド一覧

コマンド 用途
Format Document コードの自動フォーマット
Toggle Terminal ターミナルの表示/非表示
Change Language Mode ファイルの言語モードを変更
Preferences: Open Settings 設定画面を開く
Preferences: Color Theme カラーテーマの変更
Git: Clone Gitリポジトリのクローン
Developer: Reload Window VSCodeウィンドウの再読み込み
POINT

コマンドパレットでは日本語でも英語でも検索可能です。「フォーマット」と入力しても「Format Document」がマッチします。また、コマンドの右側にショートカットキーが表示されるので、よく使うコマンドのショートカットを覚えるきっかけにもなります。

コマンドパレットの接頭辞

コマンドパレットの入力欄の先頭文字によって、動作が変わります。

接頭辞 動作 例
> コマンドの実行 >format
@ ファイル内のシンボル移動 @function
# ワークスペース全体のシンボル検索 #MyClass
: 指定行番号にジャンプ :42
（なし） ファイル名検索（クイックオープン） index.html
Section 2-5

ワークスペースの使い方

ワークスペースは、VSCodeで開くプロジェクトの単位です。単純にフォルダを開く方法から、複数のフォルダをまとめて管理するマルチルートワークスペースまで、状況に応じた使い方を解説します。

フォルダを開く（シンプルなワークスペース）

最も基本的な使い方は、プロジェクトフォルダを1つ開くことです。「ファイル」→「フォルダーを開く」でフォルダを選択するか、ターミナルで code ./my-project を実行します。これだけで、エクスプローラーにファイルツリーが表示され、プロジェクト全体を管理できます。

マルチルートワークスペース

マルチルートワークスペースを使うと、関連する複数のプロジェクトフォルダを1つのVSCodeウィンドウで同時に開くことができます。

例えば、フロントエンドとバックエンドのリポジトリが分かれているプロジェクトで、両方を同時に開いて作業したい場合に便利です。

マルチルートワークスペースの作成手順

  1. 「ファイル」→「フォルダーをワークスペースに追加」で複数のフォルダを追加
  2. 「ファイル」→「名前を付けてワークスペースを保存」で .code-workspace ファイルとして保存
  3. 次回以降は .code-workspace ファイルを開くだけで同じ構成を復元できる
my-project.code-workspace
{
    "folders": [
        { "path": "./frontend" },
        { "path": "./backend" },
        { "path": "./shared-libs" }
    ],
    "settings": {
        "editor.tabSize": 2,
        "editor.formatOnSave": true
    }
}
📂 ワークスペースの種類
📁 シングルフォルダ 1つのプロジェクトを開く
📁 my-project/
  📄 index.html
  📄 style.css
  📄 app.js
📂 マルチルート 複数フォルダを同時に開く
📁 frontend/
  📄 App.tsx
📁 backend/
  📄 server.js
📁 shared/
  📄 types.ts
POINT

初心者の方は、まずシングルフォルダで1つのプロジェクトを開く使い方に慣れましょう。マルチルートワークスペースは、複数リポジトリを横断して作業する中級者以上の方に特に便利な機能です。

2章 練習問題

Q1

VSCodeの画面で、ファイルツリー（プロジェクト内のファイル一覧）が表示される領域はどれですか？

  1. アクティビティバー
  2. サイドバー（エクスプローラー）
  3. ステータスバー
  4. パネル
Q2

コマンドパレットを開くショートカットキーはどれですか？（Windows）

  1. Ctrl + P
  2. Ctrl + Shift + P
  3. Ctrl + B
  4. Ctrl + K
Q3

マルチルートワークスペースの説明として正しいものはどれですか？

  1. 1つのフォルダを複数のウィンドウで開く機能
  2. 複数のフォルダを1つのVSCodeウィンドウで同時に開く機能
  3. 複数人が同時に同じファイルを編集する機能
  4. フォルダの内容を自動バックアップする機能
Answer

2章 解説・解答

Q1の解答: B

ファイルツリーはサイドバーの「エクスプローラー」ビューに表示されます。アクティビティバーは最左端のアイコン列でビューの切替に使い、ステータスバーは最下部の情報表示欄、パネルは下部のターミナルなどが表示される領域です。サイドバーの表示/非表示は Ctrl + B で切り替えられます。

Q2の解答: B

コマンドパレットは Ctrl + Shift + P（または F1）で開きます。Ctrl + P はクイックオープン（ファイル名検索）、Ctrl + B はサイドバーの表示切替です。コマンドパレットはVSCodeのほぼすべての機能にアクセスできる最重要機能です。

Q3の解答: B

マルチルートワークスペースは、複数のフォルダを1つのVSCodeウィンドウで同時に開く機能です。例えば、フロントエンドとバックエンドのリポジトリを同時に開いて作業できます。設定は .code-workspace ファイルに保存され、次回以降も同じ構成を簡単に復元できます。
