Section 4-1

拡張機能とは

拡張機能（Extensions）は、VSCodeに新しい機能を追加するためのプラグインです。VSCode本体は軽量に設計されており、必要な機能を拡張機能として後から追加するアーキテクチャになっています。Visual Studio Marketplace には数万の拡張機能が公開されており、あらゆる開発ニーズに対応できます。

拡張機能のインストール方法

  1. アクティビティバーの拡張機能アイコン（四角形が4つ並んだアイコン）をクリック、または Ctrl + Shift + X（macOS: Cmd + Shift + X）
  2. 検索バーに拡張機能の名前を入力して検索
  3. 目的の拡張機能を見つけたら「Install（インストール）」ボタンをクリック
  4. インストールが完了すると、すぐに利用可能になります（一部は再起動が必要）

注意

拡張機能を入れすぎるとVSCodeの動作が重くなることがあります。使わない拡張機能は無効化またはアンインストールしましょう。また、信頼性の低い拡張機能にはセキュリティリスクがあるため、ダウンロード数やレビューを確認してからインストールすることを推奨します。

Section 4-2

フロントエンド開発向け拡張機能

HTML、CSS、JavaScriptを使ったフロントエンド開発で特に役立つ拡張機能を紹介します。これらは多くのWeb開発者が標準的に利用しているものです。

Live Server

ローカルの開発サーバーを起動し、HTMLファイルをブラウザでリアルタイムプレビューできる拡張機能です。ファイルを保存すると自動的にブラウザが更新（ホットリロード）されるため、コードの変更結果を即座に確認できます。

  • 使い方：HTMLファイルを右クリック →「Open with Live Server」
  • または：ステータスバー右下の「Go Live」をクリック
  • ポート：デフォルトは 5500 番（http://localhost:5500）
POINT

Live Serverは開発の必須ツールです。HTMLファイルをダブルクリックでブラウザに表示するのと異なり、ローカルサーバー経由でアクセスするため、CSSやJavaScriptの外部ファイル読み込みも正しく動作します。

Prettier - Code formatter

コードの自動フォーマット（整形）を行う拡張機能です。インデント、改行、スペース、引用符のスタイルなどを統一し、チーム全体でコードスタイルを揃えることができます。

  • 対応言語：HTML、CSS、JavaScript、TypeScript、JSON、Markdown など
  • 使い方：保存時に自動フォーマット（"editor.formatOnSave": true）
  • 手動実行：Shift + Alt + F（macOS: Shift + Option + F）
settings.json - Prettier設定
{
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "editor.formatOnSave": true,
    "prettier.tabWidth": 2,
    "prettier.singleQuote": true,
    "prettier.semi": true
}

ESLint

JavaScriptおよびTypeScriptのコード品質チェックツールです。構文エラーやアンチパターンをリアルタイムに検出し、コードの品質を維持します。

  • コードを書いている最中に問題箇所が波線でハイライトされる
  • ホバーするとエラーの詳細と修正候補が表示される
  • 保存時に自動修正する設定も可能
settings.json - ESLint自動修正設定
{
    "editor.codeActionsOnSave": {
        "source.fixAll.eslint": "explicit"
    }
}

Auto Rename Tag

HTMLの開始タグを変更すると、対応する終了タグも自動的に変更される拡張機能です。タグの不一致を防ぎ、HTMLの編集効率が向上します。

例えば、<div> を <section> に変更すると、対応する </div> が自動的に </section> に変わります。

Section 4-3

バックエンド開発向け拡張機能

Java、Python、Dockerなどのバックエンド技術に対応した拡張機能を紹介します。VSCodeはフロントエンドだけでなく、バックエンド開発にも十分対応できるエディタです。

Extension Pack for Java

Java開発に必要な拡張機能をまとめたパックです。以下の拡張機能が含まれています。

  • Language Support for Java：IntelliSense、コード補完、リファクタリング
  • Debugger for Java：Javaプログラムのデバッグ
  • Test Runner for Java：JUnitなどのテスト実行
  • Maven for Java：Mavenプロジェクトの管理
  • Project Manager for Java：Javaプロジェクトの作成・管理

Extension Packをインストールするだけで、VSCodeがJava開発環境として機能するようになります。

Python

Microsoft公式のPython拡張機能です。以下の機能を提供します。

  • IntelliSense（Pylance による高速な型推論・補完）
  • デバッグ（ブレークポイント、変数ウォッチ、ステップ実行）
  • リンティング（Pylint、flake8 など）
  • Jupyter Notebook のサポート
  • 仮想環境の自動検出と切り替え

Docker

VSCode内でDockerコンテナ、イメージ、ボリュームの管理ができる拡張機能です。

  • Dockerfileのシンタックスハイライトと補完
  • docker-compose.ymlの編集支援
  • サイドバーからコンテナの起動・停止・削除が可能
  • コンテナのログ表示やシェルアクセス

REST Client

VSCode内でHTTPリクエストを送信し、APIの動作確認ができる拡張機能です。Postmanのような外部ツールを使わずに、.http ファイルでリクエストを記述・実行できます。

api-test.http - REST Client の使用例
### ユーザー一覧を取得
GET https://jsonplaceholder.typicode.com/users
Content-Type: application/json

### 新しいユーザーを作成
POST https://jsonplaceholder.typicode.com/users
Content-Type: application/json

{
    "name": "Taro Yamada",
    "email": "taro@example.com"
}

### 特定のユーザーを更新
PUT https://jsonplaceholder.typicode.com/users/1
Content-Type: application/json

{
    "name": "Taro Yamada Updated"
}
POINT

REST Client では ### でリクエストを区切り、各リクエストの上に表示される「Send Request」リンクをクリックするだけでリクエストが送信されます。レスポンスは新しいタブに表示され、ステータスコード、ヘッダー、ボディを確認できます。

Section 4-4

生産性向上の拡張機能

特定の言語に限らず、日常の開発作業全般の生産性を向上させる拡張機能を紹介します。これらはどの言語・フレームワークを使う場合でも役立ちます。

GitLens

VSCodeのGit機能を大幅に強化する拡張機能です。標準のGitサポートだけでは不十分な場合に、以下の機能が追加されます。

  • Git Blame表示：各行に「誰が・いつ・どのコミットで」変更したかを表示
  • コミット履歴のビジュアル表示：ファイルやブランチのコミット履歴をグラフで確認
  • 差分比較：任意のコミット間やブランチ間の差分を確認
  • ホバーで詳細表示：行にカーソルを合わせるだけでコミット情報を表示

Path Intellisense

ファイルパスの入力時に、ファイル名やフォルダ名の補完候補を表示する拡張機能です。HTML の src や href 属性、JavaScript の import 文など、パスを記述するあらゆる場面で役立ちます。

Bracket Pair Colorization（ブラケットペア色付け）

対応する括弧のペア（{}、[]、()）を同じ色で表示する機能です。ネストが深いコードでも、どの括弧が対応しているかが一目でわかります。

POINT

この機能はVSCode本体に組み込まれるようになりました。settings.json で "editor.bracketPairColorization.enabled": true と設定するだけで有効化できます。拡張機能のインストールは不要です。

Material Icon Theme

エクスプローラーのファイルアイコンをリッチなアイコンセットに変更する拡張機能です。ファイルの種類が直感的にわかるようになり、プロジェクトの見通しが良くなります。

  • HTML、CSS、JS、TS、JSON、画像など、ファイル拡張子ごとに専用アイコン
  • src、dist、node_modules などのフォルダにも専用アイコン
  • 設定でアイコンのスタイルをカスタマイズ可能

Section 4-5

テーマとカスタマイズ

VSCodeの外観は、カラーテーマ、ファイルアイコンテーマ、フォント設定などで自分好みにカスタマイズできます。見た目の調整は、長時間のコーディングにおける快適性と集中力に直結します。

カラーテーマの変更

カラーテーマは、エディタの背景色、テキスト色、シンタックスハイライトの配色を変更するものです。

テーマの変更手順

  1. Ctrl + K → Ctrl + T（macOS: Cmd + K → Cmd + T）でテーマ選択画面を開く
  2. または、コマンドパレット → 「Preferences: Color Theme」
  3. 上下矢印キーでプレビューしながら、好みのテーマを選択

人気のカラーテーマ

テーマ名 特徴 種類
One Dark Pro Atom エディタの定番テーマ。コントラストが高く見やすい ダークテーマ
Dracula 紫ベースの人気テーマ。目に優しい配色 ダークテーマ
GitHub Theme GitHubのUIに合わせたテーマ。ライト/ダーク両方 両方
Tokyo Night 東京の夜景をイメージした柔らかい配色 ダークテーマ
Catppuccin パステルカラーで統一された温かみのあるテーマ 両方

ファイルアイコンテーマ

エクスプローラーに表示されるファイル・フォルダのアイコンを変更できます。

  1. コマンドパレット → 「Preferences: File Icon Theme」
  2. 人気のアイコンテーマ：Material Icon Theme、vscode-icons

フォント設定

プログラミングには等幅フォント（モノスペースフォント）が適しています。コードの桁揃えが正確になり、可読性が向上します。

フォント名 特徴 リガチャ
JetBrains Mono JetBrains製。視認性が高くプログラミングに最適 対応
Fira Code リガチャが豊富。==>, !== などが美しく表示 対応
Cascadia Code Microsoft製。Windows Terminalのデフォルトフォント 対応
Source Code Pro Adobe製。クリーンで読みやすいデザイン 非対応
settings.json - フォント設定の例
{
    "editor.fontFamily": "'JetBrains Mono', 'Fira Code', Consolas, monospace",
    "editor.fontSize": 14,
    "editor.fontLigatures": true,
    "editor.lineHeight": 1.6
}
POINT

フォントリガチャ（Ligature）とは、=== や => などの記号の組み合わせを、専用のグリフで美しく表示する機能です。有効にするには "editor.fontLigatures": true を設定し、リガチャ対応フォントを使用する必要があります。

4章 練習問題

Q1

VSCodeで拡張機能の管理画面を開くショートカットはどれですか？（Windows）

  1. Ctrl + Shift + P
  2. Ctrl + Shift + X
  3. Ctrl + Shift + F
  4. Ctrl + Shift + G
Q2

「Live Server」拡張機能の主な機能として正しいものはどれですか？

  1. コードを自動フォーマットする
  2. JavaScriptのエラーをリアルタイム検出する
  3. ローカル開発サーバーを起動し、ファイル保存時にブラウザを自動更新する
  4. Git の変更履歴を詳しく表示する
Q3

VSCodeのカラーテーマを変更する方法として正しいものはどれですか？

  1. settings.jsonを手動で編集する以外に方法はない
  2. Ctrl + K → Ctrl + T でテーマ選択画面を開ける
  3. テーマの変更には拡張機能のインストールが必須
  4. テーマの変更後はVSCodeの再インストールが必要
Answer

4章 解説・解答

Q1の解答: B

Ctrl + Shift + X で拡張機能の管理画面（Extensions ビュー）が開きます。ここから拡張機能の検索、インストール、有効化/無効化、アンインストールなどの操作が行えます。Ctrl + Shift + P はコマンドパレット、Ctrl + Shift + F はファイル横断検索、Ctrl + Shift + G はソース管理ビューです。

Q2の解答: C

Live Server はローカル開発サーバーを起動し、HTMLファイルの変更を保存すると自動的にブラウザが更新（ホットリロード）される拡張機能です。コードのフォーマットは Prettier、エラー検出は ESLint、Git 履歴表示は GitLens がそれぞれ担当する拡張機能です。

Q3の解答: B

Ctrl + K → Ctrl + T でカラーテーマの選択画面を開くことができます。上下矢印キーでプレビューしながら選択でき、VSCodeにはデフォルトで複数のテーマが内蔵されています。追加のテーマが必要な場合は Marketplace からインストールできますが、テーマの変更自体に拡張機能は必須ではありません。再インストールも不要です。
