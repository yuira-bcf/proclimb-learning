Section 15-1

MCPとは（Model Context Protocol）

MCP（Model Context Protocol）は、Anthropic社が開発したAIモデルと外部ツール・データソースを接続するためのオープンプロトコルです。MCPを使うことで、AIアシスタント（Claude、GitHub Copilotなど）がGitHub APIなどの外部サービスに直接アクセスし、より高度な支援を行えるようになります。

💻 MCPクライアント VSCode / Claude
AIアシスタント
↔
🔧 MCPサーバー GitHub MCP Server
ツール・リソース提供
↔
🌐 GitHub API リポジトリ / Issues
PR / Actions

1.1 MCPの基本概念

用語 役割 例
MCPクライアント AIアシスタントが動作するアプリケーション VSCode、Claude Desktop、Cursor
MCPサーバー 外部サービスへの接続を提供するサーバー GitHub MCP Server、Filesystem Server
ツール（Tools） AIが実行できるアクション Issue作成、PR一覧取得、ファイル検索
リソース（Resources） AIが参照できるデータ リポジトリの情報、ファイル内容

1.2 なぜMCPが重要なのか

🔌

統一されたインターフェース

MCPにより、様々な外部サービスとAIの接続が標準化されます。各サービスごとに独自のプラグインを開発する必要がなくなります。

🚀

開発効率の飛躍的向上

AIアシスタントにGitHubの操作を任せることで、IssueやPRの管理、コード検索などの作業を自然言語で指示できるようになります。

🔒

安全なアクセス管理

MCPはPersonal Access Token（PAT）を使って認証を行い、必要最小限の権限でサービスにアクセスします。

lightbulb MCPのエコシステム

GitHub以外にも、ファイルシステム、データベース（PostgreSQL）、Slack、Google Drive、Notionなど、様々なMCPサーバーが公開されています。MCPサーバーの一覧は MCP Servers リポジトリ で確認できます。

Section 15-2

VSCodeでGitHub MCPを設定する

VSCodeでGitHub MCPサーバーを設定し、AIアシスタント（Copilot Chat等）からGitHubの操作を行えるようにしましょう。MCPサーバーの実行にはDocker経由を推奨します（環境の隔離・バージョン固定のメリットがあります）。Dockerの基礎はDocker構築入門で学べます。

2.1 前提条件

  • VSCode が最新バージョンにアップデートされていること
  • GitHub Copilot 拡張機能がインストールされていること（MCP対応のAIチャット機能が必要）
  • GitHub Personal Access Token（PAT） が発行されていること
  • Docker Desktop がインストールされていること（第2章 Docker Desktopの導入参照）
lightbulb なぜDockerを使うのか？

Docker経由でMCPサーバーを実行すると、環境の隔離（MCPサーバーがホストOSのファイルにアクセスできない）、バージョン固定（チーム全員が同じ環境）、クリーンなアンインストールといったメリットがあります。詳しくはDocker構築入門 第10章を参照してください。

2.2 Personal Access Tokenの発行

  1. GitHubのSettingsを開く（右上のアイコン → Settings）
  2. 左メニューのDeveloper settingsをクリック
  3. Personal access tokens → Tokens (classic)を選択
  4. Generate new token (classic)をクリック
  5. 必要なスコープを選択：repo、read:org、read:project
  6. Generate tokenをクリックしてトークンをコピー
warning トークンの管理に注意

Personal Access Tokenはパスワードと同等の重要な秘密情報です。トークンをコードにハードコードしたり、Gitリポジトリにコミットしたりしないでください。環境変数として安全に管理しましょう。

2.3 MCP設定ファイルの作成

プロジェクトのルートに.vscode/mcp.jsonファイルを作成し、GitHub MCPサーバーの設定を記述します。

.vscode/mcp.json - GitHub MCPサーバーの設定 JSON 
{
  "servers": {
    "github": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e", "GITHUB_PERSONAL_ACCESS_TOKEN",
        "ghcr.io/github/github-mcp-server"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${env:GITHUB_TOKEN}"
      }
    }
  }
}
環境変数の設定（macOS/Linux） Terminal 
# .bashrc や .zshrc に追加
export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"

# 設定を反映
$ source ~/.zshrc
環境変数の設定（Windows PowerShell） PowerShell 
# ユーザー環境変数に設定
[System.Environment]::SetEnvironmentVariable("GITHUB_TOKEN", "ghp_xxxxxxxxxxxxxxxxxxxx", "User")
lightbulb Dockerが使えない場合（npx代替）

Docker環境が用意できない場合は、Node.jsのnpxコマンドでもMCPサーバーを起動できます。"command": "npx"、"args": ["-y", "@modelcontextprotocol/server-github"] のように設定します（Node.js 18以上が必要）。ただし環境の隔離ができないため、業務ではDocker経由を推奨します。

2.4 MCPサーバーの動作確認

  1. VSCodeでプロジェクトを開く
  2. Copilot Chat（またはMCP対応のAIチャット）を開く
  3. チャット入力欄の横にあるツールアイコンをクリック
  4. GitHub MCPサーバーのツール一覧が表示されれば成功

ツール一覧には、create_issue、list_issues、create_pull_requestなどが表示されます。

Section 15-3

Claude Code（Anthropic公式のCLIツール）でもMCPサーバーを利用できます。ターミナルから直接GitHub操作を行えるため、コマンドライン中心の開発ワークフローに最適です。

3.1 Claude Codeとは

Claude CodeはAnthropicが提供するAIコーディングエージェントです。ターミナル上で動作し、コードの読み書き・ファイル操作・Git操作・MCP連携などをAIが自律的に行えます。

💻

ターミナルで動作

コマンドラインからclaudeコマンドで起動。VSCode拡張としても利用可能です。

🔧

Git操作が組み込み

MCP無しでもgit add、git commit、git pushなどの基本操作が可能。/commitコマンドで変更を自動コミットできます。

🔌

MCP対応

GitHub MCPサーバーを追加することで、Issue作成・PR管理・コード検索などのGitHub API操作が可能になります。

3.2 Claude Codeのインストール

Claude Codeのインストール（npm） Terminal 
# npmでグローバルインストール
$ npm install -g @anthropic-ai/claude-code

# インストール確認
$ claude --version

# 起動（プロジェクトディレクトリで実行）
$ claude
lightbulb 前提条件

Claude Codeの利用にはNode.js 18以上とAnthropic APIキー（またはClaude Pro/Maxプラン）が必要です。初回起動時に認証設定を行います。

3.3 MCPサーバーの追加

Claude Codeでは、claude mcp addコマンドでMCPサーバーを簡単に追加できます。

GitHub MCPサーバーの追加 Terminal 
# GitHub MCPサーバーを追加（Docker経由 - 推奨）
$ claude mcp add github \
    -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_xxxxxxxxxxxx \
    -- docker run -i --rm \
    -e GITHUB_PERSONAL_ACCESS_TOKEN \
    ghcr.io/github/github-mcp-server

# Dockerが使えない場合（npx代替）
$ claude mcp add github \
    -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_xxxxxxxxxxxx \
    -- npx -y @modelcontextprotocol/server-github
MCPサーバーの管理コマンド Terminal 
# 登録済みMCPサーバーの一覧を確認
$ claude mcp list

# MCPサーバーの詳細を確認
$ claude mcp get github

# MCPサーバーを削除
$ claude mcp remove github
lightbulb スコープの指定

MCPサーバーは--scopeオプションで適用範囲を指定できます。local（現在のプロジェクトのみ）、project（プロジェクト共有）、user（全プロジェクト共通）の3つから選べます。デフォルトはlocalです。

Claude CodeでのGitHub操作例 Claude Code

3.5 VSCode拡張としてのClaude Code

Claude CodeはVSCode拡張機能としても利用できます。VSCodeのサイドパネルからClaude Codeを起動し、エディタと統合された環境でAI操作が行えます。

  1. VSCode拡張機能で「Claude Code」を検索してインストール
  2. サイドパネルに表示されるClaude Codeアイコンをクリック
  3. Anthropicアカウントで認証を完了
  4. チャット欄から自然言語でGit操作やMCPツールの呼び出しが可能
lightbulb ターミナルとVSCode拡張の使い分け

ターミナル版はスクリプトやCI/CDとの連携に、VSCode拡張版はエディタでコードを見ながらの作業に適しています。MCP設定はどちらでも共有されるため、同じGitHub MCPサーバーを利用できます。

開発効率化のヒント

MCPを含む最新のAIツールを活用して、日々の開発作業をさらに効率化するためのヒントをまとめます。

6.1 AIアシスタントの活用パターン

📝

コミットメッセージの自動生成

変更内容をAIに分析させて、Conventional Commitsに準拠した適切なコミットメッセージを生成してもらいましょう。

📄

PRの説明文自動生成

差分の内容をAIに読み込ませて、PRの概要・変更点・テスト方法などを自動で記述させることができます。

AIにPRのコードを分析させて、潜在的なバグやセキュリティリスク、改善点を指摘してもらいましょう。人間のレビューの前段階として活用できます。

ドキュメントの自動生成

コードの変更履歴からCHANGELOGを生成したり、READMEのAPI仕様を更新したりすることもAIに任せられます。

6.2 複数のMCPサーバーの組み合わせ

GitHub MCP以外にも、様々なMCPサーバーを組み合わせることで、より強力な開発環境を構築できます。

.vscode/mcp.json - 複数サーバーの設定例 JSON 
{
  "servers": {
    "github": {

      "args": [

      }

      ]

}

6.3 AI活用時の注意点

warning AIの出力は必ず確認する

AIが生成したコード、コミットメッセージ、Issue内容は必ず人間が確認してから使用してください。AIは間違いを含む可能性があり、特にセキュリティに関わるコードは慎重に検証する必要があります。

  • 秘密情報の取り扱い：AIにAPIキーやパスワードを入力しないよう注意する
  • 著作権の考慮：AIが生成したコードの著作権やライセンスに注意する
  • 過度な依存の回避：AIはあくまで補助ツール。基本的なGit操作は自分で理解しておく
  • チームのルールに従う：AIツールの使用範囲はチームで合意を取ってから導入する
lightbulb これからの開発者に求められるスキル

AI時代の開発者に求められるのは、AIに適切な指示を出せるプロンプトエンジニアリングのスキルと、AIの出力を評価できる技術的な基礎力の両方です。この講座で学んだGit & GitHubの基礎知識は、AIツールを効果的に使いこなすための土台になります。

Answer

15章 解説・解答

Q1の解答: B

MCPはModel Context Protocolの略称です。Anthropic社が開発した、AIモデルと外部ツール・データソースを接続するためのオープンプロトコルです。

Q2の解答: B

VSCodeのMCP設定ファイルは、プロジェクトルートの.vscode/mcp.jsonに配置します。このファイルにMCPサーバーの起動コマンドや環境変数を定義します。

Q3の解答: C

「AIの出力はすべて正確なので、そのまま使用してよい」は正しくありません。AIは間違いを含む可能性があるため、生成されたコードやテキストは必ず人間が確認・検証してから使用する必要があります。特にセキュリティに関わる部分は慎重なチェックが求められます。
