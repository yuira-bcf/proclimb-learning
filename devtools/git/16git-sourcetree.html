Section 16-1

SourceTreeとは

SourceTree（ソースツリー）は、Atlassian社が提供する無料のGit GUIクライアントです。コマンドラインを使わずに、視覚的なインターフェースでGitの操作（クローン、コミット、プッシュ、ブランチ管理など）を行えます。Windows / macOS に対応しています。

1.1 SourceTreeの特徴

💻

直感的なGUI操作

コマンドを覚えなくても、ボタンやメニューでGitの全操作が行えます。初心者にとってGitの学習コストを大幅に下げられます。

🌳

ブランチの可視化

コミット履歴やブランチの分岐・マージをグラフィカルに確認できます。プロジェクトの状態が一目で分かります。

🔄

差分の確認が簡単

変更ファイルの差分（diff）をカラー表示で確認でき、どの行が追加・削除されたか一目瞭然です。

🔗

GitHub/Bitbucketとの連携

GitHub、Bitbucket、GitLabなど主要なリモートリポジトリサービスとシームレスに連携できます。

lightbulb GUIとCLIの使い分け

SourceTreeのようなGUIツールは操作が視覚的で分かりやすいですが、基本的なGitコマンドも理解しておくことが重要です。GUIツールが使えない環境（サーバーなど）ではコマンドラインが必須になります。GUIとCLIの両方を使いこなせるのが理想です。

Section 16-2

GitHubでリモートリポジトリを作成

まず、GitHubでリモートリポジトリを作成します。このリポジトリがコードのバックアップ・共有先となります。

2.1 リポジトリ作成の手順

🚀 全体の作業フロー
📁 1. GitHub リモートリポジトリ作成
→
📂 2. ローカル準備 ディレクトリ作成
→
🌳 3. SourceTree クローン
→
📝 4. VSCode ファイル作成・編集
→
🚀 5. Commit & Push SourceTreeで操作
1
GitHubにログイン

https://github.com にアクセスし、自分のアカウントでログインします。

2
新しいリポジトリを作成

右上の「+」ボタンをクリックし、「New repository」を選択します。

3
リポジトリの設定
  • Repository name: プロジェクト名を入力（例: gittest）
  • Description: 説明を入力（任意）
  • Public / Private: 用途に応じて選択
  • Initialize this repository with a README: チェックを入れる（推奨）
4
Create repository をクリック

設定が完了したら「Create repository」ボタンをクリックしてリポジトリを作成します。

5
SSH URLをコピー

作成されたリポジトリページで「Code」ボタンをクリックし、「SSH」タブを選択して表示されるURLをコピーします。

SSH URL の例 URL 
git@github.com:username/gittest.git
warning SSHとHTTPSの違い

リモートリポジトリのURLにはSSHとHTTPSの2種類があります。SSH URLを使用すると、SSH鍵認証により毎回のパスワード入力が不要になります。事前にSSH鍵の設定が必要です（第8章参照）。

Section 16-3

ローカルプロジェクトの準備

クローン先となるローカルのプロジェクト用ディレクトリを作成し、VSCodeのプロジェクトマネージャーに登��します。

3.1 プロジェクト用ディレクトリの作成

1
ディレクトリを作成

エクスプローラー（ファイルマネージャー）を開き、以下のパスにプロジェクト用のフォルダを作成します。

ディレクトリパス Windows 
C:\academia\src\gittest
lightbulb ディレクトリ構成のポイント

C:\academia\src のように、開発用の親ディレクトリを決めておくと、プロジェクトの管理がしやすくなります。各プロジェクトはこの中にフォルダを作成して整理しましょう。

3.2 VSCode プロジェクトマネージャーへの登録

VSCodeの拡張機能「Project Manager」を使って、プロジェクトを登録しておくと素早く切り替えられます。

1
Project Manager 拡張機能を開く

VSCodeの左サイドバーからProject Managerのアイコンをクリックします。

2
Project Save でプロジェクトを保存

コマンドパレット（Ctrl + Shift + P）を開き、「Project Manager: Save Project」を実行します。プロジェクト名を入力します（例: gittest）。

3
Project Edit でパスを設定

コマンドパレットから「Project Manager: Edit Projects」を実行するとsettings.jsonが開きます。作成したディレクトリのパスを設定します。

settings.json の設定例 JSON 
[
  {
    "name": "gittest",
    "rootPath": "C:\\academia\\src\\gittest",
    "paths": [],
    "group": "",
    "enabled": true
  }
]
Section 16-4

SourceTreeでリポジトリをクローン

SourceTreeを使って、GitHubのリモートリポジトリをローカルにクローン（複製）します。これがSourceTreeでの最初のステップです。

4.1 クローンの手順（Windows版）

🌳 SourceTreeクローンの流れ
➕ タブの「+」 新規タブを開く
→
🔌 リモート選択 リポジトリを選択
→
📋 SSH URL貼付 GitHubからコピー
→
📥 クローン実行 ローカルに複製
1
SourceTreeを起動し、タブの「+」をクリック

SourceTreeを開き、上部のタブバーにある「+」（プラス）ボタンをクリックして、新しいタブを開きます。

2
「Remote」タブを選択

表示されたダイアログで「Remote」タブを選択します。GitHubアカウントと連携済みであれば、リモートリポジトリの一覧が表示されます。

3
「Show repository」のチェックを外す

不要なリポジトリを非表示にするため、「Show repository」のチェックを外してフィルタリングします。作成したリモートリポジトリのみを選択しやすくなり���す。

4
リモートリポジトリを選択

先ほどGitHubで作成したリポジトリ（例: gittest）を選択します。

5
SSH URLを設定

リモートリポジトリのURLが表示されます。SSH形式のURLになっていることを確認してください。HTTPS URLが表示されている場合は、GitHubからコピーしたSSH URLに書き換えます。

SSH URL URL 
# SSH URL（推奨）
git@github.com:username/gittest.git

# HTTPS URL（こちらではなく上のSSH URLを使用）
https://github.com/username/gittest.git
6
ローカルディレクトリを選択

クローン先のローカルディレクトリとして、先ほど作成したフォルダを選択します。

ローカルパス Windows 
C:\academia\src\gittest
7
「クローン」ボタンをクリック

「Clone」（クローン）ボタンをクリックして、リモートリポジトリをローカルにクローンします。完了すると、SourceTreeにリポジトリが表示されます。

lightbulb クローンとは？

クローン（clone）とは、リモートリポジトリの完全なコピーをローカルに作成する操作です。コードファイルだけでなく、全てのコミット履歴やブランチ情報も含めてコピーされます。コマンドラインでは git clone コマンドに相当します。

Section 16-5

VSCodeでファイルを作成・編集

クローンしたローカルリポジトリでJavaファイルを作成し、コードを記述します。VSCodeのProject Managerからプロジェクトを開くと便利です。

5.1 Main.java の作成

1
VSCodeでプロジェクトを開く

VSCodeのProject Managerから、先ほど登録したgittestプロジェクトを開きます。または、ファイル → フォルダーを開くから C:\academia\src\gittest を選択します。

2
新しいファイルを作成

エクスプローラーパネルで右クリックし、「新しいファイル」を選択します。ファイル名を Main.java と入力します。

3
コードを記述

以下のJavaコードを入力して保存します（Ctrl + S）。

Main.java Java 
public class Main {
    public static void main(String[] args) {
        System.out.println("本日のgittest");
    }
}
warning Javaファイルの注意点

Javaではファイル名とクラス名を一致させる必要があります。Main.javaというファイル名の場合、クラス名はpublic class Mainとしてください。また、System.out.printlnはmainメソッドの中に記述する必要があります。

Section 16-6

SourceTreeでコミット＆プッシュ

VSCodeでファイルを作成・保存したら、SourceTreeに戻ってコミットとプッシュを行います。これがGitワークフローの基本操作です。

6.1 コミットの手順

🚀 コミット＆プッシュの流れ
📄 ファイル変更 VSCodeで編集・保存
→
✅ ステージ 変更をステージング
→
💾 コミット メッセージを付けて記録
→
🚀 プッシュ GitHubに送信
1
SourceTreeで変更を確認

SourceTreeに戻ると、左サイドバーの「ファイルステータス」に変更が表示されています。Main.java が「作業ツリーのファイル」（未ステージ）に表示されていることを確認します。

2
ファイルをステージングに追加

Main.java のチェックボックスをオンにするか、「全てインデックスに追加」ボタンをクリックして、ファイルをステージングエリアに移動します。

lightbulb ステージングとは？

ステージング（インデックスに追加）は、コマンドラインの git add に相当します。コミットに含めるファイルを選択する操作です。

3
コミットメッセージを入力

画面下部のテキストエリアにコミットメッセージを入力します。変更内容が分かるメッセージを書きましょう。

コミットメッセージの例 メッセージ 
feat: Main.javaを追加（初回コミット）
4
「コミット」ボタンをクリック

「コミット」ボタンをクリックして、変更をローカルリポジトリに記録します。これはコマンドラインの git commit に相当します。

6.2 プッシュの手順

5
「プッシュ」ボタンをクリック

ツールバーの「プッシュ」ボタンをクリックします。プッシュ先のブランチ（通常は main）を確認して実行します。

6
プッシュの完了を確認

プッシュが成功すると、SourceTreeの履歴画面に「origin/main」のラベルがローカルの「main」と同じコミットを指していることが確認できます。

SourceTreeの操作 対応するGitコマンド 説明
チェックボックスをオン git add ファイル名 ステージングに追加
コミットボタン git commit -m "メッセージ" 変更をローカルに記録
プッシュボタン git push origin main リモートに送信
プルボタン git pull origin main リモートから取得
フェッチボタン git fetch origin リモートの情報を取得
Section 16-7

GitHubで確認

プッシュが完了したら、GitHubのリポジトリページでコードが正しくアップロードされていることを確認しましょう。

7.1 プッシュ結果の確認

1
GitHubのリポジトリページを開く

ブラウザで https://github.com/username/gittest にアクセスします。

2
ファイル一覧を確認

リポジトリのCodeタブにMain.javaが表示されていれば、プッシュ成功です。

3
ファイルの内容を確認

Main.java をクリックして、コードの内容が正しく反映されていることを確認します。

4
コミット履歴を確認

「commits」リンクをクリックすると、コミットの履歴を確認できます。先ほど入力したコミットメッセージが表示されています。

✅ 確認チェックリスト
📄 Main.java がある ファイル一覧に
表示されている
📝 コード内容が正しい System.out.println
が含まれている
💬 コミットメッセージ 入力したメッセージが
表示されている
👤 コミッター名 自分のアカウント名が
表示されている
lightbulb おめでとうございます！

ここまでの流れが確認できれば、SourceTreeを使ったGitの基本操作（クローン → 編集 → コミット → プッシュ）が完了です。この流れはチーム開発でも繰り返し使う最も基本的なワークフローです。

Exercise

16章 練習問題

Q1

SourceTreeで「ステージングに追加」する操作は、Gitコマンドのどれに相当しますか？

  1. git commit
  2. git push
  3. git add
  4. git pull
Q2

GitHubのリモートリポジトリURLとして、SSH形式はどれですか？

  1. https://github.com/user/repo.git
  2. git@github.com:user/repo.git
  3. ftp://github.com/user/repo.git
  4. ssh://github.com/user/repo
Q3

SourceTreeでクローンした後、ファイルを変更してGitHubに反映するまでの正しい順序はどれですか？

  1. プッシュ → コミット → ステージング
  2. コミット → ステージング → プッシュ
  3. ステージング → コミット → プッシュ
  4. ステージング → プッシュ → コミット
Answer

16章 解説・解答

Q1の解答: C

SourceTreeでファイルをステージングに追加する操作は、コマンドラインのgit addに相当します。ステージングとは、次のコミットに含めるファイルを選択する操作です。コミット前の「準備段階」と考えましょう。

Q2の解答: B

SSH形式のURLはgit@github.com:user/repo.gitの形式です。SSH鍵認証を使用するため、毎回のパスワード入力が不要になります。Aの形式はHTTPS URLです。

Q3の解答: C

正しい順序は「ステージング → コミット → プッシュ」です。まず変更をステージングエリアに追加し（git add）、コミットメッセージを付けてローカルに記録し（git commit）、最後にリモートリポジトリに送信します（git push）。この3ステップはGitの基本中の基本です。
