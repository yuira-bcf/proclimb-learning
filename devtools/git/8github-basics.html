Section 8-1

GitHubとは

GitHub（ギットハブ）は、Gitリポジトリのホスティングサービスです。コードの保管場所としてだけでなく、チーム開発のためのコラボレーションプラットフォームとして、世界中の開発者に利用されています。2018年にMicrosoftが買収し、現在は1億人以上の開発者が利用しています。

1.1 GitHubの主な機能

📁

リモートリポジトリ

ローカルのGitリポジトリをクラウド上にバックアップ・共有できます。PCが壊れてもコードは安全に保管されます。

👥

プルリクエスト（PR）

コードの変更をレビューしてもらい、議論した上でマージする仕組みです。チーム開発の品質管理に不可欠です。

📄

Issue管理

バグ報告、機能要望、タスク管理をIssueとして追跡できます。プロジェクト管理ツールとしても機能します。

⚙

GitHub Actions

CI/CD（継続的インテグレーション/デリバリー）のパイプラインを自動化できます。テストやデプロイを自動実行します。

lightbulb GitとGitHubは別物

Gitはバージョン管理ツール（ソフトウェア）であり、GitHubはGitリポジトリをホストするWebサービスです。Gitはインターネット接続なしでも使えますが、GitHubを使うにはインターネット接続が必要です。GitHubの他にも、GitLab、Bitbucketなど同様のサービスがあります。

Section 8-2

アカウント作成とプロフィール設定

GitHubを利用するには、まずアカウントを作成する必要があります。無料プランで十分な機能が使えるので、気軽に始められます。

2.1 アカウント作成の手順

👤 GitHubアカウント設定フロー
📝 1. アカウント作成 github.com/signup
→
👤 2. プロフィール設定 名前・アイコン・Bio
→
🔑 3. SSH鍵の設定 安全な接続を確立
→
📁 4. リポジトリ作成 最初のリポジトリ
  1. GitHubにアクセス: https://github.com/signup にアクセスします
  2. メールアドレスを入力: 普段使っているメールアドレスを登録します
  3. パスワードを設定: 15文字以上、または数字と小文字を含む8文字以上のパスワード
  4. ユーザー名を決める: 他のユーザーと重複しないユニークな名前（URLの一部になります）
  5. メール認証: 登録したメールアドレスに届く認証コードを入力
warning ユーザー名は慎重に決めよう

ユーザー名はGitHubのプロフィールURL（github.com/ユーザー名）に使われ、後から変更すると既存のリンクが無効になります。本名やニックネームなど、長期的に使える名前を選びましょう。就活のポートフォリオとしても使えるため、プロフェッショナルな印象を与える名前がおすすめです。

2.2 プロフィールの設定

アカウント作成後、プロフィールを充実させましょう。GitHubのプロフィールは開発者としての名刺代わりになります。

  • プロフィール画像: 顔写真やアバターを設定（デフォルトのアイコンのままだと信頼性が低い印象になります）
  • Name（表示名）: 本名またはハンドルネーム
  • Bio（自己紹介）: 短い自己紹介文（例: 「Web Developer / HTML, CSS, JavaScript」）
  • Location（場所）: 住んでいる地域
  • Website（Webサイト）: ポートフォリオサイトがあればURL
Section 8-3

SSH鍵の設定

SSH（Secure Shell）鍵を使うと、パスワードを入力せずに安全にGitHubと通信できます。初回のみ設定が必要ですが、一度設定すれば以降は快適に操作できます。

3.1 SSH鍵の仕組み

SSH鍵は公開鍵と秘密鍵のペアで構成されます。公開鍵はGitHubに登録し、秘密鍵は自分のPCに保管します。この仕組みにより、パスワードなしで安全な通信が実現できます。

🔒

秘密鍵（Private Key）

自分のPC内に保管するファイル（~/.ssh/id_ed25519）。絶対に他人に見せてはいけません。

🔓

公開鍵（Public Key）

GitHubに登録するファイル（~/.ssh/id_ed25519.pub）。共有しても安全です。

3.2 SSH鍵の生成手順

Step 1: SSH鍵の生成 Terminal 
# Ed25519アルゴリズムでSSH鍵を生成
ssh-keygen -t ed25519 -C "your-email@example.com"

# 出力:
# Generating public/private ed25519 key pair.
# Enter file in which to save the key (/Users/you/.ssh/id_ed25519):
#   → Enterキーを押す（デフォルトのパスでOK）
# Enter passphrase (empty for no passphrase):
#   → パスフレーズを入力（セキュリティ向上、空でもOK）
# Enter same passphrase again:
#   → 同じパスフレーズを再入力
Step 2: SSH-agentに鍵を登録 Terminal 
# SSH-agentをバックグラウンドで起動
eval "$(ssh-agent -s)"
# 出力例: Agent pid 12345

# 秘密鍵をSSH-agentに登録
ssh-add ~/.ssh/id_ed25519
Step 3: 公開鍵をコピー Terminal 
# macOSの場合
pbcopy < ~/.ssh/id_ed25519.pub

# Windowsの場合（Git Bash）
clip < ~/.ssh/id_ed25519.pub

# Linuxの場合
cat ~/.ssh/id_ed25519.pub
# 表示された内容を手動でコピー

3.3 GitHubへの公開鍵の登録

  1. GitHubにログインし、右上のアイコンをクリックしてSettingsを選択
  2. 左メニューからSSH and GPG keysをクリック
  3. New SSH keyボタンをクリック
  4. Titleに分かりやすい名前を入力（例: 「MacBook Pro」「会社PC」）
  5. Key欄にコピーした公開鍵を貼り付け
  6. Add SSH keyボタンをクリック
Step 4: SSH接続のテスト Terminal 
# GitHubへのSSH接続をテスト
ssh -T git@github.com

# 初回接続時の確認メッセージ
# The authenticity of host 'github.com (20.27.177.113)' can't be established.
# Are you sure you want to continue connecting (yes/no/[fingerprint])?
# → yes と入力

# 成功時の出力
# Hi username! You've successfully authenticated, but GitHub does not
# provide shell access.
warning 秘密鍵は絶対に共有しないこと

秘密鍵ファイル（id_ed25519、.pubが付かない方）は、パスワードと同じくらい重要な情報です。メールやチャットで送ったり、GitHubにアップロードしたりしてはいけません。共有するのは公開鍵（.pubが付くファイル）だけです。

Section 8-4

リポジトリの作成と削除

GitHubでリポジトリを作成する方法を学びましょう。Web画面から簡単に作成でき、さまざまなオプションを設定できます。

4.1 リポジトリの作成手順

  1. GitHubにログインし、右上の+ボタンからNew repositoryを選択
  2. Repository nameにリポジトリ名を入力
  3. Descriptionにリポジトリの説明を入力（任意）
  4. 公開範囲を選択（Public or Private）
  5. 初期ファイルを選択（README、.gitignore、License）
  6. Create repositoryボタンをクリック
設定項目 説明 おすすめ
Public 誰でも閲覧可能 OSS、ポートフォリオ
Private 招待されたメンバーのみ閲覧可能 業務プロジェクト、個人の下書き
README.md プロジェクトの説明ファイル チェックを入れる（推奨）
.gitignore Git管理から除外するファイルの設定 使用言語に合わせて選択
License オープンソースライセンス OSSの場合はMITなどを選択

4.2 ローカルリポジトリとの連携

既存のローカルリポジトリをGitHubに連携する Terminal 
# 1. GitHubで空のリポジトリを作成（READMEなし）

# 2. ローカルリポジトリにリモートを追加
git remote add origin git@github.com:username/my-project.git

# 3. 初回プッシュ（-u でデフォルトのリモートブランチを設定）
git push -u origin main

# 以降は git push だけでプッシュ可能
git push
GitHubのリポジトリをクローンする Terminal 
# SSH URLでクローン（推奨）
git clone git@github.com:username/my-project.git

# HTTPS URLでクローン
git clone https://github.com/username/my-project.git

# 特定のディレクトリ名でクローン
git clone git@github.com:username/my-project.git my-local-folder

4.3 リポジトリの削除

不要になったリポジトリは削除できます。ただし、削除は取り消せないため慎重に行いましょう。

  1. リポジトリのページでSettingsタブをクリック
  2. ページの最下部にあるDanger Zoneまでスクロール
  3. Delete this repositoryをクリック
  4. 確認のためリポジトリ名を入力
  5. 削除を実行
warning リポジトリの削除は元に戻せない

リポジトリを削除すると、コード、Issues、Pull Requests、Wiki、GitHub Pages、リリースなど、全てのデータが完全に削除されます。削除前にローカルにクローンしてバックアップを取っておくことを強くおすすめします。

Exercise

8章 練習問題

Q1

SSH鍵のペアのうち、GitHubに登録するのはどちらですか？

  1. 秘密鍵（id_ed25519）
  2. 公開鍵（id_ed25519.pub）
  3. 両方とも登録する
  4. どちらも登録しない
Q2

GitHubでリポジトリを作成する際、招待されたメンバーのみがアクセスできる設定はどれですか？

  1. Public
  2. Private
  3. Internal
  4. Protected
Q3

SSH鍵を生成するコマンドとして正しいものはどれですか？

  1. git keygen -t ed25519
  2. ssh-keygen -t ed25519 -C "email@example.com"
  3. github ssh-key generate
  4. git ssh-setup -t ed25519
Answer

8章 解説・解答

Q1の解答: B

GitHubに登録するのは公開鍵（id_ed25519.pub）です。秘密鍵（id_ed25519）は自分のPCに保管し、絶対に他人と共有してはいけません。公開鍵暗号方式では、公開鍵で暗号化したデータは対応する秘密鍵でしか復号できないため、公開鍵を共有しても安全です。

Q2の解答: B

Privateリポジトリは、オーナーと招待されたコラボレーターのみがアクセスできます。Publicは誰でも閲覧可能です。InternalはGitHub Enterprise向けの設定で、同じ組織内のメンバーのみがアクセスできます。Protectedはリポジトリの公開設定ではなく、ブランチの保護設定です。

Q3の解答: B

ssh-keygenはSSH鍵を生成するためのコマンドです。-t ed25519は鍵の暗号化アルゴリズム（Ed25519）を指定し、-Cはコメント（通常はメールアドレス）を付加するオプションです。このコマンドはGitではなく、SSHツールの一部です。
