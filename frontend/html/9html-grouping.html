Section 9-1

グループ分けってなに？

ここまでは主に「表示」に関するタグについて学んできました。
ここからは、ブラウザや検索エンジンの評価に対して意味を紐づける構造用のタグについて学びます。

Webサイトは、基本的にグループの塊で構成していきます。
たとえば…

  • ページ上部のヘッダー（タイトルやナビゲーション）
  • サイトの主テーマとなる記事の枠（h2が使われる塊）
  • その記事の中の章立てや本文、画像（h3~h6、pタグ、imgなどが使われる塊）
  • サイドメニューや広告関連のエリア（aタグやul+li が多い塊）
  • 著作権表記などのフッター

こうして見るとそれぞれがひとつのグループとなっており、
これらの構造の意味を伝えるために使用するのが「セマンティックタグ」と呼ばれるタグです。

例：記事をグループ化する

HTML 
<h1>今日のおすすめレシピ</h1>
<h2>ふわふわパンケーキ</h2>
<p>バターとメープルシロップがたまらない定番スイーツ！</p>
<h2>簡単オムライス</h2>
<p>子どもも喜ぶふわとろ食感。冷ご飯で簡単調理！</p>

このままでも表示がされるにはされますし、私たち人間はこれらが「今日のおすすめレシピ」という
テーマ（グループ）に対して書かれているものであり、<h2>がその中の章で、バターとメープル……が本文で1セット。
簡単オムライスが次の章テーマで、子供も喜ぶ……が本文でまた別の1セットで……
と、脳内でどこがどれと紐づいていて、どこまでがひとつのグループか、ということを理解できます。

ですが、コンピュータはこれだけではわからないのです。
コンピュータにはただ「大きい見出し」「小さい見出し」「本文」「小さい見出し」「本文」としか解釈しません。

だからこそ、コンピュータにはどこからどこまでがひとつのグループですよ！
というのを伝えてあげる必要があります。

Section 9-2

代表的なセマンティックタグを使用した基本形

Webページを作る際には、基本となるグループがあります。
それが以下になります。

タグ名 意味 使いどころ
<header> セクションのヘッダー Webページの最上部、主要テーマのヘッダー部分
<nav> グローバルナビゲーション ヘッダーやフッターの中のリンク
<main> メインコンテンツエリア Webページのメインコンテンツとなる領域
<article> 独立した主要テーマ記事 メインコンテンツの大きなコンテンツセクション
<section> テーマ内の小区切り <article>内での章ごとの区切りとして
<aside> コンテンツとは別の関連情報 サイドバーなどの広告情報やプロフィールリンク
<footer> セクションのフッター コピーライトや最後の締めに

注意

<main>はWebページ内で1つしか使えません

Section 9-3

意味のない部分をまとめる <div> タグ

最後にもうひとつ。
意味はないけれど、CSSやJavaScriptで要素を操作するのに、
グループ分けをして管理・適用したい、という場合があります。
そんなときは、意味を持たないグループとして<div>タグ、というので囲みます。

HTML 
<div class="gallery">
  <img src="images/photo1.jpg" alt="富士山の朝日">
  <img src="images/photo2.jpg" alt="河口湖の夜景">
</div>

ファーストビューなどはHTMLなどではよく「ヒーローエリア」などと言われます。
専用のセマンティックタグなどはありませんが、グループとしてまとめたいとき、
<div class = "hero-area"> というようなクラス属性を使用したりしてまとめます。

どこでも、いつでもさっと小さなグループを作れる汎用的なタグです。

POINT！

<div> は意味が存在しないため、クラスなどを付与してもその意味で解釈されるわけではありません。
セマンティックタグが存在するものはセマンティックタグを使い、
該当のタグがないけどグループ化したい、というときだけ<div>を使いましょう。

まとめ

第9章のまとめ

  • Webサイトはグループの塊で構成する
  • セマンティックタグで構造の意味をコンピュータに伝える
  • <header> <nav> <main> <article> <section> <aside> <footer>が代表的
  • <main>は1ページに1つだけ
  • 意味のないグループ化には<div>を使う
  • セマンティックタグがあるものは<div>ではなくセマンティックタグを使う
