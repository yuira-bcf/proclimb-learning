Section 9-1

Tailwind CSSとは

Tailwind CSSは、ユーティリティファーストのCSSフレームワークです。あらかじめ用意された小さなクラスを組み合わせてスタイリングします。

1.1 従来のCSSとの違い

従来のCSS 
<button class="btn">ボタン</button>

.btn {
    padding: 10px 20px;
    background: blue;
    color: white;
    border-radius: 8px;
}
Tailwind CSS 
<button class="px-5 py-2 bg-blue-500 text-white rounded-lg">
    ボタン
</button>

1.2 CDNで始める

chat プロンプト 
## 機能
Tailwind CSSをCDNで読み込んで使用開始

## 実装要件
- CDNからTailwind CSSを読み込み
- HTMLファイル単体で動作
- 学習・開発用途に最適
- すぐに試せる環境構築

## 使用するCDN
- https://cdn.tailwindcss.com
- scriptタグでhead内に配置
CDN読み込み HTML 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Tailwind CSS</title>
    <!-- Tailwind CSS CDN -->
    <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
    <h1 class="text-3xl font-bold text-blue-600">Hello Tailwind!</h1>
</body>
</html>
menu_book 構文説明

Tailwind CDNの読み込み

<script src="https://cdn.tailwindcss.com"></script>

CDN（Content Delivery Network）経由でTailwind CSSを読み込みます。scriptタグをhead内に配置するだけで、すべてのユーティリティクラスが使えるようになります。学習や開発時に便利ですが、本番環境ではnpmでのインストールとビルドを推奨します。

CDNは学習・開発用

CDN版は学習や開発に便利ですが、本番環境ではnpmでインストールしてビルドすることを推奨します。

2.1 色の指定

chat プロンプト 
## 機能

## 実装要件
- 文字色、背景色、ボーダー色を指定

## 使用するTailwindクラス

<!-- 背景色 -->

menu_book 構文説明

ユーティリティクラス（colors）

text-{色名}-{濃度} bg-{色名}-{濃度} border-{色名}-{濃度}

Tailwindの色指定はプロパティ-色名-濃度の形式です。text-は文字色、bg-は背景色、border-はボーダー色を指定します。濃度は50（最も薄い）から950（最も濃い）まであり、500が標準的な濃さです。

カラーパレット

各色には50〜950の濃淡があります。50が最も薄く、950が最も濃いです。
例：blue-50, blue-100, ... blue-500, ... blue-900

Flexboxユーティリティ

flex ... Flexコンテナ化 flex-row / flex-col ... 方向 justify-{位置} ... 主軸の配置 items-{位置} ... 交差軸の配置 gap-{数値} ... 間隔

<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
    <div>カード1</div>
    <div>カード2</div>
    <div>カード3</div>
</div>

<!-- モバイルで非表示、PC以上で表示 -->

<!-- モバイルで表示、PC以上で非表示 -->
<div class="block lg:hidden">モバイル用コンテンツ</div>

<!-- レスポンシブなパディング -->
<div class="p-4 md:p-6 lg:p-8">...</div>

<!-- レスポンシブなフォントサイズ -->
<h1 class="text-2xl md:text-4xl lg:text-6xl">見出し</h1>
menu_book 構文説明

レスポンシブプレフィックス

sm: (640px以上) md: (768px以上) lg: (1024px以上) xl: (1280px以上) 2xl: (1536px以上)

ブレークポイントプレフィックスを付けると、その画面幅以上でのみスタイルが適用されます。Tailwindはモバイルファーストなので、プレフィックスなしのクラスがモバイル用（すべての画面幅）の基本スタイルになります。

レスポンシブの記述例

class="grid-cols-1 md:grid-cols-2 lg:grid-cols-3"

この例では、モバイル（〜767px）は1列、タブレット（768px〜1023px）は2列、PC（1024px〜）は3列になります。小さい画面から順にスタイルを積み上げていく考え方です。

モバイルファースト

Tailwindはモバイルファーストです。プレフィックスなしはモバイル用、md:以上はタブレット以上という意味になります。

Exercise

9章 練習問題

Q1

Tailwind CSSでタブレット（768px）以上でのみ適用するクラスの書き方はどれですか？

  1. tablet:flex
  2. md:flex
  3. @md flex
  4. min-768:flex
Q2

ホバー時に背景色を変えるクラスの書き方はどれですか？

  1. hover-bg-blue-600
  2. bg-blue-600:hover
  3. hover:bg-blue-600
  4. on-hover:bg-blue-600
Answer

9章 解説・解答

Q1の解答: B

md:プレフィックスは768px以上に適用されます。Tailwindのブレークポイントはsm:, md:, lg:, xl:, 2xl:があります。

Q2の解答: C

Tailwindでは状態を表すプレフィックスを先に書きます。hover:bg-blue-600でホバー時にbg-blue-600が適用されます。
