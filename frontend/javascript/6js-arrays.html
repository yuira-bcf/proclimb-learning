Section 1

配列（Array）とは？

一言でいうと…「複数のデータを、整理しながらまとめて保存できる箱」

たとえば、3人の名前を変数で書くと：

JavaScript 
let name1 = "たろう";
let name2 = "はなこ";
let name3 = "さくら";

となってしまう。ただの変数では、データをひとつしか保存できない。そのため、増やせば増やすほど別名の変数を用意しなければならず、管理が大変になる。

これをまとめて保管しておけるのが「配列」という変数です。

基本構文：

JavaScript 
let 変数名 = [ 値1 , 値2, 値3……];

実際の記述：名前を3つ、配列に入れる

JavaScript 
let names = ["たろう", "はなこ", "さくら"];

上記は「names」という大箱に、3つの小箱が入った状態となります。この小箱には、それぞれインデックス番号（または添え字、と呼ばれる）が振られます。番号は0から始まり、[]（ブラケット）で管理されます。

配列の中身の例：

  • names[0]　⇒ "たろう"
  • names[1]　⇒ "はなこ"
  • names[2]　⇒ "さくら"

この配列から、任意の値を取り出すには下記のように記述します。

JavaScript 
console.log(names[0]);
console.log(names[1]);
console.log(names[2]);

コンソール表示結果：

たろう
はなこ
さくら
Section 2

配列に値を追加する

配列に値を追加する方法はいくつかあります。

2-1. インデックスを指定して追加

上記のnames 変数をそのまま使う前提とします。

JavaScript 
names[3] = "けん";
console.log(names);

結果：["たろう", "はなこ", "さくら", "けん"]

指定したインデックスにのみ値が入ります。

JavaScript 
names[5] = "なつみ";
console.log(names);

結果：["たろう", "はなこ", "さくら", "けん", "", "なつみ"]

2-2. 末尾に自動追加（pushメソッド）

インデックスを指定せずとも、自動的に最後の配列の後に値を追加することも可能です。

JavaScript 
let names = ["たろう", "はなこ", "さくら"];
names.push("なつみ");
console.log(names);

結果：["たろう", "はなこ", "さくら", "なつみ"]

2-3. 先頭に追加（unshiftメソッド）

配列の先頭に、値を挿入することもできます。

JavaScript 
let names = ["たろう", "はなこ", "さくら"];
names.unshift("なつみ");
console.log(names);

結果：["なつみ", "たろう", "はなこ", "さくら"]

Section 3

配列の値を更新・削除する

一度代入した配列の値やインデックスは、更新と削除がどちらも可能です。

3-1. 更新（値の再代入）

通常の変数の再代入と、方法は一緒です。

JavaScript 
let names = ["たろう", "はなこ", "さくら"];
names[1] = "なつみ";
console.log(names);

結果：['たろう', 'なつみ', 'さくら']

3-2. 末尾のインデックスを削除

JavaScript 
let names = ["たろう", "はなこ", "さくら"];
names.pop();
console.log(names);

結果：['たろう', 'はなこ']

3-3. 先頭のインデックスを削除

JavaScript 
let names = ["たろう", "はなこ", "さくら"];
names.shift();
console.log(names);

結果：['はなこ', 'さくら']

まとめ：配列の基本操作

配列、names に 「たろう」「はなこ」「さくら」が入っている場合

操作 コード例 結果
要素を追加（最後） names.push("けん") ["たろう","はなこ","さくら","けん"]
要素を追加（先頭） names.unshift("けん") ["けん", "たろう","はなこ","さくら"]
要素を削除（最後） names.pop() ["たろう","はなこ"]
要素を削除（先頭） names.shift() ["はなこ", "さくら"]
全部見る console.log(names) ["たろう","はなこ","さくら"]
Section 4

配列のインデックスの数（配列の長さ）を取得する（.lengthメソッド）

.lengthというメソッドを使うと、配列の長さを数字で取得することが出来ます。

JavaScript 
let fruits = ["りんご", "みかん", "ぶどう"];
console.log("配列の要素の長さは " + fruits.length + " です。");

結果：配列の要素の長さは 3 です。

これを利用することで、for文との相性が抜群に上がります。

Section 5

for文と組み合わせて、配列の中身をすべて抽出する

for文の条件式の部分に.lengthを使用することで、配列の中身をすべて抽出できます。

JavaScript 
let fruits = ["りんご", "みかん", "ぶどう"];

for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

結果：

りんご
みかん
ぶどう
POINT

条件式が「i < 配列の長さ」という形になるため、配列の中身がどれだけ増えようとも、for文は常に配列の長さ分だけを繰り返します。初期値として、i = 0とすることで、取り出すときは必ずインデックス[0]から始められます。

配列＋forEach

配列に限っては、もっと簡単に配列の中身をすべて取り出せる専用のfor文があります。それがforEachメソッドです。

JavaScript 
fruits.forEach((f) => console.log(f));

結果：

りんご
みかん
ぶどう

たったこれだけで、for文で.lengthを使った記述とまったく同じ挙動になります。

繰り返し処理の流れ

  1. 配列fruitsのインデックス[0]を取り出す
  2. 取り出した値を一時変数fに代入する
  3. console.logでfを表示する
  4. 配列の中身がまだ残っている場合は、次のインデックス[]の値を取り出す
  5. 次に取り出した値を、再び一時変数fに代入する
  6. console.logで……以下略
  7. 配列の中身が残っていない場合、繰り返し処理を終了する
まとめ

第6章のまとめ

  • 配列は複数のデータをまとめて管理できる
  • インデックスは0から始まる
  • push()で末尾に追加、unshift()で先頭に追加
  • pop()で末尾を削除、shift()で先頭を削除
  • .lengthで配列の長さを取得
  • for文やforEachで配列の中身をすべて処理できる
