Section 11-1

tailwind.config.js の基本

設定ファイルの構造

Tailwind CSSの大きな魅力のひとつは、設定ファイルを通じてフレームワーク全体をカスタマイズできることです。プロジェクトのルートに配置される tailwind.config.js ファイルが、すべてのカスタマイズの起点となります。

設定ファイルは大きく分けて content（対象ファイルの指定）、theme（デザイントークンの定義）、extend（既存テーマの拡張）、plugins（プラグインの追加）の4つのセクションで構成されています。

JavaScript
/** @type {import('tailwindcss').Config} */
module.exports = {
  // 1. Tailwindを適用するファイルのパス
  content: [
    "./src/**/*.{html,js,jsx,ts,tsx}",
    "./public/index.html",
  ],

  // 2. テーマの設定
  theme: {
    // 既存の値を完全に上書きする場合
    screens: {
      sm: '640px',
      md: '768px',
      lg: '1024px',
      xl: '1280px',
    },

    // 既存の値に追加する場合
    extend: {
      colors: {
        brand: '#06B6D4',
      },
      fontFamily: {
        sans: ['Noto Sans JP', 'sans-serif'],
      },
    },
  },

  // 3. プラグイン
  plugins: [],
}

content配列の設定

content プロパティは、Tailwindがクラス名をスキャンする対象ファイルを指定します。ここに含まれないファイルのクラスは、ビルド時に生成されるCSSに含まれません。

JavaScript
module.exports = {
  content: [
    // HTMLファイル
    "./index.html",
    "./pages/**/*.html",

    // JavaScriptファイル（React, Vue等）
    "./src/**/*.{js,jsx,ts,tsx,vue}",

    // テンプレートファイル
    "./templates/**/*.ejs",
    "./components/**/*.hbs",

    // nodeモジュール内のUIライブラリ
    "./node_modules/@my-ui-lib/**/*.js",
  ],
}
プロパティ 説明 必須
content クラス名をスキャンするファイルパスの配列 はい
theme デザイントークン（色、フォント、スペーシング等）の定義 いいえ
theme.extend 既存テーマに値を追加（上書きではない） いいえ
plugins 追加プラグインの配列 いいえ
darkMode ダークモードの戦略（'class' または 'media'） いいえ
prefix 全クラスに付与する接頭辞（例: 'tw-'） いいえ
important 全ユーティリティに !important を付与するか いいえ
separator 修飾子の区切り文字（デフォルト: ':'） いいえ
v3とv4の設定方法の違い

Tailwind CSS v4ではゼロコンフィグを目指しており、設定ファイルが不要になるケースが増えています。v4では @theme ディレクティブを使ってCSS内で直接テーマをカスタマイズできます。本章ではv3の tailwind.config.js をベースに解説しますが、v4への移行も視野に入れておきましょう。

Section 11-2

テーマの拡張（extend）

theme.extend の役割

Tailwindのデフォルトテーマには、豊富な色、スペーシング、フォントサイズなどが用意されています。theme.extend を使うと、これらのデフォルト値を保ちながら、新しい値を追加できます。

注意: theme 直下に記述すると既存の値が完全に上書きされますが、theme.extend 内に記述すると既存の値に追加されます。

カスタムカラーの追加

プロジェクトのブランドカラーを追加する場合、extend.colors にオブジェクトとして定義します。数値キーを使うとTailwindの標準的な色の濃淡パターンに合わせられます。

JavaScript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        // 単色の追加
        brand: '#06B6D4',

        // 濃淡バリエーションの追加
        primary: {
          50:  '#E0F7FA',
          100: '#B2EBF2',
          200: '#80DEEA',
          300: '#4DD0E1',
          400: '#26C6DA',
          500: '#06B6D4',  // メインカラー
          600: '#0891B2',
          700: '#0E7490',
          800: '#155E75',
          900: '#164E63',
        },

        // セマンティックカラー
        success: '#10B981',
        warning: '#F59E0B',
        danger:  '#EF4444',
      },
    },
  },
}

// 使用例:
// bg-brand, text-primary-500, border-primary-200
// bg-success, text-danger, border-warning

カスタムフォントファミリー

日本語フォントを使ったプロジェクトでは、fontFamily のカスタマイズが重要です。

JavaScript
// tailwind.config.js
const defaultTheme = require('tailwindcss/defaultTheme')

module.exports = {
  theme: {
    extend: {
      fontFamily: {
        // デフォルトのsansに日本語フォントを追加
        sans: [
          'Noto Sans JP',
          ...defaultTheme.fontFamily.sans,
        ],
        // コード表示用フォント
        mono: [
          'JetBrains Mono',
          ...defaultTheme.fontFamily.mono,
        ],
        // 見出し用フォント
        heading: [
          'Noto Sans JP',
          'sans-serif',
        ],
      },
    },
  },
}

// 使用例:
// font-sans  → Noto Sans JP, system-ui, ...
// font-mono  → JetBrains Mono, ui-monospace, ...
// font-heading → Noto Sans JP, sans-serif

カスタムスペーシングとブレークポイント

スペーシングやブレークポイントもプロジェクトに合わせて拡張できます。

JavaScript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      // カスタムスペーシング
      spacing: {
        '18': '4.5rem',   // 72px
        '88': '22rem',    // 352px
        '128': '32rem',   // 512px
        'header': '64px', // ヘッダーの高さ
        'sidebar': '280px', // サイドバーの幅
      },

      // カスタムブレークポイント
      screens: {
        'xs': '475px',     // 小さいスマホ向け
        '3xl': '1600px',   // 大画面モニター向け
        'tall': { 'raw': '(min-height: 800px)' }, // 高さベース
      },

      // カスタム最大幅
      maxWidth: {
        'content': '720px',
        'wide': '1400px',
      },

      // カスタムz-index
      zIndex: {
        'dropdown': '1000',
        'modal': '2000',
        'tooltip': '3000',
      },
    },
  },
}

// 使用例:
// p-18, mt-header, w-sidebar
// xs:flex, 3xl:grid-cols-4, tall:py-12
// max-w-content, z-modal
extendプロパティ 説明 使用例
colors カスタムカラーの追加 bg-brand, text-primary-500
fontFamily フォントファミリーの追加 font-sans, font-heading
spacing スペーシング値の追加 p-18, mt-header
screens ブレークポイントの追加 xs:flex, 3xl:grid-cols-4
fontSize フォントサイズの追加 text-xxs, text-huge
borderRadius 角丸の値の追加 rounded-4xl
boxShadow 影の追加 shadow-card, shadow-soft
animation アニメーションの追加 animate-fade-in
maxWidth 最大幅の追加 max-w-content
zIndex z-indexの追加 z-modal, z-tooltip
Section 11-3

@apply ディレクティブ

@applyとは

@apply は、TailwindのユーティリティクラスをカスタムCSSクラス内で再利用するためのディレクティブです。HTMLのクラス属性が長くなりすぎる場合や、繰り返し使うスタイルパターンをまとめたい場合に便利です。

ボタンクラスの定義

最もよく使われるパターンが、ボタンのスタイル定義です。同じボタンスタイルを複数箇所で使う場合、@apply でクラスを作成すると保守性が向上します。

CSS
/* styles.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* @applyでボタンクラスを定義 */
.btn {
  @apply inline-flex items-center justify-center
         px-6 py-2.5 rounded-lg
         font-semibold text-sm
         transition-colors duration-200
         focus:outline-none focus:ring-2 focus:ring-offset-2;
}

.btn-primary {
  @apply btn bg-cyan-500 text-white
         hover:bg-cyan-600
         focus:ring-cyan-500;
}

.btn-secondary {
  @apply btn bg-gray-100 text-gray-700
         hover:bg-gray-200
         focus:ring-gray-500;
}

.btn-outline {
  @apply btn border-2 border-cyan-500 text-cyan-600
         hover:bg-cyan-50
         focus:ring-cyan-500;
}

.btn-danger {
  @apply btn bg-red-500 text-white
         hover:bg-red-600
         focus:ring-red-500;
}

.btn-lg {
  @apply px-8 py-3 text-base;
}

.btn-sm {
  @apply px-4 py-1.5 text-xs;
}

@applyの使いすぎに注意

@apply を多用しすぎると、Tailwindの「HTMLを見ればスタイルがわかる」という利点が失われます。Tailwindの作者であるAdam Wathan氏も「@applyは最後の手段」と述べています。基本的にはHTMLにユーティリティクラスを直接書き、どうしても繰り返しが発生する場合にのみ @apply を使いましょう。

POINT: @applyを使うべき場面

使うべき場面: 同じボタンスタイルが10箇所以上で使われる場合、フォームの入力欄など統一が必要な要素、CMSなどHTMLを直接編集できない場面。
使わない方がよい場面: 1〜2箇所でしか使わないスタイル、レスポンシブやホバー状態だけが異なる場合（ユーティリティクラスの方が柔軟）、コンポーネントフレームワーク（React/Vue）を使っている場合（コンポーネント化で解決できる）。

Section 11-4

カスタムクラスとCSS変数

CSS変数との組み合わせ

CSS変数（カスタムプロパティ）とTailwindを組み合わせることで、テーマの動的な切り替えやブランドカラーの一元管理が可能になります。

@layer ディレクティブ

Tailwindでは @layer ディレクティブを使って、カスタムCSSをTailwindのレイヤーシステムに統合できます。これにより、スタイルの優先順位が適切に管理されます。

CSS
@tailwind base;
@tailwind components;
@tailwind utilities;

/* @layer base: リセットやデフォルトスタイル */
@layer base {
  html {
    @apply scroll-smooth;
  }

  body {
    @apply text-gray-800 bg-gray-50
           font-sans leading-relaxed;
  }

  h1 {
    @apply text-3xl font-bold text-gray-900;
  }

  h2 {
    @apply text-2xl font-bold text-gray-900;
  }

  h3 {
    @apply text-xl font-semibold text-gray-800;
  }

  a {
    @apply text-cyan-600 hover:text-cyan-700
           transition-colors;
  }
}

/* @layer components: 再利用可能なコンポーネント */
@layer components {
  .container-custom {
    @apply max-w-6xl mx-auto px-4 sm:px-6 lg:px-8;
  }

  .section-title {
    @apply text-2xl font-bold text-gray-900
           mb-2 pb-2 border-b-2 border-cyan-500;
  }

  .badge {
    @apply inline-flex items-center px-2.5 py-0.5
           rounded-full text-xs font-medium;
  }

  .badge-primary {
    @apply badge bg-cyan-100 text-cyan-800;
  }

  .badge-success {
    @apply badge bg-green-100 text-green-800;
  }

  .badge-warning {
    @apply badge bg-yellow-100 text-yellow-800;
  }
}

/* @layer utilities: カスタムユーティリティ */
@layer utilities {
  .text-balance {
    text-wrap: balance;
  }

  .scrollbar-hide {
    -ms-overflow-style: none;
    scrollbar-width: none;
  }

  .scrollbar-hide::-webkit-scrollbar {
    display: none;
  }
}

CSS変数でテーマカラーを管理

CSS
/* CSS変数でテーマを定義 */
:root {
  --color-primary: 6 182 212;     /* cyan-500 */
  --color-secondary: 139 92 246;  /* violet-500 */
  --color-accent: 245 158 11;     /* amber-500 */
  --color-surface: 255 255 255;
  --color-text: 31 41 55;
}

/* ダークテーマ */
.dark {
  --color-primary: 34 211 238;    /* cyan-400 */
  --color-secondary: 167 139 250; /* violet-400 */
  --color-accent: 251 191 36;     /* amber-400 */
  --color-surface: 31 41 55;
  --color-text: 243 244 246;
}
JavaScript
// tailwind.config.js でCSS変数を参照
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: 'rgb(var(--color-primary) / <alpha-value>)',
        secondary: 'rgb(var(--color-secondary) / <alpha-value>)',
        accent: 'rgb(var(--color-accent) / <alpha-value>)',
        surface: 'rgb(var(--color-surface) / <alpha-value>)',
        'on-surface': 'rgb(var(--color-text) / <alpha-value>)',
      },
    },
  },
}

// 使用例:
// bg-primary          → var(--color-primary)
// bg-primary/50       → var(--color-primary) 50%透明度
// text-on-surface     → var(--color-text)
// dark:bg-primary     → ダークモードでCSS変数が切り替わる

Section 11-5

プラグイン

公式プラグイン

Tailwind CSSには、よく使われる機能を追加する公式プラグインが用意されています。これらを導入することで、フォームのスタイルリセットやリッチテキストの表示など、実践的な機能を簡単に追加できます。

プラグイン パッケージ名 機能
Forms @tailwindcss/forms フォーム要素のベーススタイルをリセットし、ユーティリティクラスで簡単にカスタマイズ可能に
Typography @tailwindcss/typography prose クラスでマークダウンやリッチテキストを美しく表示
Aspect Ratio @tailwindcss/aspect-ratio 要素のアスペクト比を制御するユーティリティ（v3.x以降は組み込み）
Container Queries @tailwindcss/container-queries コンテナクエリに基づくレスポンシブデザイン

プラグインのインストールと設定

Bash
# 公式プラグインのインストール
npm install -D @tailwindcss/forms
npm install -D @tailwindcss/typography
npm install -D @tailwindcss/container-queries

# まとめてインストール
npm install -D @tailwindcss/forms @tailwindcss/typography @tailwindcss/container-queries
JavaScript
// tailwind.config.js
module.exports = {
  content: ["./src/**/*.{html,js}"],
  theme: {
    extend: {},
  },
  plugins: [
    require('@tailwindcss/forms'),
    require('@tailwindcss/typography'),
    require('@tailwindcss/container-queries'),
  ],
}

カスタムプラグインの作成

独自のユーティリティやコンポーネントをプラグインとして作成することもできます。チーム内で共有する共通スタイルをプラグインにまとめると便利です。

JavaScript
// tailwind.config.js
const plugin = require('tailwindcss/plugin')

module.exports = {
  plugins: [
    // カスタムプラグインの例
    plugin(function({ addComponents, addUtilities, theme }) {

      // カスタムコンポーネント
      addComponents({
        '.card-elevated': {
          backgroundColor: theme('colors.white'),
          borderRadius: theme('borderRadius.xl'),
          boxShadow: theme('boxShadow.lg'),
          padding: theme('spacing.6'),
          transition: 'box-shadow 0.2s ease',
          '&:hover': {
            boxShadow: theme('boxShadow.xl'),
          },
        },
        '.section-divider': {
          borderTop: `2px solid ${theme('colors.gray.200')}`,
          margin: `${theme('spacing.8')} 0`,
        },
      })

      // カスタムユーティリティ
      addUtilities({
        '.content-auto': {
          contentVisibility: 'auto',
        },
        '.drag-none': {
          '-webkit-user-drag': 'none',
          userDrag: 'none',
        },
      })
    }),
  ],
}

    },
  }
</script>

まとめ

1
tailwind.config.js はTailwindカスタマイズの中心。content, theme, extend, plugins の4つの主要セクションで構成される
2
theme.extend を使えば、デフォルトのユーティリティを維持したまま、カスタムカラー・フォント・スペーシングなどを追加できる
3
@apply ディレクティブでユーティリティクラスをCSS内で再利用できるが、使いすぎるとTailwindの利点が失われるため注意
4
@layer ディレクティブ（base, components, utilities）を使って、カスタムCSSをTailwindのレイヤーシステムに適切に統合する
5
CSS変数とTailwindを組み合わせることで、テーマの動的切り替えやダークモード対応が容易になる
6
公式プラグイン（forms, typography, container-queries）を活用して、よく使う機能を効率的に追加できる
7
カスタムプラグインを作成すれば、チーム共通のコンポーネントやユーティリティを再利用可能な形で共有できる
