品質チェックリスト

AI生成コードの品質確認に使うチェックリスト

HTML品質

  • セマンティックタグを使用しているか
  • 見出しレベルが適切か（h1→h2→h3）
  • 画像にalt属性が設定されているか
  • フォームにlabelが紐付けされているか

CSS品質

  • !importantを使用していないか
  • CSS変数でカラーを管理しているか
  • 重複したスタイルがないか
  • BEM命名規則に従っているか

レスポンシブ

  • モバイル（375px）で崩れないか
  • タブレット（768px）で正しく表示されるか
  • タップ領域が44px以上あるか

よくある失敗パターンと対策
NG パターン
曖昧なプロンプト
「おしゃれなボタンを作って」のような抽象的な指示では意図したUIにならない。
OK パターン
具体的な仕様を記載
サイズ・色・角丸・ホバー効果など数値で具体的に指定する。
NG パターン
検証せずに使用
AIの出力をそのまま本番に適用すると、アクセシビリティやレスポンシブの問題が残る。
OK パターン
チェックリストで確認
品質チェックリストで必ず確認してから採用する。
