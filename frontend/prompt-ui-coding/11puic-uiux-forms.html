Section 11-1

なぜフォームデザインが重要なのか

入力フォームのデザインは、ユーザー体験（UX）を左右する重要なポイントです。どれだけ良いページを作っても、フォームが「使いづらい」「分かりにくい」状態だと、ユーザーはすぐに離脱してしまいます。

「良いデザイン」とは？

ここでいう「良いデザイン」は「オシャレ」という意味ではなく、ユーザーにとって操作しやすい、分かりやすいと感じるデザインのことを指します。

フォームデザインはユーザーの負担に直結

ユーザーは「入力が簡単」「見やすい」「操作しやすい」など、入力フォームを操作する時の快適さを重視しています。

本章では、フォームUXを改善する20のポイントを「全体設計」「項目設計」「エラー表示」の3カテゴリで解説します。

Section 11-2

フォーム全体の設計（8ポイント）

特定の入力項目に依存しない、フォーム全体に適用する基本原則です。

1. 入力フォームは縦に並べる

入力項目を横並び（2カラム以上）に配置すると、ユーザーの視線が左右に揺れて、どこから入力すべきか分からないという問題が起こります。

入力項目を縦一列に並べることで、視線の流れが「上→下」になり、ユーザーは入力で迷いにくくなり、入力漏れを減らす効果があります。（姓・名などは例外的に2列でも可）

悪い例：横並び
姓
山田
名
太郎
メール
email@...

視線が左右に動き、入力しづらい

良い例：縦並び
お名前
山田 太郎
メールアドレス
email@example.com

視線が上から下に流れ、入力しやすい

2. 入力項目は上から自然な順序に並べる

入力項目の順番が不自然だと、「次に何を答えればいいの？」「なんでこのタイミングでこれ？」と迷いや違和感を覚え、ストレスを与える原因になります。

ユーザーの頭の中の流れ（自然に思い浮かべる順番）に合わせて配置します。基本的には「自己紹介の順番」に合わせると良いでしょう。

自然な順序の例

  1. お名前
  2. メールアドレス
  3. 電話番号
  4. 会社名
  5. お問い合わせ内容

3. ラベルは項目の上に配置

ラベル（項目名）を入力欄の左側に置くと、入力欄とラベルの関係が分かりにくい、エラー発生時に何の項目が原因かが分かりにくくなります。

ラベルは必ず入力欄の上に配置し、項目とラベルを「1つのまとまり」として扱いましょう。これで視線の流れが自然になり、ユーザーはスムーズに入力できます。

悪い例：左配置
メールアドレス
email@...

ラベルと入力欄の関係が分かりにくい

良い例：上配置
メールアドレス
email@example.com

1つのまとまりとして認識しやすい

4. ラベルと入力項目をグループ化

入力フォームの各項目が余白なく詰め込まれていると、ユーザーはどこからどこまでが同じ項目なのか判断しにくくなります。

入力項目とラベル、補足情報などを1つのグループとして捉え、グループごとに十分な余白を設けることを意識しましょう。

5. プレースホルダーは入力例だけ

プレースホルダーは入力を始めると消えるので、ラベルや説明文代わりに使うと、入力後に「何の入力欄だっけ」と分からなくなってしまいます。

プレースホルダーには「直感的にどのような形式で入力すれば良いか」が分かる入力例だけを表示しましょう。

悪い例
メールアドレスを入力してください

ラベル代わりに使用している

良い例
メールアドレス
example@email.com

入力例だけを表示

6. 必須・任意は一目で分かるように

必須が分かりにくいと、ユーザーの入力漏れを引き起こしやすくなります。エラーが多発して、入力にストレスを感じ、離脱に繋がります。

入力項目が必須なのか、任意なのかは、誰もが一目で分かるデザインにしましょう。任意項目をすべて削除して、「全項目必須です」と冒頭で宣言するのも良いです。

お名前必須
山田 太郎
電話番号任意
090-0000-0000

7. 入力中のフォーカスを目立たせる

入力欄のフォーカス（選択中）の見た目が弱いと、ユーザーが「今どこを入力しているのか」を見失いやすくなります。

入力中のフィールドが一目で分かるように、フォーカス時のデザインをしっかり強調します。枠線を太くする、ブランドカラーで強調するなどで構いません。（エラーとは異なる配色を使ってください）

悪い例
メールアドレス
入力中...

フォーカスが分かりにくい

良い例
メールアドレス
入力中...

今どこを入力しているか明確

8. リンクは別タブで開く

フォーム内のリンク（プライバシーポリシー・利用規約など）を同じタブで開く仕様だと、戻るボタンで入力内容が消える可能性があります。

リンクは必ず別タブ（別ウィンドウ）で開く設定にしてください。また、別タブで開くことが見た目に分かるようにしておくとさらに良いです。

HTML 
<!-- 別タブで開くリンク -->
<a href="/privacy" target="_blank" rel="noopener">
  プライバシーポリシー
  <span class="material-icons">open_in_new</span>
</a>
Section 11-3

入力項目の設計（7ポイント）

テキスト入力、選択肢、ファイルアップロードなど、項目タイプ別の設計ポイントです。

1. 自由記入欄よりも選択式

自由記入欄（テキストエリア）ばかり使うと、ユーザーは「自分で考えて打つのが面倒、、」という入力コストが発生します。

できるだけラジオボタンやチェックボックス、ドロップダウンなど、選択式のUIを採用しましょう。ユーザーは「選ぶだけでいい」と軽い気持ちでフォーム入力を進められます。（お問い合わせ内容や備考などは除く）

悪い例：自由記入
お問い合わせ種別
お問い合わせの種類を入力...

何を書けばいいか迷う

良い例：選択式
お問い合わせ種別
製品について
サービスについて
その他

選ぶだけで簡単

2. 選択肢には逃げ道を用意する

選択式の項目で逃げ道（当てはまらない時の選択肢）がないと、「自分に当てはまる選択肢がない、どうしよう」と不安になってしまいます。

選択肢には、「その他」や「未定」、「相談したい」などの当てはまらない場合に選べる逃げ道の選択肢を用意します。これでユーザーはとりあえず前に進めるようになります。

逃げ道の選択肢の例

  • その他（自由記入）
  • 未定・検討中
  • 相談したい
  • わからない
  • 該当なし

3. 選択肢が少ないならラジオボタン

プルダウン（ドロップダウン）を使うと、「タップして開くひと手間が発生する」「開けてみないと選択肢が見えない」デメリットが存在します。

選択肢が3〜5と少ない場合は、ラジオボタンを使うようにしましょう。ただし、フォームの見た目をコンパクトにまとめたいなどの理由があるならドロップダウンでも構いません。

4. 選択肢が多いならドロップダウン

選択肢が多い時にラジオボタンを使うと、すべての選択肢がフォーム上に表示されるため、ユーザーは「選べない」「探せない」のストレスを感じてしまいます。

この場合は、ドロップダウン（プルダウン）を使って選択肢を折りたたむのが最適です。特に多い場合は、キーワードなどで選択肢を絞り込めるUIを採用しましょう。

選択肢の数 推奨UI 理由
2択 ラジオボタン / トグル 一目で選択できる
3〜5個 ラジオボタン 一覧性が高い
6個以上 ドロップダウン 省スペース
10個以上 検索付きドロップダウン 絞り込みで探しやすい

5. チェックボックスは縦に並べる

チェックボックスを横並び（2列以上）に配置すると、スマホで誤タップしやすい、チェック漏れしやすいなどの問題が発生します。

行につき1項目の縦並びにすることで、視線の流れが一定になり、見やすく操作しやすくなります。ただし、選択肢が非常に少ない場合は横並びでも問題ありません。

6. ファイルアップローダーには対応形式・容量を明記

ファイルアップロード欄に「どんなファイルを」「どれくらいの容量まで」アップできるのかが記載されていないと、エラーで戻される可能性が高まります。

ファイル添付は、入力項目の中でも離脱が発生しやすいので、対応形式（拡張子）と最大容量を明確に記載しましょう。

添付ファイル任意
cloud_upload

ファイルをドラッグ&ドロップ

対応形式: PDF, JPG, PNG / 最大10MB

7. 送信ボタンのラベルは具体的に

送信ボタンに「送信」のような抽象的なラベルを使ってしまうと、ユーザーは「押すと何が起きるの？」と不安を感じてしまいます。

送信ボタンのラベルには「次に起こるアクションが明確に伝わる文言」を記載しましょう。これで心理的ハードルが一気に下がり、ボタンを押しやすくなります。

悪い例
送信

何が起きるか分からない

良い例
無料で資料をダウンロード

次のアクションが明確

Section 11-4

エラー表示の設計（5ポイント）

バリデーションエラーの表示方法とタイミングに関するポイントです。

1. 一目でエラーと分かるデザインに

エラー表示のデザインが分かりにくいと、「どの項目が間違っているのか」をすぐに理解できません。

赤色を使って、エラー発生時に「誰が見てもエラーと分かる状態」にするデザインにしましょう。ただ単に赤色にするだけでなく、より視覚的に目立つようにするとさらに良いです。

メールアドレス必須
test
error メールアドレスの形式で入力してください

2. 入力エラーは項目の近くに表示

フォーム送信後に入力エラーをフォーム上部にまとめて表示している場合がありますが、これはユーザーがエラーを修正しようとする意欲を下げる原因となります。

入力エラーは必ず、該当項目のすぐ近くに表示してください。どこを直せばいいかが一瞬で理解でき、修正がスムーズになります。

悪い例：上部にまとめて

・メールアドレスが正しくありません

・電話番号を入力してください

メールアドレス
test

どこを直せばいいか分かりにくい

良い例：項目の近く
メールアドレス
test
メールアドレスの形式で入力してください

修正箇所が一目で分かる

3. エラーメッセージは具体的に

「正しく入力してください」のような曖昧なエラーメッセージだと、ユーザーは「何をどう直せばいいのか」が分からず、入力が止まってしまう可能性があります。

エラーメッセージは「何が原因で、どう直せばいいか」を明確に伝えることが大切です。

具体的なエラーメッセージの例
  • 必須項目：「○○を入力してください」
  • 形式エラー：「メールアドレスの形式で入力してください（例: name@example.com）」
  • 文字数：「8文字以上16文字以内で入力してください」
  • 数値範囲：「1〜100の範囲で入力してください」
  • 一致エラー：「パスワードが一致しません」
  • 電話番号：「ハイフンなしの数字のみで入力してください」

4. 入力途中でエラーを表示しない

リアルタイムでエラーを表示することは大切ですが、入力途中（1文字目・2文字目など）でエラーを表示してしまうと、ユーザーは強いストレスを感じます。

エラーを出すタイミングは「入力が終わってから」または「フォーカスを外してから」にしましょう。まだ入力してるだけなのにエラーが出るストレスを軽減できます。

エラー表示の適切なタイミング

  • blur時（フォーカスを外した時）
  • 一定時間経過後（入力が止まって1秒後など）
  • 送信ボタン押下時

5. 入力のヒントを事前に表示

入力欄に何を書けば良いかのヒントが事前に表示されていないと、ユーザーに考えさせてしまうフォームになり、離脱を誘発する原因になります。

入力のヒント（マイクロコピー、入力例、説明文など）を入力欄の近くに表示しましょう。説明が長すぎると逆効果なので、短く簡潔に記載するのがおすすめです。

パスワード必須
••••••••

8文字以上、英数字を含めてください

Section 11-5

AIプロンプトで実装する

フォーム生成プロンプト例

プロンプト例 
以下の仕様で問い合わせフォームを作成してください。

【基本要件】
- 入力項目は縦一列に並べる（横並び禁止）
- ラベルは入力欄の上に配置
- 必須項目には赤い「必須」バッジを表示
- グループごとに適切な余白を設ける
- モダンなデザイン、角丸8px

【入力項目】
1. お名前（必須）- テキスト入力
2. メールアドレス（必須）- type="email"
3. 電話番号（任意）- type="tel"
4. お問い合わせ種別（必須）- ラジオボタン
   選択肢: 製品について / サービスについて / その他
5. お問い合わせ内容（必須）- テキストエリア

【UX配慮】
- placeholder は入力例のみ（ラベル代わりにしない）
- フォーカス時は枠線を青色（#007ACC）で強調
- autocomplete属性を適切に設定
- リンクは target="_blank" で別タブ

【バリデーション・エラー表示】
- 必須項目の空チェック
- メールアドレスの形式チェック
- エラーは入力欄のすぐ下に赤文字で表示
- エラー時は入力欄の枠線を赤に
- 入力途中ではエラーを出さない（blur時に検証）
- エラーメッセージは具体的に（例付き）

【送信ボタン】
- 「無料で相談する」という具体的なラベル
- 目立つ青色（#007ACC）
- ホバー時は少し暗く

HTML/CSS/JavaScriptで作成してください。

まとめ

第11章のまとめ

  • フォームデザインはユーザーの負担に直結し、離脱率に大きく影響する
  • フォーム全体（8項目）：縦並び、自然な順序、ラベル上配置、グループ化、入力例のみのplaceholder、必須明示、フォーカス強調、別タブリンク
  • 入力項目別（7項目）：選択式優先、逃げ道、UI使い分け、チェックボックス縦並び、ファイル明記、具体的ボタンラベル
  • エラー処理（5項目）：視覚的明確、項目近くに表示、具体的メッセージ、入力途中は非表示、ヒント事前表示
  • プロンプトでは20のポイントを明示して生成を依頼する
tips_and_updates 実践のヒント

フォームを作成したら、実際にスマートフォンで入力してみましょう。自分が使いづらいと感じる点は、ユーザーも同じです。入力完了までストレスなく進めるかを体験してみてください。
