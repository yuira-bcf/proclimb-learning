Section 10-1

品質チェックの習慣

AIが生成したコードは、必ず人間の目で確認する必要があります。品質チェックを習慣化することで、問題を早期に発見・修正できます。

生成コードのチェックリスト

カテゴリ チェック項目
HTML セマンティックタグが適切に使われているか
見出しレベル（h1→h2→h3）が正しい順序か
alt属性、aria属性が適切か
CSS 不要なスタイルや重複がないか
!importantが乱用されていないか
メディアクエリが適切に設定されているか
レスポンシブ モバイル、タブレット、PCで崩れないか
タップターゲットが十分なサイズか
横スクロールが発生していないか
パフォーマンス 無駄に複雑なセレクタがないか
アニメーションが重くないか

Section 10-2

AIとの効果的な協業

協業のマインドセット

AIは「アシスタント」であり「完璧な答え」ではない

  • AIの出力を出発点として捉える
  • 最終的な判断は人間が行う
  • AIの間違いを学習の機会にする
  • 対話的に改善していく姿勢を持つ

効果的なワークフロー

looks_one

1. 要件を整理する

AIに依頼する前に、作りたいものを明確にする

looks_two

2. 具体的なプロンプトを書く

曖昧さを排除し、数値や仕様を明示する

looks_3

3. 生成結果を検証する

ブラウザで確認、コードを読んで理解する

looks_4

4. 修正指示を出す

問題点を具体的に伝え、改善を依頼する

looks_5

5. 最終調整は手作業で

細かい微調整は自分で行うほうが早いことも

プロンプトテンプレートの蓄積

おすすめの習慣

うまくいったプロンプトはテンプレートとして保存しておきましょう。次回以降、類似の作業で再利用できます。

  • ボタン生成プロンプト
  • カード生成プロンプト
  • フォーム生成プロンプト
  • LP各セクション生成プロンプト
  • リファクタリング依頼プロンプト
Section 10-3

よくある失敗と対策

失敗パターンと対策

失敗パターン 原因 対策
意図と違うデザイン プロンプトが曖昧 具体的な数値、参考画像、詳細な仕様を提示
レスポンシブが崩れる ブレイクポイント未指定 各デバイスでの表示を明示的に指示
アクセシビリティ不足 要件に含めていない アクセシビリティ要件をプロンプトに追加
コードが冗長 最適化を依頼していない 「重複を削減」「最小限のコードで」と指示
古い書き方が混在 技術要件が不明確 「モダンな書き方で」「Flexbox/Gridを使用」と指定
出力が途中で切れる 依頼内容が多すぎる セクションごとに分割して依頼

AIの限界を理解する

AIが苦手なこと

  • 複雑なインタラクション - 高度なJavaScript連携
  • ブランド固有のニュアンス - 企業の世界観を完璧に再現
  • 最新の仕様 - 学習データ以降の新機能
  • ピクセルパーフェクト - デザインカンプの完全再現
  • プロジェクト固有の制約 - 既存コードベースとの整合性
Section 10-4

継続的な学習

基礎力を磨く重要性

AIを使いこなすためには、基礎的なHTML/CSS/JavaScriptの知識が不可欠です。AIが出力したコードを理解し、評価し、修正できる力が求められます。

学び続けるべきこと
  • HTML/CSSの基礎と新機能
  • レスポンシブデザインの原則
  • アクセシビリティのガイドライン
  • パフォーマンス最適化の手法
  • デザインの基礎（余白、色、タイポグラフィ）
AIで効率化すること
  • 定型的なコード生成
  • コードの雛形作成
  • リファクタリングの提案
  • デバッグのヒント取得
  • 新しい書き方の学習

最後に

プロンプトUIコーディングの心得

  1. AIは道具 - 使いこなすのは人間の役割
  2. 基礎が大事 - 知識があるほどAIを活用できる
  3. 検証を怠らない - 生成コードは必ず確認
  4. 継続的に学ぶ - Web技術もAIも進化し続ける
  5. 楽しむ - 新しい開発スタイルを楽しもう
