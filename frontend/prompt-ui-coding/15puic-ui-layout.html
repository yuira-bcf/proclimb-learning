Section 15-1

プロンプトUIレイアウトとは

プロンプトUIレイアウトは、完成したWebページのCSSを「レイヤー」に分解し、段階的に積み重ねてレイアウトを構築する学習手法です。最もシンプルな「ブロックの枠線だけ」の状態から始め、サイズ、余白、配置、装飾を1つずつ追加していくことで、CSSの各プロパティがレイアウトにどう影響するかを視覚的に理解できます。

この手法のメリット

  • CSSの各プロパティの役割を視覚的に理解できる
  • レイアウト構築の「順序」を体系的に学べる
  • AIへのプロンプト指示を正確に出せるようになる
  • デバッグ時にどのレイヤーに問題があるか特定しやすい

Section 15-2

CSSレイヤーの分類

7つのCSSレイヤー

Webページを構成するCSSプロパティは、以下の7つのレイヤーに分類できます。

1. アウトライン（デバッグ用）

outline - ブロック要素の境界を可視化。実線=主要ブロック、破線=内部コンテナ、点線=テキスト要素

2. リセット / ベース

margin: 0 padding: 0 box-sizing font-family font-size line-height color background

3. ブロックサイズ

width height max-width min-height aspect-ratio padding margin gap

4. レイアウト

display flex grid position align-items justify-content grid-template-columns overflow

5. ビジュアル装飾

background border border-radius box-shadow opacity transition transform

6. テキスト / タイポグラフィ

font-size font-weight color text-decoration text-align letter-spacing white-space

7. インタラクション（JavaScript）

ドロワーメニューの開閉、バナースライダー、タブ切り替え、カウンター操作など

Section 15-4

Step 01〜08 各ステップの詳細

各ステップのプレビューを確認しながら、CSSレイヤーが追加されるたびにレイアウトがどう変化するかを学びましょう。

01
ブロック枠線のみ
HTMLの主要ブロックだけを可視化する

最初のステップでは、ページの主要ブロックだけを色付きのoutlineで表示します。HTMLもCSSも最小限で、ページ全体の「骨組み」を把握します。

add_circle このステップで追加されるもの

  • HTMLは各ブロック要素のタグのみ（テキストなし、子要素なし）
  • outline: 2px solid [色] で主要ブロックごとに異なる色の枠線
  • ブロック要素は中身がないため1行分の高さだけ表示される
/* ========== Block Outline ========== */ .header { outline: 2px solid #E74C3C; } .quick-access { outline: 2px solid #E67E22; } .main-banner { outline: 2px solid #F1C40F; } .important-info { outline: 2px solid #2ECC71; } .section { outline: 2px solid #3498DB; } .footer { outline: 2px solid #1ABC9C; } .bottom-nav { outline: 2px solid #9B59B6; }

02
全要素の枠線
HTMLの全ネスト構造を可視化する

Step 01の主要ブロック枠線に加え、内部コンテナ（破線）とテキスト要素（点線）のアウトラインを追加します。HTMLも全ての子要素を含む完全な構造になります（ただしテキストは空）。

add_circle このステップで追加されるもの

  • HTMLに全ての子要素（div, a, span, button等）を追加（テキストは空）
  • outline: 1px dashed [色] で内部コンテナを破線表示
  • outline: 1px dotted [色] でテキスト要素を点線表示
  • 3種類の線種で要素の役割を区別できる
/* Inner Container Outline (dashed) */ .header-top { outline: 1px dashed #F1948A; } .banner-slider { outline: 1px dashed #F9E79F; } .category-grid { outline: 1px dashed #85C1E9; } .footer-links { outline: 1px dashed #76D7C4; } ... /* Text Element Outline (dotted) */ h2, h4 { outline: 1px dotted #E74C3C; } p { outline: 1px dotted #E67E22; } a { outline: 1px dotted #27AE60; } span { outline: 1px dotted #8E44AD; }
visibility Step 02 プレビュー
open_in_new 別タブで開く

info 線種の使い分け

  • 実線（solid） = 主要ブロック（header, section, footer）
  • 破線（dashed） = 内部コンテナ（grid, list, nav等）
  • 点線（dotted） = テキスト要素（h2, p, a, span等）
03
リセット + サイズガード
CSSの基盤を整える

ブラウザのデフォルトスタイルをリセットし、空要素が潰れないようサイズガードを追加します。この2つの土台があることで、後のステップでサイズや配置が正確に効くようになります。

add_circle このステップで追加されるもの

  • * { margin: 0; padding: 0; box-sizing: border-box; } - 全要素リセット
  • body { font-family, background, color, font-size, line-height } - 基本設定
  • div, section等 { min-height: 1px; min-width: 1px; } - 空要素のサイズガード
  • span, a { display: inline-block; min-width: 1px; } - インライン要素のガード
/* ========== CSS Reset ========== */ * { margin: 0; padding: 0; box-sizing: border-box; } body { font-family: 'Noto Sans JP', sans-serif; background: #F5F5F5; color: #444; font-size: 14px; line-height: 1.6; } /* ========== Empty Element Size Guard ========== */ div, section, header, footer, nav { min-height: 1px; min-width: 1px; } span, a, label { display: inline-block; min-width: 1px; min-height: 1px; }
visibility Step 03 プレビュー
open_in_new 別タブで開く

lightbulb Step 02 との違い

ブラウザデフォルトのmarginが消え、背景がグレー（#F5F5F5）になります。各要素が左上に詰まり、より「制御された」状態になります。

04
幅と高さ
ブロックのサイズを決める

リセットとサイズガードの上に、各要素のwidth/height/max-widthだけを追加します。まだ配置（display:flex等）がないため、要素は縦に並んだままですが、サイズが決まり始めます。

add_circle このステップで追加されるもの

  • .container { max-width: 1200px; } - ページ全体の最大幅
  • .logo-icon { width: 32px; height: 32px; } - ロゴアイコンのサイズ
  • .drawer-menu { width: 300px; height: 100%; } - ドロワーメニューのサイズ
  • .banner-image { width: 100%; } / .campaign-card { width: 280px; }
  • .category-icon { width: 48px; height: 48px; } - 各アイコンのサイズ
  • .bottom-spacer { height: 60px; } - 固定ナビ分のスペーサー
/* ========== Block Sizes (width/height only) ========== */ .container { max-width: 1200px; } .logo-icon { width: 32px; height: 32px; } .hamburger { width: 24px; height: 24px; } .drawer-menu { width: 300px; height: 100%; } .banner-image { width: 100%; } .banner-dot { width: 8px; height: 8px; } .category-icon { width: 48px; height: 48px; } .campaign-card { width: 280px; } .campaign-image { width: 100%; height: 140px; } .footer-sns a { width: 40px; height: 40px; } .bottom-spacer { height: 60px; }
visibility Step 04 プレビュー

lightbulb ポイント

まだ display: flex や display: grid がないため、全要素が縦に積み重なっています。サイズだけでは「配置」は変わらないことを確認しましょう。

05
余白（padding / margin / gap）
要素間に「空間」を作る

各要素にpadding（内側の余白）、margin（外側の余白）、gap（Flex/Grid間の隙間）を追加します。要素間に適切な間隔が生まれ、レイアウトに「呼吸」ができます。

add_circle このステップで追加されるもの

  • .header-top { padding: 10px 16px; } - ヘッダー内部の余白
  • .drawer-header { padding: 16px; } - ドロワーヘッダーの余白
  • .drawer-user { padding: 20px 16px; } - ユーザー情報の余白
  • .drawer-nav-item { padding: 14px 16px; } - ナビアイテムの余白
  • .drawer-user-icon { margin-bottom: 12px; } - 要素間のマージン
  • .drawer-user-login { margin-top: 8px; padding: 8px 16px; }
/* ========== Spacing (padding / margin / gap) ========== */ .header-top { padding: 10px 16px; } .drawer-header { padding: 16px; } .drawer-user { padding: 20px 16px; } .drawer-user-icon { margin-bottom: 12px; } .drawer-user-login { margin-top: 8px; padding: 8px 16px; } .drawer-nav { padding: 8px 0; } .drawer-nav-item { padding: 14px 16px; } .drawer-section-title { padding: 8px 16px 4px; }
visibility Step 05 プレビュー
open_in_new 別タブで開く

lightbulb padding vs margin

  • padding = ボックスの「内側」に空間を作る（背景色の内側に入る）
  • margin = ボックスの「外側」に空間を作る（隣の要素との間隔）
  • gap = Flex/Gridコンテナの子要素間に均等な間隔を作る
06
レイアウト + 装飾（CSS完成）
Flexbox/Gridによる配置とビジュアルスタイルの適用

Flexbox/Gridによる配置、background/border/colorの装飾、レスポンシブ対応など、全てのCSSが適用されます。テキストやJavaScriptがない状態ですが、見た目のレイアウトは完成しています。

add_circle このステップで追加されるもの

  • display: flex / display: grid - 横並びレイアウト
  • position: sticky / position: fixed - ヘッダー固定、ナビ固定
  • background / border / border-radius - ビジュアル装飾
  • color / font-size / font-weight - テキストスタイル
  • @media クエリ - レスポンシブ対応
  • アウトライン（outline）は残したまま、レイアウト構造を確認可能
/* ========== Layout (display/flex/grid) ========== */ .header-top { display: flex; align-items: center; justify-content: space-between; } .quick-access { display: grid; grid-template-columns: repeat(4, 1fr); } .category-grid { display: grid; grid-template-columns: repeat(4, 1fr); } .campaign-scroll { display: flex; overflow-x: auto; } /* ========== Visual Decoration ========== */ .header { background: #fff; position: sticky; top: 0; border-bottom: 1px solid #D0D0D0; } .bottom-nav { position: fixed; bottom: 0; display: grid; grid-template-columns: repeat(5, 1fr); }
visibility Step 06 プレビュー
open_in_new 別タブで開く

lightbulb 大きな変化のポイント

このステップで劇的にレイアウトが変わります。display: flex と display: grid が適用されることで、縦一列に並んでいた要素が横並びグリッドになります。これがCSSの「配置」レイヤーの威力です。

07
JavaScript追加
インタラクティブな動きを追加する

CSSだけでは実現できないインタラクションをJavaScriptで追加します。ドロワーメニューの開閉、バナーの自動スクロールなど、ユーザー操作に反応する動きが加わります。

add_circle このステップで追加されるもの

  • ドロワーメニューの開閉（ハンバーガーメニュークリック → スライドイン）
  • バナーの自動スライド（3秒間隔で切り替え + ドットインジケーター連動）
  • オーバーレイの表示/非表示
  • ボトムナビの「その他」タップでもドロワー表示
/* ========== JavaScript で追加される機能 ========== */ // ドロワーメニューの開閉 menuBtn.addEventListener('click', () => { drawerMenu.classList.add('open'); drawerOverlay.classList.add('active'); }); // バナー自動スクロール setInterval(() => { currentSlide = (currentSlide + 1) % totalSlides; slider.scrollTo({ left: currentSlide * slider.offsetWidth, behavior: 'smooth' }); }, 3000);

Section 15-5

プロンプトでの指示方法

レイヤー別プロンプト例

AIにCSSレイアウトを段階的に構築させる場合、以下のようにレイヤーを明示してプロンプトを書きます。

Step 1: 構造の指示（HTML）

## HTML構造 以下のセクション構成でHTMLを作成してください： - header: ロゴ + 検索ボタン + ハンバーガーメニュー - quick-access: 4つのクイックリンク（grid 4列） - main-banner: 横スクロールバナー × 3枚 + ドットインジケーター - important-info: 左ボーダー付き警告ボックス - section × 4: カテゴリ / キャンペーン / ニュース / サービスリンク - footer: SNSアイコン + リンクグループ × 3 + コピーライト - bottom-nav: 5つのタブ（モバイル用固定ナビ） - drawer-menu: 右からスライドするメニュー テキストは空で、class名のみ付与してください。

Step 2: レイアウトの指示（CSS）

## CSSレイアウト指示 以下のレイアウトプロパティを適用してください： header-top: display: flex, justify-content: space-between, align-items: center quick-access: display: grid, grid-template-columns: repeat(4, 1fr) banner-slider: display: flex, overflow-x: auto, scroll-snap-type: x mandatory category-grid: display: grid, grid-template-columns: repeat(4, 1fr) campaign-scroll: display: flex, overflow-x: auto, gap: 12px footer-links: display: grid, grid-template-columns: repeat(3, 1fr) bottom-nav: position: fixed, bottom: 0, display: grid, grid-template-columns: repeat(5, 1fr) ## ワイヤーフレーム配色 全てグレースケールで、border-radius: 0 を使用

Step 3: レスポンシブの指示

## レスポンシブブレイクポイント 768px+（タブレット）: - category-grid: 8列 - service-link-grid: 4列 - footer-links: 3列 grid 1024px+（デスクトップ）: - header: position relative（sticky解除） - quick-btn: 横並び（flex-direction: row） - category-icon: 64x64px - bottom-nav: display: none（非表示） - max-width: 1200px で中央寄せ

グレースケール → カラー変換のプロンプト

ワイヤーフレームが完成したら、ブランドカラーを適用する指示を追加します。

## カラー変換指示 ワイヤーフレームのグレースケールを以下のブランドカラーに変換してください： プライマリカラー: #e60039（ブランドレッド） グラデーション: linear-gradient(135deg, #e60039, #ff4d6d) 背景色: #f5f5f5（ライトグレー） テキスト: #333（ダークグレー） border-radius: 10px（角丸に変更） box-shadow: 0 2px 8px rgba(0,0,0,0.08) を追加 カテゴリアイコンは色分け： red=#ffebef/#e60039, blue=#e6f2ff/#0066cc, green=#e6f9ef/#00a050 ...
