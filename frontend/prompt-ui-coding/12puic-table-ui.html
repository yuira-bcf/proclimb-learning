Section 12-1

テーブルUIの基本

Webアプリケーションでは、大量のデータを一覧表示するためにテーブルUIが多用されます。基本的なHTMLテーブルから、高機能なJavaScriptライブラリまで、用途に応じた選択が重要です。

テーブルUIに求められる機能

sort

ソート（並び替え）

列ヘッダーをクリックして昇順・降順で並び替え

search

検索・フィルタリング

キーワードや条件で表示データを絞り込み

view_list

ページネーション

大量データを複数ページに分割表示

edit

インライン編集

セルをクリックしてその場で値を編集

file_download

エクスポート

CSV、Excel、PDF形式でデータを出力

smartphone

レスポンシブ対応

スマートフォンでも崩れない表示

基本的なHTMLテーブル

HTML 
<table class="data-table">
  <thead>
    <tr>
      <th>ID</th>
      <th>名前</th>
      <th>メール</th>
      <th>登録日</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>山田太郎</td>
      <td>yamada@example.com</td>
      <td>2024-01-15</td>
    </tr>
    <!-- 繰り返し -->
  </tbody>
</table>
基本テーブルの限界

HTMLのみのテーブルでは、ソートや検索などの機能を自前で実装する必要があります。JavaScriptライブラリを活用することで、これらの機能を簡単に追加できます。

Section 12-2

DataTablesの活用

DT

DataTables

jQuery製の高機能テーブルプラグイン。世界で最も使われているテーブルライブラリ

ソート 検索 ページネーション Ajax対応 エクスポート レスポンシブ 豊富な拡張機能

DataTablesの導入

HTML - CDN読み込み 
<!-- CSS -->
<link rel="stylesheet" href="https://cdn.datatables.net/1.13.7/css/jquery.dataTables.min.css">

<!-- JavaScript -->
<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
<script src="https://cdn.datatables.net/1.13.7/js/jquery.dataTables.min.js"></script>
JavaScript - 基本初期化 
$(document).ready(function() {
    $('#myTable').DataTable();
});

DataTablesの主要オプション

JavaScript - カスタマイズ例 
$('#myTable').DataTable({
    // 日本語化
    language: {
        url: '//cdn.datatables.net/plug-ins/1.13.7/i18n/ja.json'
    },
    // ページあたりの表示件数
    pageLength: 25,
    // 表示件数の選択肢
    lengthMenu: [10, 25, 50, 100],
    // 初期ソート（2列目を降順）
    order: [[1, 'desc']],
    // 列ごとの設定
    columnDefs: [
        { orderable: false, targets: [0, -1] }, // 最初と最後の列はソート無効
        { searchable: false, targets: [0] }     // 最初の列は検索対象外
    ],
    // レスポンシブ対応
    responsive: true,
    // 処理中表示
    processing: true
});

Ajax対応（サーバーサイド処理）

JavaScript - Ajaxデータ取得 
$('#myTable').DataTable({
    ajax: {
        url: '/api/users',
        dataSrc: 'data'
    },
    columns: [
        { data: 'id' },
        { data: 'name' },
        { data: 'email' },
        { data: 'created_at' }
    ],
    // サーバーサイド処理（大量データ向け）
    serverSide: true,
    processing: true
});

Section 12-3

jsGridの活用

jG

jsGrid

軽量でシンプルなグリッドコンポーネント。CRUD操作に特化した設計

ソート フィルタリング ページネーション インライン編集 行の追加・削除 バリデーション 軽量

jsGridの導入

HTML - CDN読み込み 
<!-- CSS -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/jsgrid/1.5.3/jsgrid.min.css">
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/jsgrid/1.5.3/jsgrid-theme.min.css">

<!-- JavaScript -->
<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/jsgrid/1.5.3/jsgrid.min.js"></script>

jsGridの基本設定

JavaScript - 基本初期化 
$("#jsGrid").jsGrid({
    width: "100%",
    height: "400px",

    // 機能の有効化
    inserting: true,   // 新規追加
    editing: true,     // 編集
    sorting: true,     // ソート
    paging: true,      // ページネーション
    filtering: true,   // フィルタリング

    pageSize: 10,
    pageButtonCount: 5,

    // データ
    data: clients,

    // 列定義
    fields: [
        { name: "id", title: "ID", type: "number", width: 50 },
        { name: "name", title: "名前", type: "text", width: 150 },
        { name: "email", title: "メール", type: "text", width: 200 },
        { name: "status", title: "ステータス", type: "select",
          items: [
            { Name: "有効", Id: 1 },
            { Name: "無効", Id: 0 }
          ],
          valueField: "Id", textField: "Name"
        },
        { type: "control" }  // 編集・削除ボタン
    ]
});

CRUD操作の実装

JavaScript - API連携 
$("#jsGrid").jsGrid({
    // ... 基本設定 ...

    controller: {
        // データ読み込み
        loadData: function(filter) {
            return $.ajax({
                type: "GET",
                url: "/api/clients",
                data: filter
            });
        },
        // 新規追加
        insertItem: function(item) {
            return $.ajax({
                type: "POST",
                url: "/api/clients",
                data: item
            });
        },
        // 更新
        updateItem: function(item) {
            return $.ajax({
                type: "PUT",
                url: "/api/clients/" + item.id,
                data: item
            });
        },
        // 削除
        deleteItem: function(item) {
            return $.ajax({
                type: "DELETE",
                url: "/api/clients/" + item.id
            });
        }
    }
});

バリデーション

JavaScript - 入力検証 
fields: [
    {
        name: "name",
        title: "名前",
        type: "text",
        validate: {
            validator: "required",
            message: "名前は必須です"
        }
    },
    {
        name: "email",
        title: "メール",
        type: "text",
        validate: {
            validator: function(value) {
                return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
            },
            message: "有効なメールアドレスを入力してください"
        }
    }
]
Section 12-4

ライブラリの選び方

DataTables vs jsGrid 比較

観点 DataTables jsGrid
特徴 高機能・拡張性が高い 軽量・シンプル
主な用途 データ閲覧・レポート表示 データ入力・CRUD操作
学習コスト やや高い 低い
ファイルサイズ 約100KB 約40KB
インライン編集 拡張機能で対応 標準で対応
エクスポート 豊富（CSV, Excel, PDF） なし（自前実装）
ドキュメント 非常に充実 基本的
コミュニティ 大規模 中規模

ユースケース別の選び方

analytics レポート・分析画面

DataTablesがおすすめ。ソート、検索、エクスポート機能が充実。大量データの表示に強い。

edit_note データ入力・管理画面

jsGridがおすすめ。インライン編集、CRUD操作が標準で対応。入力バリデーションも簡単。

speed 軽量さ重視

jsGridがおすすめ。ファイルサイズが小さく、初期表示が速い。シンプルな要件に最適。

extension 拡張性重視

DataTablesがおすすめ。豊富なプラグイン、カスタマイズオプションで複雑な要件にも対応。

その他の選択肢

モダンなテーブルライブラリ

  • AG Grid - 企業向け高機能グリッド（有料版あり）
  • Tabulator - jQuery不要、モダンな設計
  • Handsontable - Excel風のスプレッドシートUI
  • React Table - React用のヘッドレスライブラリ
  • TanStack Table - フレームワーク非依存

DataTablesのプロンプト例

プロンプト例 
以下の仕様でDataTablesを使った顧客一覧テーブルを作成してください。

【基本要件】
- jQuery + DataTables（CDN）
- 日本語化
- レスポンシブ対応

【テーブル列】
1. ID（数値、ソート可）
2. 顧客名（テキスト、検索対象）
3. メールアドレス（テキスト、検索対象）
4. 電話番号（テキスト）
5. 登録日（日付、ソート可）
6. ステータス（バッジ表示：有効=緑、無効=赤）
7. 操作（編集・削除ボタン、ソート無効）

【機能】
- ページネーション（10, 25, 50件）
- 検索ボックス
- 列ソート
- 初期表示は登録日の降順

【スタイル】
- モダンなデザイン
- ホバー時に行をハイライト
- ステータスはバッジスタイル

HTML/CSS/JavaScriptで作成してください。

jsGridのプロンプト例

プロンプト例 
以下の仕様でjsGridを使った商品管理テーブルを作成してください。

【基本要件】
- jQuery + jsGrid（CDN）
- CRUD操作対応（追加・編集・削除）

【テーブル列】
1. ID（数値、自動採番、編集不可）
2. 商品名（テキスト、必須）
3. カテゴリ（選択式：食品/飲料/日用品/その他）
4. 価格（数値、必須、0以上）
5. 在庫数（数値、0以上）
6. 操作（編集・削除ボタン）

【機能】
- インライン編集
- 新規追加行
- フィルタリング（各列）
- ページネーション（15件/ページ）
- バリデーション（必須チェック、数値チェック）

【API連携】
- loadData: GET /api/products
- insertItem: POST /api/products
- updateItem: PUT /api/products/:id
- deleteItem: DELETE /api/products/:id

HTML/CSS/JavaScriptで作成してください。

まとめ

第12章のまとめ

  • テーブルUIにはソート、検索、ページネーション等の機能が求められる
  • DataTablesは高機能で拡張性が高く、レポート・分析画面に最適
  • jsGridは軽量でCRUD操作に強く、データ入力画面に最適
  • 用途に応じて適切なライブラリを選択することが重要
  • プロンプトでは列定義・機能・API連携を明確に指定する
tips_and_updates 実践のヒント

まずはサンプルデータで動作確認し、その後API連携を実装しましょう。大量データの場合はサーバーサイド処理（serverSide: true）を検討してください。
