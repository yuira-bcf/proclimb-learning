Section 4-1

ボタンの基本

btnクラスの仕組み

Bootstrapでボタンを作るには、btn クラスと色を指定するクラスの
2つを組み合わせます。

基本の書き方は class="btn btn-{色の名前}" です。

HTML（基本ボタン） 
<button class="btn btn-primary">Primary</button>
<button class="btn btn-secondary">Secondary</button>
<button class="btn btn-success">Success</button>
<button class="btn btn-danger">Danger</button>
<button class="btn btn-warning">Warning</button>
<button class="btn btn-info">Info</button>
<button class="btn btn-light">Light</button>
<button class="btn btn-dark">Dark</button>
クラス 背景色 テキスト色 用途
btn-primary 青 白 主要アクション（送信、保存など）
btn-secondary グレー 白 補助的アクション（キャンセルなど）
btn-success 緑 白 成功・完了系（購入、登録など）
btn-danger 赤 白 破壊的操作（削除、退会など）
btn-warning 黄色 黒 注意が必要な操作
btn-info 水色 黒 情報表示系（詳細を見るなど）
btn-light 白に近い 黒 控えめなボタン
btn-dark 黒に近い 白 フッターやダーク系UIのボタン

POINT

btn クラスは必ず必要です。これがないとBootstrapのボタンスタイルが適用されません。
btn だけでは色が付かないので、必ず btn-primary などの色クラスとセットで使います。
btn クラスが設定するのは、padding、font-size、border-radius、cursor: pointer などの基本スタイルです。

ボタンに使えるHTML要素

Bootstrapのボタンクラスは、<button> だけでなく、
<a> や <input> にも適用できます。

HTML（さまざまな要素にボタンクラス） 
<!-- button要素（フォーム送信など） -->
<button type="button" class="btn btn-primary">
  ボタン要素
</button>

<!-- a要素（リンクとしてのボタン） -->
<a href="#" class="btn btn-success" role="button">
  リンクボタン
</a>

<!-- input要素（フォーム送信ボタン） -->
<input type="submit" class="btn btn-danger" value="送信ボタン">

<!-- input要素（リセットボタン） -->
<input type="reset" class="btn btn-warning" value="リセット">
要素 使うべき場面 注意点
<button> JavaScriptでクリックイベントを処理する場合 type="button"を明記するのが推奨
<a> 別ページへのリンクをボタンの見た目にしたい場合 role="button"を付けるとアクセシビリティが向上
<input> フォームの送信・リセットボタン value属性でボタンテキストを指定

ボタンの無効化（disabled）

ボタンをクリックできない状態（無効化）にすることもできます。

HTML（無効化ボタン） 
<!-- button要素の無効化 -->
<button class="btn btn-primary" disabled>
  無効なボタン
</button>

<!-- a要素の無効化（disabledクラスを使う） -->
<a href="#" class="btn btn-primary disabled" aria-disabled="true">
  無効なリンクボタン
</a>

POINT

<button> や <input> は disabled 属性で無効化できますが、
<a> タグは HTML仕様上 disabled 属性に対応していません。
そのため <a> の場合は class="disabled" と aria-disabled="true" を使います。

実践例：ECサイトのアクションボタン

HTML（ECサイトのボタン例） 
<div class="container mt-4">
  <h3>商品詳細ページ</h3>
  <p>手作りキャンドル - ¥2,980</p>

  <!-- メインアクション：カートに追加 -->
  <button class="btn btn-primary">カートに追加</button>

  <!-- サブアクション：お気に入り -->
  <button class="btn btn-secondary">お気に入りに追加</button>

  <!-- 購入アクション -->
  <button class="btn btn-success">今すぐ購入</button>

  <!-- 在庫切れの場合 -->
  <button class="btn btn-danger" disabled>在庫切れ</button>
</div>
Section 4-2

アウトラインボタンとサイズ

アウトラインボタン（btn-outline-*）

アウトラインボタンは、背景色がなく、枠線だけのボタンです。
通常のボタンよりも控えめな印象を与えるため、
補助的なアクションやセカンダリボタンとしてよく使われます。

HTML（アウトラインボタン） 
<button class="btn btn-outline-primary">Primary</button>
<button class="btn btn-outline-secondary">Secondary</button>
<button class="btn btn-outline-success">Success</button>
<button class="btn btn-outline-danger">Danger</button>
<button class="btn btn-outline-warning">Warning</button>
<button class="btn btn-outline-info">Info</button>
<button class="btn btn-outline-light">Light</button>
<button class="btn btn-outline-dark">Dark</button>
通常ボタン アウトラインボタン ホバー時の挙動
btn-primary btn-outline-primary 背景が青色に変わり、テキストが白になる
btn-success btn-outline-success 背景が緑色に変わり、テキストが白になる
btn-danger btn-outline-danger 背景が赤色に変わり、テキストが白になる

アウトラインボタンにマウスカーソルを乗せる（ホバーする）と、
背景色が付いて通常のボタンと同じ見た目に変わります。
このホバーエフェクトはBootstrapが自動的に付けてくれます。

実践例：通常ボタンとアウトラインの使い分け

HTML（ボタンの使い分け） 
<!-- メインアクションは通常ボタン、サブアクションはアウトライン -->
<div class="container mt-4">
  <h4>記事の投稿</h4>
  <button class="btn btn-primary">公開する</button>
  <button class="btn btn-outline-secondary">下書き保存</button>
  <button class="btn btn-outline-danger">削除する</button>
</div>

POINT

ボタンの視覚的な優先度を使い分けましょう。
通常ボタン：最も重要なアクション（1画面に1～2個が理想）
アウトラインボタン：補助的なアクション
このルールに従うことで、ユーザーが「何を押せばいいか」が直感的に分かります。

ボタンのサイズ

Bootstrapには3つのボタンサイズが用意されています。

HTML（ボタンサイズ） 
<!-- 大サイズ -->
<button class="btn btn-primary btn-lg">大きいボタン</button>

<!-- 通常サイズ（デフォルト） -->
<button class="btn btn-primary">通常ボタン</button>

<!-- 小サイズ -->
<button class="btn btn-primary btn-sm">小さいボタン</button>
クラス padding font-size 用途
btn-lg 0.5rem 1rem 1.25rem CTAボタン、ヒーローセクション
（なし：デフォルト） 0.375rem 0.75rem 1rem 一般的なボタン
btn-sm 0.25rem 0.5rem 0.875rem テーブル内のボタン、タグ

全幅ボタン（d-grid）

ボタンを親要素の横幅いっぱいに広げたい場合は、
d-grid クラスを親要素に付けます。

HTML（全幅ボタン） 
<!-- 全幅ボタン -->
<div class="d-grid">
  <button class="btn btn-primary btn-lg">
    全幅ボタン
  </button>
</div>

<!-- 間隔付き全幅ボタン -->
<div class="d-grid gap-2">
  <button class="btn btn-primary">メインボタン</button>
  <button class="btn btn-outline-secondary">サブボタン</button>
</div>

<!-- md以上では横並び、スマホでは全幅 -->
<div class="d-grid gap-2 d-md-flex">
  <button class="btn btn-primary">ボタン1</button>
  <button class="btn btn-secondary">ボタン2</button>
</div>

POINT

全幅ボタンはモバイル画面でよく使われます。
スマホではボタンが小さいとタップしにくいため、
d-grid で横幅いっぱいにすると押しやすくなります。
PCでは d-md-flex を併用して通常サイズに戻すのが一般的です。

Section 4-3

ボタングループとツールバー

ボタングループ（btn-group）

複数のボタンをひとまとまりにして表示するのがボタングループです。
btn-group クラスで囲むと、ボタン同士がくっついて表示されます。

HTML（ボタングループ基本） 
<div class="btn-group" role="group">
  <button class="btn btn-primary">左</button>
  <button class="btn btn-primary">中</button>
  <button class="btn btn-primary">右</button>
</div>

ボタングループ内のボタンは角丸が調整され、
左端のボタンは左側だけ、右端のボタンは右側だけ角丸になります。
中間のボタンは角丸なしで表示されます。

アウトラインボタンのグループ

HTML（アウトラインのボタングループ） 
<!-- アウトラインボタンのグループ -->
<div class="btn-group" role="group">
  <button class="btn btn-outline-primary">日</button>
  <button class="btn btn-outline-primary">月</button>
  <button class="btn btn-outline-primary">年</button>
</div>

実践例：表示切り替えボタン

HTML（表示切り替え） 
<!-- リスト/グリッド表示切り替え -->
<div class="btn-group" role="group">
  <button class="btn btn-outline-secondary">リスト表示</button>
  <button class="btn btn-secondary">グリッド表示</button>
</div>

<!-- カテゴリフィルター -->
<div class="btn-group" role="group">
  <button class="btn btn-primary">すべて</button>
  <button class="btn btn-outline-primary">デザイン</button>
  <button class="btn btn-outline-primary">開発</button>
  <button class="btn btn-outline-primary">マーケティング</button>
</div>

現在選択されている項目を通常ボタン、それ以外をアウトラインボタンにすると、
どの項目が選択中なのかが視覚的に分かりやすくなります。

ボタンツールバー（btn-toolbar）

複数のボタングループをさらにまとめたものがボタンツールバーです。

HTML（ボタンツールバー） 
<div class="btn-toolbar" role="toolbar">
  <!-- テキスト編集グループ -->
  <div class="btn-group me-2" role="group">
    <button class="btn btn-outline-secondary">B</button>
    <button class="btn btn-outline-secondary">I</button>
    <button class="btn btn-outline-secondary">U</button>
  </div>

  <!-- 配置グループ -->
  <div class="btn-group me-2" role="group">
    <button class="btn btn-outline-secondary">左</button>
    <button class="btn btn-outline-secondary">中</button>
    <button class="btn btn-outline-secondary">右</button>
  </div>

  <!-- 操作グループ -->
  <div class="btn-group" role="group">
    <button class="btn btn-outline-danger">削除</button>
  </div>
</div>

me-2 は margin-end（右方向の余白）を追加するユーティリティクラスです。
ボタングループ間に適度なスペースを入れて、グループの区切りを明確にします。

POINT

ボタングループやツールバーには role="group" や role="toolbar" を付けましょう。
これはアクセシビリティ（スクリーンリーダーへの対応）のための属性です。
見た目には影響しませんが、すべてのユーザーが使いやすいサイトを作るための重要な設定です。

実践例：ページネーション風ボタングループ

HTML（ページネーション風） 
<div class="btn-group" role="group">
  <button class="btn btn-outline-primary">&laquo; 前へ</button>
  <button class="btn btn-outline-primary">1</button>
  <button class="btn btn-primary">2</button>
  <button class="btn btn-outline-primary">3</button>
  <button class="btn btn-outline-primary">4</button>
  <button class="btn btn-outline-primary">5</button>
  <button class="btn btn-outline-primary">次へ &raquo;</button>
</div>

現在のページ（2ページ目）だけ btn-primary（塗りつぶし）にし、
他のページは btn-outline-primary（アウトライン）にすることで、
現在位置を視覚的に強調しています。

なお、Bootstrapには専用のpaginationコンポーネントも用意されています。
実務では pagination コンポーネントを使う方が一般的ですが、
ボタングループの応用例としてこのパターンも覚えておきましょう。

Section 4-4

バッジ

バッジとは

バッジ（Badge）は、数字やラベルを小さく表示するためのコンポーネントです。
通知の件数、ステータスの表示、カテゴリラベルなどに使われます。

基本の書き方は class="badge bg-{色の名前}" です。

HTML（バッジの基本） 
<span class="badge bg-primary">Primary</span>
<span class="badge bg-secondary">Secondary</span>
<span class="badge bg-success">Success</span>
<span class="badge bg-danger">Danger</span>
<span class="badge bg-warning text-dark">Warning</span>
<span class="badge bg-info text-dark">Info</span>
<span class="badge bg-light text-dark">Light</span>
<span class="badge bg-dark">Dark</span>

POINT

バッジはボタンと異なり、badge + bg-*（背景色クラス）の組み合わせで使います。
ボタンでは btn-primary でしたが、バッジでは bg-primary です。
warning と info の背景色は明るいので、text-dark を追加して文字を読みやすくしましょう。

見出しの中のバッジ

バッジは見出しタグの中に入れることで、タイトルに補足情報を付けられます。

HTML（見出し内バッジ） 
<h1>お知らせ <span class="badge bg-danger">New</span></h1>
<h2>記事タイトル <span class="badge bg-primary">おすすめ</span></h2>
<h3>商品名 <span class="badge bg-success">在庫あり</span></h3>
<h4>イベント名 <span class="badge bg-warning text-dark">残りわずか</span></h4>

バッジのフォントサイズは親要素の75%で表示されるため、
見出しの中に入れると見出しに合わせた大きさで表示されます。

丸型バッジ（rounded-pill）

rounded-pill クラスを追加すると、バッジの角がより丸くなり、
カプセルのような形状になります。

HTML（丸型バッジ） 
<!-- 通常の角丸バッジ -->
<span class="badge bg-primary">通常</span>

<!-- pill（丸型）バッジ -->
<span class="badge rounded-pill bg-primary">丸型</span>
<span class="badge rounded-pill bg-danger">99+</span>
<span class="badge rounded-pill bg-success">完了</span>
バッジの種類 見た目 使いどころ
通常バッジ（badge） 角がわずかに丸い四角形 カテゴリラベル、ステータス表示
丸型バッジ（badge rounded-pill） 両端が完全に丸いカプセル形 通知カウンター、タグ

ボタンの中にバッジを入れる

バッジをボタンの中に入れることで、
通知件数付きのボタンを簡単に作れます。

HTML（ボタン内バッジ） 
<!-- メール通知ボタン -->
<button class="btn btn-primary">
  メール
  <span class="badge bg-danger">4</span>
</button>

<!-- カート内アイテム数 -->
<button class="btn btn-success">
  カート
  <span class="badge bg-dark">3</span>
</button>

<!-- 通知ボタン -->
<button class="btn btn-outline-primary">
  通知
  <span class="badge rounded-pill bg-danger">99+</span>
</button>

POINT

ボタン内バッジは、ユーザーの注意を引きたいときに効果的です。
ボタンの背景色と異なる色のバッジを使うことで、
バッジが目立ちやすくなります。
例えば青いボタン（btn-primary）に赤いバッジ（bg-danger）が定番の組み合わせです。

実践例：サイドバーメニューのバッジ

HTML（サイドバーメニュー） 
<ul class="list-group">
  <li class="list-group-item d-flex justify-content-between align-items-center">
    受信トレイ
    <span class="badge bg-primary rounded-pill">14</span>
  </li>
  <li class="list-group-item d-flex justify-content-between align-items-center">
    下書き
    <span class="badge bg-secondary rounded-pill">3</span>
  </li>
  <li class="list-group-item d-flex justify-content-between align-items-center">
    スパム
    <span class="badge bg-danger rounded-pill">99+</span>
  </li>
  <li class="list-group-item d-flex justify-content-between align-items-center">
    ゴミ箱
    <span class="badge bg-dark rounded-pill">5</span>
  </li>
</ul>

上記はメールアプリのサイドバーのような表示です。
list-group コンポーネントと badge を組み合わせることで、
各項目の右端にカウンターが表示されます。

d-flex justify-content-between align-items-center は
Flexboxのユーティリティクラスで、テキストとバッジを
左端と右端に配置し、垂直方向の中央揃えにしています。

実践例：ステータスラベル

HTML（ステータスラベル） 
<!-- タスク管理のステータス表示 -->
<p>タスクA <span class="badge bg-success">完了</span></p>
<p>タスクB <span class="badge bg-warning text-dark">進行中</span></p>
<p>タスクC <span class="badge bg-secondary">未着手</span></p>
<p>タスクD <span class="badge bg-danger">期限超過</span></p>

<!-- ECサイトの商品ステータス -->
<p>商品X <span class="badge bg-success">販売中</span></p>
<p>商品Y <span class="badge bg-warning text-dark">残りわずか</span></p>
<p>商品Z <span class="badge bg-danger">売り切れ</span></p>
ステータス 推奨カラー 理由
完了・成功・販売中 bg-success（緑） 緑色は「OK」「正常」のイメージ
進行中・残りわずか bg-warning（黄） 黄色は「注意」「進行中」のイメージ
未着手・保留 bg-secondary（グレー） グレーは「まだ」「保留」のイメージ
エラー・期限超過・売り切れ bg-danger（赤） 赤色は「警告」「問題」のイメージ
