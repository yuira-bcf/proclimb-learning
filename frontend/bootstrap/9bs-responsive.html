Section 9-1

レスポンシブデザインとは

レスポンシブデザインの概念

レスポンシブデザインとは、1つのHTMLファイルで、スマートフォン・タブレット・PCなど、あらゆる画面サイズに対して最適な表示を実現するデザイン手法です。

従来はPC用サイトとモバイル用サイトを別々に作成していましたが、現在ではレスポンシブデザインが主流となっています。Googleもモバイルフレンドリーなサイトを検索結果で優遇しているため、レスポンシブ対応は必須のスキルです。

なぜレスポンシブデザインが必要なのか

2024年現在、Webサイトへのアクセスの約60%がスマートフォンからです。PC用のデザインだけでは、多くのユーザーにとって使いにくいサイトになってしまいます。1つのHTMLで全デバイスに対応できるレスポンシブデザインは、開発・保守コストの面でも大きなメリットがあります。

モバイルファーストという考え方

Bootstrapはモバイルファーストというデザイン思想を採用しています。これは「まず小さい画面（スマホ）のデザインを基準に作り、画面が大きくなるにつれてレイアウトを拡張する」という考え方です。

アプローチ 考え方 CSSの書き方
デスクトップファースト PC用デザインを基準に、小さい画面用に縮小する max-width を使うメディアクエリ
モバイルファースト スマホ用デザインを基準に、大きい画面用に拡張する min-width を使うメディアクエリ

Bootstrapではすべてのユーティリティクラスがmin-widthベースで設計されています。つまり、クラス名に何もブレイクポイントをつけなければ「全画面サイズに適用」され、-mdなどをつけると「その画面サイズ以上に適用」されます。

CSS（Bootstrapの内部実装イメージ） 
/* モバイルファースト：まず全画面サイズに適用 */
.col-12 {
  width: 100%;
}

/* md（768px）以上の画面に適用 */
@media (min-width: 768px) {
  .col-md-6 {
    width: 50%;
  }
}

/* lg（992px）以上の画面に適用 */
@media (min-width: 992px) {
  .col-lg-4 {
    width: 33.333%;
  }
}

viewport メタタグの重要性

レスポンシブデザインを正しく機能させるには、HTMLの<head>内にviewport メタタグが必須です。このタグがないと、スマートフォンのブラウザがPC用の表示をそのまま縮小表示してしまいます。

HTML 
<meta name="viewport" content="width=device-width, initial-scale=1.0">
属性 意味
width=device-width 表示幅をデバイスの画面幅に合わせる
initial-scale=1.0 初期のズーム倍率を100%にする

注意

Bootstrapのスターターテンプレートには最初からこのメタタグが含まれていますが、手動でHTMLを書く場合は忘れずに追加しましょう。これがないとレスポンシブデザインが機能しません。

Section 9-2

ブレイクポイント一覧と設計思想

Bootstrapの6つのブレイクポイント

Bootstrapには画面サイズに応じた6つのブレイクポイントが用意されています。ブレイクポイントとは「レイアウトが切り替わる画面幅の境界値」のことです。

ブレイクポイント クラス接頭辞 画面幅 対象デバイス
Extra small （なし） <576px スマートフォン（縦向き）
Small sm ≥576px スマートフォン（横向き）
Medium md ≥768px タブレット
Large lg ≥992px ノートPC
Extra large xl ≥1200px デスクトップPC
Extra extra large xxl ≥1400px 大型モニター

ブレイクポイントの読み方

例えば col-md-6 は「md（768px）以上の画面幅で、6カラム幅になる」という意味です。md未満（スマホサイズ）ではこのクラスは適用されず、col-12（全幅）がデフォルトで適用されます。

モバイルファーストの仕組み

Bootstrapのブレイクポイントはすべてmin-width（最小幅）で定義されています。これがモバイルファーストの根幹です。

クラスの適用ルールは以下の通りです。

クラス 適用される画面幅 説明
col-12 0px 以上（全サイズ） 常に12カラム幅（全幅）
col-sm-6 576px 以上 スマホ横向き以上で6カラム幅
col-md-4 768px 以上 タブレット以上で4カラム幅
col-lg-3 992px 以上 PC以上で3カラム幅

複数のブレイクポイントを組み合わせることで、画面サイズごとに異なるレイアウトを実現できます。

HTML 
<div class="container">
  <div class="row">
    <!-- スマホ: 全幅 / タブレット: 半分 / PC: 3分の1 -->
    <div class="col-12 col-md-6 col-lg-4">
      コンテンツA
    </div>
    <div class="col-12 col-md-6 col-lg-4">
      コンテンツB
    </div>
    <div class="col-12 col-md-12 col-lg-4">
      コンテンツC
    </div>
  </div>
</div>

上記の例では以下のように表示が切り替わります。

  • スマホ（768px未満）：3つのコンテンツがすべて縦1列に並ぶ（各 col-12 = 全幅）
  • タブレット（768px〜991px）：A と B が横2列、C は次の行に全幅で表示
  • PC（992px以上）：3つが横3列に均等に並ぶ（各 col-lg-4）

実践でよく使うブレイクポイントの組み合わせ

実務でよく使われるパターンをいくつか紹介します。

パターン クラスの組み合わせ 表示結果
2カラム切替 col-12 col-md-6 スマホ:1列 → タブレット以上:2列
3カラム切替 col-12 col-md-6 col-lg-4 スマホ:1列 → タブレット:2列 → PC:3列
4カラム切替 col-6 col-md-4 col-lg-3 スマホ:2列 → タブレット:3列 → PC:4列
サイドバー col-12 col-lg-8 / col-12 col-lg-4 スマホ:縦積み → PC:メイン+サイドバー
Section 9-3

レスポンシブユーティリティの活用

表示・非表示の切り替え（Display ユーティリティ）

画面サイズによって要素の表示・非表示を切り替えることは、レスポンシブデザインで非常に多い操作です。Bootstrapの d-* ユーティリティを使えば、簡単に実現できます。

クラス 効果
d-none すべての画面で非表示
d-block すべての画面で表示
d-none d-md-block スマホで非表示、タブレット以上で表示
d-block d-md-none スマホで表示、タブレット以上で非表示
d-none d-lg-block タブレット以下で非表示、PC以上で表示
HTML 
<!-- スマホでは短いテキスト、PCではフルテキストを表示 -->
<p class="d-block d-md-none">お問い合わせ</p>
<p class="d-none d-md-block">お気軽にお問い合わせください。24時間受付中です。</p>

<!-- PCのみにサイドバーを表示 -->
<div class="d-none d-lg-block">
  <h3>サイドバー</h3>
  <p>関連リンクやカテゴリ一覧など</p>
</div>

d-none と d-md-block の組み合わせ

「d-none」で一旦すべての画面で非表示にし、「d-md-block」でmd以上で表示に切り替えるという考え方です。モバイルファーストの原則に基づいて、まず非表示にしてから必要な画面サイズで表示するパターンが基本です。

レスポンシブなテキストサイズ

Bootstrapの fs-*（font-size）クラスを使うと統一的なテキストサイズが適用できます。さらに、画面サイズによってテキストサイズを変えたい場合は、Displayユーティリティと組み合わせて実現します。

HTML 
<!-- レスポンシブな見出し表示 -->
<h1 class="display-4">大きな見出し</h1>

<!-- Bootstrapのdisplayクラスはレスポンシブに縮小される -->
<h1 class="display-1">レスポンシブ見出し</h1>
<p class="lead">リード文はleadクラスで少し大きめに表示</p>
<p class="fs-6">通常サイズのテキスト</p>
クラス フォントサイズ 用途
display-1 最大（レスポンシブ対応） ヒーローセクションの見出し
display-4 やや大きめ（レスポンシブ対応） セクション見出し
lead 1.25rem リード文・導入文
fs-1 〜 fs-6 h1 〜 h6 相当 一般的なテキスト

レスポンシブなスペーシング

余白（margin / padding）も画面サイズに応じて変えることができます。ブレイクポイント付きのp-* や m-*クラスを使います。

HTML 
<!-- スマホ: padding-2 / タブレット以上: padding-4 / PC以上: padding-5 -->
<div class="p-2 p-md-4 p-lg-5 bg-light">
  <h2>セクションタイトル</h2>
  <p>画面サイズに応じて余白が変わります。</p>
</div>

<!-- スマホ: margin-bottom-3 / PC以上: margin-bottom-5 -->
<div class="mb-3 mb-lg-5">
  <p>コンテンツ間の余白もレスポンシブに調整</p>
</div>
クラス例 意味
p-2 p-md-4 スマホ: padding 0.5rem / md以上: padding 1.5rem
mb-3 mb-lg-5 スマホ: margin-bottom 1rem / lg以上: 3rem
mx-2 mx-xl-auto スマホ: 左右margin 0.5rem / xl以上: 自動中央寄せ

レスポンシブなFlexbox

Flexbox関連のユーティリティもブレイクポイント付きで使えます。「スマホでは縦並び、PCでは横並び」というよくあるパターンが簡単に実現できます。

HTML 
<!-- スマホ: 縦並び / md以上: 横並び -->
<div class="d-flex flex-column flex-md-row gap-3">
  <div class="p-3 bg-light flex-fill">カラムA</div>
  <div class="p-3 bg-light flex-fill">カラムB</div>
  <div class="p-3 bg-light flex-fill">カラムC</div>
</div>

<!-- スマホ: 左寄せ / md以上: 中央寄せ / lg以上: 右寄せ -->
<div class="d-flex justify-content-start justify-content-md-center justify-content-lg-end">
  <button class="btn btn-primary">ボタン</button>
</div>

<!-- スマホ: 折り返しあり / lg以上: 折り返しなし -->
<div class="d-flex flex-wrap flex-lg-nowrap gap-2">
  <div class="p-3 bg-light">アイテム1</div>
  <div class="p-3 bg-light">アイテム2</div>
  <div class="p-3 bg-light">アイテム3</div>
  <div class="p-3 bg-light">アイテム4</div>
</div>

レスポンシブな画像（img-fluid）

画像をレスポンシブ対応にするには img-fluid クラスを使います。このクラスは画像に max-width: 100% と height: auto を適用し、親要素の幅に合わせて自動的に縮小されるようにします。

HTML 
<!-- 基本的なレスポンシブ画像 -->
<img src="hero.jpg" class="img-fluid" alt="ヒーロー画像">

<!-- 角丸のレスポンシブ画像 -->
<img src="photo.jpg" class="img-fluid rounded" alt="写真">

<!-- サムネイル風の画像 -->
<img src="thumb.jpg" class="img-thumbnail" alt="サムネイル">
クラス 効果
img-fluid max-width: 100% で親要素に合わせて縮小
img-thumbnail ボーダー付きのサムネイルスタイル
rounded 角丸を追加
rounded-circle 丸い画像にする

img-fluid の注意点

img-fluid は画像を「縮小」はしますが「拡大」はしません。元の画像サイズ以上には大きくならないため、大きな表示エリアで使う場合は十分なサイズの画像を用意しましょう。また、Webパフォーマンスのためにsrcset属性での画像の出し分けも検討してください。

Chrome DevToolsでのレスポンシブ確認方法

作成したページのレスポンシブ表示を確認するには、Chrome DevToolsのデバイスモードが便利です。

デバイスモードの開き方

  1. Chromeでページを開く
  2. F12キー（またはCtrl + Shift + I / Cmd + Option + I）でDevToolsを開く
  3. DevTools左上のデバイスアイコン（スマホとタブレットのアイコン）をクリック
  4. 上部のドロップダウンからデバイスを選択、またはサイズを手動入力

確認すべきポイント

以下の画面幅でレイアウトが正しく切り替わるか確認しましょう。

  • 375px（iPhone SE）- 1カラム表示になるか
  • 768px（iPad）- 2カラム表示に切り替わるか
  • 1024px（ノートPC）- 3カラム表示になるか
  • 各ブレイクポイントの境界（576px, 768px, 992px, 1200px）でレイアウト崩れがないか
