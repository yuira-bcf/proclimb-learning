Section 11-1

モーダルダイアログ

モーダルとは

モーダルダイアログ（Modal Dialog）とは、画面の手前にポップアップ表示されるウィンドウのことです。背景が暗くなり、ユーザーの注意をモーダル内のコンテンツに集中させます。

Bootstrapのモーダルは、JavaScriptを自分で書かなくても data-bs-* 属性だけで動作するのが大きな特徴です。確認ダイアログ、フォーム入力、詳細情報の表示など幅広い用途で使えます。

モーダルの基本構造

モーダルは3層のネスト構造になっています。

クラス 役割
modal モーダル全体のラッパー（背景の暗幕を含む）
modal-dialog モーダルの位置・サイズを制御する枠
modal-content モーダルのコンテンツ本体（白い箱）
modal-header タイトルと閉じるボタンのエリア
modal-body モーダルの本文エリア
modal-footer アクションボタンのエリア
HTML 
<!-- モーダルを開くボタン -->
<button type="button" class="btn btn-primary"
        data-bs-toggle="modal" data-bs-target="#exampleModal">
  モーダルを開く
</button>

<!-- モーダル本体 -->
<div class="modal fade" id="exampleModal" tabindex="-1"
     aria-labelledby="exampleModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">

      <!-- ヘッダー -->
      <div class="modal-header">
        <h5 class="modal-title" id="exampleModalLabel">確認</h5>
        <button type="button" class="btn-close"
                data-bs-dismiss="modal" aria-label="閉じる"></button>
      </div>

      <!-- ボディ -->
      <div class="modal-body">
        <p>本当にこの操作を実行しますか？</p>
      </div>

      <!-- フッター -->
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary"
                data-bs-dismiss="modal">キャンセル</button>
        <button type="button" class="btn btn-primary">実行する</button>
      </div>

    </div>
  </div>
</div>

data-bs-toggle と data-bs-target

data-bs-toggle="modal" はボタンクリック時にモーダルを開く指示です。data-bs-target="#exampleModal" で開くモーダルのIDを指定します。このIDはモーダル本体の id 属性と一致させる必要があります。data-bs-dismiss="modal" はモーダルを閉じるためのトリガーです。

フォーム付きモーダル

モーダル内にフォームを配置するパターンはよく使われます。ログインフォーム、お問い合わせフォーム、簡易入力フォームなどに活用できます。

HTML 
<!-- トリガーボタン -->
<button type="button" class="btn btn-success"
        data-bs-toggle="modal" data-bs-target="#loginModal">
  ログイン
</button>

<!-- ログインモーダル -->
<div class="modal fade" id="loginModal" tabindex="-1"
     aria-labelledby="loginModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">

      <div class="modal-header">
        <h5 class="modal-title" id="loginModalLabel">ログイン</h5>
        <button type="button" class="btn-close"
                data-bs-dismiss="modal" aria-label="閉じる"></button>
      </div>

      <div class="modal-body">
        <form>
          <div class="mb-3">
            <label for="email" class="form-label">メールアドレス</label>
            <input type="email" class="form-control" id="email"
                   placeholder="name@example.com">
          </div>
          <div class="mb-3">
            <label for="password" class="form-label">パスワード</label>
            <input type="password" class="form-control" id="password">
          </div>
          <div class="form-check mb-3">
            <input type="checkbox" class="form-check-input" id="remember">
            <label class="form-check-label" for="remember">ログイン状態を保持</label>
          </div>
        </form>
      </div>

      <div class="modal-footer">
        <button type="button" class="btn btn-secondary"
                data-bs-dismiss="modal">キャンセル</button>
        <button type="button" class="btn btn-success">ログイン</button>
      </div>

    </div>
  </div>
</div>

モーダルのサイズ変更

modal-dialog に追加クラスを指定することで、モーダルのサイズを変更できます。

クラス 最大幅 用途
modal-sm 300px 確認ダイアログなどの小さなモーダル
（デフォルト） 500px 通常のモーダル
modal-lg 800px フォームや詳細表示の大きめモーダル
modal-xl 1140px 画像ギャラリーなどの特大モーダル
modal-fullscreen 画面全体 フルスクリーンモーダル
HTML 
<!-- 大きいモーダル -->
<div class="modal fade" id="largeModal" tabindex="-1">
  <div class="modal-dialog modal-lg">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title">商品詳細</h5>
        <button type="button" class="btn-close" data-bs-dismiss="modal"></button>
      </div>
      <div class="modal-body">
        <p>ここに詳細な商品情報を表示します。</p>
      </div>
    </div>
  </div>
</div>

<!-- 小さいモーダル -->
<div class="modal fade" id="smallModal" tabindex="-1">
  <div class="modal-dialog modal-sm">
    <div class="modal-content">
      <div class="modal-body text-center">
        <p>削除しますか？</p>
        <button class="btn btn-danger btn-sm">削除</button>
        <button class="btn btn-secondary btn-sm" data-bs-dismiss="modal">キャンセル</button>
      </div>
    </div>
  </div>
</div>

<!-- 中央寄せモーダル -->
<div class="modal fade" id="centeredModal" tabindex="-1">
  <div class="modal-dialog modal-dialog-centered">
    <div class="modal-content">
      <div class="modal-body">
        <p>画面の中央に表示されるモーダルです。</p>
      </div>
    </div>
  </div>
</div>

modal-dialog-centered

modal-dialog-centered を追加すると、モーダルが画面の上下中央に表示されます。デフォルトでは上寄りに表示されるため、中央表示にしたい場合はこのクラスを追加しましょう。サイズクラスとの組み合わせも可能です（例: modal-dialog modal-lg modal-dialog-centered）。

Section 11-2

カルーセル（スライドショー）

カルーセルとは

カルーセル（Carousel）は、複数の画像やコンテンツをスライドショー形式で順番に表示するコンポーネントです。ヒーローセクションやバナー、商品ギャラリーなどで広く使われています。

基本的なカルーセル

HTML 
<div id="myCarousel" class="carousel slide" data-bs-ride="carousel">

  <!-- インジケーター（下部のドット） -->
  <div class="carousel-indicators">
    <button type="button" data-bs-target="#myCarousel" data-bs-slide-to="0"
            class="active" aria-current="true" aria-label="スライド1"></button>
    <button type="button" data-bs-target="#myCarousel" data-bs-slide-to="1"
            aria-label="スライド2"></button>
    <button type="button" data-bs-target="#myCarousel" data-bs-slide-to="2"
            aria-label="スライド3"></button>
  </div>

  <!-- スライド本体 -->
  <div class="carousel-inner">
    <div class="carousel-item active">
      <img src="https://via.placeholder.com/1200x400/6c757d/ffffff?text=Slide+1"
           class="d-block w-100" alt="スライド1">
    </div>
    <div class="carousel-item">
      <img src="https://via.placeholder.com/1200x400/495057/ffffff?text=Slide+2"
           class="d-block w-100" alt="スライド2">
    </div>
    <div class="carousel-item">
      <img src="https://via.placeholder.com/1200x400/343a40/ffffff?text=Slide+3"
           class="d-block w-100" alt="スライド3">
    </div>
  </div>

  <!-- 前後の矢印ボタン -->
  <button class="carousel-control-prev" type="button"
          data-bs-target="#myCarousel" data-bs-slide="prev">
    <span class="carousel-control-prev-icon" aria-hidden="true"></span>
    <span class="visually-hidden">前へ</span>
  </button>
  <button class="carousel-control-next" type="button"
          data-bs-target="#myCarousel" data-bs-slide="next">
    <span class="carousel-control-next-icon" aria-hidden="true"></span>
    <span class="visually-hidden">次へ</span>
  </button>

</div>
クラス / 属性 役割
carousel slide カルーセル本体。slide でスライドアニメーションを有効化
data-bs-ride="carousel" ページ読み込み時に自動再生を開始
carousel-indicators 下部のドットナビゲーション
carousel-inner スライドのコンテナ
carousel-item 個々のスライド。最初のスライドに active を付与
carousel-control-prev/next 前後の矢印ボタン

キャプション付きカルーセル

各スライドにタイトルや説明文を重ねて表示できます。

HTML 
<div class="carousel-item active">
  <img src="https://via.placeholder.com/1200x500/4a3728/ffffff"
       class="d-block w-100" alt="スライド1">
  <div class="carousel-caption d-none d-md-block">
    <h5>こだわりのコーヒー</h5>
    <p>毎朝焙煎した新鮮な豆をお届けします。</p>
  </div>
</div>

d-none d-md-block に注目

キャプションに d-none d-md-block が付いているのは、スマホの小さい画面ではキャプションが画像に被って読みにくくなるためです。md以上の画面でのみキャプションを表示することで、各デバイスで最適な体験を提供しています。

自動再生とインターバル設定

カルーセルの自動再生速度は data-bs-interval 属性で制御できます。

HTML 
<!-- 全体のインターバルを5秒に設定 -->
<div id="slowCarousel" class="carousel slide" data-bs-ride="carousel"
     data-bs-interval="5000">
  <div class="carousel-inner">
    <!-- スライドごとに異なるインターバルも可能 -->
    <div class="carousel-item active" data-bs-interval="3000">
      <img src="slide1.jpg" class="d-block w-100" alt="3秒で切替">
    </div>
    <div class="carousel-item" data-bs-interval="7000">
      <img src="slide2.jpg" class="d-block w-100" alt="7秒で切替">
    </div>
    <div class="carousel-item">
      <img src="slide3.jpg" class="d-block w-100" alt="デフォルト5秒">
    </div>
  </div>
</div>

<!-- クロスフェード効果 -->
<div id="fadeCarousel" class="carousel slide carousel-fade" data-bs-ride="carousel">
  <div class="carousel-inner">
    <div class="carousel-item active">
      <img src="slide1.jpg" class="d-block w-100" alt="スライド1">
    </div>
    <div class="carousel-item">
      <img src="slide2.jpg" class="d-block w-100" alt="スライド2">
    </div>
  </div>
</div>
属性 / クラス 効果
data-bs-interval="5000" スライドの切り替え間隔を5000ミリ秒（5秒）に設定
data-bs-ride="carousel" ページ表示時に自動再生開始
data-bs-ride="true" 手動操作後も自動再生を続行
carousel-fade スライドではなくフェード（クロスフェード）で切り替え
Section 11-3

アコーディオン

アコーディオンとは

アコーディオン（Accordion）は、折りたたみ可能なコンテンツを縦に積み重ねたコンポーネントです。FAQ、設定パネル、カテゴリ分けされたコンテンツの表示に最適です。1つの項目を開くと他の項目が自動的に閉じる動作がデフォルトです。

基本的なアコーディオン

HTML 
<div class="accordion" id="faqAccordion">

  <!-- アイテム1（最初から開いている） -->
  <div class="accordion-item">
    <h2 class="accordion-header">
      <button class="accordion-button" type="button"
              data-bs-toggle="collapse" data-bs-target="#faq1"
              aria-expanded="true" aria-controls="faq1">
        営業時間を教えてください
      </button>
    </h2>
    <div id="faq1" class="accordion-collapse collapse show"
         data-bs-parent="#faqAccordion">
      <div class="accordion-body">
        平日は9:00〜20:00、土日祝は10:00〜18:00で営業しております。
        年末年始（12/31〜1/3）はお休みをいただいております。
      </div>
    </div>
  </div>

  <!-- アイテム2 -->
  <div class="accordion-item">
    <h2 class="accordion-header">
      <button class="accordion-button collapsed" type="button"
              data-bs-toggle="collapse" data-bs-target="#faq2"
              aria-expanded="false" aria-controls="faq2">
        予約は必要ですか？
      </button>
    </h2>
    <div id="faq2" class="accordion-collapse collapse"
         data-bs-parent="#faqAccordion">
      <div class="accordion-body">
        予約なしでもご来店いただけますが、週末は混雑する場合がございます。
        お電話またはWebからの事前予約をおすすめいたします。
      </div>
    </div>
  </div>

  <!-- アイテム3 -->
  <div class="accordion-item">
    <h2 class="accordion-header">
      <button class="accordion-button collapsed" type="button"
              data-bs-toggle="collapse" data-bs-target="#faq3"
              aria-expanded="false" aria-controls="faq3">
        駐車場はありますか？
      </button>
    </h2>
    <div id="faq3" class="accordion-collapse collapse"
         data-bs-parent="#faqAccordion">
      <div class="accordion-body">
        店舗前に5台分の駐車スペースがございます。
        満車の場合は、近隣のコインパーキングをご利用ください。
      </div>
    </div>
  </div>

</div>
クラス / 属性 役割
accordion アコーディオン全体のラッパー
accordion-item 個々の折りたたみアイテム
accordion-header クリック可能なヘッダー部分
accordion-button ヘッダー内のボタン（矢印アイコン付き）
accordion-collapse collapse 折りたたまれるコンテンツ部分
accordion-body コンテンツの本文
data-bs-parent 親アコーディオンを指定（他の項目を自動で閉じる）

開閉状態の初期設定

最初から開いている項目にはボタンに accordion-button（collapsed なし）、コンテンツに collapse show を指定します。閉じている項目にはボタンに accordion-button collapsed、コンテンツに collapse（show なし）を指定します。

フラッシュアコーディオン（ボーダーレス）

accordion-flush クラスを追加すると、外枠のボーダーと角丸が除去され、よりシンプルな見た目になります。親要素に枠がある場合（カード内など）に適しています。

HTML 
<div class="accordion accordion-flush" id="flushAccordion">

  <div class="accordion-item">
    <h2 class="accordion-header">
      <button class="accordion-button collapsed" type="button"
              data-bs-toggle="collapse" data-bs-target="#flush1">
        送料について
      </button>
    </h2>
    <div id="flush1" class="accordion-collapse collapse"
         data-bs-parent="#flushAccordion">
      <div class="accordion-body">
        5,000円以上のご注文で送料無料です。
        5,000円未満の場合、一律550円の送料がかかります。
      </div>
    </div>
  </div>

  <div class="accordion-item">
    <h2 class="accordion-header">
      <button class="accordion-button collapsed" type="button"
              data-bs-toggle="collapse" data-bs-target="#flush2">
        返品・交換について
      </button>
    </h2>
    <div id="flush2" class="accordion-collapse collapse"
         data-bs-parent="#flushAccordion">
      <div class="accordion-body">
        商品到着後7日以内にご連絡ください。
        未開封・未使用の商品に限り、返品・交換を承ります。
      </div>
    </div>
  </div>

</div>

            </div>

    </div>
  </div>
</div>
Section 11-4

トーストとツールチップ

トースト通知とは

トースト（Toast）は、画面の端にフワッと表示される通知メッセージです。操作完了の確認やお知らせなど、ユーザーの操作を中断せずに情報を伝えることができます。名前の由来はトースターからパンが飛び出す様子に似ていることです。

基本的なトースト

HTML 
<!-- トリガーボタン -->
<button type="button" class="btn btn-primary" id="showToastBtn">
  トーストを表示
</button>

<!-- トースト（画面右上に固定配置） -->
<div class="toast-container position-fixed top-0 end-0 p-3">
  <div id="myToast" class="toast" role="alert"
       aria-live="assertive" aria-atomic="true">
    <div class="toast-header">
      <strong class="me-auto">お知らせ</strong>
      <small>たった今</small>
      <button type="button" class="btn-close"
              data-bs-dismiss="toast" aria-label="閉じる"></button>
    </div>
    <div class="toast-body">
      カートに商品を追加しました。
    </div>
  </div>
</div>
クラス 役割
toast-container トーストを配置するコンテナ
position-fixed top-0 end-0 画面右上に固定配置
toast トースト本体
toast-header トーストのヘッダー（タイトル・時刻・閉じるボタン）
toast-body トーストの本文

トーストの配置位置

position-fixed と Bootstrap の位置ユーティリティを組み合わせて配置場所を変更できます。top-0 end-0 は右上、bottom-0 end-0 は右下、top-0 start-50 translate-middle-x は上中央です。

ツールチップとポップオーバー

ツールチップ（Tooltip）は要素にマウスを乗せた時に表示される小さな説明テキストです。ポップオーバー（Popover）はクリック時に表示される、タイトル付きのより大きな説明です。

HTML 
<!-- ツールチップ -->
<button type="button" class="btn btn-secondary"
        data-bs-toggle="tooltip" data-bs-placement="top"
        title="これはツールチップです">
  上にツールチップ
</button>

<button type="button" class="btn btn-secondary"
        data-bs-toggle="tooltip" data-bs-placement="bottom"
        title="下に表示されます">
  下にツールチップ
</button>

<button type="button" class="btn btn-secondary"
        data-bs-toggle="tooltip" data-bs-placement="left"
        title="左に表示されます">
  左にツールチップ
</button>

<button type="button" class="btn btn-secondary"
        data-bs-toggle="tooltip" data-bs-placement="right"
        title="右に表示されます">
  右にツールチップ
</button>

<!-- ポップオーバー -->
<button type="button" class="btn btn-info"
        data-bs-toggle="popover" data-bs-placement="right"
        data-bs-title="ポップオーバーのタイトル"
        data-bs-content="ここにポップオーバーの本文が入ります。クリックで表示・非表示を切り替えます。">
  ポップオーバーを表示
</button>
属性 効果
data-bs-toggle="tooltip" ツールチップを有効化
data-bs-toggle="popover" ポップオーバーを有効化
data-bs-placement 表示位置（top / bottom / left / right）
title ツールチップのテキスト
data-bs-title ポップオーバーのタイトル
data-bs-content ポップオーバーの本文

JavaScript初期化が必要なコンポーネント

Bootstrapのコンポーネントの多くは data-bs-* 属性だけで動作しますが、ツールチップ、ポップオーバー、トーストの3つはJavaScriptでの初期化が必要です。

なぜ初期化が必要なのか

ツールチップとポップオーバーはパフォーマンス上の理由からオプトイン方式（手動で有効化する方式）を採用しています。ページ内のすべての要素を自動的に監視すると処理が重くなるため、開発者が明示的に有効化する設計になっています。トーストも同様に、表示タイミングを開発者が制御する必要があるため初期化が必要です。

HTML（完全な実装例） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>JS初期化が必要なコンポーネント</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
        rel="stylesheet">
</head>
<body>

  <div class="container py-5">
    <h2 class="mb-4">ツールチップ &amp; ポップオーバー &amp; トースト</h2>

    <!-- ツールチップ -->
    <button type="button" class="btn btn-primary me-2"
            data-bs-toggle="tooltip" title="これはツールチップ">
      ホバーしてね
    </button>

    <!-- ポップオーバー -->
    <button type="button" class="btn btn-warning me-2"
            data-bs-toggle="popover"
            data-bs-title="情報"
            data-bs-content="これはポップオーバーの内容です。">
      クリックしてね
    </button>

    <!-- トーストのトリガー -->
    <button type="button" class="btn btn-success" id="showToast">
      トーストを表示
    </button>
  </div>

  <!-- トースト本体 -->
  <div class="toast-container position-fixed bottom-0 end-0 p-3">
    <div id="myToast" class="toast" role="alert">
      <div class="toast-header">
        <strong class="me-auto">通知</strong>
        <small>たった今</small>
        <button type="button" class="btn-close" data-bs-dismiss="toast"></button>
      </div>
      <div class="toast-body">
        操作が完了しました！
      </div>
    </div>
  </div>

  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
  <script>
    // ===== ツールチップの初期化 =====
    const tooltipTriggerList = document.querySelectorAll('[data-bs-toggle="tooltip"]');
    const tooltipList = [...tooltipTriggerList].map(
      tooltipTriggerEl => new bootstrap.Tooltip(tooltipTriggerEl)
    );

    // ===== ポップオーバーの初期化 =====
    const popoverTriggerList = document.querySelectorAll('[data-bs-toggle="popover"]');
    const popoverList = [...popoverTriggerList].map(
      popoverTriggerEl => new bootstrap.Popover(popoverTriggerEl)
    );

    // ===== トーストの表示 =====
    document.getElementById('showToast').addEventListener('click', function() {
      const toastEl = document.getElementById('myToast');
      const toast = new bootstrap.Toast(toastEl);
      toast.show();
    });
  </script>
</body>
</html>

初期化コードの解説

コンポーネント 初期化方法 説明
ツールチップ new bootstrap.Tooltip(element) data-bs-toggle="tooltip" を持つ全要素を取得して初期化
ポップオーバー new bootstrap.Popover(element) data-bs-toggle="popover" を持つ全要素を取得して初期化
トースト new bootstrap.Toast(element).show() 要素を取得 → Toastインスタンス作成 → show()で表示

JS初期化が不要なコンポーネント

モーダル、カルーセル、アコーディオン、ドロップダウン、コラプス、タブなどは data-bs-* 属性だけで動作します。ツールチップ・ポップオーバー・トーストの3つだけが JavaScript 初期化が必要であることを覚えておきましょう。
