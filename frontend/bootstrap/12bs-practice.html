Section 12-1

プロジェクト概要とワイヤーフレーム

プロジェクトの目標

この章では、これまでに学んだBootstrapの知識をすべて活用して、架空のカフェ「Cafe Latte」のランディングページを一から作成します。

ランディングページ（LP）とは、ユーザーが最初にアクセスするページのことで、お店や商品の魅力を1ページにまとめて伝えるWebページです。実務でもよく制作するタイプのページなので、しっかりと練習しましょう。

使用する Bootstrap の技術

このプロジェクトでは以下の技術を総合的に使用します。

  • グリッドシステム（row / col）
  • ナビゲーションバー（navbar / fixed-top）
  • カードコンポーネント（card / card-group）
  • フォーム（form-control / input-group / floating-label）
  • ユーティリティクラス（spacing / display / flex / shadow）
  • レスポンシブデザイン（ブレイクポイント / d-none）

ページ構成（ワイヤーフレーム）

ランディングページは以下の6つのセクションで構成します。

セクション 内容 使用コンポーネント
1. ヘッダー ロゴ、ナビゲーションリンク navbar, fixed-top, navbar-toggler
2. ヒーロー メインビジュアル、キャッチコピー、CTAボタン bg-dark, text-white, display-*, lead, btn
3. メニュー おすすめメニューのカード一覧 card, row-cols, g-4, shadow
4. アクセス 地図、店舗情報 row, col, iframe（地図埋め込み）
5. お問い合わせ 問い合わせフォーム form-control, form-floating, input-group
6. フッター リンク集、SNS、コピーライト row, col, text-white, bg-dark

Section 12-2

ヘッダーとヒーローセクション

固定ナビゲーションバー

まずはページ上部に固定されるナビゲーションバーを作成します。fixed-topを使うことで、ページをスクロールしても常に画面上部に表示されます。

HTML 
<!-- ナビゲーションバー -->
<nav class="navbar navbar-expand-lg navbar-dark fixed-top"
     style="background-color: rgba(74, 55, 40, 0.95);">
  <div class="container">

    <!-- ロゴ -->
    <a class="navbar-brand fw-bold fs-4" href="#">Cafe Latte</a>

    <!-- ハンバーガーメニュー（スマホ用） -->
    <button class="navbar-toggler" type="button"
            data-bs-toggle="collapse" data-bs-target="#navbarNav">
      <span class="navbar-toggler-icon"></span>
    </button>

    <!-- ナビゲーションリンク -->
    <div class="collapse navbar-collapse" id="navbarNav">
      <ul class="navbar-nav ms-auto">
        <li class="nav-item">
          <a class="nav-link active" href="#hero">ホーム</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#menu">メニュー</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#access">アクセス</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#contact">お問い合わせ</a>
        </li>
      </ul>
    </div>

  </div>
</nav>
クラス / 属性 役割
navbar-expand-lg lg以上で横並び、それ未満でハンバーガーメニュー
navbar-dark テキストとアイコンを白色にする
fixed-top ナビバーを画面上部に固定
ms-auto ナビリンクを右寄せにする
navbar-toggler スマホ用ハンバーガーボタン

fixed-top 使用時の注意

fixed-topを使うと、ナビバーがページのコンテンツの上に重なります。そのため、body や次のセクションに padding-top（ナビバーの高さ分、約70px程度）を追加して、コンテンツが隠れないようにする必要があります。ヒーローセクションが全画面表示の場合は、ヒーロー自体がナビバーの下に来るため問題ありません。

ヒーローセクション

ヒーローセクションはページの最も目立つ部分です。背景色とテキストを使って、カフェの雰囲気を伝えます。

HTML 
<!-- ヒーローセクション -->
<section id="hero" class="text-white text-center d-flex align-items-center"
         style="min-height: 100vh; background-color: #4a3728;">
  <div class="container">
    <h1 class="display-3 display-md-1 fw-bold mb-3">
      こだわりの一杯を、<br class="d-none d-md-inline">あなたに。
    </h1>
    <p class="lead mb-4 mx-auto" style="max-width: 600px;">
      厳選された豆を毎朝焙煎。心を込めて淹れる一杯が、
      あなたの一日を特別な時間に変えます。
    </p>
    <div class="d-flex justify-content-center gap-3 flex-column flex-sm-row">
      <a href="#menu" class="btn btn-light btn-lg px-4">メニューを見る</a>
      <a href="#contact" class="btn btn-outline-light btn-lg px-4">お問い合わせ</a>
    </div>
  </div>
</section>
テクニック 使用クラス 効果
全画面表示 min-height: 100vh ビューポート全体の高さにする
垂直中央寄せ d-flex align-items-center コンテンツを上下中央に配置
レスポンシブ改行 d-none d-md-inline（brタグ） PCでは改行、スマホでは改行なし
レスポンシブボタン flex-column flex-sm-row スマホ:縦並び → タブレット以上:横並び
テキスト幅制限 mx-auto + max-width: 600px リード文の幅を制限して読みやすく

カスタムCSSの追加

ヒーローセクションをより魅力的にするために、以下のカスタムCSSを <style> タグ内に追加しましょう。

CSS 
/* ヒーローセクション用カスタムスタイル */
body {
  font-family: 'Noto Sans JP', sans-serif;
}

#hero {
  background: linear-gradient(135deg, #4a3728 0%, #6b4226 50%, #8b6914 100%);
}

/* スムーススクロール */
html {
  scroll-behavior: smooth;
}

/* fixed-topの余白補正（ヒーロー以外のページ開始時用） */
.section-offset {
  scroll-margin-top: 70px;
}

Section 12-4

アクセスセクション

店舗の場所を示すために、地図と店舗情報を横並びで表示します。地図には iframe による埋め込み（プレースホルダー）を使います。

テクニック 使用クラス 効果
フローティングラベル form-floating 入力時にラベルが上部に移動する洗練されたUI

フローティングラベルの注意点

form-floating を使う場合、<input> タグを <label> タグの前に記述する必要があります（通常のフォームと順番が逆です）。また、placeholder 属性は必須です（CSSのセレクタで使用されるため）。placeholder の値は label と同じテキストにしておくのが一般的です。

フッターと仕上げ

マルチカラムフッター

フッターはグリッドシステムを使って複数カラムに分割し、リンク集やSNSリンク、コピーライトを配置します。

  <div class="container">

仕上げのチェックリスト

ランディングページが完成したら、以下のポイントを確認しましょう。

確認項目 チェック内容
レスポンシブ表示 スマホ・タブレット・PCでレイアウトが崩れないか
ナビゲーション 各リンクをクリックして正しいセクションへスクロールするか
ハンバーガーメニュー スマホサイズでメニューが正しく開閉するか
カードの高さ 同じ行のカードの高さが揃っているか
フォーム フローティングラベルが正しく動作するか
固定ナビバー 各セクションの先頭がナビバーに隠れていないか
フッター カラムが正しく分割されているか
テキスト 誤字脱字、テキストの切れがないか
