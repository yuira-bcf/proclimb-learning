Section 1-1

CSSフレームワークとは

フレームワークという考え方

Webサイトを制作するとき、CSSを一から手書きしていくのは大変な作業です。
レイアウト、ボタン、フォーム、ナビゲーションなど、どのWebサイトにも共通するパーツを
毎回ゼロから書いていては、時間がかかりすぎてしまいます。

そこで登場するのがCSSフレームワークです。
CSSフレームワークとは、Webサイトでよく使われるスタイルやレイアウトを
あらかじめ用意してくれている「CSSのテンプレート集」のようなものです。

フレームワーク（Framework）は英語で「骨組み」「枠組み」という意味です。
家を建てるときに、柱や梁といった骨組みが最初にあるように、
CSSフレームワークはWebデザインの骨組みを提供してくれます。

なぜCSSフレームワークが必要なのか

CSSフレームワークを使う理由は主に3つあります。

理由 説明
開発速度の向上 すでに用意されたクラスを使うだけで、短時間で見た目を整えられる
デザインの一貫性 ボタン・フォーム・テーブルなどのスタイルが統一され、バラバラにならない
レスポンシブ対応 スマホ・タブレット・PCに対応するレイアウトが最初から組み込まれている

手書きCSSとフレームワークの比較

同じ「青いボタン」を作る場合を比較してみましょう。

CSS（手書きの場合） 
.btn-primary {
  display: inline-block;
  padding: 8px 16px;
  font-size: 16px;
  font-weight: 400;
  color: #fff;
  background-color: #0d6efd;
  border: 1px solid #0d6efd;
  border-radius: 6px;
  text-align: center;
  text-decoration: none;
  cursor: pointer;
  transition: background-color 0.15s ease-in-out;
}
.btn-primary:hover {
  background-color: #0b5ed7;
  border-color: #0a58ca;
}

手書きの場合、ボタン1つに対して15行以上のCSSを書く必要があります。
さらに、ホバー効果や色違いのボタンも作るとなると、コード量はどんどん増えていきます。

HTML（Bootstrapを使う場合） 
<button class="btn btn-primary">ボタン</button>

Bootstrapを使えば、HTMLに class="btn btn-primary" と書くだけです。
CSSは一切書く必要がありません。ホバー効果やフォーカスのスタイルも自動で付いてきます。

POINT

CSSフレームワークは「CSSを書かなくていい」のではなく、
「よく使うスタイルを自分で書かなくていい」というものです。
独自のデザインが必要な部分は、引き続き自分でCSSを書いてカスタマイズします。

代表的なCSSフレームワーク

フレームワーク名 特徴 用途
Bootstrap 最も利用者が多い。コンポーネントが豊富 企業サイト、管理画面、プロトタイプ
Tailwind CSS ユーティリティファーストの設計思想 カスタムデザインが必要なサイト
Bulma Flexboxベース、JavaScriptなし 軽量なサイト
Foundation 高度なカスタマイズが可能 大規模プロジェクト

この中でもBootstrapは、世界中で最も使われているCSSフレームワークです。
就職・転職時の求人でも「Bootstrapが使える」ことが求められる場面が多く、
Web制作の基本スキルとして広く認知されています。

Section 1-2

Bootstrapの概要と歴史

Bootstrapとは

Bootstrap（ブートストラップ）は、Twitter社（現X社）のエンジニアである
Mark Otto（マーク・オットー）氏と Jacob Thornton（ジェイコブ・ソーントン）氏が
2011年に開発・公開した、オープンソースのCSSフレームワークです。

もともとはTwitter社内のWebアプリケーション開発で、
デザインの統一性と開発効率を高めるために作られた社内ツールでした。
それがオープンソースとして公開されると、その使いやすさから爆発的に広まりました。

Bootstrapの歴史（バージョン変遷）

バージョン 公開年 主な変更点
Bootstrap 1 2011年 Twitter社内ツールとしてオープンソース公開
Bootstrap 2 2012年 レスポンシブデザインに対応、12カラムグリッド導入
Bootstrap 3 2013年 モバイルファースト設計に転換、フラットデザイン採用
Bootstrap 4 2018年 Flexboxベースに刷新、Sassに移行、カード導入
Bootstrap 5 2021年 jQuery依存を廃止、RTL対応、ユーティリティAPI追加

POINT

本教材ではBootstrap 5（5.3.x）を使用します。
Bootstrap 5の最大の特徴は、jQuery（JavaScriptライブラリ）への依存がなくなったことです。
これにより、ページの読み込み速度が向上し、現代のWeb開発により適したものになりました。

Bootstrapの人気

Bootstrapは公開以来、世界中のWeb開発者に愛用されてきました。
GitHubで最もスターを集めているCSSフレームワークであり、
何百万ものWebサイトがBootstrapを使って構築されています。

日本国内でも、企業サイト、ECサイト、管理画面、社内ツールなど、
幅広い分野でBootstrapが活用されています。
とくに短期間で見た目の整ったWebページを作りたい場合に重宝されています。

Section 1-3

Bootstrapの特徴

Bootstrapには、Web制作を効率化するための4つの大きな特徴があります。
ここではそれぞれの特徴を、簡単なコード例とともに紹介します。

特徴1：レスポンシブデザインが標準装備

Bootstrapは「モバイルファースト」という設計思想を採用しています。
まずスマートフォン向けのレイアウトを基本とし、
画面が大きくなるにつれてレイアウトを変化させる仕組みです。

Bootstrapのグリッドシステムを使えば、CSSのメディアクエリを自分で書かなくても、
自動的にレスポンシブなレイアウトが実現できます。

HTML 
<div class="container">
  <div class="row">
    <div class="col-12 col-md-6 col-lg-4">カラム1</div>
    <div class="col-12 col-md-6 col-lg-4">カラム2</div>
    <div class="col-12 col-md-12 col-lg-4">カラム3</div>
  </div>
</div>

上記のコードでは、画面サイズに応じてカラムの幅が自動的に変わります。

画面サイズ 表示レイアウト
スマホ（～576px） 各カラムが100%幅で縦に並ぶ
タブレット（768px～） 2カラムずつ横に並ぶ
PC（992px～） 3カラムが横に並ぶ

特徴2：豊富なコンポーネント

Bootstrapには、Webサイトでよく使われるUI部品（コンポーネント）が
数多く用意されています。ボタン、ナビゲーション、カード、モーダル、
フォーム、テーブルなど、必要なパーツをクラス名を付けるだけで使えます。

HTML（アラートの例） 
<div class="alert alert-success">
  保存が完了しました！
</div>

<div class="alert alert-danger">
  エラーが発生しました。
</div>

たった1行のHTMLで、色付きの通知メッセージ（アラート）を表示できます。
alert-success で緑色、alert-danger で赤色といった具合に、
クラス名を変えるだけで色が変わります。

特徴3：ユーティリティクラス

Bootstrapには「ユーティリティクラス」と呼ばれる、
CSSの1つのプロパティに対応するクラスが多数用意されています。
これにより、CSSファイルを開かなくても、HTMLだけでスタイルを調整できます。

HTML（ユーティリティクラスの例） 
<!-- 余白の調整 -->
<div class="mt-3 mb-5 p-4">
  margin-topが3、margin-bottomが5、paddingが4
</div>

<!-- テキストの配置 -->
<p class="text-center text-primary fw-bold">
  中央寄せ・青色・太字
</p>
クラス名 効果
mt-3 margin-topを段階3（1rem）に設定
mb-5 margin-bottomを段階5（3rem）に設定
p-4 padding全方向を段階4（1.5rem）に設定
text-center テキストを中央揃えにする
text-primary テキスト色をprimary（青）にする
fw-bold font-weightをbold（太字）にする

特徴4：カスタマイズ可能（Sassベース）

Bootstrapは内部的にSass（SCSS）で書かれています。
Sass変数を上書きすることで、Bootstrapの配色やフォントサイズ、
余白のサイズなどを自分のプロジェクトに合わせてカスタマイズできます。

SCSS（カスタマイズ例） 
// Bootstrapの変数を上書き
$primary: #e74c3c;    // メインカラーを赤に変更
$font-size-base: 1rem; // 基本フォントサイズ

// Bootstrapを読み込み
@import "bootstrap/scss/bootstrap";

POINT

Sassによるカスタマイズは上級者向けの機能です。
まずはCDNで読み込んだ標準のBootstrapをそのまま使うことから始めましょう。
必要に応じて、自分でCSSを追加してデザインを調整する方法で十分です。

Section 1-4

CDNでの導入方法

CDNとは

CDN（Content Delivery Network）とは、ファイルをインターネット上のサーバーから
直接読み込む仕組みです。Bootstrapのファイルを自分のパソコンにダウンロードしなくても、
URLを指定するだけでBootstrapが使えるようになります。

CDNを使う最大のメリットは、すぐに始められることです。
HTMLファイルを1つ作るだけで、Bootstrapの全機能が使えます。

Bootstrap 5.3 のCDNリンク

Bootstrapを使うには、2つのファイルを読み込む必要があります。

ファイル 役割 記述場所
bootstrap.min.css スタイル（見た目）の定義 <head>タグ内
bootstrap.bundle.min.js 動的コンポーネント（モーダル、ドロップダウンなど）の動作 </body>の直前

完全なHTMLテンプレート

以下が、Bootstrap 5.3.3を使ったHTMLの基本テンプレートです。
新しいページを作るときは、このテンプレートをコピーして使いましょう。

HTML（Bootstrap基本テンプレート） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Bootstrap Page</title>

  <!-- Bootstrap CSS -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
        rel="stylesheet">
</head>
<body>

  <h1>Hello Bootstrap!</h1>

  <!-- Bootstrap JS Bundle (Popper.js 含む) -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js">
  </script>
</body>
</html>

POINT

<meta name="viewport"> は必ず記述してください。
この記述がないと、スマートフォンでBootstrapのレスポンシブ機能が正しく動作しません。
Bootstrap公式ドキュメントでも必須とされている設定です。

Hello Bootstrapページを作ってみよう

テンプレートを使って、実際にBootstrapが動くページを作ってみましょう。
以下のコードをコピーして、index.html として保存し、ブラウザで開いてみてください。

HTML（Hello Bootstrap） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Hello Bootstrap</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
        rel="stylesheet">
</head>
<body>

  <div class="container mt-5">
    <h1 class="text-primary">Hello Bootstrap!</h1>
    <p class="lead">Bootstrapを使った最初のページです。</p>

    <button class="btn btn-primary">プライマリボタン</button>
    <button class="btn btn-success">成功ボタン</button>
    <button class="btn btn-danger">危険ボタン</button>

    <div class="alert alert-info mt-3">
      Bootstrapが正しく読み込まれています！
    </div>
  </div>

  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js">
  </script>
</body>
</html>

ブラウザで開いたときに、青・緑・赤のボタンと水色の通知メッセージが
表示されていれば、Bootstrapが正しく読み込まれています。

CSSが読み込まれているか確認する方法

もしBootstrapが正しく読み込まれていない場合、
ボタンはただのテキストリンクのように表示され、色も付きません。
そのときは以下を確認してください。

確認項目 よくある原因
インターネットに接続されているか CDNはインターネット経由でファイルを読み込むため、オフラインでは使えない
URLが正しいか コピーミスやバージョン番号の間違い
linkタグの場所 <head>タグの中に書かれているか確認
viewportの記述 metaタグのviewportが<head>内にあるか確認

POINT

BootstrapのCSSとJSはそれぞれ別のファイルです。
CSSだけ読み込んでもボタンやテーブルの見た目は使えますが、
モーダルやドロップダウンなどの「動き」が必要なコンポーネントにはJSも必要です。
迷ったら、両方とも読み込んでおきましょう。

Section 1-5

公式ドキュメントの使い方

公式サイトにアクセスしよう

Bootstrapの公式ドキュメントは、以下のURLで公開されています。

URL 
https://getbootstrap.com/

公式ドキュメントは英語で書かれていますが、コード例が豊富なため、
英語が苦手でもコードを読むだけで十分に理解できます。
ブラウザの翻訳機能を使って日本語に翻訳することも可能です。

ドキュメントの構成

公式ドキュメントは、以下のカテゴリに分かれています。

カテゴリ 内容 用途
Getting started 導入方法、ダウンロード 初めてBootstrapを使うとき
Layout グリッドシステム、コンテナ ページの骨組みを作るとき
Content テキスト、画像、テーブル 文章や画像のスタイルを調整するとき
Forms フォーム要素のスタイル お問い合わせフォームを作るとき
Components ボタン、カード、モーダルなど UI部品を使いたいとき
Utilities 余白、色、表示の調整クラス 細かいスタイル調整をしたいとき

ドキュメントの活用方法

Bootstrapの学習で最も効率的な方法は、以下の3ステップです。

ステップ1：使いたいコンポーネントを探す

左のサイドメニューから目的のコンポーネントを選びます。
例えば「Buttons」を選ぶと、ボタンに関する全てのスタイルが表示されます。

ステップ2：コード例をコピーする

公式ドキュメントのコード例には「Copy」ボタンが付いています。
これをクリックすると、HTMLコードがクリップボードにコピーされます。
それを自分のHTMLファイルに貼り付けるだけで使えます。

HTML（公式ドキュメントからコピーした例） 
<!-- ドキュメントのButtonsページからコピー -->
<button type="button" class="btn btn-primary">Primary</button>
<button type="button" class="btn btn-secondary">Secondary</button>
<button type="button" class="btn btn-success">Success</button>

ステップ3：自分のプロジェクトに合わせてカスタマイズする

コピーしたコードのテキスト部分を変更したり、クラス名を追加して調整します。

HTML（カスタマイズ例） 
<!-- テキストを日本語に変更し、サイズクラスを追加 -->
<button type="button" class="btn btn-primary btn-lg">送信する</button>
<button type="button" class="btn btn-secondary">キャンセル</button>

POINT

Bootstrapの公式ドキュメントは、プロの現場でも日常的に参照されています。
全てを暗記する必要はありません。「こんな機能があったな」と覚えておき、
必要になったらドキュメントで調べる、という使い方が実務のスタイルです。
ブックマークに登録しておきましょう。

検索機能を使いこなそう

公式サイトの上部にある検索バーに英語のキーワードを入力すると、
関連するコンポーネントやクラスがすぐに見つかります。

調べたいこと 検索キーワード例
ボタンの種類を知りたい button
余白の調整方法 spacing
テーブルのデザイン table
カードのレイアウト card
