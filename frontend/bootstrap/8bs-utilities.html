Introduction

ユーティリティクラスとは？

Bootstrapのユーティリティクラスとは、1つのCSSプロパティに対応する小さなクラスです。「余白を追加」「要素を非表示にする」「角を丸くする」といったスタイリングを、CSSファイルを編集せずにHTMLのクラス名だけで実現できます。

ユーティリティクラスの考え方は近年のフロントエンド開発で広く採用されています（Tailwind CSSも同じ考え方）。この章では、特に実務でよく使うユーティリティクラスを体系的に学びます。

  • Spacing — マージンとパディングの制御
  • Display と Flexbox — 表示と配置の制御
  • サイズ・ボーダー・シャドウ — 幅、枠線、影の制御
  • Position と Overflow — 位置指定とはみ出し制御
Section 8-1

Spacing（余白）

Spacingユーティリティの命名規則

Bootstrapの余白クラスは {property}{sides}-{size} という命名規則に従っています。この規則を覚えれば、あらゆる余白パターンを自在に指定できます。

部分 値 意味
property m margin（外側の余白）
p padding（内側の余白）
sides t top（上）
b bottom（下）
s start（左 ※LTR言語の場合）
e end（右 ※LTR言語の場合）
x 左右の両方
y 上下の両方
（なし） 上下左右すべて
size 0 0（余白なし）
1 0.25rem（4px）
2 0.5rem（8px）
3 1rem（16px）
4 1.5rem（24px）
5 3rem（48px）
auto 自動（marginのみ）

sとeについて

Bootstrap 5では、l（left）と r（right）の代わりに s（start）と e（end）が使われています。これは、右から左に読む言語（アラビア語など）にも対応するためです。日本語やEnglishでは s = 左、e = 右と考えて問題ありません。

よく使うSpacingパターン

実務で頻繁に使うSpacingクラスの具体例を見てみましょう。

HTML
<!-- 上マージン（要素間の間隔） -->
<div class="mt-3">上に16pxのマージン</div>

<!-- 下マージン（フォームグループの間隔） -->
<div class="mb-3">下に16pxのマージン</div>

<!-- 左右パディング（カードの内側余白） -->
<div class="px-4">左右に24pxのパディング</div>

<!-- 上下パディング（セクションの内側余白） -->
<div class="py-5">上下に48pxのパディング</div>

<!-- 全方向パディング -->
<div class="p-3">上下左右に16pxのパディング</div>

<!-- 水平中央寄せ（左右autoマージン） -->
<div class="mx-auto" style="width: 300px">中央に配置</div>

<!-- 下マージンauto（Flexboxで下に押し下げ） -->
<div class="mb-auto">Flexbox内で上に配置</div>
クラス例 意味 よくある使い方
mt-3 margin-top: 1rem 見出しの上に間隔を空ける
mb-3 margin-bottom: 1rem フォームの各項目の下に間隔
px-4 padding-left/right: 1.5rem カードやボックスの左右余白
py-5 padding-top/bottom: 3rem セクション間の広い余白
mx-auto margin-left/right: auto ブロック要素の水平中央寄せ
p-0 padding: 0 デフォルトの余白を消す

レスポンシブSpacing

Spacingクラスもブレイクポイントに応じた切り替えが可能です。{property}{sides}-{breakpoint}-{size} の形式で指定します。

HTML
<!-- スマホでは上マージン2、md以上では上マージン5 -->
<div class="mt-2 mt-md-5">
  レスポンシブマージン
</div>

<!-- スマホではパディング2、lg以上ではパディング5 -->
<section class="p-2 p-lg-5">
  レスポンシブパディング
</section>

<!-- スマホでは下マージン4、md以上では下マージン0 -->
<div class="mb-4 mb-md-0">
  スマホでのみ下に間隔
</div>

モバイルファーストの考え方

Bootstrapのレスポンシブユーティリティは「モバイルファースト」です。mt-2 はすべての画面サイズに適用され、mt-md-5 はmd（768px）以上の画面でのみ上書きされます。まずモバイルのスタイルを書き、大きい画面向けに追加するのが正しい順番です。

Section 8-2

Display と Flexbox

Displayユーティリティ

CSSの display プロパティを制御するクラスです。要素の表示/非表示の切り替えや、レイアウト方式の変更に使います。

クラス CSSの値 効果
d-none display: none 要素を非表示にする
d-block display: block ブロック要素として表示
d-inline display: inline インライン要素として表示
d-inline-block display: inline-block インラインブロックとして表示
d-flex display: flex Flexboxコンテナにする
d-inline-flex display: inline-flex インラインFlexboxにする
d-grid display: grid Gridコンテナにする

レスポンシブ表示切り替え

Displayユーティリティにブレイクポイントを組み合わせることで、画面サイズに応じた表示/非表示を実現できます。これはレスポンシブデザインで非常に重要なテクニックです。

HTML
<!-- スマホでは非表示、md以上で表示 -->
<div class="d-none d-md-block">
  タブレット・PC向けコンテンツ
</div>

<!-- md以上では非表示（スマホのみ表示） -->
<div class="d-md-none">
  スマホ向けコンテンツ
</div>

<!-- スマホでは非表示、lg以上でFlex表示 -->
<div class="d-none d-lg-flex">
  PC向けFlexレイアウト
</div>
クラスの組み合わせ 意味
d-none d-md-block md未満で非表示、md以上でblock表示
d-md-none md以上で非表示（モバイルでは表示）
d-none d-sm-block d-md-none sm〜md の間だけ表示
d-block d-lg-none lg未満で表示、lg以上で非表示

表示の出し分けの仕組み

考え方のポイントは「まず d-none で消して、必要なブレイクポイントで d-{bp}-block で表示する」です。逆に「まず表示して、不要なブレイクポイントで d-{bp}-none で消す」パターンもあります。

Flexboxユーティリティ

d-flex でFlexboxコンテナにした要素に対して、配置や揃えを制御するクラスが豊富に用意されています。

カテゴリ クラス例 効果
主軸の配置 justify-content-start 左寄せ（デフォルト）
justify-content-center 中央寄せ
justify-content-end 右寄せ
justify-content-between 両端揃え（均等配置）
交差軸の配置 align-items-start 上揃え
align-items-center 垂直中央揃え
align-items-end 下揃え
方向 flex-row 横並び（デフォルト）
flex-column 縦並び
折り返し flex-wrap 幅が足りないとき折り返す
HTML
<!-- 水平中央 + 垂直中央（ど真ん中配置） -->
<div class="d-flex justify-content-center align-items-center" style="height: 200px; background: #f0f0f0">
  <p>完全に中央に配置された要素</p>
</div>

<!-- 両端揃え（左にテキスト、右にボタン） -->
<div class="d-flex justify-content-between align-items-center">
  <h5 class="mb-0">記事タイトル</h5>
  <button class="btn btn-sm btn-primary">編集</button>
</div>

<!-- 縦並び + 中央寄せ -->
<div class="d-flex flex-column align-items-center">
  <img src="avatar.jpg" alt="アバター" class="rounded-circle mb-2">
  <h5>山田太郎</h5>
  <p class="text-muted">フロントエンドエンジニア</p>
</div>

gap ユーティリティ

d-flex や d-grid と組み合わせて、子要素間のスペースを gap-{size} で簡単に設定できます。

HTML
<!-- Flexbox + gap でボタンを等間隔に配置 -->
<div class="d-flex gap-2">
  <button class="btn btn-primary">保存</button>
  <button class="btn btn-secondary">キャンセル</button>
  <button class="btn btn-danger">削除</button>
</div>

<!-- gap-3 で広めの間隔 -->
<div class="d-flex flex-wrap gap-3">
  <div class="p-3 bg-light border">カード1</div>
  <div class="p-3 bg-light border">カード2</div>
  <div class="p-3 bg-light border">カード3</div>
  <div class="p-3 bg-light border">カード4</div>
</div>

gapとmarginの使い分け

gap は子要素間の間隔を均等に設定するのに最適です。個別の要素の余白を調整するには m-* を使いましょう。Flexboxの子要素にマージンを付けるよりも、親要素に gap を設定するほうがシンプルで管理しやすいです。

実践例：レスポンシブナビゲーション

Display と Flexbox を組み合わせた実践的な例を見てみましょう。

HTML
<!-- ヘッダー：両端揃え + 垂直中央 -->
<div class="d-flex justify-content-between align-items-center p-3 bg-white shadow-sm">
  <h4 class="mb-0">マイダッシュボード</h4>

  <!-- PC向け：横並びリンク -->
  <nav class="d-none d-md-flex gap-3">
    <a href="#" class="text-decoration-none">ホーム</a>
    <a href="#" class="text-decoration-none">設定</a>
    <a href="#" class="text-decoration-none">ログアウト</a>
  </nav>

  <!-- スマホ向け：ハンバーガーアイコン -->
  <button class="btn btn-outline-secondary d-md-none">
    メニュー
  </button>
</div>
Section 8-3

サイズ・ボーダー・シャドウ

サイズユーティリティ

要素の幅と高さを制御するクラスです。パーセンテージベースで指定します。

クラス CSSの値 説明
w-25 width: 25% 親要素の25%幅
w-50 width: 50% 親要素の50%幅
w-75 width: 75% 親要素の75%幅
w-100 width: 100% 親要素の100%幅
w-auto width: auto 自動幅
h-25 height: 25% 親要素の25%高さ
h-50 height: 50% 親要素の50%高さ
h-100 height: 100% 親要素の100%高さ
h-auto height: auto 自動高さ
mw-100 max-width: 100% 最大幅を100%に制限
vh-100 height: 100vh ビューポート全体の高さ
HTML
<!-- 幅50%のボックス -->
<div class="w-50 p-3 bg-light">
  親要素の半分の幅
</div>

<!-- 幅100%のボタン -->
<button class="btn btn-primary w-100">
  幅いっぱいのボタン
</button>

<!-- ビューポート全体の高さを持つヒーロー -->
<div class="vh-100 d-flex justify-content-center align-items-center">
  <h1>フルスクリーンヒーロー</h1>
</div>

ボーダーユーティリティ

要素にボーダー（枠線）を追加・削除したり、角を丸くしたりするクラスです。

HTML
<!-- ボーダーの追加 -->
<div class="border p-3">全方向のボーダー</div>
<div class="border-top p-3">上だけボーダー</div>
<div class="border-bottom p-3">下だけボーダー</div>
<div class="border-start p-3">左だけボーダー</div>
<div class="border-end p-3">右だけボーダー</div>

<!-- ボーダーの削除 -->
<div class="border border-0">ボーダーなし</div>
<div class="border border-top-0">上のボーダーだけ削除</div>

<!-- ボーダーの色 -->
<div class="border border-primary p-3">青いボーダー</div>
<div class="border border-danger p-3">赤いボーダー</div>
<div class="border border-success p-3">緑のボーダー</div>

角丸（Border Radius）

要素の角を丸くするクラスも豊富に用意されています。

クラス 効果 よくある用途
rounded 小さい角丸（0.375rem） カード、ボックス
rounded-0 角丸なし 角丸を消す
rounded-1 微かな角丸（0.25rem） 控えめな角丸
rounded-2 小さい角丸（0.375rem） 標準的な角丸
rounded-3 中程度の角丸（0.5rem） やや大きい角丸
rounded-circle 円形（50%） アバター画像
rounded-pill 楕円形（50rem） ピル型のバッジやボタン
HTML
<!-- 円形のアバター -->
<img src="avatar.jpg" alt="ユーザー" class="rounded-circle" width="80" height="80">

<!-- ピル型のタグ -->
<span class="badge bg-primary rounded-pill">新着</span>

<!-- 角丸ボックス -->
<div class="border rounded-3 p-3">角丸のボックス</div>

シャドウユーティリティ

要素に影（シャドウ）を付けることで、奥行きや浮き上がり感を表現できます。

クラス 効果 よくある用途
shadow-none シャドウなし 既存のシャドウを消す
shadow-sm 小さいシャドウ カード、ナビバー（控えめ）
shadow 通常のシャドウ カード、モーダル
shadow-lg 大きいシャドウ ドロップダウン、ポップアップ
HTML
<div class="shadow-sm p-4 mb-4 bg-white rounded">
  小さいシャドウ（shadow-sm）
</div>

<div class="shadow p-4 mb-4 bg-white rounded">
  通常のシャドウ（shadow）
</div>

<div class="shadow-lg p-4 mb-4 bg-white rounded">
  大きいシャドウ（shadow-lg）
</div>

Opacityユーティリティ

要素の不透明度を制御するクラスです。

HTML
<div class="opacity-100">opacity-100（完全に不透明）</div>
<div class="opacity-75">opacity-75（75%不透明）</div>
<div class="opacity-50">opacity-50（50%不透明）</div>
<div class="opacity-25">opacity-25（25%不透明）</div>

opacityの使いどころ

不透明度は主に「無効状態の表現」「背景画像のオーバーレイ」「ホバー時のエフェクト」などで使います。ただし使いすぎると読みにくくなるため、控えめに使いましょう。

Section 8-4

Position と Overflow

Positionユーティリティ

CSSの position プロパティを制御するクラスです。

クラス CSSの値 説明
position-static position: static デフォルトの配置（通常フロー）
position-relative position: relative 通常位置を基準にした相対配置
position-absolute position: absolute 親要素を基準にした絶対配置
position-fixed position: fixed 画面に対する固定配置
position-sticky position: sticky スクロール到達後に固定

位置指定クラス

position-absolute や position-fixed と組み合わせて、要素の正確な位置を指定するクラスが用意されています。

クラス 効果
top-0 top: 0（上端に配置）
top-50 top: 50%（上から50%の位置）
top-100 top: 100%（下端に配置）
start-0 left: 0（左端に配置）
start-50 left: 50%（左から50%の位置）
start-100 left: 100%（右端に配置）
end-0 right: 0（右端に配置）
bottom-0 bottom: 0（下端に配置）
translate-middle transform: translate(-50%, -50%)で中心に移動
HTML
<!-- 右上に配置 -->
<div class="position-relative" style="height: 150px; background: #f0f0f0">
  <span class="position-absolute top-0 end-0 badge bg-danger">
    NEW
  </span>
</div>

<!-- 完全中央に配置 -->
<div class="position-relative" style="height: 150px; background: #f0f0f0">
  <span class="position-absolute top-50 start-50 translate-middle badge bg-primary rounded-pill">
    中央
  </span>
</div>

<!-- 左下に配置 -->
<div class="position-relative" style="height: 150px; background: #f0f0f0">
  <span class="position-absolute bottom-0 start-0 p-2 bg-dark text-white">
    左下テキスト
  </span>
</div>

position-relativeとposition-absoluteの関係

position-absolute の要素は、最も近い position-relative の親要素を基準に配置されます。そのため、絶対配置を使うときは必ず親要素に position-relative を付けましょう。付け忘れると、画面全体を基準に配置されてしまいます。

Overflowユーティリティ

要素の内容がはみ出したときの動作を制御します。

クラス CSSの値 効果
overflow-auto overflow: auto はみ出し時にスクロールバーを表示
overflow-hidden overflow: hidden はみ出した部分を非表示にする
overflow-visible overflow: visible はみ出しをそのまま表示（デフォルト）
overflow-scroll overflow: scroll 常にスクロールバーを表示
HTML
<!-- スクロール可能な固定高さのボックス -->
<div class="overflow-auto border rounded p-3" style="height: 200px">
  <p>ここに長いテキストが入ります...</p>
  <p>高さが200pxを超えるとスクロールバーが表示されます。</p>
  <p>チャットのメッセージ一覧や、ログの表示などに便利です。</p>
  <p>もっとたくさんのテキスト...</p>
  <p>さらにテキスト...</p>
</div>

<!-- はみ出しを隠す（画像のトリミングなど） -->
<div class="overflow-hidden rounded-circle" style="width: 100px; height: 100px">
  <img src="photo.jpg" alt="写真" class="w-100">
</div>

実践例：通知バッジ付きアイコン

Position ユーティリティの最も実用的な例として、ベルアイコンの右上に未読数のバッジを配置するパターンを紹介します。

HTML
<!-- 通知バッジ付きボタン -->
<button type="button" class="btn btn-primary position-relative">
  受信トレイ
  <span class="position-absolute top-0 start-100 translate-middle badge rounded-pill bg-danger">
    99+
    <span class="visually-hidden">未読メッセージ</span>
  </span>
</button>

<!-- 丸い通知ドット -->
<button type="button" class="btn btn-outline-secondary position-relative">
  通知
  <span class="position-absolute top-0 start-100 translate-middle p-2 bg-danger border border-light rounded-circle">
    <span class="visually-hidden">新しい通知</span>
  </span>
</button>

このコードでは次のような仕組みで位置を制御しています。

クラス この例での役割
position-relative（ボタンに） バッジの配置基準を設定
position-absolute（バッジに） ボタンを基準にした絶対配置
top-0 ボタンの上端に配置
start-100 ボタンの右端に配置
translate-middle バッジの中心がボタンの右上角に来るように調整

translate-middleの威力

top-0 start-100 だけでは、バッジの左上角がボタンの右上角に来てしまいます。translate-middle を追加することで、バッジの中心がその位置に来るように自動調整されます。これにより、バッジがボタンの右上角にきれいに重なります。
