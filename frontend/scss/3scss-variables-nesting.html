Section 3-1

変数（$variables）

$変数の宣言と使用

SCSSでは、$（ドルマーク）を先頭に付けて変数を宣言します。
プログラミング言語の変数と同じように、値に名前を付けて管理することができます。

SCSS - 変数の基本 
// 変数の宣言（$変数名: 値;）
$primary-color: #CF649A;
$font-size-base: 16px;
$spacing: 20px;

// 変数の使用
.button {
  background-color: $primary-color;
  font-size: $font-size-base;
  padding: $spacing;
}
CSS（コンパイル結果） 
.button {
  background-color: #CF649A;
  font-size: 16px;
  padding: 20px;
}

コンパイルされると、変数は実際の値に置き換えられます。
変数はコンパイル時に処理されるため、出力されるCSSに変数は残りません。

よく変数化する値

実務では、以下のような値を変数として定義するのが一般的です。

SCSS - 実務でよく使う変数 
// --- カラー ---
$primary: #CF649A;
$secondary: #6C757D;
$success: #28A745;
$danger: #DC3545;
$warning: #FFC107;
$text-color: #333333;
$text-light: #666666;
$bg-color: #F5F5F5;
$white: #FFFFFF;

// --- フォントサイズ ---
$font-size-sm: 12px;
$font-size-base: 16px;
$font-size-lg: 20px;
$font-size-xl: 24px;

// --- スペーシング ---
$spacing-xs: 4px;
$spacing-sm: 8px;
$spacing-md: 16px;
$spacing-lg: 24px;
$spacing-xl: 32px;

// --- ボーダー ---
$border-radius: 4px;
$border-radius-lg: 8px;
$border-color: #E0E0E0;

// --- フォントファミリー ---
$font-family-base: "Noto Sans JP", "Hiragino Sans", sans-serif;
$font-family-mono: "JetBrains Mono", monospace;
命名のコツ

変数名は「役割」で名付けるのがベストプラクティスです。
良い例：$primary-color, $text-color, $spacing-lg
避けたい例：$pink, $gray333, $twenty（具体的な値を名前にすると、値を変更したときに名前と合わなくなります）

変数のスコープ

SCSSの変数にはグローバルスコープとローカルスコープがあります。

SCSS - スコープの例 
// グローバル変数（どこからでも使える）
$color: blue;

.header {
  // ローカル変数（この { } の中でのみ有効）
  $color: red;
  color: $color;  // → red
}

.footer {
  color: $color;  // → blue（グローバル変数が使われる）
}
スコープ 宣言場所 使える範囲
グローバル セレクタの外（ファイルの直下） ファイル全体
ローカル セレクタの { } の中 そのセレクタの中だけ

SCSS変数とCSS変数（カスタムプロパティ）の違い

CSSにも「CSS変数（カスタムプロパティ）」という仕組みがあります。
SCSS変数とCSS変数は似ているようで、実は大きく異なります。

SCSS変数 
// SCSS変数（コンパイル時に値に置換される）
$primary: #CF649A;

.button {
  background-color: $primary;
}
CSS変数（カスタムプロパティ） 
/* CSS変数（ブラウザ上で動的に参照される） */
:root {
  --primary: #CF649A;
}

.button {
  background-color: var(--primary);
}
SCSS変数（$variable） CSS変数（--variable）
処理タイミング コンパイル時（ビルド時） 実行時（ブラウザ上）
出力CSS 実際の値に置換される（変数は残らない） var(--name) のまま残る
JavaScript操作 できない できる（動的に変更可能）
演算 可能（コンパイル時に計算） calc() を使えば可能
カスケード なし（単純な値の置換） あり（セレクタに応じて値が変わる）
使い分け 固定値の管理に最適 テーマ切り替え等の動的変更に最適
使い分けのポイント

SCSS変数とCSS変数は併用できます。例えば、SCSS変数で定義した値をCSS変数に代入するというパターンもよく使われます。
:root { --primary: #{$primary}; } のように書くことで、両方の利点を活かせます。

Section 3-2

ネスト（Nesting）

基本的なネスト記法

SCSSでは、セレクタを入れ子（ネスト）にして書くことができます。
HTMLの構造に合わせてCSSを整理できるため、コードの見通しが良くなります。

HTML 
<nav class="nav">
  <ul class="nav-list">
    <li class="nav-item">
      <a href="#">ホーム</a>
    </li>
    <li class="nav-item">
      <a href="#">サービス</a>
    </li>
  </ul>
</nav>
SCSS - ネストを使った書き方 
.nav {
  background-color: #333;
  padding: 10px;

  .nav-list {
    list-style: none;
    margin: 0;
    padding: 0;
    display: flex;
  }

  .nav-item {
    margin-right: 20px;

    a {
      color: white;
      text-decoration: none;
      font-size: 14px;
    }
  }
}

コンパイル結果

ネストされたSCSSがどのようなCSSに変換されるか確認しましょう。

CSS（コンパイル結果） 
.nav {
  background-color: #333;
  padding: 10px;
}
.nav .nav-list {
  list-style: none;
  margin: 0;
  padding: 0;
  display: flex;
}
.nav .nav-item {
  margin-right: 20px;
}
.nav .nav-item a {
  color: white;
  text-decoration: none;
  font-size: 14px;
}

SCSSのネストは、コンパイル時に親セレクタを連結した通常のCSSセレクタに展開されます。
.nav { .nav-item { a { } } } は .nav .nav-item a になります。

ネストの深さに注意

ネストは便利ですが、深くしすぎると問題が発生します。

SCSS - 悪い例：深すぎるネスト 
// 悪い例：5階層のネスト
.page {
  .main {
    .content {
      .article {
        .text {
          color: #333;  // → .page .main .content .article .text
        }
      }
    }
  }
}
SCSS - 良い例：3階層以内 
// 良い例：3階層以内に収める
.article {
  padding: 20px;

  .text {
    color: #333;
    line-height: 1.8;
  }
}
ネストは3階層までが推奨

ネストが深くなりすぎると、以下の問題が生じます。
1. 詳細度が高くなりすぎる：CSSの優先度が不必要に高くなり、スタイルの上書きが困難になります。
2. 可読性の低下：どのセレクタがどの要素に対応するか分かりにくくなります。
3. パフォーマンス：セレクタが長いとブラウザの処理がわずかに遅くなります。
目安として3階層以内に収めるようにしましょう。

Section 3-3

親セレクタ参照（&）

&の基本的な使い方

SCSSでは &（アンパサンド）を使って、親セレクタを参照できます。
これはネストの中で非常によく使う重要な機能です。

SCSS - &の基本 
.button {
  background-color: #CF649A;
  color: white;
  padding: 10px 20px;
  border: none;
  cursor: pointer;

  // & = 親セレクタ（.button）を参照
  &:hover {
    background-color: #B5507F;
  }

  &:active {
    background-color: #9A3D6A;
  }

  &:focus {
    outline: 2px solid #CF649A;
    outline-offset: 2px;
  }
}
CSS（コンパイル結果） 
.button {
  background-color: #CF649A;
  color: white;
  padding: 10px 20px;
  border: none;
  cursor: pointer;
}
.button:hover {
  background-color: #B5507F;
}
.button:active {
  background-color: #9A3D6A;
}
.button:focus {
  outline: 2px solid #CF649A;
  outline-offset: 2px;
}

& は「ネストの外側にある親セレクタそのもの」に置き換えられます。
&:hover は .button:hover になります（間にスペースは入りません）。

&あり と &なし の違い

&:hover → .button:hover（スペースなしで直結。同じ要素の擬似クラス）
:hover（&なし） → .button :hover（スペースが入る。子孫要素の擬似クラス）
擬似クラス（:hover, :focus等）には必ず & を付けてください。

BEM記法での&の活用

BEM（Block Element Modifier）はCSS設計手法の1つで、
.block__element--modifier の形式でクラス名を付ける方法です。
SCSSの & はBEM記法と非常に相性が良いです。

SCSS - BEM記法での&の活用 
// Block: card
.card {
  background: white;
  border-radius: 8px;
  padding: 20px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);

  // Element: card__title
  &__title {
    font-size: 20px;
    font-weight: bold;
    margin-bottom: 12px;
  }

  // Element: card__text
  &__text {
    color: #666;
    line-height: 1.6;
  }

  // Element: card__button
  &__button {
    background-color: #CF649A;
    color: white;
    padding: 8px 16px;
    border: none;
    border-radius: 4px;

    &:hover {
      background-color: #B5507F;
    }
  }

  // Modifier: card--featured
  &--featured {
    border: 2px solid #CF649A;
  }

  // Modifier: card--large
  &--large {
    padding: 32px;
  }
}
CSS（コンパイル結果） 
.card {
  background: white;
  border-radius: 8px;
  padding: 20px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}
.card__title {
  font-size: 20px;
  font-weight: bold;
  margin-bottom: 12px;
}
.card__text {
  color: #666;
  line-height: 1.6;
}
.card__button {
  background-color: #CF649A;
  color: white;
  padding: 8px 16px;
  border: none;
  border-radius: 4px;
}
.card__button:hover {
  background-color: #B5507F;
}
.card--featured {
  border: 2px solid #CF649A;
}
.card--large {
  padding: 32px;
}

&__title は .card__title に展開されます。
& が親セレクタ .card に置換され、そのまま __title が連結される仕組みです。

メディアクエリでの&の活用

SCSSでは、メディアクエリもセレクタの中にネストできます。
これにより、レスポンシブ対応のコードが非常に見やすくなります。

SCSS - メディアクエリのネスト 
.container {
  width: 100%;
  padding: 16px;

  // タブレット以上
  @media (min-width: 768px) {
    max-width: 720px;
    margin: 0 auto;
    padding: 24px;
  }

  // デスクトップ以上
  @media (min-width: 1024px) {
    max-width: 960px;
    padding: 32px;
  }
}
CSS（コンパイル結果） 
.container {
  width: 100%;
  padding: 16px;
}
@media (min-width: 768px) {
  .container {
    max-width: 720px;
    margin: 0 auto;
    padding: 24px;
  }
}
@media (min-width: 1024px) {
  .container {
    max-width: 960px;
    padding: 32px;
  }
}
メディアクエリのネストのメリット

通常のCSSでは、メディアクエリをファイルの末尾にまとめて書くことが多く、同じセレクタのスタイルが分散してしまいます。SCSSではセレクタの中にメディアクエリをネストできるため、1つの要素に関するすべてのスタイルを1箇所にまとめて管理できます。

Section 3-4

変数とネストの実践例

ナビゲーションバーの実装

実践的な例として、ナビゲーションバーをSCSSで実装してみましょう。
変数、ネスト、& のすべてを使います。

SCSS - ナビゲーションバー 
// 変数の定義
$nav-bg: #1a1a2e;
$nav-text: #e0e0e0;
$nav-hover: #CF649A;
$nav-height: 60px;
$nav-padding: 0 24px;

.navbar {
  background-color: $nav-bg;
  height: $nav-height;
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: $nav-padding;

  // ロゴ
  &__logo {
    font-size: 20px;
    font-weight: bold;
    color: white;
    text-decoration: none;

    &:hover {
      color: $nav-hover;
    }
  }

  // メニューリスト
  &__menu {
    display: flex;
    list-style: none;
    margin: 0;
    padding: 0;
    gap: 24px;
  }

  // メニュー項目
  &__link {
    color: $nav-text;
    text-decoration: none;
    font-size: 14px;
    padding: 8px 0;
    border-bottom: 2px solid transparent;
    transition: all 0.3s ease;

    &:hover {
      color: $nav-hover;
      border-bottom-color: $nav-hover;
    }

    &--active {
      color: white;
      border-bottom-color: $nav-hover;
    }
  }

  // レスポンシブ対応
  @media (max-width: 768px) {
    padding: 0 16px;

    &__menu {
      gap: 16px;
    }

    &__link {
      font-size: 12px;
    }
  }
}
CSS（コンパイル結果） 
.navbar {
  background-color: #1a1a2e;
  height: 60px;
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 0 24px;
}
.navbar__logo {
  font-size: 20px;
  font-weight: bold;
  color: white;
  text-decoration: none;
}
.navbar__logo:hover {
  color: #CF649A;
}
.navbar__menu {
  display: flex;
  list-style: none;
  margin: 0;
  padding: 0;
  gap: 24px;
}
.navbar__link {
  color: #e0e0e0;
  text-decoration: none;
  font-size: 14px;
  padding: 8px 0;
  border-bottom: 2px solid transparent;
  transition: all 0.3s ease;
}
.navbar__link:hover {
  color: #CF649A;
  border-bottom-color: #CF649A;
}
.navbar__link--active {
  color: white;
  border-bottom-color: #CF649A;
}
@media (max-width: 768px) {
  .navbar {
    padding: 0 16px;
  }
  .navbar__menu {
    gap: 16px;
  }
  .navbar__link {
    font-size: 12px;
  }
}

ボタンコンポーネントの実装

バリエーションのあるボタンコンポーネントを、変数とネストを活用して実装します。

SCSS - ボタンコンポーネント 
// カラー変数
$btn-primary: #CF649A;
$btn-secondary: #6C757D;
$btn-success: #28A745;
$btn-danger: #DC3545;
$btn-radius: 6px;

.btn {
  display: inline-block;
  padding: 10px 24px;
  font-size: 14px;
  font-weight: 500;
  text-align: center;
  text-decoration: none;
  border: none;
  border-radius: $btn-radius;
  cursor: pointer;
  transition: background-color 0.2s ease;

  // プライマリーボタン
  &--primary {
    background-color: $btn-primary;
    color: white;

    &:hover {
      background-color: darken($btn-primary, 10%);
    }
  }

  // セカンダリーボタン
  &--secondary {
    background-color: $btn-secondary;
    color: white;

    &:hover {
      background-color: darken($btn-secondary, 10%);
    }
  }

  // サクセスボタン
  &--success {
    background-color: $btn-success;
    color: white;

    &:hover {
      background-color: darken($btn-success, 10%);
    }
  }

  // サイズバリエーション
  &--small {
    padding: 6px 16px;
    font-size: 12px;
  }

  &--large {
    padding: 14px 32px;
    font-size: 16px;
  }

  // 無効状態
  &:disabled {
    opacity: 0.5;
    cursor: not-allowed;
  }
}
darken() 関数について

上のコードで使われている darken($color, $amount) はSCSSの組み込み関数です。指定した色を暗くすることができます。これにより、hover時の色を手動で指定する必要がなくなります。関数の詳細は第7章で学びます。

まとめ

第3章のまとめ

  • $変数を使うと、色・サイズ・フォントなどの値を一元管理でき、変更が1箇所で済む
  • 変数名は具体的な値（$pinkなど）ではなく、役割（$primary-colorなど）で命名するのがベスト
  • SCSS変数はコンパイル時に値に置換され、CSS変数はブラウザ上で動的に参照される
  • ネストを使うと、HTMLの構造に合わせてCSSを階層的に書ける
  • ネストは3階層以内に収めるのが推奨。深すぎると詳細度の問題が発生する
  • &（アンパサンド）で親セレクタを参照でき、:hoverや:focusなどの擬似クラスに必須
  • &はBEM記法（&__element, &--modifier）と相性が非常に良い
  • メディアクエリもセレクタ内にネストでき、レスポンシブ対応のコードを1箇所にまとめられる
