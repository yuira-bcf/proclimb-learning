Section 1-1

CSSプリプロセッサとは

プリプロセッサの概念

CSSプリプロセッサとは、CSSを拡張した言語で書いたコードを、通常のCSSに変換（コンパイル）する仕組みのことです。
「プリ（pre = 前もって）」＋「プロセッサ（processor = 処理装置）」という名前の通り、
ブラウザが読み込む前の段階で処理を行い、最終的にCSSファイルを生成します。

基本的な流れは以下の通りです。

コンパイルの流れ 
SCSSファイル（.scss）
    ↓  コンパイル（自動変換）
CSSファイル（.css）
    ↓  ブラウザが読み込み
Webページに反映

つまり、開発者はSCSSで効率よくスタイルを書き、
コンパイラが自動的に通常のCSSに変換してくれるという仕組みです。
ブラウザはSCSSを直接理解できないため、必ずCSSへの変換が必要になります。

なぜプリプロセッサが必要なのか

CSSは非常に強力なスタイリング言語ですが、大規模なプロジェクトになると次のような課題が出てきます。

CSSの課題 具体的な問題 プリプロセッサの解決策
値の繰り返し 同じ色コードやサイズを何十箇所にも書く必要がある 変数（$variable）で一元管理
セレクタの冗長さ 親子関係のセレクタを毎回フルパスで書く必要がある ネスト記法で階層構造を表現
再利用の困難さ 同じパターンのスタイルをコピー&ペーストで使い回す ミックスイン（@mixin）で再利用可能に
ファイル管理の煩雑さ 1つの巨大CSSファイルが管理しづらい パーシャル（ファイル分割）で整理
ポイント

プリプロセッサは「CSSをより効率的に書くための拡張ツール」です。最終的に出力されるのは通常のCSSなので、ブラウザ側の互換性を心配する必要はありません。

代表的なCSSプリプロセッサ

CSSプリプロセッサにはいくつかの種類がありますが、代表的なものは以下の3つです。

プリプロセッサ 拡張子 特徴 シェア
Sass / SCSS .sass / .scss 最も普及。機能が豊富でコミュニティが大きい 最も高い
Less .less Bootstrap 3で採用されていた。JavaScriptベース 中程度
Stylus .styl 柔軟な記法。Node.jsベース 低い

現在の業界ではSass（SCSS記法）が圧倒的なシェアを持っており、
多くのフレームワークやプロジェクトで採用されています。
このチュートリアルでは、最も広く使われているSCSSについて学んでいきます。

Section 1-2

SassとSCSSの違い

2つの構文

「Sass」というプリプロセッサには、実は2つの書き方（構文）が存在します。
よく混同されますが、この2つの違いを正確に理解しておきましょう。

Sass（インデント構文） SCSS（Sassy CSS）
拡張子 .sass .scss
波括弧 { } 不要（インデントで表現） 必要（CSSと同じ）
セミコロン ; 不要 必要（CSSと同じ）
CSS互換性 なし あり（CSSをそのまま書ける）
登場時期 2006年（先に登場） 2010年（後から追加）

コードの比較

同じスタイルを、Sass構文とSCSS構文で書くとこのようになります。

Sass（.sass）- インデント構文 
$primary-color: #CF649A

.header
  background-color: $primary-color
  padding: 20px

  .title
    font-size: 24px
    color: white
SCSS（.scss）- CSS互換構文 
$primary-color: #CF649A;

.header {
  background-color: $primary-color;
  padding: 20px;

  .title {
    font-size: 24px;
    color: white;
  }
}

どちらもコンパイルすると、まったく同じCSSが出力されます。

CSS（出力結果） 
.header {
  background-color: #CF649A;
  padding: 20px;
}
.header .title {
  font-size: 24px;
  color: white;
}

現在はSCSSが主流

現在のWeb開発では、SCSS構文が圧倒的に主流です。その理由は次の通りです。

check_circle

CSS互換性

既存のCSSコードをそのままSCSSファイルにコピーしても動作します。移行コストがゼロです。

check_circle

学習コストが低い

CSSを知っていれば、SCSSの基本構文は既に理解しているのと同じです。新しい機能だけ覚えればOKです。

check_circle

情報量が多い

ドキュメント、チュートリアル、Stack Overflowの回答など、SCSS記法での情報が最も多く見つかります。

注意

「Sass」という名前はプリプロセッサ全体の名称としても使われます。そのため、「Sass」と言っているがSCSS構文を指している場合が多くあります。本チュートリアルでは「SCSS」と表記して、SCSS構文（.scss）を使用していきます。

Section 1-3

SCSSのメリット

SCSSの4つの主要機能

SCSSには、CSSをより効率的に書くための強力な機能が備わっています。
ここでは代表的な4つの機能を紹介します（各機能の詳細は後の章で学びます）。

palette

変数（Variables）

色やサイズなどの値を変数として定義し、一元管理できます。値の変更が1箇所で済むため、保守性が大幅に向上します。

account_tree

ネスト（Nesting）

HTMLの構造に合わせてCSSセレクタを入れ子にできます。親子関係が視覚的に分かりやすくなり、コードの可読性が向上します。

recycling

ミックスイン（Mixins）

よく使うスタイルのパターンをテンプレート化し、何度でも再利用できます。引数を渡してカスタマイズすることも可能です。

folder_open

ファイル分割（Partials）

スタイルを複数のファイルに分割して管理できます。コンパイル時に1つのCSSファイルにまとめられるため、保守性と可読性が両立します。

CSSとSCSSの比較

同じデザインを実現するために、CSSとSCSSでどれだけコードが違うか比較してみましょう。

CSS - 従来の書き方 
.card {
  background-color: #ffffff;
  border-radius: 8px;
  padding: 24px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

.card .card-title {
  font-size: 20px;
  color: #333333;
  margin-bottom: 12px;
}

.card .card-text {
  font-size: 14px;
  color: #666666;
  line-height: 1.6;
}

.card .card-button {
  background-color: #CF649A;
  color: #ffffff;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

.card .card-button:hover {
  background-color: #B5507F;
}
SCSS - 効率的な書き方 
// 変数で色を一元管理
$white: #ffffff;
$text-dark: #333333;
$text-light: #666666;
$primary: #CF649A;
$primary-dark: #B5507F;

.card {
  background-color: $white;
  border-radius: 8px;
  padding: 24px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);

  // ネストで親子関係を表現
  .card-title {
    font-size: 20px;
    color: $text-dark;
    margin-bottom: 12px;
  }

  .card-text {
    font-size: 14px;
    color: $text-light;
    line-height: 1.6;
  }

  .card-button {
    background-color: $primary;
    color: $white;
    padding: 10px 20px;
    border: none;
    border-radius: 4px;
    cursor: pointer;

    &:hover {
      background-color: $primary-dark;
    }
  }
}
比較のポイント

SCSSでは変数で色を一元管理し、ネストでHTMLの構造を反映した見やすいコードを書けます。
もし$primaryの色を変更したい場合、SCSSなら変数の値を1箇所変えるだけで済みますが、CSSでは使われている全箇所を手動で修正する必要があります。

Section 1-4

SCSSの公式ドキュメント

公式サイト

Sass/SCSSの公式サイトは以下のURLです。英語で書かれていますが、学習を進める上で非常に重要なリソースです。

公式サイト 
https://sass-lang.com

主要なドキュメントページ

ページ URL 内容
ドキュメント トップ sass-lang.com/documentation 全機能のリファレンス
Variables sass-lang.com/documentation/variables 変数の使い方
Nesting sass-lang.com/documentation/style-rules ネストなどのスタイルルール
Mixins sass-lang.com/documentation/at-rules/mixin ミックスインの定義と使用
Modules (@use) sass-lang.com/documentation/at-rules/use モジュールシステム
Install sass-lang.com/install インストール方法
Playground sass-lang.com/playground ブラウザ上でSCSSを試せる
Playgroundを活用しよう

公式サイトの「Playground」では、ブラウザ上でSCSSコードを入力し、リアルタイムでCSSへの変換結果を確認できます。環境構築なしでSCSSを試せるため、学習の際にぜひ活用してください。

学習リソース

公式サイト以外にも、SCSSを学ぶ際に役立つリソースがあります。

リソース 種類 特徴
Sass公式ガイド ドキュメント sass-lang.com/guide で基本を網羅的に学べる
MDN Web Docs リファレンス CSSの基礎知識の確認に最適（日本語あり）
Sass Playground 実行環境 ブラウザ上でSCSSを試せる公式ツール
CodePen 実行環境 SCSSに対応したオンラインエディタ
注意：@import から @use への移行

古い記事やチュートリアルでは @import を使った書き方が紹介されていることがありますが、@import は現在非推奨（deprecated）です。新しいプロジェクトでは必ず @use を使用してください。本チュートリアルでも @use を使って解説していきます。

まとめ

第1章のまとめ

  • CSSプリプロセッサは、CSSを拡張した言語で書いたコードを通常のCSSに変換する仕組みである
  • SCSSは最も広く使われているCSSプリプロセッサで、CSS互換の構文を持つ
  • Sass（インデント構文）とSCSS（CSS互換構文）の2つの記法があり、現在はSCSSが主流である
  • SCSSの主要な機能は、変数・ネスト・ミックスイン・ファイル分割の4つ
  • SCSSで書いたコードはコンパイルしてCSSに変換する必要がある（ブラウザはSCSSを直接読めない）
  • 公式サイト（sass-lang.com）のPlaygroundでブラウザ上からSCSSを試すことができる
  • 古い記事の @import ではなく、新しい@useを使うこと
