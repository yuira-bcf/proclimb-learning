Section 17-1

AOPの基本概念

AOP（アスペクト指向プログラミング）で、横断的な関心事を分離します。

gps_fixed

@Aspect

アスペクト（横断的関心事）を定義するクラス

push_pin

Pointcut

どのメソッドにアスペクトを適用するかを定義

bolt

Advice

実際に実行される処理（Before, After, Around）

pom.xml（依存関係） 
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>

AOPの用語

用語説明
@Beforeメソッド実行前に処理
@Afterメソッド実行後に処理（成功/失敗に関わらず）
@AfterReturningメソッドが正常に終了した後に処理
@AfterThrowing例外が発生した場合に処理
@Aroundメソッドの前後を完全に制御
Section 17-2

ログ出力アスペクトの実装

LoggingAspect.java 
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.*;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.stereotype.Component;
import java.util.Arrays;

@Aspect
@Component
public class LoggingAspect {

    private static final Logger logger = LoggerFactory.getLogger(LoggingAspect.class);

    /**
     * Serviceクラスの全メソッドを対象
     */
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void serviceLayer() {}

    /**
     * メソッド実行前にログ出力
     */
    @Before("serviceLayer()")
    public void logBefore(JoinPoint joinPoint) {
        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();
        logger.info("【開始】{} 引数: {}", methodName, Arrays.toString(args));
    }

    /**
     * メソッド正常終了後にログ出力
     */
    @AfterReturning(pointcut = "serviceLayer()", returning = "result")
    public void logAfterReturning(JoinPoint joinPoint, Object result) {
        String methodName = joinPoint.getSignature().getName();
        logger.info("【終了】{} 戻り値: {}", methodName, result);
    }

    /**
     * 例外発生時にログ出力
     */
    @AfterThrowing(pointcut = "serviceLayer()", throwing = "ex")
    public void logAfterThrowing(JoinPoint joinPoint, Exception ex) {
        String methodName = joinPoint.getSignature().getName();
        logger.error("【例外】{} エラー: {}", methodName, ex.getMessage());
    }
}

Section 17-3

実行時間計測

PerformanceAspect.java 
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.stereotype.Component;

@Aspect
@Component
public class PerformanceAspect {

    private static final Logger logger = LoggerFactory.getLogger(PerformanceAspect.class);

    /**
     * @Around で実行時間を計測
     */
    @Around("execution(* com.example.service.*.*(..))")
    public Object measureExecutionTime(ProceedingJoinPoint joinPoint) throws Throwable {

        long startTime = System.currentTimeMillis();

        try {
            // 実際のメソッドを実行
            Object result = joinPoint.proceed();

            long endTime = System.currentTimeMillis();
            logger.info("{}() 実行時間: {}ms",
                joinPoint.getSignature().getName(),
                endTime - startTime);

            return result;

        } catch (Throwable ex) {
            long endTime = System.currentTimeMillis();
            logger.error("{}() 実行時間: {}ms (例外発生)",
                joinPoint.getSignature().getName(),
                endTime - startTime);
            throw ex;
        }
    }
}

warning注意点

AOPはプロキシベースで動作するため、同一クラス内でのメソッド呼び出しには適用されません。
this.method()ではなく、Springから注入されたBeanを経由する必要があります。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにAOP（アスペクト指向プログラミング）でログ出力機能を追加してください。

【要件】
- spring-boot-starter-aopの依存関係を追加
- LoggingAspectクラスを作成し、Serviceクラスの全メソッドを対象にログ出力
- @Before: メソッド実行前に「メソッド名」と「引数」をログ出力
- @AfterReturning: メソッド正常終了後に「メソッド名」と「戻り値」をログ出力
- @AfterThrowing: 例外発生時に「メソッド名」と「エラーメッセージ」をログ出力
- PerformanceAspectクラスを作成し、@Aroundで実行時間を計測

【技術スタック】
- Spring Boot 3.x
- Spring AOP
- SLF4J（ロギング）
gps_fixed ステップ別プロンプト 
# Step 1: 依存関係の追加
pom.xmlにspring-boot-starter-aopの依存関係を追加してください。

# Step 2: LoggingAspectの作成
@Aspect、@Componentを付与したLoggingAspectクラスを作成し、
Serviceクラスの全メソッドを対象にPointcutを定義してください。

# Step 3: Before/After Adviceの実装
@Before、@AfterReturning、@AfterThrowingアノテーションを使って、
メソッドの開始・終了・例外発生時にログを出力する処理を実装してください。

# Step 4: 実行時間計測Aspectの作成
@Aroundを使ってメソッドの実行時間を計測するPerformanceAspectを作成してください。
