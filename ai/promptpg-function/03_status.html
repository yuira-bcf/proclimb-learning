Section 2-2

データベースの変更

todoテーブルにcompletedカラムを追加します。

schema.sql 
CREATE TABLE todo (
    id INT PRIMARY KEY AUTO_INCREMENT,
    author VARCHAR(50) NOT NULL,
    title VARCHAR(100) NOT NULL,
    detail VARCHAR(500),
    completed BOOLEAN DEFAULT FALSE,  -- 追加
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
lightbulb 既存データがある場合

既存のテーブルにカラムを追加する場合は ALTER TABLE を使用します：
ALTER TABLE todo ADD completed BOOLEAN DEFAULT FALSE;

Section 2-3

Entityの修正

Todo.java 
package com.example.todo.entity;

import java.time.LocalDateTime;

public class Todo {

    private Integer id;
    private String author;
    private String title;
    private String detail;
    private Boolean completed;  // 追加
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    // getter / setter
    public Integer getId() {
        return id;
    }

    public void setId(Integer id) {
        this.id = id;
    }

    public String getAuthor() {
        return author;
    }

    public void setAuthor(String author) {
        this.author = author;
    }

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public String getDetail() {
        return detail;
    }

    public void setDetail(String detail) {
        this.detail = detail;
    }

    public Boolean getCompleted() {
        return completed;
    }

    public void setCompleted(Boolean completed) {
        this.completed = completed;
    }

    // 便利メソッド
    public boolean isCompleted() {
        return completed != null && completed;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }

    public void setCreatedAt(LocalDateTime createdAt) {
        this.createdAt = createdAt;
    }

    public LocalDateTime getUpdatedAt() {
        return updatedAt;
    }

    public void setUpdatedAt(LocalDateTime updatedAt) {
        this.updatedAt = updatedAt;
    }
}
Section 2-4

ステータス切り替え機能

ボタンクリックでステータスを切り替えるControllerを作成します。

TodoController.java（追加メソッド） 
/**
 * ステータスを切り替える
 */
@PostMapping("/{id}/toggle")
public String toggleStatus(@PathVariable Integer id) {
    todoService.toggleCompleted(id);
    return "redirect:/todo";
}
TodoService.java（追加メソッド） 
/**
 * 完了ステータスを切り替える
 */
public void toggleCompleted(Integer id) {
    Todo todo = todoRepository.findById(id);
    if (todo != null) {
        todo.setCompleted(!todo.isCompleted());
        todoRepository.update(todo);
    }
}
TodoMapper.xml（UPDATE文） 
<update id="update">
    UPDATE todo
    SET author = #{author},
        title = #{title},
        detail = #{detail},
        completed = #{completed},
        updated_at = CURRENT_TIMESTAMP
    WHERE id = #{id}
</update>
Section 2-5

条件分岐表示

Thymeleafでth:ifを使い、ステータスに応じた表示を切り替えます。

index.html 
<table class="data-table">
    <thead>
        <tr>
            <th>No</th>
            <th>タイトル</th>
            <th>登録者</th>
            <th>ステータス</th>
            <th>操作</th>
        </tr>
    </thead>
    <tbody>
        <tr th:each="todo : ${todoList}"
            th:classappend="${todo.completed} ? 'completed-row'">
            <td th:text="${todo.id}"></td>
            <td th:text="${todo.title}"></td>
            <td th:text="${todo.author}"></td>

            <!-- ステータス表示（条件分岐） -->
            <td>
                <span th:if="${todo.completed}" class="status-complete">完了</span>
                <span th:unless="${todo.completed}" class="status-incomplete">未完了</span>
            </td>

            <!-- 操作ボタン -->
            <td>
                <form th:action="@{/todo/{id}/toggle(id=${todo.id})}" method="post" style="display:inline;">
                    <button type="submit" th:text="${todo.completed} ? '未完了に戻す' : '完了にする'"></button>
                </form>
            </td>
        </tr>
    </tbody>
</table>

<style>
    .status-complete { background: #dcfce7; color: #166534; padding: 2px 8px; border-radius: 4px; }

Thymeleafの条件分岐

記述 説明
th:if="${todo.completed}" completedがtrueの場合に表示
th:unless="${todo.completed}" completedがfalseの場合に表示（th:ifの逆）
th:text="${condition} ? 'A' : 'B'" 三項演算子でテキストを切り替え
th:classappend="${condition} ? 'class-name'" 条件に応じてCSSクラスを追加
lightbulb 完了タスクの視覚的な区別

完了したタスクはcompleted-rowクラスで背景色を変え、
文字色をグレーにすることで視覚的に区別しています。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに完了/未完了ステータス機能を追加してください。

【要件】
- todoテーブルにcompleted（BOOLEAN）カラムを追加
- TodoエンティティにcompletedフィールドとisCompleted()メソッドを追加
- 一覧画面にステータス表示（完了/未完了のバッジ）を追加
- ボタンクリックでステータスを切り替えられる機能を実装
- 完了したタスクは背景色をグレーにして視覚的に区別

【技術スタック】
- Spring Boot 3.x
- MyBatis
- Thymeleaf (th:if, th:unless, th:classappend)
gps_fixed ステップ別プロンプト 
# Step 1: データベース変更
schema.sqlにcompleted BOOLEAN DEFAULT FALSEカラムを追加してください。

# Step 2: Entityの修正
Todo.javaにcompletedフィールドとgetter/setter、isCompleted()メソッドを追加してください。

# Step 3: ステータス切り替え機能
TodoControllerにPOST /{id}/toggleエンドポイントを追加し、ステータスを反転させる機能を実装してください。

# Step 4: 一覧画面の修正
index.htmlにth:if/th:unlessで完了/未完了バッジを表示し、切り替えボタンを追加してください。
lightbulb プロンプトのコツ

「th:classappendを使って、completedがtrueの行にcompleted-rowクラスを追加してください」のように、
使用したい技術を具体的に指定すると、期待通りの実装が得られます。
