現在の認証情報を保持するコンテキスト

TodoController.java 
import org.springframework.security.core.annotation.AuthenticationPrincipal;
import org.springframework.security.core.userdetails.UserDetails;

@Controller
@RequestMapping("/todo")
public class TodoController {

    /**
     * 一覧表示（自分のToDoのみ）
     */
    @GetMapping
    public String index(@AuthenticationPrincipal UserDetails userDetails,
                        Model model) {

        // ログインユーザー名を取得
        String username = userDetails.getUsername();

        // ユーザーのToDoのみ取得
        List<Todo> todoList = todoService.findByUsername(username);

        model.addAttribute("todoList", todoList);
        model.addAttribute("username", username);

        return "todo/index";
    }

    /**
     * 新規登録（ユーザー情報付き）
     */
    @PostMapping("/create")
    public String create(@AuthenticationPrincipal UserDetails userDetails,
                         @Valid @ModelAttribute Todo todo,
                         BindingResult result) {

        if (result.hasErrors()) {
            return "todo/new";
        }

        // 作成者を設定
        todo.setAuthor(userDetails.getUsername());
        todoService.save(todo);

        return "redirect:/todo";
    }
}

TodoMapper.xml 
<select id="findByUserId" resultType="Todo">
    SELECT * FROM todo
    WHERE user_id = #{userId}
    ORDER BY created_at DESC
</select>

<select id="findByUsername" resultType="Todo">
    SELECT t.* FROM todo t
    INNER JOIN users u ON t.user_id = u.id
    WHERE u.username = #{username}
    ORDER BY t.created_at DESC
</select>

<insert id="insert" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO todo (title, detail, author, user_id, created_at)
    VALUES (#{title}, #{detail}, #{author}, #{userId}, NOW())
</insert>

Section 13-3

Thymeleafでユーザー情報表示

index.html（ユーザー情報表示） 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:sec="http://www.thymeleaf.org/extras/spring-security">
<head>
    <title>ToDo一覧</title>
</head>
<body>
    <header>
        <!-- ログイン中のユーザー名を表示 -->
        <span sec:authentication="name"></span>さん、こんにちは！

        <!-- 認証済みの場合のみ表示 -->
        <div sec:authorize="isAuthenticated()">
            <form th:action="@{/logout}" method="post">
                <button type="submit">ログアウト</button>
            </form>
        </div>
    </header>

    <h1>あなたのToDoリスト</h1>
    <p><span th:text="${#lists.size(todoList)}"></span>件のToDoがあります</p>

    <!-- ToDo一覧 -->
    <table>
        <tr th:each="todo : ${todoList}">
            <td th:text="${todo.title}"></td>
            <td th:text="${todo.author}"></td>
        </tr>
    </table>
</body>
</html>

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにユーザー別ToDo管理機能を追加してください。

【要件】
- ToDoにuser_id（作成者）を追加し、ユーザーと紐付け
- ログインユーザーは自分のToDoのみ表示・編集可能
- 新規作成時に自動的にログインユーザーのIDを設定
- 他ユーザーのToDoにアクセスした場合は403エラー

【技術スタック】
- Spring Security (@AuthenticationPrincipal)
- Spring Boot 3.x
- MyBatis
gps_fixed ステップ別プロンプト 
# Step 1: テーブルの修正
todoテーブルにuser_id外部キーを追加し、usersテーブルと関連付けてください。

# Step 2: ログインユーザー情報の取得
@AuthenticationPrincipalでログインユーザー情報を取得し、ControllerでユーザーIDを利用してください。

# Step 3: 一覧表示の修正
findAll()をfindByUserId(userId)に変更し、ログインユーザーのToDoのみ取得するようにしてください。

# Step 4: アクセス制御の実装
編集・削除時にToDoの所有者チェックを行い、他ユーザーのToDoへのアクセスは403を返すようにしてください。
lightbulb プロンプトのコツ

「@PreAuthorize("@todoSecurityService.isOwner(#id, principal)")でメソッドレベルのセキュリティを実装してください」のように、
具体的なアノテーションを指定すると、適切な実装が得られます。
