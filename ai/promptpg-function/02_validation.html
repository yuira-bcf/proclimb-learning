Section 1-1

バリデーションとは

バリデーション（Validation）とは、ユーザーが入力したデータが正しい形式かどうかを検証することです。

学習できるSpring技術

check_circle

@Valid

Controllerでフォームデータの検証を有効にするアノテーション

edit_note

@NotBlank / @NotNull

必須入力チェックを行うアノテーション

cancel

BindingResult

検証結果（エラー情報）を保持するオブジェクト

lightbulb なぜバリデーションが必要？

不正なデータがデータベースに保存されることを防ぎ、
アプリケーションの品質とセキュリティを向上させます。

Section 1-2

依存関係の追加

Spring Boot 3.x では、バリデーション機能を使うために依存関係を追加する必要があります。

pom.xml 
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
build.gradle 
implementation 'org.springframework.boot:spring-boot-starter-validation'
Section 1-3

Formクラスの作成

入力データを受け取るFormクラスにバリデーションアノテーションを追加します。

TodoForm.java 
package com.example.todo.form;

import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.Size;

public class TodoForm {

    @NotBlank(message = "登録者は必須です")
    @Size(max = 50, message = "登録者は50文字以内で入力してください")
    private String author;

    @NotBlank(message = "タイトルは必須です")
    @Size(max = 100, message = "タイトルは100文字以内で入力してください")
    private String title;

    @Size(max = 500, message = "詳細は500文字以内で入力してください")
    private String detail;

    // getter / setter
    public String getAuthor() {
        return author;
    }

    public void setAuthor(String author) {
        this.author = author;
    }

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public String getDetail() {
        return detail;
    }

    public void setDetail(String detail) {
        this.detail = detail;
    }
}

主なバリデーションアノテーション

アノテーション 説明 例
@NotBlank null、空文字、空白のみを許可しない 文字列の必須チェック
@NotNull nullを許可しない オブジェクトの必須チェック
@Size 文字数の範囲を指定 @Size(min=1, max=100)
@Email メールアドレス形式かチェック メール入力欄
@Pattern 正規表現でチェック 電話番号など
Section 1-4

Controllerでの検証

Controllerで@ValidとBindingResultを使用してバリデーションを実行します。

TodoController.java 
package com.example.todo.controller;

import com.example.todo.form.TodoForm;
import org.springframework.stereotype.Controller;
import org.springframework.validation.BindingResult;
import org.springframework.web.bind.annotation.*;

import jakarta.validation.Valid;

@Controller
@RequestMapping("/todo")
public class TodoController {

    /**
     * 登録画面を表示
     */
    @GetMapping("/new")
    public String newTodo(@ModelAttribute TodoForm todoForm) {
        return "todo/edit";
    }

    /**
     * 確認画面へ遷移（バリデーション実行）
     */
    @PostMapping("/confirm")
    public String confirm(@Valid @ModelAttribute TodoForm todoForm,
                          BindingResult result) {

        // エラーがある場合は登録画面に戻る
        if (result.hasErrors()) {
            return "todo/edit";
        }

        // エラーがなければ確認画面へ
        return "todo/confirm";
    }
}
warning BindingResultの位置に注意

BindingResultは必ず@Validを付けた引数の直後に配置してください。
順序が違うとエラーが発生します。

Section 1-5

エラーメッセージの表示

Thymeleafでエラーメッセージを表示します。

edit.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>ToDo登録</title>
    <style>
        .error { color: #dc2626; font-size: 0.875rem; }
        .input-error { border-color: #dc2626; }
    </style>
</head>
<body>
    <h1>ToDo登録</h1>

    <form th:action="@{/todo/confirm}" th:object="${todoForm}" method="post">
        <table>
            <tr>
                <th>登録者 <span style="color:red;">*</span></th>
                <td>
                    <input type="text" th:field="*{author}"
                           th:classappend="${#fields.hasErrors('author')} ? 'input-error'">
                    <span class="error" th:if="${#fields.hasErrors('author')}"
                          th:errors="*{author}"></span>
                </td>
            </tr>
            <tr>
                <th>タイトル <span style="color:red;">*</span></th>
                <td>
                    <input type="text" th:field="*{title}"
                           th:classappend="${#fields.hasErrors('title')} ? 'input-error'">
                    <span class="error" th:if="${#fields.hasErrors('title')}"
                          th:errors="*{title}"></span>
                </td>
            </tr>
            <tr>
                <th>詳細</th>
                <td>
                    <textarea th:field="*{detail}" rows="5"></textarea>
                    <span class="error" th:if="${#fields.hasErrors('detail')}"
                          th:errors="*{detail}"></span>
                </td>
            </tr>
        </table>

        <button type="submit">確認</button>
        <a th:href="@{/todo}">戻る</a>
    </form>
</body>
</html>

Thymeleafのエラー表示

記述 説明
${#fields.hasErrors('author')} authorフィールドにエラーがあるかチェック
th:errors="*{author}" authorフィールドのエラーメッセージを表示
th:classappend 条件に応じてCSSクラスを追加
lightbulb 動作確認

登録画面で何も入力せずに「確認」ボタンを押すと、
各フィールドの下にエラーメッセージが表示されることを確認しましょう。

AIプログラミング

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、この機能を実装できます。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに入力バリデーション機能を追加してください。

【要件】
- TodoFormクラスを作成し、以下のバリデーションを実装
  - author: 必須、50文字以内
  - title: 必須、100文字以内
  - detail: 任意、500文字以内
- Controllerで@ValidとBindingResultを使用してバリデーション実行
- エラー時は入力画面に戻り、エラーメッセージを表示
- Thymeleafでエラーメッセージを各フィールドの下に赤字で表示

【技術スタック】
- Spring Boot 3.x
- Thymeleaf
- Jakarta Validation (jakarta.validation.constraints)
gps_fixed ステップ別プロンプト 
# Step 1: 依存関係の追加
pom.xmlにspring-boot-starter-validationの依存関係を追加してください。

# Step 2: Formクラスの作成
TodoFormクラスを作成し、@NotBlank、@Sizeアノテーションでバリデーションルールを定義してください。

# Step 3: Controllerの修正
TodoControllerのPOSTメソッドに@ValidとBindingResultを追加し、エラー時は入力画面に戻るようにしてください。

# Step 4: ビューの修正
Thymeleafテンプレートにth:errorsでエラーメッセージを表示する機能を追加してください。
lightbulb プロンプトのコツ

「既存のTodoController.javaを読み込んで、バリデーション機能を追加してください」のように、
既存ファイルを指定すると、プロジェクトに合わせた実装が得られます。
