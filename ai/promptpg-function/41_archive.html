Section 41-1

archived列の追加

完了済みタスクを一覧から非表示にするアーカイブ機能を実装します。archivedフラグを追加し、通常の一覧ではアーカイブ済みタスクを除外します。

archive

archivedフラグ

BOOLEAN型のフラグでアーカイブ状態を管理

filter_alt

一覧からの除外

通常の一覧クエリでarchived=falseのみ取得

event_note

アーカイブ日時

いつアーカイブされたかを記録

schema.sql（archived列追加）
ALTER TABLE todo
    ADD COLUMN archived BOOLEAN DEFAULT FALSE,
    ADD COLUMN archived_at TIMESTAMP NULL;
TodoMapper.xml（アーカイブ対応クエリ）
<!-- 通常一覧（アーカイブ済みを除外） -->
<select id="findActive" resultType="Todo">
    SELECT * FROM todo
    WHERE archived = FALSE
    ORDER BY created_at DESC
</select>

<!-- アーカイブ済み一覧 -->
<select id="findArchived" resultType="Todo">
    SELECT * FROM todo
    WHERE archived = TRUE
    ORDER BY archived_at DESC
</select>

<!-- アーカイブ実行 -->
<update id="archive">
    UPDATE todo
    SET archived = TRUE, archived_at = NOW()
    WHERE id = #{id}
</update>

<!-- アーカイブ復元 -->
<update id="unarchive">
    UPDATE todo
    SET archived = FALSE, archived_at = NULL
    WHERE id = #{id}
</update>
lightbulb削除との違い

アーカイブは論理的な非表示化であり、データは残ります。削除と違い、いつでも復元できるため、「完了したけど記録は残したい」タスクに最適です。

account_tree 処理フロー: アーカイブ機能

1
archived フラグ追加（BOOLEAN DEFAULT FALSE）
ALTER TABLE todo ADD COLUMN archived BOOLEAN DEFAULT FALSE, ADD COLUMN archived_at TIMESTAMP NULL;
2
TodoService.archive(id) でフラグを TRUE に更新
UPDATE todo SET archived = TRUE, archived_at = NOW() WHERE id = #{id}
3
通常一覧は archived = FALSE のみ表示
SELECT * FROM todo WHERE archived = FALSE ORDER BY created_at DESC
4
アーカイブ一覧画面で復元ボタンを提供
POST /todo/{id}/unarchive → SET archived = FALSE, archived_at = NULL

ポイント: アーカイブは物理削除ではなく論理的な非表示化です。データを保持したまま一覧から除外し、いつでも復元可能にすることで安全なタスク管理を実現します。

Section 41-2

アーカイブ/復元コントローラー

タスクのアーカイブと復元を行うコントローラーを実装します。ボタン1つでアーカイブ/復元を切り替えられるようにします。

inventory_2

アーカイブ処理

POSTリクエストでタスクをアーカイブ

unarchive

復元処理

アーカイブ済みタスクを通常一覧に戻す

flash_on

フラッシュメッセージ

操作結果をユーザーに通知

ArchiveController.java
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.servlet.mvc.support.RedirectAttributes;

@Controller
public class ArchiveController {

    private final TodoMapper todoMapper;

    public ArchiveController(TodoMapper todoMapper) {
        this.todoMapper = todoMapper;
    }

    /**
     * タスクをアーカイブ
     */
    @PostMapping("/todo/{id}/archive")
    public String archive(@PathVariable Integer id,
                          RedirectAttributes redirectAttributes) {
        todoMapper.archive(id);
        redirectAttributes.addFlashAttribute("message",
            "タスクをアーカイブしました");
        return "redirect:/todo";
    }

    /**
     * アーカイブ済みタスクを復元
     */
    @PostMapping("/todo/{id}/unarchive")
    public String unarchive(@PathVariable Integer id,
                            RedirectAttributes redirectAttributes) {
        todoMapper.unarchive(id);
        redirectAttributes.addFlashAttribute("message",
            "タスクを復元しました");
        return "redirect:/todo/archive";
    }

    /**
     * アーカイブ一覧ページ
     */
    @GetMapping("/todo/archive")
    public String archiveList(Model model) {
        model.addAttribute("tasks", todoMapper.findArchived());
        return "archive-list";
    }
}

Section 41-4

自動アーカイブスケジューラ

完了後一定期間が経過したタスクを自動的にアーカイブするスケジューラを実装します。Spring Bootの@Scheduledアノテーションで定期実行します。

schedule

@Scheduled

cronまたはfixedRate式で定期実行

auto_fix_high

自動アーカイブ条件

完了後30日経過したタスクを自動アーカイブ

settings

設定の外部化

アーカイブ日数をapplication.ymlで設定可能に

AutoArchiveScheduler.java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

@Component
public class AutoArchiveScheduler {

    private static final Logger log =
        LoggerFactory.getLogger(AutoArchiveScheduler.class);

    private final TodoMapper todoMapper;

    @Value("${todo.auto-archive.days:30}")
    private int archiveDays;

    public AutoArchiveScheduler(TodoMapper todoMapper) {
        this.todoMapper = todoMapper;
    }

    /**
     * 毎日深夜2時に実行
     * 完了後N日経過したタスクを自動アーカイブ
     */
    @Scheduled(cron = "0 0 2 * * *")
    public void autoArchive() {
        int count = todoMapper.archiveCompleted(archiveDays);
        if (count > 0) {
            log.info("自動アーカイブ: {}件のタスクをアーカイブ", count);
        }
    }
}
TodoMapper.xml + application.yml
<!-- TodoMapper.xml -->
<update id="archiveCompleted">
    UPDATE todo
    SET archived = TRUE, archived_at = NOW()
    WHERE completed = TRUE
      AND archived = FALSE
      AND completed_at < DATE_SUB(NOW(), INTERVAL #{days} DAY)
</update>

<!-- application.yml -->
<!--
todo:
  auto-archive:
    days: 30   # 完了後30日でアーカイブ
-->
Application.java（スケジューリング有効化）
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.scheduling.annotation.EnableScheduling;

@SpringBootApplication
@EnableScheduling   // スケジューリングを有効化
public class TodoApplication {
    public static void main(String[] args) {
        SpringApplication.run(TodoApplication.class, args);
    }
}
warning@EnableSchedulingの忘れずに

@Scheduledが動作するには、メインクラスに@EnableSchedulingを付ける必要があります。この設定がないと、cron式を書いてもスケジューラは一切実行されません。

edit_note 基本プロンプト
Spring Boot 3.xのToDoアプリに完了タスクのアーカイブ機能を追加してください。

【要件】
- todoテーブルにarchived（BOOLEAN）、archived_at（TIMESTAMP）カラム追加
- アーカイブ/復元のコントローラー（POST /todo/{id}/archive, /unarchive）
- アーカイブ一覧ページ（GET /todo/archive）：復元・完全削除ボタン付き
- 通常一覧ではarchived=falseのみ表示
- @Scheduledで毎日深夜2時に完了後30日経過タスクを自動アーカイブ
- アーカイブ日数はapplication.ymlで設定可能

【技術スタック】
- Spring Boot 3.x + MyBatis
- @Scheduled（cron式）
- @EnableScheduling
- Thymeleaf

# Step 1: DB設計

# Step 2: アーカイブ/復元コントローラー
