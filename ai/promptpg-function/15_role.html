Section 14-1

メソッドセキュリティの有効化

@PreAuthorizeを使って、ロールに基づいたアクセス制御を実装します。

shield

@PreAuthorize

メソッド実行前に権限をチェック

admin_panel_settings

ROLE_ADMIN

管理者権限を持つユーザー

SecurityConfig.java（メソッドセキュリティ有効化） 
import org.springframework.security.config.annotation.method.configuration.EnableMethodSecurity;

@Configuration
@EnableWebSecurity
@EnableMethodSecurity  // メソッドセキュリティを有効化
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/admin/**").hasRole("ADMIN")  // 管理者のみ
                .requestMatchers("/todo/**").hasAnyRole("USER", "ADMIN")
                .anyRequest().authenticated()
            )
            // ... 他の設定
        ;
        return http.build();
    }
}
Section 14-2

@PreAuthorizeによる権限制御

TodoController.java 
import org.springframework.security.access.prepost.PreAuthorize;

@Controller
@RequestMapping("/todo")
public class TodoController {

    /**
     * 全ユーザーのToDo一覧（管理者のみ）
     */
    @GetMapping("/all")
    @PreAuthorize("hasRole('ADMIN')")
    public String allTodos(Model model) {
        List<Todo> allTodos = todoService.findAll();
        model.addAttribute("todoList", allTodos);
        return "todo/all";
    }

    /**
     * ToDo削除（管理者または作成者のみ）
     */
    @PostMapping("/delete/{id}")
    @PreAuthorize("hasRole('ADMIN') or @todoService.isOwner(#id, authentication.name)")
    public String delete(@PathVariable Integer id) {
        todoService.delete(id);
        return "redirect:/todo";
    }

    /**
     * 一括削除（管理者のみ）
     */
    @PostMapping("/bulk-delete")
    @PreAuthorize("hasRole('ADMIN')")
    public String bulkDelete(@RequestParam List<Integer> ids) {
        todoService.deleteByIds(ids);
        return "redirect:/todo";
    }
}
TodoService.java（所有者チェック） 
@Service
public class TodoService {

    private final TodoMapper todoMapper;

    /**
     * 指定したToDoの所有者かどうかを確認
     */
    public boolean isOwner(Integer todoId, String username) {
        Todo todo = todoMapper.findById(todoId);
        if (todo == null) return false;
        return username.equals(todo.getAuthor());
    }
}

@PreAuthorize式

式説明
hasRole('ADMIN')ADMIN権限を持つ場合
hasAnyRole('USER', 'ADMIN')いずれかの権限を持つ場合
isAuthenticated()認証済みの場合
#idメソッド引数を参照
authentication.nameログインユーザー名
Section 14-3

管理者専用機能の実装

index.html（管理者メニュー） 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:sec="http://www.thymeleaf.org/extras/spring-security">
<head>
    <title>ToDo一覧</title>
</head>
<body>
    <nav>
        <!-- 一般ユーザー向けメニュー -->
        <a th:href="@{/todo}">マイToDo</a>
        <a th:href="@{/todo/new}">新規作成</a>

        <!-- 管理者のみ表示 -->
        <div sec:authorize="hasRole('ADMIN')">
            <span style="color: #dc2626;">[管理者メニュー]</span>
            <a th:href="@{/todo/all}">全ToDo一覧</a>
            <a th:href="@{/admin/users}">ユーザー管理</a>
        </div>
    </nav>

    <!-- 一括削除ボタン（管理者のみ） -->
    <form th:action="@{/todo/bulk-delete}" method="post"
          sec:authorize="hasRole('ADMIN')">
        <button type="submit" class="btn btn-danger">
            選択した項目を削除（管理者）
        </button>
    </form>

    <table>
        <tr th:each="todo : ${todoList}">
            <!-- 管理者のみチェックボックス表示 -->
            <td sec:authorize="hasRole('ADMIN')">
                <input type="checkbox" name="ids" th:value="${todo.id}">
            </td>
            <td th:text="${todo.title}"></td>
            <td>
                <!-- 自分のToDoまたは管理者のみ編集可能 -->
                <a th:href="@{/todo/edit/{id}(id=${todo.id})}"
                   th:if="${todo.author == #authentication.name}">編集</a>
                <a th:href="@{/todo/edit/{id}(id=${todo.id})}"
                   sec:authorize="hasRole('ADMIN')">編集（管理者）</a>
            </td>
        </tr>
    </table>
</body>
</html>
warning二重チェックの重要性

画面での表示制御だけでなく、必ずサーバー側でも権限チェックを行ってください。
@PreAuthorizeでサーバー側を保護し、sec:authorizeで画面を整理します。

lightbulb権限不足時のエラー処理

権限がない場合はAccessDeniedExceptionが発生します。
カスタムエラーページを用意しておくとユーザー体験が向上します。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにロール別権限制御を追加してください。

【要件】
- ロール：ADMIN（管理者）とUSER（一般ユーザー）
- ADMIN：全ユーザーのToDo閲覧・編集・削除可能、ユーザー管理可能
- USER：自分のToDoのみ閲覧・編集・削除可能
- 画面上でロールに応じたメニュー表示切り替え
- URLレベルでのアクセス制御

【技術スタック】
- Spring Security (@PreAuthorize, hasRole)
- Thymeleaf (sec:authorize)
- Spring Boot 3.x
gps_fixed ステップ別プロンプト 
# Step 1: SecurityConfigの修正
/admin/**パスへのアクセスをROLE_ADMINのみに制限してください。

# Step 2: メソッドセキュリティの有効化
@EnableMethodSecurityを追加し、@PreAuthorize("hasRole('ADMIN')")でメソッドレベルの認可を実装してください。

# Step 3: 画面の権限別表示
Thymeleafでsec:authorize="hasRole('ADMIN')"を使用し、管理者のみに表示する要素を制御してください。

# Step 4: ユーザー管理機能の実装
管理者専用のユーザー一覧・編集画面を作成し、ロールの変更やアカウントの有効/無効を管理できるようにしてください。
lightbulb プロンプトのコツ

「管理者が一般ユーザーのToDoを編集する場合は、編集履歴を残すようにしてください」のように、
監査要件を追加すると、より堅牢な実装が得られます。
