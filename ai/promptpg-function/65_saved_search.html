Section 65-1

検索条件テーブル設計

ユーザーが頻繁に使う検索条件を名前を付けて保存できるテーブルを設計します。検索条件はJSON形式で柔軟に保存します。

JSON形式で条件保存

キーワード、カテゴリ、優先度、日付範囲を1つのJSONに

format_list_numbered

schema.sql（saved_searchesテーブル） 
CREATE TABLE saved_searches (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    name VARCHAR(100) NOT NULL,
    criteria_json TEXT NOT NULL,       -- 検索条件をJSON形式で保存
    use_count INT DEFAULT 0,           -- 使用回数
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 検索条件JSONの例:
-- {"keyword":"報告書","category":"仕事","priority":"HIGH","status":"未完了"}

lightbulbJSONで柔軟に保存

検索条件をJSON文字列で保存することで、将来フィルタ項目が増えてもテーブル構造を変更せずに対応できます。JacksonのObjectMapperで簡単にJava ↔ JSON変換が可能です。

JSON変換

ObjectMapperでSearchCriteria ↔ JSON文字列を変換

SavedSearchController.java 
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/saved-searches")
public class SavedSearchController {

    private final SavedSearchMapper savedSearchMapper;
    private final ObjectMapper objectMapper;

    public SavedSearchController(SavedSearchMapper savedSearchMapper,
                                 ObjectMapper objectMapper) {
        this.savedSearchMapper = savedSearchMapper;
        this.objectMapper = objectMapper;
    }

    /**
     * 検索条件を保存
     */
    @PostMapping
    public Map<String, Object> save(
            @AuthenticationPrincipal UserDetails user,
            @RequestBody SaveSearchRequest request) throws Exception {

        Integer userId = ((CustomUserDetails) user).getUserId();

        // SearchCriteriaをJSON文字列に変換
        String json = objectMapper.writeValueAsString(request.getCriteria());

        SavedSearch ss = new SavedSearch();
        ss.setUserId(userId);
        ss.setName(request.getName());
        ss.setCriteriaJson(json);

        savedSearchMapper.insert(ss);
        return Map.of("status", "ok", "id", ss.getId());
    }

    /**
     * 保存済み検索一覧を取得
     */
    @GetMapping
    public List<SavedSearch> list(
            @AuthenticationPrincipal UserDetails user) {
        Integer userId = ((CustomUserDetails) user).getUserId();
        return savedSearchMapper.findByUserId(userId);
    }

    /**
     * 保存済み検索を削除
     */
    @DeleteMapping("/{id}")
    public Map<String, String> delete(
            @AuthenticationPrincipal UserDetails user,
            @PathVariable Integer id) {
        Integer userId = ((CustomUserDetails) user).getUserId();
        savedSearchMapper.deleteById(id, userId);
        return Map.of("status", "ok");
    }
}

/**
 * 現在の検索条件を保存するダイアログを表示
 */
function showSaveDialog() {
    const name = prompt('検索条件の名前を入力してください:');
    if (!name || !name.trim()) return;

    // 現在のフォーム値から検索条件を収集
    const criteria = {
        keyword: document.getElementById('search-keyword').value,
        category: document.getElementById('search-category').value,
        priority: document.getElementById('search-priority').value,
        status: document.getElementById('search-status').value
    };

    fetch('/api/saved-searches', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ name: name.trim(), criteria })
    })
    .then(res => res.json())
    .then(data => {
        if (data.status === 'ok') {
            location.reload(); // 一覧を更新
        }
    });
}

/**
 * 保存済み検索を削除
 */
function deleteSavedSearch(btn) {
    if (!confirm('この検索条件を削除しますか？')) return;
    const id = btn.dataset.id;

    fetch('/api/saved-searches/' + id, { method: 'DELETE' })
    .then(res => res.json())
    .then(data => {
        if (data.status === 'ok') {
            btn.closest('.saved-search-item').remove();
        }
    });
}

JavaScript（ワンクリック検索実行） 
/**
 * 保存済み検索を実行
 */
function executeSavedSearch(btn) {
    const id = btn.dataset.id;
    const criteria = JSON.parse(btn.dataset.criteria);

    // 1. 検索フォームに条件を反映
    if (criteria.keyword)
        document.getElementById('search-keyword').value = criteria.keyword;
    if (criteria.category)
        document.getElementById('search-category').value = criteria.category;
    if (criteria.priority)
        document.getElementById('search-priority').value = criteria.priority;
    if (criteria.status)
        document.getElementById('search-status').value = criteria.status;

    // 2. 使用回数を更新
    fetch('/api/saved-searches/' + id + '/use', { method: 'POST' });

    // 3. URLパラメータを構築してリダイレクト
    const params = new URLSearchParams();
    Object.entries(criteria).forEach(([key, value]) => {
        if (value) params.set(key, value);
    });

    window.location.href = '/todo?' + params.toString();
}

    savedSearchMapper.incrementUseCount(id);
    return Map.of("status", "ok");

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに検索条件保存機能を追加してください。

【要件】
- saved_searchesテーブルで検索条件をJSON形式で保存
- 現在の検索条件に名前を付けて保存できるUI
- 保存済み検索の一覧表示（使用頻度順）
- ワンクリックで保存済み検索を実行

【技術スタック】
- Spring Boot 3.x + MyBatis
- Jackson ObjectMapper（JSON変換）
- JavaScript fetch API
- Thymeleaf
gps_fixed ステップ別プロンプト 
# Step 1: 検索条件テーブル
saved_searchesテーブルを作成してください。検索条件はJSON文字列で保存し、使用回数カウンターを含めてください。

# Step 2: 保存・削除API
POST /api/saved-searchesで検索条件を保存、DELETE /api/saved-searches/{id}で削除するREST APIを作成してください。

# Step 3: 一覧UI
保存済み検索をサイドバーに使用頻度順で表示してください。各項目に削除ボタンを付けてください。

# Step 4: ワンクリック実行
保存済み検索をクリックしたら検索フォームに条件を反映し、URLパラメータでリダイレクトして検索を実行してください。
