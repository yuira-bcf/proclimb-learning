Section 45-1

完了率計算ロジック

全タスクに対する完了タスクの割合をパーセンテージで計算するサービスロジックを実装します。0除算への対応も含めます。

calculate

完了率計算

完了タスク数 / 全タスク数 x 100の計算

data_object

ProgressDTO

進捗データを格納するデータ転送オブジェクト

safety_check

0除算対策

タスクがない場合の安全な処理

ProgressInfo.java（進捗DTO） 
public class ProgressInfo {

    private String label;       // 表示ラベル（全体 or カテゴリ名）
    private long totalCount;    // 全タスク数
    private long completedCount; // 完了タスク数
    private int percentage;     // 完了率（0-100）

    public ProgressInfo(String label, long total, long completed) {
        this.label = label;
        this.totalCount = total;
        this.completedCount = completed;
        // 0除算を防止
        this.percentage = total == 0 ? 0
            : (int) Math.round((double) completed / total * 100);
    }

    // getter/setter
}
ProgressService.java（進捗計算サービス） 
import org.springframework.stereotype.Service;
import java.util.*;

@Service
public class ProgressService {

    private final TodoMapper todoMapper;
    private final CategoryMapper categoryMapper;

    public ProgressService(TodoMapper todoMapper,
                           CategoryMapper categoryMapper) {
        this.todoMapper = todoMapper;
        this.categoryMapper = categoryMapper;
    }

    /**
     * 全体の進捗情報を取得
     */
    public ProgressInfo getOverallProgress(Integer userId) {
        long total = todoMapper.countByUserId(userId);
        long completed = todoMapper.countByUserIdAndStatus(
            userId, "COMPLETED");
        return new ProgressInfo("全体", total, completed);
    }

    /**
     * カテゴリ別の進捗情報を取得
     */
    public List<ProgressInfo> getCategoryProgress(Integer userId) {
        List<Category> categories = categoryMapper.findAll();
        List<ProgressInfo> result = new ArrayList<>();

        for (Category cat : categories) {
            long total = todoMapper.countByUserIdAndCategory(
                userId, cat.getId());
            long completed = todoMapper
                .countCompletedByUserIdAndCategory(
                    userId, cat.getId());
            result.add(new ProgressInfo(
                cat.getName(), total, completed));
        }
        return result;
    }
}

lightbulbMath.roundで整数化

Math.round()で四捨五入することで、表示用のパーセンテージを整数にしています。小数点付きの表示が必要な場合はdouble型にしてString.format("%.1f", value)でフォーマットします。

Section 45-3

カテゴリ別進捗表示

カテゴリごとの進捗状況を複数のプログレスバーで一覧表示します。どのカテゴリが遅れているかを一目で把握できます。

category

カテゴリ別集計

各カテゴリのタスク完了率を個別に計算

view_list

一覧表示

th:eachでカテゴリ別プログレスバーを繰り返し描画

sort

ソート表示

進捗率の低い順に表示して注意喚起

ProgressController.java 
import org.springframework.security.core.annotation.AuthenticationPrincipal;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class ProgressController {

    private final ProgressService progressService;

    public ProgressController(ProgressService progressService) {
        this.progressService = progressService;
    }

    @GetMapping("/progress")
    public String progress(
            @AuthenticationPrincipal LoginUser loginUser,
            Model model) {

        Integer userId = loginUser.getUserId();

        // 全体の進捗
        ProgressInfo overall =
            progressService.getOverallProgress(userId);
        model.addAttribute("overall", overall);

        // カテゴリ別進捗（進捗率の低い順でソート）
        List<ProgressInfo> categoryProgress =
            progressService.getCategoryProgress(userId);
        categoryProgress.sort(
            Comparator.comparingInt(ProgressInfo::getPercentage));
        model.addAttribute("categoryProgress", categoryProgress);

        return "progress";
    }
}

Section 45-4

リアルタイム更新

タスクのステータスが変更されたときに、プログレスバーをリアルタイムで更新する仕組みを実装します。fetch APIで最新データを取得しDOMを更新します。

sync

ポーリング更新

定期的にAPIを呼び出して最新の進捗を取得

api

進捗API

現在の進捗データをJSON形式で返すAPI

refresh

DOM更新

バー幅とパーセンテージ表示を動的に更新

ProgressApiController.java（進捗API） 
import org.springframework.web.bind.annotation.*;
import org.springframework.security.core.annotation.AuthenticationPrincipal;

@RestController
@RequestMapping("/api/progress")
public class ProgressApiController {

    private final ProgressService progressService;

    public ProgressApiController(ProgressService progressService) {
        this.progressService = progressService;
    }

    @GetMapping
    public Map<String, Object> getProgress(
            @AuthenticationPrincipal LoginUser loginUser) {

        Integer userId = loginUser.getUserId();
        ProgressInfo overall =
            progressService.getOverallProgress(userId);
        List<ProgressInfo> categories =
            progressService.getCategoryProgress(userId);

        return Map.of(
            "overall", overall,
            "categories", categories
        );
    }
}
progress.js（リアルタイム更新） 
/**
 * 進捗バーをリアルタイムで更新
 */
function updateProgressBars() {
    fetch('/api/progress')
        .then(res => res.json())
        .then(data => {
            // 全体の進捗を更新
            const overall = data.overall;
            const overallBar = document.querySelector(
                '#overall-progress .progress-bar-fill');
            const overallValue = document.querySelector(
                '#overall-progress .progress-value');

            overallBar.style.width = overall.percentage + '%';
            overallValue.textContent = overall.percentage + '%';

            // 色クラスを更新
            overallBar.className = 'progress-bar-fill';
            if (overall.percentage < 30) {
                overallBar.classList.add('low');
            } else if (overall.percentage < 70) {
                overallBar.classList.add('medium');
            } else {
                overallBar.classList.add('high');
            }
        });
}

// 30秒ごとに更新
setInterval(updateProgressBars, 30000);
lightbulbポーリングの間隔

30秒間隔のポーリングは簡易的なリアルタイム更新です。より即座な反映が必要な場合は、Server-Sent Events（SSE）やWebSocketの使用を検討してください（第66章で解説）。
