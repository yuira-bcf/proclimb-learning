Section 28-1

親子関係テーブル設計

タスクにサブタスク（子タスク）を持たせるために、todoテーブルにparent_id列を追加し、自己参照の親子関係を構築します。

account_tree

自己参照外部キー

parent_idでtodoテーブル自身のidを参照する設計

schema

NULL許容設計

parent_idがNULLなら親タスク、値があればサブタスク

storage

カスケード削除

親タスク削除時にサブタスクも連動して削除

schema.sql（parent_id列の追加） 
-- todoテーブルにparent_id列を追加
ALTER TABLE todo
ADD COLUMN parent_id INT NULL,
ADD CONSTRAINT fk_todo_parent
    FOREIGN KEY (parent_id) REFERENCES todo(id)
    ON DELETE CASCADE;

-- インデックス追加（検索高速化）
CREATE INDEX idx_todo_parent_id ON todo(parent_id);
Todo.java（エンティティ拡張） 
public class Todo {

    private Integer id;
    private String title;
    private String description;
    private boolean completed;
    private Integer userId;
    private Integer parentId;          // サブタスクの親ID（NULLなら親タスク）
    private List<Todo> subtasks;       // 子タスク一覧（マッピング用）
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    // --- 親タスク判定 ---
    public boolean isParent() {
        return this.parentId == null;
    }

    // --- サブタスク判定 ---
    public boolean isSubtask() {
        return this.parentId != null;
    }

    // getter/setter
}
TodoMapper.xml（親子リレーション取得） 
<!-- 親タスクのみ取得（parent_id IS NULL） -->
<select id="findParentTasks" resultMap="todoWithSubtasks">
    SELECT id, title, description, completed, user_id,
           parent_id, created_at, updated_at
    FROM todo
    WHERE user_id = #{userId} AND parent_id IS NULL
    ORDER BY created_at DESC
</select>

<!-- サブタスク取得 -->
<select id="findSubtasks" resultType="Todo">
    SELECT id, title, description, completed, user_id,
           parent_id, created_at, updated_at
    FROM todo
    WHERE parent_id = #{parentId}
    ORDER BY created_at ASC
</select>

<!-- 親タスク + サブタスクのResultMap -->
<resultMap id="todoWithSubtasks" type="Todo">
    <id property="id" column="id"/>
    <result property="title" column="title"/>
    <result property="completed" column="completed"/>
    <collection property="subtasks" column="id"
                select="findSubtasks"/>
</resultMap>
lightbulb自己参照テーブルのポイント

parent_idがNULLのレコードが親タスク、値が入っているレコードがサブタスクです。
MyBatisの<collection>タグを使うことで、親タスク取得時にサブタスクも自動的にロードできます。

Section 28-2

サブタスク追加・削除API

親タスクに対してサブタスクを追加・削除するコントローラーとサービスを実装します。サブタスクは親タスクの詳細画面から操作します。

check_box

SubtaskController.java 
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.servlet.mvc.support.RedirectAttributes;

@Controller
@RequestMapping("/todo/{parentId}/subtasks")
public class SubtaskController {

    private final SubtaskService subtaskService;

    public SubtaskController(SubtaskService subtaskService) {
        this.subtaskService = subtaskService;
    }

    /**
     * サブタスク追加
     */
    @PostMapping
    public String addSubtask(@PathVariable Integer parentId,
                             @RequestParam String title,
                             RedirectAttributes ra) {
        subtaskService.addSubtask(parentId, title);
        ra.addFlashAttribute("message", "サブタスクを追加しました");
        return "redirect:/todo/" + parentId;
    }

    /**
     * サブタスク削除
     */
    @PostMapping("/{subtaskId}/delete")
    public String deleteSubtask(@PathVariable Integer parentId,
                                @PathVariable Integer subtaskId,
                                RedirectAttributes ra) {
        subtaskService.deleteSubtask(subtaskId);
        ra.addFlashAttribute("message", "サブタスクを削除しました");
        return "redirect:/todo/" + parentId;
    }

    /**
     * サブタスク完了トグル
     */
    @PostMapping("/{subtaskId}/toggle")
    public String toggleSubtask(@PathVariable Integer parentId,
                                @PathVariable Integer subtaskId) {
        subtaskService.toggleComplete(subtaskId);
        return "redirect:/todo/" + parentId;
    }
}
SubtaskService.java 
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class SubtaskService {

    private final TodoMapper todoMapper;

    public SubtaskService(TodoMapper todoMapper) {
        this.todoMapper = todoMapper;
    }

    /**
     * サブタスクを追加
     */
    @Transactional
    public void addSubtask(Integer parentId, String title) {
        Todo parent = todoMapper.findById(parentId);
        if (parent == null || parent.isSubtask()) {
            throw new IllegalArgumentException("無効な親タスクです");
        }

        Todo subtask = new Todo();
        subtask.setTitle(title);
        subtask.setParentId(parentId);
        subtask.setUserId(parent.getUserId());
        subtask.setCompleted(false);
        todoMapper.insert(subtask);
    }

    /**
     * サブタスクを削除
     */
    @Transactional
    public void deleteSubtask(Integer subtaskId) {
        todoMapper.deleteById(subtaskId);
    }

    /**
     * 完了状態をトグル
     */
    @Transactional
    public void toggleComplete(Integer subtaskId) {
        Todo subtask = todoMapper.findById(subtaskId);
        subtask.setCompleted(!subtask.isCompleted());
        todoMapper.update(subtask);
    }
}
TodoMapper.xml（サブタスク登録） 
<insert id="insert" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO todo (title, description, completed, user_id,
                      parent_id, created_at)
    VALUES (#{title}, #{description}, #{completed}, #{userId},
            #{parentId}, NOW())
</insert>
warningネスト制限

この実装では1階層のみのサブタスクをサポートしています。サブタスクの子タスク（孫タスク）は作成できないよう、parent.isSubtask()チェックを入れています。多階層が必要な場合は再帰的な設計が必要です。

Section 28-3

進捗率の自動計算

サブタスクの完了数から親タスクの進捗率を自動計算する機能を実装します。全サブタスクが完了すると親タスクも自動的に完了になります。

percent

進捗率の計算式

完了サブタスク数 / 全サブタスク数 * 100

auto_fix_high

自動完了判定

全サブタスク完了時に親タスクを自動で完了に

analytics

進捗バー表示

一覧画面に進捗率をバーで可視化

SubtaskService.java（進捗率計算） 
/**
 * 親タスクの進捗率を計算（0〜100）
 */
public int calculateProgress(Integer parentId) {
    List<Todo> subtasks = todoMapper.findSubtasks(parentId);
    if (subtasks.isEmpty()) {
        return 0;
    }

    long completedCount = subtasks.stream()
            .filter(Todo::isCompleted)
            .count();

    return (int) (completedCount * 100 / subtasks.size());
}

/**
 * サブタスク完了トグル + 親タスクの自動完了判定
 */
@Transactional
public void toggleComplete(Integer subtaskId) {
    Todo subtask = todoMapper.findById(subtaskId);
    subtask.setCompleted(!subtask.isCompleted());
    todoMapper.update(subtask);

    // 親タスクの自動完了チェック
    updateParentCompletion(subtask.getParentId());
}

/**
 * 全サブタスク完了なら親も完了にする
 */
private void updateParentCompletion(Integer parentId) {
    List<Todo> subtasks = todoMapper.findSubtasks(parentId);

    boolean allCompleted = subtasks.stream()
            .allMatch(Todo::isCompleted);

    Todo parent = todoMapper.findById(parentId);
    parent.setCompleted(allCompleted);
    todoMapper.update(parent);
}
TodoMapper.xml（進捗率集計SQL） 
<!-- サブタスク数と完了数を一度に取得 -->
<select id="countSubtaskProgress" resultType="map">
    SELECT
        COUNT(*) AS total,
        SUM(CASE WHEN completed = true THEN 1 ELSE 0 END) AS done
    FROM todo
    WHERE parent_id = #{parentId}
</select>
lightbulbN+1問題に注意

一覧画面で各親タスクの進捗率を個別に取得するとN+1問題が発生します。
一覧表示では集計SQLで一括取得するか、MyBatisの<collection>でサブタスクをまとめてロードしましょう。

TodoController.java（詳細画面に進捗率を渡す） 
@GetMapping("/todo/{id}")
public String detail(@PathVariable Integer id, Model model) {
    Todo todo = todoService.findByIdWithSubtasks(id);

    // 進捗率を計算してモデルに追加
    int progress = subtaskService.calculateProgress(id);
    long completedCount = todo.getSubtasks().stream()
            .filter(Todo::isCompleted).count();

    model.addAttribute("todo", todo);
    model.addAttribute("progress", progress);
    model.addAttribute("completedCount", completedCount);
    model.addAttribute("totalCount", todo.getSubtasks().size());

    return "todo-detail";
}

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにサブタスク機能を追加してください。

【要件】
- todoテーブルにparent_id列を追加（自己参照外部キー）
- 親タスクの詳細画面にサブタスク一覧を表示
- サブタスクの追加・削除・完了トグル機能
- 進捗率の自動計算（完了サブタスク数/全サブタスク数）
- 全サブタスク完了時に親タスクも自動完了

【技術スタック】
- Spring Boot 3.x + MyBatis
- Thymeleaf
- ON DELETE CASCADEでカスケード削除
- @Transactionalでデータ整合性を保証
gps_fixed ステップ別プロンプト 
# Step 1: テーブル設計
todoテーブルにparent_id列（INT, NULL許容）を追加するALTER TABLEを作成してください。外部キー制約とON DELETE CASCADEを含めてください。

# Step 2: エンティティとMapper
TodoエンティティにparentIdフィールドとsubtasksリストを追加してください。MyBatisのResultMapでcollectionを使い、親タスク取得時にサブタスクも自動ロードしてください。

# Step 3: サブタスクCRUD
/todo/{parentId}/subtasks配下でサブタスクの追加・削除・完了トグルを行うコントローラーとサービスを作成してください。

# Step 4: 進捗率と自動完了
サブタスクの完了数から進捗率を計算し、全サブタスク完了時に親タスクも自動で完了にする処理を実装してください。

# Step 5: UI実装
親タスクの詳細画面にプログレスバー、サブタスク一覧（チェックボックス付き）、インライン追加フォームを表示してください。
lightbulb プロンプトのコツ
