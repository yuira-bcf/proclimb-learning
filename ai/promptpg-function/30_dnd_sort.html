Section 30-1

display_order列の追加

タスクの表示順をユーザーが自由に変更できるよう、todoテーブルにdisplay_order列を追加します。

sort

表示順カラム

INT型のdisplay_orderで並び順を管理

low_priority

デフォルト値

新規タスクは末尾に追加（最大値+1）

swap_vert

順番の入れ替え

ドラッグ操作で複数レコードのorderを更新

schema.sql（display_order列の追加） 
-- display_order列を追加
ALTER TABLE todo
ADD COLUMN display_order INT DEFAULT 0;

-- 既存データの表示順を初期化（ID順）
UPDATE todo SET display_order = id
WHERE display_order = 0;

-- インデックス追加
CREATE INDEX idx_todo_display_order
ON todo(user_id, display_order);
TodoMapper.xml（表示順でソート取得） 
<select id="findByUserIdOrdered" resultType="Todo">
    SELECT id, title, description, completed,
           user_id, display_order, created_at
    FROM todo
    WHERE user_id = #{userId} AND parent_id IS NULL
    ORDER BY display_order ASC
</select>

<!-- 新規タスクの表示順を末尾に設定 -->
<select id="getMaxDisplayOrder" resultType="int">
    SELECT COALESCE(MAX(display_order), 0)
    FROM todo
    WHERE user_id = #{userId}
</select>
lightbulb表示順の設計パターン

連番（1,2,3...）方式はシンプルですが、並べ替え時に多数のレコードを更新する必要があります。
間隔を空ける方式（100,200,300...）にすると、間に挿入する際の更新を最小限にできます。

Section 30-2

HTML5 Drag & Drop API

ブラウザ標準のHTML5 Drag & Drop APIを使って、外部ライブラリなしでドラッグ操作を実装します。

open_with

draggable属性

HTML要素にdraggable="true"を付与

drag_indicator

イベントハンドラ

dragstart/dragover/dropの3つのイベント

data_object

DataTransfer

ドラッグ中のデータをDataTransferで受け渡し

drag-sort.js（ドラッグ&ドロップ処理） 
document.addEventListener('DOMContentLoaded', () => {
    const list = document.getElementById('todo-list');
    let draggedItem = null;

    // --- ドラッグ開始 ---
    list.addEventListener('dragstart', (e) => {
        draggedItem = e.target.closest('.todo-item');
        draggedItem.classList.add('dragging');
        e.dataTransfer.effectAllowed = 'move';
        e.dataTransfer.setData('text/plain', draggedItem.dataset.id);
    });

    // --- ドラッグ中（並べ替え位置の決定） ---
    list.addEventListener('dragover', (e) => {
        e.preventDefault();
        e.dataTransfer.dropEffect = 'move';

        const afterElement = getDragAfterElement(list, e.clientY);
        if (afterElement == null) {
            list.appendChild(draggedItem);
        } else {
            list.insertBefore(draggedItem, afterElement);
        }
    });

    // --- ドロップ完了 ---
    list.addEventListener('drop', (e) => {
        e.preventDefault();
        draggedItem.classList.remove('dragging');
        saveNewOrder();
    });

    list.addEventListener('dragend', () => {
        if (draggedItem) {
            draggedItem.classList.remove('dragging');
        }
    });

    /** マウス位置から挿入先の要素を取得 */
    function getDragAfterElement(container, y) {
        const items = [...container.querySelectorAll(
            '.todo-item:not(.dragging)')];
        return items.reduce((closest, child) => {
            const box = child.getBoundingClientRect();
            const offset = y - box.top - box.height / 2;
            if (offset < 0 && offset > closest.offset) {
                return { offset, element: child };
            }
            return closest;
        }, { offset: Number.NEGATIVE_INFINITY }).element;
    }

    /** 新しい並び順をサーバーに送信 */
    function saveNewOrder() {
        const items = list.querySelectorAll('.todo-item');
        const orderData = [...items].map((item, index) => ({
            id: parseInt(item.dataset.id),
            displayOrder: index + 1
        }));

        fetch('/api/todo/reorder', {
            method: 'PUT',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(orderData)
        });
    }
});
warningモバイル対応

HTML5 Drag & Drop APIはタッチデバイスでは動作しません。モバイル対応が必要な場合は、touch イベントを別途実装するか、SortableJSなどのライブラリを使用してください。

Section 30-3

並び順更新のREST API

ドラッグ完了後に新しい並び順をREST APIで一括更新するエンドポイントを実装します。

api

PUTエンドポイント

/api/todo/reorderで並び順を一括更新

sync

バッチ更新

foreachで全タスクのdisplay_orderを更新

security

権限チェック

自分のタスクのみ並べ替え可能に制限

TodoApiController.java（並び順更新API） 
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/todo")
public class TodoApiController {

    private final TodoService todoService;

    public TodoApiController(TodoService todoService) {
        this.todoService = todoService;
    }

    /**
     * タスクの表示順を一括更新
     */
    @PutMapping("/reorder")
    public ResponseEntity<Void> reorder(
            @RequestBody List<ReorderRequest> requests) {
        todoService.updateDisplayOrder(requests);
        return ResponseEntity.ok().build();
    }
}

// リクエストDTO
public class ReorderRequest {
    private Integer id;
    private Integer displayOrder;
    // getter/setter
}
TodoMapper.xml（display_order一括更新） 
<update id="updateDisplayOrder">
    UPDATE todo
    SET display_order = #{displayOrder}
    WHERE id = #{id}
</update>

<!-- バッチ更新（foreach版） -->
<update id="batchUpdateDisplayOrder">
    <foreach collection="list" item="item" separator=";">
        UPDATE todo
        SET display_order = #{item.displayOrder}
        WHERE id = #{item.id}
    </foreach>
</update>
lightbulbパフォーマンス最適化

タスク数が多い場合、foreachで個別UPDATEを発行すると遅くなります。
MyBatisのallowMultiQueries=trueを接続URLに追加することで、1回のリクエストで複数UPDATEをまとめて実行できます。

todo-list.html（ドラッグ対応リスト） 
<ul id="todo-list" class="todo-list">
    <li th:each="todo : ${todos}"
        th:data-id="${todo.id}"
        class="todo-item"
        draggable="true">

        <!-- ドラッグハンドル -->
        <span class="drag-handle">
            <span class="material-icons">drag_indicator</span>
        </span>

        <!-- タスク内容 -->
        <span th:text="${todo.title}" class="todo-title"></span>

        <!-- 操作ボタン -->
        <div class="todo-actions">
            <a th:href="@{/todo/{id}(id=${todo.id})}"
               class="btn btn-sm">詳細</a>
        </div>
    </li>
</ul>

<script th:src="@{/js/drag-sort.js}"></script>

AIプログラミング

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにドラッグ&ドロップでタスクの並び替えができる機能を追加してください。

【要件】
- todoテーブルにdisplay_order列（INT）を追加
- HTML5 Drag & Drop APIで並び替えUI（外部ライブラリ不要）
- ドロップ完了後にfetch APIで並び順をサーバーに送信
- PUT /api/todo/reorder で並び順を一括更新

【技術スタック】
- Spring Boot 3.x + MyBatis
- Thymeleaf + Vanilla JavaScript
- @RestControllerでJSON API
- @Transactionalでバッチ更新
gps_fixed ステップ別プロンプト 
# Step 1: テーブル変更
todoテーブルにdisplay_order列を追加し、既存データの表示順をID順で初期化するSQLを作成してください。

# Step 2: 並び順でソート取得
display_order ASCでタスクを取得するMyBatis SQLを追加してください。新規タスクのdisplay_orderは末尾（MAX+1）にしてください。

# Step 3: REST API
PUT /api/todo/reorderエンドポイントで、[{id, displayOrder}]のJSON配列を受け取り、一括更新するAPIを作成してください。

# Step 4: ドラッグ&ドロップJS
HTML5 Drag & Drop APIでリスト項目を並べ替え、ドロップ完了後にfetch APIで新しい順番をサーバーに送信するJavaScriptを作成してください。
lightbulb プロンプトのコツ

「ドラッグ中の要素は半透明にし、ドロップ先には青い線のインジケータを表示してください」のように、
視覚的フィードバックの仕様を明示すると、直感的なUIの実装が得られます。
