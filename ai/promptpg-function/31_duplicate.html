Section 31-1

複製コントローラー

既存タスクをワンクリックで複製するコントローラーを実装します。POSTリクエストで元タスクIDを受け取り、コピーを作成します。

content_copy

POST /todo/{id}/duplicate

既存タスクを複製する専用エンドポイント

flash_on

即時リダイレクト

複製後は新しいタスクの詳細画面に遷移

verified_user

権限チェック

自分のタスクのみ複製可能

TodoController.java（複製エンドポイント） 
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.servlet.mvc.support.RedirectAttributes;

@Controller
@RequestMapping("/todo")
public class TodoController {

    private final TodoService todoService;

    public TodoController(TodoService todoService) {
        this.todoService = todoService;
    }

    /**
     * タスクを複製
     */
    @PostMapping("/{id}/duplicate")
    public String duplicate(@PathVariable Integer id,
                            @AuthenticationPrincipal UserDetails user,
                            RedirectAttributes ra) {
        Todo newTodo = todoService.duplicate(id);
        ra.addFlashAttribute("message",
            "タスク「" + newTodo.getTitle() + "」を複製しました");
        return "redirect:/todo/" + newTodo.getId();
    }
}
lightbulbPOSTメソッドを使う理由

複製はデータを変更する操作なので、GETではなくPOSTメソッドを使います。
GETで実装するとブラウザの「戻る」やクローラーで意図しない複製が発生するリスクがあります。

3

4

ポイント: 複製時はid・created_at・completedをリセットして新規レコードとして保存。サブタスクがある場合はparent_idを新しい親IDに書き換えて連動複製します。

Section 31-2

データコピーロジック

元タスクのフィールドを新しいエンティティにコピーするディープコピーロジックを実装します。ステータスはリセットし、タイトルに「(コピー)」を付与します。

file_copy

フィールドコピー

タイトル・説明・カテゴリ等をコピー

restart_alt

ステータスリセット

完了状態はfalseに、日時は現在に設定

label

タイトル区別

「(コピー)」を末尾に追加して識別

TodoService.java（複製ロジック） 
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class TodoService {

    private final TodoMapper todoMapper;

    public TodoService(TodoMapper todoMapper) {
        this.todoMapper = todoMapper;
    }

    /**
     * タスクを複製（サブタスク含む）
     */
    @Transactional
    public Todo duplicate(Integer originalId) {
        Todo original = todoMapper.findById(originalId);
        if (original == null) {
            throw new IllegalArgumentException("タスクが見つかりません");
        }

        // 新しいタスクを作成（IDはnullで自動採番）
        Todo copy = new Todo();
        copy.setTitle(original.getTitle() + "（コピー）");
        copy.setDescription(original.getDescription());
        copy.setUserId(original.getUserId());
        copy.setCompleted(false);       // ステータスはリセット
        copy.setParentId(null);         // 親IDはコピーしない
        todoMapper.insert(copy);

        // サブタスクも複製
        duplicateSubtasks(originalId, copy.getId());

        return copy;
    }

    /** サブタスクを再帰的に複製 */
    private void duplicateSubtasks(Integer originalParentId,
                                    Integer newParentId) {
        List<Todo> subtasks = todoMapper.findSubtasks(originalParentId);
        for (Todo sub : subtasks) {
            Todo subCopy = new Todo();
            subCopy.setTitle(sub.getTitle());
            subCopy.setDescription(sub.getDescription());
            subCopy.setUserId(sub.getUserId());
            subCopy.setCompleted(false);
            subCopy.setParentId(newParentId);
            todoMapper.insert(subCopy);
        }
    }
}
TodoMapper.xml（複製用INSERT） 
<!-- SQL一発でコピーする方式（サブタスクなしの場合） -->
<insert id="duplicateById" useGeneratedKeys="true"
        keyProperty="id">
    INSERT INTO todo (title, description, completed, user_id,
                      parent_id, created_at)
    SELECT CONCAT(title, '（コピー）'), description, false,
           user_id, NULL, NOW()
    FROM todo
    WHERE id = #{originalId}
</insert>
warningコピーしないフィールド

id（自動採番）、created_at（現在日時）、completed（falseにリセット）は元タスクからコピーしません。
期限がある場合は、コピー元の期限をそのまま使うか、期限なしにするかをユーザーに選ばせるのも有効です。

Section 31-3

サブタスクの連動複製

親タスクを複製する際に、サブタスクも一緒にコピーする処理を実装します。新しい親IDへの紐付けがポイントです。

account_tree

親子関係の維持

コピーされたサブタスクは新しい親IDに紐付け

loop

ループ処理

元のサブタスク一覧を取得してINSERT

lock

トランザクション

親+サブタスクの複製を一つのトランザクションで

TodoMapper.xml（サブタスク一括複製SQL） 
<!-- サブタスクを一括コピー（INSERT...SELECT） -->
<insert id="duplicateSubtasks">
    INSERT INTO todo (title, description, completed, user_id,
                      parent_id, created_at)
    SELECT title, description, false, user_id,
           #{newParentId}, NOW()
    FROM todo
    WHERE parent_id = #{originalParentId}
    ORDER BY created_at ASC
</insert>
TodoController.java（複製オプション付き） 
/**
 * タスク複製（サブタスク連動オプション付き）
 */
@PostMapping("/{id}/duplicate")
public String duplicate(@PathVariable Integer id,
                        @RequestParam(defaultValue = "true")
                        boolean includeSubtasks,
                        RedirectAttributes ra) {

    Todo newTodo = todoService.duplicate(id, includeSubtasks);

    String msg = "タスクを複製しました";
    if (includeSubtasks) {
        int subCount = todoService.countSubtasks(newTodo.getId());
        msg += "（サブタスク" + subCount + "件を含む）";
    }
    ra.addFlashAttribute("message", msg);

    return "redirect:/todo/" + newTodo.getId();
}
lightbulbINSERT...SELECT の活用

MyBatisではINSERT INTO ... SELECT FROM ...を使うことで、Javaのループなしにデータベース側だけでデータをコピーできます。サブタスクが多い場合にパフォーマンスの差が出ます。

warning連打防止

複製ボタンをダブルクリックすると2つのコピーが作成されてしまいます。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにタスク複製機能を追加してください。

【要件】
- POST /todo/{id}/duplicateで既存タスクを複製
- タイトルに「（コピー）」を付与、ステータスはリセット
- サブタスクがある場合は連動して複製（オプション選択可）
- 一覧画面と詳細画面に複製ボタンを配置

【技術スタック】
- Spring Boot 3.x + MyBatis
- Thymeleaf
- @Transactionalで親+サブタスクの一括複製
gps_fixed ステップ別プロンプト 
# Step 1: 複製コントローラー
POST /todo/{id}/duplicateエンドポイントを作成してください。複製後は新しいタスクの詳細画面にリダイレクトしてください。

# Step 2: コピーロジック
元タスクのtitle（末尾に「（コピー）」付与）、description等をコピーし、completed=false、created_at=NOW()で新しいレコードをINSERTするサービスを作成してください。

# Step 3: サブタスク連動
サブタスクも連動して複製する処理を追加してください。INSERT...SELECTで一括コピーし、parent_idは新しい親のIDに書き換えてください。

# Step 4: UI実装
タスク一覧の各行にcontent_copyアイコンの複製ボタン、詳細画面には「サブタスクも複製する」チェックボックス付きの複製ボタンを配置してください。
lightbulb プロンプトのコツ

「タスクに添付ファイルやタグがある場合もコピーしてください」のように、
関連データの扱いを明示すると、抜け漏れのない実装が得られます。
