Section 5-1

確認ダイアログの必要性

削除ボタンをクリックしたときに、確認ダイアログを表示して誤操作を防ぎます。

code

JavaScript連携

ThymeleafとJavaScriptを組み合わせた実装

mouse

th:onclick

DELETE操作

データベースからのレコード削除

warning なぜ確認が必要？

削除は取り消しができない操作です。
誤クリックによるデータ消失を防ぐため、必ず確認ダイアログを表示しましょう。

Section 5-2

confirm()による実装

最もシンプルな方法は、JavaScriptのconfirm()関数を使う方法です。

index.html（シンプル版） 
<!-- 削除ボタン（シンプル版） -->
<form th:action="@{/todo/{id}/delete(id=${todo.id})}" method="post"
      onsubmit="return confirm('本当に削除しますか？');">
    <button type="submit" class="delete-btn">削除</button>
</form>
index.html（タイトル表示版） 
<!-- 削除ボタン（タイトル表示版） -->
<form th:action="@{/todo/{id}/delete(id=${todo.id})}" method="post"
      th:data-title="${todo.title}"
      onsubmit="return confirmDelete(this);">
    <button type="submit" class="delete-btn">削除</button>
</form>

<script>
function confirmDelete(form) {
    const title = form.dataset.title;
    return confirm('「' + title + '」を削除しますか？');
}
</script>

confirm()の動作

戻り値 ユーザー操作 結果
true 「OK」をクリック フォームが送信される（削除実行）
false 「キャンセル」をクリック フォーム送信がキャンセル
Section 5-3

削除処理の実装

TodoController.java 
/**
 * 削除処理
 */
@PostMapping("/{id}/delete")
public String delete(@PathVariable Integer id,
                     RedirectAttributes redirectAttributes) {

    todoService.delete(id);

    // 削除完了メッセージをフラッシュ属性で渡す
    redirectAttributes.addFlashAttribute("message", "ToDoを削除しました");

    return "redirect:/todo";
}
TodoService.java 
/**
 * ToDoを削除
 */
public void delete(Integer id) {
    todoRepository.deleteById(id);
}
TodoMapper.xml 
<delete id="deleteById">
    DELETE FROM todo WHERE id = #{id}
</delete>
Section 5-4

カスタムモーダルダイアログ

ブラウザ標準のconfirm()ではなく、デザインをカスタマイズできるモーダルダイアログを作成します。

index.html（カスタムモーダル） 
<!-- 削除ボタン -->
<button type="button" class="delete-btn"
        th:data-id="${todo.id}"
        th:data-title="${todo.title}"
        onclick="showDeleteModal(this)">
    削除
</button>

<!-- 削除確認モーダル -->
<div id="deleteModal" class="modal-overlay" style="display: none;">
    <div class="modal-content">
        <h3>削除確認</h3>
        <p>「<span id="deleteTitle"></span>」を削除しますか？</p>
        <p style="color: #dc2626; font-size: 0.9rem;">この操作は取り消せません。</p>
        <div style="margin-top: 1.5rem;">
            <form id="deleteForm" method="post" style="display: inline;">
                <button type="submit" class="delete-btn">削除する</button>
            </form>
            <button type="button" onclick="hideDeleteModal()"
                    style="margin-left: 0.5rem; padding: 4px 12px;">
                キャンセル
            </button>
        </div>
    </div>
</div>

<script th:inline="javascript">
function showDeleteModal(button) {
    const id = button.dataset.id;
    const title = button.dataset.title;

    document.getElementById('deleteTitle').textContent = title;
    document.getElementById('deleteForm').action = '/todo/' + id + '/delete';
    document.getElementById('deleteModal').style.display = 'flex';
}

function hideDeleteModal() {
    document.getElementById('deleteModal').style.display = 'none';
}

// モーダル外クリックで閉じる
document.getElementById('deleteModal').addEventListener('click', function(e) {
    if (e.target === this) {
        hideDeleteModal();
    }
});
</script>

<style>
.modal-overlay {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    background: rgba(0, 0, 0, 0.5);

    justify-content: center;
    align-items: center;
    z-index: 1000;
}
.modal-content {
    background: white;
    padding: 2rem;
    border-radius: 8px;
    text-align: center;
    max-width: 400px;
    box-shadow: 0 10px 25px rgba(0, 0, 0, 0.2);
}
.delete-btn {
    background: #ef4444;
    color: white;
    border: none;
    padding: 8px 16px;
    border-radius: 4px;
    cursor: pointer;
}
.delete-btn:hover {
    background: #dc2626;
}
</style>

scriptタグにth:inline="javascript"を付けると、
JavaScript内でThymeleaf変数を[[${変数名}]]の形式で使用できます。

フラッシュ属性（RedirectAttributes）

メソッド 説明
addFlashAttribute() リダイレクト先で1回だけ使えるデータを設定
addAttribute() URLパラメータとしてデータを渡す
lightbulb なぜフラッシュ属性？

削除後はリダイレクトするため、通常のModelではデータを渡せません。
フラッシュ属性を使うと、リダイレクト先で1回だけデータを受け取れます。
ブラウザを更新すると消えるため、メッセージ表示に最適です。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに削除確認ダイアログを追加してください。

【要件】
- 削除ボタンクリック時にJavaScriptで確認ダイアログを表示
- 「本当に削除しますか？」というメッセージを表示
- OKで削除実行、キャンセルで何もしない
- 可能であればモーダルダイアログで実装

【技術スタック】
- JavaScript (confirm() または カスタムモーダル)
- Thymeleaf
- Spring Boot 3.x (DELETE処理)
gps_fixed ステップ別プロンプト 
# Step 1: 簡易版（confirm使用）
削除ボタンにonclick="return confirm('本当に削除しますか？')"を追加してください。

# Step 2: モーダル版
カスタムモーダルダイアログを作成してください。削除対象のタイトルを表示し、確認・キャンセルボタンを配置してください。

# Step 3: 削除処理
TodoControllerにDELETE /{id}エンドポイントを追加し、削除後は一覧画面にリダイレクトしてください。
lightbulb プロンプトのコツ

「アクセシビリティを考慮し、ESCキーでモーダルを閉じられるようにしてください」のように、
UX要件を追加すると、より使いやすい実装が得られます。
