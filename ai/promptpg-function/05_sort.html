Section 4-3

Controllerの実装

TodoController.java 
/**
 * 一覧画面（検索・ソート機能付き）
 */
@GetMapping
public String index(@RequestParam(required = false) String keyword,
                    @RequestParam(required = false, defaultValue = "created_at") String sort,
                    @RequestParam(required = false, defaultValue = "desc") String order,
                    Model model) {

    // ソートカラムのバリデーション（SQLインジェクション対策）
    List<String> allowedSortColumns = Arrays.asList("id", "title", "author", "created_at");
    if (!allowedSortColumns.contains(sort)) {
        sort = "created_at";
    }

    // ソート順のバリデーション
    if (!order.equals("asc") && !order.equals("desc")) {
        order = "desc";
    }

    List<Todo> todoList = todoService.findAll(keyword, sort, order);

    model.addAttribute("todoList", todoList);
    model.addAttribute("keyword", keyword);
    model.addAttribute("sort", sort);
    model.addAttribute("order", order);

    return "todo/index";
}
warning セキュリティ対策

ソートカラム名はユーザー入力から来るため、
許可されたカラム名のみを受け付けるバリデーションが必要です。
これによりSQLインジェクション攻撃を防ぎます。

Section 4-4

動的ORDER BYの実装

TodoMapper.xml 
<select id="findByCondition" resultType="Todo">
    SELECT id, author, title, detail, completed, created_at, updated_at
    FROM todo
    <where>
        <if test="keyword != null and keyword != ''">
            title LIKE CONCAT('%', #{keyword}, '%')
        </if>
    </where>
    <!-- 動的ORDER BY -->
    ORDER BY
    <choose>
        <when test="sort == 'id'">id</when>
        <when test="sort == 'title'">title</when>
        <when test="sort == 'author'">author</when>
        <otherwise>created_at</otherwise>
    </choose>
    <if test="order == 'asc'">ASC</if>
    <if test="order == 'desc'">DESC</if>
</select>
TodoRepository.java 
@Mapper
public interface TodoRepository {

    List<Todo> findByCondition(
        @Param("keyword") String keyword,
        @Param("sort") String sort,
        @Param("order") String order
    );
}

MyBatisの動的SQLタグ

タグ 説明
<if> 条件が真の場合にSQL断片を含める
<choose><when><otherwise> switch文のような条件分岐
<where> WHEREを自動付与、不要なAND/ORを除去
Section 4-5

ソート方向の切り替え

同じカラムを再度クリックすると、昇順/降順が切り替わる仕組みを実装します。

TodoService.java 
@Service
public class TodoService {

    private final TodoRepository todoRepository;

    public TodoService(TodoRepository todoRepository) {
        this.todoRepository = todoRepository;
    }

    /**
     * 検索・ソート条件を指定して取得
     */
    public List<Todo> findAll(String keyword, String sort, String order) {
        return todoRepository.findByCondition(keyword, sort, order);
    }
}

ロジックの説明

<!-- Thymeleafでの切り替えロジック -->
order=${sort == 'id' and order == 'asc'} ? 'desc' : 'asc'

解説：
1. sort == 'id' → 現在のソートがidカラムか？
2. order == 'asc' → 現在が昇順か？
3. 両方trueなら → 降順(desc)に切り替え
4. それ以外なら → 昇順(asc)に切り替え
lightbulb 動作確認

以下を確認しましょう：
・ヘッダークリックでソートされる
・同じヘッダーを再クリックで昇順/降順切り替え
・矢印アイコンが正しく表示される
・検索結果を維持したままソートできる

AIプログラミング

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにソート機能を追加してください。

【要件】
- テーブルヘッダーをクリックでソート可能にする
- ソート対象：作成日、タイトル、ステータス
- 昇順/降順の切り替えが可能
- 現在のソート状態を視覚的に表示（▲▼アイコン）
- URLパラメータでソート状態を保持

【技術スタック】
- Spring Boot 3.x (@RequestParam)
- MyBatis (ORDER BY動的SQL)
- Thymeleaf
gps_fixed ステップ別プロンプト 
# Step 1: Mapperの修正
TodoMapper.xmlのSELECT文にORDER BY句を動的に追加してください。sortとdirectionパラメータを受け取るようにしてください。

# Step 2: Controllerの修正
TodoControllerに@RequestParamでsortとdirectionを受け取り、Serviceに渡すようにしてください。

# Step 3: ビューの修正
テーブルヘッダーにソートリンクを追加し、現在のソート状態に応じてアイコンを表示してください。
lightbulb プロンプトのコツ

「ORDER BY句にはホワイトリスト方式で安全なカラム名のみ許可してください」のように、
セキュリティ要件を明示すると、SQLインジェクション対策済みの実装が得られます。
