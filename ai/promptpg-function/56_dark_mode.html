Section 56-1

CSS変数でテーマ定義

ライトモードとダークモードの色をCSS カスタムプロパティ（変数）で一元管理します。:rootにライト用、[data-theme="dark"]にダーク用の変数を定義することで、テーマ切替時にすべてのUIが連動して切り替わります。

palette

CSS変数の基本

--bg-primary、--text-primaryなど意味のある変数名で色を管理

dark_mode

ダークテーマ定義

data-theme属性でダーク用の変数値を上書き

brush

既存CSSの書き換え

ハードコードされた色を変数参照に置き換え

    --input-border: #d1d5db;

    --nav-text: #e2e8f0;

lightbulbCSS変数を使う理由

CSS変数（カスタムプロパティ）を使うと、テーマ切替時に変数値だけ差し替えるだけで全体の色が変わります。
各要素のスタイルを個別に書き換える必要がないため、メンテナンス性が大幅に向上します。

    <div class="nav-right">

theme-toggle.js（テーマ切替ロジック） 
document.addEventListener('DOMContentLoaded', () => {
    const toggleBtn = document.getElementById('theme-toggle');
    const themeIcon = document.getElementById('theme-icon');
    const htmlEl = document.documentElement;

    /**
     * テーマを適用する
     * @param {string} theme - 'light' または 'dark'
     */
    function applyTheme(theme) {
        if (theme === 'dark') {
            htmlEl.setAttribute('data-theme', 'dark');
            themeIcon.textContent = 'light_mode';   // 太陽アイコン
            toggleBtn.title = 'ライトモードに切替';
        } else {
            htmlEl.removeAttribute('data-theme');
            themeIcon.textContent = 'dark_mode';     // 月アイコン
            toggleBtn.title = 'ダークモードに切替';
        }
    }

    /**
     * 現在のテーマを取得
     */
    function getCurrentTheme() {
        return htmlEl.getAttribute('data-theme') === 'dark'
            ? 'dark' : 'light';
    }

    // ボタンクリックで切替
    toggleBtn.addEventListener('click', () => {
        const newTheme = getCurrentTheme() === 'dark'
            ? 'light' : 'dark';
        applyTheme(newTheme);
    });
});
style.css（トグルボタンのスタイル） 
/* テーマ切替ボタン */
.btn-icon {
    background: none;
    border: none;
    cursor: pointer;
    padding: 8px;
    border-radius: 50%;
    color: var(--text-primary);
    transition: background-color 0.2s, transform 0.2s;
}

.btn-icon:hover {
    background-color: var(--border-color);
    transform: scale(1.1);
}

/* テーマ切替時のアニメーション */
html {
    transition: background-color 0.3s ease,
                color 0.3s ease;
}

.card, .todo-item, input, select, textarea {
    transition: background-color 0.3s ease,
                border-color 0.3s ease,
                color 0.3s ease;
}
lightbulbtransitionでなめらかに切替

transitionプロパティを使うことで、テーマ切替時に色がなめらかにアニメーションします。
急激な色変化はユーザー体験を損なうため、0.3秒程度のイージングを入れるのが定番です。

Section 56-3

LocalStorageで設定保存

ユーザーが選んだテーマ設定をLocalStorageに保存し、次回アクセス時にも同じテーマで表示されるようにします。ページ読み込み時にLocalStorageから設定を復元します。

save

設定の永続化

LocalStorageにテーマ名を保存して次回起動時に復元

refresh

初期読み込み時の復元

DOMContentLoaded時に保存済みテーマを適用

flash_on

FOUCの防止

ページ描画前にテーマを適用してちらつきを防ぐ

theme-toggle.js（LocalStorage対応版） 
const STORAGE_KEY = 'todo-app-theme';

document.addEventListener('DOMContentLoaded', () => {
    const toggleBtn = document.getElementById('theme-toggle');
    const themeIcon = document.getElementById('theme-icon');
    const htmlEl = document.documentElement;

    function applyTheme(theme) {
        if (theme === 'dark') {
            htmlEl.setAttribute('data-theme', 'dark');
            themeIcon.textContent = 'light_mode';
            toggleBtn.title = 'ライトモードに切替';
        } else {
            htmlEl.removeAttribute('data-theme');
            themeIcon.textContent = 'dark_mode';
            toggleBtn.title = 'ダークモードに切替';
        }
        // LocalStorageに保存
        localStorage.setItem(STORAGE_KEY, theme);
    }

    function getCurrentTheme() {
        return htmlEl.getAttribute('data-theme') === 'dark'
            ? 'dark' : 'light';
    }

    // 保存済みテーマを復元
    const savedTheme = localStorage.getItem(STORAGE_KEY);
    if (savedTheme) {
        applyTheme(savedTheme);
    }

    toggleBtn.addEventListener('click', () => {
        const newTheme = getCurrentTheme() === 'dark'
            ? 'light' : 'dark';
        applyTheme(newTheme);
    });
});

lightbulbFOUC（Flash of Unstyled Content）とは

ダークモードユーザーがページを読み込んだ際に、一瞬ライトモードが表示されてからダークに切り替わる現象です。
<head>内にインラインスクリプトを配置し、CSSの読み込みより前にテーマ属性を設定することで防止できます。

Section 56-4

prefers-color-scheme対応

OSのダークモード設定をprefers-color-schemeメディアクエリで検出し、ユーザーが明示的に設定していない場合はOS設定に従うようにします。

settings_suggest

OS設定の検出

window.matchMediaでOSのダークモード設定を取得

sync

リアルタイム追従

OSの設定変更をリスナーで検知して即座に反映

priority_high

優先度の設計

ユーザー設定 > OS設定 > デフォルトの優先順位

theme-toggle.js（OS設定対応・完全版） 
const STORAGE_KEY = 'todo-app-theme';

document.addEventListener('DOMContentLoaded', () => {
    const toggleBtn = document.getElementById('theme-toggle');
    const themeIcon = document.getElementById('theme-icon');
    const htmlEl = document.documentElement;

    // OSのダークモード設定を検出
    const prefersDark = window.matchMedia(
        '(prefers-color-scheme: dark)'
    );

    function applyTheme(theme) {
        if (theme === 'dark') {
            htmlEl.setAttribute('data-theme', 'dark');
            themeIcon.textContent = 'light_mode';
            toggleBtn.title = 'ライトモードに切替';
        } else {
            htmlEl.removeAttribute('data-theme');
            themeIcon.textContent = 'dark_mode';
            toggleBtn.title = 'ダークモードに切替';
        }
    }

    /**
     * テーマを決定する（優先順位）
     * 1. LocalStorageのユーザー設定
     * 2. OSのprefers-color-scheme
     * 3. デフォルト（light）
     */
    function resolveTheme() {
        const saved = localStorage.getItem(STORAGE_KEY);
        if (saved) return saved;
        return prefersDark.matches ? 'dark' : 'light';
    }

    // 初期テーマを適用
    applyTheme(resolveTheme());

    // ボタンクリックで切替（ユーザー設定として保存）
    toggleBtn.addEventListener('click', () => {
        const current = htmlEl.getAttribute('data-theme')
            === 'dark' ? 'dark' : 'light';
        const newTheme = current === 'dark' ? 'light' : 'dark';
        localStorage.setItem(STORAGE_KEY, newTheme);
        applyTheme(newTheme);
    });

    // OS設定の変更をリアルタイムで検知
    prefersDark.addEventListener('change', (e) => {
        // ユーザーが明示的に設定していない場合のみ追従
        if (!localStorage.getItem(STORAGE_KEY)) {
            applyTheme(e.matches ? 'dark' : 'light');
        }
    });
});
FOUC防止（OS設定対応版） 
<script>
    (function() {
        var saved = localStorage.getItem('todo-app-theme');
        var theme = saved
            || (window.matchMedia('(prefers-color-scheme: dark)')
                .matches ? 'dark' : 'light');
        if (theme === 'dark') {
            document.documentElement.setAttribute(
                'data-theme', 'dark'
            );
        }
    })();
</script>

テーマ解決の優先順位

優先度ソース説明
1（最高）LocalStorageユーザーがトグルボタンで明示的に選択した設定
2prefers-color-schemeOSのダークモード設定に自動追従
3（最低）デフォルトいずれも未設定の場合はライトモード
lightbulb設定リセット機能

「OS設定に戻す」ボタンを追加する場合は、localStorage.removeItem(STORAGE_KEY)を呼び出してからテーマを再解決します。
これによりユーザーはいつでもOS設定への追従に戻すことができます。

edit_note 基本プロンプト 
Spring Boot 3.x + ThymeleafのToDoアプリにダークモード切替機能を追加してください。

【要件】
- CSS変数（カスタムプロパティ）でライト/ダークの色を管理
- トグルボタンでテーマを切り替え（月/太陽アイコン）
- LocalStorageにユーザーの選択を保存
- OSのprefers-color-schemeに自動対応
- FOUC（ちらつき）防止のhead内インラインスクリプト

【技術スタック】
- CSS Custom Properties（:root / [data-theme="dark"]）
- JavaScript（localStorage, matchMedia）
- Material Icons（dark_mode / light_mode）
gps_fixed ステップ別プロンプト 
# Step 1: CSS変数の定義
既存のstyle.cssのハードコードされた色をCSS変数に置き換え、:rootにライト用、[data-theme="dark"]にダーク用の変数を定義してください。

# Step 2: テーマ切替ボタン
ナビゲーションバーにダークモード切替ボタンを追加し、クリックでdata-theme属性を切り替えるJavaScriptを作成してください。

# Step 3: LocalStorage保存
テーマ設定をLocalStorageに保存し、次回アクセス時に復元する機能を追加してください。head内にFOUC防止のインラインスクリプトも配置してください。

# Step 4: OS設定対応
prefers-color-schemeメディアクエリでOS設定を検出し、ユーザー設定がない場合はOS設定に追従するようにしてください。

「FOUCを防止するため、CSSよりも前にhead内にインラインスクリプトを配置してください」のように、
ユーザー体験に関わる実装上の注意点を明示すると、より実用的なコードが得られます。
