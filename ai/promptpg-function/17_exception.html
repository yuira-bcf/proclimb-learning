TodoNotFoundException.java 
public class TodoNotFoundException extends RuntimeException {

    private final Integer todoId;

    public TodoNotFoundException(Integer todoId) {
        super("ToDo not found: id=" + todoId);
        this.todoId = todoId;
    }

    public Integer getTodoId() {
        return todoId;
    }
}
BusinessException.java（業務例外） 
public class BusinessException extends RuntimeException {

    private final String errorCode;

    public BusinessException(String errorCode, String message) {
        super(message);
        this.errorCode = errorCode;
    }

    public String getErrorCode() {
        return errorCode;
    }
}

// 使用例
throw new BusinessException("E001", "この操作は許可されていません");
Section 16-2

@ControllerAdviceで共通処理

GlobalExceptionHandler.java 
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.ui.Model;

@ControllerAdvice
public class GlobalExceptionHandler {

    /**
     * ToDoが見つからない場合
     */
    @ExceptionHandler(TodoNotFoundException.class)
    public String handleTodoNotFound(TodoNotFoundException ex, Model model) {
        model.addAttribute("errorMessage", "指定されたToDoが見つかりません");
        model.addAttribute("todoId", ex.getTodoId());
        return "error/404";
    }

    /**
     * 業務例外
     */
    @ExceptionHandler(BusinessException.class)
    public String handleBusinessException(BusinessException ex, Model model) {
        model.addAttribute("errorCode", ex.getErrorCode());
        model.addAttribute("errorMessage", ex.getMessage());
        return "error/business";
    }

    /**
     * その他の例外（フォールバック）
     */
    @ExceptionHandler(Exception.class)
    public String handleException(Exception ex, Model model) {
        model.addAttribute("errorMessage", "システムエラーが発生しました");
        // 本番環境では詳細を表示しない
        // model.addAttribute("detail", ex.getMessage());
        return "error/500";
    }
}
RestExceptionHandler.java（API用） 
import org.springframework.web.bind.annotation.RestControllerAdvice;
import org.springframework.http.ResponseEntity;
import org.springframework.http.HttpStatus;

@RestControllerAdvice(basePackages = "com.example.api")
public class RestExceptionHandler {

    @ExceptionHandler(TodoNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleNotFound(TodoNotFoundException ex) {
        ErrorResponse error = new ErrorResponse(
            "NOT_FOUND",
            ex.getMessage()
        );
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidation(
            MethodArgumentNotValidException ex) {
        String message = ex.getBindingResult()
            .getFieldErrors()
            .stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.joining(", "));

        ErrorResponse error = new ErrorResponse("VALIDATION_ERROR", message);
        return ResponseEntity.badRequest().body(error);
    }
}

例外処理のポイント

ポイント説明
例外の階層化業務例外とシステム例外を分ける
適切なログ出力例外発生時はログを記録する
ユーザーへの通知技術的詳細は隠し、分かりやすいメッセージを表示
復旧手段の提示一覧に戻るリンクなどを用意
warning本番環境での注意

スタックトレースやシステム内部情報をユーザーに表示しないでください。
攻撃者に情報を与えてしまう可能性があります。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに例外ハンドリング機能を追加してください。

【要件】
- @ControllerAdviceでグローバル例外ハンドリング
- 404エラー用のカスタムエラーページ
- 500エラー用のカスタムエラーページ
- ビジネス例外（TodoNotFoundException等）の定義
- REST APIの場合はJSON形式でエラーレスポンス
- エラーログの出力

【技術スタック】
- Spring Boot 3.x (@ControllerAdvice, @ExceptionHandler)
- Thymeleaf (error.html)
- SLF4J (ログ出力)
gps_fixed ステップ別プロンプト 
# Step 1: カスタム例外クラスの作成
TodoNotFoundExceptionとBusinessExceptionを作成してください。HTTPステータスコードをアノテーションで指定してください。

# Step 2: GlobalExceptionHandlerの作成
@ControllerAdviceクラスを作成し、各例外タイプに対応する@ExceptionHandlerメソッドを実装してください。

# Step 3: エラーページの作成
templates/error/404.html, 500.htmlを作成し、ユーザーフレンドリーなエラー画面を表示してください。

# Step 4: REST API用エラーハンドリング
@RestControllerAdviceで/api/**パス用のJSON形式エラーレスポンスを実装してください。
