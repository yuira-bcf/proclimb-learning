TaskMergeController.java（マージ画面表示） 
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.servlet.mvc.support.RedirectAttributes;

@Controller
@RequestMapping("/todo/merge")
public class TaskMergeController {

    private final TaskMergeService mergeService;
    private final TodoMapper todoMapper;

    public TaskMergeController(TaskMergeService mergeService,
                               TodoMapper todoMapper) {
        this.mergeService = mergeService;
        this.todoMapper = todoMapper;
    }

    @GetMapping
    public String showMergeForm(Model model) {
        model.addAttribute("tasks", todoMapper.findAll());
        return "merge";
    }

    @PostMapping
    public String merge(@RequestParam Integer targetId,
                        @RequestParam Integer sourceId,
                        RedirectAttributes redirectAttributes) {
        if (targetId.equals(sourceId)) {
            redirectAttributes.addFlashAttribute("error",
                "同じタスクは選択できません");
            return "redirect:/todo/merge";
        }

        mergeService.merge(targetId, sourceId);
        redirectAttributes.addFlashAttribute("message",
            "タスクをマージしました");
        return "redirect:/todo/" + targetId;
    }
}
warning不可逆操作

マージ操作は元に戻せない破壊的な操作です。confirm()ダイアログに加え、マージ後のプレビュー表示を実装することで、誤操作を防止します。

1
マージ元タスク複数選択 + マージ先タスク指定

2
TaskMergeService.merge() でデータ統合（説明文連結・優先度・期限）

3
サブタスク・タグ・依存関係をマージ先に付け替え

4
マージ元を削除 + 履歴記録

ポイント: @Transactionalで全操作を1トランザクションにまとめ、途中エラー時は全てロールバック。自己依存が発生した場合は自動削除。

Section 38-2

データ統合ロジック

マージ元タスクの説明文やメモをマージ先タスクに統合するロジックを実装します。テキスト情報を連結し、重要な情報を失わないようにします。

text_fields

説明文の連結

マージ元の説明文をマージ先に追記

low_priority

優先度の統合

より高い優先度を採用

event

期限の統合

より早い期限を採用

TaskMergeService.java（マージロジック） 
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class TaskMergeService {

    private final TodoMapper todoMapper;
    private final SubtaskMapper subtaskMapper;
    private final TaskDependencyMapper dependencyMapper;
    private final MergeHistoryMapper mergeHistoryMapper;

    public TaskMergeService(TodoMapper todoMapper,
                            SubtaskMapper subtaskMapper,
                            TaskDependencyMapper dependencyMapper,
                            MergeHistoryMapper mergeHistoryMapper) {
        this.todoMapper = todoMapper;
        this.subtaskMapper = subtaskMapper;
        this.dependencyMapper = dependencyMapper;
        this.mergeHistoryMapper = mergeHistoryMapper;
    }

    /**
     * タスクをマージする
     * @param targetId マージ先（残すタスク）のID
     * @param sourceId マージ元（削除するタスク）のID
     */
    @Transactional
    public void merge(Integer targetId, Integer sourceId) {
        Todo target = todoMapper.findById(targetId);
        Todo source = todoMapper.findById(sourceId);

        // 1. テキスト情報の統合
        mergeTextFields(target, source);

        // 2. 優先度の統合（高い方を採用）
        if (source.getPriority() > target.getPriority()) {
            target.setPriority(source.getPriority());
        }

        // 3. 期限の統合（早い方を採用）
        if (source.getDeadline() != null) {
            if (target.getDeadline() == null ||
                source.getDeadline().isBefore(target.getDeadline())) {
                target.setDeadline(source.getDeadline());
            }
        }

        // 4. 関連データの付け替え
        reassociateData(targetId, sourceId);

        // 5. マージ先を更新
        todoMapper.update(target);

        // 6. マージ履歴を記録
        saveMergeHistory(targetId, sourceId, source.getTitle());

        // 7. マージ元を削除
        todoMapper.deleteById(sourceId);
    }

    private void mergeTextFields(Todo target, Todo source) {
        // 説明文の連結
        if (source.getDescription() != null
                && !source.getDescription().isBlank()) {
            String merged = (target.getDescription() != null
                ? target.getDescription() : "")
                + "\n\n--- マージ元 (#" + source.getId()
                + ": " + source.getTitle() + ") ---\n"
                + source.getDescription();
            target.setDescription(merged.trim());
        }
    }
}
lightbulb@Transactionalの重要性

マージ処理は複数テーブルにまたがる操作のため、@Transactionalで全体を1つのトランザクションにまとめます。途中でエラーが発生した場合は全てロールバックされ、データの整合性が保たれます。

Section 38-3

関連データの付け替え

マージ元タスクに紐づくサブタスク・タグ・依存関係などの関連データを、マージ先タスクに付け替える処理を実装します。

subdirectory_arrow_right

サブタスク移管

マージ元のサブタスクをマージ先に付け替え

label

タグの統合

重複を除いてタグを統合

link

依存関係の移管

依存関係をマージ先に付け替え

TaskMergeService.java（関連データの付け替え） 
/**
 * 関連データをマージ元からマージ先に付け替え
 */
private void reassociateData(Integer targetId, Integer sourceId) {
    // サブタスクの付け替え
    subtaskMapper.updateParentId(sourceId, targetId);

    // タグの統合（重複除外）
    List<Integer> targetTagIds = todoMapper.findTagIds(targetId);
    List<Integer> sourceTagIds = todoMapper.findTagIds(sourceId);
    for (Integer tagId : sourceTagIds) {
        if (!targetTagIds.contains(tagId)) {
            todoMapper.insertTodoTag(targetId, tagId);
        }
    }

    // 依存関係の付け替え（マージ元→マージ先）
    dependencyMapper.replacePredecessor(sourceId, targetId);
    dependencyMapper.replaceSuccessor(sourceId, targetId);
    // 自己依存が発生した場合は削除
    dependencyMapper.deleteSelfReference(targetId);
}
Mapper.xml（付け替えSQL） 
<!-- SubtaskMapper.xml -->
<update id="updateParentId">
    UPDATE subtask
    SET parent_id = #{newParentId}
    WHERE parent_id = #{oldParentId}
</update>

<!-- TaskDependencyMapper.xml -->
<update id="replacePredecessor">
    UPDATE task_dependencies
    SET predecessor_id = #{newId}
    WHERE predecessor_id = #{oldId}
      AND successor_id != #{newId}
</update>

<update id="replaceSuccessor">
    UPDATE task_dependencies
    SET successor_id = #{newId}
    WHERE successor_id = #{oldId}
      AND predecessor_id != #{newId}
</update>

<delete id="deleteSelfReference">
    DELETE FROM task_dependencies
    WHERE predecessor_id = #{taskId}
      AND successor_id = #{taskId}
</delete>
warning自己依存の防止

依存関係を付け替えると、マージ先タスクが自分自身に依存する状態が発生する可能性があります。付け替え後にdeleteSelfReferenceで自己依存レコードを削除してください。

Section 38-4

マージ履歴の記録

マージ操作の履歴をテーブルに記録し、いつ・どのタスクがマージされたかを追跡できるようにします。監査やトラブルシューティングに役立ちます。

history

履歴テーブル

マージ先/元ID、タイトル、実行日時を記録

visibility

履歴表示

タスク詳細画面にマージ履歴を一覧表示

person

実行者の記録

マージを実行したユーザー情報も記録

schema.sql（マージ履歴テーブル） 
CREATE TABLE merge_history (
    id INT AUTO_INCREMENT PRIMARY KEY,
    target_id INT NOT NULL,
    source_id INT NOT NULL,
    source_title VARCHAR(255) NOT NULL,
    merged_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    merged_by VARCHAR(50),
    FOREIGN KEY (target_id) REFERENCES todo(id)
);
MergeHistoryMapper.java / .xml 
// --- MergeHistoryMapper.java ---
@Mapper
public interface MergeHistoryMapper {

    void insert(@Param("targetId") Integer targetId,
                @Param("sourceId") Integer sourceId,
                @Param("sourceTitle") String sourceTitle,
                @Param("mergedBy") String mergedBy);

    List<MergeHistory> findByTargetId(
        @Param("targetId") Integer targetId);
}

// --- MergeHistoryMapper.xml ---
<insert id="insert">
    INSERT INTO merge_history
        (target_id, source_id, source_title, merged_by)
    VALUES
        (#{targetId}, #{sourceId}, #{sourceTitle}, #{mergedBy})
</insert>

<select id="findByTargetId" resultType="MergeHistory">
    SELECT id, target_id, source_id, source_title,
           merged_at, merged_by
    FROM merge_history
    WHERE target_id = #{targetId}
    ORDER BY merged_at DESC
</select>

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに重複タスクを統合するマージ機能を追加してください。

【要件】
- マージ先（残す）とマージ元（削除される）を選択するUI
- 説明文の連結、優先度は高い方・期限は早い方を採用
- サブタスク・タグ・依存関係をマージ先に付け替え
- merge_historyテーブルにマージ履歴を記録
- @Transactionalでデータ整合性を保証

【技術スタック】
- Spring Boot 3.x + MyBatis
- Thymeleaf
- @Transactional
gps_fixed ステップ別プロンプト 
# Step 1: マージ画面
マージ先とマージ元のタスクをセレクトボックスで選択する画面を作成してください。確認ダイアログ付きにしてください。

# Step 2: データ統合ロジック
マージ元の説明文をマージ先に追記、優先度は高い方を採用、期限は早い方を採用するServiceメソッドを実装してください。

# Step 3: 関連データの付け替え
サブタスク・タグ・依存関係をマージ元からマージ先に移管するSQLを作成してください。自己依存が発生した場合は削除してください。

# Step 4: マージ履歴
merge_historyテーブルを作成し、マージ実行時に履歴を記録してください。タスク詳細画面に履歴を表示してください。

「依存関係の付け替えで自己依存が発生した場合は自動削除してください」のように、
データ整合性の問題を具体的に指摘すると、堅牢な実装が得られます。
