Section 61-1

操作履歴テーブル設計

タスクに対する全ての操作を記録する操作履歴テーブルを設計します。操作の種類、対象タスク、変更前後のデータをJSON形式で保存し、Undoに必要な情報を保持します。

history

操作履歴テーブル

操作種別、対象ID、変更前後のデータを記録

data_object

JSON形式保存

変更前後のデータをJSON文字列として保存

undo

Undo可能判定

操作から一定時間内のみUndoを許可

schema.sql（操作履歴テーブル）
CREATE TABLE operation_history (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    operation_type VARCHAR(30) NOT NULL,  -- CREATE, UPDATE, DELETE, STATUS_CHANGE
    target_table VARCHAR(50) NOT NULL,    -- 'todo'
    target_id INT NOT NULL,
    before_data JSON,                     -- 変更前データ（JSON）
    after_data JSON,                      -- 変更後データ（JSON）
    is_undone BOOLEAN DEFAULT FALSE,      -- Undo済みフラグ
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
OperationHistory.java（エンティティ）
import java.time.LocalDateTime;

public class OperationHistory {
    private Integer id;
    private Integer userId;
    private String operationType;   // CREATE, UPDATE, DELETE
    private String targetTable;     // "todo"
    private Integer targetId;
    private String beforeData;      // JSON文字列
    private String afterData;       // JSON文字列
    private boolean undone;
    private LocalDateTime createdAt;
    // getter/setter
}
lightbulbJSON型カラムの利点

変更前後のデータをJSON形式で保存することで、タスクのどのフィールドが変更されたかを後から特定できます。また、テーブル構造が変わっても履歴データに影響しません。

Section 61-2

変更前後データの保存

タスクの更新・削除時に変更前のデータをJSON化して履歴テーブルに保存します。ObjectMapperを使ってJavaオブジェクトをJSON文字列に変換します。

compare_arrows

Before/After記録

更新前のデータを取得してからDB更新を実行

transform

ObjectMapper変換

JavaオブジェクトをJSON文字列にシリアライズ

layers

サービス層で記録

トランザクション内で更新と履歴保存を一括実行

HistoryService.java（履歴記録）
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.stereotype.Service;

@Service
public class HistoryService {

    private final HistoryMapper historyMapper;
    private final ObjectMapper objectMapper;

    public HistoryService(HistoryMapper historyMapper,
                           ObjectMapper objectMapper) {
        this.historyMapper = historyMapper;
        this.objectMapper = objectMapper;
    }

    /**
     * 操作履歴を記録する
     */
    public void record(Integer userId, String operationType,
                        String targetTable, Integer targetId,
                        Object before, Object after) {
        try {
            OperationHistory history = new OperationHistory();
            history.setUserId(userId);
            history.setOperationType(operationType);
            history.setTargetTable(targetTable);
            history.setTargetId(targetId);
            history.setBeforeData(before != null
                ? objectMapper.writeValueAsString(before) : null);
            history.setAfterData(after != null
                ? objectMapper.writeValueAsString(after) : null);
            historyMapper.insert(history);
        } catch (Exception e) {
            // 履歴記録の失敗でビジネスロジックを止めない
            e.printStackTrace();
        }
    }
}
TodoService.java（履歴記録付き更新）
@Service
public class TodoService {

    private final TodoMapper todoMapper;
    private final HistoryService historyService;

    @Transactional
    public void updateTodo(Todo newData, Integer userId) {
        // 変更前データを取得
        Todo before = todoMapper.findById(newData.getId());

        // DB更新
        todoMapper.update(newData);

        // 操作履歴を記録
        historyService.record(userId, "UPDATE",
            "todo", newData.getId(), before, newData);
    }

    @Transactional
    public void deleteTodo(Integer todoId, Integer userId) {
        // 削除前データを取得（Undo用）
        Todo before = todoMapper.findById(todoId);

        todoMapper.deleteById(todoId);

        // 削除の履歴を記録（afterはnull）
        historyService.record(userId, "DELETE",
            "todo", todoId, before, null);
    }
}
lightbulb履歴記録のエラーハンドリング

履歴記録が失敗しても本来の操作（更新・削除）は正常に完了させるべきです。try-catchで例外を吸収し、ログに記録する設計にしています。

Section 61-3

Undoコントローラー

操作履歴のbeforeDataを使って、直前の操作を取り消す（Undo）コントローラーを実装します。Undo可能な期間を制限し、安全に元に戻します。

undo

Undo処理

beforeDataからオブジェクトを復元してDBを更新

timer

時間制限

操作から5分以内のみUndoを許可

replay

復元ロジック

UPDATE→beforeDataで上書き、DELETE→再INSERT

UndoController.java
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.web.bind.annotation.*;
import java.time.LocalDateTime;
import java.util.Map;

@RestController
@RequestMapping("/api/undo")
public class UndoController {

    private static final int UNDO_LIMIT_MINUTES = 5;

    private final HistoryMapper historyMapper;
    private final TodoMapper todoMapper;
    private final ObjectMapper objectMapper;

    public UndoController(HistoryMapper historyMapper,
                           TodoMapper todoMapper,
                           ObjectMapper objectMapper) {
        this.historyMapper = historyMapper;
        this.todoMapper = todoMapper;
        this.objectMapper = objectMapper;
    }

    @PostMapping("/{historyId}")
    public Map<String, Object> undo(@PathVariable Integer historyId) {
        OperationHistory history = historyMapper.findById(historyId);

        // Undo可能かチェック
        if (history == null || history.isUndone()) {
            return Map.of("success", false,
                "message", "この操作は取り消せません");
        }

        LocalDateTime limit = LocalDateTime.now()
            .minusMinutes(UNDO_LIMIT_MINUTES);
        if (history.getCreatedAt().isBefore(limit)) {
            return Map.of("success", false,
                "message", "Undo可能な時間を過ぎています（5分以内）");
        }

        try {
            switch (history.getOperationType()) {
                case "UPDATE":
                    // 変更前データで上書き
                    Todo before = objectMapper.readValue(
                        history.getBeforeData(), Todo.class);
                    todoMapper.update(before);
                    break;
                case "DELETE":
                    // 削除されたデータを再INSERT
                    Todo deleted = objectMapper.readValue(
                        history.getBeforeData(), Todo.class);
                    todoMapper.insert(deleted);
                    break;
            }

            // Undo済みフラグを立てる
            historyMapper.markAsUndone(historyId);

            return Map.of("success", true,
                "message", "操作を取り消しました");

        } catch (Exception e) {
            return Map.of("success", false,
                "message", "Undoに失敗しました: " + e.getMessage());
        }
    }
}
lightbulbUndoの時間制限

Undoを無制限に許可するとデータの整合性が崩れるリスクがあります。5分以内など時間制限を設けることで、「うっかり操作」の取り消しに限定しています。

history.js（Undo実行）
async function undoAction(historyId) {
    if (!confirm('この操作を元に戻しますか？')) return;

    const response = await fetch(`/api/undo/${historyId}`, {
        method: 'POST'
    });
    const result = await response.json();

    if (result.success) {
        alert(result.message);
        location.reload(); // ページをリロードして反映
    } else {
        alert('エラー: ' + result.message);
    }
}

AIプログラミング

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、この機能を実装できます。

edit_note 基本プロンプト
Spring Boot 3.xのToDoアプリに操作履歴・Undo機能を追加してください。

【要件】
- operation_historyテーブルで操作種別・変更前後データ(JSON)を記録
- タスクの更新・削除時に変更前データを保存
- /api/undo/{historyId}でUndoを実行（5分以内制限）
- タイムライン形式の操作履歴画面（Undoボタン付き）

【技術スタック】
- Spring Boot 3.x + MyBatis
- JSON型カラム（MySQL）
- ObjectMapper（Java↔JSON変換）
- @Transactional（更新+履歴保存の一括実行）
gps_fixed ステップ別プロンプト
# Step 1: 操作履歴テーブル
operation_historyテーブルを作成してください。操作種別、対象テーブル、対象ID、変更前後のJSON、Undo済みフラグを含めてください。

# Step 2: 変更前後データの保存
TodoServiceの更新・削除メソッドで変更前のデータをObjectMapperでJSON化し、HistoryServiceで履歴テーブルに保存する仕組みを作成してください。

# Step 3: Undoコントローラー
/api/undo/{historyId}でUndoを実行するRESTコントローラーを作成してください。5分以内制限と二重Undo防止を含めてください。

# Step 4: タイムラインUI
操作履歴をタイムライン形式で表示するThymeleafテンプレートを作成し、各項目にUndoボタンを配置してください。
lightbulbプロンプトのコツ

「変更前データのJSON化にはObjectMapperを使い、履歴記録が失敗してもメインの操作は止めないようにしてください」のように、エラーハンドリング方針を明示すると安全な実装が得られます。
