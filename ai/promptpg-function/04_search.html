lightbulb GETメソッドを使う理由

検索はGETメソッドを使用します。これにより：
・URLにキーワードが含まれ、ブックマークや共有が可能
・ブラウザの戻るボタンで検索結果に戻れる

Section 3-3

Controllerの実装

TodoController.java 
/**
 * 一覧画面（検索機能付き）
 */
@GetMapping
public String index(@RequestParam(required = false) String keyword,
                    Model model) {

    List<Todo> todoList;

    if (keyword != null && !keyword.isEmpty()) {
        // キーワードがある場合は検索
        todoList = todoService.searchByTitle(keyword);
    } else {
        // キーワードがない場合は全件取得
        todoList = todoService.findAll();
    }

    model.addAttribute("todoList", todoList);
    model.addAttribute("keyword", keyword);  // 検索キーワードを保持

    return "todo/index";
}

@RequestParamの属性

属性 説明 例
required 必須かどうか（デフォルトtrue） required = false
defaultValue パラメータがない時のデフォルト値 defaultValue = ""
name パラメータ名（変数名と異なる場合） name = "q"
Section 3-4

動的SQLの作成

MyBatisの<if>タグを使って、条件に応じたSQLを生成します。

TodoMapper.xml 
<!-- タイトルで検索 -->
<select id="searchByTitle" resultType="Todo">
    SELECT id, author, title, detail, completed, created_at, updated_at
    FROM todo
    WHERE title LIKE CONCAT('%', #{keyword}, '%')
    ORDER BY created_at DESC
</select>

<!-- 動的SQLで全件取得と検索を1つにまとめる場合 -->
<select id="findByCondition" resultType="Todo">
    SELECT id, author, title, detail, completed, created_at, updated_at
    FROM todo
    <where>
        <if test="keyword != null and keyword != ''">
            title LIKE CONCAT('%', #{keyword}, '%')
        </if>
    </where>
    ORDER BY created_at DESC
</select>
TodoRepository.java 
@Mapper
public interface TodoRepository {

    List<Todo> findAll();

    List<Todo> searchByTitle(@Param("keyword") String keyword);

    // または動的SQLを使う場合
    List<Todo> findByCondition(@Param("keyword") String keyword);
}
warning SQLインジェクション対策

MyBatisの#{}を使用するとプレースホルダーとして処理され、
SQLインジェクション対策が自動で行われます。
${}は直接文字列が埋め込まれるため、使用を避けてください。

Section 3-5

検索結果の表示

TodoService.java 
@Service
public class TodoService {

    private final TodoRepository todoRepository;

    public TodoService(TodoRepository todoRepository) {
        this.todoRepository = todoRepository;
    }

    public List<Todo> findAll() {
        return todoRepository.findAll();
    }

    public List<Todo> searchByTitle(String keyword) {
        return todoRepository.searchByTitle(keyword);
    }
}

AIプログラミング

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、この機能を実装できます。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにタイトル検索機能を追加してください。

【要件】
- 一覧画面の上部に検索フォームを追加
- タイトルの部分一致検索（LIKE検索）を実装
- GETメソッドでURLパラメータにキーワードを含める
- 検索キーワードを入力欄に保持
- 検索結果件数を表示
- 「クリア」リンクで全件表示に戻る

【技術スタック】
- Spring Boot 3.x (@RequestParam)
- MyBatis (動的SQL、LIKE検索)
- Thymeleaf
gps_fixed ステップ別プロンプト 
# Step 1: Mapperの作成
TodoMapper.xmlにLIKE検索用のSQLを追加してください。CONCAT('%', #{keyword}, '%')を使用してください。

# Step 2: Repositoryの修正
TodoRepositoryにsearchByTitle(@Param("keyword") String keyword)メソッドを追加してください。

# Step 3: Controllerの修正
TodoControllerの一覧メソッドに@RequestParam(required = false) String keywordを追加し、検索処理を実装してください。

# Step 4: 検索フォームの追加
index.htmlにGETメソッドの検索フォームを追加し、検索結果件数とクリアリンクを表示してください。
lightbulb プロンプトのコツ

「MyBatisの#{}を使用してSQLインジェクション対策をしてください」のように、
セキュリティ要件を明示すると、安全な実装が得られます。
