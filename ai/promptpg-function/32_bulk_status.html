Section 32-1

チェックボックス複数選択UI

タスク一覧の各行にチェックボックスを配置し、複数タスクを選択できるUIを実装します。「全選択/解除」機能も付けます。

check_box

行ごとのチェックボックス

各タスク行にname="selectedIds"のチェックボックス

select_all

全選択トグル

ヘッダーのチェックで全行を一括選択/解除

numbers

選択数カウンター

選択中のタスク数をリアルタイム表示

<form id="bulk-form" th:action="@{/todo/bulk-status}"

    <div class="bulk-action-bar" id="bulkBar" style="display:none">
        <span id="selectedCount">0</span>件選択中
        <select name="newStatus">
            <option value="true">完了にする</option>
            <option value="false">未完了に戻す</option>
        </select>
        <button type="submit" class="btn btn-sm btn-primary">
            一括変更
        </button>
    </div>

    <table class="table">
        <thead>
            <tr>
                <th><input type="checkbox" id="selectAll"
                           onchange="toggleAll(this)"></th>
                <th>タイトル</th>
                <th>ステータス</th>
                <th>操作</th>
            </tr>
        </thead>
        <tbody>
            <tr th:each="todo : ${todos}">
                <td>
                    <input type="checkbox" name="selectedIds"
                           th:value="${todo.id}"
                           onchange="updateCount()">
                </td>
                <td th:text="${todo.title}"></td>
                <td>
                    <span th:if="${todo.completed}"
                          class="badge badge-success">完了</span>
                    <span th:unless="${todo.completed}"
                          class="badge badge-secondary">未完了</span>
                </td>
                <td>...</td>
            </tr>
        </tbody>
    </table>
</form>
bulk-select.js（全選択・カウント更新） 
/** 全選択/解除トグル */
function toggleAll(master) {
    const checkboxes = document.querySelectorAll(
        'input[name="selectedIds"]');
    checkboxes.forEach(cb => cb.checked = master.checked);
    updateCount();
}

/** 選択数を更新し、操作バーの表示を切り替え */
function updateCount() {
    const checked = document.querySelectorAll(
        'input[name="selectedIds"]:checked');
    const count = checked.length;

    document.getElementById('selectedCount').textContent = count;
    document.getElementById('bulkBar').style.display =
        count > 0 ? 'flex' : 'none';
}
lightbulbフォーム設計のポイント

同じname="selectedIds"のチェックボックスが複数ある場合、Spring MVCは自動的にList<Integer>として受け取れます。
フォーム全体を一つの<form>で囲み、一括操作バーもその中に配置しましょう。

account_tree 処理フロー: 一括ステータス変更

1

2

3

ポイント: 同じname属性のチェックボックスをList<Integer>で受け取り、MyBatis foreachのIN句で1SQLのバッチ更新を実行。WHERE句にuser_id条件を含めてセキュリティを確保します。

Section 32-2

一括更新コントローラー

選択されたタスクIDリストとステータスを受け取り、一括でステータスを変更するコントローラーを実装します。

playlist_add_check

複数ID受け取り

@RequestParamでList<Integer>として受け取り

swap_horiz

ステータス指定

完了/未完了を指定して一括変更

shield

入力検証

空のリストが送信された場合のガード処理

TodoController.java（一括ステータス変更） 
/**
 * 選択されたタスクのステータスを一括変更
 */
@PostMapping("/todo/bulk-status")
public String bulkUpdateStatus(
        @RequestParam List<Integer> selectedIds,
        @RequestParam boolean newStatus,
        RedirectAttributes ra) {

    if (selectedIds == null || selectedIds.isEmpty()) {
        ra.addFlashAttribute("error",
            "タスクが選択されていません");
        return "redirect:/todo";
    }

    int count = todoService.bulkUpdateStatus(
        selectedIds, newStatus);

    String statusText = newStatus ? "完了" : "未完了";
    ra.addFlashAttribute("message",
        count + "件のタスクを「" + statusText + "」に変更しました");

    return "redirect:/todo";
}
TodoService.java（一括更新サービス） 
/**
 * 指定されたIDリストのステータスを一括更新
 */
@Transactional
public int bulkUpdateStatus(List<Integer> ids,
                            boolean completed) {
    return todoMapper.bulkUpdateStatus(ids, completed);
}
warningセキュリティ考慮

渡されたIDがログインユーザーのタスクであることを検証してください。他人のタスクIDを直接指定して変更される恐れがあります。WHERE句にuser_id条件を必ず含めましょう。

Section 32-3

動的SQLバッチ更新

MyBatisの動的SQL（foreach）を使って、複数レコードを1つのSQL文で一括更新します。

code

foreach タグ

IDリストをIN句に展開する動的SQL

speed

1SQL実行

UPDATE ... WHERE id IN (...)で一括更新

query_stats

更新件数の取得

UPDATEの戻り値で実際の更新件数を確認

TodoMapper.xml（動的SQLバッチ更新） 
<!-- 一括ステータス更新（foreachでIN句を動的生成） -->
<update id="bulkUpdateStatus">
    UPDATE todo
    SET completed = #{completed},
        updated_at = NOW()
    WHERE id IN
    <foreach collection="ids" item="id"
             open="(" separator="," close=")">
        #{id}
    </foreach>
</update>
TodoMapper.java（インターフェース） 
@Mapper
public interface TodoMapper {

    /**
     * 指定IDリストのステータスを一括更新
     * @return 更新された行数
     */
    int bulkUpdateStatus(@Param("ids") List<Integer> ids,
                         @Param("completed") boolean completed);
}
lightbulb@Paramアノテーション

MyBatisでメソッドに複数のパラメータがある場合、@ParamでXMLのパラメータ名を明示する必要があります。
@Param("ids")はXMLのcollection="ids"と対応します。

Section 32-4

操作結果のフラッシュメッセージ

一括操作の完了後に操作件数を含む結果メッセージをフラッシュ属性で表示し、ユーザーにフィードバックを返します。

check_circle

成功メッセージ

「5件のタスクを完了にしました」のように件数を表示

error

エラーメッセージ

未選択時やエラー時に警告を表示

auto_delete

自動消去

数秒後にメッセージを自動で非表示

todo-list.html（フラッシュメッセージ表示） 
<!-- 成功メッセージ -->
<div th:if="${message}" class="alert alert-success flash-msg">
    <span class="material-icons">check_circle</span>
    <span th:text="${message}"></span>
</div>

<!-- エラーメッセージ -->
<div th:if="${error}" class="alert alert-danger flash-msg">
    <span class="material-icons">error</span>
    <span th:text="${error}"></span>
</div>

<!-- 自動消去スクリプト -->
<script>
document.addEventListener('DOMContentLoaded', () => {
    const flashMsgs = document.querySelectorAll('.flash-msg');
    flashMsgs.forEach(msg => {
        setTimeout(() => {
            msg.style.transition = 'opacity 0.5s';
            msg.style.opacity = '0';
            setTimeout(() => msg.remove(), 500);
        }, 4000);
    });
});
</script>

lightbulbRedirectAttributesの仕組み

RedirectAttributes.addFlashAttribute()はリダイレクト先の1回のリクエストでのみ使えるデータです。
セッションに一時保存され、ページ表示後に自動で削除されるため、ブラウザのリロードで再表示されることはありません。

AIプログラミング

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、この機能を実装できます。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリに一括ステータス変更機能を追加してください。

【要件】
- タスク一覧の各行にチェックボックスを追加
- ヘッダーの全選択チェックボックスで全行を一括選択/解除
- 選択数をリアルタイム表示する操作バー
- 「完了にする」「未完了に戻す」の一括変更ボタン
- MyBatis foreachでIN句を動的生成して一括UPDATE
- 操作結果をフラッシュメッセージで表示（自動消去付き）

【技術スタック】
- Spring Boot 3.x + MyBatis（動的SQL）
- Thymeleaf + JavaScript
- RedirectAttributes（フラッシュメッセージ）
gps_fixed ステップ別プロンプト 
# Step 1: チェックボックスUI
タスク一覧テーブルの各行にチェックボックス（name="selectedIds"）を追加してください。ヘッダーに全選択チェックボックスを配置し、JavaScriptで全選択/解除と選択数カウント表示を実装してください。

# Step 2: 一括更新コントローラー
POST /todo/bulk-statusでList<Integer> selectedIdsとboolean newStatusを受け取り、ステータスを一括変更するコントローラーを作成してください。

# Step 3: MyBatis動的SQL
foreachタグでIDリストをIN句に展開し、completedを一括更新するSQLを作成してください。@Paramアノテーションを使用してください。

# Step 4: フラッシュメッセージ
更新完了時に「5件を完了にしました」のような成功メッセージをRedirectAttributesで表示してください。4秒後に自動でフェードアウトするJavaScriptも追加してください。
lightbulb プロンプトのコツ

「セキュリティ対策として、WHERE句にuser_id条件を含め、他ユーザーのタスクを変更できないようにしてください」と追加すると、
安全な実装が得られます。
