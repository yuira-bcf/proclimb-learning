Section 26-1

Javadocの概要とMaven設定

Javadocコメントを記述し、maven-javadoc-pluginでHTMLドキュメントを自動生成します。

edit_note

Javadocコメント

クラス・メソッド・フィールドにドキュメントを記述

build

Maven Plugin

ビルド時にHTMLドキュメントを自動生成

inventory_2

パッケージ情報

package-info.javaでパッケージの概要を記述

主要なJavadocタグ

タグ用途
@paramメソッドのパラメータを説明
@return戻り値を説明
@throws発生する例外を説明
@author作者名
@versionバージョン番号
@since導入バージョン
@see関連クラス・メソッドへのリンク
@deprecated非推奨であることを示す
{@code}コードとしてフォーマット
{@link}インラインリンク
{@inheritDoc}親クラスのJavadocを継承
pom.xml（Javadocプラグイン設定） 
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-javadoc-plugin</artifactId>
            <version>3.6.0</version>
            <configuration>
                <encoding>UTF-8</encoding>
                <docencoding>UTF-8</docencoding>
                <charset>UTF-8</charset>
                <locale>ja_JP</locale>
                <author>true</author>
                <version>true</version>
            </configuration>
        </plugin>
    </plugins>
</build>

<!-- Javadoc生成コマンド -->
<!-- mvn javadoc:javadoc -->
<!-- 出力先: target/site/apidocs/index.html -->
lightbulbMaven Javadocプラグインのポイント
  • encoding / docencoding / charset をすべてUTF-8に設定して文字化けを防止
  • locale を ja_JP にすると日本語のドキュメントが生成される
  • mvn javadoc:javadoc コマンドで target/site/apidocs/ にHTML出力
Section 26-2

Javadocコメントの記述

クラスレベル・メソッドレベルのJavadocコメントの書き方を学びます。

TodoService.java（Javadoc付き） 
/**
 * ToDoのビジネスロジックを提供するサービスクラス。
 *
 * <p>ToDoのCRUD操作とビジネスルールの適用を行います。</p>
 *
 * <h2>使用例</h2>
 * <pre>{@code
 * Todo todo = new Todo();
 * todo.setTitle("買い物");
 * todoService.create(todo);
 * }</pre>
 *
 * @author yamada
 * @version 1.0
 * @since 2024-01-01
 * @see TodoController
 * @see TodoMapper
 */
@Service
public class TodoService {

    /**
     * 指定されたIDのToDoを取得します。
     *
     * @param id 取得するToDoのID（{@code null}不可）
     * @return 指定されたIDのToDo
     * @throws ResourceNotFoundException 指定されたIDのToDoが存在しない場合
     * @throws IllegalArgumentException idが{@code null}の場合
     * @see #findAll()
     */
    public Todo findById(Integer id) {
        if (id == null) {
            throw new IllegalArgumentException("id must not be null");
        }
        return todoMapper.findById(id)
            .orElseThrow(() -> new ResourceNotFoundException("ToDo not found: " + id));
    }

    /**
     * 新しいToDoを作成します。
     *
     * <p>作成日時は自動的に現在時刻が設定されます。</p>
     *
     * @param todo 作成するToDo情報（{@code null}不可）
     * @return 作成されたToDo（IDが設定済み）
     * @throws IllegalArgumentException todoが{@code null}の場合
     */
    @Transactional
    public Todo create(Todo todo) {
        todo.setCreatedAt(LocalDateTime.now());
        todoMapper.insert(todo);
        return todo;
    }
}
lightbulbJavadocコメントの書き方のポイント
  • クラスレベル - 概要、使用例、@author、@version、@since、@seeを記述
  • @param - パラメータ名と説明、nullの許容可否を明記
  • @return - 戻り値の内容を具体的に説明
  • @throws - 発生する可能性のある例外と条件を明記
  • {@code} - コード片をそのまま表示（HTMLエスケープ不要）
  • @see - 関連するクラスやメソッドへのリンクを記述

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、この機能を実装できます。

edit_note 基本プロンプト 
Spring Boot 3.xのToDoアプリにJavadocを追加してください。

【要件】
- pom.xmlにmaven-javadoc-pluginを追加（UTF-8、日本語ロケール）
- クラスレベルJavadoc: 概要、@author、@version、@since、@see
- メソッドレベルJavadoc: 概要、@param、@return、@throws
- package-info.javaでパッケージの概要を記述
- {@code}、{@link}、{@inheritDoc}を適切に使用

【対象クラス】
- TodoService（全メソッド）
- TodoController（全エンドポイント）
- Todo（エンティティ）
- TodoForm（フォームクラス）

【技術スタック】
- Java 17+
- Maven
- maven-javadoc-plugin 3.6.0
gps_fixed ステップ別プロンプト 
# Step 1: Maven設定
pom.xmlにmaven-javadoc-pluginを追加してください。
UTF-8エンコーディング、日本語ロケールで設定してください。

# Step 2: サービスクラスのJavadoc
TodoServiceの全メソッドにJavadocコメントを追加してください。
@param, @return, @throwsを漏れなく記述してください。

# Step 3: コントローラーのJavadoc
TodoControllerの全エンドポイントにJavadocコメントを追加し、
HTTPメソッドやパスも説明に含めてください。

# Step 4: パッケージドキュメント
各パッケージにpackage-info.javaを作成し、
パッケージの概要と主要クラスへの@linkを記述してください。
lightbulb プロンプトのコツ

「既存のJavaコードを貼り付けて、Javadocコメントを追加してください」のように、
実際のコードを添付すると、より正確なドキュメントが生成されます。
