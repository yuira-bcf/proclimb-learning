スレッドプール設定のパラメータ

  • corePoolSize - 基本スレッド数（常に維持）
  • maxPoolSize - 最大スレッド数
  • queueCapacity - キューの容量
  • threadNamePrefix - スレッド名のプレフィックス

サンプルソースコード

AsyncConfig.java 
@Configuration
@EnableAsync
public class AsyncConfig implements AsyncConfigurer {

    @Bean(name = "taskExecutor")
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);       // 基本スレッド数
        executor.setMaxPoolSize(10);       // 最大スレッド数
        executor.setQueueCapacity(100);    // キュー容量
        executor.setThreadNamePrefix("Async-");
        executor.initialize();
        return executor;
    }

    // 非同期処理の例外ハンドラー
    @Override
    public AsyncUncaughtExceptionHandler getAsyncUncaughtExceptionHandler() {
        return (ex, method, params) -> {
            log.error("Async error in {}: {}", method.getName(), ex.getMessage());
        };
    }
}

lightbulb コード解説

  • @EnableAsync - 非同期処理を有効化（これがないと@Asyncが動作しない）
  • ThreadPoolTaskExecutor - スレッドプールを管理するExecutor
  • corePoolSize - タスクがなくても維持されるスレッド数
  • queueCapacity - corePoolSize以上のタスクはキューで待機
  • AsyncUncaughtExceptionHandler - void戻り値の非同期メソッドで発生した例外を処理
NotificationService.java 
@Service
public class NotificationService {

    // 戻り値なしの非同期処理（Fire-and-Forget）
    @Async
    public void sendNotification(String userId, String message) {
        // 時間のかかる処理（別スレッドで実行）
        Thread.sleep(3000);  // 実際はPush通知やメール送信
        log.info("Notification sent to {}", userId);
    }

    // 結果を返す非同期処理
    @Async
    public CompletableFuture<Report> generateReport(LocalDate date) {
        // レポート生成処理（別スレッドで実行）
        Report report = createReport(date);
        return CompletableFuture.completedFuture(report);
    }
}

// 呼び出し側（Controller）
@GetMapping("/generate-report")
public String generateReport() {
    // 非同期でレポート生成を開始（すぐに画面を返す）
    CompletableFuture<Report> future = notificationService.generateReport(LocalDate.now());

    // 結果が必要な場合は join() で待機
    Report report = future.join();

    return "redirect:/reports";
}

lightbulb コード解説

  • @Async - このメソッドを別スレッドで実行
  • void戻り値 - Fire-and-Forget（呼び出し後すぐに戻る、結果は待たない）
  • CompletableFuture - 非同期処理の結果を後から取得できる
  • join() - 非同期処理の完了を待って結果を取得

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ─────────────────────────────────────────────────────────

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

auto_awesome AIプロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリに非同期処理機能を追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- 全ての処理が同期的に実行されている
- メール送信やレポート生成など時間のかかる処理がある

【実装要件】
1. 非同期処理の有効化
   - @EnableAsyncをメインクラスまたはConfigクラスに追加
   - AsyncConfigでThreadPoolTaskExecutorを設定
   - corePoolSize=5, maxPoolSize=10, queueCapacity=100

2. 非同期サービスの実装
   - @Asyncアノテーションを使用した非同期メソッド
   - 戻り値なしの非同期メソッド（void）
   - CompletableFutureを返す非同期メソッド

3. 使用例
   - ToDo作成時に非同期で通知を送信
   - レポート生成を非同期で実行
   - 複数の非同期処理の結果を待ち合わせ（CompletableFuture.allOf）

4. 例外処理
   - 非同期処理での例外をハンドリング
   - AsyncUncaughtExceptionHandlerの実装

5. スケジューリング（オプション）
   - @Scheduledで定期実行タスクを追加
   - Cron式でのスケジュール設定

【技術スタック】
- Spring Boot 3.x
- @Async / @EnableAsync
- CompletableFuture
- ThreadPoolTaskExecutor

必要なコードの変更点を提示してください。
lightbulb ポイント

@Asyncメソッドが同一クラス内で呼び出された場合、非同期にならないことに注意。別のBeanから呼び出す必要があります。
