Section 26-1

工程概要

Javadocコメントを記述し、HTMLドキュメントを自動生成します。

Section 26-2

成果物

この機能で作成・更新するファイルの一覧です。

ファイル 変更内容
pom.xml maven-javadoc-pluginの設定
package-info.java パッケージレベルのドキュメント
各Javaファイル Javadocコメントの追加
target/site/apidocs/ 生成されるHTMLドキュメント

主要なJavadocタグ

  • @param - メソッドのパラメータを説明
  • @return - 戻り値を説明
  • @throws - 発生する例外を説明
  • @see - 関連するクラスやメソッドへのリンク
  • {@code} - コードスニペットを表示
  • {@link} - インラインリンクを作成

サンプルソースコード

pom.xml（Javadocプラグイン設定） 
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-javadoc-plugin</artifactId>
            <version>3.6.0</version>
            <configuration>
                <encoding>UTF-8</encoding>
                <docencoding>UTF-8</docencoding>
                <charset>UTF-8</charset>
                <locale>ja_JP</locale>
                <author>true</author>
                <version>true</version>
            </configuration>
        </plugin>
    </plugins>
</build>

<!-- Javadoc生成コマンド -->
<!-- mvn javadoc:javadoc -->
<!-- 出力先: target/site/apidocs/index.html -->
TodoService.java（Javadoc付き） 
/**
 * ToDoのビジネスロジックを提供するサービスクラス。
 *
 * <p>ToDoのCRUD操作とビジネスルールの適用を行います。</p>
 *
 * <h2>使用例</h2>
 * <pre>{@code
 * Todo todo = new Todo();
 * todo.setTitle("買い物");
 * todoService.create(todo);
 * }</pre>
 *
 * @author yamada
 * @version 1.0
 * @since 2024-01-01
 * @see TodoController
 * @see TodoMapper
 */
@Service
public class TodoService {

    /**
     * 指定されたIDのToDoを取得します。
     *
     * @param id 取得するToDoのID（{@code null}不可）
     * @return 指定されたIDのToDo
     * @throws ResourceNotFoundException 指定されたIDのToDoが存在しない場合
     * @throws IllegalArgumentException idが{@code null}の場合
     * @see #findAll()
     */
    public Todo findById(Integer id) {
        if (id == null) {
            throw new IllegalArgumentException("id must not be null");
        }
        return todoMapper.findById(id)
            .orElseThrow(() -> new ResourceNotFoundException("ToDo not found: " + id));
    }

    /**
     * 新しいToDoを作成します。
     *
     * <p>作成日時は自動的に現在時刻が設定されます。</p>
     *
     * @param todo 作成するToDo情報（{@code null}不可）
     * @return 作成されたToDo（IDが設定済み）
     * @throws IllegalArgumentException todoが{@code null}の場合
     */
    @Transactional
    public Todo create(Todo todo) {
        todo.setCreatedAt(LocalDateTime.now());
        todoMapper.insert(todo);
        return todo;
    }
}

lightbulb コード解説

  • クラスレベルJavadoc - 概要、使用例、作者、バージョン、関連クラスを記述
  • @param - パラメータ名と説明、nullの許容可否を明記
  • @return - 戻り値の説明
  • @throws - 発生する可能性のある例外と条件
  • {@code} - コード片をそのまま表示（HTMLエスケープ不要）
  • @see - 関連するクラスやメソッドへのリンク

Section 26-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

auto_awesome AIプロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリにJavadocを追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- 各クラスにはコメントがほとんどない
- APIドキュメントが存在しない

【実装要件】
1. pom.xmlにmaven-javadoc-pluginを追加
   - UTF-8エンコーディング
   - 日本語ロケール
   - ビルド時にJavadoc jarを生成

2. 各クラスにJavadocコメントを追加
   - クラスレベル: 概要、@author、@version、@since、@see
   - メソッドレベル: 概要、@param、@return、@throws
   - フィールドレベル: 各フィールドの説明

3. package-info.javaの作成
   - 各パッケージの概要を説明
   - 主要クラスへのリンク

4. Javadocタグの使用例
   - {@code コード}
   - {@link クラス名}
   - {@inheritDoc}
   - 
、
    、
  • などのHTML

5. 以下のクラスにJavadocを追加
   - TodoService（全メソッド）
   - TodoController（全エンドポイント）
   - Todo（エンティティ）
   - TodoForm（フォームクラス）

【技術スタック】
- Java 17+
- Maven
- maven-javadoc-plugin 3.6.0

必要なコードの変更点を提示してください。
また、Javadoc生成コマンド（mvn javadoc:javadoc）の実行方法も説明してください。
lightbulb ポイント

Swagger/OpenAPIを併用すると、REST APIのドキュメントも自動生成できます。フロントエンド開発者との連携に便利です。
