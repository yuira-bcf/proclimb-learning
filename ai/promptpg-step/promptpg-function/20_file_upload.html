セキュリティ上の注意点

  • ファイル名サニタイズ - パストラバーサル対策（..を含むパスを拒否）
  • ファイル種類制限 - 許可する拡張子・MIME型を制限
  • ファイルサイズ上限 - application.propertiesで設定
  • 保存ディレクトリ - Webから直接アクセスできない場所に配置
  • UUID形式のファイル名 - 元のファイル名は露出させない

サンプルソースコード

FileStorageService.java 
@Service
public class FileStorageService {

    @Value("${app.upload.dir}")
    private String uploadDir;

    private final AttachmentMapper attachmentMapper;

    public TodoAttachment store(Integer todoId, MultipartFile file) {
        // ファイル名サニタイズ（パストラバーサル対策）
        String originalFilename = StringUtils.cleanPath(
            file.getOriginalFilename());

        if (originalFilename.contains("..")) {
            throw new StorageException("Invalid file path");
        }

        // UUID形式でファイル名を生成
        String storedFilename = UUID.randomUUID() + getExtension(originalFilename);
        Path targetPath = Paths.get(uploadDir).resolve(storedFilename);

        // ファイル保存
        Files.copy(file.getInputStream(), targetPath);

        // メタ情報をDBに保存
        TodoAttachment attachment = new TodoAttachment();
        attachment.setTodoId(todoId);
        attachment.setOriginalFilename(originalFilename);
        attachment.setStoredFilename(storedFilename);
        attachment.setContentType(file.getContentType());
        attachment.setFileSize(file.getSize());
        attachmentMapper.insert(attachment);

        return attachment;
    }
}

lightbulb コード解説

  • StringUtils.cleanPath() - ファイルパスを正規化し、危険な文字を除去
  • ".."チェック - パストラバーサル攻撃（../../../etc/passwd等）を防止
  • UUID形式 - 元のファイル名を隠し、ファイル名衝突を回避
  • メタ情報のDB保存 - 元ファイル名はDBに保存し、ダウンロード時に復元
TodoController.java（アップロード・ダウンロード） 
@Controller
public class TodoController {

    // アップロード処理
    @PostMapping("/{id}/attach")
    public String uploadFile(
            @PathVariable Integer id,
            @RequestParam("file") MultipartFile file) {
        fileStorageService.store(id, file);
        return "redirect:/todo/" + id;
    }

    // ダウンロード処理
    @GetMapping("/attachments/{attachmentId}/download")
    public ResponseEntity<Resource> downloadFile(
            @PathVariable Integer attachmentId) {
        TodoAttachment attachment = attachmentMapper.findById(attachmentId);
        Path filePath = Paths.get(uploadDir).resolve(attachment.getStoredFilename());
        Resource resource = new UrlResource(filePath.toUri());

        // Content-Dispositionヘッダーで元のファイル名を指定
        String encodedFilename = URLEncoder.encode(
            attachment.getOriginalFilename(), StandardCharsets.UTF_8);

        return ResponseEntity.ok()
            .contentType(MediaType.parseMediaType(attachment.getContentType()))
            .header(HttpHeaders.CONTENT_DISPOSITION,
                "attachment; filename*=UTF-8''" + encodedFilename)
            .body(resource);
    }
}

lightbulb コード解説

  • MultipartFile - Spring MVCがアップロードファイルを自動的にバインド
  • ResponseEntity<Resource> - ファイルデータをHTTPレスポンスとして返却
  • Content-Disposition - ブラウザにダウンロードファイル名を指示
  • filename*=UTF-8'' - 日本語ファイル名に対応するRFC 5987形式

【要件】

- application.propertiesでファイルサイズ上限（10MB）を設定

- ファイル名のサニタイズ（パストラバーサル対策）
- ダウンロード機能（Content-Dispositionヘッダーでファイル名を指定）
- 添付ファイルの削除機能
