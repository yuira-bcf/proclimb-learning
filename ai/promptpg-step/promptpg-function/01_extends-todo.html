Section 01-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

smart_toy AIプロンプト

あなたはJavaプログラマーです。

Phase 1で作成したToDoアプリのEntityを拡張してください。

【現在のTodo Entity】
- id (Long)
- title (String)
- completed (Boolean)

【追加するフィールド】
1. description (String) - タスクの詳細説明（任意入力、最大500文字）
2. priority (Integer) - 優先度（1:高, 2:中, 3:低、デフォルト2）
3. createdAt (LocalDateTime) - 作成日時（自動設定）
4. updatedAt (LocalDateTime) - 更新日時（自動設定）

【変更が必要なファイル】
1. Todo.java - フィールド追加、getter/setter追加
2. schema.sql - 新規カラム追加
3. TodoMapper.xml - INSERT/UPDATE文の修正
4. form.html - 入力フィールド追加
5. list.html - 一覧表示に優先度と説明を追加

【要件】
- createdAtは新規作成時に自動設定
- updatedAtは更新時に自動設定
- priorityはドロップダウンで選択
- descriptionはtextareaで入力

必要なコードの変更点を提示してください。

auto_awesome 生成されるソースコード

Todo.java（拡張後） 
package com.example.todo.entity;

import jakarta.persistence.*;
import lombok.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "todos")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String title;

    @Column(length = 500)
    private String description;  // 追加: 詳細説明

    @Column(columnDefinition = "INTEGER DEFAULT 2")
    private Integer priority = 2;  // 追加: 優先度

    private Boolean completed = false;

    private LocalDateTime createdAt;  // 追加: 作成日時
    private LocalDateTime updatedAt;  // 追加: 更新日時

    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
    }

    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }
}
lightbulb コード解説
  • description - タスクの詳細を記録（最大500文字）
  • priority - 1:高、2:中、3:低 でタスクの重要度を管理
  • @PrePersist - INSERT時に自動でcreatedAt/updatedAtをセット
  • @PreUpdate - UPDATE時に自動でupdatedAtを更新
  • columnDefinition - DBのデフォルト値を指定

lightbulb ポイント

LocalDateTimeを使用する場合、@DateTimeFormatアノテーションでフォーマットを指定します。フォーム入力とのマッピングに注意しましょう。
