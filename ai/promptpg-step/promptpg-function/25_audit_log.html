監査ログテーブルのカラム

  • action - 操作種別（CREATE, UPDATE, DELETE）
  • entity_type - 対象エンティティ（TODO, USER等）
  • entity_id - 対象のID
  • old_value / new_value - 変更前後の値（JSON形式）
  • user_id / username - 操作したユーザー情報
  • ip_address - IPアドレス

サンプルソースコード

Auditable.java（カスタムアノテーション） 
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Auditable {

    // 操作種別
    AuditAction action();

    // エンティティ種別
    String entityType();
}

public enum AuditAction {
    CREATE, UPDATE, DELETE
}

lightbulb コード解説

  • @Target(ElementType.METHOD) - メソッドに付与できるアノテーション
  • @Retention(RetentionPolicy.RUNTIME) - 実行時にも情報を保持（AOPで読み取り可能）
  • action() - CREATE/UPDATE/DELETEの操作種別を指定
  • entityType() - 対象エンティティ（TODO, USERなど）を文字列で指定
AuditAspect.java 
@Aspect
@Component
public class AuditAspect {

    private final AuditLogService auditLogService;
    private final ObjectMapper objectMapper;

    @Around("@annotation(auditable)")
    public Object audit(ProceedingJoinPoint joinPoint, Auditable auditable) throws Throwable {

        // 変更前の値を取得（UPDATE/DELETEの場合）
        String oldValue = null;
        if (auditable.action() != AuditAction.CREATE) {
            Integer entityId = extractEntityId(joinPoint);
            Object oldEntity = findEntityById(auditable.entityType(), entityId);
            oldValue = objectMapper.writeValueAsString(oldEntity);
        }

        // 本体処理を実行
        Object result = joinPoint.proceed();

        // 変更後の値を取得
        String newValue = objectMapper.writeValueAsString(result);

        // 監査ログを記録（独立トランザクション）
        auditLogService.log(
            auditable.action(),
            auditable.entityType(),
            extractEntityId(result),
            oldValue,
            newValue
        );

        return result;
    }
}

lightbulb コード解説

  • @Around - メソッド実行の前後で処理を挟む
  • @annotation(auditable) - @Auditableが付いたメソッドを対象
  • ProceedingJoinPoint - 元のメソッドを実行するための参照
  • oldValue/newValue - 変更前後の値をJSON形式で保存
  • proceed() - 元のメソッド（本体処理）を実行
TodoService.java（使用例） 
@Service
public class TodoService {

    @Auditable(action = AuditAction.CREATE, entityType = "TODO")
    @Transactional
    public Todo create(Todo todo) {
        todoMapper.insert(todo);
        return todo;
    }

    @Auditable(action = AuditAction.UPDATE, entityType = "TODO")
    @Transactional
    public Todo update(Todo todo) {
        todoMapper.update(todo);
        return todo;
    }

    @Auditable(action = AuditAction.DELETE, entityType = "TODO")
    @Transactional
    public void delete(Integer id) {
        todoMapper.delete(id);
    }
}

Section 25-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

auto_awesome AIプロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリに監査ログ機能を追加してください。

【現在の状態】
- Spring Boot 3.x + MyBatis + Thymeleafで構築されたToDoアプリ
- Spring Securityでログイン認証済み
- CRUD操作は実装済みだが、操作履歴は記録していない

【実装要件】
1. 監査ログテーブル設計
   - audit_logsテーブル（action, entity_type, entity_id, user_id, username, old_value, new_value, ip_address, created_at）
   - 変更前後の値をJSON形式で保存

2. 監査ログエンティティとMapper
   - AuditLog.java（エンティティ）
   - AuditLogMapper.java/xml（INSERT、検索クエリ）

3. カスタムアノテーション @Auditable
   - action: CREATE, UPDATE, DELETE
   - entityType: TODO, USER など

4. AOPで自動記録（AuditAspect.java）
   - @Auditableが付いたメソッドの前後で処理
   - 変更前の値を取得（UPDATE/DELETEの場合）
   - 変更後の値を戻り値から取得
   - Propagation.REQUIRES_NEWで独立したトランザクション

5. 監査ログ検索画面（管理者向け）
   - エンティティ種別、ユーザー、日付範囲で検索
   - 変更内容の詳細表示

【技術スタック】
- Spring Boot 3.x
- Spring AOP
- MyBatis
- Jackson（JSON変換）
- Spring Security（ユーザー情報取得）

必要なコードの変更点を提示してください。
lightbulb ポイント

監査ログは独立したトランザクションで記録することで、本体の処理がロールバックされても監査記録は残ります。
