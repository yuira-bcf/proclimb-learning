Section 16-3

生成プロンプト

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、実装コードを生成できます。

サンプルソースコード

TodoApiController.java 
@RestController
@RequestMapping("/api/todo")
public class TodoApiController {

    @GetMapping
    public ResponseEntity<List<Todo>> list() {
        return ResponseEntity.ok(todoService.findAll());
    }

    @PostMapping
    public ResponseEntity<Todo> create(@RequestBody Todo todo) {
        Todo created = todoService.save(todo);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        todoService.delete(id);
        return ResponseEntity.noContent().build();
    }
}

menu_book コード解説

  • @RestController: 戻り値を自動的にJSONに変換
  • ResponseEntity: HTTPステータスコードとボディを制御
  • @RequestBody: リクエストJSONをオブジェクトに変換
  • HttpStatus.CREATED: 201ステータス（新規作成成功）

auto_awesome AIプロンプト

あなたはJavaプログラマーです。

Spring Boot 3.xのToDoアプリをREST APIとして実装してください。

【要件】
- RESTful設計：GET/POST/PUT/DELETE
- エンドポイント：/api/todo
- JSONでリクエスト/レスポンス
- 適切なHTTPステータスコード（200, 201, 204, 400, 404）
- レスポンスは統一フォーマット（success, message, data）
- CORS設定（フロントエンドからのアクセス許可）

【実装箇所】

■ TodoApiController.java
- @RestController、@RequestMapping("/api/todo")
- @GetMapping - 一覧取得（List<Todo>を返す）
- @GetMapping("/{id}") - 1件取得（見つからない場合は404）
- @PostMapping - 新規作成（201 Created）
- @PutMapping("/{id}") - 更新
- @DeleteMapping("/{id}") - 削除（204 No Content）

■ ApiResponse.java（レスポンスラッパー）
- boolean success - 成功/失敗フラグ
- String message - メッセージ
- T data - 実データ
- static success(T data), static error(String message)メソッド

■ WebConfig.java（CORS設定）
- @CrossOriginまたはWebMvcConfigurerでCORS許可
- 許可オリジン、メソッド、ヘッダーを設定

【技術スタック】
- Spring Boot 3.x (@RestController)
- Jackson (JSON変換)
- ResponseEntity

必要なコードを提示してください。
