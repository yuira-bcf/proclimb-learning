info このChapterで学ぶこと

このChapterでは、一覧→新規登録→確認→完了という典型的な画面遷移フローを実装します。 ControllerとThymeleafテンプレートを使い、Aタグ（リンク）によるGETリクエストで画面を遷移させます。

使用するアノテーション

  • @Controller - クラスがMVCのControllerであることを示す
  • @GetMapping - GETリクエストを処理するメソッドを定義
  • @RequestMapping - クラス全体のベースURLを設定

Section 2-1

一覧画面の作成

まずはControllerを作成し、ToDoの一覧画面を表示できるようにします。

psychology Controllerの役割とは？

Controllerは、ユーザーからのリクエストを受け取り、適切な処理を行って画面を返す役割を担います。 MVCアーキテクチャの「C」にあたり、ModelとViewの橋渡し役です。

1 リクエスト受付

HTTPリクエスト（GET）を受け取る

2 Service呼び出し

ビジネスロジックをServiceクラスに委譲

3 Model設定

Viewに渡すデータをModelに追加

4 View名を返す

表示するテンプレート（HTML）名を返す

smart_toy AIプロンプト

Spring BootでToDoアプリ用のControllerと全画面のテンプレートを作成してください。

要件:
- クラス名: TodoController
- @Controllerアノテーションを使用
- すべてGETリクエストで画面遷移する構成:
  1. GET /todo → 一覧画面（list.html）
  2. GET /todo/new → 新規登録画面（new.html）
  3. GET /todo/confirm → 確認画面（confirm.html）
  4. GET /todo/complete → 完了画面（complete.html）

テンプレート（場所: src/main/resources/templates/todo/）:
- list.html: ToDoリストをテーブルで表示、「新規登録」ボタンで /todo/new へ遷移
- new.html: タイトル入力欄、「確認画面へ」で /todo/confirm へ遷移（Aタグ）
- confirm.html: 確認内容表示、「登録」で /todo/complete、「戻る」で /todo/new へ遷移（Aタグ）
- complete.html: 完了メッセージ、「一覧へ戻る」で /todo へ遷移（Aタグ）
- Bootstrap 5のCDNを使用してスタイリング

auto_awesome 生成されるソースコード

TodoController.java 
package com.example.todo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;

/**
 * ToDoアプリのControllerクラス
 * 一覧→新規登録→確認→完了の画面遷移を管理
 */
@Controller
@RequestMapping("/todo")
public class TodoController {

    /**
     * 一覧画面を表示
     * GET /todo でアクセス
     */
    @GetMapping
    public String list(Model model) {
        // ToDoリストをModelに追加（後でServiceから取得）
        return "todo/list";
    }

    /**
     * 新規登録画面を表示
     * GET /todo/new でアクセス
     */
    @GetMapping("/new")
    public String showCreateForm() {
        return "todo/new";
    }

    /**
     * 確認画面を表示
     * GET /todo/confirm でアクセス
     */
    @GetMapping("/confirm")
    public String confirm() {
        return "todo/confirm";
    }

    /**
     * 完了画面を表示
     * GET /todo/complete でアクセス
     */
    @GetMapping("/complete")
    public String complete() {
        return "todo/complete";
    }
}
lightbulb コード解説

@Controller と @RequestMapping

  • @Controller - このクラスがMVCのControllerであることをSpringに伝えます
  • @RequestMapping("/todo") - クラス全体のベースURLを /todo に設定

@GetMapping

  • @GetMapping - リンククリックなどのGETリクエストを処理
  • 引数なしの場合、ベースURL（/todo）そのものを処理
  • 引数ありの場合、ベースURL + 引数のパスを処理（例: "/new" → /todo/new）

戻り値（View名）

  • return "todo/list" - templates/todo/list.html を表示
  • Thymeleafが自動的に src/main/resources/templates/ 配下を探す

link URLマッピングの一覧

@RequestMapping("/todo") ← クラスのベースURL
@GetMapping → /todo → 一覧画面
@GetMapping("/new") → /todo/new → 新規登録画面
@GetMapping("/confirm") → /todo/confirm → 確認画面
@GetMapping("/complete") → /todo/complete → 完了画面

lightbulb コード解説

Thymeleaf名前空間の宣言

  • xmlns:th="http://www.thymeleaf.org" - Thymeleaf属性を使うための宣言

リンク式 @{...}

  • th:href="@{/todo/new}" - 新規登録画面へのリンク
  • Aタグのhref属性をThymeleafで動的に生成する

繰り返し処理 th:each

  • th:each="todo : ${todos}" - Controllerから渡されたtodosリストをループ
  • 各ループで変数 todo に1件ずつデータが入る

テキスト出力 th:text

  • th:text="${todo.title}" - todoオブジェクトのtitleプロパティを出力
  • HTMLタグ内のテキスト（サンプルタスク等）は、実行時にth:textの値で置き換えられる
Section 2-2

新規登録画面の作成

一覧画面の「新規登録」リンクから遷移する新規登録画面を作成します。ここではAタグで確認画面へ遷移します。

smart_toy AIプロンプト

ThymeleafでToDoの新規登録画面を作成してください。

要件:
- ファイル名: new.html
- 場所: src/main/resources/templates/todo/
- タイトル入力欄（text）
- 「確認画面へ」リンクで /todo/confirm へ遷移（Aタグ）
- 「戻る」リンクで一覧画面 /todo へ遷移（Aタグ）

auto_awesome 生成されるソースコード

src/main/resources/templates/todo/new.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>ToDo新規登録</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
          rel="stylesheet">
</head>
<body>
    <div class="container mt-4">
        <h1>ToDo新規登録</h1>

        <!-- タイトル入力欄 -->
        <div class="mb-3">
            <label class="form-label">タイトル</label>
            <input type="text" name="title" class="form-control">
        </div>

        <!-- 一覧画面へ戻るリンク -->

Aタグによる画面遷移

  • th:href="@{/todo/confirm}" - 確認画面へのリンク
  • th:href="@{/todo}" - 一覧画面へ戻るリンク
  • AタグのクリックはGETリクエストを発生させる

リンク式 @{...} の仕組み

  • Thymeleafの @{...} はコンテキストパスを自動補完してURLを生成
  • Controllerの @GetMapping に対応するURLを指定する

Section 2-3

確認画面の作成

新規登録画面のリンクから遷移する確認画面を作成します。登録ボタンと戻るボタンをAタグで実装します。

smart_toy AIプロンプト

ThymeleafでToDoの登録確認画面を作成してください。

要件:
- ファイル名: confirm.html
- 場所: src/main/resources/templates/todo/
- 「登録内容の確認」のタイトルを表示
- 確認内容をカードで表示（タイトル: サンプルタスク）

確認画面のリンクから遷移する完了画面を作成します。一覧画面へ戻るリンクを配置します。

smart_toy AIプロンプト

ThymeleafでToDoの登録完了画面を作成してください。

要件:
- ファイル名: complete.html
- 場所: src/main/resources/templates/todo/
- 「登録が完了しました」のメッセージを表示
- 「一覧へ戻る」リンクで /todo へ遷移（Aタグ）

lightbulb コード解説

完了画面のポイント

  • alert alert-success - Bootstrapの成功メッセージ用スタイル
  • th:href="@{/todo}" - 一覧画面へ戻るリンク
  • 完了画面は最終画面なので、一覧へ戻るリンクのみを配置

Thymeleaf基本構文まとめ

  • th:text - テキストを出力
  • th:each - 繰り返し処理
  • th:href - リンクURLを動的に生成
  • @{...} - Thymeleafのリンク式（URLを生成）

  • 一覧画面の「新規登録」をクリック → 新規登録画面が表示される
