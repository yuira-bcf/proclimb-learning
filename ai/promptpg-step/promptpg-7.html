Section 7-1

削除処理の実装

RepositoryのdeleteById()メソッドを使用して、指定したIDのデータを削除します。

smart_toy AIプロンプト

Spring BootでToDoの削除機能を実装してください。

要件:
- POST /todo/{id}/delete で削除処理
- Serviceクラスで削除処理を実行
- 削除成功後は一覧画面にリダイレクト
- 存在しないIDの場合のエラーハンドリング

Controller、Service、例外処理を含めてください。
TodoService.java（削除処理） 
@Transactional
public void delete(Long id) {
    if (!todoRepository.existsById(id)) {
        throw new IllegalArgumentException("指定されたToDoが見つかりません: " + id);
    }
    todoRepository.deleteById(id);
}
TodoController.java（削除処理） 
@PostMapping("/todo/{id}/delete")
public String delete(
        @PathVariable Long id,
        RedirectAttributes redirectAttributes) {
    try {
        todoService.delete(id);
        redirectAttributes.addFlashAttribute("message", "ToDoを削除しました");
        redirectAttributes.addFlashAttribute("messageType", "success");
    } catch (IllegalArgumentException e) {
        redirectAttributes.addFlashAttribute("message", "削除に失敗しました");
        redirectAttributes.addFlashAttribute("messageType", "danger");
    }
    return "redirect:/todo";
}

psychology try-catchとは？

try-catchは、エラー（例外）が発生する可能性のある処理を安全に実行するための構文です。 エラーが起きてもプログラムがクラッシュせず、適切なエラーメッセージをユーザーに返すことができます。

基本構文
try {
    // エラーが起きる可能性のある処理
    todoService.delete(id);
    // 成功時の処理
    redirectAttributes.addFlashAttribute("message", "成功しました");
} catch (IllegalArgumentException e) {
    // エラー発生時の処理
    redirectAttributes.addFlashAttribute("message", "失敗しました");
}
try-catchの3つの役割
1 エラー検知

try ブロック内で発生したエラーを自動的にキャッチ

2 クラッシュ防止

エラーが起きてもアプリが止まらず処理を継続

3 ユーザー通知

catch ブロックで適切なエラーメッセージを表示

info
catch(IllegalArgumentException e) は「IllegalArgumentException」型のエラーをキャッチします。
e にはエラーの詳細情報が入っており、e.getMessage() でエラーメッセージを取得できます。

lightbulb コード解説
  • existsById() - IDの存在確認（削除前のチェック）
  • deleteById() - 指定IDのデータを削除
  • @PathVariable - URLのパス変数からIDを取得
  • try-catch - エラーハンドリングでユーザーにフィードバック
  • addFlashAttribute - リダイレクト先で1回だけ使えるメッセージ

warning 注意

Section 7-2

確認ダイアログの実装

誤操作を防ぐため、削除前に確認ダイアログを表示します。JavaScriptのconfirm()を使用します。

smart_toy AIプロンプト

削除ボタンに確認ダイアログを追加してください。

要件:
- JavaScriptのconfirm()を使用
- 「本当に削除しますか？」というメッセージを表示
- 「OK」を押した場合のみ削除処理を実行
- 「キャンセル」を押した場合は何もしない

フォーム送信と組み合わせた実装を示してください。
確認ダイアログ付き削除フォーム 
<form th:action="@{/todo/{id}/delete(id=${todo.id})}"
      method="post"
      onsubmit="return confirm('本当に削除しますか？')">
  <button type="submit" class="btn btn-danger">削除</button>
</form>
lightbulb コード解説
  • onsubmit="return confirm(...)" - フォーム送信前に確認
  • confirm()がfalseを返すと送信がキャンセルされる
  • ユーザーが「OK」を押した場合のみ削除処理が実行される
help_outline 確認ダイアログの動作フロー
check_circle OKを押した場合
confirm() → true → フォーム送信 → 削除実行
cancel キャンセルを押した場合
confirm() → false → 送信キャンセル → 何も起きない
Section 7-3

削除完了メッセージ

削除完了後にフラッシュメッセージを表示して、ユーザーに結果を通知します。

smart_toy AIプロンプト

Spring Bootでフラッシュメッセージを実装してください。

要件:
- RedirectAttributesを使用
- 削除成功時:「ToDoを削除しました」（成功メッセージ）
- 削除失敗時:「削除に失敗しました」（エラーメッセージ）
- Bootstrapのアラートコンポーネントで表示
- 数秒後に自動で消える機能も追加

Controller側とテンプレート側の両方を示してください。
templates/todo/list.html（フラッシュメッセージ表示） 
<!-- フラッシュメッセージ表示 -->
<div th:if="${message}"
     th:classappend="'alert alert-' + ${messageType ?: 'info'}"
     class="alert alert-dismissible fade show"
     role="alert">
  <span th:text="${message}"></span>
  <button type="button" class="btn-close" data-bs-dismiss="alert"></button>
</div>

<!-- 自動で消えるJavaScript -->
<script th:if="${message}">
  setTimeout(function() {
    const alert = document.querySelector('.alert');
    if (alert) {
      alert.classList.remove('show');
      setTimeout(() => alert.remove(), 150);
    }
  }, 3000); // 3秒後に消える
</script>
lightbulb コード解説
  • RedirectAttributes.addFlashAttribute() - リダイレクト先で1回だけ使えるデータ
  • th:if="${message}" - メッセージがある場合のみ表示
  • messageType - success/danger/info等でアラートの色を変更
  • setTimeout() - 指定時間後にアラートを自動で非表示
check_circle 確認ポイント

一覧画面から削除ボタンを押して、確認ダイアログが表示されることを確認してください。削除後、メッセージが表示され、データが削除されていることも確認しましょう。
