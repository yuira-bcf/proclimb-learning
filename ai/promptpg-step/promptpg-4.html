ポイント: データベースを使うことで、アプリを再起動してもデータが消えずに残ります。今回はまず開発用のH2データベース（インメモリ型）を使い、データベースの基本を学びます。

smart_toy AIプロンプト

Spring BootでH2データベースを設定してください。

要件:
- application.propertiesに必要な設定を追加
- H2コンソールを有効化（/h2-console）
- データベース名: tododb
- ユーザー名: sa
- パスワード: なし
- schema.sqlでtodosテーブルを作成（id, title, completed）
- spring.sql.init.mode=alwaysで起動時にテーブルを自動作成

psychology AIプロンプト解説 ── なぜこの書き方をするのか？

1
「Spring BootでH2データベースを設定してください」
最初に技術名 + やりたいことを明示します。AIが「何の文脈の話か」をすぐに理解でき、的確な回答が得られます。
2
「データベース名: tododb」「ユーザー名: sa」── 具体的な値を指定
AIに値を任せるとtododbやtodo_dbなど出力がブレます。使いたい値を明記することで、毎回同じ結果が得られます。
3
「schema.sqlでtodosテーブルを作成（id, title, completed）」── ファイル名とカラム名を指定
どのファイルに・どんな構造で作るかを伝えます。「テーブルを作成して」だけだと、カラム名や型がAIの判断で変わってしまいます。
4
「spring.sql.init.mode=alwaysで起動時にテーブルを自動作成」── 設定キーを明記
Spring Bootの設定キーは正確な名前を書くのがコツです。AIが別のアプローチ（JPA自動生成など）を提案するのを防ぎます。

プロンプトのコツ: AIへの指示は「曖昧に任せる」より「具体的な値・ファイル名・設定キーを明記する」ほうが、安定した出力が得られます。特にデータベース名や設定値は毎回ブレやすいので、必ず指定しましょう。

auto_awesome 生成されるソースコード

application.properties（H2設定例） 
# H2 Database
spring.datasource.url=jdbc:h2:mem:tododb
spring.datasource.driverClassName=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# H2 Console
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console

# テーブル初期化（schema.sqlを自動実行）
spring.sql.init.mode=always
schema.sql（テーブル定義） 
-- src/main/resources/schema.sql
-- 起動時に自動実行され、テーブルを作成する

CREATE TABLE IF NOT EXISTS todos (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    completed BOOLEAN DEFAULT FALSE
);
lightbulb 設定項目の解説
  • spring.datasource.url - データベース接続URL（mem:はインメモリ）
  • spring.h2.console.enabled - H2コンソールを有効化
  • spring.sql.init.mode=always - 起動時にschema.sqlを自動実行してテーブルを作成
