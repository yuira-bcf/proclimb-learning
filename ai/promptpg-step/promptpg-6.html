Section 6-1

データの取得

RepositoryのfindAll()メソッドで全件取得し、Modelに格納してテンプレートに渡します。

smart_toy AIプロンプト

Spring Boot + MyBatisでToDoの一覧を取得してテンプレートに渡す処理を実装してください。

現在の状態:
- TodoMapper: findAll() メソッドが定義済み
- TodoService: 登録処理のみ実装済み

要件:
- GET /todo で一覧画面を表示
- TodoServiceにfindAll()メソッドを追加し、TodoMapperを呼び出す
- Controllerで取得したリストをModelに格納

Controller、Serviceのコードを示してください。
TodoService.java（一覧取得） 
@Service
@RequiredArgsConstructor
public class TodoService {

    private final TodoRepository todoRepository;

    public List<Todo> findAll() {
        return todoRepository.findAll(Sort.by(Sort.Direction.DESC, "createdAt"));
    }

    public List<Todo> findByCompleted(boolean completed) {
        return todoRepository.findByCompleted(completed);
    }
}
TodoController.java（一覧表示） 
@Controller
@RequiredArgsConstructor
public class TodoController {

    private final TodoService todoService;

    @GetMapping("/todo")
    public String list(Model model) {
        List<Todo> todos = todoService.findAll();
        model.addAttribute("todos", todos);
        return "todo/list";
    }
}
lightbulb コード解説
  • findAll(Sort.by(...)) - ソート条件を指定して全件取得
  • Sort.Direction.DESC - 降順（新しい順）でソート
  • model.addAttribute() - テンプレートに渡すデータをセット
  • テンプレートでは ${todos} でデータにアクセス可能
Section 6-2

一覧テンプレートの作成

Thymeleafのth:eachを使用して、データをループ処理で表示します。

smart_toy AIプロンプト

Chapter 2で作成したlist.htmlを更新し、DBから取得したToDoデータを表示してください。

現在のlist.html:
- 静的なサンプルデータをテーブルで表示中

要件:
- Modelから受け取ったtodosリストをth:eachで繰り返し表示
- 表示項目: ID、タイトル、完了状態
- 完了状態はtrue/falseを「完了」「未完了」で表示
- 「新規登録」ボタンで /todo/new へ遷移
- Bootstrap 5のテーブルを使用
templates/todo/list.html 
<table class="table table-striped">
  <thead>
    <tr>
      <th>タイトル</th>
      <th>期限日</th>
      <th>優先度</th>
      <th>状態</th>
      <th>操作</th>
    </tr>
  </thead>
  <tbody>
    <tr th:each="todo : ${todos}">
      <td th:text="${todo.title}">タイトル</td>
      <td th:text="${#temporals.format(todo.dueDate, 'yyyy/MM/dd')}"
          th:classappend="${todo.dueDate != null && todo.dueDate.isBefore(#temporals.createToday())} ? 'text-danger'">
        2024/01/01
      </td>
      <td>
        <span th:each="i : ${#numbers.sequence(1, todo.priority)}">★</span>
      </td>
      <td>
        <span th:if="${todo.completed}" class="badge bg-success">完了</span>
        <span th:unless="${todo.completed}" class="badge bg-secondary">未完了</span>
      </td>
      <td>
        <a th:href="@{/todo/{id}(id=${todo.id})}" class="btn btn-sm btn-info">詳細</a>
        <a th:href="@{/todo/{id}/edit(id=${todo.id})}" class="btn btn-sm btn-warning">編集</a>
      </td>
    </tr>
  </tbody>
</table>
lightbulb コード解説
  • th:each="todo : ${todos}" - todosリストをループ処理
  • #temporals.format() - 日付を指定フォーマットで表示
  • th:classappend - 条件に応じてCSSクラスを追加
  • #numbers.sequence(1, n) - 1からnまでの連番を生成（星表示用）
  • th:if / th:unless - 条件による表示切替
  • @{/path/{id}(id=${todo.id})} - パス変数を含むURL生成

Section 6-3

データが空の場合の処理

データが0件の場合にメッセージを表示するよう、条件分岐を追加します。

smart_toy AIプロンプト

Thymeleafでデータが空の場合の処理を追加してください。

要件:
- th:if / th:unless を使用
- データが0件の場合:「ToDoがまだ登録されていません」と表示
- 「新規作成」ボタンへの誘導メッセージも表示
- データがある場合: テーブルを表示

空の状態でも見栄えの良いデザインにしてください。
templates/todo/list.html（空データ処理） 
<!-- データがある場合 -->
<div th:if="${!#lists.isEmpty(todos)}">
  <table class="table">
    <!-- テーブル内容 -->
  </table>
</div>

<!-- データがない場合 -->
<div th:unless="${!#lists.isEmpty(todos)}" class="text-center py-5">
  <div class="mb-4">
    <span class="material-icons" style="font-size: 4rem; color: #9ca3af;">
      inbox
    </span>
  </div>
  <h4 class="text-muted">ToDoがまだ登録されていません</h4>
  <p class="text-muted mb-4">最初のToDoを作成しましょう</p>
  <a th:href="@{/todo/new}" class="btn btn-primary">
    <span class="material-icons">add</span> 新規作成
  </a>
</div>
lightbulb コード解説
  • #lists.isEmpty() - リストが空かどうかを判定
  • th:if - 条件がtrueの場合に表示
  • th:unless - 条件がfalseの場合に表示（th:ifの逆）
  • 空の状態でも見栄えの良いデザインにすることでUXが向上
check_circle 確認ポイント

一覧画面にアクセスして、登録したデータが正しく表示されることを確認してください。データが0件の場合の表示も確認しましょう。
