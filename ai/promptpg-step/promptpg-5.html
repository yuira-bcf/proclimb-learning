Section 5-1

Entityクラスの作成

Entityは、データベースのテーブルに対応するJavaクラスです。JPAアノテーションを使用して定義します。

psychology Entityの役割とは？

Entityは、データベースのテーブル構造をJavaオブジェクトとして表現する役割を担います。 ORM（Object-Relational Mapping）の中心的な概念で、SQLを意識せずにデータを扱えるようになります。

Entityが必要な3つの理由
1 オブジェクト指向

テーブルの1行をJavaオブジェクトとして扱える

2 型安全性

コンパイル時にデータ型のエラーを検出できる

3 自動マッピング

JPAがJava↔DBの変換を自動で行う

❌ Entityを使わない場合
ResultSet rs = stmt.executeQuery();
String title = rs.getString("title");
✅ Entityを使う場合
Todo todo = repository.findById(1);
String title = todo.getTitle();

smart_toy AIプロンプト

Spring BootでToDoアプリ用のEntityクラスを作成してください。

要件:
- クラス名: Todo
- テーブル名: todos
- フィールド:
  - id（主キー、自動採番）: Long
  - title（タイトル）: String（必須、最大100文字）
  - description（説明）: String（任意、最大500文字）
  - dueDate（期限日）: LocalDate
  - priority（優先度）: Integer（デフォルト: 1）
  - completed（完了フラグ）: Boolean（デフォルト: false）
  - createdAt（作成日時）: LocalDateTime
  - updatedAt（更新日時）: LocalDateTime

Lombokを使用し、@PrePersistと@PreUpdateで日時を自動設定してください。
Todo.java（Entityクラス） 
package com.example.todo.entity;

import jakarta.persistence.*;
import lombok.*;
import java.time.LocalDate;
import java.time.LocalDateTime;

@Entity
@Table(name = "todos")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String title;

    @Column(length = 500)
    private String description;

    private LocalDate dueDate;

    @Column(columnDefinition = "INTEGER DEFAULT 1")
    private Integer priority = 1;

    @Column(columnDefinition = "BOOLEAN DEFAULT FALSE")
    private Boolean completed = false;

    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
    }

    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }
}
lightbulb コード解説
  • @Entity - JPAがこのクラスをDBテーブルにマッピング
  • @Table(name = "todos") - マッピング先テーブル名を指定
  • @Id + @GeneratedValue - 主キーの自動採番
  • @Column - カラムの制約（NOT NULL、長さ等）を設定
  • @PrePersist - INSERT前に自動実行（作成日時セット）
  • @PreUpdate - UPDATE前に自動実行（更新日時セット）
  • Lombokアノテーション（@Data等）でgetter/setter自動生成

Repositoryインターフェースの作成

Repositoryは、データベースへのCRUD操作を提供するインターフェースです。Spring Data JPAが実装を自動生成します。

psychology Repositoryの役割とは？

Repositoryは、データベースへのアクセスを抽象化し、CRUD操作を簡単に実行する役割を担います。 Spring Data JPAを使うと、インターフェースを定義するだけでSQLを書かずにDB操作ができます。

smart_toy AIプロンプト

Spring Data JPAでTodoエンティティ用のRepositoryを作成してください。

要件:
- インターフェース名: TodoRepository
- JpaRepositoryを継承
- カスタムクエリメソッド:
  - 完了状態でフィルタリング
  - タイトルで部分一致検索
  - 期限日が今日以前のもの
  - 優先度でソート

@Queryアノテーションの使い方も示してください。
TodoRepository.java 
package com.example.todo.repository;

import com.example.todo.entity.Todo;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.stereotype.Repository;
import java.time.LocalDate;
import java.util.List;

@Repository
public interface TodoRepository extends JpaRepository<Todo, Long> {

    // 完了状態でフィルタリング
    List<Todo> findByCompleted(Boolean completed);

    // タイトルで部分一致検索
    List<Todo> findByTitleContaining(String keyword);

    // 期限日が指定日以前のもの
    List<Todo> findByDueDateLessThanEqual(LocalDate date);

    // 優先度でソート（降順）
    List<Todo> findAllByOrderByPriorityDesc();

    // @Queryを使ったカスタムクエリ
    @Query("SELECT t FROM Todo t WHERE t.completed = false ORDER BY t.dueDate ASC")
    List<Todo> findUncompletedOrderByDueDate();
}
lightbulb コード解説
  • JpaRepository<Todo, Long> - Entityクラスと主キーの型を指定
  • 基本CRUD操作（save, findById, findAll, delete等）は自動で提供される
  • メソッド名から自動でSQLが生成される（Spring Data JPA機能）
  • @Query - JPQLでカスタムクエリを記述可能

Spring Data JPAのクエリメソッド命名規則

  • findBy〇〇 - 条件で検索
  • findBy〇〇And△△ - AND条件
  • findBy〇〇OrderBy△△ - ソート付き検索
  • countBy〇〇 - 件数カウント
  • deleteBy〇〇 - 条件削除
layers Repository メソッド → SQL 自動生成
findByCompleted(true) → SELECT * FROM todos WHERE completed = true
findByTitleContaining("買い物") → SELECT * FROM todos WHERE title LIKE '%買い物%'
findAllByOrderByPriorityDesc() → SELECT * FROM todos ORDER BY priority DESC
Section 5-3

データ登録処理の実装

Repositoryのsave()メソッドを使用して、フォームから送信されたデータをデータベースに登録します。

smart_toy AIプロンプト

Spring Bootでフォームデータをデータベースに登録する処理を実装してください。

要件:
- ServiceクラスでRepositoryを呼び出す
- フォームオブジェクト（TodoForm）からエンティティ（Todo）への変換
- 登録成功後は一覧画面にリダイレクト
- フラッシュメッセージで「登録が完了しました」を表示

Controller、Service、変換処理のすべてを示してください。
TodoService.java 
package com.example.todo.service;

import com.example.todo.entity.Todo;
import com.example.todo.repository.TodoRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
public class TodoService {

    private final TodoRepository todoRepository;

    @Transactional
    public Todo create(String title, String description) {
        Todo todo = new Todo();
        todo.setTitle(title);
        todo.setDescription(description);
        return todoRepository.save(todo);
    }
}
TodoController.java（登録処理） 
@PostMapping("/todo/create")
public String create(
        @RequestParam String title,
        @RequestParam(required = false) String description,
        RedirectAttributes redirectAttributes) {

    todoService.create(title, description);

    redirectAttributes.addFlashAttribute("message", "登録が完了しました");
    return "redirect:/todo";
}
lightbulb コード解説
  • @Service - サービス層のクラスであることを示す
  • @Transactional - トランザクション管理（エラー時は自動ロールバック）
  • repository.save() - INSERT（新規）またはUPDATE（既存）を自動判定
  • RedirectAttributes - リダイレクト先にメッセージを渡す
  • redirect:/todo - PRGパターン（Post-Redirect-Get）でリダイレクト
