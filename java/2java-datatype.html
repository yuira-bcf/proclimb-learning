Section 2-1

リテラル

リテラルとは、ソースコード中に直接記述される値のことです。数値、文字、文字列などがあります。

1.1 数値リテラル

種類 例 説明
整数（10進数） 123 通常の整数表記
整数（2進数） 0b1010 0bまたは0Bで始まる
整数（8進数） 0755 0で始まる
整数（16進数） 0xFF 0xまたは0Xで始まる
long型 123L 末尾にLまたはlを付ける
浮動小数点（double） 3.14 デフォルトはdouble型
浮動小数点（float） 3.14f 末尾にFまたはfを付ける
lightbulb アンダースコア区切り

Java 7以降、数値リテラルにアンダースコアを入れて読みやすくできます：1_000_000

Section 2-2

変数や定数の宣言と初期化

変数はプログラムの中でデータを一時的に保存するための「箱」です。適切な変数名と型を選ぶことは、コードの可読性とバグ防止に直結します。チーム開発では命名規則を統一することで、コードレビューの効率が大幅に向上します。

2.1 変数と命名規則

変数名のルール：

  • 英字、数字、アンダースコア、ドル記号が使用可能
  • 数字で始めることはできない
  • 予約語は使用できない
  • 大文字と小文字は区別される

Section 2-4

配列

配列は同じ型のデータを複数まとめて扱うための基本的なデータ構造です。ループと組み合わせることで、大量のデータを効率的に処理できます。後に学ぶArrayListなどのコレクションも、内部では配列を使用しています。配列の仕組みを理解することは、パフォーマンスを意識したプログラミングの基礎となります。

配列の生成

newキーワードで指定した要素数の配列を生成します。要素は型に応じたデフォルト値で初期化されます。

配列初期化子

型名[] 配列名 = {値1, 値2, 値3};

波括弧内に初期値を列挙して配列を初期化します。要素数は自動的に決定されます。

配列要素へのアクセス

warning 配列の初期値

配列をnewで生成すると、要素は自動的にデフォルト値で初期化されます（int:0、boolean:false、参照型:null）。

Section 2-6

文字列の操作

文字列操作はほぼすべてのプログラムで必要になります。ユーザー入力の処理、ファイル名の生成、ログ出力、API通信など、文字列を扱わないプログラムはほとんどありません。Stringクラスのメソッドを使いこなすことで、効率的なコードが書けるようになります。

6.1 Stringクラスの特徴

Stringクラスの重要な特徴：

  • イミュータブル（不変）：一度作成された文字列は変更できない
  • 文字列プール：同じ文字列リテラルは共有される

指定した範囲の部分文字列を返します。終了インデックスは含まれません（開始以上、終了未満）。

equals() - 文字列の等価比較

文字列1.equals(文字列2)

文字列の内容が等しいかをbooleanで返します。==は参照の比較なので、文字列の比較にはequals()を使用します。

menu_book 構文説明

StringBuilderの生成

StringBuilder sb = new StringBuilder();

変更可能な文字列を扱うクラスです。頻繁な文字列操作ではStringより効率的です。

append() - 文字列の追加

sb.append(追加する値);

末尾に文字列を追加します。数値やオブジェクトも自動的に文字列に変換されます。StringBuilder自身を返すため、メソッドチェーンが可能です。

insert() - 文字列の挿入

sb.insert(位置, 挿入する値);

指定した位置に文字列を挿入します。既存の文字は後ろにずれます。

toString() - Stringへの変換

sb.toString()

StringBuilderの内容をStringオブジェクトに変換します。最終結果を取得する際に使用します。

lightbulb StringBufferとの違い

StringBufferはスレッドセーフですが、StringBuilderより遅いです。シングルスレッド環境ではStringBuilderを推奨します。

String s = "Hello";

Q3

int[] arr = new int[3];で生成された配列の各要素の初期値は何ですか？

  1. null
  2. 0
  3. 未定義（ランダム）
  4. コンパイルエラー
Answer

2章 解説・解答

Q1の解答: B, C

A: 変数名は数字で始められません。D: classは予約語です。B: アンダースコアで始まる名前は有効です。C: ドル記号で始まる名前も有効です。

Q2の解答: B

Stringはイミュータブル（不変）なので、toUpperCase()は新しい文字列を返しますが、元の変数sは変更されません。結果を反映するにはs = s.toUpperCase();とする必要があります。

Q3の解答: B

int型配列をnewで生成すると、各要素はデフォルト値の0で初期化されます。参照型の場合はnull、booleanの場合はfalseになります。
