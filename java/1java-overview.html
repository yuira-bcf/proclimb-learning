Section 1-1

Javaの概要

Javaは1995年にサン・マイクロシステムズ（現Oracle）によって開発されたプログラミング言語です。「Write Once, Run Anywhere（一度書けばどこでも動く）」をスローガンに、プラットフォームに依存しない実行環境を提供しています。

1.1 Java言語の特徴

Javaには以下のような特徴があります：

💻

オブジェクト指向

クラスとオブジェクトを基本単位としてプログラムを構成します。継承、カプセル化、ポリモーフィズムといった概念を活用できます。

🔒

型安全性

静的型付け言語であり、コンパイル時に型チェックが行われます。これにより実行時エラーを未然に防ぐことができます。

📦

ガベージコレクション

メモリ管理を自動的に行うため、プログラマがメモリの解放を意識する必要がありません。

🌐

プラットフォーム非依存

Java仮想マシン（JVM）上で動作するため、OSに依存せずプログラムを実行できます。

1.2 実行環境の特徴

Javaプログラムは、以下の流れで実行されます：

1

ソースコードの作成

.java拡張子のファイルにJavaのソースコードを記述します。

2

コンパイル

javacコマンドでソースコードをバイトコード（.classファイル）に変換します。

3

実行

javaコマンドでJVMがバイトコードを解釈・実行します。

lightbulb バイトコードとは

バイトコードは、JVMが理解できる中間コードです。ネイティブコード（機械語）とは異なり、どのプラットフォームのJVMでも実行可能な形式になっています。

Section 1-2

Java環境とJDK

Javaプログラムを開発・実行するためには、適切な環境を整える必要があります。このセクションでは、開発に必要なツールと、それぞれの役割を理解します。環境構築はプログラミング学習の第一歩であり、ここをしっかり理解することで、後のトラブルシューティングにも役立ちます。

2.1 Java環境とは

Javaの開発・実行には、以下の環境が必要です：

環境 正式名称 用途
JDK Java Development Kit 開発に必要なツール一式（コンパイラ、デバッガなど）
JRE Java Runtime Environment 実行に必要な環境（JVMとライブラリ）
JVM Java Virtual Machine バイトコードを解釈・実行する仮想マシン
warning 注意

Java 11以降、JREは単独で配布されなくなりました。JDKにJREが含まれているため、開発者はJDKをインストールすれば開発と実行の両方が可能です。

2.2 環境セットアップ

JDKのインストール手順：

  1. Oracle公式サイトまたはAdoptium（旧AdoptOpenJDK）からJDKをダウンロード
  2. インストーラーを実行してJDKをインストール
  3. 環境変数JAVA_HOMEを設定
  4. 環境変数PATHに%JAVA_HOME%\binを追加
  5. コマンドプロンプトでjava -versionを実行して確認
バージョン確認コマンド Terminal 
$ java -version
java version "21.0.1" 2023-10-17 LTS
Java(TM) SE Runtime Environment (build 21.0.1+12-LTS-29)
Java HotSpot(TM) 64-Bit Server VM (build 21.0.1+12-LTS-29, mixed mode, sharing)

3.2 ソースファイルの作成

最も基本的なJavaプログラム「Hello World」を作成してみましょう：

chat プロンプト 
## 機能
Hello Worldを出力する最初のJavaプログラム

## 実装要件
- publicクラスとしてHelloWorldクラスを作成
- mainメソッドをエントリーポイントとして定義
- コンソールに「Hello, World!」を出力

## 使用するJava構文
- public class宣言
- public static void main(String[] args)
- System.out.println()
HelloWorld.java Java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
menu_book 構文説明

クラス宣言

public class クラス名 { }

Javaプログラムの基本単位です。publicは他のクラスからアクセス可能であることを示します。クラス名は大文字で始めるのが慣習です。

mainメソッド

public static void main(String[] args) { }

プログラムのエントリーポイント（開始点）です。publicはどこからでもアクセス可能、staticはインスタンス化不要、voidは戻り値なし、String[] argsはコマンドライン引数を受け取る配列です。

標準出力

System.out.println(出力内容);

コンソールに文字列を出力し、改行します。print()は改行なし、println()は改行ありです。

lightbulb ファイル名の規則

publicクラスを含むファイルは、クラス名と同じファイル名にする必要があります。上記の場合、HelloWorld.javaという名前で保存します。

3.3 コンパイル

javacコマンドを使ってソースファイルをコンパイルします：

コンパイルコマンド Terminal 
$ javac HelloWorld.java

コンパイルが成功すると、HelloWorld.classというバイトコードファイルが生成されます。

3.4 クラスの実行

javaコマンドでプログラムを実行します：

実行コマンド Terminal 
$ java HelloWorld
Hello, World!
warning 注意

実行時には.class拡張子を付けません。クラス名のみを指定します。

3.5 ソースファイルとクラスファイルの関係

  • 1つのソースファイルには複数のクラスを定義できますが、publicクラスは1つだけ
  • ソースファイル内の各クラスに対して、それぞれのクラスファイルが生成される
  • 内部クラスがある場合は外部クラス$内部クラス.classという名前で生成される

3.6 ソースファイルモードでの実行

Java 11以降では、コンパイルせずに直接ソースファイルを実行できます：

ソースファイルモード（Java 11以降） Terminal 
$ java HelloWorld.java
Hello, World!

この機能は単一ファイルのプログラムをテストする際に便利です。

Section 1-4

パッケージ宣言とインポート

大規模なプロジェクトでは、数百から数千のクラスを扱うことがあります。パッケージはこれらのクラスを論理的に整理し、名前の衝突を防ぐ仕組みです。Spring BootやMavenプロジェクトでは、パッケージ構成がプロジェクト構造の基盤となるため、この概念をしっかり理解しておくことが重要です。

4.1 パッケージ

パッケージは、関連するクラスをグループ化するための仕組みです。名前空間を提供し、クラス名の衝突を防ぎます。

- publicクラスとしてMyClassを定義
- パッケージ宣言はファイルの先頭に記述

## 使用するJava構文
- package文
- public class宣言
パッケージ宣言 Java 
package com.example.myapp;

public class MyClass {
    // クラスの内容
}
menu_book 構文説明

パッケージ宣言

package パッケージ名;

クラスが所属するパッケージを宣言します。ファイルの先頭（コメント以外）に記述します。パッケージ名はドット区切りで階層を表し、通常はドメイン名を逆にした形式（例：com.example.myapp）を使用します。

4.2 パッケージ化

パッケージ化するには：

  • ソースファイルの先頭にpackage文を記述
  • パッケージ名に対応するディレクトリ構造にファイルを配置
ディレクトリ構造の例 Directory 
src/
  └── com/
      └── example/
          └── myapp/
              └── MyClass.java

4.3 パッケージ化されたクラスのコンパイルと実行

コンパイルと実行 Terminal 
# コンパイル（srcディレクトリで実行）
$ javac com/example/myapp/MyClass.java

# 実行（完全修飾クラス名を指定）
$ java com.example.myapp.MyClass

4.4 クラスパス

クラスパスは、JVMがクラスファイルを検索する場所を指定します。

クラスパスの指定 Terminal 
# -cpまたは-classpathオプションで指定
$ java -cp ./classes com.example.myapp.MyClass

# 複数のパスを指定（Windowsは;、Unix系は:で区切る）
$ java -cp ./classes:./lib/* com.example.myapp.MyClass

4.5 インポート

他のパッケージのクラスを使用するには、import文を使います。

chat プロンプト 
## 機能
Javaのインポート文の使用例

## 実装要件
- 特定のクラス（ArrayList）を個別にインポート
- ワイルドカードを使ってパッケージ全体をインポート
- staticインポートで定数やメソッドを直接使用可能にする

## 使用するJava構文
- import文（単一クラス指定）
- import文（ワイルドカード *）
- import static文
インポート文 Java 
// 特定のクラスをインポート
import java.util.ArrayList;

// パッケージ内の全クラスをインポート（ワイルドカード）
import java.util.*;

// staticメンバのインポート
import static java.lang.Math.PI;
import static java.lang.Math.*;
menu_book 構文説明

単一クラスのインポート

import パッケージ名.クラス名;

特定のクラスを1つだけインポートします。使用するクラスを明示できるため、コードの可読性が高まります。

ワイルドカードインポート

import パッケージ名.*;

パッケージ内の全クラスをインポートします。ただし、サブパッケージは含まれません。

staticインポート

import static パッケージ名.クラス名.メンバ名;

クラスのstaticメンバ（フィールドやメソッド）を直接使用できるようにします。Math.PIの代わりにPIと書けるようになります。

lightbulb java.langパッケージ

java.langパッケージのクラス（String、Systemなど）は自動的にインポートされるため、明示的なimport文は不要です。

4.6 標準APIのパッケージ

Javaの標準ライブラリには多くのパッケージが含まれています：

パッケージ 説明
java.lang 基本クラス（String、Math、Systemなど）
java.util ユーティリティクラス（List、Map、Dateなど）
java.io 入出力処理（File、InputStreamなど）
java.nio 新しい入出力API（Path、Filesなど）
java.sql データベース接続（Connection、Statementなど）
java.time 日時API（LocalDate、LocalTimeなど）

1章 練習問題

Q1

Javaプログラムの実行手順として正しいものを選んでください。

  1. 実行 → コンパイル → ソースファイル作成
  2. ソースファイル作成 → コンパイル → 実行
  3. コンパイル → ソースファイル作成 → 実行
  4. ソースファイル作成 → 実行 → コンパイル
Q2

以下のコードを実行するためのコマンドとして正しいものはどれですか？

package com.example;
public class Sample { }
  1. java Sample
  2. java Sample.class
  3. java com.example.Sample
  4. java com/example/Sample
Q3

JDK、JRE、JVMの関係として正しいものを選んでください。

  1. JVM ⊃ JRE ⊃ JDK
  2. JDK ⊃ JRE ⊃ JVM
  3. JRE ⊃ JDK ⊃ JVM
  4. すべて独立した関係
Answer

1章 解説・解答

Q1の解答: B

Javaプログラムは「ソースファイル作成 → コンパイル → 実行」の順序で実行されます。ソースファイル（.java）をjavacでコンパイルしてクラスファイル（.class）を生成し、javaコマンドで実行します。

Q2の解答: C

パッケージ化されたクラスを実行するには、完全修飾クラス名（パッケージ名を含むクラス名）を指定します。パス区切りは/ではなく.を使用します。

Q3の解答: B

JDKはJREを含み、JREはJVMを含んでいます。JDK ⊃ JRE ⊃ JVMの関係です。JDKは開発ツール、JREは実行環境、JVMは仮想マシンです。
